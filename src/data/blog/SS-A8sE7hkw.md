## *소유권과 오픈 모델: 구글 DeepMind의 Gemma 4 발표 및 활용 전략* 핵심 요약

- 이 영상은 구글 DeepMind의 Gus Martins와 Ian Ballantyne가 **최신 오픈모델 시리즈 Gemma 4의 특징, 소유권(ownership), 활용사례 및 배포/커스터마이징 가능성**에 대해 상세히 설명함
- **Gemini와 Gemma** 두 모델 계열의 차이점을 통해, 구글이 폐쇄형(호스팅 기반)과 오픈형(로컬/맞춤) 모델을 모두 제공해야 하는 이유를 밝힘
- **Gemma 4는 네 가지 사이즈(2B, 4B, 26B, 31B)**로 제공되며, 2B/4B 모델은 **모바일·엣지 장치에서 구동 가능**, 26B/31B는 **데스크탑/서버에서 구동 가능**
- **효율적인 파라미터 관리 방식(2B/4B지만 성능은 5B급 등)**으로, **경쟁 모델 대비 적은 메모리 자원**에도 뛰어난 “지능 대비 크기(intelligence per parameter)”를 달성
- **ELO 스코어(LeMarina) 기준 오픈모델 상위 4/7위**에 올랐으며, 경쟁 모델 대비 파라미터 규모는 2~20배 적으면서도 높은 성능 유지
- Gemma 31B는 일반 데스크탑 단일 GPU에서도 실행 가능하며, **경쟁사의 동급 모델들이 200GB 메모리를 요구하는 데 비해 월등히 경제적**
- **Gemma 4부터 라이선스 정책이 패스 2.0(PaS 2.0)으로 변경**되어, 법무팀 검토 등 도입 절차가 대폭 간소화되고 자주권/커스터마이징 요건에 적합
- 우크라이나·불가리아·브라질 등 **국가 맞춤형 LLM 파인튜닝 사례**가 존재하며, 멀티링구얼 지원도 탁월(LeMarina TOP 2~3 수준)
- **모바일 현장/데스크탑/엔터프라이즈/의료 등 실제 데모와 활용 예시**가 제시되어, 실질적 도입 구상에 활용 가능
- 영상은 **실행 환경/하드웨어, 에너지·토큰 비용, 커스터마이징 범위, 확장 전략** 등 구체적 고려사항까지 폭넓게 다룸

## 세부 요약 - 주제별 정리

### Gemma 4 공개로 구글은 오픈과 폐쇄형 모델을 모두 제공하여 사용자 선택권을 확대함

- 발표자는 Gus Martins(구글 DeepMind, Gemma 담당)와 Ian Ballantyne임
- 2024년 5월 마지막 주에 **새로운 Gemma 4 모델군**을 공식 출시했다는 사실을 알림
- 구글은 **Gemini(호스팅형, API 이용 가능, 최고 성능 멀티모달)**와 **Gemma(로컬 소유권 중심, 직접 배포/커스터마이즈 가능)** 두 계열을 병행
- Gemini는 가장 강력한 모델이지만, API를 통해서만 접근 가능하며 데이터를 외부에 노출해야 함
- Gemma는 사용자가 자신의 하드웨어에서 독립적으로 구동 및 맞춤형 변경이 가능하도록 설계됨
- 보안, 데이터 주권, 커스터마이징 등 개별/기관·국가적 요건에 대응하기 위해 **두 모델이 보완적으로 제공됨**을 강조

### Gemma 4 시리즈는 다양한 크기와 파라미터 효율성을 바탕으로 모바일부터 서버까지 폭넓은 적용이 가능함

- Gemma 4는 **E2B(2B), E4B(4B), 26B(MoE), 31B(Dense)** 네 가지 크기로 공개
    - E2B, E4B는 IoT 및 모바일 디바이스 실행에 최적화
    - E 시리즈(E2B/E4B)는 “Effective”의 의미로, 실제 GPU 메모리는 2B/4B만 필요하지만 논리적 모델 파라미터는 약 5B/10B 수준
    - 추가 파라미터 일부는 외부 메모리 또는 매핑 토큰으로 이관해 효율 확보
- **E2B/E4B 모델**은 텍스트, 비전, 오디오 인풋을 처리하며, 텍스트 출력만 지원
- 추론, 코딩, 함수 호출 등 **다양한 지능적 작업을 휴대폰에서 직접 실행 가능**
- **26B(Mixture of Experts)**: 여러 개 4B 수준 모델을 앙상블하여 26B 성능 확보, 실질 메모리 사용은 4B급에 가깝게 효율화
- **31B(Dense)**: Gemma 시리즈 내 최고 성능, 언어/코딩/멀티링구얼 기능 모두 탁월하며, 일반 데스크탑 등에서도 단일 GPU로 실행 가능

### 파라미터 효율성에서 경쟁 모델 대비 ‘지능 대비 크기’ 우위를 입증함

- LM Marina의 ELO 스코어 기준, **Gemma 26B와 31B 모델은 각각 오픈모델 상위 4위와 7위 기록**
- 상위 20~30개 모델과 비교 시, 파라미터 규모는 2~20배 작지만 유사한 또는 더 뛰어난 응답 품질 보여줌
- **31B 모델**은 실제 코드 작성, 다중 언어 처리, 에이전트 작업 등 거의 모든 지능적 태스크를 수행 가능
- 경쟁 모델들은 유사 수준 성능을 위해 200GB 메모리(4~5 GPU) 필요하지만, Gemma 31B는 **1 GPU로도 충분**
- 효율성과 비용 경쟁력(경제적 인프라 구동) 강조

### 다양한 벤치마크 결과와 실제 사용자 경험에서 Gemma 4의 품질이 입증됨

- LM Marina, LeMarina 등의 **학술 및 사용성 벤치마크에서 강력한 성능** 확인
- 실제 사용자가 질의할 때 체감하는 답변 품질(User Query Response) 역시 중요한 평가 기준으로 간주
- Gemma 26B, 31B는 클라우드 배포시 서버 엔드포인트로 이상적이며, 데스크탑, 퍼블릭 클라우드, 오프라인 환경 등 다양한 시나리오에서 활용 가능

### 라이선스 정책 개선(PaS 2.0 적용)으로 공공·자주권 기관 및 기업의 채택 장벽을 낮춤

- Gemma 4 이전은 카스타마이즈형 Gemma License(상업 친화적)이었으나, 법률 검토 등 복잡한 절차가 필요해 도입에 어려움 있었음
- 실제로 변호사, 공공기관 등이 **장기간(최장 18개월) 내부 절차**를 거쳐야 하는 경우가 많았음
- Gemma 4부터는 **PaS 2.0 라이선스**로 전환, 명확한 사용 권한 및 커스터마이징/배포의 자유를 보장
- 우크라이나(국가 서비스 파트로 Gemma 활용), 불가리아(불가리아어 전용 파인튜닝 버전), 브라질(브라질 포르투갈어 버전) 등 실제 **국가별 LLM 도입 사례** 제시

### 멀티링구얼 성능도 대폭 향상되어, 추가 파인튜닝 없이 각국 언어에 높은 성능을 보임

- Gemma 시리즈는 영어 외 타 언어(불가리아어, 포르투갈어 등)로 **파인튜닝된 버전**이 실제 서비스 적용 사례 있음
- 최근 모델은 기본 멀티언어 지원이 높아 별도 추가 파인튜닝 시 약 1% 미만 증가라는 한계점도 존재
- 그럼에도 LM Marina(LeMarina) 다언어 벤치마크에서 **상위 2~3위**를 차지, 31B 모델임에도 높은 언어 범용성 보유

### 오픈모델 소유권은 에이전트·토큰/에너지 비용 관리 측면에서 실질적 이점을 제공함

- 폐쇄형 모델의 API 사용은 토큰 비용이 빠르게 증가하지만, **직접 소유 가능한 오픈모델은 하드웨어 활용 극대화 및 반복 작업 비용 절감** 효과
- OpenRouter 데이터: 프로그래밍 등 “고토큰 생성” 작업에서 오픈모델 소유 및 사용이 뚜렷한 강점
- 개인/팀 보유 하드웨어, 클라우드 임대, 오프라인 환경에 맞춰 자체 배포가 가능하며, 작업별 요구 대기 시간·비용에 따라 유연하게 조정할 수 있음

### 모바일, 엣지, 데스크탑, 엔터프라이즈 등 하드웨어별 실제 데모와 사용 사례가 소개됨

- Coremax팀의 **모바일 에이전트 데모** 사례: 2B/4B 모델이 폰에서 구동되며,
    - 이미지/음성 인식, 실시간 대화, 기기 내 앱 호출(캘린더, 지도 등)까지 지원
    - 유저 정의 함수 호출 및 에이전트 스킬 조합 가능
- **Google AI Gallery** 앱(iOS/Android)로 누구나 실시간 플레이그라운드에서 체험 가능
- 데스크탑/서버 환경에서는 **26B/31B 모델**도 일반 PC 또는 단일 GPU에서 구동 가능한 데모 시연
    - 예: M4 맥북에서 26B 구동, 26GB 메모리 활용, LM Studio 오픈소스 플랫폼 시연
    - 하나의 입력을 여러 언어로 동시 번역, 각 Sub-agent가 병렬 처리 및 결과 통합
- 오프라인 분석, 에이전트 기반 문서 처리, 다국어 웹페이지 생성 등 실제적 태스크 데모 제공

### 실제 적용 시 하드웨어·운영비·에너지 비용 등 종합적 고려가 필요함

- GPU/MPU 등 하드웨어 활용률과 과업의 실시간성(즉각응답, 야간배치 등)에 따라 배치·실행 정책 변화
- 직접 GPU 구입, 임대, 관리시 초기 및 유지비용까지 반드시 감안해야 함
- 모바일 오프로딩 지원시 기기별 엑셀러레이터, RAM 용량 등 기종별 성능 분석 필요
- 오프라인/개인 데이터 보호(사용자 기기에서 유출 없이 구동)라는 추가적 이점도 제공

### 엔터프라이즈 및 특수목적(의료 등)에서 온프레미스 커스터마이징과 경제적 확장성을 실질적으로 확보함

- 31B 이상의 모델도 단일 H100, A100(L4 포함) 등에서 유연하게 스케일링 가능
- 예산, 인프라 환경에 따라 스타트업·조직·기업 규묘로 맞춤형 배포 가능
- 예: **MediGemma** (Gemma에서 파인튜닝한 의료특화 LLM) 출시, 병원 전산망에 자체적으로 1~2 GPU로 배포 및 운영 가능

### 오픈모델 실전 적용을 위한 빠른 실험·기존 워크플로우 접목, 벤치마킹·튜닝 절차가 제안됨

- LM Studio, Olama 등 오픈AI 호환 인터페이스로 빠르게 Gemma를 실험·업무에 투입 가능
- 기존 워크플로우에 모델만 교체하거나 추가 연결하여 실질적 태스크 적합성, 벤치마크, 한계점 확인 권장
- 직접 GPU 호스팅시 가동/중단, 유지관리 등 실질적 운영 이슈 고려 필요
- 엔터프라이즈 확장 시 하드웨어/운영비 평가 및 용도별 적절한 리소스 투입 설계가 요구됨
- 데이터 프라이버시, 오프라인 처리 등 맞춤 도입 가능성이 강조됨

### Gemma 4는 벤치마크 성능과 활용 유연성, 소유권까지 겸비하면서 오픈모델 시대의 주역임을 알림

- 영상은 “무엇이 가능한가?” 실험 정신을 유도하며, 실제 사용·피드백·튜닝을 적극 장려
- 사용자별, 과업별 실질적 이득과 한계를 직접 확인하고, 필요에 따라 더욱 발전 가능한 기반 기술로서 Gemma를 소개함
