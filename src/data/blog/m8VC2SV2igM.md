## *Vercel Eve: 프로덕션 AI 에이전트 구축 방식을 완전히 바꾸는 오픈소스 프레임워크* 핵심 요약

- 영상은 Vercel에서 최근 공개한 오픈소스 AI 에이전트 프레임워크인 Eve를 소개하며, “파일 시스템 우선(file system first)” 아키텍처가 기존 AI 에이전트 설계 방식을 혁신적으로 바꾸고 있음을 강조함
- Eve는 단순한 개인/세컨드 브레인용이 아닌, 수천~수백만 사용자까지 스케일 가능한 프로덕션 환경용 AI 에이전트 구축을 위해 설계됨
- 에이전트 전체가 하나의 “폴더”로 구성되고, 각 기능(프롬프트, 모델, 스킬, 툴, 채널, 연결, 서브에이전트, 스케줄 등)이 개별 서브폴더에 정의되어 높은 조합성과 관리 편의성을 제공함
- Eve의 핵심은 “컴파일 단계”로, 사용자가 직접 모듈 연결 작업 없이 폴더 내 구성 요소를 자동으로 탐색·통합하여 단일 매니페스트를 생성함
- Vercel 인프라와 통합되어 내결함성(durable session), 격리된 샌드박스 실행, 휴먼 인더 루프(Human-in-the-loop), 자동 배포 평가(Evals), 대규모 확장(수천~수백만 사용자 동시 지원) 등 프로덕션 등급 기능을 기본 지원함
- Claude Code, Cursor 등 코딩 에이전트와 Vercel 플러그인 연계를 통해 Eve 기반 에이전트 빌드 및 배포가 명령어 몇 개로 간단하게 구현됨
- 에이전트 구성은 TypeScript와 마크다운 파일만으로 빠르게 가능하며, 새로운 기능(스킬, 서브에이전트, 툴 등) 추가 시 별도의 임포트 연결 과정 없이 폴더에 파일만 추가하면 자동 인식·적용됨
- 슬랙(Slack)과의 연동과 실시간 데모 시연을 통해, Eve 기반 에이전트의 대화 지속(단기 기억), 병렬 대화, 인간 승인(결정 버튼 제공), 스킬·툴 동작을 입증함
- 사용자 프롬프트만으로 Eve 에이전트 뼈대를 자동 생성하는 등 개발 경험이 혁신적으로 단순화됨
- 영상은 Eve가 파일 시스템 기반 에이전트의 표준을 정립하는 계기가 될 것이며, OKF 등 기존 오픈소스 표준과 결합 가능성을 시사함

## 세부 요약 - 주제별 정리

### Vercel Eve는 파일 시스템 아키텍처로 개인용이 아닌 프로덕션 환경까지 지원함을 강조함

- Vercel이 발표한 오픈소스 AI 에이전트 프레임워크 ‘Eve’를 첫 공개
- Eve는 파일 시스템 기반 구조인데, 기존에는 주로 개인 에이전트·세컨드 브레인 용도로 쓰였으나 Eve는 프로덕션 환경까지 확장 가능
- “에이전트 = 하나의 폴더”라는 단순한 개발 구조를 지향
- 파일 시스템 기반 구조의 장점(조합성, 관리 편의성)에 더해, 실 서비스 배포에 필수적인 신뢰성까지 확보

### 폴더 중심의 구조와 컴포넌트 관리는 조합성과 간결성을 혁신적으로 높인다

- Eve에서 에이전트는 상위 폴더이고, 내부에 각각의 기능(시스템 프롬프트, 에이전트 정의, 스킬, 툴, 샌드박스, 채널, MCP 서버 연결, 서브에이전트, 스케줄 등)이 개별 폴더로 구성
- 각 기능 요소별로 독립적으로 파일 관리가 가능하여 협업 및 기능 추가·수정이 용이
- 이전의 개인 에이전트들은 이런 아키텍처를 가져도 실 서비스 배포에는 한계가 있었으나, Eve는 이를 극복함

### Eve의 ‘컴파일 단계’가 개발자의 연결·정의 부담을 완전히 해소함

- 사용자는 각 기능별 파일·폴더만 추가하면 연결 작업(임포트, 레퍼런스 등) 필요 없음
- 컴파일 시 Eve가 전체 폴더 구조를 파싱하여 에이전트의 모든 기능적 연결을 자동으로 통합
- 하나의 매니페스트에서 모든 스킬, 서버, 채널, 툴 등이 연결된 “컴파일된 에이전트”를 생성
- Claude Code의 스킬 정의 방식과 유사하나, Eve는 여기에 프로덕션 등급의 배포 및 신뢰성까지 지원

### Vercel 인프라와 통합되어 프로덕션 등급 필수 기능을 기본 제공함

- 세션이 ‘내결함성(durable, checkpointed workflow)’으로 관리되어 에이전트가 중단·재배포 시에도 중간 상태 복원 가능
- 격리 샌드박스(sandbox) 환경에서 코드 실행 가능하여, 에이전트의 코드 작성 및 실행이 보안적으로 안전
- Human-in-the-loop 지원: 위험 단계에서 작업을 중단시키고 인간의 승인을 받도록 흐름 설계
- Evals 폴더를 통해 사전 동작 평가(테스트)가 가능, 에이전트 업데이트 전 자동 검증 절차 구현
- Vercel 클라우드가 전체 인프라 호스팅을 담당, 수천~수백만 명의 사용자를 대상으로 대규모 확장 지원

### 표준화의 필요성과 Eve가 파일 시스템 기반 에이전트 아키텍처의 새 표준이 될 잠재력을 갖고 있음을 설명함

- 기존 여러 조직들이 내부적으로 비슷한 구조의 에이전트 시스템을 각각 구현해왔으나, 통일된 표준은 부재
- Eve는 가장 간결한 방식으로 구조를 표준화하고, 내부 로직(배관 작업)을 대변하여 개발 효율 상승
- MCP, A2A, OKF 등 기존 AI 표준(삼문자 약어)과의 연동·조합 용이
- 오픈소스 기반이기 때문에 업계 표준화 및 상호운용성 확산에 기여 가능
- OKF는 에이전트에 연결되는 지식 베이스 표준 역할, Eve 구조는 에이전트 본체 표준 역할

### Eve는 에이전트의 ‘핵심 기능 및 확장 모듈’만을 스케일하며, 거대한 마크다운 기반 지식베이스 방식과는 구별됨

- Eve의 스케일링 대상은 ‘핵심 프리미티브’(에이전트, 스킬, MCP 서버 등), 대규모 마크다운 기반 LLM 위키(지식베이스)는 별도
- 지식베이스 자체를 자동으로 무제한 확장하는 방식은 아니며, 에이전트 구조와 연결성에 집중
- Eve + OKF 조합으로 “에이전트와 지식베이스”를 각 표준화 구조로 결합할 수 있음

### Vercel 플러그인(Claude Code, Cursor 등)으로 구축과 배포 과정이 명령어 수준으로 단순해짐

- Eve 기반 에이전트 개발을 위해 Vercel 공식 플러그인 별도 제공
- Claude Code, Cursor 등 최신 코딩 에이전트에 플러그인만 설치하면 모든 에이전트 구조·빌드·배포가 통합 지원
- 플러그인 설치 및 접속 방법은 영상 설명란 링크 및 공식 가이드 제공
- 플러그인 덕분에 고급 구조 이해 없이도 누구나 손쉽게 프로덕션 등급 에이전트 생성 가능

### 실제 시연(데모)에서 에이전트 구조, 스킬 추가, 실시간 대화, 툴 활용 방식이 구체적으로 입증됨

- ‘Eve Analyst’라는 예시 에이전트 클론(Claude Code + Vercel Eve 플러그인 기반)으로 상세 구조 시연
- 기본 구조: agent.ts(TypeScript) + 다양한 마크다운 및 TS 파일(스킬, 서브에이전트, 채널, 지시문 등)
- 모델은 Anthropic API, 환경변수로 키만 등록하면 사용 시작 가능
- 예시: revenue 관련 스킬(질의응답 개선), investigator 서브에이전트(토큰 소모 큰 질문 분산처리) 등 파일만 해당 폴더에 추가하면 자동 반영
- 툴 추가: 단순 TS 함수 파일을 tools 폴더에 넣으면 데이터베이스 질의 등 기능 확장, Zod로 타입 안정성 부여
- 모든 조립·연결 작업을 자동화하여 수작업 최소화

### 배포 및 실시간 동작, Slack 연동에서 Eve의 ‘생산성’과 ‘신뢰성’이 드러남

- 모든 기능 완성 후 “Deploy this Eve agent” 명령만으로 빌드→테스트→프로덕션 배포 자동화 (MCP 서버 활용)
- 에이전트 기본 생성도 “Scaffold a new Eve agent called [이름]” 프롬프트만으로 가능, 세부기능(서브에이전트·스킬 등)도 상세 설명만 입력시 자동 생성
- Slack 연동 데모: /@eveanalyst로 직접 대화, 단기기억(이전 메시지 회상) 및 병렬 대화(여러 유저 동시 채팅) 정상 동작 확인
- 휴먼 인더 루프: 위험한 작업(대량 SQL select 등) 시 승인·거부 버튼 자동 생성, Slack 내에서 클릭만으로 승인 처리

### Eve의 휴먼 인더 루프, 자동화-신뢰성 균형 및 실제 운용 팁이 상세히 안내됨

- 휴먼 인더 루프(HITL)는 민감 작업 발생 시 Slack 메시지 큐에 “승인/거부” 버튼 노출
- 관리자는 실시간으로 승인하여 작업 진행/차단 결정 가능, 에이전트의 실제 프로덕션 활용에 적합
- 너무 강한 통제는 에이전트의 효용성을 저하시켜, 신뢰성과 자동화 균형을 Eve가 쉽게 실현함

### 에이전트 제작 경험이 단순 파일 조작 수준으로 혁신적으로 단순화됨이 강조됨

- 과거에는 에이전트 구조 설계·배포가 복잡했지만, Eve+플러그인 조합으로 “마크다운/TS 파일 생성→폴더에 추가→명령 실행”만으로 구현 가능
- 더 이상 복잡한 임포트·레퍼런스·서비스 확장·연결 작업 필요 없음
- 영상 제작자(콜 메딘)는 실제 Vercel 측과 협업해 내용의 정확성 보장
- 영상 마지막에 Eve 공식 레포와 추가 자료(가이드, 플러그인, 분석 에이전트 등) 안내
