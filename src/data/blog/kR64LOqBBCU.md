## *에이전트크래프트: 오케스트레이션(Orchestration)에 게임의 지혜를 담다 — 이도 살로몬* 핵심 요약

- 이도 살로몬(Ido Salomon)은 AgentCraft, MC I, MC apps의 개발자로, 에이전트 기반 생산성 향상을 목표로 하는 오케스트레이터 AgentCraft를 소개함
- 단일 에이전트의 성과에 비해 다수 에이전트(10~100개)의 확장 적용은 인간의 오케스트레이션(관리) 능력 한계로 실질적으로 어려움을 겪는다고 진단
- AgentCraft는 게임 플레이(특히 RTS 게임)에서 영감을 얻어 다수 에이전트를 통합 관리하는 새로운 작업 방식을 제안
- 실시간 시각화, 파일/디렉토리 기반 작업 추적, 히트맵 등 풍부한 모니터링 기능을 통해 에이전트 행동의 가시성과 계통을 보장
- 에이전트의 자율성을 적극 활용해 사전 정의된 과제(퀘스트, 캠페인, 반복 스케줄링) 수행, 작업 분해/계획/검토의 자동화·분산화 추진
- 사람간·에이전트간·사람과 에이전트간의 즉각적·간접적 협업(워크스페이스, 대화, 작업 분담)을 지향하여 오케스트레이션 효율 극대화
- 리뷰 번들, 시각적 증거(스크린샷, 동영상), 변경 이력 등 실제 코드 리뷰 편의성 대폭 향상
- 개발자는 모든 작업을 직접 수행하기보다는 계획·검토 중심으로 역할을 이동, 더 넓은 작업량을 소화할 수 있게 됨
- 익숙지 않은 미래형 업무 방식처럼 보이지만, 실은 게임 등 익숙한 환경에서 체득했던 역량임을 강조하며 적응의 중요성 역설
- AgentCraft는 무료·실험적 프로젝트로, 사용자 피드백과 커뮤니티 참여(Discord 등)를 통해 예리하게 발전시킬 계획임

## 세부 요약 - 주제별 정리

### AgentCraft는 다수 에이전트 오케스트레이션의 실질적 병목을 인간 관리자의 한계에서 발견함

- 다수의 AI 에이전트가 각자 뛰어난 역량을 가지고 있으나, 병렬적으로 많이 사용하려 해도 실질적 오케스트레이션이 어렵다는 문제가 있음
- 에이전트를 여러 개 띄우는 행위 자체는 기술적으로 어렵지 않으나, 이를 통합 운영하는 관리자의 인지적, 조직적 한계가 업무 효율의 병목이 됨
- 여러 명의 충동적인(‘reckless’) 직원을 통제해야 하는 상황을 개발자의 현실에 비유하며, 실제 회사에서 하지 않는 일을 새롭게 학습해야 함을 강조
- 이런 오케스트레이션 기술은 전혀 새로운 것이 아니라 ‘예상치 못한 곳’, 즉 게임(특히 실시간 전략 게임, RTS) 속 유닛 관리에서 이미 익숙함을 지적

### 게임에서 영감을 얻은 AgentCraft의 설계는 생산성 협업 툴로 발전함

- AgentCraft는 게임에서 유닛을 관리하듯 에이전트들을 다루는 인터페이스와 구조를 채택
- 실제 작동 환경은 에이전트가 시각적으로 구현되는 라이브 세션(Active session) 형태
- 코드 편집용 커서, 클라우드 코드, CodeX, OpenClaw 등 다양한 플러그형 에이전트 연결 가능
- 사용자는 AgentCraft 내에서 에이전트를 직접 생성(‘spawn’) 또는 외부에서 감지 가능
- 텍스트, 음성, 이미지 등 다양한 방식으로 에이전트에 명령을 전달하여 특정 기능 개발 등 작업을 할당 가능
- 건물(Building) 단위의 기능 구성을 도입, 각 건물이 플러그인·스킬 관리, 통합 터미널, Git 등 다양한 End-to-End 워크플로우 지원

### 실시간 가시성과 워크플로우 시각화로 작업 투명성을 대폭 강화함

- 모든 에이전트를 한 눈에 파악할 수 있는 사이드 패널과 상태 요약, 미션 진행상황 제공
- 파일 시스템 전체가 지도(map) 형태로 시각화되며, 각 디렉토리와 파일(방/Room)이 시각적으로 구분되어 나타남
- 사용자는 실제로 어떤 에이전트가 어떤 파일·작업을 하고 있는지 실시간으로 추적 가능
- 모든 작업에 대해 변경 내역(change list), 작업 연대기(Full lineage), 작업 주체(어떤 에이전트가 언제 작업)를 확인할 수 있음
- 히트맵 기능을 통해 작업 충돌(collisions)을 사전에 시각화·예방 가능

### 에이전트간 임무 분해와 자동 플래닝, 긴밀한 반응성(reactivity)을 구현함

- 에이전트가 유관 과제를 탐지(미션 찾기) 및 할당받아, 반복적인 리팩터링·테스트 등 개발자가 원치 않는 일도 수행 가능
- 개발자는 플랜 승인, 질문 응답 등 필요한 부분에 빠르게 개입(react)하며, 실시간으로 에이전트간 전환도 게임식으로 지원
- 단순 관찰·빠른 반응만으로는 충분하지 않다는 한계도 언급
- 작업 ‘캠페인(Campaign)’ 기능을 통해 대규모 작업을 하나의 컨테이너에 할당해 완전 자동화된 분해·계획·수행 과정 실현
- 여기서부터 개발자는 업무의 ‘기획’과 ‘검토’에 집중, 작업 실행 관리는 캠페인 오케스트레이터가 전담함

### 반복 실행, 크론 스케줄 등 자동화로 확장성을 높임

- 스케줄(크론잡) 기능을 이용, 예시로 매일 트위터에서 아이디어를 수집·자동 프로젝트 생성이 가능함을 시연
- 다양한 풀 리퀘스트(PRs)가 자동 생성, 리뷰 번들 기능으로 전체 업무 내역·변경 내역·작업 사유를 한 화면에서 검토 가능
- 스크린샷, 동영상 등 시각적 증거 제시로 코드 리뷰 효율과 신뢰성 강화

### 계획(플래닝) 기능에서 선택·검토 중심의 업무로 전환 유도

- 캠페인별 다양한 플랜이 수립되면, 여러 번 시도한 후 가장 적합한 결과만 선택적으로 채택하는 방식 추천
- 플래닝에 드는 시간 부담이 줄어들며, 리뷰 중심의 멀티태스킹이 가능해짐

### 인간 협업과 에이전트 조율을 융합한 새로운 워크스페이스 제공

- 인간 작업자간(예: 개발자-디자이너) 공동 Workspace 구성·작업 이력 공유로 협업 강화
- 각자 컴퓨터의 Agent가 서로 작업 현황을 실시간으로 파악·공유, 역할 분배·계속 진행 가능
- 예) 디자이너가 새 페이지를 설계하면, 개발자가 이어 받아 작업 또는 검토 가능
- 에이전트 공동 소유·활용(our agents) 등 유연한 분업 체계 구현

### 대화형 협업 시스템으로 인간-에이전트 및 인간-인간 소통을 촉진함

- 양방향 채팅 시스템 내장, 인간↔에이전트 및 인간↔인간 메시징 가능
- 에이전트도 작업 개시와 중단을 메시지로 공지, 변경 파일 공유, 협업상의 충돌 예방 가능
- 소프트 협업(Soft collaboration) 구조가 작업 충돌 감지·알림 등 부드러운 통합 워크플로우 실현

### 기존에 익숙했던 게임 내 역량을 업무에 접목할 수 있음을 강조함

- 완전히 새로운 오케스트레이션 스킬이 아니라 과거 게임에서 사용했던 대량 유닛관리 스킬임을 재발견
- 인간과 에이전트 사이의 시야 확장, 자율성 제고, 효율적 협업이 목표임을 명확히 밝힘
- 신기술 두려움보다는 익숙한 스킬의 전환·응용이라는 메시지로 사용자 동기 부여

### 에이전트크래프트는 실험적·커뮤니티 기반 오케스트레이션 혁신을 지향함

- AgentCraft는 누구나 무료로 사용할 수 있는 프로젝트(open beta)임을 안내
- 발전 방향은 실제 사용자 경험·피드백에 따라 결정하겠다고 밝힘
- 피드백 통로로 공식 웹사이트 및 Discord 커뮤니티를 운영, 집단적 혁신을 호소
- 발전 가능성, 실험적 성격, 커뮤니티 참여의 필요성을 끝으로 강조하며 발표를 마침
