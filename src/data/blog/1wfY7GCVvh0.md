## 구글 에이전트 CLI: CLI와 스킬 결합으로 AI 에이전트 개발을 획기적으로 단순화한 도구 핵심 요약

- 구글의 오픈소스 에이전트 CLI를 활용하면 누구나 AI 에이전트를 쉽고 빠르게 개발하고, 실제 환경에 배포할 수 있음
- 에이전트 CLI와 ‘스킬’의 조합 덕분에 코딩 어시스턴트가 아이디어 단계부터 신뢰성 있는 프로덕션 배포까지 대부분의 과정을 자동화함
- 과거 LangChain 등 프레임워크로 손수 코딩하던 것과 달리, 자연어 지시를 통해 프로젝트 스캐폴딩, 평가, 배포 등이 가능해짐
- “스킬”은 구글 ADK(Agent Development Kit) 공식 문서의 내용을 코드 어시스턴트에 내장하여, 따로 문서를 볼 필요 없이 일련의 작업을 수행하게 해줌
- 실제 예시로 CSV와 질문을 입력하면 파이썬 코드로 답을 구하는 ‘Ask Your Data’ 에이전트를 몇 분 만에 구축·배포함
- 코드 실행은 ADK의 내장 샌드박스 환경에 의해 안전하게 처리되며, 이 점이 특히 프로덕션에서 유용함
- CLI와 스킬을 통한 에이전트 구축 과정은 Streamlit UI를 곁들여 간단히 테스트 및 평가함
- 기존의 코드 에이전트 SDK로 ‘세컨드 브레인’ 같은 개별 워크플로우도 만들 수 있으나, 토큰 효율성과 속도 측면에서 프로덕션용에는 ADK+CLI가 더 적합함을 강조
- 에이전트 배포 후, GCP에서 바로 접근 가능한 웹 UI, audit 로그, 트레이스, 메모리, 코드 실행 등 생산환경의 모든 필수 기능 제공
- 영상 전 과정에서 사용자는 ‘한 줄의 커맨드도 직접 입력하지 않고’ 음성-자연어 지시만으로 아이디어에서 프로덕션까지 전 과정을 실연함

## 세부 요약 - 주제별 정리

### AI 에이전트 개발은 예전에 비해 월등히 간소화되어 ‘자연어’로 전체 과정이 가능해짐

- 예전에는 LangChain 등 프레임워크로 소스코드를 일일이 작성해 에이전트를 만들었음
- 당시엔 AI 코딩 어시스턴트도 없어, 공식 문서 참고와 수동 수정이 필수였음
- 구글 ADK, Pydantic AI 등 다양한 프레임워크가 있지만, 문서를 따로 참고해야 했고 버그도 자주 발생
- 지금은 CLI+스킬 덕분에 설치·구축·배포 등 여러 과정이 하나의 자연어 명령과 워크플로우로 완결됨
- 개발자는 단일 명령(명령 프롬프트) 혹은 코딩 어시스턴트에 지시만 해도 프로덕션까지 수행 가능함

### 구글 에이전트 CLI는 ADK와의 결합으로 프로덕션 환경까지 연결되는 자동화 파이프라인을 제공함

- Agent CLI는 오픈소스로 공개되어 누구나 사용할 수 있음
- 다양한 CLI 명령(프로젝트 스캐폴딩, 평가, 단일 프롬프트 테스트, eval 데이터 기반 프롬프트 최적화 등) 제공
- CLI의 각 기능은 대응되는 스킬과 맞물려 작동하며, ‘스킬’이 실제로 각 명령의 활용 방법을 코딩 에이전트에 전달
- 스킬 속에는 ADK 공식 문서 내용이 내장되어 있어, 별도로 문서를 찾아보지 않아도 됨
- ADK 스킬은 코드 작성, 평가, 배포, 운영 등 모든 단계에 존재하여 자동화를 실현함

### ‘코딩 에이전트’가 모든 명령을 자동으로 실행하며, 스킬이 명령 실행법을 체계적으로 안내함

- CLI는 코딩 에이전트가 실행할 수 있는 ‘능력’ 집합이며, 스킬은 각 능력을 ‘어떻게’ 사용할지 알려주는 매뉴얼
- 코딩 어시스턴트(예: Claude Code)는 사용자의 자연어 지시만으로 프로젝트를 셋업하고 명령 실행을 대행함
- 스킬이 도입되면서 문서 전달, context 주입 등 번거로운 작업이 불필요해짐
- 코딩 에이전트가 에이전트 제작~배포까지 ‘엔드 투 엔드’로 리드할 수 있음
- 실습 예시에서는 “Google Agent CLI를 설치해달라”는 음성 명령만으로 글로벌 환경에 도구가 설치됨

### 실전 예시: CSV 데이터 기반 질의응답 에이전트를 구축·배포하는 과정을 단계별로 시연함

- 스킬이 로드된 환경에서, “CSV와 질문을 입력받아 파이썬 코드로 처리하는 에이전트 생성” 요청
- 코드 실행에는 ADK의 내장 샌드박스(secure code execution) 기능을 활용함
- 사용자가 직접 명령어나 CLI 실행 없이, 어시스턴트가 작업을 대행
- Claude Code 등 코딩 에이전트에 스펙을 충분히 알려주면 복잡한 기능 구현도 가능
- 예제의 경우, 사용자가 프롬프트를 붙여 넣고, 에이전트가 실행에 필요한 ADK 코드 익스큐터 종류를 질의
- 내장 샌드박스를 선택해 안전한 코드 실행 환경을 확보함

### 생산 환경(Production) 배포와 관련한 전 과정이 CLI+스킬 만으로 자연스레 처리 가능함

- 에이전트가 CLI의 스캐폴딩, 코드 작성, 테스트 등 일련의 단계를 주도
- Streamlit으로 제작된 간단한 UI를 통해 구축한 에이전트를 로컬에서 우선 테스트
- “주문별 중간값 매출은 얼마이며, 평균보다 가장 큰 이상값이 포함된 행은 어디인가?” 등의 질문에 파이썬 코드로 직접 답을 생성
- 반환 결과에 실제로 작성된 코드를 함께 시각화
- 이러한 과정을 모두 자동화하여 신속하게 반복 테스트가 가능함

### 에이전트 성능 평가(Evaluation), 프롬프트 개선 및 배포까지 ‘한 번에’ 진행할 수 있음

- 자연어로 “ADK 스킬을 활용해 방금 구축한 에이전트 평가” 요청
- 새로운 대화 세션에서 context 없이도 평가 스킬이 코드베이스를 분석, 사전 정의된 테스트 세트를 실행
- 3가지 샘플 테스트 케이스 모두 정상 통과
- 평가 후, 추가적인 성능 개선(시스템 프롬프트/툴 조정 등)과정을 거쳐 배포 준비를 완료할 수 있음

### AI 코딩 어시스턴트 SDK와 구글 ADK 기반 전통적 프레임워크의 용도가 다름을 명확히 설명함

- 코딩 에이전트 SDK(예: Anti-Gravity SDK)는 개인 용도의 ‘세컨드 브레인’ 등 활용에 매우 유용
- 그러나 높은 토큰 소모와 비교적 느린 응답속도는 대규모 서비스(프로덕션) 배포에 비효율적
- 실제 고객 대상 플랫폼에서는 4초 내 첫 응답이 없으면 이탈률이 급증한다는 연구 자료 인용
- 전문가용/대규모 서비스에는 미니멀하고 경량화된 프레임워크(예: Google ADK + CLI) 사용을 권고

### 실제 배포: GCP에 에이전트 인스턴스 자동 생성, 관리 및 모니터링 기능까지 기본 내장됨

- “이 에이전트를 배포해달라”는 자연어만으로 배포 절차 시작
- 구글 클라우드에 자동으로 서비스 인스턴스가 생성되고, 콘솔 플레이그라운드 URL도 자동 제공
- 인증이나 프로젝트 설정이 필요한 경우 어시스턴트가 단계별로 안내
- 배포된 에이전트는 곧바로 웹에서 테스트 가능하며, Streamlit UI와 연동도 가능
- 에이전트별 식별자, 권한 설정, 트레이스, 감사로그, 메모리, 코드샌드박스, 평가 등 운영 필수 기능을 내장
- 외부 플랫폼에서 손쉽게 에이전트를 호출할 수 있는 URL 제공

### 배포 이후 실제 프로덕션 환경(웹)과 로컬 환경 모두에서 신속하게 동작 확인 가능함

- Streamlit UI에서 ‘배포 완료된 에이전트’로 샘플 데이터를 재실행
- “분기별로 가장 성장한 지역과 성장률은?” 같은 질의에 파이썬 코드 및 결과 반환
- 운영 중인 환경에서도 동일한 인터페이스로 동작 확인
- 결과와 코드 텍스트 모두 실시간으로 제공됨

### 전체 프로세스를 통해 기존 대비 AI 에이전트 개발·배포가 혁신적으로 간편해졌음을 실증함

- 사용자는 단 한 줄의 터미널 명령 없이, 자연어 기반 지시만으로 전체 워크플로우를 완성함
- 복잡도가 높아질수록 사전 스펙 정의가 더 많이 필요하지만, 프로세스 자체는 변함없이 단순하게 유지 가능
- 복수 에이전트, RAG 시스템 등도 추가적인 스펙 작업만 선행하면 CLI+스킬 조합이 전체 개발을 안내함
- 마지막으로 영상 설명란에 레포 및 장비 링크도 안내함

### 현 시점(2024년 기준) AI 에이전트 구축과 배포의 난이도와 효율성이 불과 1년 전과 비교해 현저히 개선되었음을 강조하며 마무리함

- 과거 방식과 달리, 코딩 어시스턴트가 전체 프로젝트를 주도·자동화
- 배포 단계까지 논리적이고 신뢰도 높게 진행이 가능해짐
- 영상 제작자는 본 도구들을 직접 실무에 활용 중임을 언급
- 변환 과정 전체를 자연스럽고 직관적으로 시연하며, 더 많은 AI 에이전트/어시스턴트 관련 영상 예고와 함께 종료
