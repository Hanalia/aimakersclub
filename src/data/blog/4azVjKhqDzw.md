## *에이전틱 엔지니어링을 위한 앱스토어 구축하기* 핵심 요약

- 본 라이브 스트림에서는 오픈소스 AI 하니스 빌더인 Archon의 "워크플로우 마켓플레이스" 기능 구축 과정을 다룸
- Archon은 사용자가 자신만의 AI 코딩 프로세스를 워크플로우로 패키징, 병렬 실행 및 재사용할 수 있도록 설계되어 있음
- 워크플로우 마켓플레이스는 사용자들이 빌드한 워크플로우를 다른 이들과 손쉽게 공유하거나 가져올 수 있게 하는 시스템임
- 마켓플레이스에 워크플로우를 등록할 때 자동화된 PR(풀리퀘스트) 리뷰 및 머지 프로세스를 구축: AI가 PR을 자동 검토, 승인, 또는 반려
- 코딩 에이전트 기반 워크플로우 개발의 전형적인 프로세스(탐색–플래닝–인플리멘테이션–검증–코드리뷰–피드백 순환)가 "PIV loop"란 워크플로우로 구현되어 있음
- 실시간 작동 예제로 워크플로우를 빌드하고 PR 제출 및 자동 리뷰 테스트를 진행
- 클라우드 코드, Sonnet, Haiku 등 다양한 LLM 모델을 선택·혼합하여 사용하며, 비용 및 처리 효율을 고려한 세밀한 구성 적용
- PR 머지, 이슈 수정, 모델 교체 등 실제 개발 과정에서 발생하는 문제 해결 및 토큰 사용량, 병렬 실행, 보안 등 현실적인 이슈 논의
- 유저 커뮤니티와의 상호작용(Q&A, 개선 제안, 워크플로우 제출 독려 등)과 실시간 피드백 수렴, 오픈소스 개발 문화 강조
- 전체적으로 Archon 워크플로우 시스템이 기존의 스킬 기반 AI 자동화의 한계를 뛰어넘어, 신뢰성과 확장성을 대폭 강화한 플랫폼임을 실제 코드 예시 및 작업 흐름을 통해 입증

## 세부 요약 - 주제별 정리

### Archon은 오픈소스 하니스 빌더로 AI 코딩 프로세스를 개인화·자동화할 수 있도록 설계됨

- Archon은 "하니스"가 아니라 "하니스 빌더"로, 사용자가 자신만의 워크플로우를 직접 설계·패키징할 수 있도록 지원
- 기존의 AI 자동화 도구(예: BMAD)와 달리, 아키텍처/기술스택에 맞게 워크플로우를 자유롭게 조합 가능
- 다양한 코드베이스, 도구, 기술스택(PostgreSQL, SQLite, Docker, DAG, loops 등)과 쉽게 연동됨
- 제공된 기본 워크플로우(PIV loop 등) 외에도 사용자가 자유롭게 워크플로우 확장, 커스텀, 재작성이 매우 용이
- 주요 목표는 "내 코드베이스, 내 방식"으로 AI 코딩 프로세스를 완전히 내재화하는 것

### 워크플로우 마켓플레이스는 사용자 맞춤 워크플로우의 손쉬운 공유 및 활용을 가능하게 함

- 누구나 GitHub 레포에 워크플로우 패키징 후, 마켓플레이스(manifest TypeScript 파일)에 PR로 추가 가능
- 커뮤니티 중심의 공개 등록 구조(아직 별도 Discord 없음, Dynamis 커뮤니티에서 일부 교류 및 워크샵 진행)
- 워크플로우 검색·필터링 위한 태깅 시스템, 상세한 컨트리뷰팅 가이드 제공(구조, 의존성, 소스, 리드미 체계화)
- 기존 워크플로우 외에, AI 동영상 생성 등 다양한 에이전틱, 비코딩 워크플로우도 공유 및 확장 가능

### PR 자동 리뷰 및 머지 프로세스 구축을 위해 Archon 내에 자체 워크플로우를 사용하는 메타 구조 구현

- "워크플로우 마켓플레이스 PR 리뷰 및 머지"용 Archon 워크플로우를 새롭게 설계 및 커스터마이즈
- 제출된 PR이 marketplace.ts만 변경할 것 등 스코프 제약 자동 검증
- 코딩 에이전트(Claude Haiku 등)가 소스 패치, 스키마 검증, 보안 스캔, AI 리뷰 등 단계별 체크
- 결과에 따라 PR을 자동 병합(approve) 또는 댓글과 함께 반려(reject)하는 구조
- PR에서 악의적/품질 문제 탐지, 스팸 방지, 자동 보안점검 등 자동화된 관리 프로세스 적용

### PIV 루프(Plan-Implement-Validate) 워크플로우를 통한 AI 코딩의 신뢰성 극대화

- 탐색(Explore) → 명확화 Q&A → 플랜 작성 → 이행 준비(Setup) → 구현(Implement) → 검증·코드리뷰(Validate) 순서의 자동화 루프
- 각 단계별로 휴먼 인 더 루프(HITL) 확인 및 피드백 인터페이스 존재
- 여러 에이전트 세션(플랜, 인플리멘트, 리뷰 등)을 독립적으로 분리하여 신뢰성과 반복성을 확보
- 계획 문서(plan.md)의 엄밀한 사전 검토, 구체적 성공 기준·리스크 문서화, 각 노드별 디터미니스틱 작업 실행
- 예: 단일 기능 추가, 전체 아키텍처 워크플로우 등 다양한 규모에 대응 가능

### 다양한 AI LLM(Claude Sonnet, Opus, Haiku, Codeex 등)을 워크플로우 노드별로 혼합·활용하는 멀티에이전트 설계

- 각 워크플로우 노드별로 LLM 모델 종류와 제공자를 개별 지정 가능(예: 구현은 Opus, 리뷰는 Sonnet 등)
- 비용 최적화를 위해 PR 리뷰 시 Claude Haiku(저렴/효율적)를 활용
- 워크플로우 유지 및 실행에서 각 LLM의 토큰 사용량과 5시간/주간 한도 등 세밀하게 모니터링
- Codeex, OpenCode, Pi 등 타 LLM·프로그래밍 모델 연동도 커뮤니티가 활발히 개발(문제점·비교피드백 공유)

### 실시간 라이브 빌드 과정에서 발생하는 개발/운영상의 실제적 문제와 그 해결 절차 공개

- 워크플로우 과정에서 명령어 입력 오류, LLM의 명령어 '환각', 디버깅 필요, 머지 충돌 등 다양한 상황 발생
- GitHub Actions에서 API 키 설정, 포크 PR시 시크릿 전달 제한, pull_request vs pull_request_target 트리거 이슈 설명
- 실제 PR 적용 후 워크플로우 오류(예: bun 경로문제) 발생 시 LLM(QA)으로 자동 수정보완, 재시도 반복
- 개발 중 실시간 버그트래킹(GitHub Issue 기록), 내부 메시지 전달 등 투명한 문제해결 절차 노출

### Archon 워크플로우와 Skill 기반 AI 자동화의 차별점 및 장점 심층 설명

- ‘Skill’은 “에이전트 내부에 시스템을 내장”하는 개념(비결정론적, 신뢰성 저하, 대규모 프롬프트 오류 위험)
- Archon 워크플로우는 “코딩 에이전트들을 러너블 시스템(워크플로우)으로 래핑”하여 처리 흐름을 결정적으로 구성
- 각 단계별 체크, 동시병렬 처리, 휴먼 피드백, 작업 격리(Worktree) 등 시스템적 신뢰성을 보장
- 필요시 BMAD 등 기존 스킬/하니스 구조를 Archon 방식으로 변환해 장점만 흡수 가능
- “Skill은 시스템을 에이전트에 넣고, Archon 워크플로우는 에이전트를 시스템에 넣는다”라는 구분으로 본질적 차이 설명

### 대규모·다중 레포지토리/조직 환경에서의 Archon 활용 및 오케스트레이션 전략 정리

- 수십 개 레포를 병렬로 처리할 때 PIV loop 워크플로우 맞춤화(공통 프롬프트, sub-agent 사용 등) 권장
- 각 워크플로우는 격리된 Worktree와 컨텍스트 관리로 병렬 실행 가능
- ‘Second Brain’(코딩 오케스트레이터) 개념 도입: 유저 맞춤형 orchestrator로 각 워크플로우/이슈/PR 처리 자동화
- Web UI에서도 채팅 기반으로 다수 워크플로우 병렬 터치 및 오케스트레이션 수행
- 엔터프라이즈급 워크플로우 설계/이관 위한 별도 워크샵 진행 계획 예고

### 마켓플레이스 자동화 프로세스의 보안·검증·품질 관리 체계 및 향후 개선 방안 논의

- 동적 PR 리뷰/병합이 보안적 위험(서플라이 체인 등) 요인을 내포하므로, PR은 marketplace.ts 한정, 자동화 보안 스캔, 악의적 패턴/스팸 검증 등 다수 체크 적용
- 커뮤니티 참여 리뷰 혹은 신뢰 검증 절차(Trusted Submitter, fork PR 제한 등) 추가 논의
- 코드 리뷰, 테스트, 위험 등 2차 AI Judge(RAG, Sonnet 등) 등 다중검증 구조 내장 필요성 강조
- 인증, 해시확인(Sign/Hash for Install), 워크플로우 등록/사용 과정 추가 보안 기능 제안
- 완벽한 보안은 불가(커뮤니티 주도형 오픈마켓 특성상), 개발자 권장사항·배너·책임 한계 고지 등으로 보완

### 실시간 사용 경험(토큰/속도/에러 등) 및 음성입력, 도구 활용 노하우 공유

- AquaVoice 등 음성입력 도구로 신속한 브레인덤프, 대형 코딩 입력 효율화(타자 110~120wpm, 발화 224wpm)
- Archon 워크플로우는 실행속도보다 신뢰성·확장성·비용효율성에 우선순위
- LLM 사용량, 컨텍스트, 오케스트레이션, 오픈소스/상업적 사용 시 체크포인트 등 실전 팁 제공

### 라이브 스트림 내 실시간 커뮤니티 Q&A, 협업 제안, Archon 신규 기능 개발 논의

- 탭 추가, 멀티에이전트 워크플로우, 디스코드/깃허브 커뮤니티, Dynamis 오프라인 워크숍 등 다양한 상호작용
- BMAD, Hermes, Pi, OpenCode, Gemini CLI 등 타 도구 연동 지원 및 구현상황 공유
- 대규모 회사/실서비스 워크플로우 구축, 코드 품질 관리, 커뮤니티 검증, 레지스트리, 오프라인 LLM 활용 등 심도 있는 사용자 참여 논의

### 마켓플레이스의 실제 구현 예시와 향후 로드맵 소개, 완성 단계 직전의 진행상황 정리

- 실제 Archon 워크플로우를 marketplace에 등록, 자동 리뷰·머지까지 시연하며 각 단계별 완성도 점검
- CI 연동, Anthropic API 키 설정, action 트리거, repo 구조, 오류 수정, fork 제한 등 실전상황 공개
- 남은 오류 수정을 반복적 LLM 지시로 자동화하며, 출시 목표 시점(다음 릴리즈) 및 완성된 서비스 시연 예고
- 추후 라이브 스트림에서 완성 데모와 공동 사용법 소개 예정, 지속적 개선과 커뮤니티 참여 촉진 선언
