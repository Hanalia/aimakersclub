## *대규모 생성 이미지 & 비디오 모델 구축: Veo와 Nano Banana의 실제* 핵심 요약

- 이 영상은 구글 딥마인드의 연구원 Sander Dieleman이 대규모 생성 이미지 및 비디오(Generative Image & Video) 모델(Veo 및 Nano Banana)을 실제로 구축하는 과정의 뒷이야기를 다룬다.
- 주요 초점은 최근 생성 미디어 분야에서 표준이 된 "확산 모델(Diffusion Models)"의 원리, 샘플링 체계, 데이터 처리 방식, 그리고 대규모 학습 구조에 있다.
- 데이터 큐레이션(구성)이 모델 성능에 매우 큰 영향을 미치며, 오히려 모델 구조 및 옵티마이저 개선보다 데이터의 질이 더 중요하다고 강조한다.
- 픽셀 기반 표현 대신 오토인코더(Autoencoder)로 생성된 "잠재 표현(latent representation)" 기반의 압축 형태를 활용해 메모리 효율성과 구조적 정보를 모두 보존하고 있다.
- 확산 모델의 핵심은 "노이즈 추가-제거" 반복 과정을 통해 점진적으로 이미지를 생성하는데, 주로 고수준 대 저수준 주파수 정보를 분석하며 이를 '스펙트럴 오토리그레션'으로 해석한다.
- 네트워크 아키텍처는 원래 UNet 기반에서 Transformer로 발전했으며, 대규모 언어 모델에서 축적된 확장성과 병렬처리 노하우가 이미지/비디오 생성에서도 강점으로 작용한다.
- 효율적 분산 학습에는 JAX 활용이 두드러지며, 특히 TPU 다중 칩 환경에 최적화되어 데이터/모델 병렬성을 쉽게 지원한다.
- 샘플링(guidance)은 다양성-품질 트레이드오프의 핵심 역할을 하며, "가이던스 스케일"이라는 하이퍼파라미터를 통해 사용자 의도 및 결과 품질을 크게 좌우할 수 있다.
- 샘플링 속도(단계 단축)을 위한 '디스틸레이션(distillation)' 및 '컨시스턴시 모델(consistency model)' 활용법을 소개한다.
- 텍스트 프롬프트 외 다양한 컨트롤 시그널(예: 참조 영상, 카메라 움직임)에 대한 대응, 그리고 실제 생성 모델 활용까지 포함한 실무적 이해를 폭넓게 제공한다.

## 세부 요약 - 주제별 정리

### 데이터 큐레이션이 대규모 생성 모델의 성공에 결정적 영향을 미침

- 고품질 생성 데이터(이미지, 비디오)를 엄선·구성하는 과정이 모델 결과의 질에 직접적인 영향을 미침
- 과거 연구(박사과정 등)에서는 공인된 데이터셋을 단순히 사용하는 경향이 있었지만, 대규모 생성 모델에서는 데이터 자체를 면밀히 검토·가공해야 함
- 데이터 세밀화 작업에 투입한 시간이 모델 구조 및 옵티마이저 개선보다 오히려 효율성·성과에 더 큰 영향을 끼침
- 데이터 큐레이션의 세부적인 노하우는 기업의 ‘비밀 소스’로, 연구공개가 어렵고 실무자들 각자가 경험을 통해 익히는 편임

### 잠재 표현 기반 압축이 모델 효율성과 성능을 동시에 확보함을 보여줌

- 기존엔 이미지/비디오를 픽셀 격자로 직접 입력했으나, 해상도 및 길이가 늘면 메모리 과부하로 직접 처리 불가
- JPEG, H.265 등 기성 압축코덱 대신, 자동 인코더(Autoencoder)로 학습한 ‘잠재 표현(latent representation)’ 방식을 사용
    - 예시: Stable Diffusion은 256x256x3 RGB 이미지를 32x32x4 ‘잠재그리드’로 압축
- 이런 방식은 공간구조(톱롤로지)를 최대한 보존하면서 고주파 상세정보는 일부 희생
- 영상에서는 시간축까지 포함된 3차원 텐서를 최소 수십~수백 배로 압축, 실질적 학습이 가능하게 됨
- EQVE 논문 등에서는 잠재공간의 주성분을 RGB로 시각화하여 원본의 물체 구조가 일정 부분 유지됨을 보여줌

### 확산 모델은 노이즈 추가(오염)와 제거(복원) 반복을 통해 이미지를 생성함

- 시퀀스 예측(Autoregression) 방식과 달리, 확산 모델은 이미지를 점진적으로 노이즈로 오염시키고, 역으로 그 노이즈를 단계적으로 제거함
- 한꺼번에 복원하지 않고, 각 단계마다 부분적 복원을 반복하여 점진적으로 이미지를 구체화
- 한 단계에서 예측은 수많은 원본 후보의 평균이 되기에 1단계로 예측하면 항상 흐릿해지며, 여러 단계를 거쳐 점차 디테일이 살아남
- 오염-복원 반복 도중, 외부에서 신규 노이즈를 약간씩 추가하면 누적 오류를 줄여 ‘샘플이 망가지는 현상’을 예방할 수 있음

### 주파수 분석을 통해, 확산 모델이 ‘거칠게부터 정밀하게’ 생성함을 직관적으로 파악함

- 원본 이미지의 푸리에(Fourier) 스펙트럼은 일반적으로 파워로우(기울기)가 나타남(=고주파수 세부 정보가 적음)
- 가우시안 노이즈는 전 대역에 평평하게 깔림(=모든 주파수가 동등)
- 이미지에 노이즈를 더하면, 점진적으로 고주파(세부) 정보부터 손실 → 충분히 넣으면 저주파(전체 형태) 정보도 소멸
- 따라서 확산 복원 과정은 저주파부터(전체 구조) → 고주파(세부 묘사) 순서로 이미지를 만들어내는 셈
- 이를 통해 이미지 생성에서 ‘의미 형상화→세부 조정’의 자연스러운 절차가 이루어짐

### UNet에서 Transformer로의 아키텍처 변화가 확장성과 유연성을 제공함

- 초기 확산 모델은 UNet(컨볼루션 신경망 기반·이미지 복원용) 구조를 주로 사용
- 최근에는 Transformer 구조가 도입, 단일/양방향 어텐션이 모두 허용되어 입출력 동시 예측이 용이함
- LLM(대규모 언어모델) 연구에서 축적된 파라미터 크기 확장성, 병렬 처리 기술을 이미지/비디오 생성모델에도 동일하게 활용 가능
- 특히 비디오 모델에서는 ‘시간축에서만 오토리그레션, 공간축은 디퓨전’ 등 하이브리드 모델도 적용(예시: Genie)

### 대규모 학습에서는 JAX와 칩 병렬화가 실용적으로 가장 우수함

- 일정 크기 이하에서는 데이터 병렬화(배치 분할)만으로도 충분하나, 초대형 모델에서는 모델 자체 병렬화 필요
- JAX는 모델 파라미터 자동 분할, 칩 간 통신 최소화 등 대규모 병렬 훈련에 매우 적합
- 특히 구글 TPU 환경에서는 JAX의 활용도가 극대화됨
- PyTorch도 지원 가능하나, JAX의 초기 설계가 대규모 병렬·TPU 최적화에 더 초점을 맞췄음

### 확산 모델의 샘플링(생성) 단계에서 '가이던스'가 품질을 결정적으로 향상시킴

- 가이던스(guidance)는 프롬프트(텍스트 등) 조건을 넣을 때, ‘조건 없음’(unconditional) 예측과의 차이(Δ)를 증폭시켜 진행 방향을 정함
- 가이던스 스케일(배수) 조정으로 다양성과 품질 트레이드오프 가능: 높일수록 결과는 더 선명하지만, 다양성은 줄어듦
- 최신 논문 이후로는 가이던스가 사실상 필수가 되어, 끄는 경우 모델 성능이 극단적으로 저하됨
- 오픈AI Glide(2021) 등에서 가이던스 효과 전후 사례 공개: 품질-다양성 차이 명확히 관찰됨
- 샘플 질이 너무 ‘스타일화’되는 효과나, 과도한 가이던스시 ‘색 채도 과잉’ 문제가 생기기도 하며, 이를 위한 스케일 조정 기법도 연구됨

### 샘플링 속도 단축 위해 Distillation 및 Consistency Model을 적극 활용함

- 확산 모델은 본래 수십~수백 단계 반복 샘플링이 필요하나, 이를 한 번(또는 극소수번)에 가까운 단계로 줄이려는 시도가 활발
- Distillation에서의 목표는 모델 크기 축소가 아니라, '경로 전체' 예측을 통해 샘플링 단계 자체를 줄이는 것
- Consistency Model은 하나의 노이즈-데이터 쌍에 대해 전체적인 예상 결말을 직접 예측(‘선형 경로’화)함
- 실제로는 단 1단계에 전체 경로를 예측하는 것은 신경망에 무리이므로, 여러 방법을 통해 점진적 경로 단순화 실시

### 텍스트 프롬프트 외에도 다양한 컨트롤 신호가 실제 활용·요구됨을 강조함

- 텍스트 프롬프트(설명문)는 기본적이지만, 실사용자는 ‘참조 이미지/동영상’, ‘카메라 동작’, ‘시간 지정’ 등 다차원 제어 신호를 원함
- 특히 생성되는 결과에 참여자를 직접 넣거나, 정확한 움직임·시퀀스 제어 등 고급 활용이 늘고 있음
- 대부분의 데이터셋에는 텍스트 외 신호가 없어, 모델을 프롬프트만으로 사전학습 후 ‘후학습(post-training)’ 단계에서 추가 컨트롤을 결합하는 방법 사용
- 후학습시 사람 피드백·선호도 기반 강화학습, 직접 최적화 등도 병행 실험됨

### 확산 과정의 결정론/확률론적 차이가 적용 목적에 직접 영향 미침

- 각 샘플링 단계마다 새 노이즈를 추가하면 본질적으로 ‘확률적’, 노이즈를 추가하지 않으면 완전 ‘결정론적’이 됨
- 완전 결정론적 샘플링은 디스틸레이션 등에서 유용(즉, 입력 노이즈가 같으면 결과도 항상 동일하게 나옴)
- 확률적 방식은 누적 오류에 더 강인하며 다양성을 보장함
- 사용 목적(양방향 샘플링, 단일결과 반복생성 등)에 맞는 샘플링 방식을 선택할 수 있음

### 실제 컨트롤 신호의 삽입 위치, 형식, 모델 구조에 따라 도입 전략이 달라짐

- Transformer 구조에서는 입력 토큰 추가, 값 브로드캐스팅 등 여러 방식으로 컨트롤 신호 삽입 가능
- 조건 신호가 토큰 수준, 잠재공간 수준, 입력 전체 브로드캐스트 등 다양한 방식 가능
- 예를 들어, 시간축/카메라 움직임 등은 시뮬레이터 등 고유 데이터로 추가되거나, 토큰화/브로드캐스트 모두 사용될 수 있음
- 실제 각 신호별 적합 삽입 방식은 많은 실험과 노하우가 축적되어야 결정됨
