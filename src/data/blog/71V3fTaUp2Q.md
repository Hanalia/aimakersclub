## *LLM이 스스로 탐색하며 강화학습 환경을 배우는 법: RL 환경 엔지니어링* 핵심 요약

- 발표자 스테파노 피오루치(Stafano Fiorucci)는 Haystack(Deepset사 오픈소스 LLM 프레임워크) 개발자이며, 개인적으로 작은 언어모델, 미세조정, 강화학습 등에 관심이 많음
- 최근 대형 AI 연구실과 스타트업들은 언어모델 성능 향상 및 지능 확장을 위해 수천 개의 RL(강화학습) 환경을 활용 중이며, 구체적 사례로 DeepSeek, MiniMax의 리포트와 OpenAI O1 모델을 언급
- LLM(언어모델) 평가 및 훈련에 특화된 강화학습 환경은 자연스러운 체험 학습/도구 사용/다단계 문제 해결이 가능하다는 점에서 중요성이 증가함
- 기존 언어모델 훈련 방식(대규모 사전학습 → 지도 미세조정 → 강화학습(RL))의 한계가 지적되며, RL 환경을 통해 ‘턴 기반 상호작용 및 검증 가능한 보상(verifiable rewards)’으로 성능을 개선할 수 있음을 입증
- DeepSeek R1은 경량화된 GRPO(새로운 RL 알고리즘)와 ‘검증 가능한 보상’ 도입, 도구 활용, 최적화된 환경 설계 등 다방면 혁신 사례로 소개됨
- 오픈소스 라이브러리인 Verifiers를 통해 다양한 RL 환경(싱글턴/멀티턴/도구형/상태관리 등)을 쉽고 유연하게 구축 및 훈련 가능함
- 실제 사례로, 작은 언어모델(LFM-2 등)이 RL 환경 내에서 틱택토(Tic-Tac-Toe) 게임을 학습해 뛰어난 실력자가 되었으며, 훈련 세부 전략(상벌 방식, 상대 난이도 설정, 일관된 환경 확보 등)이 결과에 결정적 영향 제공
- RL 훈련의 안정성과 성능(배치 크기, 탐험, 편향 방지 등)에 관한 실험적 인사이트도 심도 있게 공유됨
- 최종 결론: RL 환경과 검증 가능한 보상 체계는 언어모델에 ‘탐험의 자유’를 주어, 지도학습으로 어려운 고차원적 사고 및 전략 탐색을 가능케 하며, 작은 오픈소스 모델이 폐쇄형 대형 모델을 특정 과업에서 능가할 수 있는 길을 제시
- 실습용 코드, 환경 공유허브, 심화 리소스, 차기 과제 아이디어 등 구체적 제안을 마지막에 안내

## 세부 요약 - 주제별 정리

### 강화학습(RL)의 핵심 개념을 언어모델 영역에 어떻게 적용하는지가 발표의 출발점임

- 에이전트(agent)와 환경(environment)은 RL의 두 축을 이룸
- 환경은 에이전트가 행동(action)을 취해 상태(state)가 변화하고, 이에 대한 보상(reward)을 받음
- 에이전트의 목표는 누적 보상을 극대화하기 위한 탐험(exploration)과 착취(exploitation)의 균형
- 한 번의 전 과정을 trajectory(롤아웃/에피소드)라 부르며, 본 발표에서는 1게임 또는 1세트 상호작용을 의미
- LLM(언어모델)은 주어진 프롬프트에 대한 답변(완성)을 생성하는 통계적 모델로, 일반적으로 ①대규모 사전학습 ②지도 미세조정 ③강화학습의 3단계로 훈련
- 지도 미세조정의 예시: 프롬프트-응답 쌍으로 통계적 모방을 학습하는 SFT(Supervised Fine-Tuning)
- 최근 Ilya Sutskever(NeurIPS 2024 발표) 등은 기존 패러다임의 정체 한계를 강조, OpenAI O1 시리즈도 RL이 ‘다단계 사고(chain of thought)’ 훈련에 기여한다고 명시

### 검증 가능한 보상(Verifiable Rewards)을 활용한 새로운 RL 환경이 언어모델에 혁신을 제공함

- DeepSeek R1 발표자료에서, 체이닝(추론 과정)을 모델에게 학습시키기 위해 대규모 고품질 지도 데이터는 비용상 비현실적임을 지적
- 대신, 모델이 추론 과정(trace)과 정답(answer)을 생성하고, 이를 환경에서 자동 검증하여 보상을 주는 RL을 적용
- RL의 강점: SFT는 ‘주어진 예시만 모방’ 하지만, RL은 실험을 통해 더 나은 전략을 ‘스스로 발견’ 가능(예: 틱택토 승리법)
- 문서 정답, 게임의 승리, 도구 호출 성공 등 기계적 검증이 가능한 과제라면 이 방식이 강하게 작동함

### RL 환경에서 에이전트(언어모델), 환경, 도구 정의 및 상호작용 방식이 확장되고 있음

- RL 환경은 데이터, 하네스, 채점 규칙, 도구 등 전반의 동적 시스템 포함
- 과거 SFT 환경은 단순히 일괄적(정적) 데이터셋 활용, RL 환경은 상호작용 중심이며, 진화 가능
- 에이전트(언어모델)는 단순 텍스트 응답 대신, API호출, 시스템 명령, 도구 활용 등 복합적 액션 가능
- 소프트웨어적으로도 ‘환경’ 자체가 코드화(recipes, 패키지화)되어 재사용, 확장성 보장

### Verifiers 오픈소스 라이브러리는 LLM 전용 RL 환경을 쉽고 확장성 있게 구축할 수 있도록 지원함

- Prime Intellect가 개발한 Verifiers 라이브러리는 파이썬 환경 패키지로, 쉽게 설치/배포 가능
- 단일턴(Single-Turn), 다중턴(Multi-Turn), 도구 환경(Tool Env), 상태 기반(Stateful) 등 다양한 베이스클래스 및 파서, 보상추상화 제공
- OpenAI API 등과 호환되어 다양한 모델 접속 지원, Training은 자체 Trainer 및 Prime RL/Tinker/Sky RL 연계 가능
- 환경 논리에만 집중해 빠르고 안정적으로 실험·훈련 가능(인프라 관리 불필요)
- 예시: Reverse Text(문자열 뒤집기), Double-Check(수학문제-재확인), Wiki Search(도구/검색 활용 등)

### 실제 코드 구조로 살펴본 다양한 RL 환경(싱글턴, 멀티턴, 도구 활용 환경) 구현법

- 싱글턴 환경: Reverse Text 예시에서 데이터셋 1,000개 프롬프트→정답 매핑, XML 파서 적용, LCS(최장 공통 부분수열) 기반 보상 정의, 전형적인 평가·훈련 플로우 설명
- 다중턴 환경: Double-Check에서는 수학문제→'Are you sure?' 확인→최종응답 등 여러 차례 메시지 교환, state 딕셔너리/정지조건(via_stop) 데코레이터 활용 등 설명
- 도구 환경: 파이썬 함수로 정의된 도구를 모델에 부여, 모델이 호출(예: 검색, 계산)하고 환경이 실행 후 결과 반환, Wiki Search 환경 예시 권장
- 추가적 추상화: 모델컨텍스트 프로토콜 자동연결(NCP Env), Rollout별 상태를 유지하는 Stateful Tool Env, ‘REPL환경-재귀적 LLM’ 탐색 등도 지원

### RL 환경의 오픈소스 공유와 커뮤니티 협력의 중요성과 실제 ‘Environments Hub’ 활용법

- Verifiers와 Environments Hub는 환경 분열(Fragmentation) 문제를 막고, 오픈소스 모델이 폐쇄형 환경에 뒤처지지 않도록 지원
- 환경 묶음(리포지토리) 형태로 누구나 만들어 배포·활용 가능, 다양한 실험 아이디어 공유
- 폐쇄형 RL 환경 시장이 커지는 현실에서, 오픈환경의 확장성과 즐거움 강조

### 틱택토 환경 구현 예시: RL의 실제 효용성과 훈련 전략, 세밀한 환경 제어를 중점적으로 해부

- 이유: 틱택토는 간단하지만 다중턴, 동적 환경, 승리/무승부/패배 판단 등이 RL로 구현하기 적합
- 초기 버전: X(모델)가 항상 선공, 0~8 사이 숫자(move 태그), 상대는 무작위 전략
- 환경 구조: 데이터셋에서 시작 메시지 추출, 상태(state) 업데이트(보드・승자 등), 응답 처리(잘못된 움직임→즉시 패배·후에는 패널티)
- 두 가지 보상함수 결합: 승리(1점)/패배(0점)와 XML 형식 준수(+0.2점)
- 이후 점진적 개선: 모델의 선/후공 무작위 결정, 상대 수준 강화(임의vs최적전략), 환경 seed를 통한 일관성 유지, 실패(형식 오류, 잘못된 선택 등)에는 일정 패널티 부여
- 상대 난이도(무작위 확률)에 대한 stratified sampling으로 훈련 안정성 극대화

### RL 훈련 전략의 세부 Tuning 기법: 안정적 학습과 최적화의 실전 팁 제공

- GRPO(Group Relative Policy Optimization) 등 RL 알고리즘 도입
- 각 데이터(초기 상태)별 그룹 내 다양한 롤아웃 생성, 승리/형식/실수 등 각 보상별 가중치 적용, 평균치와 각 롤아웃 비교하여 이득(advantage) 계산
- 배치사이즈(batch size)와 안정성: 값이 크면 학습이 느리지만 안정적, 작으면 불안정(부분적 상황 반복 및 이상 학습 위험)
- 환경 편향(상대 알고리즘의 의도치 않은 편식-특정 첫번째 칸만 선택 등)에 주의
- 모델 선택: reasoning 기반 모델은 롤아웃 길이와 GPU 사용량에 신경, base 모델 평가·선택 및 단계적 향상이 효율적
- 훈련 중 모니터링은 로그/플롯 확인에 집착하지 말고, RL은 시간이 필요하다는 점 강조

### 실험 결과: 작은 언어모델이 RL 환경 내에서 특정 과업에 대해 대형-상위 모델을 능가함을 증명

- Liquid AI의 LFM-2(소형 오픈모델)는 불량한 답변 및 실수 많았으나, RL·SFT 병행 적용으로 형식 준수율·승률 대폭 향상
- GPT-5 Mini와 비교 시, 임의 상대에겐 유사·최적 상대에겐 본인 모델이 우수(85% 무승부, 거의 실수 없음)
- 실수 유형(트랩 등) 파악 후 RL 환경 개선 및 탐험(temperature 조정) 포함하여 궁극의 틱택토 마스터 모델 도출

### 실험 과정에서 얻은 실패 경험과 실전적인 RL 환경 훈련 조언

- 작은 배치크기, 다양한 상대 배분 불균형, 환경 편향(상대 전략 단순화), 모델 최대 길이 문제 등 언급
- base 모델 단계 평가, 롤아웃 직접 관찰, 자동 평가 결과 외 실제 조작 테스트 병행 권장

### RL 환경·보상 기반 트레이닝으로 ‘합리적 비용’에 특정 분야에 특화된 모델을 키울 수 있음을 강조하며 실습 방법과 아이디어 안내

- 명확한 평가 기준/보상함수, RL 환경, 오픈소스 Verifiers와 Hub 활용하면, 누구나 단일 과업에서 작은 모델이 대형 폐쇄모델을 능가하도록 훈련 가능
- 전체 실습 코드, 환경 구현법, 실전 튜토리얼은 LLM RL environment free course 및 Prime RL, Environments Hub에서 제공
- 다음 단계 제안: 도구 연동(검색, API 등) 다양한 RL 환경 설계, Wiki search 등으로 도전 권장
