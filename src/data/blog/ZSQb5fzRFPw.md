## *컴퓨터 사용 2.0: 에이전트가 멀티 커서를 얻다* 핵심 요약

- **본 발표는 Francesco Bonacci(Cua CEO), CTO Don, 인프라 책임자 Rob이 컴퓨터 사용 에이전트의 과거, 현재, 미래를 소개함**
- **초창기 에이전트는 스크린샷 기반 작업 및 사용자의 전체 화면 제어라는 한계가 있었음(Computer Use 1.0)**
- **오픈소스 프로젝트 “Quad Driver”는 백그라운드에서 실행되며, Mac, Windows, Linux 모두 지원; 화면을 takeover하지 않으면서도 에이전트가 OS 내 작업을 수행하게 함**
- **Quad Driver는 접근성 트리 및 스크린샷을 활용해 윈도우 상태를 파악하고, 백그라운드에서 다양한 입력 액션 구현 지원**
- **신뢰할 수 있는 에이전트 행동 평가를 위해 “KuaBench”라는 평가 프레임워크 개발(Setup/Oracle/Evaluator 구조, Python SDK 기반)**
- **KuaBench에는 130+개 검증 가능한 태스크와 42개의 환경이 구축되어 있으며, 5개 데스크톱 플랫폼 지원 및 CLI 기반 재현 가능**
- **Snorkel AI와 협력해 “KuaBench Kyad” 데이터셋(전자공학 실무 소프트웨어 임무) 개발, 실제 테스트 결과 현존 최고 에이전트도 25개 중 6개만 완벽 통과**
- **Quad Driver 적용 시, 토큰 사용량은 34% 감소·성공률은 62%→80%로 도약함(4K 해상도 테스트 기준)**
- **에이전트 지능 평가 시, 단순 행동 성공률뿐만 아니라 '세상 이해 능력'도 정량적으로 측정 가능(트래젝토리 fork, 모델 예측력 비교)**
- **컴퓨터 사용 에이전트 RL 트레이닝 시 GPU유휴시간으로 인해 비용 손실이 크다는 문제를, 온디맨드 오토스케일 워크플로우 및 워밍풀 전략으로 극복**
- **앞으로 Mac, Windows, Linux, Android 등 다양한 OS 지원 지속, 백그라운드 환경에서의 GUI 제어 및 자동화 발전 추구**

## 세부 요약 - 주제별 정리

### 초기 컴퓨터 사용 에이전트는 사용자의 전체 데스크톱을 제어하는 구식 방법에 머물러 있었음

- 1년 전의 "컴퓨터 사용 에이전트" 개념은 대중에게 익숙하지 않았음
- Microsoft 재직 시절 GUI 에이전트 실험이 시초
- 구식 “Computer Use 1.0” 방식은 스크린샷을 초반 입력으로 받아, 클릭/타이핑/스크롤 등 일련의 액션을 통해 사용자가 직접 화면을 제어했던 ‘인간 루프’에 의존
- 이 방식은 에이전트가 사용자의 화면 전체를 takeover하면서, 실제 사용의 불편함, 방해, 비효율이 뒤따랐음

### Quad Driver는 모든 OS에서 백그라운드 에이전트 동작을 지원하며 사용성을 혁신함

- 2개월 전 오픈소스로 공개된 “Quad Driver”
- 가장 핵심적인 변화: 에이전트가 백그라운드에서 동작, 사용자의 화면/활동을 takeover하지 않음
- Apple 프레임워크의 undocumented API 활용 등, 백엔드에서 복잡한 기술적 ‘블랙 매직’ 작동
- Mac뿐만 아니라 Windows, Linux 크로스 플랫폼 지원 달성
- 최초로 에이전트가 접근성 트리 또는 스크린샷 레벨에서 OS와 상호작용하는 로컬 드라이버 구현
- 예시: get_window_state 호출 → 접근성 트리+스크린샷 형태로 윈도우 상태 추출 → 백그라운드에서 입력 액션 실행(필요시 픽셀 기반 클릭)
- 정책적으로 “행동은 윈도우 단위(native), 실패하면 픽셀 정책 fallback” 전략 적용
- OS별 행동 일관성 확보를 위해, 내부적으로 8개 이상의 애플리케이션 하니스(app harness)로 수많은 테스트 진행
- Clicky, Mass Queno, H Company, Droid Factory 등 초기 리뷰어/어답터의 공헌 소개

### Quad Driver 적용으로 에이전트의 행동 기반 자동화 신뢰성과 효율이 비약적으로 증가함

- Quad Driver는 기존 전체 스크린·데스크톱 기반이 아닌 ‘윈도우 단위’ 집중 → 불필요한 토큰(맥락) 사용량 크게 절감
- 실제 평가(4K 해상도): 기본 평가툴 이용시 에이전트 패스율 62%에서 Quad Driver 적용시 80%로 증가
- 토큰 소모량 34% 절감(성능효율성 대폭 개선)
- 플랫폼 크로스오버: Mac, Windows, Linux 동시지원

### 평가 프레임워크로 KuaBench를 개발해 에이전트 행동의 신뢰성과 정확도를 체계적으로 측정함

- KuaBench는 자동화된 Task 평가 표준화 도구
- 구성: setup(환경 세팅 함수), oracle(정답 시나리오 함수-GUI 액션 기반), evaluator(환경을 실제로 체크)
- Python SDK 기반으로, 여러 OS에 걸쳐 동일한 코드로 태스크 및 평가 환경 구성 가능
- 현재 130개 이상 검증 가능한 태스크, 42개 환경, 5개 플랫폼 지원(모두 CLI 재현 가능)
- 외부 누구나 태스크·환경 저작 가능 → 커뮤니티 확장성 보장
- Snorkel AI와의 협력을 통한 “KuaBench Kyad” 데이터셋: 실제 회로 설계 소프트웨어를 이용해 전문가가 다양한 작업을 직접 측정하도록 설계
- 실제 결과: 모든 평가 모델에서 누적 보상 30% 한계, 최고 성과 모델도 25개 중 6개 태스크만 완벽 패스(특히 기존 회로 도식 편집 시에만 성공, 빈 도식부터 생성은 전무)

### 에이전트의 진정한 세계 모델(‘이해’)을 측정하는 새로운 평가 방법이 도입되고 있음

- 행동 성공만이 아닌, 에이전트의 ‘세계 이해’ 능력(월드 모델)을 정량적으로 측정
- 트래젝토리(행동 궤적) 내 어느 순간이든 상태를 fork하여, 해당 시점에서 모델에게 ‘보상’, ‘내부상태’, ‘화면 관찰’ 등 예측시킴 → 실제결과와 비교해 예측력 측정
- 즉, 행동·결과만이 아닌 잠재적 세계 인식 능력까지 평가 대상으로 삼음

### KuaBench 평가 신뢰성을 확보하기 위해, 환경/태스크를 '리워드 해킹' 등으로 직접 검증하는 체계를 구축함

- 각 평가 태스크별로 다양한 에이전트가 ‘리워드 해킹’ 시도를 거침
- 환경/코드 리뷰 결과, 이를 통과한 태스크만 데이터셋에 추가
- 수집된 검증 데이터를 코드래빗(codereview-style) 방식으로 문서화 및 검토

### RL 트레이닝에서 발생하는 GPU 유휴비용 문제를 “온디맨드 오토스케일 풀”로 해결함

- RL(강화학습) 환경에서 샌드박스 초기화/리셋 대기로 GPU 자원이 놀게 되는 것이 큰 비용 부담
- 대규모 또는 용량(40GB 등)이 큰 환경은 빠른 초기화가 불가피하게 어려움
- 이를 대응하기 위해 온디맨드 방식의 자동화된 풀(autoscaler pool) 시스템 설계
- GPU가 필요로 할 때만 ‘따뜻한(warm) 풀’ 샌드박스 인스턴스를 즉시 할당/확장—최적 사이즈는 실시간 수요에 따라 자동 결정
- 훈련이 진행될수록 샌드박스 풀의 실제 적합 size를 동적으로 조절, 불필요한 오버헤드 방지
- 샌드박스 인스턴스 비용은 일반적으로 GPU의 1/2~1/4수준이어서 일부 redundancy(여유) 비용 감수해도 전체적으로 저렴
- 윈도우, 리눅스, 안드로이드(곧 Mac OS) 등 다양한 OS 대상 즉시 환경 제공

### 안드로이드 등 모바일 플랫폼에서도 다양한 백그라운드 자동화를 추진 중임

- 안드로이드는 컨테이너화(예: GUI Docker 컨테이너 or Ubuntu 등으로) 등을 통해 백그라운드 툴 사용은 가능
- iOS 대비 상대적으로 유연성 높음(도구 기반 백그라운드 작업에 더 적합)
- Android Activity Framework와 연동하며, 현재는 GUI 전체 자동화보다는 '툴 사용' 위주로 구현

### Quad Driver, KuaBench, 인프라 자동화는 커뮤니티 기반의 오픈소스 협력과 활발한 실사용 피드백을 바탕으로 성장함

- 오픈소스 프로젝트로써, 다양한 기업/개인의 기여 및 검증을 받아 빠르게 발전
- 각종 초기 적용사례(clicky mass queno h company 등)를 통한 upstream feedback과 기능 발전
- CLI 및 SDK 기반 저작성, 검증성 강조

### 앞으로도 멀티 OS, 백그라운드 자동화, 신뢰할 수 있는 에이전트 평가 자동화 등 차세대 컴퓨터 사용 패러다임 확장에 집중할 계획임

- Mac, Windows, Linux, Android 등 멀티플랫폼 지원 지속 강화
- 백그라운드 제어, 다양한 환경 자동평가 확장 모색
- 커뮤니티 Q&A, 실사용 피드백, 산업 협력(예: Snorkel AI) 등 오픈 생태계와의 연계 강화
