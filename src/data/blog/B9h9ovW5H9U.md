## *에이전트에겐 단순 문서가 아닌 결정 추적(Decision Trace)이 반드시 필요하다 — Zach Blumenfeld, Neo4j* 핵심 요약

- Neo4j의 Zach Blumenfeld가 "컨텍스트 그래프(context graph)"의 필요성과 주요 구현 방안을 설명하고, 단순 문서나 데이터베이스 이상으로 ‘결정 추적(Decision Trace)’이 에이전트의 정확성과 설명 가능성에 왜 중요한지 강조함
- 컨텍스트 그래프란 전통적인 데이터베이스, 지식베이스에서 확장된 형태로, 과거 결정 과정과 그 근거·전거(Precedent), 정책·맥락까지 포함해 에이전트가 더 적합하고 신뢰할 수 있는 결정을 내릴 수 있게 함
- 예시로 금융 분석 에이전트가 요청을 받았을 때, 단순 고객·거래 정보만이 아니라 과거의 결정 이력·유사 사례·정책 등 결정 추적 데이터를 참조해 ‘승인/거부’ 및 그 이유까지 답변할 수 있음을 시연
- 결정 추적을 효과적으로 활용하기 위해 그래프 임베딩(graph embedding)·하이브리드(구조+의미)검색 등 다양한 방법으로 유사 트레이스를 탐색함
- 본 데모는 Claude, OpenAI 임베딩, Neo4j, Next.js 중복 결합 등으로 구성되어 실제 컨텍스트 그래프를 질의-응답에 적용하는 과정을 보여줌
- “UVX create context graph”와 같은 명령 한 줄로 프론트/백엔드가 결합된 컨텍스트 그래프 앱을 빠르고 손쉽게 구축할 수 있는 도구 패키지도 소개함
- 데이터 소스로 GitHub, Jira, Notion, Slack 등의 SaaS에서도 불러올 수 있는 커넥터 제공, 22개 내장 도메인(금융, 의료, 커스텀 등) 및 자동 온톨로지 설계 지원
- Neo4j Agent Memory 패키지는 단기/장기 메모리와 추론이 결합, 다양한 데이터의 엔터티 추출·통합·중복제거 등 컨텍스트 그래프의 정보 구조화 지원
- Cypher 등 그래프 질의어를 이용해 실제 정보를 질의·시각화·추론 가능, 새로운 결정 추적 추가 등 발전 방향 연구 중
- 실습용 오픈소스 데모, 문서, GitHub 소스 등 개발자를 위한 학습 자원도 제공됨

## 세부 요약 - 주제별 정리

### Neo4j는 연결성과 설명력을 강화하는 그래프 인텔리전스 플랫폼임을 강조함

- Zach Blumenfeld는 Neo4j에서 근무하며, 그래프 데이터베이스와 인공지능 결합에 초점을 맞춘 연구 및 개발을 수행
- Neo4j는 정보를 네트워크형으로 연결하고 해석하며, 지식 계층(Knowledge Layer)에서 AI 시스템의 정확성 및 설명 가능성(Explainability)을 높이는 역할
- 발표자는 최근 기술 마케팅에서 AI/머신러닝 연구 엔지니어링으로 전환했음을 밝힘
- ‘컨텍스트 그래프’라는 개념이 아직 신선하고 다수 청중이 실질 구현 경험이 부족하다고 언급

### 컨텍스트 그래프는 에이전트의 ‘더 나은 결정’을 위한 구조적 기반을 제공함

- 전통적 지식베이스는 질의응답(RAG, Retrieval)에는 적합하나, ‘왜 그런 결정을 내렸는가’ 설명에 한계가 있음
- 컨텍스트 그래프는 기존 정보(고객, 거래, 정책) 외 ▲과거 결정 추적(Decision Trace), ▲전거(Precedent), ▲동적 정책 등 결정을 둘러싼 고차원 맥락 데이터를 통합
- 실무의 금융분석 에이전트 예시에서 단순 응답이 아닌 “승인/거부+이유” 제시가 가능해짐
- 컨텍스트 그래프는 사회적·도메인 전문지식, 인과관계, 정책 변화 등 현실적인 의사결정 맥락을 구조적으로 표현

### 이전 결정 추적과 전거(Precedent)가 실시간 AI 판단에 핵심적 역할을 함

- 과거 결정, 승인, 거절 등 실제 이력(트랜잭션·이벤트·정책 변천 등)을 그래프에 저장
- 새 결정을 내릴 때 구조적·의미적으로 유사한 과거 트레이스를 활용, ‘유사 판례’처럼 가이드로 삼음
- 예를 들어 위험 평가(Risk Score), 주요 리스크 요인(Key Risk Factor) 추출과 설명이 구체적으로 가능
- ‘find precedents’ 도구로 그래프에서 직접 전거를 탐색, 에이전트 판단의 객관성·재현성을 제공

### 하이브리드 검색과 그래프 임베딩으로 유사 결정 트레이스를 효율적으로 탐색함

- 의미적 유사성과 구조적 유사성을 결합한 하이브리드 검색 방식 적용
- Neo4j에서 각 결정 트레이스(노드 및 경로)를 ‘그래프 임베딩’(vector)으로 변환하여 연관도 기반 빠른 유사성 탐색 가능
- 예: ‘fraud rejection’처럼 의미 기반 탐색은 백터 인덱스 활용, 과거 결정 프로세스 맥락까지 구조적으로 비교
- 전통 문서 기반 시스템에서는 어렵던 동적 의사결정 흐름 파악이 그래프 기반에서는 용이함

### 실제 데모 시연을 통해 컨텍스트 그래프 기반 질의-응답을 구체적으로 보여줌

- 데모 환경 구성: Claude LLM, OpenAI 임베딩, Neo4j 그래프DB(+vector), Next.js 프론트엔드
- CRM/고객지원 등 외부 데이터 수집, “질문 입력→도구 연쇄 실행→그래프 데이터/결정 이력 수집→최종 결정/설명 반환”의 플로우 시연
- 예시 질의(처방약 관련)는 시각적 스키마와 함께 여러 트레이스를 보여주며, 왜 그런 결정을 내렸는지 구조적으로 확인

### “Create Context Graph” 도구를 통해 개발자가 1분 만에 컨텍스트 그래프 앱 구축 가능함

- UVX create context graph 명령 한 줄로 백엔드/프론트엔드 기반 앱 구조 자동 생성
- 앱 이름, 도메인(헬스케어, 금융, 커스텀 등 22개 내장/맞춤 지원), 프레임워크(Pydantic AI, OpenAI 등), 샘플데이터 입력 등 설정 가능
- 생성 폴더 내 fixure JSON으로 더미 데이터 세팅, 알맞은 그래프 스키마로 애플리케이션 곧바로 구동
- Cypher(그래프 질의어)로 직접 데이터 추출, 트레이스 탐색, 시각화 등 즉시 활용 환경 제공

### 외부 SaaS 연동 및 다양한 도메인 지원으로 현업 활용도가 높음

- Github, Notion, Jira, Slack 등 다양한 SaaS 데이터 커넥터 내장
- 앱 생성 시 자체 도메인/온톨로지 정의 입력 가능, 필요시 자동 그래프 스키마 생성 및 엔터티 추출 프로세스 제공
- MCP 서버 연동, 멀티턴 대화, 구조화 질의 등 실제 서비스 환경에서 바로 실험/테스트 가능
- 오픈소스(공개 저장소) 기반으로 누구나 코드 확인·수정·기여 가능

### Neo4j Agent Memory 패키지는 단기/장기 메모리 및 추론 체계를 효과적으로 결합함

- 단기 메모리: 대화 세션 기록, 최근 컨텍스트 등 일시적 정보 저장
- 장기 메모리: 세션에서 추출된 엔터티, 반복 정보, 지속되는 데이터의 누적 및 통합 관리
- 추론(Reasoning): 장기메모리에 기록된 결정 트레이스와 연결고리로 인과적 추론 실현
- 메모리 API는 Microsoft Agent Framework, Google ADK 등 다수 프레임워크와 통합 지원

### 비구조 텍스트 데이터의 그래프 추출 및 온톨로지 생성도 자동화됨

- spaCy, gliner, LLM fallback 등 다양한 엔터티 추출 방식 내장
- 추출데이터의 통합, 중복제거, 풍부화 등 일련의 처리과정 지원
- 생성된 그래프 스키마: 회색(대화), 노랑(엔터티), 트레이스(결정들) 연결로 시각화 가능
- 구조적 데이터(.csv, 테이블 등)는 Cypher 매핑으로 신속 적용

### 아직 개발 초기지만, 다양한 확장 및 품질/신뢰도 개선 논의도 이루어짐

- 결정 트레이스, 감성/품질 Score 등 추가 관리 기능 논의
- 사용자가 의도적으로 결정 트레이스를 저장하는 방법, 자동 추적 활성화 등 연구 진행 중
- 새 커스텀 도메인 생성, 온톨로지 정의, 엔터티 추출 사이클 개선 등 실무 적용성 지속 강화

### 실습 데모, 문서, 오픈 리소스 등 학습·테스트를 위한 폭넓은 지원이 제공됨

- 공식 블로그 및 데모 사이트에서 실시간 컨텍스트 그래프 앱 실습·테스트 가능
- ‘Create context graph’ 명령 및 문서, GitHub 저장소에서 전체 소스코드·모듈 공개
- 질의응답 세션에서 다양한 확장 요구, 직접 온톨로지 설계 지원 등 Q&A도 상세히 진행됨
