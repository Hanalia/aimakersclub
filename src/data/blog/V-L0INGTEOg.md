## *바이킹 VOIP 전화 프로토콜을 Claude Code로 리버스 엔지니어링하기* 핵심 요약

- 11 Labs의 보리스 스타코프가 영국 AI 역사 퀴즈를 AI 음성 에이전트와 상호 작용하는 데모 전화 부스를 구현하는 과정을 소개함
- 프로젝트의 핵심 하드웨어인 바이킹 VOIP 전화는 호환 소프트웨어가 Windows XP에서만 작동하며, 11 Labs에는 Windows 노트북이 없어 설정에 어려움을 겪음
- 1년 전 샌프란시스코 11 Labs 팀이 이 기기를 도입했지만, 한동안 3명의 시니어 엔지니어와 ChatGPT를 활용해도 활용법을 찾지 못하고 방치됨
- 보리스는 Claude Code를 활용해 하드웨어 리버스 엔지니어링에 도전, 포트 스캐닝과 프로토콜 분석을 진행하며 명령 체계를 추출함
- 바이킹 전화는 공개 문서가 없는 2글자 명령 체계로 동작하며, 약 80개의 유효 명령을 브루트포스 방식을 통해 Claude가 자동 추출함
- 운영 과정 중 임시 메모리에 이용자 정보가 남지만 재부팅 시 사라지는 문제가 발생, 영구 저장 방법을 찾기 위해 프로토콜을 심층 분석함
- 윈도우 가상머신(UTM)과 TCP 프록시를 이용, 공식 소프트웨어와 전화기의 통신을 가로채어 패킷을 분석하고 체크섬 등 암호화 부분을 역공학함
- Claude가 1바이트 단순 체크섬 암호화 형식을 추론하여 직접 명령 저장 및 영구화에 성공함
- 이제 윈도우 환경이나 서비스툴 없이 누구나 Claude Code와 오픈소스 스킬을 이용해 바이킹 전화 설정 가능
- 전체 프로젝트에서 Claude Code의 역할과 AI 기반 리버스 엔지니어링의 실질적인 효용성을 강조함
- 구현 과정에서 어느 정도 실시간 협업(Claude 지시 + 물리적 조작)이 필요했으며, 프로젝트 비용은 약 10~100달러였음
- 이 경험과 스킬은 다른 하드웨어에도 확장 가능성이 있음을 시사함

## 세부 요약 - 주제별 정리

### 3명의 엔지니어와 ChatGPT로도 풀지 못했던 문제를 Claude Code로 1년 만에 해결함

- 1년 전 샌프란시스코 11 Labs 팀은 바이킹 VOIP 전화를 이벤트용으로 구입했으나, 세 명의 시니어 소프트웨어 엔지니어와 ChatGPT가 협업해도 활성화 및 네트워크 연동 실패
- 기기는 Windows XP 호환 소프트웨어만 지원하였고, 11 Labs 대부분 직원은 Mac만 사용하여 환경 준비에 어려움을 겪음
- ChatGPT만으로는 역공학 및 통신 분석 한계에 봉착, 문제를 해결하지 못하고 1년간 방치됨

### Mac 환경에서 Claude Code를 통해 하드웨어-소프트웨어 연결의 첫 걸음을 내딛음

- Windows 노트북이 없어 맥북과 라우터, 이더넷(PoE) 케이블을 이용해 바이킹 전화와 네트워크 연결 시도
- Claude Code가 nmapping으로 네트워크에 연결된 디바이스 탐지, 포트 스캐닝을 통해 통신 가능한 포트를 찾음 (1001번 포트가 초깃값이었으나 잘못된 선택, 재탐색 후 성공적 포트 획득)
- CLI(명령줄) 테스트 및 최초 랜덤 데이터 송신 후 전화로부터 응답 획득, 독자적인 인터페이스 존재 확인

### 프로토콜 문서가 전무한 상황에서 브루트포스와 응답분석으로 명령어 체계를 도출함

- 바이킹 전화의 프로토콜은 인터넷 등에서 정보를 찾을 수 없어 전적으로 직접 분석 필요
- Claude Code가 2글자 명령어 전수검사를 자동화하여 브루트포스 실행, 대부분 에러(ER 반환), 80개 내외의 유효 명령어 단위 추출
- 도출된 명령어 중 일부는 의미 파악 가능(예: SA는 status로 추정)
- 예시 : “SA” 등 다양한 명령의 응답 패턴을 관찰하며 분석 진행

### 임시 메모리와 영구 메모리의 차이를 확인하고, 데이터 영구 저장 패턴을 역공학함

- 명령어로 전화기에 자격증명(전화번호 등) 설정 가능하지만, 재부팅 시 임시 메모리에만 남아 영구 저장이 되지 않는 결함 발견
- Claude Code와 수 시간에 걸친 반복 & 다각도 분석으로 저장 명령 추가 및 장기저장 방식 추적 시도
- 세 글자 명령어도 전수 검색했으나 아무런 단서 획득 실패

### 가상머신과 TCP 프록시를 통한 ‘맨인더미들’ 방식으로 공식 소프트웨어-전화 통신을 실시간 분석함

- 바이킹 공식 소프트웨어는 Windows 환경 전용이어서 가상머신(UTM)에 설치
- 다만 맥 환경에서는 VM에서 이더넷 브릿지가 불가해, 클로드의 지시로 맥북에서 TCP 프록시 서버를 구축하여
- 공식 프로그램과 전화 사이의 모든 트래픽을 중계 및 기록, 데이터 패킷을 상세히 탐색하는 ‘맨인더미들’ 분석 구조를 완성함

### 덤프한 패킷에서 1바이트 체크섬 규칙을 분석해 암호화 패턴을 완벽히 해독함

- 통신에서 확인된 “TS” 명령은 바이너리 페이로드, 마지막 1바이트는 체크섬
- Claude Code가 보내는 데이터와 전화 응답 관계, 체크섬 규칙(단순 가감산 기반 1바이트)을 역으로 추론 및 검증
- 실제로 여러 값 실험, 반복적, 폐쇄 루프형 프로토콜 완전 해석 달성

### 직접 명령어로 바이킹 전화기 영구 설정이 가능해지고, 모든 과정을 자동화·오픈소스화함

- 체크섬 및 저장 명령 체계까지 해독 이후, 더 이상 Windows 환경이나 공식 툴에 의존하지 않음
- Claude Code에 ‘스킬’ 형태로 명령어 자동 전송 구현, 다른 누구라도 Mac 등에서 손쉽게 전화기를 설정 가능
- 오픈소스 수준의 문서 및 코드 공개, 설정 복잡도가 현격하게 줄어듦

### AI가 ‘물리적’ 단계에서도 직접 리드를 하며 인간은 ‘손과 눈’ 역할로 협업하게 됨

- 구현 과정에서 사용자는 Claude Code의 지시(예: 수화기를 들었을 때 몇 번 소리 들리는지 등)에 따라 물리적 조작만 수행
- Claude Code가 네트워크 분석, 명령 송수신, 프로토콜 추론 등 모든 지적 판단을 주도, 사용자는 단순히 수행만 함
- 폐쇄 루프 실행, 일부 단계(가상머신 내 소프트웨어 컨트롤)는 자동화도 가능했으나 시간상 직접 처리

### 전체 과정 비용, 진행 상황 및 실제 구현 자리(데모 부스) 등 구체적 내용 공유

- Claude Code/API 이용 등 드는 비용은 약 10~100달러(기업 체험 토큰 활용)
- 데모 완성 후 실제로 대중들이 체험 가능한 전화부스(런던, 3층)에 설치
- 수화기를 들면 Michael Caine 성우 AI 에이전트가 5가지 AI 역사 퀴즈를 진행, 참가자가 한 문제 이상 맞히면 11 Labs의 상품 제공
- PoE 이더넷선, 라우터, Wi-Fi 등 물리적 네트워크 구성 상세 설명

### 이번 리버스 엔지니어링 경험과 오픈스킬은 다양한 하드웨어 해킹에 확장 가능함을 강조함

- 과거에는 폐쇄적 하드웨어 설정 시 제조사 소프트웨어에 의존하는 것이 일반적이었으나, 이제는 AI가 네트워크 패킷 분석 및 명령 자동화가 가능
- 바이킹 외 다양한 IoT/하드웨어에도 Claude Code 활용 가능, 일반 개발자도 손쉽게 접근 가능함 시사
- 모든 과정은 직접 ‘해킹’이 아니라 ‘리버스 엔지니어링’임을 명확히 하며, AI의 실제적 문제해결 파워와 확장성 부각

### Claude Code가 아니었다면 성공하지 못했을 만큼 AI 기반 리버스 엔지니어링의 혁신성을 입증함

- 보리스는 자신이 보안 전문가가 아니며, 초반에 등장하는 복잡한 메시지/ 명령 구조를 스스로는 해체하지 못했음을 언급
- Claude Code의 역할이 단순 ‘도우미’가 아니라 전 과정의 지적 엔진이었음
- 과거보다 10배 빠를 뿐 아니라, 아예 불가능했던 작업을 가능하게 만들어 주었음을 강조하며 세션을 마무리함
