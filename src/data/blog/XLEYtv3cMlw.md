## **과학적 업무를 위한 자율 에이전트** 핵심 요약

- 영상에서는 Sina Shahandeh가 자율 에이전트(특히 LLM 기반 코딩 에이전트)를 활용하여 실제 과학 연구 업무를 자동화하고, 기존 한계를 어떻게 극복할 수 있는지 집중적으로 설명함
- 기존 'Auto Researcher'는 코드 최적화와 반복 실험에 강점을 보이지만, 혁신적인 가설이나 새로운 아이디어를 스스로 도출하는 데는 한계가 있음
- 영상의 핵심 사례로 RadiCait에서 진행 중인 ‘CT 영상을 PET 스캔 이미지로 변환하는 ML 모델’ 개발 과정을 구체적으로 제시
- 연구 현장에서는 반복적 실험과 코드 구현은 비교적 쉽게 자동화되지만, 연구자의 창의적 가설 생성이 병목 지점임을 강조
- 문제를 명확한 계층적(하이어라키) 하위 컴포넌트로 분해하고, 각 파트를 문서화/자동화하여 새로운 아이디어 생성 및 실험이 용이해지는 구조를 구현함
- GPT-4, Gemini, Oracle CLI 등 최신 LLM과 클라우드 도구를 루프 내에서 협업적/상호비판적으로 활용하는 방식을 도입
- 이미지 등록(image registration)∼품질 관리 등에서 멀티모달 모델과 서로 다른 에이전트의 역할 분업·협업 구조를 구체적으로 설명
- 현재 멀티모달 LLM 및 AutoML은 관찰(특히 미세한 의료 영상 관찰) 부분에서 한계가 있으며, 향후 미세조정(fine-tuning) 필요성을 제시
- 체계적 계층 분해 및 가설 루프 구조 덕분에 인간 연구자에 근접하는 자율적 연구 프로세스 실행 가능성을 보여줌
- 결론적으로 근미래에는 더 성능 좋은 모델이 나오더라도, 현재는 분명한 문제 분해와 루프 구조 설계가 자율 연구 도입의 핵심임을 시사

## 세부 요약 - 주제별 정리

### 코딩 에이전트의 오토리서치 성과와 한계를 명확히 분석함

- Auto Researcher 혹은 Auto Research는 코드 자동화와 실험 반복에 있어 매우 강력하다.
- Andrej Karpathy의 GitHub 리포를 예시로, ML 모델이 주어진 지표(metric)를 최적화하며 반복적으로 개선하는 구조(힐클라임)를 소개.
- 이러한 구조는 명확히 정의된 코딩 업무에는 매우 효과적이나, 목표가 모호하거나 장기적인 과학 문제(롱 호라이즌)에선 아이디어 고갈 및 한계점(Research Taste의 결여)이 드러남.
- 인간 연구자는 상위 1%에 속할수록 반복 루프에서도 계속해서 더 좋은 가설과 혁신적 접근을 도출해냄.

### 과학적 문제는 가설 생성 능력이 에이전트의 성패를 좌우함을 강조함

- 과학의 본질적 루프는: 관찰 → 질문 제기 → 가설 수립 → 실험 및 결과 분석의 반복임.
- 현재 AI/코딩 에이전트는 ‘기억, 실수로부터 학습, 구현’은 비교적 잘 수행 가능함.
- 그러나 ‘좋은 가설 도출’, 즉 새로운 아이디어 창출은 극복이 어려운 병목 지점임.
- 실제 현장에선 코딩 에이전트가 plateau(효율 정체점)에 다다르며 획기적 가설이 필요해지는 지점이 명확히 존재함.

### RadiCait의 ‘CT 영상을 PET 영상으로 변환’하는 ML 프로젝트 과정을 구체적 사례로 제시함

- RadiCait에서 진행하는 예제로, CT 영상을 바탕으로 PET(양전자방출단층촬영) 영상을 ML로 생성하는 인실리코 PET(insilicopet) 솔루션을 개발 중임.
- 실제 환자의 CT 슬라이스 이미지로부터 결절(nodule)이 암인지 여부 판단이 목적.
- PET 스캔은 실제 의료 현장에선 고비용·고시간 작업이기에 ML기반 변환이 필요.
- GAN(생성적 적대 신경망) 등으로 CT → PET 이미지를 생성, 학습 과정과 2.5D→3D 아키텍처 전환 등 도전 사례가 존재.
- ML이 단순 파라미터 변경을 넘어, 아예 구조적 혁신(예: 3D 컨볼루션 도입 등)을 하려면 새로운 가설 생성 루프가 필수임.

### 계층적 문제 분해와 문서화로 혁신적 아이디어 생성이 가능해짐을 보여줌

- 복잡한 과학업무는 ‘문제→하위 문제→컴포넌트’ 식 계층적 분해가 핵심임을 강조.
- 예: “폴리머 CT 패치 → 동등 PET 변환” 과제를, 데이터·코어 코드·아키텍처·트레이닝·손실함수·측정지표·스크립트 등 세부 계층으로 세분화함.
- 각 계층별로 README, 세부 문서 등으로 정확히 구조화·문서화하여, LLM이 각 레벨별로 창의적 개선을 “관념적으로” 수행 가능해짐.
- 실제 실습에서는 Obsidian 등 도구로 계층적 문제 문서화를 실행.

### 프롬프트 설계와 다층 문서 활용으로 코드 베이스 혁신을 실현함

- LLM/코딩 에이전트에게 ‘계층 문서’를 인식시키는 프롬프트를 제공하면, 기존보다 훨씬 근본적·다각도의 개선 아이디어가 생성됨.
- 예전 방식(그냥 코드베이스 + 목표 제시)은 빠른 포화(한계)에 도달하였으나, 계층적 분해 + 가설생성 프롬프트 혼합 시 수십∼수백 가지 새로운 솔루션이 도출될 수 있음.
- 각 컴포넌트마다 루프 내에서 개선을 시도할 수 있어 전체적으로 ‘실질적 진보’가 이루어짐.
- 이렇게 하면 2D→3D 영상, 트레이닝 로직, 데이터 증강 등 기존에는 잘 시도하지 않았던 변화가 자발적으로 등장함.

### 에이전트 간 상호비판과 협업 구조를 도입하여 루프 효율이 대폭 개선됨

- 각 루프(실험-가설-비판-재실험)에 여러 LLM/에이전트가 협력·평가·반론하는 adversarial/collaborative 구조를 포함시킴.
- 한 에이전트(코딩)이 만들어낸 플랜/가설을, 또 다른 모델(예: Reasoning에 강한 LLM)이 검토 및 반론함으로써 실제 개선 폭이 커짐.
- 반복적인 검증-실행을 통해, 실제 인간 연구자의 협업 방식과 유사하게 진화 가능.

### 데이터 준비 및 품질 관리에서도 멀티에이전트 활용 효과를 구체적으로 설명함

- 데이터 품질 관리, 예를 들어 이미징 레지스트레이션(CT-PET 정합) 작업은 또다른 루프 구조임.
- CT와 PET는 시간차·장치차로 완전히 정렬(align)되지 않음 → 최적정합이 필요(옵티마이즈 루프)
- 실제로는 ‘수치적 지표’뿐 아니라 ‘정성 평가’(예: 이미지를 보고 마스크가 잘 적용되었는지 등)까지 필요함.
- 이를 위해 멀티모달 LLM(예: Gemini 모델)이나 특화된 모델에 QA Task를 맡기는 식으로 루프를 설계.
- Oracle CLI, GPT-4.5 Pro API 등 최신 도구와 연동하여, 코드/데이터 패키징→모델 평가→Feedback 적용→재실험을 자동화함.

### 현실에서는 멀티모달 LLM의 미묘한 관찰능력, 특히 의료영상 해독력에 근본 한계가 존재함을 지적함

- 현재의 LLM/멀티모달 모델은 미세한 이상(결절 등) 관찰은 제대로 못함.
- 대부분의 대형 LLM은 과학적/의료 이미지를 충분히 학습하지 않아 실제 전문 연구자 수준의 ‘관찰-판단-가설’ 루프를 못 갖춤.
- 앞으로 미세조정(fine-tuning) 및 특별한 데이터셋을 통한 최적화가 반드시 필요함을 강조.

### 명확한 계층/루프 구조와 역할 분할로 근미래 자율 연구 스케일업의 실질적 가능성을 보여줌

- 문제의 구조적 분해, 각 계층별 개선 시도, 루프 내 협업 구조 등은 현존 역량 내에서도 상당한 ‘준자율’ 연구 자동화를 가능하게 함.
- 효율적인 ‘테스트 타임 컴퓨트’ 확대, 루프 반복·분기, 토큰 생산량(아이디어 수) 최대화 등 실질적 생산성 향상 확인됨.
- 향후 더 우수한 모델들이 등장하면 인위적 문제 분해나 트릭이 줄어들 수 있음.
- 현재로선 ‘계층 구성+루프+에이전트 협업’이 자율 과학 연구 자동화의 실질적 전략임을 보여줌.
