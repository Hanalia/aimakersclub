## *AI 코딩 마켓플레이스가 드디어 라이브로 오픈됨!* 핵심 요약

- 본 영상은 오픈소스 AI 코딩 워크플로우 도구인 **Archon**의 개발자 라이브 스트림으로, **워크플로우 마켓플레이스가 최초로 정식 오픈**된 과정을 상세히 시연함.
- **커뮤니티 기여 워크플로우(idea-to-work-order)**가 최초로 마켓플레이스에 등록·머지되는 전 과정을 보여주며, 기여 방법, PR(풀리퀘스트) 프로세스, 자동 리뷰와 배포의 흐름을 실시간으로 설명함.
- 워크플로우 등록 프로세스 자동화(자동 PR 리뷰 & 머지) 덕분에 Archon 유지보수자 부담이 줄고, **빠르고 투명하게 누구나 워크플로우 기여가 가능**해짐.
- 'idea-to-work-order' 워크플로우는 사용자의 새로운 개발 아이디어를 여러 단계별 작업 명세로 분해하는 기능을 제공, 실제 활용 시나리오도 라이브 데모로 진행함.
- 워크플로우 추가/리뷰 자동화는 **GitHub Actions**와 Archon/CloudCode의 결합으로 깃허브 PR만 제출하면 심사 및 머지가 진행되도록 설계됨.
- 커뮤니티 워크플로우는 보안 점검 및 기본적인 자동 검증을 거치며, 사용자는 신뢰할 수 있는 출처(예: 공식/저자)인지 확인 권장.
- **두 번째 커뮤니티 워크플로우(GitLab 스마트 MR 리뷰)**도 실시간으로 추가, 자동 리뷰/머지되는 모습 확인.
- 실시간 배포(릴리즈) 및 최신 워크플로우가 Archon 공식 웹사이트(workflows 마켓플레이스)에 실시간 반영되는 전 과정을 화면/스크립트 로그와 함께 공개.
- 마켓플레이스, 워크플로우 실행, 업데이트, 설치 등에서 발견되는 다양한 버그와 개선점도 즉석에서 이슈로 등록 및 논의.
- 전체 라이브는 Archon 생태계의 개방성, 자동화, 협업 문화, 빠른 피드백 사이클을 보여주는 데 초점을 맞춤.

## 세부 요약 - 주제별 정리

### Archon 워크플로우 마켓플레이스가 자동화 기반으로 정식 오픈됨을 시연함

- Archon 스트림 시작과 동시에 워크플로우 마켓플레이스가 드디어 라이브로 공개됐음을 언급함.
- 단순히 직접 제공하는 기본 워크플로우 외에, **커뮤니티 기여 워크플로우가 공식적으로 등록, 관리되는 구조**를 처음 도입함.
- 마켓플레이스의 첫 공식 커뮤니티 워크플로우(idea-to-work-order)가 실제로 머지되고 배포되는 과정을 실시간으로 보여줌.
- 워크플로우는 오픈소스 레포의 manifest 파일(PR)에 추가돼 관리되며, 그 이후 자동화된 PR 리뷰/머지/빌드/배포가 진행됨.
- 자동화 덕분에 유지보수자 확인 없이도 워크플로우가 빠르고 신속하게 커뮤니티로 퍼질 수 있게 됨.

### 커뮤니티 워크플로우 기여 방식은 오픈소스 운영 원칙을 따르고 누구나 손쉽게 접근 가능함

- Archon 워크플로우를 직접 제작하여, 자신의 퍼블릭 레포에 올리고, 해당 워크플로우를 manifest에 PR로 추가하면 됨.
- 컨트리뷰팅 문서(contributing.md)에 절차가 상세히 안내되어 있음.
- 과정 시연 중, 실제로 클라우드 에이전트(Speech-to-Text 등)로 명령 내리는 모습도 보여줌.
- PR 생성 시, 기존 오픈소스 프로젝트의 일반 규칙(contributing.md 활용, credit 부여, 개별 저장소 관리 등)을 따름.

### idea-to-work-order 워크플로우의 구조와 특징을 상세하게 소개 및 데모함

- 사용자가 새 기능이나 프로젝트 아이디어(예: 새로운 패키지 설계, 앱 신규 개발 등)를 입력하면 Archon이 일련의 질의 과정을 통해 요구사항을 파악함.
- 입력된 아이디어를 바탕으로 **작업을 단계별로 세분화(phase로 분할)**해, 각 작업에 대한 구체적인 작업 오더(명세, 코드 구현 단계 포함)를 자동으로 생성.
- 단순 PRD(제품 요구 명세) 생성이 아니라, 실제 코드 레벨 구체적 구현 방안까지 명세로 분해함.
- 모든 과정에서는 사용자의 **수동 승인(approval gates)** 절차가 여러 번 포함되어, 인간이 중간 피드백을 줄 수 있도록 설계됨.
- 실제로 'GitHub CLI 유틸 통합 리팩터링' 예시를 통해 데모 진행, 사용자가 각 단계(질의 응답, 위험성 분석, 단계별 작업 분해)에서 직접 응답함.

### 워크플로우 등록·리뷰·배포는 GitHub Actions 및 클라우드 코드로 완전 자동화됨

- 각 워크플로우 PR은 **manifest.ts** 파일 접근 여부만으로 자동 트리거, 자동 리뷰 및 머지/거절 기능 동작.
- 보안 점검(코드 스캔), 스키마 유효성, AI 기반 리뷰가 포함됨. 예시: AI 리뷰 결과 "리스크: 낮음, 스키마: 유효, 보안: 클린" → automerge 결정.
- 허용되지 않을 경우 피드백 코멘트로 PR에 자동 댓글 및 개선사항 노출.
- 배포 릴리즈는 '/release' 명령어 한 번으로 실행; Archon 자체 워크플로우가 새로운 워크플로우까지 포함한 릴리즈 전체 절차 자동화.

### 실시간 배포 및 검증 과정이 스크립트 단계별로 전부 투명하게 공개됨

- 배포 후 실제 Archon 공식 웹사이트(archon.diy/workflows)에 신규 워크플로우가 반영됐는지 스크린샷 자동 캡쳐 등으로 검증.
- 사용자는 CLI/웹 UI에서 곧바로 새 워크플로우를 설치·실행할 수 있음.
- 설치 과정에서 발생하는 커맨드 미지원(업데이트 필요), 바이너리 버전, 명령어 경로 관련 이슈 등도 실시간으로 파악·이슈화.

### 두 번째 커뮤니티 워크플로우(스마트 GitLab MR 리뷰)도 라이브 등록 및 리뷰 자동화 과정을 거침

- 라이브 도중 시청자 Raphael이 두 번째 커뮤니티 워크플로우를 PR로 제출.
- 해당 워크플로우도 자동 리뷰, 보안·스키마 검증 통과 후 자동 머지·릴리즈 처리됨.
- GitLab 전용 PR 리뷰로 Archon의 유연한 통합 가능성과 마켓플레이스 확장성을 보여줌.

### 마켓플레이스 운영에서 발생하는 실시간 버그 및 개선점도 즉시 이슈로 등록·대응함

- 명령어 실행 경로, 명령 프롬프트 경로, 클라우드 바이너리 인식 문제, UI 새로고침, 업데이트 알림 미비 등 다양한 문제가 논의됨.
- 각 문제는 AI에 명시적으로 GitHub 이슈 등록 요청을 하며, Archon 코드베이스와 프론트엔드 컨텍스트를 바탕으로 생성.
- 예: "마켓플레이스에서 설치한 워크플로우가 명령어를 인식하지 않음" → 관련 코드를 조사해 공식 이슈화하도록 지시.

### “인증/추천 시스템(Upvote)” 등 마켓플레이스의 향후 발전 방향도 언급됨

- 현재 마켓플레이스는 정적사이트 기반으로 인증 없이 운영되나, 향후에는 로그인·직접 제출·댓글·투표 시스템(Upvote/Downvote) 등 커뮤니티 기능 확장 계획.
- 워크플로우별로 플랫폼/통합 태그화 등 분류도 구상 중임.

### 릴리즈 자동화 및 Doc/ChangeLog 관리의 혁신이 AI 및 자동화 덕분에 대폭 향상됨

- 릴리즈 과정에서 체인지로그, 버전 관리 등 모든 과정이 AI(CloudCode)의 도움으로 빨라지고 단순화됨을 강조.
- API 업데이트, 명령어 자동 추가, 문서화, UI 반영까지 한 번의 주요 워크플로우로 일괄 처리.

### Anthropic SDK 과금 전환 이슈 및 LLM/모델 운용의 현장 경험도 실시간 토론

- 라이브 중 Anthropic SDK가 6월 15일부터 과금제로 전환된 소식, $200 무료 크레딧 한도 등 논의.
- 클라우드 코드 사용시 구독/Production 모드에 따라 API키 정책 차이 설명, 대체모델(Codeex, Pi 등) 고려 상황 공유.
- 클라우드 기반 또는 자체 CPU LLM 학습 등 시청자와 기술적 토론도 이루어짐.

### Archon을 활용한 실제 코드 작업 결과물(Work Order) 구조를 상세하게 시연함

- 워크플로우 실행 결과, 작업 주문(Work Order)은 자세한 context, todo 단계, validation, 정의된 anti-pattern, test, definition of done 등 표준화된 명세 파일(ReadMe/구조화 파일)로 생성됨.
- 추가적인 이슈 생성, 문서화, 티켓 등록 등 연속 개발 흐름에서 바로 활용 가능함을 시연.

### Archon 생태계의 개방성, 협업, 빠른 피드백과 커뮤니티 중심 발전 철학을 몸소 보여줌

- 스트림의 빠른 템포, 즉각적 이슈 등록과 피드백, AI와의 협업, 실사용자 참여(실시간 워크플로우 제출, 토론) 등 Archon이 지향하는 오픈 커뮤니티 엔지니어링 문화가 잘 드러남.
- 워크플로우 마켓플레이스는 "AI 코딩 자동화·협업의 진짜 출발점"을 열었고, 마켓플레이스 참여를 누구나 쉽게 할 수 있음을 강조하며 참관자들의 피드백 수렴, 신규 플랜(평가시스템 등)까지 비전을 구체화함.
