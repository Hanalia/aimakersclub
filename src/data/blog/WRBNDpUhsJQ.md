## *이종 지능의 다음 패러다임 확장하기* 핵심 요약

- 영상은 Adrian Bertagnoli(콜로시안 창립 엔지니어)가 인공지능에서 "이종 지능(heterogeneous intelligence)"이 왜 중요한지, 그리고 이를 통해 AI 시스템을 어떻게 더 효율적이고 확장 가능하게 만들 수 있는지 설명함
- 기존의 AI 발전은 "동질적 지능(homogeneous intelligence)" 즉, 동일한 모델·하드웨어에서 규모만 확장하는 방식(예: 동일 GPU상에서 대형 Dense 모델)에서 출발하였으나, 이제는 모델·워크플로우·하드웨어가 서로 다른 이질성을 갖는 방향으로 패러다임이 전환되고 있음
- 실제로 아키텍처(모델)는 단일 큰 모델에서 'mixture of experts'로, 워크플로우는 단일 LLM 호출에서 멀티 에이전트 시스템으로, 하드웨어는 동일 칩에서 분리·특화된 시스템으로 변화 중임
- 이종 지능이 효과적인 이유로 현실 세계의 문제들은 다단계·복합적 구조이며, 각각의 하위과제가 다양한 지능 유형을 요구하기 때문이라고 주장
- 산출함수(production function) 이론과 각종 수치·도메인(신경과학, 경제, 생태 등) 분석에서 동질적 시스템보다 이종 시스템이 여러 제약 조건에서도 더 뛰어난 성과를 내는 것으로 수학적으로 입증됨
- 실전 활용 사례로 '이종 재귀(heterogeneous recursion)'와 멀티모달 비디오 액션 언어 모델의 조합을 들며, GPT-5.2 등 대형 단일 모델 대비 현격히 빠르고(최대 5배), 저렴하며(최대 18배), 효율적인 결과를 보임
- 특히, 웹 비주얼 내비게이션 등 복잡한 과제에서 각 하위과제(줌 인/아웃 등)마다 최적화된 저사양 모델을 분배 사용해, 비용은 크게 낮추고 속도와 정확도는 향상시킴
- 최종적으로, 미래의 컴퓨팅 패러다임은 다중 에이전트적 워크로드를 최적의 하드웨어에 자동 사상(mapping)해서 더욱 스마트하고 빠르고 저렴하게 만드는 이종 지능이 될 것이라 전망함
- 영국 ARIA에서 후원받아 첫 번째 이종형 공존 클러스터(3백만 파운드 규모)를 설계·운영 중이라고 밝힘
- 단일(homogeneous) 성장의 시대는 끝났으며, 앞으로는 모델, 워크플로우, 하드웨어가 함께 진화하는 이종 지능이 새로운 혁신의 핵심임을 강조

## 세부 요약 - 주제별 정리

### 동질적 지능 패러다임의 한계가 AI 시스템의 발전을 제약함

- 기존 인공지능 아키텍처는 주로 동일한 모델을 동일한 GPU 등에서 대량으로 학습·추론하는 방식(동질적 지능)에 집중되어 있었음
- 큰 기여를 한 배경은 "Neural Scaling Laws"(신경 망 확장 법칙)로, 데이터와 파라미터가 많을수록 모델의 성능이 좋아진다는 규칙이 관찰되었기 때문임
- 하지만 이 법칙은 주로 ‘학습(training)’ 단계에 한정되어 있으며, 실제 ‘추론(inference)’에서는 그 효율성이 떨어지는 것으로 나타남
- 산업 현장에서도 아키텍처뿐 아니라 워크플로우와 하드웨어 수준에서 점차 이질화(heterogeneity)의 조짐이 나타나기 시작함

### 실제로는 아키텍처·워크플로우·하드웨어 세 축에서 이미 이질화가 진행되고 있음

- 아키텍처 수준에서 기존의 대형 Dense 모델이 ‘mixture of experts(전문가 혼합)’ 구조로 대체되는 경향 발생
- 워크플로우에서는 기존에 단일 LLM 호출(즉, 하나의 거대한 언어모델만)에서 벗어나, 여러 에이전트(모델)들이 협업하는 멀티에이전트 시스템으로 전환 중
- 하드웨어 배치 또한 단일 GPU/TPU에서 칩 분리, prefill/decode 디스어그래깃 등 역할별 분화가 일어나고 있음
- 현재는 경미한 이질화 단계로, 주로 프롬프트나 일부 하위 에이전트에서만 미묘한 다양성이 존재하고 있음
- 미래에는 각 하위과제별로 완전히 프로파일된 모델(LLM, state-space models, diffusion models 등)을 최적의 칩에 할당하는 방식으로 진화할 전망

### 현실 세계의 복합 과제가 다양한 지능의 조합을 요구한다는 점이 이종 지능의 필요성을 뒷받침함

- 실제 문제는 다단계적, 개방적(open-ended), 각 하위 문제마다 필요한 지능이 매우 상이함
- 따라서 단일 유형의 아키텍처로 문제를 해결하려면 비효율적이고, 제한적일 수밖에 없음
- 이상적인 접근은 서로 다른 아키텍처·규모의 모델들이 긴 time horizon(지속적 협력)에서 팀으로 문제를 해결하는, '멀티에이전트 이종 지능' 모델임

### 수학적·과학적으로 이종 시스템의 이점이 여러 분야에서 검증됨

- '최대 이질성 원리(principle of maximum heterogeneity)'를 그림과 함께 설명: 색상이 다른 에이전트가 여러 기술 집합(skill space)을 커버
- 각 에이전트간 링 토폴로지 구조로 연결, 특정 문제의 수요곡선(demand function)에 맞추려면 이질적 조합이 더 적합
- 동질적 모델(일관된 일반 모델들)의 경우 역량 확장이 제한적이며, 실제 수요함수를 충족하기 어려움이 수치로 제시됨
- 신경과학, 경제, 생태 여러 분야에서 합리적 제약 조건 내에서 이종 시스템이 일관적으로 동질 시스템보다 더 뛰어난 성과를 냄이 관찰됨

### 실무에서는 하드웨어·에이전트·워크플로우 세 단계에서 이종성을 적극 활용하고 있음

- 실제 적용 시, 하드웨어별로 에이전트가 구동되는 방식, 에이전트 간 상호작용, 워크플로우 구성이 모두 이질적으로 최적화 가능
- 사례별로 이종성 도입이 속도·비용 등 다방면의 극적인 성과 개선을 유도함을 강조

### 이종 재귀(heterogeneous recursion) 개념이 LLM 한계 극복에 효과적임을 입증함

- "재귀적 언어모델(recursive language model)"은 MIT 논문(2023년 10월 발표)에서 소개된 방식임
- 기존 대형 LLM은 프롬프트의 정보 복잡도가 커질수록, 맥락창(context window)이 크더라도 실질 성능 저하(context rot)가 있음
- 특히 선형, 2차원적 정보 요구(O(n), O(n^2))가 늘어나면 성능은 급격히 감소함(60~30%에서 성능붕괴)
- 재귀적 LLM은 전체 문맥을 prompt에 넣는 대신, 맥락(파일 등)을 외부 환경처럼 두고, 코딩 에이전트가 프로그래밍적으로 이를 순차적으로 검색(파이썬 REPL, 키워드검색 등)하여 하위 맥락들을 추출함
- 각각의 하위 맥락은 동일한 재귀적 에이전트에 전달되고, 필요시 추가 스폰/재귀 진행됨
- 콜로시안은 이 아이디어를 확장, 각각의 하위 맥락을 서로 다른 칩, 모델에 자동 사상해, 동일 모델 대비 더 빠르고 저렴하게 성능을 재현
- Ulong 벤치마크 결과: GPT-5.2 모델(2000초, 3.75달러/task) 대비 Cerebras에서는 7배 저렴·5배 빠르고, SambaNova에서는 12배 저렴·3배 빠름

### 시각적 웹 네비게이션 문제에서 이종 멀티모달 액션 언어모델이 SOTA(최신 최고 성능)를 능가함

- 실험에서 오픈/클로즈 소스 비디오 액션 언어 모델들을 혼합하여 사용(예: Quant 3 VL8B-Instruct, Kimik A 2.5 등)
- Video Web Arena 벤치마크에서 GPT-5.2, Gemini 2.5 대비 각각 18%·25% 성능 향상, 더 빠르고 저렴함을 입증
- 각 하위 과제(예: 화면 줌, 시각 정보 해석 등)는 복잡성이 달라 GPT급이 아닌 경량 모델 활용이 더 경제적이며, 정확도 저하 없이 비용/속도를 극적으로 개선함
- Quant 3 + GPT 조합 시, Kimi Kimi만 사용할 때보다 1.3배, GPT-5.2보다 3배 더 빠르고 3.7배 저렴
- ChatGPT 단독 사용 대비 서브태스크(줌 등)에서 최대 11배 빠르고 43배 더 저렴
- 결론적으로, 이종 혼합 방식은 현존 단일 대형 모델보다 압도적으로 비용/속도/정확도 측면에서 탁월

### 태스크 난이도에 따라 최적 모델·하드웨어 배분을 자동화하는 계층을 구현함

- 초반에는 각 하위 태스크별로 수작업(bespoke)으로 적합한 저사양 모델을 할당
- 최근에는 태스크 복잡도를 자동 인지하고 최적 하드웨어와 모델을 자동 배치하는 오토메이션 레이어를 개발해 완전 자동화
- 결과적으로 운영비용·속도를 사용자 개입 없이 자동 최적화 가능

### 미래는 이종형 컴퓨팅-지능이 패러다임이 될 것이며, 이를 위한 인프라를 구축 중임

- 과거 1세대 컴퓨팅 확장은 CPU 위주로 속도 향상, 2세대는 NVIDIA 중심의 병렬 컴퓨팅 확장, 3세대인 현재는 다중 에이전트, 이종 워크로드를 다양한 칩에 최적 매핑해 처리하는 단계로 진입함
- 영국 UK ARIA Institute로부터 3백만 파운드 그랜트를 받아, 첫 이종형 공존 클러스터를 기획·운영 중이라고 언급
- 앞으로는 모델, 워크플로우, 칩(실리콘) 계층이 함께 진화(co-evolution)하고, 새로운 다양성이 시스템 전체를 더 스마트·빠름·저렴하게 만듦
- "지금이 인프라가 가장 못난 상태일 것(This is the worst our infrastructure will ever be)"이라며 혁신 낙관론을 피력함

### 질의응답에서 모델 자동 분배(매핑) 방식과 아키텍처 의사결정에 대한 추가 설명

- 질문: "어떤 태스크를 더 빠르고 저렴한 모델에 할당하는 기준은 무엇인가?"
- 응답: 초기에 각 단계를 별도로 지정(bespoke)했으나, 현재는 태스크의 난이도를 감지하고, 자동화 계층에서 최적 모델·하드웨어를 할당함
- 사용자는 명령어를 지정할 필요 없이 시스템이 알아서 저사양 서브태스크에는 저렴한 모델을, 복잡한 태스크에는 고성능 모델을 자동 배정

### 결론에서, 동질성(homogeneous) 집중 시대가 끝나고 이종적 지능(heterogeneous intelligence)이 앞으로의 AI 혁신을 견인할 것임을 선언

- 과거 동일 모델·하드웨어 확장을 통한 발전에 감사하나, 더 이상 이 방법만으로는 한계에 봉착했다고 진단
- 앞으로는 ‘이종 지능’ 시대: 다양한 모델·워크플로우·하드웨어가 동시에 진화하며, 다양성 자체가 시스템 성능 혁신의 핵심 동력이 됨을 강조
- 혁신적 인프라 도입이 이 패러다임 전환을 가속화시킬 것임을 강조하며 발표를 마침
