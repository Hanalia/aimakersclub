## *모든 AI 에이전트를 하나로 관리하는 엔터프라이즈 레지스트리 구축 전략* 핵심 요약

- Amplifon은 26개국, 10,000개 매장, 2만여 임직원을 보유한 세계 최대 청각 케어 기업으로 글로벌 AI 전환을 추진하고 있음
- AI 솔루션이 여러 대륙, 다양한 팀에서 개별적으로 개발됨에 따라 보안·인프라·운영 및 거버넌스 상의 혼란이 발생
- Amplifon은 2025년 1월 'Amplify 프로그램'을 출범해 AI 도입 및 확산의 규칙과 실행안을 구조화함
- Amplify 프로그램 운영 모델은 '컨트롤 타워'(중앙 전략 및 가이드라인 수립)와 '위원회'(현장 실행 및 우선순위 결정)로 구성됨
- Amplify는 거버넌스, 플랫폼, 팩토리(실행조직)라는 3가지 축에 집중해 AI 활용의 준수·확장성·재사용성 확보를 목표로 함
- 주요 해결 과제로 모델 수명주기 관리, 컴플라이언스, 활용 내역 추적, 개발자 생산성 및 중앙 집중형 가이드라인 제시를 꼽음
- AI Gateway를 통해 모델 사용 통합 접근/보안 인증/예산 관리/사용현황 모니터링 체계를 도입함
- 엔터프라이즈 차원에서 MCP(도구), A2A(에이전트간), 유즈케이스(활용사례) 3종 레지스트리와 메타데이터를 설계, 정보관계와 영향관계까지 한 눈에 파악하게 만듦
- FastAPI 기반의 표준화된 블루프린트, GitHub템플릿, CICD, LongFuse 관측도구 등 개발자 지원체계를 갖춰 생산성·일관성·추적성 높임
- 이 플랫폼을 통해 조직 전반의 AI 사용 현황, 영향도, 연관 데이터 흐름, 오류 대응까지 완전한 가시성과 거버넌스 실현이 가능해짐

## 세부 요약 - 주제별 정리

### Amplifon은 글로벌 AI 도입을 위해 Amplify 프로그램이라는 통합 운영 체계를 구축함

- Amplifon은 전 세계 26개국에서 10,000개의 매장을 운영하며 20,000명 이상의 직원이 근무하는 세계 최대 청각 케어 솔루션 기업임
- AI 혁신을 추진하며 다양한 대륙 및 팀에서 독자적으로 AI 에이전트 및 인프라를 개발하는 과정에서 혼란이 가중됨
- 2025년 1월 글로벌 및 크로스 펑셔널 AI 도입 프로그램 ‘Amplify’를 시작함
- Amplify는 “운영 모델”과 “실행 플랜”으로 구조화됨
- 운영모델은 ‘컨트롤 타워’(최고 전략, 보안, 법무, 기술 가이드라인 수립)와 ‘위원회’(현장 전략 실행, 국가/코퍼레이트 단위 우선순위 조정)로 구성
- Amplify의 3대 초점: 거버넌스(준수/전사 알림), 플랫폼(인프라 및 개발생태계 확립), 팩토리(실행조직 및 재사용성 기반 확산)

### 대규모 AI 확산 과정에서 유지보수, 컴플라이언스, 스케일 관리가 최대 과제로 드러남

- LLM 기반 AI 솔루션의 짧은 수명 주기, 잦은 모델 교체/고장 이슈로 인한 유지보수와 신속 대응의 중요성 대두
- 각 팀/국가별 규정 준수, 활용 사례·데이터·AI 자산의 정확한 파악과 중앙 보고 필요성 발생
- 다양하게 분산된 인프라, 자산 현황, 자체 개발물들의 일원화 관리 필요
- 개발자가 매 과제마다 보안·배포·운영 문제를 재발명하지 않고, 비즈니스 로직에 집중할 수 있는 환경 제공이 중요

### AI Gateway 도입으로 모든 모델 사용 시 통합 접근, 인증, 비용관리, 모니터링을 실현함

- 모든 AI 모델 접근을 통합하는 통일된 엔드포인트(AI Gateway) 구축
- 개발자는 게이트웨이를 통해 Amplifon 사내 모든 LLM 및 에이전트에 접근 가능
- 보안 강화를 위해 ‘intra ID’ 인증 시스템을 게이트웨이에 연동
- 예산 관리를 게이트웨이에 도입: 각 유즈케이스별로 정해진 예산 설정 및 사용량 실시간 추적 가능
- 게이트웨이를 통한 모든 요청/응답은 중앙화된 감시·모니터링/분석 도구와 연결, 엔드투엔드 사용 현황 추적

### MCP(도구), A2A(에이전트 간), 유즈케이스(활용사례) 3종 레지스트리로 모든 AI 자산을 중앙에서 통합 관리함

- MCP Registry: Amplifon 내 모든 자체/승인된 공개 MCP 서버(도구/통합 시스템) 카탈로그
- A2A Registry: Agent-Card 표준 기반, 배포된 모든 에이전트의 신원/엔드포인트/기능/인증/모듈리티 기록. 배포시, CI/CD로 자동 등록
- Use Case Registry: 위 2가지 도구/에이전트 맵핑, 각 유즈케이스별로 사용 자산/모델/배포위치/라인리지 및 상호 영향관계 데이터 통합
- 각 레지스트리는 메타데이터(책임팀, 환경, 인증방식, 비용, 연관 유즈케이스 등)까지 포함해 추적성과 영향도 분석 실현

### MCP Registry는 공식 커뮤니티 MCP 카탈로그를 확장하여 Amplifon만의 엔터프라이즈 요구를 반영함

- Amplifon은 외부 MCP 공식 레지스트리를 기반으로, 내부 사용 환경에 맞는 확장 메타데이터 및 기능 추가
- 사내 구축 커스텀 MCP 서버와 Amplifon 승인 공개 MCP 서버로 구분해 등록·관리
- 각 서버별 메타데이터: 소유팀, 환경(dev/test/prod), 인증모델, 비용 귀속, 활용 유즈케이스 등
- 모니터링/운영 관점에서 중요한 ‘영향도 분석’을 위해 필수 데이터로 삼고, 레지스트리 자체가 추적·감사 기능 내장

### A2A Registry에서는 배포시마다 Agent Card를 자동 갱신하여 모든 에이전트가 자가 문서화됨

- Agent Card는 에이전트 신원, 엔드포인트, 기능, 인증조건 등 서술
- 새로운 에이전트 배포시, CI/CD 연동을 통해 메타데이터가 자동 등록됨
- 다른 팀/에이전트도 중앙 레지스트리에서 신규/기존 에이전트 검색·재사용 가능
- 이를 통해 에이전트 간 정보 교환, 생태계 확산, 문서 자동화 실현

### Use Case Registry는 도구·에이전트·모델과의 관계 및 라이프사이클 등 비즈니스 활용 전반을 연결함

- 각 유즈케이스별로 사용 도구, 에이전트, 연관 모델, 배포 위치, 활용 내역, 영향 관계 등 포괄적 정보 저장
- 다수의 유즈케이스가 어떻게 상호 연결되고, 단일 장애가 어떤 연쇄 영향을 주는지 분석 가능
- 중앙 인터페이스에서 유즈케이스별 자산맵/라인리지 조회 기능 제공

### 실제 플랫폼 데모에서 주요 기능(카탈로그, 대시보드, 상세정보, 계보도 등) 구현 과정을 상세히 소개함

- 플랫폼 홈→카탈로그→MCP, A2A에이전트, 유즈케이스 정보로 한 눈에 데이터 연결
- 특정 유즈케이스 상세: 상태, 버전, 설명, 사용 에이전트/도구/AI 모델, 라이프사이클 히스토리 표시
- 신규 유즈케이스 생성 시 메타데이터·링크 자산·소유자 입력 및 관리 가능
- 도구/에이전트의 server.json, agent card 등도 자동 파싱·등록
- Inspector 기능으로 MCP나 A2A 에이전트의 기능/호환성 실시간 확인
- 위젯을 통해 초심자도 쉽게 JSON 규격 생성·미리보기 지원
- 계보(Lineage) 뷰를 통해 유즈케이스-에이전트-도구-모델 연관관계를 시각화, 장애 대응시 영향 경로 추적 가능

### 표준화된 템플릿(블루프린트)과 자동화된 개발/배포 파이프라인(CICD)으로 개발 생산성을 대폭 향상시킴

- MCP, A2A 프로토콜 각각에 맞는 GitHub 템플릿/레포지토리 제공
- FastAPI, Docker, 패키지 관리자 등 공통 인프라/툴링 내장→즉시 확장/배포 가능
- 인증/비용 관리·관측(LongFuse) 등 기본기능 내장→개발자는 비즈니스 로직에 집중 가능
- A2A 서버용 블루프린트는 프레임워크 불문(예: LangChain, Agno 등) 인터페이스 기반 설계
- 표준화된 인터페이스 제공으로, 자체 프레임워크와 무관하게 조직 전체가 동일 기준에서 개발/운영

### CICD와 Metadata Registration Flow 도입으로 배포-등록-추적 전 과정이 자동화됨

- 개발 완료 후 GitHub 브랜치 태깅→Action이 도커 이미지 생성/배포 및 메타데이터(에이전트 카드, server.json) 자동 등록 처리
- AI Gateway 및 MCP/A2A 프록시가 중앙 카탈로그로부터 도구/에이전트 백엔드 엔드포인트를 실시간 조회 및 인증 운용

### Amplify 플랫폼을 통해 조직 내 모든 AI 자산의 라이프사이클, 영향도, 연관성, 오류 복구 관점의 가시성과 관리체계를 완성함

- 카탈로그/라인리지 관리로 어느 도구·에이전트·모델이 어디에 사용되고 있는지, 실패시 연쇄 영향까지 한 눈에 파악 가능
- 표준화된 블루프린트, CICD로 빠른 개발/배포와 더불어 전사적인 일관성 확보
- 메타데이터 내장 운영체계 덕분에 거버넌스, 감사, 리스크 관리 체계를 중앙에서 견고하게 수행
- 플랫폼 기능이나 범위는 계속 확장 중임을 명시하고, 외부 협업/토론도 권장하며 발표 마무리
