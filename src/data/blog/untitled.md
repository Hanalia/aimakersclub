- **코더(coder) 플랫폼에서 제공하는 무료 오픈소스 툴 'Workspaces'를 활용해 구현**
- **클라우드 상에서 독립적으로 인스턴스가 생성되며, 개인 PC에서 어떤 소프트웨어도 실행할 필요 없음**
- **Workspaces는 AI 코딩 에이전트 세션을 대규모로 원격 오케스트레이션할 수 있도록 하는 인프라 제공**

- **간단히 GitHub 이슈에 ‘coder’ 태그를 붙이면 자동으로 프로세스가 시작되며, 진행상태 트래킹 링크도 제공됨**
- **동일 인프라에서 여러 건의 GitHub 이슈를 병렬 처리할 수 있어, 협업 및 대량 작업에 탁월**
- **모든 워크플로우가 격리된 환경에서 처리되므로, 팀 단위로도 효율적으로 인프라를 공유 및 관리 가능**
- **최종적으로 GitHub에 풀리퀘스트까지 자동 생성되어, 검토 및 머지만 진행하면 완료됨**
- **Workspaces는 직접 호스팅이 가능한 무료 오픈소스 툴로, 다양한 개발 환경에서 쉽게 적용 가능**

## 세부 요약 - 주제별 정리

### 사용자가 한 줄의 코드도 작성하지 않고 AI가 GitHub 이슈를 자동 해결함을 시연함

- 발표자는 GitHub 이슈에 대해 “한 줄의 코드도 작성하지 않았다”고 언급
- 모든 작업은 코딩 에이전트가 사용자의 컴퓨터가 아닌 별도의 환경에서 처리함
- 사용자는 단지 GitHub 이슈를 생성하고 ‘coder’ 태그만 붙이면 됨
- 이후 모든 코딩 작업, 풀리퀘스트 생성까지 자동화로 진행됨

### 클라우드에서 원격 인스턴스가 실시간으로 생성되어 작업을 담당함

- ‘coder’ 태그가 감지되면, 자동으로 독립된 인스턴스가 원격 환경에 생성
- AI 기반 코딩 작업(Cloud code)이 이 인스턴스 위에서 수행됨
- 사용자의 로컬(개인 PC)에서는 어떠한 소프트웨어나 프로세스도 동작하지 않음

### 무료 오픈소스 도구 ‘Workspaces’가 AI 코딩 에이전트 오케스트레이션을 담당함

- 이 모든 자동화 및 인프라 관리는 ‘Workspaces’(플랫폼: coder)라는 무료 오픈소스 툴을 통해 구현
- Workspaces는 AI 코딩 에이전트들이 효율적으로 오케스트레이션될 수 있도록 인프라·환경을 제공
- 여러 개의 코딩 세션을 병렬로 관리·운영 가능하게 함

### Workspaces는 인프라인 것이며, 코딩 에이전트 자체가 아님을 명확히 설명함

- Workspaces는 각 AI 코딩 에이전트의 실행 환경(인프라·컨테이너) 관리에 해당
- 코딩 에이전트(예: Claude Code)를 직접 수행하는 엔진이 아니라, 이를 여러 개 원격에서 손쉽게 관리·확장 가능케 하는 역할
- 예시로 VPS(가상 서버) 환경에서 워크스페이스를 배포해 직접 활용했다고 언급

### GitHub 이슈 ‘coder’ 태그 추가만으로 자동 프로세스가 촉발됨

- 사용자는 GitHub 이슈에 ‘coder’ 태그만 추가하면 워크플로우가 자동화되어 실행됨
- 태그가 감지되면, 배포된 인프라 상에서 해당 이슈 처리를 위한 작업(task)이 자동 할당됨
- Workspaces 관리 화면(task view)에서 새로운 작업(예: 3번째 task)이 실시간으로 등록되는 모습 시연

### 처리 진행상태는 댓글과 실시간 링크로 투명하게 공개됨

- 해당 task가 시작되면 GitHub 이슈 페이지에 작업 트래킹용 실시간 링크가 자동 댓글로 게재됨
- 문제 해결 과정은 클라우드 환경에서 진행되며, 사용자는 링크를 통해 진행상황 확인 가능

### AI 에이전트(Cloud code)가 클라우드 환경에서 이슈를 처리하고 풀리퀘스트까지 자동 생성함

- 클라우드 코드(agent)가 생성된 작업 환경(인스턴스)에서 코드 작성 및 수정, 문제 해결을 수행함
- 해당 이슈에 대한 처리 및 구현이 완료되면, GitHub에는 자동으로 풀리퀘스트(PR)가 생성됨
- 사용자는 최종적으로 PR을 확인하고 머지 여부만 결정하면 됨

### Workspaces는 복수 이슈의 병렬 작업 및 팀 단위 인프라 공유가 가능함을 강조함

- 각 작업(코딩 에이전트 세션)은 격리된 환경에서 독립적으로 실행
- 수십 개의 GitHub 이슈도 동시에 병렬로 처리 가능하여 생산성 대폭 향상
- 동일 인프라를 팀원 모두와 공유하며 효율적으로 관리·확장할 수 있음

### Workspaces와 도입 방식, 실사용 환경을 구체적으로 설명함

- Workspaces는 무료, 오픈소스, 자체 호스팅(VPS 등)에 용이하다고 강조
- Coder 플랫폼에서 공식적으로 배포하는 툴이며, 누구든 쉽게 시작할 수 있음
- 실제 예시 영상에서는 VPS에 직접 호스팅, GitHub 이슈와 연동해 전체 자동화 과정을 시연

### 영상 마무리에서는 강력한 생산성 향상 및 무료 툴의 시도 권장함

- Workspaces가 제공하는 병렬 작업 환경 및 자동화 인프라의 강점을 재차 강조
- 개인·팀 개발환경 모두에 무료로 적용 가능, 생산성 극대화
- 공식 설명 링크가 영상 설명란에 추가되어 있으니 참고 권장
