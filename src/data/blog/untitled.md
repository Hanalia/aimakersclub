## *Agent Harnesses가 바이브 코딩을 다시 불러오고 있는가?* 핵심 요약

- 최근 LLM(대형 언어 모델)의 원초적인 성능 향상 속도는 예전만큼 폭발적이지 않다는 냉엄한 현실을 강조함
- 대표적으로 Gemini 3, Claude Opus 4.5와 같은 모델이 꾸준히 벤치마크 갱신을 보이긴 하지만, 이는 점진적임
- 진정한 차별화 포인트는 LLM 위에 구축하는 "레이어", 즉 에이전트 및 래퍼가 되고 있기 때문에 주목해야 함
- 이런 상위 레이어 개발은 누구나 창의적으로 만들고 최적화할 수 있는 새로운 기회로 작용함
- 월드 모델(world model)처럼 LLM을 감싸는 구조나, 이번 영상 주제인 Agent Harness(에이전트 하니스)가 핵심 예시임
- Agent Harness 개발은 차세대 에이전트의 역량을 대폭 확장하는 "다음 단계의 잠금 해제"임을 주장
- 상상력이 필요하지만, Harness의 발전이 실제로 코딩 작업의 99% 이상을 AI 코딩 에이전트에 위임하는 기반이 될 것이라 확신함
- 장기적인 에이전트 운영에도 Harness가 적용되어 프로그래머의 역할이 근본적으로 변화할 것으로 전망함
- 2026년 경에는 바이브 코딩(vibe coding)이 현실 가능해지나, 그 방식은 시스템 및 Harness 엔지니어링 중심으로 완전히 달라질 것임을 예견함
- 이를 통해 인간 개발자는 대부분의 코드를 직접 작성하지 않고, Harness 설계에 집중하는 미래를 언급함

## 세부 요약 - 주제별 정리

### LLM의 성능 폭발이 둔화되며 새로운 변화가 필요해짐

- 지난 몇 년 동안 LLM은 괄목할 만한 기술적 도약을 보여왔으나, 최근에는 그 원초적인 성능 향상이 예전처럼 극적이지 않음
- "LLM의 원초적 파워가 더 이상 예전처럼 폭발하고 있지 않다"는 점을 냉정하게 인식해야 함을 강조함
- Gemini 3, Claude Opus 4.5 등 최신 LLM이 계속 벤치마크에서 발전 중이나, 변화의 폭은 점진적임
- 단순히 LLM의 '성능 경쟁'만으로는 앞으로의 혁신적 진전을 기대하기 어려운 상황임을 시사함

### LLM 위에 구축되는 에이전트와 래퍼의 중요성이 부각되고 있음

- 현재 진정한 경쟁력은 LLM 자체가 아니라, LLM 위에 어떤 레이어(layer)를 구축하고 그것을 어떻게 활용하느냐에 달림
- 즉, 우리가 직접 구축하고 최적화할 수 있는 상위 구조(예: 에이전트, 래퍼)가 창의성의 무대가 되고 있음
- 이러한 구조는 기업/개인에 의해 각기 다르게 개발될 수 있으므로, 차별화된 역량 확보의 핵심이 됨

### 월드 모델과 에이전트 하니스가 새로운 기회의 장을 열고 있음

- 예시로 '월드 모델(world model)' 같은 LLM을 감싸는 상위 구조가 거론되며, 별도의 영상으로 다룰 계획을 언급함
- 이번 영상에서는 특히 'Agent Harness(에이전트 하니스)'라는 새로운 래퍼 개념에 초점을 맞춤
- Agent Harness는 LLM과 외부 도구·시스템 간의 복잡한 상호작용을 통제하고, 기능을 대폭적으로 확장해주는 계층임
- Harness 개발이 차기 혁신의 중심 키가 될 것임을 암시함

### Agent Harness가 에이전트의 역량을 획기적으로 끌어올리게 됨

- Agent Harness를 LLM 위에 쌓아올림으로써 에이전트(즉, 자동화된 인공지능 작업자)의 기능 수준이 크게 향상됨
- 이 하니스 개발이 "에이전트의 다음 단계(capability unlock)로 가는 열쇠"임을 명확히 밝힘
- 기존 단순 에이전트와 달리, 하니스는 복잡한 작업 위임, 대규모 코드 생성, 멀티태스킹 등에서 차별적 이점을 제공함

### Harness의 가능성을 현실로 만들려면 상상력과 선제적 엔지니어링이 필요함

- 현 시점에서 Harness의 잠재력이 완전히 보이진 않지만, 상상력을 가지고 발전시키면 엄청난 가능성을 열 것이라고 함
- Harness를 통해 장기적으로 "코딩의 99% 이상을 AI 코딩 에이전트에 위임할 수 있다"는 구체적 전망 제시
- 에이전트 하니스는 단순 의사결정이나 반복 작업을 넘어서, 사실상 인간 대신 거의 모든 코딩을 처리하는 역할로 확장될 수 있음

### 장기 실행되는 에이전트에도 Harness 구조가 필수적으로 적용됨

- 영상 내내 언급되어온 바와 같이, 단기 작업뿐만 아니라 "long-running agent(장기 작동 에이전트)"에도 Harness가 중요한 역할을 하게 됨
- 이는 지속적으로 작동하는 백엔드 에이전트나 인프라 관리, 복합적인 프로젝트 관리에도 동일하게 적용됨
- Harness 구조를 통해 에이전트가 다양한 운영 환경에서 자율적으로 진화하고, 문제 해결 방식을 개발할 수 있게 됨

### 바이브 코딩이 다시 현실화되지만, 방식은 근본적으로 달라질 것임

- Harness 및 시스템 엔지니어링 기반의 개발 패러다임이 자리잡으면서 "바이브 코딩(vibe coding)"이 재등장할 것임을 예고함
- 다만 예전의 바이브 코딩과 달리, 대부분의 코드 작성은 에이전트가 담당하게 되고 인간은 Harness 설계 및 시스템 엔지니어링에 집중하는 방식으로 바뀔 것임
- 2026년 즈음에는 이러한 구조가 본격적으로 현실화될 것이라 전망함

### 미래에는 대부분의 코드를 인간이 쓰지 않고 Harness 엔지니어링만 담당하게 됨

- 머지않은 미래(2026년)에는 시스템 설계, Harness 구조 설계가 주요 프로그래밍 역할이 될 것임을 강조
- 프로그래머는 대부분의 저수준 코드 작성에서 벗어나, Harness 및 에이전트 구조를 설계/관리하는 역할로 이동하게 됨
- 요컨대, Harness의 등장이 코딩과 개발 방식에 근본적인 패러다임 변화를 가져올 것임
