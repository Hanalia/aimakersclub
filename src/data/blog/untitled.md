## *Glue.ai는 Slack이 지향했어야 할 차세대 AI/MCP 네이티브 채팅 플랫폼* 핵심 요약

- 영상 제작자는 매일 Slack을 사용하여 업무를 진행하지만, 점점 비효율적이고 산만해지는 점에 실망감을 표현함
- 최근 몇 일 전, Slack 내 중요한 버그 수정 쓰레드를 3일간 놓쳐 업무에 차질이 생겼던 경험을 공유함
- Slack에서 실제로 업무를 처리하려면 여러 앱을 오가야 하고, 이는 불편함과 번거로움을 유발함
- 이런 단점을 해결하기 위해 ‘Glue’를 테스트했으며, 이것이 Slack이 추구했어야 할 진화라고 평가함
- Glue는 업계 최초의 멀티플레이어 MCP(Multi-Context Platform) 기반 채팅 플랫폼으로, 실질적인 협업과 업무 수행을 지원하도록 설계되었음
- Glue의 AI는 채팅 쓰레드 내에 완전히 통합되어, 대화 맥락 전체를 인지하며 팀 전원이 효율적으로 활용할 수 있음
- MCP로 구동되어 다양한 협업 툴과 직접 연동: ASA에서 업무 업데이트, Notion에서 문서 읽기, GitHub 이슈 확인 등 모두 Glue 내에서 가능
- Glue는 35가지 주요 툴의 즉시 연결이 가능한 MCP 디렉토리를 갖췄으며, 맞춤 MCP 서버를 통해 수천 개의 추가 연동이 지원됨
- 영상은 Glue가 단순히 AI 기능을 추가한 Slack이 아니라, 본질부터 업무 생산성과 빠른 협업을 위해 만들어진 AI 네이티브 채팅 플랫폼임을 강조함
- 최신 AI 툴에 투자하면서도 팀 내 채팅 환경이 과거에 머무른 느낌이 든다면 Glue를 직접 사용해 볼 것을 추천함

## 세부 요약 - 주제별 정리

### Slack을 업무용 채팅 툴로 사용하면서 겪는 문제점이 누적되고 있음

- 영상 제작자는 본인의 비즈니스 운영에 Slack을 필수적으로 활용한다고 밝힘
- Slack의 대화 쓰레드가 점차 비효율적으로 관리되어 점점 산만해지는 경향이 강조됨
- 쓰레드 관리(“thread hygiene”)에 본인 책임도 있겠지만, 실제로 불편이 누적되는 상황을 고백함
- 예시로 며칠 전, 중요한 프로젝트 버그 수정 관련 주요 쓰레드를 3일간 놓쳐 업무 대응이 늦어진 사례를 언급함
- 이런 문제의 본질은 단순히 ‘알림 기능(notification problem)’에 국한되지 않음을 시사함

### Slack 기반 협업에서 앱 전환이 업무 효율을 크게 저해함

- Slack에서 실질적 업무 처리를 위해서는 여러 개의 외부 앱들 간 이동(“swapping between all my different apps”)이 필수적임을 강조
- 반복적 앱 전환으로 인해 업무 흐름이 끊기고, 실제 생산성 저하를 경험하게 됨
- 이러한 앱 간 ‘컨텍스트 스위칭’이 번거로움을 유발하는 핵심 원인임을 드러냄

### Glue는 Slack의 다음 세대로, MCP 기반 협업에 최적화되어 있다고 평가됨

- 제작자는 최근 ‘Glue’를 직접 사용해보고, Slack이 발전했어야 할 방향을 구현했다고 느꼈다고 밝힘
- Glue를 “업계 최초의 멀티플레이어 MCP(Multi-Context Platform) 기반 채팅 플랫폼”으로 규정
- Glue는 단순 메시징이 아닌, 실제로 팀별 실무를 ‘효율적으로 처리’하는 데 초점을 맞춘 채팅 플랫폼임을 강조

### Glue의 AI는 쓰레드 내부에 완벽히 통합되어 팀 모두에게 실시간 도움을 제공함

- Glue AI는 각각의 채팅 쓰레드 내부에 ‘살아 있는 것처럼’ 임베드되어 있음
- 단순 외부 챗봇 호출이 아니라, 대화의 전체 맥락(“full context”)을 실시간으로 인지함
- Glue AI는 팀 내 모든 구성원이 동시에 접속해 활용 가능한 구조임
- 이런 통합성을 통해 단편적 답변이나 기능 호출이 아니라, 복합적 문제 해결과 심도 깊은 협업이 가능해짐

### Glue는 MCP를 통해 다양한 협업 도구와 채팅 내에서 직접 연동하여 생산성을 비약적으로 높임

- Glue는 MCP(Multi-Context Platform) 엔진을 통해 사용자의 모든 업무 도구에 직접 접근이 가능함
- 예시로 ASA에서 업무(Task)를 업데이트, Notion 내 문서를 읽기, GitHub 이슈 확인 등 일반적으로 외부 앱에서만 가능했던 일들을 Glue 내에서 처리 가능함
- 이 모든 작업이 Glue를 벗어나지 않고 채팅 대화 내에서 즉시, 자연스럽게 이루어짐을 강조

### Glue는 업계 최대 수준의 in-app MCP 디렉토리와 방대한 연동성을 제공함

- Glue는 현재 35종의 주요 협업 툴을 버튼 한 번으로 바로 연결할 수 있는 MCP 디렉토리를 자체 내장
- 더 나아가 ‘커스텀 MCP 서버’를 구축하면 수천 개 이상의 다양한 업무 툴 연동까지 확장 가능함
- 이로 인해 업무 자동화와 협업 효율화에서 경쟁 툴 대비 현저한 이점을 제공함

### Glue는 기존 AI 플러그인 채팅이 아니라 본질적으로 AI 네이티브 채팅 플랫폼임을 내세움

- Glue가 Slack 등과 본질적으로 다른 점은, 단순히 AI를 후처리로 붙인 ‘AI 플러그인’이 아니라 플랫폼 설계 단계부터 AI·MCP에 최적화된 구조임을 강조
- AI와 MCP 통합 덕분에 실무 생산성, 협업 속도, 컨텍스트 스위칭 해소 등에서 높은 효과를 거둠
- AI 네이티브 플랫폼으로서 팀 기반 협업에 특화된 최신 사용자 경험을 실현함을 내세움

### Glue는 현재의 팀 채팅 경험이 여전히 과거에 머물러 있다면 적극 도입을 권장함

- 영상 마지막에서, “AI 툴을 도입하고 있지만 팀 채팅 환경은 2015년에 멈춰 있는 기분”이 든다면 Glue AI를 사용해볼 것을 촉구함
- Glue가 변화한 업무 환경에 꼭 맞는 차세대 팀 채팅 경험을 제공한다고 자신 있게 제안함
