## *나의 AI 메모리가 모든 도구에서 나를 따라다닌다!* 핵심 요약

- 영상은 Claude Code, Codex, Pi 등 다양한 AI 코딩 도구를 사용할 때 ‘개인화된 메모리 시스템’의 호환성 문제를 소개함
- 사용자는 기존에 Claude Code 내에만 국한되던 ‘세컨드 브레인’ 메모리 시스템의 한계를 느끼고 더 확장된 솔루션을 탐색함
- Walrus and Memory라는 탈중앙화, 암호화 기반 외부 메모리 시스템이 대안으로 제시됨
- Walrus는 사용자의 모든 AI 도구에서 동일한 메모리(지식, 정보, 경험)를 접근하고 활용할 수 있게 해줌
- Walrus의 시작 방법은 홈페이지에서 복사한 프롬프트 한 줄만으로 간단하게 설정 가능
- Walrus SDK로 다양한 앱과 도구에 손쉽게 통합할 수 있음
- MCP 서버를 통해 대용량 텍스트를 요약, 주요 정보 추출 및 저장, 나아가 검색까지 제공함
- Claude Code, Pi 등 서로 다른 AI 에이전트에서 동일한 질문으로 Walrus에 저장된 기억을 정확히 불러올 수 있음
- 모든 데이터는 암호화되어 사용자가 통제하며, 접근 권한 위임(Delegate Key) 등 세분화된 관리 기능이 제공됨
- Walrus는 설치와 사용이 무료로 매우 빠르고 간편하게 시작할 수 있다는 점이 강조됨

## 세부 요약 - 주제별 정리

### 자체 메모리 시스템은 한 도구 안에만 국한돼 확장성에 한계가 있었음

- Claude Code를 활용해 세컨드 브레인 형태의 개인화 메모리 시스템을 구축해 사용함
- 해당 메모리 시스템은 사용자가 진행 중인 프로젝트 및 개인 정보 등 모든 내용을 저장하고 관리하는 역할을 함
- 이 시스템 덕분에 생산성이 크게 향상됐으나, Claude Code에만 고정되어 있음에 불편을 느낌
- 최근 Codex 및 Pi 등 각기 다른 강점을 가진 여러 AI 코드 어시스턴트를 병행해서 사용하게 됨
- 각 도구에 따라 전문성이 다르고, 사용 목적에 따라 효율이 다름
- 그러나 이전에 구축한 메모리 시스템 때문에 여전히 Claude Code에 종속되는 문제 발생

### AI 메모리를 다양한 도구 간에 자유롭게 공유할 필요성이 대두됨

- Claude Code 전용 메모리 시스템의 한계 때문에, 도구를 바꿀 때마다 과거 정보와 맥락을 잃는 문제 발생
- 여러 도구와 프로젝트를 오가며 일하는 현대적 환경에서는 ‘메모리의 이동성’이 필수라고 판단
- 사용자는 ‘자신 및 비즈니스와 관련된 모든 정보’를 어떤 도구로도 옮겨 다닐 수 있는 방안을 모색함
- 오랜 기간 해답을 찾던 중, 외부 메모리 플랫폼 Walrus and Memory를 발견함

### Walrus and Memory는 탈중앙화 암호화 메모리 시스템을 제공함

- Walrus and Memory는 한 특정 코딩 도구나 회사에 의존하지 않는 외부 메모리 저장소임
- 사용자의 기억(데이터)은 탈중앙화, 암호화되어 오직 사용자 자신이 통제함
- 데이터는 별도의 기업 인프라가 아니라, 사용자의 권한 아래에 소유됨
- 접근 통제는 사용자 설정에 따라 가능하며, 각기 다른 시스템이나 사용자에게 Delegate Key(위임키)를 만들어 저장된 기억 사용 권한을 세분화시켜 부여할 수 있음

### Walrus는 단 하나의 프롬프트로 빠르게 시작할 수 있는 점이 매우 특징적임

- Walrus 홈페이지에서 제공하는 시작용 프롬프트를 복사해 AI 코딩 어시스턴트에게 보내는 것만으로 설치됨
- 별도의 복잡한 셋업, 회원가입 등이 불필요하여 학습 및 적용에 매우 용이함
- 무료 시작이 가능하다는 점도 강조됨

### Walrus SDK로 여러 앱과 도구에 메모리 시스템을 확장할 수 있음

- Walrus는 자체 SDK를 제공하여, 원하는 인공지능 도구나 개발 환경에 기억 시스템을 추가로 통합 가능
- 하나의 기억 저장소를 여러 도구, 여러 세션에서 동시에 활용하는 환경이 구현됨

### MCP 서버 연결로 저장, 요약, 검색 등 다양한 메모리 활용이 가능해짐

- Walrus 세팅 후 MCP 서버가 정상적으로 연결되었는지 화면에서 확인 가능
- 각종 AI 도구 내에서 ‘기억 저장 도구’와 ‘검색’, ‘대규모 텍스트 요약’ 등의 기능을 쓸 수 있음
- 대화형 사용뿐만 아니라, 필요에 따라 원문 텍스트를 붙여넣고 중요한 정보만 추출해 메모리화 가능

### 구체적 사용 예: 긴 텍스트의 정보 추출 및 암호화 저장 시연

- 예시로 긴 텍스트를 붙여넣고, “Walrus를 이용해 기억하라”고 명령함
- MCP 서버가 텍스트에서 핵심 정보를 자동 추출하여, 20개의 주요 팩트로 분리해 메모리에 저장
- 이 정보들은 암호화되어 저장되고, 어느 도구에서든 즉시 활용 가능

### Claude Code와 Pi 등 다른 AI 도구에서도 동일한 기억 접근과 활용이 즉각적으로 가능함을 보여줌

- 예를 들어, Claude Code 세션에서 “Walrus 기억에서 Dark Factory의 기술 스택을 알려달라”고 검색
- Walrus 메모리 MCP가 바로 연결되어 정확하고 상세한 답변이 반환됨
- 이어서 Pi 에이전트 세션에서도 같은 질문으로 Walrus 기억 접근 시, 조금 더 간결하지만 동일 정보의 응답을 즉시 얻음
- 각 도구에서 추가 설정 없이 MCP 서버만 연결하면 동일한 메모리 활용 가능

### 암호화 및 접근 제어 등 보안과 프라이버시 면에서도 탁월함

- 모든 기억 데이터는 암호화되어 있어 외부에 노출될 걱정이 적음
- 접근 권한 및 위임 키 생성이 가능해, 다양한 시스템이나 사용자에게 안전하게 메모리 공유 및 열람 권한 부여 가능
- 사용자 데이터 소유 및 통제권이 완전히 사용자에게 있음이 강조됨

### Walrus는 매우 빠르고 무료로 설치 가능한 범용 인공지능 메모리 계층임

- Walrus는 모든 AI 툴에서 쓸 수 있는 범용 메모리 레이어를 구현
- 링크와 프롬프트만으로 손쉽고 빠르게 설치 가능하며, 진입 장벽이 매우 낮음
- 무료이기 때문에 누구나 바로 사용해볼 수 있음을 재차 강조하며 영상이 마무리됨
