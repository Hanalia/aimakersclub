## *스택드 디프(누적 변경사항) 방식이 AI 코드 리뷰 효율을 혁신적으로 높임* 핵심 요약

- 영상 제목: *스택드 디프(누적 변경사항) 방식이 AI 코드 리뷰 효율을 혁신적으로 높임*
- AI를 통한 코드 리뷰는 매우 강력하며, 반드시 코드 검증 과정에 포함해야 한다고 강조함
- 전통적인 풀 리퀘스트(변경사항 요청) 방식은 승인을 기다려야 하므로 반복 과정이 비효율적임
- 스택드 디프(workflow)를 사용하면 작은 단위의 변경 사항을 계속 쌓아가며 큰 기능을 완성할 수 있음
- 각 단계의 변경마다 기다림 없이 작업을 이어갈 수 있어 개발 효율이 향상됨
- 스택드 디프는 AI 코드 리뷰에 최적화되어 각 단계별로 집중적이고 정밀한 피드백을 신속히 받을 수 있음
- Graphite라는 플랫폼은 스택드 디프 관리를 위한 CLI(명령줄 도구)와 AI 코드 보조 대시보드를 제공함
- 실제 Graphite의 데모를 따라가며, 첫 스택 디프 작성 및 일련의 누적 풀 리퀘스트 생성 과정을 소개
- 각 변경 사항은 별도 브랜치에 만들고 누적적으로 병합하며, 최종적으로 메인 브랜치에 통합
- AI가 각 단계별 변경점을 개별적으로 리뷰하여, 큰 규모의 PR 한 번에 검토할 때보다 훨씬 세분화된 결과를 얻을 수 있음
- Claude Code 등 AI 리더가 각 PR을 분석·리뷰하는 실제 예시도 시연
- Graphite 및 AI 활용으로 코드 리뷰와 병합 과정이 훨씬 직관적이고 생산적으로 진행됨을 구체적으로 시각적으로 보여줌

## 세부 요약 - 주제별 정리

### AI 코드 리뷰는 반드시 포함해야 할 강력한 검증 도구임이 강조됨

- AI가 코드 리뷰에 혁신적인 도움을 주고 있으며, 이는 필수적인 검증 절차로 자리 잡았음을 분명히 함
- AI 코드 리뷰는 코드의 오류 탐지, 개선점 제안, 코드 품질 유지에 핵심 역할을 담당
- 영상 제작자는 AI 코드 리뷰가 일상 작업에서 빠질 수 없는 존재라고 직접 언급

### 기존의 전통적 풀 리퀘스트 방식은 승인 지연과 비효율성을 내포함

- 전통적인 PR(Pull Request) 워크플로우는 변경사항 작성 → 승인 대기 → 반복의 구성을 가짐
- 승인 대기 과정에서 개발이 중단되는 병목 현상이 발생함
- 큰 기능 개발 시 한꺼번에 많은 코드 변경이 발생하여, 검토와 리뷰가 더욱 어렵고 시간 소모가 커짐

### 스택드 디프 방식은 작은 변경의 단계적 누적으로 개발 효율을 극대화함

- 스택드 디프(stack diffs)란, 기능 개발을 작은 단위로 쪼개 각 변경 사항을 순차적으로 누적하는 방식임
- 작은 단위의 변경(디프)을 쌓아가며 전반적인 큰 기능을 완성해 나감
- 각 단계 별로 독립적으로 작업을 계속 진행할 수 있으므로 승인 대기를 생략 가능
- 누적 방식이어서 이후 단계의 작업이 이전 단계에 그대로 이어짐

### 이 방식은 AI 코드 리뷰에서 각 단계별 집중 피드백을 가능하게 만듦

- 각 작은 디프마다 AI의 집중적이고 정밀한 코드 리뷰를 받을 수 있음
- 한 번에 전체 코드와 변경사항을 검토하는 대신, 각 단계별로 문제점을 찾고 해결
- 이전에는 확인하기 어렵던 미세한 논리 오류나 구조적 개선점을 조기에 발견 가능

### 스택드 디프 관리를 혁신적으로 지원해 주는 도구로 Graphite가 소개됨

- Graphite는 스택드 디프 방식의 코드를 관리해 주는 플랫폼임
- CLI(명령줄 인터페이스) 도구를 제공하여 GitHub CLI와 유사하게 사용 가능
- 각 저장소(레포지토리)에서 누적 변경 사항을 효과적으로 관리 가능

### Graphite 데모를 통한 스택드 디프 실제 워크플로우 경험이 구체적으로 언급됨

- Graphite 플랫폼 가입 후 제공되는 데모에서 첫 스택드 디프 생성 과정을 자세히 안내받았음을 언급
- 사용자는 첫 변경 사항을 만들어 PR 생성 → 추가 PR 단계로 누적적인 스택드 디프 작성 흐름을 경험
- 명령줄에서 3개의 PR을 누적적으로 만들어가는 실제 화면도 제공
- 각 변경 사항에 대한 검토, 병합(submit), 관리 과정을 대시보드에서 모두 시각적으로 확인

### Graphite 대시보드는 풀 리퀘스트 생성과 관리, AI 리뷰 모두를 한 곳에서 지원함

- 각 풀 리퀘스트 상태를 한번에 대시보드에서 확인 가능
- Graphite Agent라는 AI 코드 리뷰어 기능 내장
- 모든 누적 PR을 대시보드 내에서 병합 가능하여, GitHub 사이트 방문이 필요 없음
- 각 변경 사항이 일반적인 Git 흐름(브랜치/병합)과 동일하게 처리됨을 설명

### 각 단계별 브랜치 관리와 메인 브랜치로의 최종 통합이 직관적으로 이루어짐

- 작은 변화마다 별도의 기능 브랜치 생성 → 브랜치 간 병합을 반복적으로 수행
- 모든 변경 사항 확정 후에는 최종적으로 메인 브랜치에 병합(fetch)됨
- 각 변경 사항마다 독립적으로 AI 리뷰가 들어가 코드 품질이 향상됨

### 실전 예시로 Claude Code가 각 PR을 개별적으로 리뷰하는 과정이 제시됨

- 영상에서는 실제로 Claude Code라는 AI 코드 분석기가 3개의 누적 PR 각각을 검토하는 장면 제시
- 각 PR을 분리해서 리뷰하므로, 대규모 단일 PR을 한 번에 검토하는 것보다 효과적인 피드백이 가능
- 이 과정을 통해 전반적인 코드 품질과 팀 협업 효율이 극적으로 증가함을 강조

### 대규모 단일 PR에 비해 스택드 디프가 리뷰 효율과 품질에서 월등함이 실제로 입증됨

- 전체 큰 PR 하나로 통합해 리뷰하는 것과, 각 변화마다 PR을 분리해 리뷰하는 것의 결과 차이가 크다고 직접 비교
- 작은 PR마다 AI의 집중 리뷰 결과, 직관적이며 체계적인 피드백과 향상된 코드 통합 과정이 가능
- 영상 끝에서 Graphite 플랫폼과 AI 코드 리뷰의 결합이 코드 리뷰의 새로운 방향임을 보여주고 마무리
