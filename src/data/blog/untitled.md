## *Claude Code로 아름다운 다이어그램을 만드는 전체 워크플로우* 핵심 요약

- Claude Code와 같은 코드 에이전트는 기본적으로 시각적 결과를 제공하는 데 한계가 있음
- 사용자가 복잡한 프롬프트와 추가 기능 없이 시각적 설명을 원할 때, 기존 코드 에이전트의 성능은 만족스럽지 않음
- 이를 개선하기 위해, 발표자는 자신의 전체 다이어그램 생성 및 검증 워크플로우를 "기술(skill)" 형태로 패키징함
- 이 기술을 적용하면 Claude Code와 같은 에이전트가 Excal(Excalra) 다이어그램의 JSON 파일을 생성할 수 있음
- 생성된 다이어그램은 자동으로 렌더링되고, PNG로 스크린샷을 찍어 시각적으로 결과를 검증함
- 에이전트는 효율적인 자기 검증 과정을 거쳐, 다이어그램의 미흡한 부분을 반복적으로 개선함
- 사용자는 ‘Excalra 다이어그램 스킬’ 설치를 코드 에이전트에게 요청할 수 있고, 에이전트가 README를 참고해 자동 설정을 진행함
- 다이어그램의 색상, 구조 등 사용자 브랜드 맞춤형 커스터마이징도 쉽게 구현됨
- ‘시각적으로 논증하는 방법’을 프롬프트와 기술에 포함하여, 단순 블록 조합이 아닌 구조적이고 의미 있는 다이어그램 생성 가능
- 발표자는 해당 저장소를 클론하고 기술을 시도해볼 것을 권장하며, PDF나 코드베이스 등 다양한 영역에 적용 가능함
- 이 워크플로우는 매주 여러 시간의 반복 작업을 절감시키고, 생산적인 출발점을 제공함

## 세부 요약 - 주제별 정리

### 기존 코드 에이전트는 시각적 설명에 한계를 보임

- Claude Code 등 대부분의 코드 에이전트는 텍스트 분석과 코드 생성에는 강하지만, 시각적 설명 및 시각 자료 제작에는 약점이 있음
- 사용자가 복잡한 시각 자료를 요청할 경우, 추가 지시 및 복잡한 프롬프트 없이는 만족스러운 결과를 얻기 어려움
- 이러한 한계가 있을 때, 사용자는 직접 많은 노력을 들여야 하며 자연스러운 다이어그램 제작이 저해됨

### 워크플로우 전체를 기술(skill)화하여 코드 에이전트 능력을 확장함

- 발표자는 본인의 다이어그램 제작 과정을 하나의 기술(skill)로 패키징함
- 이 skill은 Claude Code 뿐 아니라, 다른 다양한 코드 에이전트와도 호환 가능하도록 설계됨
- 사용자는 이 skill을 통해, 코드 에이전트로부터 직관적이고 실용적인 다이어그램을 손쉽게 제작 가능

### 코드 에이전트에 시각적 논증(visual argument) 능력을 부여함

- 핵심은 코드 에이전트가 단순히 정보를 블록 형태로 나열하는 대신, 구조적이고 논리적으로 시각화하도록 유도하는 점임
- Skill에서 제공하는 'primary instruction set'을 통해, Claude Code가 어떻게 시각적으로 논증(visual argument)할지 구체적으로 가이드함
- 결과적으로 다이어그램 생성이 단순 블록 나열에서 벗어나 더 설득력있고 이해를 돕는 구조가 됨

### Excal(Excalra) 다이어그램을 자동 생성·검증하는 전 과정을 소개함

- 에이전트는 매뉴얼 없이도 Excalra 다이어그램의 JSON 파일을 작성
- 작성된 다이어그램은 자동으로 렌더링되어 PNG로 변환되고, 생성 즉시 결과물을 시각적으로 검증함
- 에이전트가 다이어그램을 직접 검사하고, PNG 이미지를 분석하여 부족한 부분을 반복적으로 개선함
- 이 과정에서 “self-validation”(자체 검증) 기능이 강력하게 작동함

### 다이어그램의 시각 품질 향상 및 반복 개선 과정이 강조됨

- 본 워크플로우의 차별점은, 에이전트가 생성–검증–수정 과정을 완전 자동화한다는 것임
- PNG 생성 후 실제 시각적 품질을 직접 체크해서, 미흡한 점을 스스로 수정
- 반복 개선 과정 덕분에, 코드 에이전트도 전문 디자이너가 그린 듯한 수준 높은 다이어그램을 생성 가능

### Excalra 다이어그램 기술(skill) 설치·셋업이 자동화됨

- 사용자는 코드 에이전트에게 간단히 “Excalra 다이어그램 스킬 설정을 도와줘”라고 요청 가능
- Claude Code 등 에이전트는 skill의 README 파일을 읽고, 필요한 모든 수동 절차를 자동으로 진행함
- 복잡한 환경설정 없이 곧바로 다이어그램 제작에 착수할 수 있도록 지원됨

### 개개인 맞춤 스타일 및 브랜드 적용이 손쉽게 이루어짐

- 사용자는 다이어그램 색상, 브랜드 스타일 등 선호하는 스타일을 손쉽게 적용할 수 있음
- 색상표, 구조, 라벨링 등 다양한 커스터마이징 옵션 지원
- 자신만의 브랜드 색상이나 스타일과 일관성 있게 다이어그램 출력을 유지 가능

### ‘시각적 논증’ 프롬프트 설계로 구조적이고 의미 있는 다이어그램이 탄생함

- skill의 core philosophy는 ‘시각적으로 논증하는 방법’을 코드 에이전트에게 훈련시키는 것임
- 단순히 박스를 나열하는 형태가 아니라, 각 요소의 구조와 라벨을 통해 전체 컨셉을 명확히 설명하도록 유도
- 예를 들어, 단순한 블록 다이어그램이 아니라 흐름, 그룹핑, 연결선 등 전체 맥락 이해를 돕는 설계가 가능

### 저장소(repo) 활용 및 다양한 활용처 예시가 제시됨

- 발표자는 Github 저장소를 클론해 skill을 도입해볼 것을 권장함
- 단순한 도식뿐만 아니라 PDF 요약, 코드베이스 맵핑 등 실용적이고 응용 가능한 케이스도 제시
- 사용자는 원하는 다양한 데이터나 문서에 쉽게 적용 및 변환 가능

### 생산성 향상 및 워크플로우 반복 작업 절감 효과가 입증됨

- 해당 워크플로우와 skill을 통해, 매주 여러 시간의 반복 작업이 절약됨
- 코드 에이전트가 효율적인 시각화 출발점을 제공하여, 사용자의 작업 시간이 대폭 단축됨
- 단순 예시 제공이 아니라 실제 업무 프로세스의 효율화가 이루어졌음을 강조함
