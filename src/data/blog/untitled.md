## *AI 에이전트를 초강화하는 가장 간단한 방법(스킬)* 핵심 요약

- 이 영상은 Anthropic이 최근 공개한 ‘스킬(Skills)’ 기능이 AI 에이전트 발전에 어떻게 중요한 변화를 일으켰는지 설명함
- Anthropic의 스킬은 그 간결함 덕분에 AI 시스템 설계에서 “더 단순할수록 더 낫다”는 모토를 잘 구현함
- AI 에이전트에 다양한 기능을 추가하는 것은 중요하지만, 너무 많은 정보를 한 번에 주면 컨텍스트 윈도우를 과부하시켜 오히려 성능이 떨어짐
- MCP 서버 등 기존 방법은 에이전트에게 수많은 툴들을 한 번에 제공하여, 실제 필요하지 않은 기능도 모두 로딩하게 만드는 비효율 문제를 가짐
- Anthropic 스킬의 핵심 혁신은 ‘점진적 공개(Progressive Disclosure)’로, 기능을 한 번에 모두 주지 않고 실제 필요할 때만 정보를 단계적으로 공개하도록 설계함
- 예를 들어, 시스템 프롬프트에는 단순히 “이런 기능(PDF 처리 등)이 가능하다”는 설명만 포함되며, 실제 요청이 들어올 때만 구체적 지침(skill.md)이 열람됨
- skill.md 파일은 스킬의 정확한 사용법 전체를 담고 있으며, 추가 설명이 필요한 문서는 추가 단계에서만 로딩되어 과도한 컨텍스트 낭비를 방지함
- 이러한 계층적 공개 방식은 작업 상황에 맞게 점진적으로 정보가 AI에게 노출되므로, 에이전트가 효율적으로 기능을 활용하면서도 문맥 붕괴나 리소스 낭비를 최소화함

## 세부 요약 - 주제별 정리

### Anthropic 스킬이 최근 AI 분야에서 중요한 진전을 보여줌

- Anthropic이 새롭게 선보인 ‘스킬’ 기능은 AI 발전 흐름에서 매우 의미 있는 진보로 간주됨
- 영상에서는 “최근 가장 중요한 AI의 발전 중 하나”라고 언급
- 이 기술이 주목받는 가장 큰 이유는 그 구현 방식이 매우 단순하면서도 강력하다는 점에 있음

### 단순함이 주는 강력함: Anthropic의 설계 철학이 구현됨

- Anthropic의 모토는 “더 단순할수록 더 낫다(The simpler the better)”임을 강조
- 복잡하고 많은 툴을 한 번에 쓰기보다, 간결하면서도 목적에 부합하는 기능 추가를 지향함
- 영상 속 설명자는 이 단순함이 실제 에이전트 강화에 핵심적 역할을 한다고 말함

### AI 에이전트에 과도한 정보 제공이 문제를 초래함

- 에이전트에 여러 기능(툴), 대화 이력, 시스템 프롬프트 등 너무 많은 정보를 한 번에 제공하면 ‘컨텍스트 윈도우’가 금방 가득 참
- 에이전트가 크게 혼란을 겪거나, 맥락이 훼손되어 답변 품질이 저하되는 문제 발생
- 이는 모든 정보를 미리 한 번에 제공하는 기존 툴 설계 방식의 한계로 지적됨

### 기존 MCP 서버 방식의 비효율성과 한계가 지적됨

- MCP 서버 같은 기존 구조에서는 다양한 툴을 사용하지 않아도 ‘미리’ 모두 에이전트에게 할당
- 실제 대화나 작업에서 필요하지 않은 툴까지 항상 로딩됨으로써, 리소스 낭비 및 연산 성능 저하를 야기함
- 이런 접근 방식은 “에이전트가 필요 없는 툴까지 모두 알아야 한다”는 비효율로 이어짐

### 점진적 공개(Progressive Disclosure) 방식이 도입됨

- Anthropic 스킬의 강점은 사용에 ‘점진적 공개’를 도입한 것임
- 에이전트에 필요한 모든 도구와 설명을 한 번에 제공하지 않고, 필요할 때마다 단계적으로 정보가 공개됨
- 툴 설명 → 상세 지침 → 추가 문서 등 한 단계씩 정보를 에이전트에게 제공해, 컨텍스트 과부하를 방지함

### 시스템 프롬프트에 ‘스킬 설명만’ 제공하는 1단계 공개 방식

- 최초 시스템 프롬프트에는 오직 스킬의 간단한 설명만 포함됨
- 예시로, “이 에이전트는 PDF 처리가 가능하다”라는 식의 문장이 들어감
- 사용자가 그 기능(PDF 처리 등)을 실제로 요청하지 않으면, 별도의 상세 문서나 설명은 로딩되지 않음

### 실제 작업 요청이 들어올 때에만 skill.md 파일이 열람됨

- 사용자가 PDF 처리 등 특정 스킬을 직접적으로 요청하는 순간, 에이전트는 skill.md 파일을 참고함
- skill.md 파일에는 해당 스킬(기능)에 대한 전체 사용법과 구체적인 안내가 담겨 있음
- 이 방식 덕분에, 시스템은 필요할 때만 ‘2단계 정보’를 AI에 제공함

### skill.md 내부 참고 자료와 파일이 추가로 로딩되는 3단계 공개

- 어떤 작업에서는 skill.md 안에 또 다른 참고 문서가 포함되어 단계적으로 정보가 추가됨
- 예를 들어, 난이도 높은 작업이나 복잡한 처리에는 skill.md에 명시된 별도의 서브 문서가 참고될 수 있음
- 이를 통해, 정말 필요한 상황에만 ‘최대한의 맥락 정보’가 AI에게 주어짐

### 계층적(단계적) 정보 제공 방식이 컨텍스트 효율성과 에이전트 강화에 기여함

- 1단계: 스킬에 관한 간단한 설명만 초기에 제공
- 2단계: 실제구현 필요 시 skill.md 파일의 상세 지침 로딩
- 3단계: 더 복잡한 작업일 때 추가 참고 문서까지 순차적으로 로딩
- 이런 계층적 방식 덕분에, 에이전트는 작업 흐름에 맞춰 필요한 정보만 점진적으로 습득하며, 불필요한 정보로 인한 성능 저하나 문맥 붕괴가 방지됨

### 모든 예시와 논리는 Anthropic 공식 스킬 및 커스텀 구현에도 동일하게 적용됨

- 영상의 예시는 Anthropic에서 공식으로 제공하는 skill.md 파일 구조뿐 아니라, 커스텀 스킬 구현에도 적용됨
- Anthropic 생태계의 모든 스킬 활용에서 이러한 3단계 공개 방식이 유지됨
- 설명자는 향후 자신의 커스텀 스킬 구현 사례에서도 동일 원리를 적용한다고 언급
