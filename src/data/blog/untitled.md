## *AI 에이전트를 슈퍼차지하는 가장 간단한 방법: 스킬(기능)* 핵심 요약

- Anthropic은 최근 ‘스킬(Skills)’ 기능을 공개하며 AI 발전의 중요한 전기를 마련함
- ‘단순성이 곧 강점’이라는 Anthropic의 철학에 따라 설계된 스킬은 사용이 간편하면서도 강력함
- 에이전트에게 다양한 기능을 부여해 역량을 극대화하면서도, 정보 과부하 문제를 해결하는 것이 핵심 목표임
- 기존 MCP 서버 방식은 많은 툴을 한 번에 부여해 에이전트의 컨텍스트 창을 복잡하게 만들고, 실제로 필요하지 않은 도구까지 미리 로딩되는 단점이 있음
- ‘점진적 공개(Progressive Disclosure)’ 원칙을 통해, 스킬은 사용자가 필요로 할 때만 단계적으로 관련 정보를 보여줌
- 초기 시스템 프롬프트에는 해당 기능의 간략한 설명만 제공, 불필요한 맥락 로드 방지
- 예시로 ‘PDF 처리 스킬’의 경우, 최초에는 기능 설명만 알리고, 실제 요청이 있을 때만 상세 지침(skill.md)을 추가로 읽음
- skill.md 파일에는 각 기능별 실행 세부 방법과 추가 참조 문서가 구성되어 있어, 점차적으로 필요한 맥락만을 제공함
- 1단계: 기능 설명 → 2단계: skill.md 로 세부 안내 → 3단계: 추가 문서 참조 방식으로 과부하 없이 성능을 극대화함
- 이 구조는 에이전트가 실제 상황에 맞춰 필요한 정보만 효율적으로 활용하도록 돕는다

## 세부 요약 - 주제별 정리

### Anthropic이 ‘스킬(Skills)’ 기능을 도입하며 AI 발전의 단순성에 집중함

- Anthropic은 최근 AI 에이전트에 ‘스킬’이라는 새로운 기능을 추가하여, 자연어 처리 에이전트의 역량을 대폭 향상시킴
- Cole Medin은 이를 “최근 AI 분야에서 가장 중요한 발전 중 하나”라고 평가함
- Anthropic은 ‘더 단순할수록 더 좋다(The simpler the better)’라는 모토를 내세움
- 복잡한 설정이나 과도한 컨텍스트 정보 없이도, 실질적으로 필요한 기능을 효과적으로 사용할 수 있게 설계됨

### 에이전트에 다양한 역량을 부여할 때 겪는 ‘맥락(컨텍스트) 과부하’ 문제를 해결함

- AI 에이전트에게 기능을 많이 줄수록 기존에는 그만큼 많은 도구 설명, 대화 이력, 시스템 프롬프트 등을 한 번에 컨텍스트 창에 넣어야 했음
- 모든 기능 설명을 미리 로딩하면, 에이전트가 오히려 혼란스럽거나 맥락 창이 불필요하게 빨리 차는 문제가 발생함
- “에이전트는 많은 정보(툴, 시스템 프롬프트, 히스토리 등)를 한 번에 받으면 쉽게 압도된다”는 점을 강조

### 기존 MCP 서버 방식은 비효율적인 도구 과다 제공 문제를 가짐

- MCP 서버 등 기존 방식은, 실제 대화에서 필요하지 않더라도 미리 다수의 도구를 에이전트에 제공함
- 이는 어떤 대화에서는 사용되지도 않을 도구들이 컨텍스트 창을 불필요하게 차지하는 문제로 이어짐
- 비효율적이고, 에이전트의 성능 저하 및 메모리 낭비로 연결될 수 있음
- 요약하면, “실제로 필요 없는 기능까지 미리 제공하는 것은 바람직하지 않음”을 지적

### ‘점진적 공개(Progressive Disclosure)’ 원칙으로 불필요한 정보를 단계적으로 차단함

- 스킬 시스템의 핵심은, 에이전트가 모든 도구 정보를 시작부터 갖지 않고 ‘필요할 때’ 점진적으로 정보를 받아들이도록 설계됐다는 것
- “점진적 공개”란, 사용자의 실제 요청에 따라 에이전트가 관련 능력을 순차적으로 발견하며 활성화하는 방식
- 예: 초기엔 단순히 ‘이런 기능이 있다’고 알려주고, 필요할 때만 단계별로 상세 문서를 로딩함

### 시스템 프롬프트에는 ‘기능 설명’만 미리 제공해 초기 컨텍스트를 최적화함

- 시스템 프롬프트(에이전트가 참고하는 기본 안내)에는 각 도구/스킬의 간략한 ‘설명’만 삽입함
- 예시: PDF 처리 스킬의 경우 “PDF 관련 요청이 있을 때 사용할 수 있는 기능이 있다”는 설명만 등록
- 이를 통해 모든 세부 문서를 미리 불러오는 방식을 탈피, 비효율을 제거함

### 에이전트가 실제로 기능을 활용할 때만 skill.md 등 상세 문서를 단계별로 불러옴

- 사용자가 실제로 특정 작업(예: PDF 처리)을 요청하면, 에이전트가 해당 기능의 skill.md 파일을 읽음
- skill.md는 Anthropic의 모든 스킬이 따르는 표준 형식으로, 실제 사용법, 처리 방법, 중요 안내사항 등 상세 정보를 담고 있음
- 한마디로, ‘필요 시점’에만 상세 맥락이 컨텍스트로 로딩됨

### 모든 스킬은 세 단계의 정보 공개 레이어로 설계되어, 정보 과부하를 효과적으로 관리함

- 1단계: 기본 시스템 프롬프트에 ‘스킬 설명’만 미리 제공
- 2단계: 실제 요청 시 ‘skill.md’ 파일을 불러와 상세 맥락 채움
- 3단계: skill.md에서 또 필요한 경우 ‘추가 참조 문서’(예: 기술적 문서, 데이터 시트 등)로 연결
- 점진적으로 정보의 범위와 세부사항이 확장되는 구조임

### 점진적 정보 공개 방식이 실제 태스크 수행 효율성과 확장성 모두를 동시에 확보함

- 실제로 과부하나 성능 저하 없이, 에이전트가 점차 필요한 정보만 습득하므로 효율적임
- 사용자가 어떤 요청을 하느냐에 따라 컨텍스트가 ‘맞춤형’으로 제공되어, 유연하게 새 스킬 도입도 가능
- 이 구조가 Anthropic이 지향하는 꾸준한 ‘단순성과 효율성’의 구현 방식임

### PDF 처리 스킬 구현 사례를 통해 세부 동작과 장점을 구체적으로 설명함

- 예제에서는 PDF 관련 요청을 받는 상황에서, “PDF 처리 기능이 있다”는 1단계 설명 제공
- 사용자가 실제 PDF 처리를 요구하면, 2단계에서 skill.md 로 세부 처리 방법이 주어짐
- 필요하다면 skill.md가 다른 문서(3단계)로 안내하여 복잡한 로직, 외부 도구 연결 등도 지원
- 결국 에이전트는 가장 단순한 상태에서 출발해, 사용 요청이 있을 때마다 점진적으로 똑똑해짐

### Anthropic의 스킬 시스템은 실용성과 단순함을 동시에 추구하며 AI 개발의 생산성을 대폭 높임

- 강력한 기능이라도 ‘필요할 때만’ 로딩해 오버헤드 최소화
- 에이전트 개발자는 스킬 구조를 통해 다양한 기능을 유연하게 추가 및 관리 가능
- 궁극적으로, “단순하지만 강력한 구조”를 통해 AI 에이전트의 생산성, 확장성, 사용성을 모두 향상시킴
