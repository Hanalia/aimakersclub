## *클로드에게 24시간 쉬지 않고 코딩을 시켰더니 생긴 일* 핵심 요약

- 영상의 주제는 Claude Opus 4.5 모델을 이용해 24시간 동안 자동으로 코딩을 시켜 실제 서비스 수준의 소프트웨어를 만들 수 있는지 실험한 과정과 결과임
- 제작자는 거의 손대지 않고 자동화된 루프를 통해 애플리케이션의 주요 기능들을 구현하는 데 성공했다고 강조함
- 최종 산출물은 claw.ai의 완전한 복제판에 가까울 정도로 완성도가 높았으며, 실제 용도를 고려한 다양한 기능이 포함됨
- 과거 대화 기록을 순차적으로 검증하면서 생성된 콘텐츠를 다시 확인할 수 있는 기능이 구현됨
- 마크다운 형식으로 깔끔하게 출력되는 보기 좋은 인터페이스와 여러 개의 HTML 페이지 생성, 코드 작성 및 실행 기능까지 추가됨
- 사용자가 테마와 기본 AI 모델, 최대 토큰 슬라이더 등 다양한 설정을 직접 조정할 수 있는 UI가 탑재됨
- “엄청나게 풍부한 기능”이라고 평가될 만큼 기능 면에서 부족함이 거의 없음을 언급함
- 아직 ‘휴먼 인 더 루프’(human in the loop, 즉 중간에 사람이 검증)에 대한 부분은 더 보완하고 싶다고 밝힘
- 24시간이 아닌 더 오랜 시간 자동 실행한다면 실제로 완벽에 가까운 제품까지 만들 수 있을 것이라는 가능성을 제시함
- Claude Opus 4.5와 같은 대규모 언어모델, 그리고 이를 자유롭게 검증·반복할 수 있는 런타임 환경이 혁신적 개발을 가능하게 함을 실증함

## 세부 요약 - 주제별 정리

### 클로드 Opus 4.5의 자율 코딩 실험으로 손대지 않고도 거의 완전한 서비스를 구현함

- 제작자는 Claude Opus 4.5를 활용해 24시간 동안 인간의 개입 없이 지속적으로 코딩을 시도함
- 자동화된 루프(loops)를 통해 코드의 각 단계별 결과를 스스로 검증하고, 문제 발견 시 반복 개선 과정을 진행함
- 실험 결과, claw.ai의 기능을 대부분 복제한 “완전히 동작하는 클론”에 가까운 결과물을 얻었음
- 제작자 스스로도 기대 이상이라고 언급, 마지막 테스트 절반의 용도마저 의문스러울 정도로 완성도를 보여줌

### 자동 생성된 전체 대화 이력과 검증 결과를 모두 확인할 수 있는 기능이 탑재됨

- 개발 과정 중 Claude가 생성한 모든 대화 및 검증(verification) 과정을 히스토리 형태로 열람 가능하게 구현
- 각 단위 기능·루프 별로 어떤 메시지가 오고 갔는지, 어떻게 실패를 인지하고 고쳤는지 투명하게 확인 가능
- 인간 개발자가 직접 손대지 않고도 작업 전과정을 추적할 수 있다는 점이 강조됨

### 마크다운 기반의 깔끔한 인터페이스와 다양한 기능성 페이지가 자동 생성됨

- 코드 결과물은 마크다운(markdown) 포맷으로 보기 쉽고 정갈하게 나타나 사용자 편의성이 높음
- 다양한 HTML 페이지가 자동으로 생성되어, 각 기능별로 독립적인 UI 환경이 구축됨
- 코드 작성 및 실행 기능까지 함께 구현돼 실제 개발 및 결과 확인의 즉각성이 극대화됨

### 클로드를 통해 사용자 맞춤 설정 (테마, 모델, 토큰 등)을 지원하는 고급 설정 UI가 만들어짐

- 사용자 환경설정(Settings) 메뉴가 추가되어, 직접 테마를 바꾸거나 기본 모델 선택, 최대 토큰 갯수조절 슬라이더 기능 등이 포함됨
- 이런 설정 UI를 Claude 자체 코드 생성 루프 만으로 자동 구현했다는 점이 주요 포인트임

### 인간의 개입이나 추가 보완 없이도 풍부한 기능성을 확보할 수 있었음

- 제작자는 “so so featurerich(정말 기능이 넘친다)”라고 표현하며, 자동 생성 소프트웨어의 다채롭고 수준 높은 기능성을 강조함
- 기존에 수작업을 요하던 많은 UI 및 서버 코드까지 AI가 알아서 만들어내는 모습을 보여줌

### 인간(휴먼 인 더 루프)의 개입을 추가하면 안정성을 더 높일 수 있음을 시사함

- 아직 100% 자동화가 아닌, 중간에 사람이 검증하거나 승인해주는 ‘휴먼 인 더 루프’ 기능은 별도로 도입하고 싶다고 밝힘
- 이 부분을 제외하면 이미 대부분의 요구사항이 충족된 상태임을 시사함

### 하루가 아닌 더 긴 실행 시간을 주면 완벽한 제품에 근접할 가능성이 높게 나타남

- 실험 결과를 바탕으로, 하루가 아닌 며칠만 더 자동 코딩을 진행한다면 완벽에 가까운 애플리케이션이 탄생할 수도 있다고 예상함
- 기능 추가, 반복 개선, 디테일 보완 등 거의 모든 개발과정을 AI가 스스로 해결할 수 있음을 보여줌

### Claude Opus 4.5와 같은 대규모 언어모델 및 반복 검증 환경이 혁신적 소프트웨어 개발에 큰 역할을 함

- Claude Opus 4.5의 자체적인 코드 생성 역량, 복잡한 로직 처리, 에러 감지 및 수정 루프의 효율성 등을 언급
- 자유롭게 검증, 반복시킬 수 있는 “하니스(harness)” 환경의 중요성을 함께 강조
- 짧은 실험 만으로도 AI 기반 개발 방법론의 현주소와 가능성을 직접 입증함
