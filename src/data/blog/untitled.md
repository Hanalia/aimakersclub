## *AI 에이전트를 초강력화하는 가장 간단한 방법: Skills* 핵심 요약

- **Anthropic 사가 공개한 ‘Skills’ 기능은 최근 AI 분야에서 가장 중요한 발전 중 하나로 평가됨**
- **‘Skills’의 강점은 복잡성을 줄이고 단순함을 극대화하는 데 있음**
- **AI 에이전트에 다양한 능력을 부여하되, 에이전트가 받을 정보량(컨텍스트 윈도우) 과부하를 피하는 것이 핵심 목표임**
- **이전 방식(MCP 서버 등)은 처음부터 수많은 도구와 정보를 에이전트에 일괄 제공해 비효율과 혼란을 초래**
- **Skills는 ‘점진적 공개(Progressive Disclosure)’ 방식을 채택해 에이전트가 필요에 따라 점진적으로 능력을 탐색하도록 설계됨**
- **시스템 프롬프트에선 해당 스킬의 간단한 ‘설명’만 제시하고, 실제 활용 시에만 자세한 가이드(skill.md)가 컨텍스트에 추가됨**
- **skill.md 파일이 각 스킬의 구체적 동작법과 추가 참고 자료를 단계적으로 제공(2~3단계 점진적 공개)**
- **이 방식은 에이전트가 작업에 꼭 필요한 정보만 시점에 맞게 받아 최적화된 성능과 효율을 보장함**

## 세부 요약 - 주제별 정리

### Anthropic의 Skills 기능은 AI 발전에 기여하며 ‘단순함’ 철학이 핵심임

- Anthropic는 최근 ‘Skills’ 기능을 세계에 공개함
- 이는 최근 AI 발전 중 가장 중요한 도전 중 하나로 언급됨
- Anthropic의 핵심 모토는 ‘단순할수록 더 좋다(The simpler the better)’임
- Skills는 AI에 여러 능력을 부여할 때 복잡성을 유발하지 않는 간단한 구현 방식을 제공

### 정보 과부하(Overwhelm) 문제는 기존 방법의 가장 큰 약점으로 지적됨

- 다양한 툴, 대화 이력, 시스템 프롬프트 등 수많은 정보를 에이전트가 한 번에 받으면 쉽게 과부하에 빠짐
- 컨텍스트 윈도우의 한계로 인해 너무 많은 정보가 투입되면 에이전트 성능 저하 유발
- MCP 서버(기존 방식)에서는 필요 여부와 관계없이 처음부터 모든 툴을 일괄 제공해 과부하 문제가 심각함
- 예를 들어 한 번의 대화에서 사용하지 않을 기능까지 포함된 도구가 모두 투입됨

### Skills는 ‘점진적 공개’ 구조로 필요할 때만 정보를 추가 제공함

- Skill 방식은 ‘프로그레시브 디스클로저(Progressive Disclosure)’에 기반함
- 처음부터 모든 기능을 공개하지 않고, 실제 필요에 따라 점진적으로 툴과 정보를 탐색하도록 설계됨
- 시스템 프롬프트에는 각 스킬의 간단한 설명만 포함(1단계)
- 사용자 요청이 해당 능력을 필요로 할 때만 구체적 지침이 추가됨(2단계: skill.md)
- 필요 시 skill.md에서 다른 문서를 참조하여 추가 자료와 세부 정보를 확인(3단계)

### 스킬 사용 예시는 PDF 처리 능력으로 설명됨

- PDF 처리(PDF processing) 스킬이 예시로 등장
- 시스템 프롬프트에서 “PDF를 처리할 수 있다”는 간단한 설명만 에이전트에 제시
- 사용자가 PDF 관련 요청을 할 경우, 에이전트가 skill.md 파일을 확인
- skill.md에는 PDF 기능의 구체적인 사용 방법과 자세한 설명이 포함되어 있음

### skill.md 파일은 모든 주요 스킬의 핵심 동작 가이드 역할을 수행함

- skill.md가 각 스킬의 작동을 이끄는 주요 파일임(Anthropic에서 제공 및 커스텀 구현 사례 모두 포함)
- skill.md에는 해당 능력의 전체 실행 지침이 기록되어 있음
- 추가적인 상세 설명이나 참고 문서가 필요할 경우 skill.md 내에 해당 자료 링크나 정보가 포함됨
- 단계별 점진적 공개의 두 번째 계층 역할을 함

### 3단계 점진적 공개 구조가 에이전트의 효율과 성능을 극대화함

- 1단계: 시스템 프롬프트에 스킬 ‘설명’만 제공(최소한의 정보)
- 2단계: 실제로 필요할 때만 skill.md(실행 가이드)가 추가됨
- 3단계: skill.md에서 다른 문서나 자료가 추가적으로 제공될 수 있음(확장적 참조)
- 이 구조 덕분에 작업에 필요한 정보만 바로바로 부여되어 컨텍스트 용량도 절약되고 다양한 상황에 유연하게 대처 가능

### Skills 방식은 컨텍스트 최적화와 도구 활용의 균형을 성공적으로 달성함

- 항상 모든 도구를 제공하는 것이 아닌, 실제 상황에 맞춰 점진적으로 적재적소에 정보 제공
- 에이전트가 과부하에 빠지지 않으며 필요한 때에만 기능을 활성화 가능
- 정보의 레이어별 제공으로 컨텍스트 윈도우를 효율적으로 활용
- Skill 구조로 에이전트가 다양한 태스크에서 최고의 성능 발휘 가능

### Skills 도입은 사용성과 구현의 난이도 모두 낮추는 효과를 가짐

- 개발자는 스킬의 설명 및 skill.md만 준비하면 복잡한 설정/도구 관리 불필요
- 사용자 입장에서도 불필요한 도구 설명과 옵션 없이, 요청에 따라 자연스럽게 기능 활용 가능
- Skills 방식은 단순하면서도 강력한 AI 에이전트 구현의 새로운 표준으로 제시됨
