## *이 무료 툴로 Claude 코드 리뷰가 10배 더 편해짐* 핵심 요약

- 영상은 AI 개발 도구 Claude Code를 사용할 때 발생하는 코드 리뷰의 불편함을 지적하며, 이를 해결하는 무료 툴 ‘Kinugi(키누기)’를 소개함
- Claude Code는 강력하고 효율적이지만, 여러 인스턴스를 병렬로 사용할 경우 각 세션의 코드 출력을 각각 검토해야 해서 번거로운 문제가 있음
- Kinugi는 Claude Code 위에 만들어진 에이전틱(Agentic) 개발 환경으로, 기존 워크플로우를 대체하지 않고 혁신적인 시각적 레이어를 제공함
- Kinugi를 통해 모든 병렬 Cloud Code 세션을 한 화면에서 한눈에 확인하고, 각각의 코드 변경점과 리뷰 대상, 실행 중인 작업을 쉽게 파악 가능
- 각 에이전트가 생성한 ‘공격 계획(plan of attack)’을 실시간으로 검토하고 코멘트를 달 수 있어, 코드 구현 및 리뷰 과정이 크게 개선됨
- 기존에는 Cloud Code의 모든 로그와 파일을 개별적으로 확인해야 했으나, Kinugi는 깃허브의 풀 리퀘스트처럼 직관적으로 리뷰 가능함
- AI가 생성한 코드에 다양한 유형의 코드 리뷰(특히 보안 리뷰)를 자동으로 적용 가능한 점이 특징이며, 이는 개발사 Sonar의 전문성이 반영된 결과임
- Kinugi는 현재 Mac OS에서만 사용 가능하고 매우 실험적인 단계이기 때문에 무료로 제공되지만, 이미 Cloud Code 관리 효율성을 크게 높이고 있음
- 영상 제작자는 Sonar로부터 직접 연락을 받고 Kinugi를 테스트했으며, “정말 쿨하다”고 극찬함
- 영상 하단 설명란에 Kinugi 다운로드 링크가 걸려 있음

## 세부 요약 - 주제별 정리

### Claude Code는 매우 강력하지만 코드 리뷰가 큰 단점으로 지적됨

- 영상 제작자는 개발 과정 대부분에 Claude Code를 활용할 정도로 이 툴의 강력함을 강조함
- 하지만 사용 시간이 길어질수록 Claude Code의 출력 결과물, 특히 코드를 전부 검토하는 과정에서 '큰 틈(new gap)'을 체감하게 됨
- Claude가 병렬 처리로 여러 변경사항을 빠르게 생성하는 덕분에, 이 모든 출력을 사람이 일일이 리뷰해야 하는 불편함이 커짐
- AI가 코드를 자동 생성하더라도, 개발자가 직접 리뷰해야 하는 과정에서 생산성 저하 발생
- 특히 여러 Claude Code 인스턴스를 동시에(병렬로) 사용하는 환경에서는 출력물을 체계적으로 파악하고 검토하기가 더욱 힘듦

### 개발자는 Claude Code 병렬 세션 관리의 비효율을 직접 경험함

- 제작자는 “여러 인스턴스 병렬 실행”을 자주 사용하는 사용 패턴을 언급함
- 병렬 실행 시, 각각의 세션에서 나오는 결과코드를 따로따로 확인해야 하므로 관리와 검토의 번거로움이 커짐
- 코드 리뷰는 “여전히 사람이 꼭 해야 하는 작업”임을 명확히 지적
- 이로 인해 생산성이 저하되고 개발 경험이 복잡해지는 점을 실증적으로 설명함

### Kinugi는 Claude Code 위에 구축된 시각적 에이전틱 환경임

- Kinugi(키누기)는 Claude Code 자체를 대체하거나 경쟁하는 것이 아님
- “Claude Code를 기반으로 그 위에 시각적이고 에이전틱한 레이어가 추가된 개발 환경”임을 강조
- 사용자는 기존 Claude Code 워크플로우와 모든 기능을 그대로 활용하면서, Kinugi의 시각적 장점을 그대로 누릴 수 있음
- Kinugi를 “에이전틱 개발 환경(agentic development environment)”이라고 소개

### Kinugi를 통해 병렬 세션 한 화면에서 확인 및 관리 가능함

- Kinugi의 가장 큰 강점으로 “내 모든 병렬 세션을 한눈에 볼 수 있다”는 점을 꼽음
- 각 세션의 상태별(리뷰 대기, 실행 중 등) 목록화가 가능
- 한 화면에서 어떤 세션이 리뷰를 필요로 하는지, 어떤 작업이 아직 진행 중인지 시각적으로 파악할 수 있음
- 여러 Claude Code 인스턴스를 동시에 관리하는 사용자가 큰 효율 개선을 체감 가능

### ‘플랜 오브 어택(Plan of Attack)’, 실시간 피드백 및 코드 리뷰 기능이 제공됨

- AI 에이전트가 작업 계획(plan of attack)을 세우면, 사용자는 그것을 검토하고 실시간으로 코멘트를 달 수 있음
- AI 에이전트는 사용자의 피드백을 바로 반영하여 코드 구현을 진행함
- 코드 구현이 끝나고 리뷰 단계로 넘어오면, 기존에는 로그와 파일을 일일이 확인해야 했으나 Kinugi에서는 풀 리퀘스트 형식으로 한눈에 결과물을 검토 가능
- 기존 Cloud Code 사용 방식(모든 로그와 파일 개별 확인)에 비해 훨씬 높은 생산성을 제공함

### GitHub 풀 리퀘스트 스타일 리뷰 경험으로 코드 검토가 쉬워짐

- Kinugi에서는 코드를 한 줄 한 줄 로그로 확인하는 것이 아니라, 풀 리퀘스트(Pull Request) 형태로 직관적으로 비교/검토 가능
- 개발자는 변화된 코드와 주요 변경사항을 이전 코드와 나란히 놓고 검토함으로써 오류나 개선점을 쉽게 파악할 수 있음
- 이런 UI/UX의 변화는 기존 Cloud Code만 사용하는 것과 비교했을 때 매우 큰 개선임

### 자동 코드 리뷰(특히 보안 리뷰) 기능이 내장되어 있음

- Kinugi에는 AI가 생성한 코드에 대해 여러 유형의 자동 코드 리뷰를 적용할 수 있는 기능이 내장됨
- 보안(Security) 리뷰 등 클라우드 코드 자동 생성 코드의 리스크를 자동 진단하는 시나리오를 지원
- 이 기능은 개발사 Sonar가 보안 및 코드 품질 영역의 전문성을 갖춘 팀임을 근거로, 플랫폼의 경쟁력을 강화함

### Kinugi는 Mac OS 한정, 매우 실험적인 단계이므로 무료 제공 중임

- Kinugi는 현 시점에서 Mac OS에서만 설치 및 사용이 가능함
- Sonar 팀에서 제공하는 설명에 따르면 “아주 실험적인 단계”라고 강조
- 이러한 이유로 Kinugi는 현재 무료로 사용할 수 있으며, 향후 확장 가능성 여지를 남김
- 비록 실험적이지만 이미 Cloud Code 관리 효율을 실질적으로 크게 높여줌

### Sonar 개발팀이 Kinugi를 소개하고 영상 제작자가 직접 추천함

- Sonar 팀이 제작자에게 연락해 Kinugi 사용을 권유했고, 테스트 후 영상으로 소개하게 됨
- 제작자는 Kinugi를 “정말 쿨한 툴”이라고 표현하며 강력 추천
- 영상 설명란에 Kinugi의 공식 다운로드 링크가 제공됨
