## *스택드 디프는 AI 코드 리뷰 효율성을 혁신적으로 높임* 핵심 요약

- 영상 제목: *스택드 디프는 AI 코드 리뷰 효율성을 혁신적으로 높임*
- AI 코드 리뷰는 매우 강력한 도구이며, 코드 검증 과정에서 필수적이라고 강조함
- 기존의 PR(풀 리퀘스트) 방식은 승인 대기와 반복 작업으로 인해 비효율적임을 지적
- 스택드 디프(층층이 쌓인 변경분) 방식은 작은 단위의 변화를 연속적으로 쌓을 수 있어 전체 기능 구현에 효과적임
- 각 스택드 디프는 독립적이고 작은 변경점을 포함하므로, 각 단계마다 AI를 통한 집중적 리뷰가 가능함
- Graphite 플랫폼은 스택드 디프 워크플로우를 간편하게 지원하며, 전용 CLI(명령줄 도구)와 대시보드, AI 코드 어시스턴트를 제공함
- 플랫폼 데모를 통해, 첫 스택드 디프 생성부터 연속된 PR 작업, 대시보드 관리까지 직관적으로 안내받을 수 있음
- 기능 브랜치를 활용하여 각 변경분을 순차적으로 통합하며, 필요시 전체를 메인 브랜치로 병합함
- Claude 코드를 활용해 각 디프(PR)를 개별 리뷰할 수 있으며, 단일 대형 PR 리뷰 대비 큰 이점을 얻음
- Graphite의 참여와 실 사례 시연을 통해, 스택드 디프 기반 AI 리뷰의 효과를 세밀하게 전달함

## 세부 요약 - 주제별 정리

### AI 코드 리뷰는 검증 과정의 필수 요소로 자리 잡았음을 강조함

- AI 기반 코드 리뷰 도구의 강력함을 직접 언급
- AI 리뷰가 코드 품질 유지 및 검증에 얼마나 중요한지 개인적인 경험을 바탕으로 밝힘
- AI 코드 리뷰를 자체적인 필수(validation) 단계로 채택하고 있다고 강조함
- 리뷰 자동화가 개발 프로세스의 효율성을 크게 향상시킴

### 기존 풀 리퀘스트(PR) 방식은 승인 대기와 반복 작업에서 비효율을 노출함

- 전통적인 PR 워크플로우는 한 번에 하나의 수정 사항을 생성 및 제출
- 각 변경사항마다 검토와 승인을 기다려야 하므로 작업이 지연됨
- 피드백을 받기 전까지 다음 단계를 진행할 수 없어 전체 개발 흐름이 느려짐
- 개발 속도와 협업 효율성에 본질적인 한계를 지적함

### 스택드 디프는 작은 변화를 연속적으로 쌓아 전체 기능 구현을 효과적으로 함

- 스택드 디프란 여러 개의 작은 변경(디프)들을 순차적으로 쌓아 최종 기능을 구현하는 방식
- 각 디프는 서로 의존적이며 상위 디프 위에 쌓임
- 큰 변경 사항을 작은 단위 과제로 쪼개어 개발 및 리뷰가 진행됨
- 각 단계마다 독립적 검증과 피드백이 가능하여 오류 범위와 소통이 명확해짐

### 스택드 디프 방식은 AI 코드 리뷰 최적화를 가능하게 함

- 디프가 작고 명확하므로 AI 통해 초점이 맞춰진(초점화된) 피드백을 받을 수 있음
- 각 작은 디프에 대해 개별적으로 AI 코드 리뷰 수행이 용이
- 통합적인 대형 PR 리뷰보다 오류 탐지 및 논리적 검증에서 더 우수한 결과 도출
- 반복적인 PR 승인 대기 없이 개발자 생산성 극대화

### Graphite 플랫폼이 스택드 디프 워크플로우를 직관적이고 효율적으로 지원함

- Graphite는 스택드 디프 전용 명령줄 도구(CLI)와 대시보드를 제공
- GitHub CLI와 유사하지만, 스택드 디프 작업에 최적화됨
- 대시보드를 통해 모든 PR 및 디프 상태를 한눈에 관리 가능
- AI 코딩 어시스턴트가 내장되어 PR 리뷰를 자동화하여 지원

### Graphite 데모를 통해 스택드 디프가 어떻게 작성되는지 체험할 수 있음

- Graphite 신규 사용자 데모 과정 설명
- 첫 번째 스택드 디프 생성 및 PR 요청 - Git이나 GitHub CLI 사용과 매우 유사함
- 첫 PR 이후, 추가된 디프를 계속해서 PR로 쌓아 올릴 수 있음
- 이렇게 3개의 PR이 순차적으로 스택을 이루는 과정을 명령줄에서 시각적으로 경험

### Graphite 대시보드를 이용한 PR 관리 및 AI 코드 리뷰 활용 방법 시연

- Graphite 대시보드에서 각 PR(스택드 디프)을 관리할 수 있음
- Graphite agent를 활용해 PR 리뷰를 지원받을 수 있음
- 모든 PR을 대시보드 상에서 쉽게 병합 가능
- GitHub 저장소에 직접 들어갈 필요 없이 대시보드만으로 모든 과정 완료 가능

### 기능 브랜치 체계를 활용해 각 PR을 연속적으로 통합하는 구조를 명확히 밝힘

- 각 인크리멘탈(점진적) 변경은 각각의 기능 브랜치에서 이루어짐
- 변경분이 각 기능 브랜치 간에 병합(merge)됨
- 최종적으로 준비 완료된 모든 스택을 메인 브랜치에 반영함
- Git을 더욱 지능적으로 응용하는 방식임을 피력

### AI가 각각의 스택드 PR을 별도로 리뷰함으로써 품질과 효율이 크게 향상됨

- AI 리뷰 도구(Claude code)를 활용해 각 스택드 PR을 개별적으로 분석
- 1개의 대형 PR을 AI가 한 번에 리뷰하는 것과 비교해 월등한 분석 결과 도출
- 각 단계별 집중적 검증으로 버그, 논리 오류 탐지력 강화

### Graphite와의 협업을 통해 스택드 디프 기반 AI 코드리뷰의 실제 작동 방식을 실질적으로 보여줌

- Cole Medin이 Graphite와 협업하여 영상 제작 및 데모를 진행
- 실제 워크플로우를 시연함으로써 시청자가 그대로 따라할 수 있게 설명
- 결과적으로 스택드 디프와 AI 코드 리뷰 결합의 혁신적 가치를 전달
