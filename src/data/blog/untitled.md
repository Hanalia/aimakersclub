## *RAG 2.0: 에이전틱 RAG와 지식 그래프 결합(무료 템플릿 소개)* 핵심 요약

- 이 영상은 최신 RAG(검색-증강 생성) 전략 중 에이전틱 RAG와 지식 그래프(Knowledge Graphs)를 결합한 AI 에이전트 구축 사례를 소개함
- 영상 제작자는 다양한 RAG 전략을 실험한 끝에 두 가지 방식(에이전틱 RAG, 지식 그래프)이 특히 효과적임을 발견
- 개발된 AI 에이전트는 질문 유형에 따라 정보를 가장 잘 찾을 수 있는 데이터 소스를 스스로 선택해 검색
- 벡터 데이터베이스로는 Postgres에 PG vector 확장 기능을 붙이고, 서버리스 플랫폼 Neon을 이용해 구축함
- 주요 문서는 벡터 임베딩을 생성 후 Postgres에 저장되고, 동시에 지식 그래프(Graffiti, Neo4j에도 저장)로 변환 및 저장
- 지식 그래프를 활용하면 예시처럼 “Amazon이 어디에서 Enthropic 모델을 호스팅하는가?” 등 회사 간 관계 질의가 가능해짐
- 에이전트는 질문에 따라 벡터 DB(단일 엔티티 중심 질문) 또는 지식 그래프(관계성 질문) 중 최적의 방법을 자동으로 결정
- 사용자가 원할 경우 두 데이터 소스를 모두 활용하도록 프롬프트나 명령어를 조정할 수 있음
- 해당 AI 에이전트의 무료 템플릿이 제공되며, 영상의 풀 버전 링크를 통해 Cloud Code 활용, 단계별 구현법, 직접 실행 방법 등도 안내됨

## 세부 요약 - 주제별 정리

### 에이전틱 RAG와 지식 그래프가 결합된 AI 에이전트의 필요성이 강조됨

- 제작자는 여러 RAG 구현 전략(검색-증강 생성)을 깊이 있게 실험함
- 수많은 방법 중 에이전틱 RAG(Agentic RAG)와 지식 그래프가 반복적으로 가장 성과가 좋음을 경험
- 두 방식을 결합해 매우 강력한 AI 에이전트를 개발했고, 해당 에이전트가 정보를 더 정밀하게 제공함
- 사용자가 던지는 질문의 맥락과 성격에 따라 최적화된 정보 검색 경로를 스스로 선택함

### 벡터 데이터베이스는 Postgres와 Neon 서버리스 플랫폼으로 구축됨

- 벡터 데이터베이스로는 Postgres를 사용하고, PG vector 확장(extension)을 적용함
- 서버리스 기반 Postgres 플랫폼인 Neon을 이용하여 클라우드에서 효율적으로 데이터베이스를 운영함
- 문서를 분할(chunk)하여 저장하고, 각 조각에 대해 임베딩(embeddings)을 생성함
- 이렇게 저장된 데이터는 빠른 벡터 유사도 검색에 활용

### 동일 문서를 지식 그래프에 Graffiti, Neo4j로 저장하여 관계 질의 기능 확보

- 동일한 정보를 단순 텍스트 벡터 외에도 ‘관계형’ 방식인 지식 그래프 형태로도 저장함
- Graffiti와 Neo4j라는 유명 그래프 DB 솔루션을 활용해 정보를 노드(회사 등), 엣지(관계 등) 형태로 변환
- 예시 문서에는 빅테크 기업들의 AI 이니셔티브 및 상호 파트너십 정보가 포함됨
- 지식 그래프를 통해, 예컨대 “Amazon이 Enthropic의 모델을 호스팅함”과 같은 관계 질의가 가능해짐

### 에이전트는 질문을 분석해 최적의 검색 도구(벡터 DB/그래프 DB)를 자동 선정함

- 에이전트는 질문의 종류와 맥락을 판단한 뒤, 어떤 데이터 소스를 쓸지 스스로 결정함
- 한 회사의 정보 등 단일 엔티티 질문에는 벡터 데이터베이스만 검색
- 두 회사 간의 관계와 같이 복합적 관계를 요구하는 질문은 지식 그래프를 사용
- 실제 영상 예시: “Google의 AI 이니셔티브는?” → 벡터 DB 검색, “Amazon과 Enthropic의 관계는?” → 그래프 DB 검색

### 답변 근거와 사용된 도구 정보를 에이전트가 투명하게 제공함

- 결과값에는 에이전트가 어떤 도구(벡터 DB/그래프 DB)를 활용했는지 표시됨
- 매 쿼리마다, 어떤 검색 경로를 활용했는지 명확히 안내함
- 이를 통해 사용자는 AI가 어떤 분석 체계를 활용했는지 쉽게 파악 가능

### 사용자가 시스템 프롬프트나 명령으로 데이터 소스 활용 방식을 세밀하게 조정 가능함

- 기본적으로 에이전트가 질문을 분석해 소스를 결정하지만, 사용자가 명시적으로 데이터 소스 활용을 지정할 수도 있음
- 프롬프트 문구(시스템 프롬프트) 조정이나 “두 소스 모두 써라”와 같은 명령 입력이 가능함
- 영상 데모에서는 사용자가 직접 “벡터 DB와 지식 그래프 모두 사용” 명시 → 두 곳에서 모두 결과를 종합하여 응답

### 실제 구현 예시와 문서 구조, 정보 저장 방식이 구체적으로 소개됨

- 예시 문서는 실제로 빅테크 기업(예: Amazon, Google 등)의 AI 이니셔티브 정보와 기업 간 파트너 정보로 구성
- 문서 저장 방식: chunking(문서 조각화) → 벡터 임베딩 생성 → Postgres 저장, 동시 지식 그래프 DB 변환 및 저장
- 엔티티(회사 등)와 그 관계(파트너십, 모델 호스팅 등)가 그래프에 노드/엣지로 구성됨

### 무료 템플릿과 전체 구축법, Cloud Code 연계 방법, 실행 안내가 영상 전체에서 제공됨

- 이 AI 에이전트의 전체 소스 및 템플릿을 ‘무료’로 공개함을 강조
- 영상 하단(혹은 설명 링크)을 통해 전체 구현 영상(Full Video)로 연결
- 구현 과정에서 Cloud Code를 활용해 개발 생산성을 높인 방법까지 공유
- 사용자가 단계별로 따라할 수 있는 구체적 가이드와 바로 실행할 수 있는 방법까지 안내
