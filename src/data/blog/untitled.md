## *에이전트 하니스(Agent Harness)가 바이브 코딩(Vibe Coding)의 부활을 이끄는가* 핵심 요약

- LLM(대형 언어 모델)의 '순수한' 성능 향상은 최근 몇 년 전처럼 폭발적으로 증가하지 않고 있음
- 최신 LLM(Gemini 3, Claude Opus 4.5 등)이 벤치마크에서는 계속 발전하고 있지만, 더 중요한 변화는 LLM 위에 구축되는 '레이어', 즉 에이전트와 그 래퍼(wrapper)에 있음
- 에이전트 하니스(Agent Harness) 같은 상위 레이어를 구축하고 최적화하는 것이 현재 AI 발전의 결정적인 열쇠로 부상함
- 이러한 하위 구조(레이어)는 개발자가 직접 설계하고 실험할 수 있어, 혁신의 여지가 큼
- 월드 모델 등 다양한 구조적 시도들이 앞으로 별도의 영상 주제가 될 수 있음
- 이 상위 레이어의 혁신이 바로 새로운 에이전트 능력의 "언락(unlock)", 즉 한계 돌파를 가능하게 함
- 진화된 하니스를 통해 전체 코딩 작업의 99% 이상을 코드 에이전트에게 위임하는 미래가 가까워지고 있음
- 장기 실행형(Long-running) 에이전트에도 이러한 하니스 적용이 핵심이 될 것임
- 따라서 바이브 코딩(vibe coding, 창의적/추상적 코딩)이 부활할 것으로, 다만 이전과는 전혀 다른 모습이 될 것
- 2026년에는 개발자가 시스템과 하니스만 설계하고, 직접 코드는 거의 작성하지 않는 시대가 도래할 것으로 전망됨

## 세부 요약 - 주제별 정리

### LLM의 순수 연산력 향상은 더 이상 폭발적이지 않다는 사실을 받아들여야 함

- 최근 몇 년 간 LLM(대형 언어 모델)의 발전은 매우 인상적이었으나, 현재는 이전만큼 급격하게 향상되지 않고 있음
- "우리가 마주해야 할 냉정한 진실"로 언급, LLM 자체의 파워가 더 이상 핵심 경쟁력이 아님을 시사
- 예전과 같은 기하급수적 성장 곡선은 완만해졌다는 점이 강조됨

### 여전히 벤치마크는 상승세지만, 진짜 변화의 주역은 LLM 위에 구축되는 레이어임

- Gemini 3, Claude Opus 4.5 등 최신 모델의 벤치마크 수치는 여전히 올라가고 있음
- 하지만 실질적으로 발전의 차이를 만들어내는 것은 LLM 자체가 아니라, 그 위에 구축하는 구조들임을 강조
- "진짜 차이를 만드는 것은 우리가 우리의 에이전트 위에 무엇을 구축하는가"라는 직접적 언급

### 에이전트 하니스(Agent Harness)와 같은 인프라가 AI 혁신에서 핵심 역할을 함

- 에이전트 하니스 또는 LLM 래퍼(Wrapper) 등 '상위 구조'의 중요성이 특별히 부각됨
- 영상에서 예로 든 'world models' 등 다양한 메타 구조도 발전 가능성이 큼
- 이들 구조가 바로 다음 단계를 열어줄 'unlock'임을 구체적으로 밝힘

### 이 새로운 레이어는 개발자에게 창의적 실험과 최적화의 기회를 제공함

- LLM 위에 덧씌울 수 있는 구조들을 개발자가 스스로 설계하고 만들 수 있음
- "이것은 우리가 창의적으로 만들고 최적화할 수 있는 판"이라는 표현에서 가능성 강조
- 예를 들어 world models, 다양한 하니스 배리에이션 등 많은 실험이 이루어질 것임

### 이러한 레이어 혁신이 에이전트의 능력을 근본적으로 확장시킬 수 있음

- "이것이 우리가 가진 에이전트 능력의 다음 언락"이라고 명시
- 즉, LLM 자체 업그레이드가 아니라 에이전트 하니스의 혁신이 새로운 수준의 자동화와 창의적 대행을 가능하게 함

### 코딩의 99% 이상을 에이전트에게 위임하는 기술적 가능성이 가까워지고 있음

- 현재 상상력이 필요하지만, 하니스가 발전하면 코딩의 99% 이상을 대행할 수 있을 것이라고 전망
- 단기적 예측이 아닌, 중장기적 비전을 담고 있음('in the very near future', 'what we have in store for 2026')
- "코딩 작업의 대부분을 에이전트가 처리"라는 개념 초점

### 장기 실행형 에이전트(Long-running Agent)에서도 하니스가 핵심 역할을 하게 됨

- 긴 시간 동안 동작하며 복잡한 과제를 처리하는 에이전트에도 이 하니스 구조가 필수적임을 언급
- 실제로 영상에서 "long-running agent"를 별도로 강조함

### 바이브 코딩(Vibe Coding)은 새로운 형태로 부활할 것이며, 개발자는 시스템과 하니스 설계에 집중하게 될 것임

- "Vibe coding, it's going to sort of become viable, but also look a lot different"라고 직접 언급
- 과거의 바이브 코딩(창의적, 추상적 방식의 코딩)이 복귀하지만, 실제로는 시스템과 하니스 구조를 엔지니어링하는 것이 핵심이 됨
- 대부분의 코드는 에이전트가 자동으로 작성하며, 인간은 방향성과 구조 설계에 주력할 것으로 전망

### 2026년경 인간 개발자가 직접 코딩하는 시대는 끝나고, 하니스 설계가 핵심이 될 것으로 전망됨

- "우리가 매우 가까운 미래(2026년)에 대부분의 코드를 작성하지 않게 될 것"이라는 명확한 시간 언급
- 개발자의 역할이 시스템 아키텍처, 하니스 엔지니어링 중심으로 재편됨을 반복적으로 강조
- 이 변화가 필연적이고 가깝다는 점을 확고하게 전달함
