## *RAG 2.0: 에이전틱 RAG와 지식 그래프의 결합 (무료 템플릿 제공)* 핵심 요약

- 최근 몇 달간 다양한 RAG(Retrieval-Augmented Generation) 전략을 실험한 끝에, **에이전틱 RAG(Agentic RAG)**와 **지식 그래프(Knowledge Graph)**가 가장 탁월한 방식임을 강조
- 두 전략을 결합하여, **질문 의도에 따라 스스로 검색 경로와 데이터 소스를 선택하는 AI 에이전트**를 구축함
- 벡터 데이터베이스로는 **Postgres의 PG Vector 확장**과 서버리스 플랫폼 **Neon**을 활용하여 문서를 청크로 분할 및 임베딩함
- 동일한 문서를 **Graffiti와 Neo4j**를 이용해 **지식 그래프**에도 저장, 정보의 관계성을 표현함
- 에이전트는 **단일 회사에 대한 질문**(예: “구글의 AI 이니셔티브는 무엇인가?”)에는 벡터 검색만 활용해 자동으로 답변
- **기업 간 관계를 묻는 질문**(예: “Amazon과 Anthropic의 관계”)에는 지식 그래프 쿼리로 전환하여 응답
- **시스템 프롬프트나 명시적 지시**를 통해, 필요시 벡터 DB와 지식 그래프 **두 소스 모두를 동원**한 종합적인 답변도 제공
- 개발 과정에는 **Cloud Code**를 사용하여 구현 과정을 지원함
- 이 모든 구성요소와 코드는 **무료 템플릿**으로 공개, 전체 빌드 과정은 본 채널의 장문 영상을 참고할 수 있도록 안내

## 세부 요약 - 주제별 정리

### 다양한 RAG 전략을 검토한 끝에 에이전틱 RAG와 지식 그래프가 최상의 조합임을 확인함

- 최근 수 개월 동안 발표자(Cole Medin)는 거의 모든 종류의 RAG 방식을 심도 있게 실험함
- 다양한 방법을 탐색해본 결과, “에이전틱 RAG(Agentic RAG)”와 “지식 그래프(Knowledge Graph)” 조합이 반복적으로 가장 좋은 결과를 보여줌
- 해당 조합은 단순한 정보 검색을 넘어, 문맥적 추론과 관계 기반 검색까지 모두 지원함
- 기존 RAG의 한계를 보완하고, 더 유연하고 똑똑한 정보 추출이 가능해짐

### AI 에이전트는 질문의 유형에 따라 최적의 정보 소스를 스스로 선택하여 활용함

- 새로 구축한 AI 에이전트는 질문을 이해하고, 답변에 가장 적합한 정보원을 **자동으로 판단**함
- 질문의 유형 및 의도에 맞춰 벡터 검색 또는 그래프 쿼리를 선택적으로 실행
- 이러한 방식으로 **정확성과 효율성** 모두를 강화함
- 사용자는 별도의 작업 없이도 복잡한 질의에 대해 자연스럽게 답을 받을 수 있음

### 벡터 데이터베이스는 Postgres + Neon + PG Vector로 구현됨

- 벡터 데이터베이스 계층을 **Postgres**의 **PG Vector 확장**으로 구현
- 서버리스 아키텍처 기반의 **Neon 플랫폼**을 활용하여 관리 및 확장성 확보
- 문서는 미리 청크(chunk) 단위로 나누어 저장하며, 각 청크에 대해 임베딩을 생성함
- 예시 문서에는 **빅테크 기업들의 AI 이니셔티브와 파트너십 정보**가 포함됨

### 지식 그래프는 Graffiti와 Neo4j로 구축하여 관계 정보도 효율적으로 저장함

- 동일한 원본 문서를 **Graffiti**와 **Neo4j**를 사용해 지식 그래프로 구축
- 지식 그래프 방식은 **데이터 간의 관계성**을 직관적으로 표현할 수 있도록 도와줌
- 예시: “Amazon은 Anthropic이 모델을 호스팅하는 곳”과 같은 기업 간 파트너십 관계 표현 가능
- 구조화된 관계형 저장방식으로 쿼리가 다양해지고, 다층적인 질의가 가능함

### 에이전트는 벡터 DB·지식 그래프 중 적합한 쿼리 방식을 상황별로 결정함

- 예를 들어 **“구글의 AI 이니셔티브가 무엇인가?”**라는 질문이 입력되면, 에이전트는 벡터 데이터베이스만 검색함
- 이때 에이전트는 검색에 사용한 도구 또한 명확히 표시해 줌(‘vector database used’ 등)
- **“기업 간 관계”**를 묻는 질문이 들어오면, 에이전트는 프롬프트의 안내에 따라 **지식 그래프** 쿼리로 전환함
- 어떤 데이터 소스를 어떤 이유로 사용했는지 투명하게 표기함

### 필요시 사용자는 벡터 검색과 그래프 검색을 동시에 실행하도록 에이전트를 제어할 수 있음

- 시스템 프롬프트를 조정하거나, 직접 “벡터 데이터베이스와 그래프 모두를 사용하라”고 명령 가능
- 이렇게 명시적으로 지시하면 에이전트는 **두 데이터 소스를 모두 탐색**해 종합적인 답변 제공
- 병행 검색을 통해 훨씬 더 깊이 있는 정보 도출 및 다양한 관점 제공이 가능해짐

### Cloud Code를 활용해 개발 및 배포 과정이 최적화됨

- RAG 2.0 에이전트의 구현 과정에는 **Cloud Code** 도구를 적극 활용
- Cloud Code는 개발, 배포, 협업을 간편하게 만들어 주어 전체 개발 프로세스가 가속화됨
- 영상의 전체 빌드, 테스팅, 배포 시나리오 등은 추가 영상에서 자세하게 안내함

### 누구나 무료로 사용할 수 있도록 에이전트 템플릿을 전면 공개함

- 전체 RAG 2.0 에이전트는 **무료 템플릿** 형태로 제공됨
- 필요한 코드 및 설정값, 사용 예시는 공개된 링크(영상 하단 참고)에서 바로 확인 가능
- 본 영상은 단편 소개 버전으로, 세부 구현법은 메인 영상에서 가이드함

### 결론적으로 이 에이전트는 다양한 AI 응용 사례에 즉시 적용 가능하며 확장성도 높음

- 벡터 데이터베이스와 지식 그래프의 **융합적 AI 구조**를 쉽고 빠르게 구현 가능
- 복잡한 정보 요청에 유연하게 대응하며, 대화형 AI, 비즈니스 인텔리전스 등 여러 분야로 확장 가능
- 기업 내부 데이터, 복합적 산업 관계 처리 등 다양한 실제 사례에 활용될 수 있음
