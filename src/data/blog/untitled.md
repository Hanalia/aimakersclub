## *Claude에게 24시간 쉬지 않고 코딩을 시킨 결과* 핵심 요약

- 영상 제목은 ‘Claude에게 24시간 쉬지 않고 코딩을 시킨 결과’임
- 제작자는 Claude Opus 4.5를 24시간 동안 연속으로 코딩 작업에 투입함
- 테스트의 절반이 남았음에도 이미 claw.ai와 거의 동일한 기능의 클론을 완성했다고 언급함
- 과거의 대화 기록과 각 반복(Loop) 단계에서의 검증 내용을 모두 볼 수 있는 기능을 갖춤
- 마크다운(Markdown) 포맷팅이 뛰어나 각종 문서와 코드가 보기 좋게 정리됨
- HTML 페이지 생성, 코드 작성 및 실행 기능을 포함해 다양한 부가기능이 구현됨
- 환경설정(Settings)에서는 테마 변경, 기본 모델 선택, 최대 토큰 수를 조절할 수 있는 슬라이더 기능이 추가됨
- 기능의 풍부함(Feature rich)에 제작자가 감탄하며, 한 번도 손을 대지 않고도 이렇게 많은 기능을 구현한 점을 강조함
- 향후 ‘사람이 확인하는 절차’(Human-in-the-loop)를 도입할 계획이라고 밝힘
- Claude Opus 4.5와 같은 LLM과 검증·반복용 하니스의 조합이 애플리케이션 개발에 혁신적인 가능성을 제시함
- 하루 더 같은 방식으로 추가 개발한다면 완벽에 가까운 애플리케이션이 될 수 있다고 시사함
- 최신 AI 도구의 자가 검증 및 반복 개선 능력의 놀라움을 강조함

## 세부 요약 - 주제별 정리

### Claude는 24시간 만에 claw.ai와 유사한 완전한 클론 애플리케이션을 구축함

- 제작자는 Claude를 24시간 ‘논스톱’으로 코딩시키는 실험을 진행함
- 모든 테스트가 완료되지 않은 시점(절반 이상 남음)임에도 불구하고, claw.ai의 거의 모든 주요 기능을 재현함
- 개발자는 "이미 완전히 동작하는 클론 같다"며 놀라움을 표함
- claw.ai는 AI 채팅 앱의 일종으로 이해됨

### 각 반복 단계의 대화와 검증 기록을 손쉽게 조회할 수 있음

- 애플리케이션 내에서 Claude가 이전에 생성했던 대화 이력을 전부 확인할 수 있음
- 각 반복(Loop)마다 무엇을 검증하며 어떻게 발전시켰는지 추적 가능함
- 검증 및 작업의 자동화된 ‘루프’ 구조를 통해 AI가 스스로 발전해 나가는 과정을 투명하게 노출함

### 뛰어난 마크다운(Markdown) 포맷팅과 문서화가 구현됨

- Claude가 생성한 모든 결과물과 코드가 마크다운 형식으로 보기 좋게 정리됨
- 문서 및 코드 설명, 예시 등이 구조적으로 제공되어 개발 결과를 한눈에 파악할 수 있음
- 가독성 및 유지보수 용이성 측면에서 강점을 보임

### HTML 페이지 생성 및 코드 작성·실행 기능이 모두 지원됨

- 애플리케이션 내에서 다양한 HTML 페이지를 자동 생성 가능
- Claude가 직접 코드도 작성하고 이를 바로 실행해볼 수 있는 환경을 갖춤
- 코드 실행 결과 역시 마크다운 형식 등으로 깔끔하게 출력됨

### 테마 변경, 기본 모델 선택, 토큰 수 조절 등 풍부한 환경 설정 옵션이 추가됨

- 환경설정(settings) 항목에서 사용자가 다양한 옵션을 커스터마이즈할 수 있음
- 테마(theme)를 자유롭게 선택 가능
- 기본 사용할 AI 모델을 변경할 수 있는 옵션이 있음
- 대화 시 허용되는 최대 토큰 수를 슬라이더(slider)로 세밀하게 조정 가능하게 구현됨

### 기능의 다양성과 완결성에 대해 제작자가 감탄을 표함

- 애플리케이션이 ‘so so featurerich’(기능이 아주 풍부하다)라고 언급
- 제작자는 자신이 손을 대지 않고도 대부분의 기능이 자동으로 구현됐다는 점을 강조하며 놀라움을 드러냄
- 실제 손을 거의 쓰지 않고도, 대부분 Claude에게 맡김에도 불구하고 완성도 높은 결과가 나왔음을 강조

### 향후 ‘Human-in-the-loop’를 통한 사람의 검증 절차 도입 계획이 있음

- 현재까지는 완전히 자동화된 형태이지만, 앞으로는 사람의 개입(Human-in-the-loop)을 추가할 계획을 밝힘
- 이는 AI의 요구사항 분석 혹은 품질 검증 단계에서, 사람의 판단을 결합하려는 의지로 해석됨
- 인간의 피드백이 추가될 경우 완성도 및 신뢰성이 더 높아질 것으로 기대

### Claude Opus 4.5와 하니스(Harness)의 결합이 검증과 반복개선에 큰 역할을 함

- Claude Opus 4.5의 뛰어난 생성 능력과 반복 검증 시스템(하니스)이 결합됨
- 하니스는 애플리케이션의 유효성과 품질을 자동으로 테스트하고 개선점을 도출하는 역할을 함
- AI가 자체적으로 수많은 수정·검증 과정을 거쳐 품질 높은 결과물을 산출하도록 설계

### 하루만 더 같은 방식으로 계속 개발한다면 완벽한 앱이 될 수 있다고 평가함

- 제작자는 기능 추가와 반복 검증을 하루(24시간)만 더 계속하면, 높은 완성도의 애플리케이션이 될 것으로 내다봄
- Claude와 자동화된 개발 도구의 한계와 가능성 모두를 엿볼 수 있음

### Claude Opus 4.5 및 최신 AI 도구의 잠재력에 대한 감탄으로 마무리함

- 제작자는 Claude와 같은 최신 LLM이 ‘crazy’(말도 안 되게 놀랍다) 수준의 생산성을 보여준다고 평가
- 자동화된 하니스와 결합했을 때 AI가 개발 현장에 가져올 변화에 대해 언급
- 영상 전체를 통해 LLM의 빠른 반복·검증·기능 확장 능력을 전시하며, AI가 직접 소프트웨어를 개발하는 시대의 도래를 암시함
