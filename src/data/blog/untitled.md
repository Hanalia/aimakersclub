## *나의 AI 기억이 이제 모든 툴에서 연동된다!* 핵심 요약

- Cole Medin은 AI 코딩 도우미 'Claude Code'를 자신의 '세컨드 브레인'처럼 사용하며 개인과 프로젝트 관련 모든 정보를 기억하게 함
- 하지만 Claude Code 외에도 Codex, Pi 등 다양한 AI 코딩 툴을 병행 사용 중이며, 각 툴별로 강점이 달라 사용용도를 다르게 함
- 기존 Claude Code에 구축한 기억 시스템은 해당 툴에만 묶여 여러 도구 간 기억(메모리) 전환이 불가능한 한계가 있었음
- 이 문제를 해결하기 위해 외부 기억 시스템인 'Walrus and Memory'를 발견, 도입함
- Walrus는 특정 AI툴이 아닌 '툴 외부에 존재'하는 분산형, 암호화 기반 메모리 시스템으로, 사용자가 자신의 데이터를 직접 소유하고 통제할 수 있음
- Walrus는 한 번의 프롬프트만으로 무료로 설정 가능하며, 홈페이지에서 프롬프트를 복사하여 모든 코딩 AI에 입력하면 바로 설치됨
- SDK를 통해 다양한 앱에 직접 메모리 시스템을 통합할 수 있고, MCP 서버 연동으로 각종 AI 도구에서 즉시 사용 가능
- 실제 예시로 대량 텍스트에서 핵심 정보 20개를 추출·암호화하여 저장하는 등 강력한 기억 저장·검색 기능 시연
- Claude Code, Pi 등 여러 AI 에이전트 간 동일한 메모리(기억) 데이터 공유 가능하며, delegate key 등으로 접근 권한 제어도 손쉽게 제공함
- Walrus는 모든 AI 툴에 적용하는 범용 기억 레이어로서 빠르고 간편하게 설치·사용할 수 있음

## 세부 요약 - 주제별 정리

### Claude Code의 개인화된 '세컨드 브레인' 기억 시스템 구축과 한계

- Cole Medin은 'Claude Code' AI 코딩 도우미를 통해 자신의 정보와 진행 중인 프로젝트 관련 세부 사항을 기억하도록 만듦
- Claude Code에 구축한 기억 시스템 덕분에 생산성이 크게 향상됨
- 하지만 최근 Codex, Pi 등의 다양한 AI 툴을 동시에 사용하게 됨
- 각 툴별로 독특한 강점이 있어, 작업 특성마다 최적의 AI를 선택 사용함
- 문제점: Claude Code 전용으로 만든 기억 시스템이 해당 툴에 '잠겨' 있어, 다른 AI 툴과 기억을 공유할 수 없음

### 다양한 AI 코딩 도구 간 기억 데이터의 자유로운 이동 필요성 발생

- 여러 AI 코딩 에이전트를 병행 사용하면서, 일관된 기억 데이터(메모리)를 다양한 툴과도 공유하려는 요구가 생김
- 기존에는 기억 시스템의 툴 종속성 때문에, 도구를 변경할 때마다 같은 내용을 다시 셋업해야 했음
- 이러한 비효율성을 극복하기 위해, 툴 간 자유로운 기억 전환과 공유가 가능한 솔루션을 모색함

### 'Walrus and Memory' 도입으로 분산형 AI 기억 시스템 실현

- 오래 찾아 헤맨 끝에 Cole Medin은 'Walrus and Memory'라는 솔루션을 발견하여 직접 적용함
- Walrus는 특정 AI툴 내부가 아닌, 완전히 툴 외부(분산형)에서 기억 시스템이 동작함
- 저장되는 기억 데이터는 모두 암호화되며, 사용자가 직접 데이터의 소유권을 가짐
- 기억 데이터가 특정 IT기업이 아닌 사용자의 인프라에 안전하게 저장됨

### Walrus는 한 번의 프롬프트로 무료 설치 및 환경구축이 용이함

- Walrus의 가장 큰 특징은 '원 프롬프트(One Prompt)' 방식의 간단한 설치 과정
- Walrus 공식 홈페이지에서 프롬프트를 복사해, 사용 중인 AI 코딩 어시스턴트 창에 붙여넣기만 하면 셋업이 거의 완료됨
- 이 과정을 통해 누구나 별도의 복잡한 설정 없이 무료로 시작 가능함

### SDK와 MCP 서버 연동으로 어떤 앱이나 도구에도 자유롭게 확장 가능

- Walrus는 개발자용 SDK(Software Development Kit)를 제공
- SDK를 통해 Walrus 기반 기억 시스템을 사용자 자신이 직접 원하는 앱에 탑재할 수 있음
- MCP 서버 연결로 Claude Code, Pi 등 다양한 AI 도구에서 동일한 기억 데이터에 즉각 접근 가능

### 대화 속 또는 대용량 텍스트로부터 핵심 정보(메모리) 자동 추출 기능 시연

- 자연스러운 대화를 통한 기억 저장뿐 아니라, 대량의 텍스트를 복사해서 붙여넣으면 핵심 정보를 알아서 추출·기억함
- 데모에서는 세컨드 브레인 관련 대용량 텍스트를 입력, 'Walrus로 이 내용을 기억하라' 명령 후, MCP 서버가 20개의 주요 사실을 추출
- 추출된 정보들은 암호화되어 Walrus 메모리에 자동 저장됨

### Claude Code 및 Pi 등 모든 AI 툴에서 동일한 기억 데이터 공유 및 검색 가능

- Claude Code에서 'Walrus memories에서 Dark Factory의 기술 스택을 알려달라'고 질의하자, Walrus 메모리 내의 관련 정보(전체 스택)를 정답으로 반환함
- 같은 방식으로 Pi에서 동일 질의를 입력하면, 약간 더 간결하지만 본질적으로 동일한 정보를 다시 얻음
- 사용자는 어떤 AI 도구에서든지 Walrus 메모리만 연결하면 같은 기억 데이터풀을 즉시 검색·활용 가능

### 기억 데이터 접근 권한 관리와 Delegate key 발급으로 비즈니스 활용도 극대화

- Walrus에 저장된 기억은 모두 암호화되어 보안성이 높음
- 사용자별·시스템별로 delegate key를 발행해, 특정 기억에 대한 접근 권한 관리가 가능함
- 다양한 시스템 혹은 특정 인물에게 기억 데이터 일부에 대한 접근 허용 등, 유연한 권한 설정이 가능함

### 하나의 기억 레이어로 모든 AI 툴에 적용해 빠르게 생산성 향상 가능

- Walrus는 기존 기억 시스템과 달리, 특정 툴에 종속되지 않는 범용 기억 레이어(메모리 플러그인·API 역할)를 수행함
- AI 도구 추가·변경 시에도 한 번 만든 기억 자산을 계속 사용할 수 있어, 생산성 손실 없이 새로운 툴을 채택 가능
- 실제 설치 및 연동 과정은 매우 빠르고 직관적임

### 무료 사용과 간편한 온보딩으로 누구나 바로 실험 가능함

- 소개 영상 설명란에 Walrus 홈페이지 링크가 제공됨
- 프롬프트 한 줄로 즉시 무료 체험 및 설치 가능
- 쉬운 온보딩과 빠른 셋업으로 많은 사람들이 다양하게 활용하기 용이함
