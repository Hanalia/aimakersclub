## *가장 간단한 방법으로 AI 에이전트의 기능을 극대화하는 법 (스킬)* 핵심 요약

- Anthropic이 최근 선보인 "스킬(skills)"은 AI 에이전트 기능 향상에 있어 가장 단순하면서도 강력한 방식을 제시함
- 이 방식이 중요한 이유는 "단순함이 곧 강점"이라는 Anthropic의 철학에 충실하기 때문임
- AI 에이전트는 다양한 도구, 대화 기록, 시스템 프롬프트 등 많은 정보를 한 번에 제공받게 되면 컨텍스트 윈도우가 쉽게 과부하되거나 혼란스러워질 수 있음
- 기존의 MCP 서버 방식은 미리 모든 도구를 에이전트에게 제공해버려, 실제 필요하지 않은 기능까지 노출시켜 비효율과 정보 과부하 문제 유발
- 스킬의 핵심은 "점진적 노출(progressive disclosure)"로, 필요한 순간에만 기능을 한 단계씩 추가 정보를 제공하는 방식임
- 예시로 PDF 처리 기능을 사용할 때, 처음에는 오직 '이런 기능이 있다'는 설명만 내려주고, 실제 활용 요청이 오면 구체적인 skill.md 문서를 불러와 지침을 전달
- skill.md 파일은 해당 스킬의 명확한 수행법과 추가로 필요한 문서(3단계 노출)를 참조하며 실제 컨텍스트에 적절히 로드됨
- 이처럼 단계별로 필요한 정보만 제공함으로써 에이전트의 효율성과 확장성을 동시에 확보하는 혁신적 방법임
- Anthropic의 스킬 구조는 앞으로 AI 에이전트 설계, 활용에 중요한 표준으로 자리 잡을 것으로 전망됨

## 세부 요약 - 주제별 정리

### Anthropic의 '스킬' 출시가 AI 발전에 미치는 영향이 매우 큼

- Anthropic이 최근 선보인 ‘스킬’은 AI 진화에 있어 핵심적인 발전으로 꼽힘
- 해당 기능은 출시된 지 오래되지 않았으나, 현 시점에서 가장 중요한 AI 기술 중 하나로 언급됨
- 단순함과 직관성에 초점을 맞춘 이 방식이 AI 에이전트 설계에서 혁신적 전환점이 되고 있음
- 본 영상은 Anthropic의 스킬 구조와 그 실용적 가치를 설명함

### '단순함이 곧 강점'이라는 Anthropic의 설계 모토가 구현됨

- Anthropic의 모토는 ‘더 단순할수록 더 낫다(The simpler the better)’임
- 복잡한 입력보다, 필수적인 능력을 단순명료하게 전달하는 것이 중요하다고 강조
- AI 시스템은 사용하기 쉽고, 확장성 있게 기능이 추가되어야 한다고 주장

### AI 에이전트의 컨텍스트 윈도우 과부하 문제를 효과적으로 해결함

- 기존에는 에이전트에 다양한 도구와 정보, 대화 이력, 시스템 프롬프트 등을 한꺼번에 전달함
- 모든 정보가 한 번에 컨텍스트 윈도우로 들어갈 경우, 에이전트가 쉽게 과부하되거나 혼란스러워짐
- 도구가 많아질수록 컨텍스트 윈도우의 부담이 기하급수적으로 증가함

### MCP 서버 방식과 비교해 스킬 방식의 단점 극복 방식을 설명함

- MCP 서버와 같은 기존 접근은 에이전트에게 필요한지 여부와 무관하게 모든 도구를 미리 제공
- 실제 대화나 작업에서 필요하지 않은 도구까지 억지로 로드되어, 비효율적으로 작동함
- “모든 도구를 upfront로 주는 것은 좋지 않다”는 명확한 한계점 지적
- 이러한 문제점을 스킬의 점진적 노출로 극복함

### '점진적 노출(progressive disclosure)' 개념이 스킬 구조의 핵심임

- 점진적 노출이란 에이전트에 필요한 정보를 단계적으로 주입하는 설계 철학
- 처음에는 기능 설명(‘기능이 있다’는 사실)만 시스템 프롬프트에 간단히 기재
- 사용자가 그 기능을 실제로 요구하면, 구체적인 정보와 문서를 추가로 로드
- The agent discovers and loads only as needed—즉, 필요할 때만 능력을 트리거함

### 예시: PDF 처리 기능 도입 절차가 단계적으로 이뤄짐

- 예시로 PDF 처리 스킬을 사례로 선정
- 1단계: 시스템 프롬프트에 ‘PDF 처리 기능이 있다’고 알림
- 2단계: 사용자가 실제 PDF 관련 요청을 보내면, 에이전트가 skill.md 문서를 읽어 상세 절차를 파악
- 3단계: skill.md 문서가 추가적으로 연결된 관련 자료(예: 다른 설명 문서 등)를 단계별로 참조, 점진적으로 전체 컨텍스트를 확장
- 각 단계별로 필요한 정보만 로드되어 에이전트의 효율성과 정확도를 높임

### skill.md 파일이 스킬의 핵심 구현 역할을 담당함

- skill.md는 각 스킬(기능)에 대한 구체적이고 명확한 지침을 담은 파일임
- Anthropic의 모든 스킬의 핵심은 이 문서에 구현됨
- 사용자가 요청을 트리거하면, 에이전트는 skill.md의 내용을 읽고 그에 따라 행동
- skill.md에서 추가문서를 참조할 수 있고, 그때마다 컨텍스트가 점차 늘어남

### 정보는 요청이 생길 때만, 단계별로 에이전트에 추가됨

- PDF 처리 예시처럼, 시스템 프롬프트에는 오직 간단한 설명만 먼저 제공
- 필요 시에만 상세 정보와 추가 문서가 차례차례 로딩되는 2~3단계 구조
- 덕분에 컨텍스트 윈도우 부담이 최소화되고, 실제 필요한 작업에만 자원을 집중 가능

### Anthropic 스킬 구조의 혁신이 다른 AI 툴과 차별화되는 이유 명확히 설명됨

- 기존 AI 툴들이 정보 과잉과 복잡성 문제를 안고 있지만, Anthropic의 접근은 필요 중심, 최소 정보 구조임
- 기능 확장과 성능 최적화를 동시에 달성하는 새로운 패러다임을 제시
- 이러한 구조는 다양한 커스텀 구현에도 일관성 있게 적용 가능

### 점진적 노출 기반 설계가 향후 AI 에이전트 표준이 될 수 있음을 암시함

- Anthropic의 스킬 아키텍처는 단순함, 효율성, 확장성을 고루 갖춘 미래지향적 모델임
- 영상 내에서는 이 구조를 활용한 커스텀 구현이 실제로 어떻게 이뤄지는지 예시를 들어 설명
- 단계별 정보 노출이 지닌 이점(최적의 리소스 분배, 사용성 향상 등)이 반복적으로 강조됨
