## *AI 에이전트의 성능을 비약적으로 높이는 가장 간단한 방법(스킬)* 핵심 요약

- 이 영상은 최근 Anthropic이 공개한 ‘스킬(Skills)’이라는 기능이 AI 발전에서 갖는 중요성에 대해 다룸
- Anthropic 스킬의 가장 큰 특징은 ‘단순함’으로, 복잡한 정보를 한 번에 주지 않고 필요한 순간에만 단계적으로 공개(점진적 공개: Progressive Disclosure)함
- 기존 MCP 서버 방식은 사용하지 않을 도구까지 모두 한꺼번에 에이전트에게 제공, 컨텍스트 창을 금세 과부하시키는 문제점이 있음
- 스킬 방식은 에이전트가 필요한 정보를 필요할 때마다 점진적으로 얻을 수 있게 해 과부하를 방지함
- 수행 가능한 '스킬'은 시스템 프롬프트에 간단한 설명만 우선 전달되고, 실제로 필요 시 자세한 설명 문서(skill.md)를 읽게 됨
- skill.md 파일에는 해당 역량에 대한 구체적이고 완전한 지침이 담겨 있어 두 번째 단계의 정보 제공을 담당
- 경우에 따라 skill.md가 추가 문서들을 참조함으로써 세 번째 단계의 심화 정보 제공이 이루어짐
- 이 구조 덕분에 사용하지 않는 정보로 에이전트의 컨텍스트 창이 낭비되지 않고, 실제 필요한 때 최적의 정보만 제공 가능
- 예시로 ‘PDF 처리 스킬’을 들면서 사용자가 관련 요청할 때만 세부 지침이 로드되는 구조를 설명
- 전체적으로 영상은 Anthropic의 스킬 시스템이 AI 활용을 더 효율적이고 강력하게 만드는 핵심 원리와 구현 방식을 강조함

## 세부 요약 - 주제별 정리

### Anthropic 스킬 도입은 최근 AI 분야의 가장 중요한 발전 중 하나임

- 영상은 Anthropic이 최근 도입한 스킬(skills) 기능을 ‘AI에서 최근 이루어진 가장 중요한 발전’ 중 하나라고 평가함
- 스킬 시스템은 에이전트의 성능을 비약적으로 향상시키는 계기라고 강조
- 단순함과 효율성을 핵심 가치로 내세우는 앤트로픽의 철학이 잘 반영됐음을 언급

### 복잡한 정보 제공은 에이전트의 컨텍스트 창 과부하를 초래함

- AI 에이전트의 컨텍스트 창(context window)은 과도한 정보(프롬프트, 툴, 대화 이력 등)에 쉽게 압도됨
- 기존 방식(MCP 서버 등)은 한 번에 매우 많은 도구/스킬을 에이전트에게 주입하는데, 실제 필요하지 않은 정보까지 포함되어 컨텍스트 과포화 발생
- 이런 과부하는 에이전트의 응답 정확도 및 성능 저하로 이어짐

### 스킬의 혁신은 ‘점진적 공개(Progressive Disclosure)’ 전략에 있음

- 스킬 방식의 본질은 ‘점진적 공개(progressive disclosure)’임
- 에이전트가 실제 태스크 수행 중에 점점 더 많은 정보를 탐색하며 얻을 수 있도록 설계됨
- 모든 툴을 한꺼번에 제공하는 기존과 달리, 에이전트가 필요할 때마다 점차적으로 도구/정보를 습득

### 시스템 프롬프트에는 역량 설명만 포함하여 정보 과부하를 방지함

- 에이전트에게 처음부터 주입되는 정보는 각 스킬(기능)에 대한 매우 간단한 ‘설명’ 뿐임
- 예시: PDF 처리 스킬이 있다면, “PDF 파일을 다룰 수 있는 기능이 있음” 정도만 시스템 프롬프트에 기술
- 에이전트가 해당 기능이 필요하다고 판단되면 그때서야 본격적인 사용법 등 추가 정보를 얻게 됨

### skill.md 파일이 실제 역량 실행을 위한 메인 가이드 역할을 수행함

- skill.md 파일은 각 스킬의 ‘메인 지침서’로서, 해당 역량을 어떻게 사용할지에 대해 구체적으로 안내함
- 사용자가 관련 요청을 할 때 에이전트는 skill.md 파일을 참조하여 구체적인 실행 단계를 이해
- 이로써 스킬의 두 번째 정보 제공 레이어가 시작됨(1차: 설명, 2차: 상세 지침)

### 경우에 따라 skill.md는 추가 참고 문서를 통해 심화 정보 제공이 가능함

- skill.md 문서 내부에서는 필요한 경우 별도의 서브 문서 혹은 관련 자료를 참고하게 설계됨
- 이렇게 3단계(기초 설명 → 상세 지침 → 심화 문서)의 점진적 정보 층이 구성되어 있음

### PDF 파일 처리 스킬 예시로 단계적 정보 로딩 과정을 설명함

- 예시: ‘PDF 처리 스킬’을 스킬 시스템에 적용할 경우
- 최초 대화에선 에이전트가 PDF 기능이 있다는 사실만 알고 있음
- 사용자가 PDF를 처리해달라는 요청을 해야 에이전트가 skill.md(실행 지침)를 읽고 실제 처리 단계로 진입
- 추가 설명이나 특정 플러그인 기능이 필요하면 skill.md가 별도 문서로 연결함

### 불필요한 정보 로딩을 방지해 에이전트가 더욱 강력하게 작동함

- 이 구조는 사용하지 않는 기능까지 에이전트가 처음부터 기억하게 하는 비효율을 없앰
- 실제 상황에 맞는 정보만 실시간으로 조달하면서 에이전트의 컨텍스트 창을 슬림하게 유지
- 자연스럽게 에이전트의 응답 속도와 활용 효율이 극대화됨

### Anthropic 스킬 시스템이 AI 활용에 혁신적인 방향성을 제시함

- 영상은 Anthropic 스킬 시스템이 단순함과 점진적 정보 제공 전략만으로도 AI 에이전트의 성능을 극대화할 수 있음을 시사함
- 복잡한 도구 체계 없이도, 효과적으로 다양한 기능을 곧바로 효율적으로 사용할 수 있다는 점을 강조
- 이러한 구조적 접근이 AI 도구 개발 및 응용 방법론 전반에 중요한 시사점을 제공함
