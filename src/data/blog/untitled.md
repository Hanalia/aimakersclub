## *AI 에이전트를 극적으로 강화하는 가장 단순한 방법 (Anthropic Skills)* 핵심 요약

- Anthropic가 최근 공개한 ‘Skills’(스킬) 기능은 AI 분야에서 가장 중요한 혁신 중 하나로 평가됨
- Skills의 최대 장점은 매우 간결하고 단순한 설계 방식에 있음
- 복잡한 기능을 한 번에 제공하는 대신, 필요한 순간에 점진적으로 기능을 공개(Progressive Disclosure)함
- 기존 MCP 서버같은 도구들은 대량의 도구와 정보를 처음부터 에이전트에 제공하여 컨텍스트 윈도우를 과부하시키는 단점이 있음
- Skills는 시스템 프롬프트에 오직 기능(스킬)의 ‘설명’만 우선적으로 포함시킴
- 사용자가 예를 들어 PDF 처리와 같은 특정 요청을 할 때에만, 관련 skill.md 문서를 로딩하여 자세한 사용 지침을 제공
- skill.md 문서는 모든 스킬의 핵심으로 작동하며 기능별 세부 지침을 단계적으로 에이전트에 읽히도록 유도
- 추가적인 세부 파일이 필요하면 skill.md에서 이를 참조, 3단계 이상으로 점점 더 많은 컨텍스트를 필요에 따라 주입
- 이렇게 함으로써 컨텍스트 윈도우를 절약하고, 에이전트가 실제로 필요한 정보만 효율적으로 활용하도록 설계됨
- Skills의 구조는 복잡성을 최소화하면서도 에이전트의 기능을 유연하게 확장할 수 있는 방식임

## 세부 요약 - 주제별 정리

### Anthropic의 'Skills' 기능은 최근 AI 발전의 가장 중요한 진보 중 하나로 평가됨

- Anthropic는 얼마 전 Skills라는 새로운 기능을 일반에 공개함
- 이번 기능은 AI 에이전트의 활용성 및 성능을 크게 강화한 것으로 주목받고 있음
- Skills는 기존의 복잡한 도구 관리방식보다 훨씬 간단하고 효율적임
- Anthropic 팀의 모토는 "더 단순할수록 더 낫다(The simpler the better)"임
- Skills 도입의 주된 목적은 AI 에이전트에 다양한 역량을 부여하되, 컨텍스트 윈도우의 한계를 넘지 않는 것임
- Skills가 가진 간결성과 단계적 정보 공개 방식이 최신 AI 흐름과 맞물리며, 시장에서 관심을 받는 중임

### 기존 MCP 서버 방식은 에이전트에 비효율적임이 드러남

- MCP 서버나 기존 도구 관리 시스템은 모든 도구 목록과 정보를 한 번에 에이전트에 제공함
- 이 방식은 에이전트가 실제 대화에서 필요하지 않은 도구까지 모두 기억해야 하므로 컨텍스트 윈도우가 쉽게 과부하됨
- 시스템 프롬프트, 대화 기록, 도구 설명 등 모든 정보가 한 번에 컨텍스트에 투입되어 과다한 부하를 발생시킴
- 컨텍스트 윈도우의 적재 한계로 인해 실질적인 작업 효율이 떨어질 수밖에 없음
- 이런 한계 때문에, 실제 대화에 불필요한 도구 제공은 바람직하지 않다는 비판적 설명이 추가됨

### 'Skills'는 점진적 공개(Progressive Disclosure)를 통해 효율적인 기능 관리가 가능함을 보여줌

- Skills의 핵심 설계 철학은 Progressive Disclosure, 즉 ‘점진적(순차적) 공개’임
- 에이전트가 필요시마다 단계적으로 더 많은 기능과 정보를 스스로 발견하도록 설계됨
- 사용자가 관련 요청을 하지 않는 한, 모든 도구의 사용법이나 세부 지침이 항상 컨텍스트에 포함되지 않음
- Skills 방식은 실제 대화를 진행하는 동안 필요한 순간에만 구체적 정보를 로딩함

### 시스템 프롬프트에는 스킬의 ‘설명’만 우선적으로 제시함

- 시스템 프롬프트에 우선적으로 전달되는 것은 각 기능(스킬)에 대한 간단한 ‘설명’임
- 예시: PDF 처리 기능이 있다면, "PDF 처리가 가능합니다" 정도의 간단한 설명만 초기 시스템 프롬프트에 포함됨
- 이로써 컨텍스트 윈도우가 간소화되고, 에이전트의 기본 기능 인식만 확보함

### 실제 요청이 있을 때에만 skill.md를 읽어 세부 지침을 단계적으로 로딩함

- 사용자가 예를 들어 "PDF를 요약해달라"고 요청할 경우, 에이전트는 해당 skill.md 파일을 읽음
- skill.md에는 그 기능(예시: PDF 처리)에 대한 자세한 사용 방법, 절차, 구체적 작업 지시가 포함됨
- 에이전트는 이렇게 받아들인 상세 정보를 근거로 실제 처리를 진행함
- skill.md는 각각의 스킬마다 존재하는 핵심 안내 파일임

### skill.md 자체도 추가 서브파일을 참조하며 2~3단계로 세부 정보를 점진적으로 제공함

- skill.md는 필요시 더 구체적인 세부 지침 파일을 내부에서 참조하도록 구성됨
- 예컨대, PDF 처리에 필요한 세부 명령어나 예제, 파라미터 파일 등이 있을 수 있음
- 정보 제공이 일괄적으로 이루어지지 않고, 단계별로 점점 더 심화된 파일들이 작업 진행에 따라 연결됨
- 이를 통해 처음에는 큰 틀의 설명 → 점점 세부 안내로 이어지는 2~3단계 점진 구조가 완성됨

### Skills 구조는 불필요한 컨텍스트 적재를 최소화하면서도 확장성이 뛰어남

- 에이전트가 한 번에 모든 정보를 갖고 있을 필요가 없으므로, 컨텍스트 윈도우 사용이 매우 효율적임
- 필요시마다 정보와 도구를 추가적으로 습득(Discover)하게 하여 기능 확장과 효율 관리가 동시에 이루어짐
- Skills 구조는 복잡성 증가 없이도 새로운 역량 추가와 기존 역량 수정이 유연함
- Anthropic이 설계한 이 구조 덕분에 개발자와 에이전트 사용자가 모두 더 나은 경험을 할 수 있음

### 결론적으로 Skills 통한 점진적 기능 공개가 AI 에이전트 설계의 새 표준이 될 가능성을 시사함

- Skills의 사례는 불필요한 정보 전달을 줄이면서도 에이전트의 사용성을 극대화함을 보여줌
- Progressive Disclosure(점진적 공개) 원칙이 앞으로 더 많은 에이전트 설계에 적용될 전망임
- 영상에서는 Anthropic Skills 구조와 예시(custom implementation)로 PDF 처리 도구를 들어 설명함
- ‘필요할 때만’ 단계적으로 정보를 주입하는 핀 포인트 방식이 AI 에이전트 효율 개선에 중요한 전환점임
