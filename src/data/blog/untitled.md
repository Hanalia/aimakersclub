## *Glue.ai는 슬랙이 지향했어야 할 AI/멀티플레이어 네이티브 채팅 플랫폼* 핵심 요약

- 영상은 슬랙(Slack) 이용 중 겪는 '조직화 문제'와 생산성 저하를 지적하며 시작함
- 발표자는 최근 중요한 디버그 쓰레드를 3일이나 늦게 발견하는 등 슬랙의 알림 및 쓰레드 관리에 불만을 토로함
- 업무 진행 시 여러 앱(Asana, Notion, GitHub 등)을 오가야 해서 큰 불편을 느낌을 언급
- 이러한 문제의 해결책으로 'Glue'라는 새로운 채팅 플랫폼을 소개함
- Glue는 MCP(Multiplayer Control Platform) 기반 최초의 '실제 업무 전용' AI/멀티플레이어 채팅 플랫폼임을 강조함
- Glue AI는 각 대화 스레드 내에 상주해 맥락을 완전히 파악하며, 팀 전체가 실시간 협업 가능
- MCP 통합 덕분에 Glue AI는 Asana의 태스크 수정, Notion 문서 열람, GitHub 이슈 확인 등 주요 도구와의 연동을 대화 내에서 즉시 지원
- Glue의 인앱 MCP 디렉터리를 통해 35개의 업무 툴을 클릭 한 번으로 즉시 연결 가능하며, 수천 개의 커스텀 MCP 서버도 확장 지원
- Glue는 기존 슬랙처럼 외부 AI 도구를 단순히 덧붙이는 것이 아니라, 업무용 AI 네이티브 채팅 플랫폼임을 차별점으로 내세움
- 영상은 구체적으로 팀 내 채팅과 업무 연동, AI의 맥락 파악력, 앱 전환 없는 편의성 등을 근거로 Glue의 우수성을 강조함
- 발표자는 구식 채팅 툴에 불편을 느끼는 AI 팀, 최신 워크플로우를 원하는 조직에 Glue AI 체험을 적극 권장함

## 세부 요약 - 주제별 정리

### 발표자는 슬랙 사용 경험에서 업무 흐름 단절과 알림 문제를 겪음

- 발표자는 일상적으로 슬랙을 활용해 업무를 진행하지만, 점점 더 혼란스럽고 조직적이지 못하다고 느낌
- 최근 3일이나 늦게 중요한 버그 수정(thread)에 대해 알게 되었다며, 이는 알림 체계의 문제라고 지적
- 이러한 혼란은 사용자의 쓰레드 관리 미흡 때문일 수 있지만, 소프트웨어의 구조적 한계에도 주목함
- 슬랙에서 중요한 대화를 놓침으로써 발생하는 업무 지연 및 생산성 저하의 실례를 상세히 언급
- 반복적으로 디지털 워크스페이스에서의 '정보 유실'과 '알림 피로'가 실제 작업에 악영향을 끼침을 토로

### 업무 처리 과정에서 여러 앱을 오가는 번거로움이 심각함을 설명

- 슬랙 내 채팅에서 실제로 액션을 취하려면 각종 도구(Asana, Notion, GitHub 등)로 이동해야 함
- 이 과정에서 빈번한 앱 전환과 작업 맥락의 단절(context switching)이 발생
- 단순한 대화 확인이 아니라, 채팅을 통한 실제 업무 진행이 힘들다고 느껴 불편함을 암시
- 다양한 도구의 연계 미흡이 팀 협업 전체에 비효율을 유발한다고 강조

### Glue는 AI 및 MCP 네이티브 설계로 기존 슬랙과 근본적으로 다르다고 강조

- Glue를 "슬랙이 발전했어야 할 모습"으로 소개
- MCP(Multiplayer Control Platform) 기반 최초의 멀티플레이어 채팅 솔루션임을 밝힘
- Slack이나 타 플랫폼처럼 외부 AI를 끼워 맞춘 형태가 아님을 명확히 함
- Glue의 설계 구조 자체가 AI와 여러 협업 도구 통합을 '기본'으로 내장함을 강조

### Glue AI는 모든 대화 쓰레드에서 '풀 컨텍스트'로 팀원들과 협력

- Glue AI는 각 대화 스레드 내부에 상주(Embed)되며, 해당 대화의 전체 맥락을 일관되게 파악
- AI를 통한 정보 제공 및 요약, 파일 또는 태스크 업데이트 등 팀원 업무를 AI가 직접 지원
- 팀 전체가 Glue AI를 통해 동일한 맥락에서 협업하도록 돕는 구조
- 'AI 비서' 이상의 역할을 하며, 팀별 실질적인 생산성 도구로 작동함을 시사

### MCP 기술 덕분에 Glue AI는 주요 업무 도구들과 즉시 상호작용 가능

- Glue AI는 Asana 태스크 변경, Notion 문서 조회, GitHub 이슈 확인 및 업데이트 등 핵심 협업 툴에 직접 접속
- 대화창 내에서 즉시 관련 작업을 실행할 수 있어, 별도 앱 실행 및 전환이 필요 없음
- 전형적인 팀의 워크플로우에서 발생하는 비효율적 '앱 점프'를 없앰
- 가장 최신의 MCP와 도구 연동 기술을 적극 활용한 구조임을 분명히 언급

### Glue 인앱 MCP 디렉터리로 35개 이상의 앱을 즉시 연동 및 확장할 수 있음

- Glue 플랫폼에는 인앱 MCP 디렉터리가 존재, 35개 업무 툴을 클릭 한 번으로 연동 가능
- 사용자는 확장성을 위해 수천 개의 커스텀 MCP 서버에 접근할 수 있음
- 이 덕분에 팀 특성이나 요구에 따라 추가적인 도구 연동 및 맞춤화가 자유로움

### Glue는 'AI를 외부 도구로 붙인' 형식이 아닌, 본질적 'AI 네이티브' 채팅 플랫폼임

- Slack과 달리 Glue는 외부 AI 챗봇이나 플러그인에 의존하지 않음
- 플랫폼 구조 자체가 AI와 실시간 협업, 도구 통합 중심으로 설계됨
- 이로써 '채팅+AI'가 아닌, '실제 업무 전용 AI-네이티브 멀티플레이어 채팅' 플랫폼이라는 차별성을 강조

### Glue 도입 시, 생산성/효율성 증대와 과거식 채팅툴의 한계 극복 가능성을 시사

- 기존 슬랙-중심 팀 체팅이 '2015년에 머무른 느낌'을 준다고 지적
- Glue AI를 통해 최신 AI 도구를 활용해도 팀 채팅의 생산성이 제약받지 않음을 강조
- 특히 기술 스타트업, AI 툴 개발·배포 팀에게 Glue가 상당한 효용을 줄 수 있음을 암묵적으로 시사
