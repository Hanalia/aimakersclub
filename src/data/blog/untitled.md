## *RAG 2.0: 에이전틱 RAG와 지식 그래프 결합(무료 템플릿 공개)* 핵심 요약

- 영상 제목은 ‘RAG 2.0: 에이전틱 RAG + 지식 그래프(무료 템플릿)’로, 최신 RAG(검색·생성 증강) 전략의 결합 방식과 이를 활용한 AI 에이전트 제작 방법을 소개함
- 제작자는 최근 몇 달간 다양한 RAG 전략을 연구했으며, 특히 ‘에이전틱 RAG’와 ‘지식 그래프’ 방식에 주목했다고 밝힘
- 두 방식을 결합한 강력한 AI 에이전트를 직접 개발하여 도입
- AI 에이전트는 ‘질문에 최적인 정보 소스’를 스스로 판단하여, 벡터 데이터베이스 또는 지식 그래프를 적절히 탐색해 답변을 도출함
- 데이터 저장에는 Postgres 데이터베이스와 PG vector 확장, Neon 서버리스 플랫폼을 활용하여, 문서 임베딩 및 저장을 구현함
- 동일한 문서를 Graffiti 및 Neo4j로 지식 그래프 형태로도 구축, 기업간 관계·제휴 정보까지 반영
- 실제 예시로, 단일 기업 쿼리(예: "구글의 AI 이니셔티브")는 벡터 DB 검색, 회사 관계 쿼리(예: "Amazon은 어느 업체에게 Anthropic을 호스팅하나?")는 지식 그래프를 에이전트가 스스로 선택하여 검색함
- 시스템 프롬프트를 조정하면 두 소스를 동시에 검색하도록 할 수도 있으며, 이를 통해 더욱 안전하고 풍부한 답변 획득 가능
- 영상 마지막에서 이 강력한 에이전트를 ‘무료 템플릿’으로 제공하며, 전체 제작 과정을 설명하는 링크와 Cloud Code 활용 노하우도 안내함

## 세부 요약 - 주제별 정리

### 최근 RAG(검색 증강 생성) 연구 결과, 에이전틱 RAG와 지식 그래프 방식이 가장 뛰어남

- 제작자는 최근 몇 달 동안 거의 모든 RAG 전략을 깊이 있게 실험하였음을 강조함
- 다양한 접근법이 있지만, 반복적으로 돌아오게 된 방식이 ‘에이전틱 RAG’와 ‘지식 그래프’ 두 가지임
- 두 방법 모두 각기 다른 강점이 있으며, 실제 응용에서 효과성이 증명됨
- 영상의 핵심은 두 방식을 결합해 시너지 효과를 창출하는 AI 에이전트 제작임

### 두 방식을 결합한 강력한 AI 에이전트가 개발되었음

- 제작자는 직접 에이전트 모델을 구축했으며, 두 방식(RAG, Knowledge Graph)을 결합해 탁월한 정보 추천 및 추론이 가능하도록 설계함
- 에이전트는 질문의 성격에 따라 ‘정보 탐색 경로’를 자동으로 결정함
- 사용자가 무엇을 묻느냐에 따라, 더 적합한 데이터 소스를 선택하여 답변 품질을 높임
- 구축한 에이전트를 '정말 멋진 AI 에이전트'라고 직접 언급함

### Postgres, PG vector, Neon 등을 활용해 벡터 데이터베이스 환경을 실현함

- 벡터 DB로 PostgreSQL과 PG vector 확장 기능 사용
- 데이터베이스 구축에는 서버리스(PostgreSQL 기반) 플랫폼인 Neon을 이용하여 간편하고 유연한 운영이 가능함
- 주요 문서는 정보를 잘게 ‘청크(chunk)’ 단위로 나누어 저장
- 각 청크에는 Embedding(임베딩) 생성 후 Postgres에 저장, AI 모델이 쿼리 가능한 상태로 만듦

### 문서 내용은 ‘빅테크 기업, AI 사업 현황, 기업 간 제휴’ 등 폭넓게 포함됨

- 시연에 사용된 문서는 빅테크 기업들의 이름, 각 사의 AI 관련 이니셔티브(사업) 정보, 그리고 서로 간의 파트너십 사례 등이 담겨 있음
- 예: “Amazon은 어떤 AI 스타트업과 제휴했는가?”, “Anthropic은 어디에 모델을 호스팅하는가?” 등의 관계형 질문 가능
- 동일 문서가 여러 데이터 형식(벡터, 그래프)으로 중복 저장되어 다각적 검색 실험이 가능함

### 동일 문서를 Graffiti와 Neo4j로도 저장해, 지식 그래프를 구축함

- 기존 임베딩 기반 데이터 외에도, 같은 정보를 지식 그래프 구성으로 변환해 저장함
- Graffiti와 Neo4j를 활용하여 각 기업, 활동, 관계 등이 노드 및 엣지 구조로 연결됨
- 관계형 데이터 기반으로, 더 복잡한 연결성(예: “Amazon과 Anthropic은 어떤 식으로 파트너십을 맺었는가?”) 질의 가능

### AI 에이전트는 질문 유형에 맞춰 데이터 소스를 스스로 선택·활용함

- 단일 엔티티(기업 등)에 대한 질문(예시: “구글의 AI 이니셔티브는?”)에는 벡터 DB를 활용함
- 관계형 질의(예시: “Amazon은 Anthropic 모델을 어디서 호스팅하는가?” 등)는 지식 그래프를 선택함
- 에이전트는 모든 질의 뒤에 ‘어떤 도구(데이터베이스 또는 그래프)’를 사용했는지 투명하게 밝혀 줌
- 사용된 툴(예: ‘vector database 검색’, ‘graph search’) 정보도 실시간으로 반환

### 시스템 프롬프트 조정 또는 명시 호출로 벡터 DB, 그래프 DB 동시 활용도 가능함

- 시스템 프롬프트 혹은 사용자 명령에서 “두 소스 모두 사용”을 요청하면, 에이전트가 벡터 DB와 지식 그래프를 모두 쿼리하여 최적 답변 도출
- 각 질의마다 사용한 데이터 소스 및 도구가 결과 화면에 표시됨
- 상황과 목적에 따라 다양한 혼합 검색 패턴 적용이 가능

### 에이전트는 복합 검색 기능과 유연한 응답 구조로 매우 실용적임을 강조함

- 벡터 DB와 지식 그래프가 결합된 복합 검색이 사용자의 다양한 질의 유형에 뛰어난 대응력을 제공함
- AI 에이전트는 각 데이터 저장 방식의 장점만을 취해, 답변의 신뢰성과 풍부함을 극대화함
- 제작자는 “정말 놀라운 AI 에이전트”라며 실사용 가치에 대해 강하게 주장함

### 무료로 제공되는 템플릿 및 전체 구현 영상 링크 정보도 포함됨

- 본 영상은 템플릿을 누구나 이용 가능하게 무료로 공개함을 분명히 알림
- 전체 AI 에이전트 구축 및 Cloud Code 활용법, 구현 과정 전체를 담은 본편 영상의 링크를 안내함
- Cloud Code 도구가 개발 및 배포 과정에서 중요한 역할을 했음을 언급함
