## *Glue.ai는 Slack이 진화했어야 할 모습: AI/MCP 네이티브 채팅 플랫폼* 핵심 요약

- 영상 제작자는 자신의 비즈니스 운영에 매일 Slack을 사용하지만, 점점 비효율적이고 혼란스러워진 구조에 불만을 품고 있음
- 중요한 버그 수정과 관련된 쓰레드를 3일 늦게 확인하는 등 Slack의 알림 시스템과 쓰레드 관리 한계로 인해 실제로 업무 지연이 발생했음을 설명
- Slack 내에서 논의된 작업을 수행하려면 여러 개의 앱을 오가야 해 번거롭고 비효율적인 업무 흐름 발생
- 최근 제작자가 실험적으로 사용해 본 Glue는, Slack이 진화해야 했던 이상적인 형태로 소개됨
- Glue는 최초의 멀티플레이어 MCP(Multi-agent Collaboration Platform) 기반 채팅 플랫폼으로, 실제 업무 진행에 초점을 맞춤
- Glue AI는 대화 쓰레드 내에 상시 배치되며, 팀 전체가 동일한 컨텍스트로 AI를 활용할 수 있음
- MCP 기반으로 아사나(Asana), 노션(Notion), 깃허브(GitHub) 등 다양한 외부 툴과 직접 연동되어 Glue 내에서 바로 태스크 관리, 문서 확인, 이슈 추적 등이 가능
- Glue는 클릭 한 번으로 연결 가능한 35개의 툴 인앱 MCP 디렉토리를 구축하였고, 별도 MCP 서버 연동 시 수천 개의 툴도 활용 가능함
- Glue는 Slack에 써드파티 AI를 단순히 얹는 것이 아닌, AI를 네이티브하게 탑재하고 '업무 완결'에 집중한 팀용 채팅 플랫폼임
- 최신 AI 도구를 개발하고 있지만 팀 채팅만은 과거 방식에 머물러 있다면, Glue AI 도입을 고려해볼 만하다고 제안

## 세부 요약 - 주제별 정리

### 제작자는 Slack의 작업 조직력 부족으로 업무에 실질적 지장이 있었음을 고백함

- 영상 시작에서 제작자는 매일 Slack을 사용하는 사실을 밝히며, 최근 들어 점점 더 비효율적으로 느껴진다고 토로함
- 중요한 버그 수정과 관련된 쓰레드를 3일이나 늦게 확인하는 실수를 실제로 겪었다고 설명
- 이러한 문제의 원인 중 일부는 개인의 쓰레드 관리 습관에도 있음을 인정하지만, Slack의 알림 및 정보 구조 한계도 지적

### Slack 내 채팅 기반 사업 운영이 여러 앱을 번갈아 써야 하는 비효율로 이어진다고 지적함

- Slack에서 해야 할 작업을 실행하려면 다양한 외부 앱을 계속 전환해야 해서 번거롭다고 표현
- 예를 들어, 대화 중 할 일을 태스크 관리 툴(예: 아사나)로 옮기고, 필요한 문서는 노션에서 찾고, 깃허브 이슈는 별도로 확인해야만 함
- 반복적인 앱 간 이동이 생산성을 저하시키고, 실제 업무 흐름을 방해한다고 밝힘

### Glue는 Slack이 진화했어야 할 AI/MCP 네이티브 협업 플랫폼임을 주장함

- Glue를 "Slack이 되어야 했던 것"이라고 소개하며, 단순한 메시징 플랫폼을 넘어선 본질적인 변화를 시도한다고 평함
- Glue는 AI와 MCP(Multi-agent Collaboration Platform) 네이티브 구조로 설계되었으며, 업무 완결 및 협업에 초점을 둠
- “최초의 멀티플레이어 MCP 기반 채팅 플랫폼”이라고 명확히 표현

### Glue AI는 대화 쓰레드 안에서 팀 전체 맥락을 바탕으로 실질적 업무 지원을 제공함

- Glue AI는 대화 쓰레드 내에 항상 존재하며, 전체 대화 컨텍스트를 인지하고 작업에 참여할 수 있음
- 동일한 AI 인스턴스를 팀원 모두가 함께 활용할 수 있어, 각자 다른 정보를 주고받는 것이 아니라 공동의 맥락 속에서 소통이 가능
- AI가 단순한 자동 답변 수준이 아니라, 실질적으로 기반 데이터와 연동함

### MCP 아키텍처 덕분에 Glue는 여러 주요 도구와 심층적으로 연동함

- Glue는 MCP 구조를 통해 다양한 툴과 즉시 연결되고, 빠르게 실무에 반영할 수 있도록 설계됨
- Glue를 이용하면 아사나(Asana)의 태스크를 업데이트하거나, 노션(Notion)의 문서를 읽거나, 깃허브(GitHub)의 이슈를 확인하는 등 실제 업무 흐름을 이탈하지 않아도 됨
- 이러한 모든 작업이 Glue 대화 내에서 끊김 없이 이뤄짐

### Glue의 인앱 MCP 디렉토리는 35개 주요 툴을 클릭 한 번에 연결 가능하게 해줌

- Glue에는 현재 35개의 툴을 인앱(앱 내) MCP 디렉토리로 제공, 버튼 한 번에 즉시 연동
- 별도의 커스텀 MCP 서버 구성 시 수천 개의 외부 툴 추가 연동도 가능
- 다양한 SaaS 및 생산성 관리 툴과의 깊은 통합성을 강조

### Glue는 Slack에 AI를 써드파티로 덧댄 것이 아니라 AI를 기본 구조로 삼음

- Glue는 “Slack처럼 AI를 제3자 툴로 덧붙인 것이 아니라, 태생적으로 AI를 근간에 두고 설계”됐다고 설명
- 이는 단순 메시징에 그치지 않고, AI를 체계적으로 도입하여 실제 공동 작업 생산성을 높이려는 의도임
- “팀이 빠르게 일하고 맥락 전환(Context Switching)을 피하는” 데 초점을 둔다고 언급

### 최신 AI 도구를 개발하지만 팀 채팅은 구식이라면 Glue AI 사용을 고려하라고 권유함

- 제작자는 “AI 도구를 출시하고 있으면서도, 팀 내 채팅 환경은 여전히 2015년도에 머물러 있다 느끼면 새 Glue AI를 써볼 것”을 적극 제안
- 이는 혁신적인 팀커뮤니케이션과 AI 기반의 업무플로우 도입 필요성을 암시하는 권유로 마무리됨
