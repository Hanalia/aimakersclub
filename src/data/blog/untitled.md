## *가장 간단하게 AI 에이전트의 기능을 극대화하는 방법 (스킬)* 핵심 요약

- 이 영상은 Anthropic社가 최근에 공개한 '스킬(Skills)' 기능이 AI 발전사에서 얼마나 중요한 변화인지를 다룸
- Anthropic의 핵심 철학은 "더 단순할수록 더 좋다"는 것이며, 이 철학이 스킬 기능에 적용됨
- 기존에는 많은 기능과 도구들을 한 번에 에이전트에 제공해 문맥 창(context window)을 쉽게 과부하 시켰음
- 예전 방식(MCP servers 등)은 여러 도구를 에이전트에게 처음부터 모두 노출, 실제 대화에서 필요하지 않은 도구까지 포함됨
- 스킬 기능의 혁신은 '점진적 공개(Progressive Disclosure)'에 있음: 필요할 때만 해당 기능/문서를 참조하도록 설계
- 시스템 프롬프트에는 각 스킬의 ‘간단한 설명’만 포함시켜 에이전트의 문맥 과부하를 막음
- 사용자가 특정 기능(예: PDF 처리)을 원할 때만, 에이전트가 관련 ‘skill.md’ 파일(상세 지침, 실전 사용 설명서)에 접근
- skill.md와 추가 문서들은 다층적으로 점진적으로 에이전트가 참조, 사용에 필요한 맥락만 그때그때 불러옴
- 이로써 기존 도구 기반 AI 시스템의 비효율(용량 낭비, 정보 과다)을 개선하면서도 에이전트를 유연하고 강력하게 만듦
- 영상에서는 PDF 프로세싱 스킬을 예시로, 점진적 공개가 실제 어떻게 작동하는지 구체적으로 설명함

## 세부 요약 - 주제별 정리

### Anthropic이 최근 도입한 스킬(능력) 기능은 AI 발전에서 매우 중요한 전환점임

- Anthropic은 '스킬'이라는 새로운 기능을 최근 공개했음
- 영상 제작자는 이를 AI 최근 발전 중 가장 중요한 변화로 꼽음
- 혁신의 핵심은 구현이 매우 '아름다울 정도로 단순하다'는 것임
- Anthropic의 모토는 "단순할수록 더 낫다(The simpler, the better)"임
- 기존 복잡한 기능 제공 방식과 대비해 월등히 효율적임

### 더 많은 기능 제공이 오히려 에이전트 문맥 창 수준을 쉽게 초과시킴

- 우리는 AI 에이전트(Agent)에 다양한 능력을 부여하고 싶어함
- 하지만 conversation history, system prompt, tool 정보 등 모든 것이 문맥 창(context window)에 함께 들어감
- 한 번에 너무 많은 정보를 주면 에이전트가 쉽게 정보를 처리하지 못하고 성능 저하를 일으킴
- 복잡한 프롬프트, 불필요한 도구 노출 등이 대표적 문제임

### 기존 방식(MCP 서버 등)은 모든 도구를 미리 공개해 에이전트를 오히려 불필요하게 부담시킴

- MCP servers 같은 이전 방식은 다양한 도구(tool)를 에이전트에게 시작 단계에서 전부 제공함
- 이러한 도구 중 실제 대화에서 필요 없는 것도 포함되어 비효율적으로 작동함
- 결과적으로 context window 용량이 낭비되고, 에이전트가 혼란을 겪음
- 지나치게 많은 정보 제공이 오히려 품질 저하를 초래함

### 스킬 기능의 본질은 '점진적 공개(Progressive Disclosure)'에 있음

- 스킬의 핵심 원리는 '필요할 때만 자세한 정보를 노출'하는 점진적 공개임
- 에이전트에 모든 기능, 도구, 설명을 한꺼번에 주지 않음
- 사용자의 필요에 따라, 실제 필요한 순간에만 추가 문서와 정보를 순차적으로 공개
- 이 아키텍처 덕분에 context window 효율 극대화 및 에이전트 유연성 확보 가능

### 시스템 프롬프트에선 스킬의 '간단 명료한 설명'만 즉시 제공해 부담을 최소화함

- 초기 system prompt에는 각 능력(스킬)의 간단한 설명문만 포함됨
- 예: "PDF를 처리할 수 있는 능력이 있음" 수준의 안내
- 에이전트는 본인이 어떤 일을 할 수 있는지 정도만 사전 인지함
- 구체적, 상세한 설명이나 실행 절차 등은 포함하지 않음으로써 메모리 자원을 아낌

### 사용자가 실제 해당 능력을 요청할 때만 세부 지침(skill.md)이 불러와짐

- 예로 'PDF 처리가 가능한 스킬'을 설명
- 사용자가 PDF 관련 작업을 요청하면 에이전트가 해당 skill.md를 조회함
- skill.md에는 해당 능력을 실질적으로 사용하는 방법, 단계별 지침 등 상세 절차가 포함
- 이처럼 필요 시에만 구체적 파일을 읽어들이는 구조

### skill.md 파일과 다른 참조 문서들이 ‘다층적 점진적 공개’ 구조로 필요에 따라 순차적으로 사용됨

- skill.md가 1차적으로 주요 지침을 제공하는 메인 파일 역할
- skill.md 내부에서 추가적으로 참고해야 하는 문서가 있으면 그것을 3단계로 불러오게 됨
- 예: 1단계(설명), 2단계(skill.md 세부 지침), 3단계(참고문서)
- 각각의 레이어를 필요시에만 열람해 context window 낭비를 최소화함

### 점진적 공개 구조로 에이전트는 유연성을 잃지 않으면서 효율과 기능성을 동시에 확보함

- 모든 능력을 한꺼번에 제공하는 대신, 실제 필요시만 상세 정보에 접근해 효율적임
- 에이전트는 항상 가볍고, 상황에 따라 필요한 만큼만 기능을 확장하는 구조
- 컨텍스트(문맥) 과부하 방지와 동적 리소스 할당이라는 두 마리 토끼를 잡음

### 예시(PDF 처리 스킬)를 통해 점진적 공개가 실제 어떻게 구현되는지 설명함

- 사용자의 특정 요청(예: PDF 처리)이 있을 때만 문맥에 기능 지침(skill.md 및 추가 문서)이 순차적으로 추가됨
- 이렇게 함으로써 AI 에이전트가 필요한 순간 올바른 리소스만 효율적으로 읽어들임
- 복잡성을 최소화하면서도 실전 사용에서는 강력한 확장성과 실용성을 제공

### 결론적으로 점진적 공개 기반의 Anthropic 스킬은 기존 도구 기반 방식의 비효율을 혁신적으로 대체함

- 결국 스킬이란, AI 에이전트를 위한 '모듈 및 능력 관리'의 새로운 표준임을 재차 강조
- 복잡한 도구 노출 대신, '필요시 레이어별 노출'로 AI의 능력·유지보수·확장성까지 모두 개선
- Anthropic은 '단순함'을 지향하는 철학을 실제로 구현했으며, 이로 인해 AI 시스템 개발 및 운용에 큰 변화를 예고
