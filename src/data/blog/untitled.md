## *유튜브 콘텐츠를 위한 전체 스택 RAG 에이전트 최초 라이브 빌드* 핵심 요약

- 영상은 채널 운영자가 오랫동안 만들어보고 싶었던 ‘개인 AI 코치(에이전트)’ 구현 과정을 다루며, 주제를 AI 에이전트 및 AI 코딩에 특화시킴
- 최종 목표는 자신의 모든 유튜브 콘텐츠로 훈련된 RAG(검색 기반 생성, Retrieval-Augmented Generation) 에이전트를 개발하여, ChatGPT처럼 자유롭게 질문·답변할 수 있는 채팅 인터페이스를 구현하는 것임
- 데이터 처리 과정에는 최근 성능 향상이 두드러진 “하이브리드 청킹(Hybrid Chunking)” 전략을 사용하며, 이 방법이 기존의 다양한(심지어 일부 커스텀) 청킹 방식보다 훨씬 뛰어남을 강조함
- 해당 하이브리드 청킹은 ‘Dockling’ 라이브러리를 채택해 적용되며, 발표자는 이 기술 도입에 큰 기대감을 표명함
- 먼저 유튜브 데이터에 대한 RAG 파이프라인을 구축한 뒤, 전체 라이브 개발이 진행될 것임을 명시
- RAG 파이프라인 구성 후, 핵심 데이터는 Supabase(슈퍼베이스)에 저장할 예정임
- 생성 에이전트 구현엔 ‘Pideantic AI’를 사용할 계획으로, RAG 모델의 프론트 취급함
- 이번 영상은 라이브 스트림 형식으로 기획되어, 시청자가 개발 과정을 단계적으로 따라갈 수 있도록 설계됨
- 이전에 사용했던 여러 다른 청킹 전략에 비해 하이브리드 방식의 실제 효과가 “엄청나다(insane)”고 언급하며, 본 영상에서 그 차이를 직접 시연할 예정임

## 세부 요약 - 주제별 정리

### 개인 AI 코치를 만들고자 하는 목표와 라이브 스트림의 배경이 상세히 설명됨

- 발표자는 자신의 채널에서 사용할 수 있는 AI 코치를 직접 구축하는 것을 오랫동안 바랐다고 언급
- 해당 AI 코치는 AI 에이전트와 AI 코딩에 모두 적용 가능하도록 설계됨을 강조
- 라이브 스트림을 통해 실시간으로 개발 과정을 보여주는 것이 목표임을 밝힘
- 시청자들이 프로세스를 쉽게 따라올 수 있도록, “이 라이브는 단순하게 RAG 파이프라인부터 시작해서 전체 빌드를 보여주는 구조”임을 설명

### 모든 유튜브 데이터를 학습한 RAG 기반 에이전트 구축이 핵심임이 강조됨

- 최종 결과물은 발표자의 유튜브 콘텐츠로 학습시킨 RAG 에이전트임
- 사용자는 ChatGPT와 유사한 자연어 채팅 UI에서 질문을 던지고, 맞춤형 답변을 받을 수 있음
- “아래에서 돌아가는 엔진은 고도화된 RAG 에이전트”라는 점을 명확히 언급
- AI 에이전트가 유튜브 데이터의 방대한 정보를 실시간 질의에 활용하도록 기획

### 하이브리드 청킹 전략이 RAG 성능에 결정적 영향을 주었음을 밝힘

- 데이터 분할(청킹)을 위한 여러 전략 중, 최근 하이브리드(Hybrid) 청킹이 가장 효과적이었다고 평가
- 발표자는 “이전에는 좀 더 커스텀한 청킹 등 다양한 전략을 실험했으나, 하이브리드가 월등했다”고 언급
- 하이브리드 방식에서 성능 개선이 매우 두드러져 “사실상 게임 체인저”였다고 칭찬
- 해당 청킹의 원리 및 성능 우위는 본 라이브 초반부에서 직접 구현 및 시연할 예정

### Dockling 라이브러리 도입이 구체적이고 핵심적인 의도로 제시됨

- 하이브리드 청킹을 위해 ‘Dockling’이라는 라이브러리를 본격 활용할 계획임을 밝힘
- Dockling은 본인의 RAG 파이프라인을 실질적으로 “새로운 수준”으로 발전시킨 기술임을 강조
- Dockling 도입은 발표자의 최근 경험을 바탕으로 한 선택임
- 라이브 내에서 Dockling의 실제 적용 예시와 성능 향상 사례를 보여줄 계획임

### 데이터 저장소로 Supabase를 활용한다는 구성 계획이 명확히 공유됨

- 세그먼트된 데이터 및 관련 정보는 Supabase 데이터베이스에 저장될 예정임
- Supabase 활용으로 빠른 데이터 접근성과 관리를 도모
- 데이터 파이프라인 구현 후, Supabase 내 데이터 입출력 작업까지 시연이 계획됨
- 클라우드 기반 스토리지가 RAG 파이프라인과 연계됨

### Pideantic AI를 통한 에이전트 개발 계획이 구체적으로 제시됨

- RAG 기반 데이터 구축 후, AI 에이전트의 본격적인 개발에는 ‘Pideantic AI’ 라이브러리를 사용할 계획임을 명확히 밝힘
- Pideantic AI는 RAG 파이프라인 위에 얹혀, 사용자가 대화할 수 있는 인터페이스 및 대화 분석을 담당할 구조임
- 최종적으로 사용자는 ChatGPT와 유사한 채팅 환경에서 자신만의 AI 코치와 상호작용 가능

### 전체 개발 순서와 라이브 구조가 사전에 체계적으로 정리되었음을 알 수 있음

- 라이브 스트림 처음에는 RAG 파이프라인 구축(유튜브 데이터 -> 하이브리드 청킹 -> Dockling 적용 -> Supabase 저장) 작업을 진행
- 그 이후, 에이전트 구현(Pideantic AI 도입) 및 대화형 인터페이스 개발로 이어지는 순서임을 언급
- 발표자는 각 단계별로 구체적인 전략과 방법론을 제공할 계획임

### 하이브리드 청킹 전략의 효과를 ‘이전 방식과 비교하여 압도적’이라 강조함

- “이전에도 직접 만든 커스텀 청킹 전략들을 많이 써봤지만, 이 방법이 단연코 최고”라고 발언
- 성능 차이가 실제로 “엄청나게(insane)” 크다고 판단하여, 라이브에서 직접 경험 공유 예정
- 해당 부분은 영상 초반부에 비중 있게 다룰 계획임

### 채팅 인터페이스의 형태와 사용자 경험이 미리 소개됨

- 완성된 제품은 사용자 입장에서 ChatGPT와 매우 흡사한 대화형 UI를 제공할 것임
- 누구나 직접 질문을 하고, 콘텐츠 데이터 기반으로 최적의 답을 얻는 시스템 구축이 궁극적 지향점임
- 인터페이스는 직관적이며, AI 에이전트가 채널의 풍부한 자료를 실시간 검색·분석해 답변함

### 전체 프로젝트가 유튜브 데이터 기반 AI 시스템 구축의 모범적 사례임을 보여줌

- 영상 내내, 기존 RAG+청킹 전략의 한계와, 하이브리드 방법의 게임 체인저 역할을 구체적으로 부각
- Dockling, Supabase, Pideantic AI 등 오픈소스·최신 라이브러리/플랫폼 채택이 잘 드러남
- 라이브 코딩 기반으로, 시청자가 실시간으로 개발 흐름·방법론을 따라올 수 있도록 구성됨
