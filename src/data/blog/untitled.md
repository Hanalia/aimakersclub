## *Postman이 Git-네이티브로 진화하며 API 구축 방식에 혁신을 가져옴* 핵심 요약

- 영상은 그동안 API 작업이 복잡하고 비효율적이었던 문제를 지적하며 시작함
- API 엔드포인트, 컬렉션, 테스트 관리가 각각 다른 도구 및 환경에서 따로 이뤄졌던 기존 워크플로의 문제점을 상세히 설명함
- Postman이 최근 대규모 업데이트를 통해 AI-네이티브 API 플랫폼으로 혁신, 그리고 Git-네이티브 지원 기능을 통합함을 강조함
- 업데이트 후 Postman의 컬렉션, 스펙, 목, 테스트 등 모든 리소스를 로컬 코드베이스에서 관리 가능한 YAML 파일로 저장할 수 있게 됨
- 파일들은 로컬에서 Postman 데스크톱 앱으로 바로 작업 가능하고, Postman 클라우드와 양방향 동기화 기능을 지원함
- Git diff 등 버전 관리 툴을 통해 코드와 동일하게 스펙, 테스트의 변경사항을 체계적으로 추적·관리 가능함을 시연함
- Postman의 에이전트 모드를 통해 전체 API 라이프사이클 자동 관리: 자연어로 요청만 하면 AI가 코드베이스 스캔, 경로 탐지, 스펙·테스트 생성, 목 세팅까지 자동화함
- 생성된 모든 변경 사항은 리뷰 가능한 git 커밋 단위로 정리되며, Claude, GPT, 셀프 호스트 등 다양한 AI 모델 및 MCP 서버와 호환됨
- GitHub, Sentry, Linear 등 기존 툴과의 연동도 자연스럽게 지원함
- 동일한 컬렉션을 로컬 및 CI 파이프라인(Pre-commit, GitHub Actions 등)에서 환경 변경 없이 반복적으로 사용할 수 있어 효율 향상
- 영상 제작자는 10년 넘게 Postman을 사용해왔으며, 이런 변화에 큰 만족을 표함
- 예시 저장소 및 Postman 공식 링크를 영상 설명란에서 제공함

## 세부 요약 - 주제별 정리

### API 작업의 기존 워크플로는 비효율과 단절 문제로 인해 복잡함을 유발함

- API 구현을 위해 엔드포인트, 스펙, 컬렉션, 테스트를 각각 분리된 툴과 위치에서 생성·관리해야 했음
- IDE나 코딩 에이전트에서 엔드포인트를 작성 후, 별도의 API 테스팅 도구(예: Postman)에서 컬렉션, 테스트를 다시 만들어야 했음
- CI(Continuous Integration) 환경에서 테스트를 운용하려면, 추가 설정 및 연동 작업이 필요했음
- 결과적으로 스펙, 컬렉션, 코드가 따로따로 존재하고, 버전 관리도 별도로 이루어져 혼란을 야기함

### Postman이 최근 대규모 업데이트로 AI-네이티브 및 Git-네이티브 플랫폼으로 도약함

- Postman은 영상 제작자가 10년 가까이 주요하게 사용해 온 협업·API 테스팅 도구임
- 2024년 상반기, Postman은 대규모 업데이트를 발표
- 이번 업데이트로 AI-네이티브 API 플랫폼이 되었고, Git-네이티브 지원 기능도 도입함
- 업계에서 큰 반향을 일으키는 변화라고 언급

### 컬렉션, 스펙, 목, 테스트 등 모든 리소스를 YAML 파일로 코드베이스에 직접 저장 가능해짐

- Postman 리소스들을 로컬 프로젝트 폴더 내 YAML 파일로 쉽게 저장 및 관리할 수 있음
- 파일 기반이므로 Git/SVN 등 버전관리 시스템과 완벽하게 연동 가능함
- YAML 형식은 유지보수가 쉽고, 협업 및 자동화에 적합함
- 이 기능은 데스크톱 앱에서 직접 사용하거나, 코드 에디터(IDE)에서도 자유롭게 편집 가능함

### Postman의 로컬-클라우드 양방향 동기화로 작업 유연성과 협업성이 극대화됨

- 로컬 코드베이스와 Postman 클라우드 간에 실시간으로 두 방향 동기화 지원
- 한 곳에서 변경 시 다른 곳에도 자동 반영되어, 실시간 협업 및 중앙 집중 관리가 가능함
- 사용자는 필요에 따라 로컬 혹은 클라우드 환경에서 자유롭게 전환해 작업할 수 있음

### Git diff와 같은 도구로 스펙과 테스트 변화를 손쉽게 추적하게 됨

- 스펙, 테스트는 코드와 동일하게 버전 관리(git diff 등)를 활용하여 체계적으로 변경 이력을 관리할 수 있음
- 개발자가 코드에 변경을 가하면, 해당 항목의 변동사항을 git diff 명령어 등으로 바로 확인 가능함
- 리뷰 과정에서 모든 변경사항과 이력을 투명하게 검토할 수 있음

### Postman 에이전트 모드가 전체 API 라이프사이클의 자동화를 이끌어냄

- 에이전트 모드를 활성화하면, 자연어 명령(예: "이 Flask 앱에 대한 컬렉션을 생성해줘")만으로 자동화 지원
- Postman이 코드베이스 스캔 → 경로(route) 탐지 → 스펙 생성 → 테스트 생성 → 목 작성 과정을 자동 실행
- 사용자가 별도의 수작업 없이도, 전체 API 관련 리소스를 일관되게 관리할 수 있음
- 이 과정에서 모든 변경 내역이 git 커밋 단위로 자동 기록됨

### 다양한 AI모델과 인프라 환경 지원으로 유연성과 확장성이 확보됨

- 사용자가 선호에 따라 Claude, GPT, 자체 호스팅 모델 등 다양한 AI 모델 선택 가능
- MCP 서버 지원을 통해, API 테스트 자동화 및 인프라 관리에서도 유연하게 대처 가능함
- 조직별, 프로젝트별로 맞춤형 환경 구축이 한층 쉬워짐

### GitHub, Sentry, Linear 등 실무 툴들이 Postman과 자연스럽게 연동 가능함

- Postman이 GitHub 등 코드 호스팅 서비스, Sentry, Linear와의 연동을 공식 지원함
- 기존에 팀에서 사용하는 업무 관리/버그 추적/이슈 관리 도구와의 통합이 쉬워, 워크플로 최적화에 탁월함
- 별도의 복잡한 설정 없이, 관련 정보를 API 테스트 및 문서화에도 즉시 활용 가능

### Postman 컬렉션은 로컬 개발환경과 CI 파이프라인 모두에서 동일하게 재사용 가능함

- 작성된 컬렉션, 스펙, 테스트는 로컬 개발 환경과 CI 파이프라인(Pre-commit, GitHub Actions)에서 동일하게 사용됨
- 환경에 따라 다시 작성하거나 별도의 셋업 없이 동일한 파일, 동일한 테스트 코드가 구동됨
- 반복적이고 일관된 테스트, 배포가 가능하여 생산성과 신뢰성이 증가함

### 영상 제작자는 Postman의 발전에 큰 만족을 표하며 예시 리포지토리도 함께 공유함

- 영상 제작자는 약 10년간 Postman을 사용해왔으며, 이번 변화에 대해 "분명히 레벨업했다"고 평가함
- Postman 공식 홈페이지 링크 및 직접 작성한 GitHub 예시 리포지토리 링크를 설명란에 함께 안내함

### 사용자는 별도 환경 변경이나 설정 없이 Postman의 모든 기능을 즉시 활용할 수 있게 됨

- YAML 파일 기반이라 별도 환경 구성, 호환성 문제 없이 빠르게 적용 가능
- 개발 속도, 협업성, 자동화 등 여러 측면에서 "아름다운 경험"이라 극찬함
