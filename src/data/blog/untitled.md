## *AI 에이전트의 기능을 극대화하는 가장 단순한 방법(스킬)* 핵심 요약

- Anthropic이 최근 선보인 ‘스킬’(skills)은 AI 발전에 있어 가장 중요한 혁신 중 하나로 평가됨
- 스킬의 중요성은 ‘단순함’에 있으며, Anthropic의 모토인 “더 단순할수록 더 좋다”와 일치함
- AI 에이전트에 너무 많은 기능(도구, 대화 이력, 시스템 프롬프트 등)을 한 번에 제공하면 ‘컨텍스트 윈도우 과부하’로 성능이 저하됨
- 기존 방식(MCP 서버 사용 등)은 불필요한 도구까지 한꺼번에 주입, 필요 없는 정보로 에이전트를 혼란스럽게 함
- 스킬의 핵심은 ‘점진적 공개(progressive disclosure)’로, 에이전트가 필요한 순간에만 필요한 정보를 단계적으로 받아들일 수 있도록 설계됨
- 초기에는 기능(스킬)에 대한 간단한 ‘설명’만 시스템 프롬프트에 전달하여 에이전트가 “능력이 있다”는 사실만 알게 함
- 사용자가 실제로 특정 기능(예: PDF 처리)을 요청할 경우에만 에이전트가 해당 스킬의 구체적 설명(skill.md 파일)을 읽음
- skill.md 파일에는 해당 기능에 대한 사용법 및 자세한 안내가 정리되어 있음(두 번째 공개 단계)
- 필요한 경우, skill.md가 추가적인 참고자료(Documents 등, 세 번째 공개 단계)를 참조하여 정보를 더 가져올 수도 있음
- 이로써 에이전트는 작업이 진행되면서 점진적으로 필요한 오직 최소한의 컨텍스트만을 받아 효율적으로 동작할 수 있음

## 세부 요약 - 주제별 정리

### Anthropic의 ‘스킬’은 최근 AI에서 가장 중요한 혁신 중 하나로 부상함

- Anthropic이 세계에 ‘스킬’을 공개한 지 오래되지 않았으나, 이 기술은 AI 영역에서 큰 변화를 이끌고 있음
- 영상 제작자는 최근의 AI 발전 가운데 ‘스킬’이 가장 중요한 성과 중 하나라고 강조
- 혁신의 진정한 가치는 이 기능의 단순함에 있으며, 이는 Anthropic이 추구해온 가치와도 연결됨

### ‘단순함’이 Anthropic 기술 혁신의 핵심이며 불필요한 복잡성은 배제함

- Anthropic의 기본 철학은 “단순할수록 좋다”(the simpler, the better)는 신념
- 영상 초반에 소개된 ‘스킬’의 간결함이 전체 접근방식을 특징지음

### 너무 많은 정보와 도구를 제공하면 에이전트가 오히려 제 역할을 못하게 됨

- AI 에이전트에게 다양한 능력을 부여하고 싶겠지만 동시에 과도한 정보 제공은 오히려 독임
- 시스템 프롬프트, 도구(tool), 대화 이력(conversation history) 등 모든 정보를 한 번에 제공하면 에이전트의 컨텍스트 윈도우가 과부하됨
- 윈도우가 넘치면 맥락을 파악하지 못해 성능 저하나 에러가 발생할 수 있음

### 기존 방법(MCP 서버)은 불필요한 모든 도구를 한 번에 제공해 ‘과부하’를 유발함

- MCP 서버 등 전통적 접근방식에서는 사용 여부와 관계없이 수많은 도구를 에이전트에게 미리 제공함
- 실제 필요한 도구가 아닌 것까지도 한꺼번에 노출되며, 불필요한 자원 소모와 혼란을 야기함
- 이런 구조는 비효율적일 뿐 아니라, 특정 대화에서 절대 쓰이지 않을 도구까지 입력되어 컨텍스트 공간을 낭비함

### 스킬의 진정한 강점은 ‘점진적 공개’ 방식에 있음

- 스킬의 핵심은 ‘progressive disclosure’, 즉 필요한 순간에 필요한 정보만 단계적으로 제공하는 원리
- 초기에는 시스템 프롬프트에 스킬(기능)에 대한 요약적 설명만 포함시켜 에이전트가 자신의 능력을 인지함
- 예를 들어, ‘PDF 처리 스킬’이 있다면, “PDF 관련 작업을 수행할 수 있다”는 정도만 안내
- 이 단계에서는 기능의 구체적 작동방식이나 세부 지침까지는 노출하지 않음

### 사용자의 요청이 들어오면 에이전트가 구체 스킬 정보를 skill.md에서 읽음

- 사용자가 실제로 특정 기능(예: “PDF를 처리해달라”)을 요청할 때까지 상세 정보는 제공되지 않음
- 해당 요청 시, 에이전트는 ‘skill.md’라는 메인 문서를 읽어 자세한 지시와 동작 방식을 획득함
- skill.md에는 실제 동작을 위한 상세 사용법, 인수 및 결과 기대치 등 구체적 설명이 담겨 있음
- 이 과정이 점진적 공개의 두 번째 레이어에 해당

### 추가 정보가 필요하면 skill.md가 참조하는 별도의 자료를 불러오는 세 번째 레이어가 작동함

- skill.md 외에도, 보다 심화된 안내가 필요하다면 추가 문서(docs 등)로부터 정보를 가져올 수 있음
- 이로써 에이전트는 작업 과정에서 필요한 만큼만 점진적으로 컨텍스트를 확장할 수 있음
- 점진적 공개는 각 작업 단계별로 정보 제공이 누적·보완되는 구조임

### 단계적으로 컨텍스트가 쌓이면서 최적의 효율성과 성능을 확보할 수 있음

- 점진적 공개 덕분에, 에이전트는 초반에는 복잡한 정보를 부담하지 않고 상황에 따라 필요한 내용만 받아들임
- 결국 컨텍스트 윈도우의 효율적 활용이 가능해지고, 과부하 없이 더 다양한 기능 확장 가능
- 불필요한 정보 입력 최소화로 작업 맥락 파악력과 응답 품질이 향상됨

### Anthropic의 스킬 구조는 맞춤형 구현(custom implementation)에도 쉽게 적용될 수 있음

- 영상 제작자는 자신이 실습 중인 커스텀 구현(custom implementation)에도 이 스킬 구조를 사용하는 예를 언급
- Anthropic의 모든 스킬 사례에서 해당 구조(skill.md, 참조 문서 등)를 일관되게 적용하고 있음을 밝힘
- 구조적 단순함과 확장성 덕분에 다양한 에이전트 개발에 용이하게 접목 가능함

### AI 에이전트 개발에서 핵심은 ‘단계적 기능 노출’과 ‘최소 컨텍스트 소비’임을 실례로 강조함

- 영상의 전체 구조는 단계별 공개의 흐름을 예시와 함께 반복적으로 설명
- PDF 처리 스킬의 예시는 실제 AI 에이전트에게 어떻게 기능이 활성화되는지 구체적으로 보여줌
- 결론적으로, 장황한 도구 셋업이 아니라 ‘간단한 소개 → 실제 요청 → 상세 정보 로딩’의 흐름이 가장 효율적임을 시현함
