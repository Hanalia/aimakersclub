## *Claude Code로 아름다운 다이어그램을 만드는 전체 워크플로우* 핵심 요약

- Claude Code와 같은 코딩 에이전트는 본래 시각적 표현 능력이 부족해, 시각적 설명이나 다이어그램 생성에 한계가 있음
- 발표자는 이러한 한계를 극복하기 위해 전체 워크플로우를 하나의 스킬로 패키징하여 코딩 에이전트가 실질적이고 미려한 다이어그램을 쉽게 만들도록 함
- Excalra 다이어그램 스킬을 활용해 Claude Code 등 코딩 에이전트에게 시각적 논증 및 다이어그램 작성 능력을 부여함
- 워크플로우의 핵심은 JSON 파일을 생성하여 Excal 다이어그램을 만들고, 이를 시각적으로 검증 및 반복 수정하는 과정임
- 생성된 다이어그램을 직접 렌더링하여 PNG로 된 이미지 파일을 살펴보고, 결함이나 미비점을 자동으로 개선함
- 이러한 자기 검증(self-validation) 프로세스는 코딩 에이전트가 비주얼 작업에서 약점을 보완하도록 돕고, 결과물의 품질을 크게 향상시킴
- Excalra 다이어그램 스킬은 사용자의 브랜드, 원하는 색상 테마 등 맞춤화가 가능하도록 설계됨
- 주된 프롬프트 설계 철학은 단순히 블록을 나열하는 대신 시각적으로 논증 구조와 라벨로 개념을 잘 설명하는 다이어그램 제작임
- 스킬 설정 방법도 자동화되어, 코딩 에이전트가 README를 읽고 수동 단계를 알아서 처리, 사용자는 곧바로 다이어그램 요청 가능
- 실사용 예시로 PDF와 코드베이스 등 다양한 데이터에 적용하며, 반복 작업 시간을 매주 수 시간 단축할 수 있음

## 세부 요약 - 주제별 정리

### 코딩 에이전트는 기본적으로 비주얼 도구로 적합하지 않음을 지적

- Claude Code 등 코딩 에이전트는 텍스트 기반으로 동작하여 시각적 문제 설명이나 다이어그램 생성에는 약함
- 시각적 표현이나 설명을 원하는 경우, 다수의 프롬프트와 추가 설정이 필요함
- 만족스러운 시각적 결과물을 얻기가 어렵다는 점이 근본적인 문제로 지적됨

### 전체 워크플로우를 스킬로 패키징하여 접근성을 높임

- 발표자는 자신이 개발한 전체 워크플로우를 하나의 "스킬"로 패키징함
- 이 스킬은 Claude Code 혹은 다른 코딩 에이전트가 다이어그램을 쉽고 빠르게 제작할 수 있게 함
- 미적으로 뛰어나면서 실용적인 다이어그램을 효율적으로 만드는 것이 핵심 목표임

### 코딩 에이전트에게 시각적 논증 능력을 체계적으로 부여함

- 기본적으로 Claude Code 등은 시각적으로 "주장(argue)"하는 기능이 내장되어 있지 않음
- Excalra 다이어그램 스킬은 시각적으로 논증하고 구조적으로 설명하는 방법을 코딩 에이전트에 가르침
- 이를 통해 다이어그램이 단순한 블록 조합에서 벗어나, 전달하고자 하는 개념과 논리를 명확히 반영

### Excalra 다이어그램 스킬의 주요 작동 프로세스 소개

- 스킬은 Claude Code에 적용되는 주요 “지시문 세트(instruction set)”로 구성됨
- Claude Code는 이 지시문에 따라 Excal 다이어그램용 JSON 파일을 자동 생성함
- 사용자는 이 JSON 파일을 기반으로 다이어그램을 만들고 다듬을 수 있음

### 워크플로우는 생성과 검증 반복을 통해 결과물을 최적화함

- 스킬의 강점은 단순히 다이어그램을 생성하는 데 그치지 않고, 자체적인 결과물 검증 과정을 포함함
- 생성된 다이어그램을 실제로 렌더링(render)하여 이미지(PNG)로 저장
- 이후 이 이미지를 분석하여 미흡한 부분이나 오류를 자동으로 찾아내고 반복적으로 수정하며 결과를 개선함
- 이러한 자기 검증(self-validation) 루프를 통해 다이어그램 품질이 월등히 향상됨

### Excalra 다이어그램 스킬 설치 및 세팅 방법의 자동화

- 코딩 에이전트에 “Excalra 다이어그램 스킬” 설치를 요청하면, 에이전트가 GitHub 저장소의 README를 읽고 자동으로 매뉴얼 단계를 수행
- 사용자는 별도의 복잡한 세팅 없이 바로 다이어그램 생성 프롬프트를 입력 가능
- 간단한 데모에서 보인 것처럼, 복잡한 설정없이 즉시 다이어그램 제작에 활용할 수 있음

### 맞춤 색상 및 브랜드 적용 등 커스터마이징 기능 강조

- 다이어그램의 색상 테마(Color Scheme), 브랜드 이미지 등 개인적/회사별 맞춤화가 쉽게 가능함
- 사용자가 선호하는 스타일 및 브랜딩에 맞춘 일관성 있는 시각 자료 제작이 가능

### 프롬프트 설계 철학은 ‘시각적 논증 구조’에 집중

- skill.mmd 파일 내부의 프롬프트 설계에서는 “시각적으로 논증하는 방법”에 초점을 둠
- 기존의 코딩 에이전트 다이어그램 생성 결과는 단순한 블록 나열에 그치는 경우가 많음
- Excalra 스킬을 사용하면, 구조와 라벨링을 활용해 전체 개념을 이해하기 쉬운 형태로 시각화할 수 있음

### 다이어그램 생성 적용 사례와 시간 절약 효과

- 발표자는 Excalra 다이어그램 스킬을 PDF 문서, 코드베이스 등 다양한 데이터에 적용해 봄
- 스킬 활용을 통해 일상적으로 소비되는 반복적인 다이어그램 작업 시간을 매주 수 시간 단축함
- 시작점으로서 높은 품질을 자동 생성하여, 추가 수정 없이 바로 업무 활용 가능

### 소스코드 공개 및 다양한 용도 제안

- 발표자는 누구나 이 GitHub 리포지터리(clone this repo)를 복제하여 사용해 보기를 권장함
- PDF 문서, 코드베이스 등 다양한 입력 데이터에 바로 적용하여 다이어그램을 제작하는 것을 제안
- “가능성은 끝이 없다(possibilities are endless)”는 언급을 통해 활용의 폭이 넓음을 강조함
