## *스택드 디프(누적 변동) 방식이 AI 코드 리뷰의 효율을 극대화함* 핵심 요약

- 영상은 AI 기반 코드 리뷰의 강력함을 강조하며, 이를 극대화하는 '스택드 디프(stacked diffs)' 워크플로우를 집중적으로 소개함
- 스택드 디프는 대규모 변경이 아닌, 작은 변경을 하나씩 차례대로 쌓아 올려 전체 기능을 완성하는 방식임
- 각 단계의 변경사항(diff)에 대해 독립적인 코드 리뷰와 AI 피드백을 받을 수 있어, 각 변경의 품질과 집중도를 높일 수 있음
- 이 방식은 전통적 PR(pull request)처럼 매 승인마다 기다릴 필요 없이, 여러 개의 변경사항을 연속적으로 작업이 가능하여 효율적임
- 그래파이트(Graphite) 플랫폼을 활용하면 스택드 디프 워크플로우를 CLI(커맨드라인 툴)와 대시보드로 손쉽게 진행할 수 있음
- 그래파이트의 명령줄 인터페이스는 GitHub CLI와 유사하지만, 스택드 디프에 최적화되어 있음
- 그래파이트 대시보드에서는 PR 관리, AI 코드 리뷰, 병합 등 모든 작업을 한곳에서 처리 가능함
- 실제 그래파이트 데모 과정을 통해, 첫 스택 디프 생성부터 연속적인 PR 작성, AI 기반 리뷰, 대시보드 관리 등 전체 과정이 직관적으로 소개됨
- 각 변경마다 AI(예: Claude code)로부터 집중적 리뷰를 받아, 한 번에 큰 PR을 검토하는 기존 방식보다 퀄리티와 결과가 탁월하다고 설명함
- 영상 마지막에서는, 이 모든 과정을 그래파이트와 함께 협업해 진행했음을 밝힘

## 세부 요약 - 주제별 정리

### AI 기반 코드 리뷰는 검증 과정의 필수 요소로 자리 잡음

- AI 코드 리뷰 도구는 매우 강력하며, 코드 품질 검증에 반드시 활용되어야 한다고 주장함
- AI가 코드의 변경 사항을 빠르게 분석하고, 잠재적 문제나 개선점을 지적해줌
- 기존 개발 흐름에서도 AI 코드 리뷰는 점점 표준으로 자리잡고 있음
- 영상에서는 AI 코드 리뷰를 최대한 잘 활용할 수 있는 '특별한 워크플로우'로서 스택드 디프를 소개

### 스택드 디프란 작은 변화의 '누적' 방식을 적용해 큰 기능을 완성하는 기법임

- 기존의 전통적인 PR 방식에서는, 한 번에 큰 변화를 생성하고 그에 대한 리뷰를 받음
- 이 방식으로는 매번 변경사항 승인 대기, 반복 작업 등 비효율적인 면이 있었음
- 스택드 디프는 하나의 큰 작업을 여러 개의 작은 변경(diff)으로 쪼개서, 순차적으로 쌓아올리는 구조임
- 각 변경(diff)이 이전 단계에 종속되어 차례로 누적되며, 전체적으로 큰 기능 혹은 개편이 완성됨
- 덕분에 각각의 변경에 대해 빠른 피드백과 리뷰가 가능하고 작업 병목이 줄어듦

### 스택드 디프 방식은 AI 코드 리뷰에 '집중도와 효율'을 가져다줌

- 소규모 변경(diff)마다 AI로부터 레이어별 집중 리뷰를 받을 수 있음
- 각 변경이 작고 명확하므로, AI가 더 정확한 컨텍스트 내에서 피드백 제공 가능
- 큰 PR을 한 번에 리뷰하는 것보다, 작은 단위를 분리해 단계적으로 리뷰하면 오점이 줄어듦
- AI의 코드 리뷰 결과가 더 세밀하고 신속하면서도 실용적으로 변환됨

### 그래파이트(Graphite) 플랫폼이 스택드 디프를 쉽게 구현할 수 있게 지원함

- 그래파이트는 스택드 디프 작업에 특화된 플랫폼임
- GitHub CLI와 유사한 자체 커맨드라인 툴을 제공하여, 기존 개발자 환경과 연속해서 쓸 수 있음
- 각 레이어(diff)별로 독립적인 작업 브랜치와 PR 생성을 CLI로 쉽게 할 수 있음
- 추가적으로, 대시보드를 통해 전체 PR 스택 및 각각의 상태를 직관적으로 관리 가능함

### 그래파이트에는 AI 코드 어시스턴트가 내장되어 있어 리뷰 자동화가 가능함

- 그래파이트 내에는 AI 코딩 어시스턴트가 내장되어 있음
- 각 PR별로 자동 코드 리뷰와 피드백을 받을 수 있음
- 변경 승인, 병합 등도 대시보드에서 손쉽게 처리 가능
- GitHub 레포지토리로 직접 이동하지 않아도, 모든 작업을 한 플랫폼 내에서 통합적으로 진행 가능

### 그래파이트 입문 과정과 실제 워크플로우가 직관적으로 안내됨

- 플랫폼 가입 시 데모가 제공되어 입문자가 쉽게 따라할 수 있도록 설계됨
- 예시 과정을 통해, 첫 스택 디프 생성 → PR 작성 → 추가 PR을 연이어 생성(스택 형식) 하는 과정을 시연함
- 실제 명령어 예시 화면도 영상에 포함되어 있음
- 두 번째, 세 번째 PR을 차례대로 첫 PR 위에 쌓는 컨셉을 직관적으로 보여줌

### 그래파이트 대시보드를 통한 PR 관리와 병합, AI 리뷰 활용법이 소개됨

- 모든 PR 스택을 대시보드에서 체계적으로 확인 가능
- 그래파이트 에이전트를 통해 각 PR별 AI 리뷰 요청 및 적용 가능
- 모든 변경 사항들은 대시보드 내에서 병합 작업도 손쉽게 이뤄짐
- 필요 시 GitHub 레포지토리에도 동일하게 결과를 반영함

### 각각의 변경을 별도의 feature 브랜치에서 관리하고 머지하는 매우 지능적인 Git 흐름을 활용함

- 각 단계의 변경(diff)은 별도 'feature 브랜치'에서 생성
- 이전 단계 PR이 승인되면, 다음 스택의 PR을 그 위에 쌓는 방식으로 관리
- 최종적으로 단계별 병합을 마치고 모든 변경을 메인(main) 브랜치로 통합
- 기존 Git 방식과 유사하나, 훨씬 체계적이고 명확한 변경 누적(workflow) 제공

### 한 번에 큰 PR을 한번에 리뷰하는 것보다 '작은 변화의 스택' 방식이 월등히 효율적이라는 경험적 결과가 강조됨

- 실질적으로 Claude code(예시 AI 도구) 등으로 각 PR을 개별 심층 리뷰함
- 이 방식으로 얻는 피드백의 퀄리티 및 오류 발견률이, 대형 PR 한 번 검토할 때보다 크게 높다고 평가
- 스택드 디프 리뷰 결과가 기존의 일괄 리뷰와는 '차원이 다르다'고 표현

### 영상 마무리에서 스택드 디프와 그래파이트 협업 경험 및 참조 자료를 안내함

- 전체 과정을 그래파이트와 협력하여 진행했음을 언급
- 구체적 워크플로우 확인을 위한 추가 영상 링크와 구독자 참고용 자료 공유(영상 설명란)
- 본 영상이 스택드 디프와 그래파이트의 핵심 사용법 및 AI 코드 리뷰 강화 효과를 전달하는 데에 중점 두었음을 반복 강조
