## *AI 에이전트를 강화하는 가장 간단한 방법, Skills 소개* 핵심 요약

- 본 영상에서는 Anthropic이 최근 공개한 'Skills(스킬)'가 AI 분야에서 가장 중요한 진보 중 하나라고 강조함
- Skills는 단순하면서도 강력한 구조로, 사용이 매우 간편하다는 점에서 높은 평가를 받음
- 기존의 MCP 서버 방식은 많은 도구와 정보를 한 번에 제공해 에이전트의 컨텍스트 윈도우를 쉽게 과부하 시키는 문제가 있었음
- Skills는 ‘점진적 공개(progressive disclosure)’라는 전략을 도입하여 필요한 순간에만 세부 정보를 추가로 제공함
- 시스템 프롬프트에는 오직 해당 에이전트가 사용할 수 있는 기능(스킬)에 대한 간략한 설명만 전달됨
- 이후 사용자가 PDF 처리 등 특정 요청을 할 때에만 skill.md 파일의 상세 지침이 컨텍스트에 추가되고, 필요할 경우 참조 문서까지 순차적으로 입력됨
- 이러한 1-2-3단계 점진적 공개 방식은 불필요한 정보로 인한 에이전트 성능 저하를 방지하고, 효율적인 컨텍스트 관리가 가능하게 함
- PDF 처리 스킬 등 구체적 예시를 들어, Skills의 구조와 작동 방식을 설명함
- Anthropic의 “단순할수록 좋다(The simpler the better)” 철학이 Skills 시스템 전반에 반영됨
- Skills 방식은 맞춤형 구현에도 활용 가능하며, 확장/응용 잠재력이 큼

## 세부 요약 - 주제별 정리

### Anthropic의 Skills는 최근 AI 발전에서 가장 중요한 혁신으로 평가받음

- Anthropic은 최근 ‘Skills’라는 기능을 공개하여 AI 에이전트에 새로운 가능성을 제시함
- 영상 제작자는 Skills가 최근 AI 발전 중 가장 중요한 혁신의 하나라고 강조함
- Skills의 핵심 가치는 ‘심플함’에서 기인함
- Anthropic은 제품 설계 모토로 “더 단순할수록 더 좋다(The simpler the better)”를 내세움
- Skills 구조가 기존 복잡한 도구 및 시스템 대비 월등히 간결함

### 기존 MCP 방식은 너무 많은 도구를 한 번에 전달해 에이전트 성능을 저하시키는 문제가 있었음

- 전통적 MCP 서버 방식 사용 시, 대화 에이전트에 다양한 도구와 정보를 한꺼번에 제공함
- 여러 도구가 프롬프트에 한 번에 들어가면, 에이전트의 컨텍스트 윈도우(작업 메모리 영역)를 금방 소진시킴
- 컨텍스트 윈도우가 과부하되면, 에이전트는 핵심 정보만 참조하는 데에도 어려움을 겪음
- 도구가 실제로 필요하지 않은 상황에서도 모두 입력되어 비효율적임
- 이러한 방식은 사용성 및 성능 모두에 악영향을 미칠 수 있음

### Skills는 점진적 공개(progressive disclosure) 방식을 도입하여 효율적으로 정보를 제공함

- Skills 시스템의 본질은 ‘점진적 공개’임
- 필요한 도구 및 정보만, 필요한 순간에만 에이전트에 단계적으로 제공함
- 모든 기능의 세부 지침을 한꺼번에 제공하지 않고, 처음엔 극도로 간략한 설명만 전달함
- 업무 과정에서 실질적으로 필요할 때에만 상세 지침이 로드됨
- 이런 방식으로 컨텍스트의 효율성과 집중도를 높임

### 시스템 프롬프트에는 오직 기능 설명만 입력되어 에이전트의 초기 부담을 최소화함

- 첫 단계 시스템 프롬프트에는 각 기능(스킬)의 짧고 명확한 소개만 기입함
- 예) “이 에이전트는 PDF 처리 기능을 사용할 수 있습니다”와 같은 설명이 제공됨
- 상세한 실행 방법이나 매뉴얼은 이때 제공되지 않음
- 에이전트는 자신이 어떤 능력을 보유하게 되었는지 명확히 인지함

### 사용자가 특정 기능(PDF 처리 등)을 요청할 때만 상세 설명이 컨텍스트에 추가됨

- 사용자가 예를 들어 “PDF 문서를 분석해줘”와 같은 요청을 했을 때, 에이전트가 스킬 세부 내용을 확인함
- 이 과정에서 skill.md 파일이 로드되어 세부적인 동작 지침과 프로세스가 전달됨
- skill.md는 해당 스킬(예: PDF 처리) 실행에 필요한 모든 설명이 포함된 핵심 파일임
- 복수의 스킬을 사용할 때에도 각각 해당 상황에서만 상세 문서가 순차적으로 주입됨

### skill.md 파일이 2차 점진적 공개의 핵심, 추가 문서가 3차 점진적 공개를 담당함

- 전체 구조는 단계적으로 더 많은 정보를 에이전트에게 전달하는 3중 레이어 방식임
- 1차: 시스템 프롬프트 내 스킬 설명
- 2차: skill.md 파일(상세 설명) 로드
- 3차: skill.md가 필요할 때 참조하는 추가 문서(예: 부가 설명서, 예시 파일 등)
- 실제로 필요한 상황에서만 정보를 ‘점차적으로’ 공개함

### 이러한 방식은 불필요한 정보 입력을 줄이고, 컨텍스트 오염 없이 다양한 기능 확장이 가능하게 함

- 각 단계마다 최소한의 필수 정보만 전달되어, 작업 부담이 크지 않음
- 이전과 달리 모든 잠재적 기능을 한 번에 로드하지 않으므로, 컨텍스트 윈도우 오염 최소화 가능
- 추가적 기능 확장 시에도 시스템 프롬프트에 간단한 설명만 추가하면 됨
- 필요할 때만 상세 설명, 부가자료가 투입되어, 전체 처리 효율이 매우 높아짐

### Skills 구조는 커스텀 구현 및 다양한 확장 응용에도 손쉽게 적용될 수 있음

- Anthropic에서 제공하는 skill.md류 문서를 바탕으로 다양한 커스텀 스킬 구현이 가능함
- 새로운 기능이나 목적에 따라 관련 문서를 준비하여 같은 점진적 방식으로 적용 가능
- 제작자는 앞으로도 다양한 Skills 스킬 예시와 맞춤형 구현 방법을 소개할 것임을 언급함

### PDF 처리 스킬을 구체 사례로 동작 방식을 상세히 설명함

- 우선 시스템 프롬프트에 “PDF 처리 기능을 사용할 수 있다”는 설명만 기록
- 사용자가 PDF 분석 등 요청을 하면, skill.md에서 해당 기능 실행에 필요한 단계별 설명 제공
- skill.md가 추가 문서(내용 해설, 샘플 등)을 다시 참조하면, 실제 PDF 처리 작업에 활용됨
- 이 3단계 과정이 실제 Skills 시스템의 대표적 활용 사례로써 제시됨

### Anthropic의 철학은 Skills 전체 기능 설계에 일관되게 반영되어 있음

- “더 단순할수록 더 좋다(The simpler the better)”라는 모토가 Skills 설계 전반에 담겨 있음
- 복잡성 최소화 및 기능 효율성 극대화를 동시에 추구
- 컨텍스트 관리 최적화와 유연한 확장 모두를 가능케 함

### Skills 시스템은 AI 에이전트의 실제 활용성과 맞춤형 개발에 큰 시사점을 제공함

- Skills의 점진적 공개 방식 덕분에, AI 에이전트에 다양한 기능을 추가하더라도 성능 저하가 적음
- 기업 또는 개발자는 자신만의 다양한 스킬을 제작하여 상황별로 적용할 수 있음
- 세부 문서화 및 기능 확장성을 바탕으로 AI 활용의 전략적 폭을 넓혀줌
