## *클로드 코드의 진짜 목적(생각보다 더 크다)* 핵심 요약

- 본 영상에서 제작자는 자신의 스마트폰으로 "클로드 코드"라는 맞춤형 인공지능 에이전트에게 스스로를 개선하라고 직접 지시함
- 목적은 클로드 코드가 자신의 파일을 편집하여 ‘순차적 사고 MCP(Multi-Component Processing) 서버’를 추가하도록 하는 것임
- 편집 대상 파일은 제작자가 미리 보여준 Sentry가 연결된 텔레그램 봇의 코드임
- 제작자는 이전에 오브시디언(Obsidian) 통합 예시에서 이미 ‘순차적 사고’ 기능을 구현한 적이 있음을 언급
- 클로드에게 요청을 보내는 과정이 스마트폰 상에서 실시간으로 진행되며, 클로드는 요청을 받아 바로 코드 수정에 착수함
- 제작자는 실시간으로 클로드가 옵션값이 저장된 부분을 변경하는 장면을 관찰함
- 최종적으로, 클로드 코드가 ‘순차적 사고 MCP’와 해당 서버 내 모든 도구 사용 권한을 부여하는 내용까지 자동으로 추가함
- 이 과정을 통해 클로드 코드가 스스로를 신속히 개선할 수 있음을 직접 입증함

## 세부 요약 - 주제별 정리

### 클로드 코드가 스스로를 개선하는 실험이 스마트폰으로 시도됨

- 제작자는 영상 시작에서 “클로드 코드가 스스로를 개선하도록 하겠다”고 언급함
- 이 작업은 개발용 컴퓨터가 아닌 자신의 스마트폰에서 수행되는 점을 강조
- 맞춤형 클로드 에이전트에게 직접 명령을 내림으로써 인공지능이 자기 개선을 함을 실시간으로 보여줌
- 실험의 목적은 클로드 코드의 자기 개선 가능성과 편의성을 시현하는 데 있음

### ‘순차적 사고 MCP 서버’ 추가 요청 및 구체적 명령 과정이 공개됨

- 사용자(제작자)는 “내 클로드 에이전트에 순차적 사고 MCP 서버를 추가하라”는 구체적 명령을 내림
- 편집할 대상은 “Sentry가 연결되어 있는 텔레그램 봇”의 파일임을 명확하게 지정
- 클로드에게 “이전에 오브시디언(Obsidian) 통합 예제에서 순차적 사고를 이미 연결했다”고 언급, 참고하도록 지시함
- 본 요청은 스마트폰 내에서 즉시 처리되고 클로드가 입력을 받아들여 작업에 착수함

### 클로드 코드가 실시간으로 파일을 직접 수정하는 과정이 보여짐

- 영상 중 제작자가 “작업 요청을 보냈다(bom), 클로드가 바로 타이핑을 시작했다”고 언급
- 클로드는 즉시 요청에 따라 서버 관련 코드를 작성하기 시작함
- 제작자는 ‘옵션이 저장된 부분’을 집중적으로 관찰하며, 클로드가 해당 영역을 어떻게 편집하는지 지켜봄
- 변경된 내용이 아주 짧은 시간 내로 실시간으로 반영되는 모습이 강조됨

### 기능 추가와 동시에 권한(permission) 자동 부여가 이루어짐

- 클로드가 ‘순차적 사고 MCP’ 기능을 추가함과 동시에, 서버 내 모든 도구 사용 권한도 자동으로 파일에 반영함
- 이로써 개발자가 도구를 사용할 때 별도의 추가 권한 확인 작업을 하지 않아도 됨
- 제작자는 “도구들을 쓸 때마다 허가를 받을 필요가 없다”고 설명

### 클로드 코드의 열린 확장성과 자기 개선 능력이 직접 입증됨

- 전체 과정이 스마트폰에서 간단한 지시 한 번으로 이루어진다는 점이 특별히 강조됨
- 클로드 코드가 직접 파일을 파악하고, 기존 예시(Obsidian 통합)까지 참고하면서 스스로 코드를 발전시킴
- 제작자는 이 과정을 “정말 미쳤다(so so)”고 반복 언급하며 기술적 진보에 놀라움을 표현
- 본 시연을 통해 인공지능 에이전트가 스스로 개발을 개선하고, 사용자는 더욱 간편하게 서비스 확장이 가능함을 실증함
