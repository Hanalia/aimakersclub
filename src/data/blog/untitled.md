## *AI에이전트의 기능을 가장 단순하게 강화하는 방법 (Skills)* 핵심 요약

- Anthropic이 최근 공개한 "Skills" 기능은 최신 AI 분야에서 가장 중요한 발전 중 하나로 평가됨
- "Skills"의 가장 큰 강점은 놀랍도록 단순하면서도 효과적이라는 점에 있음
- Anthropic의 개발 모토는 "단순함이 곧 더 나음(the simpler the better)"에 근거함
- 복잡하고 많은 정보를 에이전트에게 한 번에 제공하면, 컨텍스트 윈도우가 과부하되어 성능 저하가 발생함
- 예전의 MCP 서버 방식처럼 수많은 도구(tool)를 미리 다 주는 것은 비효율적이고 실제 대화에 불필요한 도구까지 에이전트가 인지하게 만듦
- Skills 기능은 "점진적 노출(progressive disclosure)"이라는 방식으로 이 문제를 해결함
- 에이전트는 시스템 프롬프트에 '기능의 간단한 설명'만을 먼저 받고, 실제로 필요한 순간에만 해당 기능의 전체 지침(skill.md)을 불러와 사용함
- skill.md 파일이 바로 각 기능(스킬)의 구체적 사용법과 추가 문서를 제공하며, 필요 시 여러 레이어로 더 많은 정보를 참조함
- 궁극적으로, 에이전트는 실제 작업에 맞춰 필요한 컨텍스트만을 순차적으로 받아 업무 효율과 융통성을 극대화함

## 세부 요약 - 주제별 정리

### Anthropic의 Skills는 복잡한 AI 기능 강화 방법을 극적으로 단순화함

- Anthropic이 최근 공개한 Skills는 최신 AI 분야에서 상당히 중요한 발전으로 주목받고 있음
- 단순하면서도 파워풀한 구현방식이 Skills의 본질적 매력으로 강조됨
- 영상 제작자는 Skills의 등장이 최근 AI 발전 중 “가장 혁신적”이라 평가
- Anthropic 개발 철학 중 “The simpler the better” 원칙이 이 기술에 그대로 적용됨

### 많은 기능을 한 번에 투입하면 AI 컨텍스트 윈도우가 쉽게 과부하되고 비효율이 발생함

- AI 에이전트에게 수많은 도구, 대화이력, 프롬프트 등 정보를 한꺼번에 주면 컨텍스트 윈도우가 빨리 꽉 참
- 과도한 정보 투입은 요약 실패, 답변 저하, 문맥 무시 등 다양한 부작용을 유발
- 기존 MCP 서버 방식은 에이전트가 한 대화에서 쓸모없는 도구까지 모두 인지하게 만들어 효율성이 크게 저하됨

### Skills는 ‘점진적 노출’ 방식을 적용해 에이전트에 정보와 기능을 순차적으로 제공함

- Skill 방식의 핵심 차별점은 “progressive disclosure(점진적 공개)”임
- 모든 도구와 설명을 한꺼번에 시스템 프롬프트에 포함하지 않고, 필요한 순간에 필요한 정보만 단계별로 공개
- 에이전트는 우선 각 기능(skill/capability)의 간단한 설명만 시스템 프롬프트로 제공받음

### 단일한 기능(예: PDF 처리)을 예시로 들어 Skills 적용 원리를 설명함

- 예시: PDF 처리 스킬을 에이전트에게 부여할 때, 처음에는 단순히 “PDF 처리 가능 기능이 있음” 정도만 전달
- 사용자가 “PDF를 처리해달라”는 구체 요청을 했을 때에만 에이전트가 skill.md 파일 전체를 읽어본 뒤 해당 기능을 적용
- skill.md 파일이 바로 그 스킬의 상세한 사용법, 명령, 예시를 포함

### Skill.md 파일과 추가 문서를 단계적으로 공개하여 컨텍스트 사용 최적화함

- skill.md는 개별 스킬(기능)에 대한 ‘본문 설명서’ 역할을 하며, 첫 번째 레이어(설명) 직후 필요한 만큼만 추가 참고
- skill.md에서 더 세부적인 문서(참고 매뉴얼, 샘플, 규칙 등)가 필요하다면 추가로 불러오도록 설계(3단계 레이어)
- 이렇게 단계적으로 점진적으로 필요한 만큼만 문서와 컨텍스트가 에이전트에 해금됨

### Skills 방식이 기존 MCP 서버 대비 현저히 불필요한 정보 과부하를 줄임

- MCP 서버 방식은 대화에서는 실제 사용하지 않을 수많은 툴도 항상 에이전트에 모두 노출
- Skills 방식은 “필요할 때 필요한 만큼만” 열람하게 하여 컨텍스트 윈도우를 효율적으로 유지
- 실제 구현에서는 각 기능이 skill.md라는 파일로 구분 관리되어, 확장과 유지가 용이함

### 모든 기능이 단순화된 구조 덕분에 에이전트 확장성과 사용성이 동시에 극대화됨

- 스킬 추가가 시스템 프롬프트에는 단순 설명만 추가하는 식으로 매우 용이
- 에이전트의 활용 가능성과 워크플로우 자동화가 간편하게 확장 가능
- 복잡한 시스템이나 긴 셋업 없이도 빠른 인공지능 기능 강화가 구현됨

### 사용자 요구에 따라 실제 필요한 순간에만 완전한 능력을 불러오는 구조가 작업 효율을 높임

- “점진적 노출”의 3단계(설명→지침→추가참고) 구조로, 실질적 필요가 생길 때에만 추가정보가 호출됨
- 이는 실제 업무 상황이나 다양한 사용자 요청에 맞춤형으로 유연하게 대처할 수 있도록 하는 구조임
- 불필요한 전 정보 공개를 피함으로써 사용성, 응답 품질, 속도를 모두 잡을 수 있음

### Anthropic Skills의 철학과 구현 방식을 통해 미래 AI 확장 패러다임이 제시됨

- “필요한 정보의 단계별 노출”이라는 간명한 원리가 모든 기능에 적용됨
- 실제 엔터프라이즈나 커스텀 분야에서 복합적인 기능을 가진 에이전트 구현에 큰 장점 제공
- 영상자는 Anthropic Skills를 실제 커스텀 구현 예시로도 여러 번 시연 예정임

### progressive disclosure 개념이 AI 도구 설계의 새로운 기준이 되고 있음을 강조함

- 모든 AI 시스템 설계에서 “얼마나 단순하게, 얼마나 점진적으로 정보를 제공할지”가 핵심 설계 포인트로 부상
- Skills는 이러한 최신 설계법의 대표적 예시로 제시됨
- 최종적으로, AI 에이전트의 확장과 유지보수의 난이도를 낮추고, 실제 사용 효율을 극적으로 개선함
