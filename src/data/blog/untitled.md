## *AI 에이전트의 성능을 극대화하는 가장 간단한 방법(스킬)* 핵심 요약

- Anthropic이 최근 공개한 “스킬(Skills)”은 AI 분야에서 가장 중요한 발전 중 하나로 평가받음
- “더 간단할수록 더 좋다”는 앤트로픽의 개발 철학이 스킬의 설계에 반영되어 있음
- 기존의 MCP 서버와 달리, 모든 도구를 한꺼번에 에이전트에게 제공하지 않고 필요한 순간에 점진적으로 노출시키는 ‘점진적 공개(progressive disclosure)’ 구조가 핵심
- 에이전트에게 처음에는 스킬(기능)의 ‘설명’만 시스템 프롬프트로 제공함
- 사용자의 요청이 실제로 필요할 때만, 자세한 지침이 담긴 skill.md 파일이 로드되어 두 번째 맥락(컨텍스트) 계층이 활성화됨
- skill.md 파일은 해당 스킬의 모든 주요 작동 지시사항을 포함하고 있음
- 세 번째 계층에서는 추가 참조 문서들을 skill.md가 필요시 로드하여 보다 세밀한 컨텍스트를 제공
- 이런 방식 덕분에 에이전트의 컨텍스트 윈도우가 불필요하게 복잡해지거나 과부하되는 문제가 예방됨
- 예시로는 PDF 처리 스킬이 언급되었으며, 실사용 시 워크플로우가 상세히 설명됨
- 마지막으로, 이는 사용자 요청에 따라 필요한 정보와 기능만 단계별로 제공하는 구조로, 도구 활용의 효율성과 에이전트의 성능이 대폭 향상됨

## 세부 요약 - 주제별 정리

### Anthropic의 “스킬”은 AI 분야에서 혁신적인 진보로 평가받음

- Anthropic이 최근 “스킬” 기능을 공개했으며, 이는 AI 발전의 중요한 전환점으로 소개됨
- 스킬은 에이전트의 역량을 비약적으로 확장해주기 위한 간단하면서도 효과적인 방법으로 강조됨
- 단순함을 통한 강력함이 Anthropic의 모토임이 반복적으로 언급됨
- 영상 제작자는 “최근 AI 발전 중 가장 중요하다”고 표현함

### 단순함을 지키면서 에이전트의 기능을 최대한 확장하는 것이 스킬의 핵심임

- 에이전트에게 너무 많은 정보(도구, 대화 이력, 시스템 프롬프트 등)를 한 번에 제공하면 컨텍스트 윈도우가 빠르게 포화됨
- 기존에는 모든 도구를 미리 한꺼번에 에이전트에 주입, 이로 인해 불필요한 혼란과 복잡함 발생
- Anthropic 스킬 시스템은 “최소한의 정보”만 먼저 제시하고, 필요할 때 적시에 세부사항을 공개하는 방식을 따름

### 기존 MCP 서버 방식은 불필요하게 많은 도구를 에이전트에 한꺼번에 부여하는 문제가 있음

- MCP 서버 같은 기존 접근법은, 특정 대화에서 사용하지 않을 수도 있는 다양한 도구를 미리 모두 공개함
- 이런 방식은 대화에 각 도구가 필요하지 않아도 에이전트의 컨텍스트 윈도우를 불필요하게 소모시킴
- 결과적으로 에이전트의 맥락 이해력 저하, 처리 과부하, 응답 질 저하 등 부작용 초래

### 점진적 공개(progressive disclosure) 구조가 Anthropic 스킬의 핵심 혁신임

- 모든 도구 설명과 지침을 한 번에 주입하는 것이 아니라, 3단계의 ‘점진적 공개’로 관리됨
- 1단계: 시스템 프롬프트에 에이전트가 가진 ‘능력(스킬)’의 이름과 설명만 가볍게 담음
- 2단계: 실제로 필요할 때, 해당 능력의 작동 방법/지침이 구체적으로 담긴 skill.md 파일을 로드
- 3단계: skill.md 파일이 참조하는 추가 문서까지 필요한 시점에만 로드되어 더 깊은 맥락을 제공
- 이렇게 계층적으로 정보가 공개됨으로써, 맥락 복잡성 최소화 및 적시성 확보

### PDF 처리 스킬을 예시로, 실제 활용 구조가 설명됨

- 예시로, PDF 파일을 처리하는 스킬을 가진 에이전트가 소개됨
- 시스템 프롬프트에는 “PDF 처리가 가능합니다” 정도의 설명만 포함됨
- 사용자가 PDF 관련 요청을 하면, 그때 skill.md가 활성화되어 구체적인 처리 지침이 에이전트에 전달됨
- skill.md 파일에는 PDF 처리에 필요한 모든 지침과 워크플로우가 상세히 기록되어 있음
- 추가로, skill.md가 참고하는 별도 문서가 있다면 3단계에서 연동하여 더 깊은 정보 제공

### 앤트로픽 스킬의 계층적 구조는 에이전트 맥락 관리의 효율성을 극대화함

- “설명 → 세부 지침(skill.md) → 추가 문서”의 3단계 계층이 에이전트의 컨텍스트 윈도우 관리에 최적화되어 있음
- 실제 요청 때 필요한 정보만 단계적으로 공개, 컨텍스트 창 포화 위험 최소화
- 결과적으로 대화 자연스러움, 응답 질, 도구 활용 효율 모두 향상됨

### 사용성 관점에서, 각 스킬은 필요 시에만 단계별로 활성화되어 오버로드가 방지됨

- 모든 “능력”을 한꺼번에 노출하지 않기 때문에 에이전트가 혼란을 겪지 않음
- 사용자가 요구할 때만 일부 기능(지침, 참조 문서 등)이 추가적으로 공개됨
- 필요할 때마다 단계적으로 정보가 추가되어, 전체적인 시스템 반응성과 직관성 모두 증대

### custom 스킬 구현에도 Anthropic의 구조가 그대로 적용됨

- 영상에서는 기본 제공 스킬뿐만 아니라, 직접 구현한 custom 스킬에도 동일한 progressive disclosure 구조를 적용함을 강조
- 모든 Anthropic 스킬의 중심은 skill.md 파일에 있음
- skill.md가 명확하게 각 스킬의 구체적 지침과 연동 문서들을 연결하는 허브로 작동

### 맥락 창(컨텍스트 윈도우)의 효율적 사용이 에이전트 성능에 직접적 영향을 미침

- 영상 내내 에이전트 용량(맥락 창) 관리 문제와 스킬 구조의 장점이 반복적으로 강조됨
- 도구와 지침, 대화 이력, 시스템 프롬프트가 모두 컨텍스트 창을 차지하므로, 불필요한 정보 최소화가 핵심
- 스킬 시스템 자체가 정보 과다 문제의 효과적 해결책임을 시사

### 단계적으로 확장하는 ‘스킬’로 AI 에이전트는 더욱 강력하고 유연하게 진화 가능함

- 점진적으로 도구/기능을 활성화시켜주는 구조 덕분에, 여러 용도에서 맞춤형 워크플로우 설계가 가능해짐
- 필요 기능만 적시에 호출하여, 복잡한 시스템에서도 높은 성능과 안정적인 대응력을 확보할 수 있음
