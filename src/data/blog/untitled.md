## *스택드 디프(누적 변경 이력) 방식이 AI 코드리뷰의 효율을 극대화함* 핵심 요약

- AI 코드리뷰는 강력한 검증 도구로, 개발 검수 과정에 필수라고 평가함
- 전통적 풀 리퀘스트 방식에서는 각 변경 후 승인을 기다려야 하므로 개발 속도가 느림
- 스택드 디프(누적 변경 이력) 방식을 사용하면 작은 변화 단위들이 누적, 각 단위별 빠르고 집중적인 AI 코드리뷰가 가능
- 이 방식은 여러 개의 작은 변경이 쌓여 하나의 큰 기능을 만들며, 중간 승인 대기 없이 연속된 개발이 가능함
- 그래파이트(Graphite)라는 툴이 스택드 디프를 쉽게 적용할 수 있게 해줌
- 그래파이트는 CLI(커맨드라인 도구)와 대시보드를 제공하며, AI 코드 어시스턴트를 통한 PR(풀 리퀘스트) 리뷰 지원 기능 포함
- 툴 가입 후 데모에서는 최초 스택드 디프 생성과 여러 단계의 PR 작성, 대시보드를 통한 관리 과정을 직관적으로 안내함
- 각각의 작은 변화(기능 브랜치)는 개별 PR로 관리되며, 각 PR마다 AI(클로드 코드 등)가 집중 코멘트를 남김
- 최종적으로 모든 PR을 그래파이트 대시보드에서 손쉽게 병합 가능, GitHub 방문 없이도 모든 관리가 이루어짐
- 누적 변경 단위별 리뷰 결과는 대규모 PR 한 번에 검토하는 것보다 월등히 명확하고 효과적임
- 추가 데모 영상 링크도 제공, 그래파이트와 AI 코드를 활용한 PR 리뷰 전 과정을 시연함

## 세부 요약 - 주제별 정리

### AI 코드리뷰가 코드 검증에 있어 필수적 도구로 자리매김함

- AI를 이용한 코드리뷰는 개발자가 자신의 변경 사항을 효과적으로 검증할 수 있도록 도와줌
- “AI 코드리뷰는 매우 강력하며 제 검증 과정에서 반드시 활용하는 요소”라고 언급함
- 단순한 자동화 검토를 넘어, 코드의 논리적 오류나 설계 결함까지 찾아냄
- 판별력이 높고 집중도 또한 우수함

### 전통적 풀 리퀘스트 방식의 한계로 개발 프로세스가 느려짐

- 기존 PR(풀 리퀘스트)은 변경 사항을 만든 뒤 승인 대기, 승인 후 다음 변경을 반복하는 절차임
- 각 단계 사이에 의무적으로 승인을 받아야 하므로 생산성 저하 발생
- 대규모 기능 개발이나 여러 변경이 연속적으로 필요한 경우, 속도 측면에서 비효율적임

### 스택드 디프를 활용하면 작은 단위의 변경을 빠르게 누적 적용 가능함

- 스택드 디프란, 여러 개의 소규모 변경이 차례차례 쌓여 하나의 큰 기능이나 개선을 완성하는 작업 방식임
- 각 변경 사항을 독립적이고 작은 단위로 쪼개어 작업함
- 매번 승인 대기 없이 다음 변경을 이어서 작성할 수 있으므로 개발 속도 증가
- 작은 단위기 때문에 코드리뷰 역시 집중적이고 명확해짐

### 스택드 디프 방식은 AI 코드리뷰 효율을 극대화함

- 각 작은 변경(디프) 단위에 대해서 AI가 더욱 집중적으로 리뷰를 진행할 수 있음
- 한꺼번에 많은 파일과 코드를 검토하는 것이 아니라, 단일 관심사/기능에만 집중
- 리뷰 결과가 구체적이고 actionable한 피드백으로 나옴
- 개발 및 코드 품질 양쪽 모두에서 큰 이점을 제공

### 그래파이트(Graphite)가 스택드 디프 및 AI 리뷰 적용을 혁신적으로 지원함

- Graphite는 스택드 디프 작업을 쉽게 해주는 커맨드라인 툴과 대시보드를 제공
- GitHub CLI와 유사하지만, 스택드 디프 작업에 특화됨
- 리포지토리 내 스택드 디프 관리, PR 생성, 병합 등 모든 과정을 포괄적으로 처리 가능
- 대시보드 내에 AI 코드 어시스턴트가 내장되어 PR 품질 검토 자동화 가능

### 그래파이트 가입 및 데모에서 스택드 디프 생성/관리를 쉽게 익힐 수 있음

- 플랫폼 가입 시, 직관적인 데모를 통해 첫 스택드 디프 작성과 워크플로우를 안내받음
- 첫 번째 변경을 만들고, 그 후 추가로 두 번째, 세 번째 PR까지 계층적으로 생성함
- 모든 과정을 대시보드와 CLI(명령어 창) 양쪽에서 시각적으로 확인 가능

### 각각의 PR(풀 리퀘스트)을 별개로 분리하고 AI를 통해 집중적으로 리뷰함

- 예시로 세 개의 PR을 연속적으로 생성하고, 각각이 계단식으로 연결됨(누적)
- 각 PR은 독립적으로 기능 브랜치에서 작성, 상위 PR에 병합되는 구조임
- 각 incremental change마다 AI(클로드 코드 등)로부터 구체적인 리뷰 피드백을 받음

### 그래파이트 대시보드에서 모든 변경사항을 통합 관리 및 병합 가능함

- 그래파이트의 웹 대시보드에서 각 PR을 한 눈에 파악, 관리 및 병합이 쉬움
- 필요시 그래파이트 에이전트를 활용해 AI 리뷰 프로세스를 PR 별로 실행 가능
- 모든 프롤 하나씩 혹은 전체를 합치는 병합(merge)도 대시보드에서 실행
- 이 모든 과정 동안 GitHub 자체 페이지 방문 없이도 작업 가능

### 실제 변경 이력 관리에는 Git의 feature branch 전략을 활용함

- 각 incremental change는 별도의 기능 브랜치(feature branch)에서 작성됨
- 작업이 끝난 후, 상위 변경에 merge하며 계단식 누적 구조가 완성됨
- 개발이 마무리되면 모든 변경을 통합(main 브랜치에 머지함)

### AI를 활용한 소규모 분할 리뷰가 대규모 PR 전체 리뷰보다 훨씬 낫다는 결과가 도출됨

- 세분화된 각 PR마다 AI의 섬세한 리뷰가 가능, 실질적 코드 품질 개선에 용이
- “한꺼번에 거대한 PR을 리뷰하는 것과 비교하면, 결과가 정말 엄청나다(insane)”고 강조
- AI(예: Claude code)를 각 계층별로 투입, 극대화된 품질·생산성 효과 확인

### 추가 영상에서 자세한 스택드 디프 적용 방법을 따로 안내함

- 영상 하단 설명란에 추가 데모 영상 링크 제공
- 본 영상에서 미처 다루지 못한 그래파이트·AI 활용법과 PR 운영 팁 등을 심도있게 안내함
