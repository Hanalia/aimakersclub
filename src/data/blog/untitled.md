## *MCP 서버의 심각한 문제점과 그 해결책* 핵심 요약

- 영상 제목은 “MCP 서버의 심각한 문제점과 그 해결책”으로, MCP 프로토콜이 인기를 얻으면서 밝혀진 중요한 결함에 대해 다룸
- MCP는 최근 토큰 소비와 컨텍스트 소실(‘context rot’) 문제로 인해 비판받고 있음
- 영상은 MCP 문제의 본질과 심각성을 알리기 위해 구체적 예시를 들어 설명함
- 다섯 개의 대표적인 MCP 서버가 AI 코딩에 사용될 때, 각 서버가 AI 에이전트에 자신의 기능을 설명하는 데 수천 개의 토큰을 소모함
- 이러한 방식으로 여러 MCP 서버를 동시에 같은 세션에서 사용할 경우, 토큰 소모량이 기하급수적으로 증가해 세션 관리와 AI 응답 효율에 큰 부하를 줌
- MCP의 토큰 문제로 인해 “같은 세션 내 여러 MCP 서버 동시 사용”을 절대 피해야 한다고 강조함
- 해결책으로 ‘Agent Skills’라는 기능이 소개되며, 이는 최근 Anthropic(엔트로픽)에서 공개함
- Agent Skills는 스크립트 자동 생성, API 엔드포인트 상호작용, 동적 코드 생성 등으로 MCP의 근본적인 문제를 해결함
- Agent Skills의 등장으로 MCP 관련 토큰 및 컨텍스트 문제를 효과적으로 극복할 수 있게 됨
- 영상은 “이것이 통념상의 MCP 시대의 종말을 뜻하는가?”라는 의문을 던지며 마무리함

## 세부 요약 - 주제별 정리

### MCP 프로토콜은 인기와 함께 심각한 문제점을 드러내고 있음

- MCP(Multi-Component Protocol)는 최근 인기를 얻으며 다양한 AI 코딩 환경에서 활용되고 있음
- 그러나 프로토콜의 확산과 함께 치명적인 문제가 있음을 사용자와 개발자들이 점점 더 인식하게 됨
- 영상에서는 “토큰 소비와 컨텍스트 소실(context rot)” 문제가 현재 MCP의 가장 뜨거운 논쟁임을 언급
- 이 문제들은 특히 AI 시스템과의 상호작용 효율, 비용, 기능 한계 등 여러 측면에 악영향을 미침

### MCP 환경에서 토큰 소비와 컨텍스트 소실이 심각한 이슈로 등장함

- MCP 서버는 AI 코딩 보조 시스템에서 주요 요소로 사용됨
- 각 서버는 AI 에이전트에게 자신이 무엇을 할 수 있는지 설명하기 위해 상당한 분량의 토큰을 사용
- 이 과정에서 세션마다 설명이 반복되므로 토큰 낭비가 심각하게 발생할 수 있음
- 토큰 소모가 많아질수록 AI의 컨텍스트 창(상황 인식)이 빠르게 소실됨(‘컨텍스트 rot’)
- 이에 따라, 코딩 보조 효율과 AI의 대화 품질이 급격히 저하될 위험이 존재함

### 여러 MCP 서버를 한 세션에서 병렬 사용하면 토큰 소모가 폭증함

- 저자가 예를 든 사례: “다섯 개의 MCP 서버를 AI 코딩 세션에 동시 적용”
- 각 서버가 자신을 설명하는 데만 수천 개의 토큰을 소모
- 최종적으로 세션 당 토큰 소모량이 수만 개 이상 될 수 있음
- 저자는 “이렇게 여러 MCP 서버를 한 세션에서 절대 사용하지 않겠다”고 강조함
- 토큰 사용량이 많아질수록 프로젝트 비용이나 AI 응답 속도, 정확도에 치명적 영향

### 기존 MCP 방식은 효율성과 확장성에 본질적 한계를 지님

- 기존 방식: 각 MCP 서버가 각자 AI 에이전트에게 모든 기능을 설명해야 하므로 중복, 비효율 발생
- 여러 서버가 동시에 기능 설명을 반복하면 컨텍스트 창을 빠르게 소진하게 됨
- 이는 세션 내에서 긴 코드 생성 또는 복잡한 업무를 처리하는 데 큰 제약으로 작용
- “서로 다른 MCP 서버의 활용성” 자체가 제한되는 상황을 초래

### Anthropic의 ‘Agent Skills’ 기능이 MCP 문제에 근본적 해결책을 제시함

- Anthropic(엔트로픽)에서 최근 출시한 ‘Agent Skills’가 MCP 병목 문제의 해법으로 제시됨
- 이 기능은 사용자 및 AI 에이전트가 다양한 스킬(스크립트 등)을 동적으로 생성할 수 있는 구조
- Agent Skills를 통해 서버들은 자신의 모든 기능을 매번 자세히 설명할 필요 없이, 필요한 스킬만을 AI가 즉시 호출하거나 생성 가능함
- 이로써 불필요한 토큰 사용이 막대하게 줄어듦

### Agent Skills는 API 엔드포인트 상호작용 및 동적 코드 실행을 가능하게 함

- AI 에이전트가 Agent Skills를 활용해 API 엔드포인트와 상호작용하는 스크립트를 생성/실행할 수 있음
- 예시: API 호출을 위한 명령 생성, 상세 인스트럭션 제공, 즉시 코드 생성 등 다양한 실시간 기능
- 영상에서는 “동적 코드 실행(code execution) 능력의 극대화”로 이를 강조
- 기존 MCP 서버 설명 방식보다 훨씬 빠르고 효율적임

### Agent Skills 채택으로 MCP의 기본 한계를 극복할 수 있음

- Agent Skills는 토큰 낭비와 컨텍스트 소실이라는 MCP의 근본 문제를 크게 완화
- AI 코딩 도우미 또는 생산성 도구에서 여러 서버를 효과적으로 활용하는 것이 가능해짐
- 사용자는 더 적은 토큰으로 더 많은 서버 기능을 유연하게 활용할 수 있음
- AI 소프트웨어의 확장성과 실제 응용 가능성을 크게 높여줌

### 영상은 MCP 패러다임의 변화, 혹은 끝을 암시하며 마무리됨

- 저자는 영상 마지막에 “이것이 우리가 알고 있던 MCP의 끝인가?”라는 의문을 남김
- Agent Skills와 코드 실행 방식이 MCP의 전통적인 한계를 뛰어넘으며, 산업 패러다임 변화 신호탄임을 시사
- 영상의 결론: Agent Skills와 같은 신기능이 기존 MCP 방식의 비효율과 문제를 해결하며, 혁신의 새로운 시대를 열고 있음을 강조함
