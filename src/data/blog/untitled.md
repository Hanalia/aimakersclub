## *스택드 디프(누적 변경 사항)로 AI 코드 리뷰 효율이 극대화됨* 핵심 요약

- 영상 제목은 "스택드 디프가 AI로 코드 리뷰하는 방식을 어떻게 바꿨는가"이며, AI 코드 리뷰의 효과적 활용법을 소개함
- AI 기반 코드 리뷰는 매우 강력하며 필수적 검증 단계로 작동함을 강조함
- 기존 풀 리퀘스트(PR)는 변경 후 승인 대기를 반복하는 비효율적 프로세스를 가짐
- 스택드 디프는 작은 변경들을 차례로 쌓아가면서 큰 기능을 완성하는 방식으로, 각 단계마다 대기 없이 연속 작업이 가능함
- 이 구조는 각 변경 사항마다 AI가 더욱 집중적으로 피드백을 제공할 수 있도록 최적화됨
- 스택드 디프 활용을 쉽게 돕는 도구로 ‘Graphite’를 소개함: 커맨드라인 툴, 통합 대시보드, AI 코드 어시스턴트 포함
- Graphite 플랫폼은 가입 시 직관적인 데모를 제공, 첫 스택 디프 생성 및 풀 리퀘스트 과정을 단계별로 안내함
- 각 변경은 개별 피처 브랜치에 커밋되고, 점차적으로 머지되어 최종적으로 메인 브랜치로 머지됨
- AI(예: Claude code)를 활용해 각 스택 풀 리퀘스트를 개별적으로 리뷰한 결과, 한 번에 큰 PR을 리뷰할 때와 비교해 극적인 차이를 경험함
- Graphite 덕분에 GitHub 리포지토리 안에 직접 들어가지 않고도 모든 PR 관리, 리뷰, 병합이 가능함

## 세부 요약 - 주제별 정리

### AI 코드 리뷰는 개발 검증 과정에서 필수적인 도구로 자리잡음

- 영상 시작과 동시에 AI 코드 리뷰의 강력함을 강조하며, 개인적으로도 코드를 검증하는 필수 수단으로 사용한다고 밝힘
- AI 코드 리뷰를 통해 코드의 정확성, 일관성, 버그 발견 등에 강점을 갖는 사례를 전제함
- AI 도구의 사용이 개발 검수 품질을 향상시키는 핵심 임을 시사

### 기존 풀 리퀘스트 흐름은 승인 대기 때문에 비효율적임을 설명함

- 전통적 풀 리퀘스트(PR) 방식은 작업자가 변경사항을 만든 후, 승인받기까지 기다려야 하며, 이를 여러 번 반복하므로 비효율이 큼
- 각 변경이 순차적이고 반복적으로 승인받는 과정이 전체 개발 흐름을 지연시킴
- 이러한 구조로는 큰 규모의 기능 도입이나 다수의 코드 변경 시 생산성이 저하됨

### 스택드 디프 방식 도입이 반복 작업을 없애고 효율을 높임

- 스택드 디프(Stacked Diffs)란, 각각의 작은 변경을 위에 차곡차곡 쌓아가면서 나중에 큰 기능을 완성하는 개발 방법을 의미
- 각 단계가 독립적이고, 중간에 승인 대기가 필요 없으므로 지속적으로 다음 작업으로 넘어가 효율이 극대화됨
- 스택드 디프 방식으로는 대규모 변경도 작은 단위(디프)로 나누어 작업할 수 있어 개발자의 생산성 증진됨

### 각 변경 사항에 대한 AI 집중 피드백이 가능해짐

- 스택드 디프 구조에서는 각 디프(변경 사항)마다 AI가 별도로 리뷰를 수행함
- 한 번에 많은 코드를 리뷰하는 기존 방식을 탈피해, 각 단계마다 집중적이고 구체적인 피드백을 받을 수 있음
- 결과적으로 AI 코드 리뷰의 품질과 민감도가 대폭 상승함

### Graphite 플랫폼은 스택드 디프 기반 개발을 매우 쉽게 만듦

- Graphite는 스택드 디프 작업에 최적화된 커맨드라인 툴을 제공, 이는 GitHub CLI와 유사하지만 스택드 디프 관리에 더욱 특화됨
- 모든 변경 이력과 풀 리퀘스트, 스택 구조를 한 눈에 볼 수 있는 대시보드 제공
- 대시보드 내에는 AI 코드 어시스턴트가 내장되어 있어, 각 PR 리뷰가 손쉽게 가능

### 회원 가입 즉시 제공되는 데모를 통해 직관적으로 활용법을 익힐 수 있음

- Graphite 가입 시 제공하는 데모가 매우 직관적이고 실전적임
- 첫 스택 디프 생성부터 풀 리퀘스트 생성, 연속적으로 쌓아가는 방식까지 단계별로 안내
- git이나 GitHub CLI를 사용하듯 자연스럽게 습득 가능하게 설계됨

### 단계별 풀 리퀘스트 생성 및 누적 병합 과정을 명확하게 시연함

- 첫 변경을 커밋하고 PR 생성, 그 위에 추가 변경을 쌓으면서 새 PR을 만드는 과정을 직접 시연함
- 세 개의 PR이 연속적으로 쌓이고, 각 PR은 모두 이전 것 위에 빌드됨
- 최종적으로 스택 전체가 대시보드에서 어떻게 보여지는지, 어떻게 각 단계를 머지하는지 명확하게 시각적으로 설명

### Graphite 대시보드에서 모든 PR 관리와 AI 리뷰, 병합이 가능함

- 각각의 스택 PR을 대시보드 상에서 쉽게 관리 및 리뷰할 수 있음
- Graphite Agent(에이전트)를 통해 각 PR에 대해 AI 기반 리뷰를 받을 수 있음
- 필요 시 전체 스택의 내용을 한꺼번에 머지할 수 있고, GitHub로 이동하지 않고도 모든 작업이 처리됨

### 각 스택 단계별로 AI 코드 리뷰(예: Claude code)를 받아 집중 관리할 수 있음

- 각 PR마다 개별적으로 AI(Claude code)로 리뷰를 수행하며 품질을 검증
- 한 번의 대규모 PR을 AI로 검토할 때보다, 작은 단위별 검토 결과가 훨씬 정확하고 세밀했다고 강조
- 실제 세 개의 스택된 PR 각각을 Claude code로 리뷰받는 구체적인 활용 예시를 전달

### Stack Diffs와 Graphite의 결합으로 코드 리뷰 방식이 혁신됨을 결론지음

- Stack Diffs 방식과 Graphite 플랫폼의 결합으로, 기존 PR 중심의 개발 및 코드 리뷰 패러다임이 효과적으로 혁신됨을 전달
- 끝으로, 방법 소개와 Graphite 협업에 감사를 표하며 영상을 마무리함
