## *Postman의 Git-네이티브 기능 도입은 API 개발 전 과정을 혁신적으로 변화시킴* 핵심 요약

- 기존에는 API 작업이 개발자에게 매우 번거롭고 분산되어 있어 생산성이 저하됨
- API 엔드포인트, 컬렉션, 테스트, 스펙 등을 각기 다른 도구와 위치에서 관리해야 했음
- 이러한 리소스들은 코드베이스, Postman, CI 등 서로 다른 시스템에 분산되어 있어 버전관리와 동기화가 어려웠음
- Postman이 대규모 업데이트를 통해 API 플랫폼을 AI 네이티브이자 Git-네이티브로 개편함
- 이제 Postman 리소스(컬렉션, 스펙, 목, 테스트 등)를 단순한 YAML 파일로 코드베이스(로컬)에 직접 저장 및 관리 가능
- Postman 데스크톱 앱과 클라우드 간 양방향 동기화 지원, 예시로 테스트 수정 시 곧바로 git diff로 변경 사항 추적 가능
- 파일 기반 관리로 인해 스펙과 테스트를 코드와 동일하게 git 버전 관리를 할 수 있음
- Postman의 AI 에이전트 모드는 코드 전체를 분석하여 자연어로 API 컬렉션, 스펙, 테스트, 목을 자동으로 생성·동기화함
- AI 모델 선택(Claude, GPT, 셀프호스팅 등)과 GitHub, Sentry, Linear와 같은 여러 개발 도구와의 컨텍스트 연동을 지원함
- 개발 환경과 CI 파이프라인 내에서 동일한 컬렉션과 테스트 실행 가능(프리커밋 훅, GitHub 액션 지원)으로 환경별 재작성 불필요
- 영상 제작자는 실제 Git repo 예시를 공유하며 Postman의 변화와 효용성을 강조함

## 세부 요약 - 주제별 정리

### Postman 도입 전까지는 API 작업이 번거롭고 분산된 환경이었다

- API 작업 시 엔드포인트 작성, 컬렉션 생성, 테스트 관리 등이 모두 별개의 위치와 도구에서 수행되었음
- 보통 개발자는 IDE나 코딩 에이전트에서 엔드포인트를 만들고, 그 후 Postman 같은 별도 도구에서 테스트와 컬렉션을 구성함
- 스펙, 컬렉션, 테스트, 코드 등이 파일과 도구에 따라 분산되어 저장됨
- 각 리소스가 서로 연결되지 않은 채 관리되어 변경 관리 및 버전 통합에 문제 발생
- 특히 AI 에이전트 등 복잡한 API를 개발할 때 이러한 복잡성이 가중됨

### Postman은 대규모 업데이트로 AI 네이티브이자 Git-네이티브 플랫폼으로 재탄생하였다

- Postman은 최근 큰 규모의 기능 업데이트를 단행함
- 이번 업데이트로 Postman은 "AI 네이티브 API 플랫폼"이 되었다고 언급됨
- 동시에 "Git 네이티브" 기능이 도입되어 Git 중심 개발 플로우와 자연스럽게 통합됨
- 영상 제작자는 10년 가까이 Postman을 사용해왔으며, 이번 변화가 매우 혁신적이라고 평가함

### 모든 Postman 리소스를 단일 코드베이스에서 관리할 수 있게 되었다

- 컬렉션, 스펙, 목, 테스트 등 Postman 리소스가 모두 로컬 코드베이스에서 YAML 파일로 저장 관리됨
- YAML 파일 포맷은 유지보수가 쉽고 읽기·편집에 용이함
- Postman 데스크톱 앱에서도 이러한 파일을 직접 불러와서 작업 가능
- 개발자는 로컬에서 모든 컬렉션, 엔드포인트, 테스트 등 전 리소스를 한 번에 관리 가능

### Postman 데스크톱과 클라우드 간 양방향 동기화와 실시간 git 변경 추적이 지원된다

- 언제든 Postman 앱에서 Postman 클라우드로 직접 전환할 수 있음
- 두 환경 간 실시간 양방향 동기화(Sync)가 지원됨
- 예를 들어, 엔드포인트 테스트를 수정한 뒤 에디터로 이동하면, git diff 명령을 통해 변경 사항을 바로 확인 가능
- API 스펙과 테스트도 소스코드처럼 버전 컨트롤이 쉬워짐

### 파일 기반 관리로 스펙과 테스트를 코드와 마찬가지로 git 버전관리를 할 수 있다

- Postman 리소스가 파일화되면서 개발자 작업 흐름과 Git 버전관리 시스템이 완전히 통합됨
- 변경 내역이 git commit으로 남으므로 코드 변경과 함께 API 문서/테스트의 변경 이력을 추적 관리 가능
- 코드, 스펙, 테스트가 한 곳에서 동일하게 리뷰·반영·롤백 등 유지관리 용이

### Postman의 AI 에이전트 모드는 전체 API 주기 자동화를 지원한다

- 파일 기반 저장 덕분에 Postman의 에이전트 모드가 전체 API 주기에서 역할을 확장함
- 에이전트 모드에서 자연어로 처리 요구 사항(ex. "플라스크 앱용 컬렉션 생성")을 입력하면 자동 코드 해석
- 코드베이스를 스캔하여 라우트(엔드포인트) 추출, 스펙 자동 생성, 테스트·목 파일 세팅 등을 한번에 처리
- 생성된 모든 리소스가 항상 동기화되며, git 커밋 단위로 리뷰 가능

### AI 모델 선택과 다양한 개발 툴 연동으로 생산성이 극대화된다

- Postman은 Claude, GPT, 셀프호스팅 등 다양한 AI 모델과 연계 가능
- GitHub, Sentry, Linear 등 주요 협업 및 개발 툴에서 컨텍스트를 직접 불러올 수 있음
- 이미 사용 중이던 개발 환경과 자연스럽게 통합 가능

### 동일한 API 컬렉션/테스트 셋이 개발 환경과 CI에서 완벽하게 재사용된다

- 로컬 개발 환경에서 사용하는 컬렉션, 테스트 세트가 CI 파이프라인(GitHub Actions 등)에서도 100% 동일하게 동작
- 프리-커밋 훅, GitHub 액션 등 자동화 단계까지도 완전히 일관성 유지
- 환경 별로 코드를 별도 재작성하거나 테스트 케이스 이중 관리 필요 없음

### 최신 Postman의 혁신적 변화는 모든 API 작업 흐름 자체를 크게 단순화한다

- 영상 소개자는 Postman이 최근 큰 수준으로 발전했다며 사용을 적극 권장함
- 실제 예시 GitHub 저장소와 공식 Postman 링크도 영상 설명란에 제공함
- 이번 변화로 API 개발, 테스트, 배포, 관리 전 과정을 한 번에 연결·효율화할 수 있게 됨
