## *RAG 2.0: 에이전틱 RAG와 지식 그래프 통합(무료 템플릿 소개)* 핵심 요약

- 최근 몇 달간 다양한 RAG(검색-증강 생성) 전략을 심층적으로 탐구한 결과, 핵심적으로 돌아오게 되는 두 가지는 에이전틱 RAG(Agentic RAG)와 지식 그래프(Knowledge Graphs)임
- 두 전략을 결합한 강력한 AI 에이전트를 직접 개발하여 시연함
- 이 에이전트는 사용자의 질문에 가장 적합한 답변을 제공하기 위해 정보 탐색 경로를 스스로 추론함
- 백터 데이터베이스로는 서버리스 Postgres 플랫폼인 Neon을 활용, PGVector 확장 사용
- 예시 데이터는 빅테크 기업, 각사 AI 이니셔티브, 그리고 서로 간의 제휴 관계를 포함
- 동일한 데이터를 그래피티(Graffiti)와 네오포제(Neo4j)를 통한 지식 그래프에도 저장, 관계형 구조 활용
- 단일 회사에 대한 AI 이니셔티브 질의는 벡터 DB 검색(임베딩 활용), 관계 질의는 지식 그래프 쿼리 적용
- 에이전트는 사용 도구(벡터 검색, 그래프 검색)를 직접 명시해 알려주며, 복합 쿼리의 경우 두 데이터 소스를 모두 활용 가능
- 전체 에이전트 개발 방법 및 무료 템플릿은 풀 버전 영상과 링크를 통해 제공
- Cloud Code 활용 개발법 및 빠른 셋업 가이드까지 포함하여 누구나 에이전트 직접 구축 가능

## 세부 요약 - 주제별 정리

### 최근 RAG 전략 심층 탐구 후 남은 두 가지 핵심: 에이전틱 RAG와 지식 그래프

- 영상 제작자는 지난 몇 달간 수많은 RAG 전략(검색-증강 생성 방법론)을 직접 실험하고 분석함
- 다양한 방식 중에서도 에이전틱 RAG(Agentic RAG)와 지식 그래프(Knowledge Graphs)가 가장 실용적이자 반복적으로 주목하게 되었음
- 두 방식을 왜 결합해야 하는지, 각각이 가지는 장단점에 대한 내재적 필요성을 체감함

### 에이전틱 RAG와 지식 그래프를 결합한 AI 에이전트 직접 개발 및 구현

- Agentic RAG의 장점(추론 경로 결정 능력)과 지식 그래프의 관계성 탐색 기능을 한 에이전트에 통합
- 에이전트는 질문 유형에 따라 어떤 정보원(벡터 DB 또는 그래프)에서 검색할지 스스로 판단함
- 목적: 최적의 답변을 위해 정보 탐색 경로를 자동으로 조정하는 AI 시스템 구축

### Postgres + PGVector 기반 백터 데이터베이스 구성 및 활용 사례

- 벡터 데이터베이스로는 PostgreSQL에 PGVector 확장(extension)을 추가해 사용
- 서버리스 PostgreSQL 플랫폼인 'Neon'을 데이터베이스 인프라로 채택함
- 예시 데이터는 빅테크 관련 문서를 여러 청크로 분할 및 임베딩 생성 후 벡터 DB에 저장

### 그래피티(Graffiti)와 네오포제(Neo4j)로 구축한 지식 그래프 구조의 활용

- 동일한 빅테크 정보 문서를 지식 그래프에도 저장, 네오포제(Neo4j)와 그래피티(Graffiti) 툴 사용
- 각 사의 관계(예: '아마존이 엔트로픽의 모델 호스팅 위치') 등 연결성을 관계형 구조로 저장
- 관계 정보를 신속하고 유의미하게 탐색할 수 있도록 설계

### 단일 주체 질의엔 벡터 DB, 관계 중심 질의엔 그래프 DB가 선택적으로 활용됨

- Google의 AI 이니셔티브처럼 '한 회사'에 대한 정보 질의는 벡터 데이터베이스에서 처리
- 에이전트는 시스템 프롬프트와 질문 내용을 해석해 검색 소스를 자동 판단함
- 질의 결과와 함께 '어떤 도구(벡터/그래프 검색)를 활용했는지'도 출력

### 관계 질의나 복합적 정보 요청 시 지식 그래프 쿼리로 자동 전환

- 두 기업 간의 파트너 관계, 협업 모델 같이 '관계'를 묻는 질문엔 지식 그래프를 액세스
- 시스템 프롬프트 조건 또는 질문 유형 자체로 도구 선택이 달라짐
- 실제 예시 질의와 'graph search' 도구 사용 내역이 확인됨

### 복합 데이터 소스 요청 시 에이전트가 벡터와 그래프 모두 조회 가능

- 질문이 벡터 DB와 그래프 DB 양쪽 정보 모두 필요하도록 명시하거나 프롬프트 조정 가능
- 에이전트가 두 소스를 병행 조회해, 더 신뢰성 있는 답변을 제공
- 사용자는 System Prompt가 아니라 직접 "vector와 graph 모두 사용" 명령도 전달 가능

### 도구 사용 내역(툴 리스트)을 답변과 함께 명확히 안내하여 신뢰도 제고

- 에이전트가 답변을 생성할 때마다 어떠한 데이터 소스와 분석 도구를 적용했는지 명확하게 알려줌
- 벡터 검색, 그래프 검색 등 각각의 도구가 어떻게 활용되었는지 확인 가능
- 답변의 신뢰성과 파악 용이성을 높이는 방식으로 설계

### 무료 AI 에이전트 템플릿 및 구축 가이드를 풀 영상 링크에서 제공

- 영상에서 시연한 라그 2.0 에이전트는 무료 템플릿 형태로 제공
- 영상 하단 링크를 통해 전체 개발 과정 및 상세 코드 예시 제공
- 직접 따라 하며 동일한 시스템을 구축할 수 있도록 문서화

### Cloud Code를 활용한 개발 방법론 및 셋업 과정 전체를 강의 영상에서 안내

- Cloud Code 도구(클라우드 기반 개발 환경)를 활용해 에이전트 개발 효율화
- 노코드 및 빠른 배포, 확장 적용 등 실제 현업 활용 방안도 수록
- 구체적 사용법과 셋업부터 에이전트 실운영까지 단계별 실습 안내 포함
