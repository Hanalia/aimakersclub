## *AI 에이전트를 한층 더 강력하게 만드는 가장 단순한 방법(스킬)* 핵심 요약

- Anthropic이 최근 공개한 Skills(스킬)는 최근 AI 분야에서 가장 중요한 진전 중 하나로 평가됨
- 이 솔루션의 핵심 성공 요인은 탁월하게 단순하다는 점임; Anthropic의 모토 역시 "간단할수록 좋다"
- AI 에이전트에 다양한 기능을 부여하려 할 때 컨텍스트 윈도우(이해할 수 있는 정보량) 과부하가 심각한 문제로 대두됨
- 기존의 MCP 서버 방식 등은 불필요하게 수많은 도구를 한 번에 에이전트에 제공하기 때문에 효율성 저하와 혼란을 야기함
- Skills는 프로그레시브 디스클로저(점진적 공개) 원리를 기반으로 필요할 때만 단계적으로 기능을 노출함으로써 과부하 문제를 해결함
- 첫 단계에서는 시스템 프롬프트에 단순히 "이런 기능(스킬)이 있다"는 설명만 입력됨
- 예를 들어, PDF 처리 스킬의 경우 에이전트에게 해당 기능을 사용할 수 있다고만 안내함
- 실제로 사용자가 PDF 관련 요청을 했을 때만 skill.md(기능 설명 파일)가 참조되어 구체적인 지침과 정보가 제공됨
- skill.md 파일은 실제 기능 수행에 필요한 상세 설명을 포함하며, 추가 세부 문서를 참조하는 3단계 공개도 지원함
- 이러한 구조 덕분에 필요한 순간에만 필요한 정보를 단계적으로 로딩하여, 에이전트의 성능과 효율성을 극대화함

## 세부 요약 - 주제별 정리

### Anthropic의 Skills는 AI 기능 확장의 패러다임을 혁명적으로 단순화함

- 최근에 공개된 Anthropic의 ‘Skills’는 AI 도구 설계에서 단순함이 강점임을 실증적으로 보여줌
- Anthropic은 '간단할수록 좋다'는 개발 철학을 고수하며, 복잡성 대신 단순 명료함을 추구함
- 기존 방법처럼 에이전트에 처음부터 수많은 기능을 과도하게 주입하지 않음
- Skills로 인해 여러 기능을 추가해도 에이전트가 압도당하지 않고 효율적으로 작동 가능
- 간편하지만 강력한 기능 확장이 가능해져 개발자 및 사용자 모두에게 높은 실용성을 제공

### 정보 과부하 문제는 기존 MCP 서버 방식의 대표적 한계로 지적됨

- 일반적으로 MCP 서버 등 기존 방식은 모든 도구와 정보를 미리 에이전트에게 제공함
- 이렇게 되면 실제 사용 필요성 여부와 무관하게 많은 정보가 한꺼번에 컨텍스트 윈도우에 입력됨
- 에이전트가 툴, 대화 이력, 시스템 프롬프트까지 한번에 모두 받아들이므로 오버로드 위험이 커짐
- 필요 없는 기능과 불필요한 정보가 에이전트를 혼란스럽게 하며, 효율성과 정확성 모두 저해
- Skills는 이 문제를 점진적 정보 공개(프로그레시브 디스클로저)로 혁신적으로 해결함

### 프로그레시브 디스클로저 구조는 단계별 정보 로딩으로 과부하를 차단함

- Skills의 기본 개념은 ‘점진적으로 필요한 정보만 공개’하는 것임
- 출력 창(컨텍스트 윈도우)에 처음부터 모든 정보를 담지 않고, 상황에 따라 정보를 단계적으로 추가
- 첫 단계에서는 시스템 프롬프트에 각 기능(스킬)의 단순한 ‘설명’만 명시
- 두 번째 단계로, 사용자가 관련 요청을 했을 때 구체적 기능 설명(skill.md)이 실제로 제공됨
- 필요에 따라 skill.md 내부에서 추가 문서나 세부 지침(3단계)을 읽어들일 수 있음

### 스킬의 실질적 구현 예시는 PDF 처리 기능을 통해 설명됨

- 예시로, PDF 처리(PDF processing) 스킬이 언급됨
- 에이전트는 초기에 ‘PDF로 무엇이든 할 수 있는 기능이 있음’만 알고 있음
- 사용자가 실제로 “PDF를 변환해줘” 등 요청할 때, skill.md의 전체 설명을 읽어 기능을 활성화
- skill.md 파일은 Anthropic의 모든 공식 스킬 및 커스텀 구현 시 핵심 파일로 작동
- 이 파일을 통해 기능의 구체적 실행 방식과 추가적인 문서를 순차적으로 참조

### 각 스킬은 여러 단계로 컨텍스트를 '발견'하며 점진적으로 확대함

- 스킬의 설명(1단계), skill.md의 상세 지침(2단계), 필요시 연결되는 추가 문서(3단계)로 세분화
- 에이전트는 실제로 작업을 진행하면서만 정보와 지침을 하나씩 참조하게 됨
- 이러한 점진적 발견 구조가 필요없는 정보로 인한 컨텍스트 윈도우 오염을 방지
- 기능 활성화 시 진정 필요한 정보만 단계별로 전달되어 정확성과 속도, 효율이 모두 개선됨

### Skills 방식은 스케일링, 사용자 경험 측면에서 큰 이점을 제공함

- 필요할 때만 정보가 로딩되므로 여러 기능을 한 번에 탑재해도 성능 저하 없음
- 사용자 대화 내역, 시스템 프롬프트, 툴 안내 모두 간소화 구현 가능
- 개발자는 추가 기능 도입 시마다 전체 시스템을 재설계할 필요 없이 skill.md 파일만 작성·참조하면 됨
- 사용자 입장에서도 에이전트가 한결 명확하고 신속하게 반응함
- 점진적 정보 공개 덕분에 대규모 여러 스킬 적용도 안전하게 가능

### Anthropic Skills 구성이 AI 에이전트의 목적 지향적 작업수행 역량을 실질적으로 향상시킴

- 에이전트는 실제 과업 수행 시점에만 구체적 실행 정보와 지침을 빠르게 활용
- 모든 도구 및 정보가 실제 필요할 때만 활성화되어 불필요한 혼란 및 중복 피함
- 각 스킬별 세분화된 안내와 예시, 추가 문서 덕분에 복잡한 작업도 안전하게 처리 가능
- 점진적인 정보 접근으로 실제 작업 품질(accuracy and efficiency) 향상
- Anthropic Skills 방식이 차세대 AI 에이전트 설계에서 필수적인 패러다임으로 자리매김할 가능성 시사
