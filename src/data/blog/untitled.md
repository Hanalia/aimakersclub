## *스택 디프가 AI 코드 리뷰 방식을 어떻게 혁신했는가* 핵심 요약

- 영상 제목은 "스택 디프가 AI와의 코드 리뷰 방식을 어떻게 바꾸었는가"로 번역됨
- AI를 활용한 코드 리뷰는 매우 강력하며, 검증 과정에서 반드시 필요한 도구로 평가됨
- 전통적인 풀 리퀘스트(Pull Request) 방식은 각 변경 사항마다 승인 대기 시간이 발생해 개발 속도를 저해함
- 스택 디프(Stacked Diff) 워크플로우는 작은 단위의 변경을 점진적으로 쌓아올려 큰 기능을 완성하는 방식임
- 이 방식은 각 단계마다 대기할 필요 없이 연속적으로 작업이 가능함
- AI 코드 리뷰는 작은 단위의 변경에 세밀한 피드백 제공이 가능해 효율성이 극대화됨
- 스택 디프를 쉽게 사용할 수 있도록 지원하는 플랫폼으로 Graphite가 소개됨
- Graphite는 커맨드라인 툴, 대시보드, AI 코딩 어시스턴트 등 스택 디프 관리를 위한 다양한 기능을 제공함
- 실제로 Graphite의 온보딩 데모를 통해 첫 스택 디프 생성, 연속 풀 리퀘스트 작성, 대시보드 검토 과정을 상세히 보여줌
- 각 단계 변화에 AI를 적용하면 한 번에 큰 변경을 리뷰할 때와 비교해 훨씬 뛰어난 결과를 얻는다고 강조함
- 영상에서는 Claude Code와 같은 AI 도구를 이용해 각 스택별 풀 리퀘스트를 개별적으로 리뷰하는 실제 예시를 공개함
- 전반적으로 스택 디프와 AI 리뷰의 결합이 개발 프로세스와 코드 품질 향상에 크게 기여함을 시연함

## 세부 요약 - 주제별 정리

### AI 코드 리뷰는 지금 필수 도구로 자리잡았음

- 제작자는 AI 코드 리뷰의 효과를 매우 높이 평가하며, 자신의 코드 검증 과정에서 반드시 활용한다고 밝힘
- AI 코드 리뷰는 오류 탐지, 성능 최적화, 코드 스타일 일관성 등 다방면에 걸쳐 유용하게 작용한다고 설명함
- "AI code reviews are incredibly powerful and I do consider them a must as a part of my validation"라는 직접 인용으로 중요성을 강조함

### 전통적인 풀 리퀘스트 방식은 개발 속도에 한계를 드러냄

- 기존의 개발 워크플로우에서는 기능 수정이나 추가가 필요할 때마다 Pull Request(이하 PR)를 생성함
- 각 PR은 변경 후 승인되기까지 대기해야 하며, 이 과정이 반복되어 개발의 흐름이 단절됨
- 승인이 나기를 기다리는 동안 추가 작업이 불가능하여 전체적인 개발 효율이 저하됨

### 스택 디프는 작은 변경의 연쇄로 대기 시간 없이 개발하도록 설계됨

- 스택 디프(스택형 변경사항)는 각 기능을 아주 작은 단위로 쪼개어 점진적으로 쌓아감
- 각각의 변경은 이전의 변경 위에 올라가는 방식으로, 하나의 커다란 기능을 여러 개의 작은 조각으로 만드는 셈
- 모든 변경이 '쌓여' 있기 때문에, 각 단계마다 승인 대기를 할 필요 없이 계속 새로운 변경을 이어갈 수 있음

### 스택 디프 방식은 AI 코드 리뷰에 최적화되어 초집중 피드백을 이끌어냄

- 작은 단위로 조각낸 각각의 변경사항에 대해, AI는 매우 집중적이고 세밀한 피드백을 제공함
- 기존처럼 대규모 PR 하나를 리뷰할 때보다, 각 변경마다 필요한 리뷰 포인트에만 초점을 맞추기가 수월함
- 결과적으로 코드 품질 개선, 검토 시간 단축, 실수 예방 등의 이점을 극대화함

### Graphite는 스택 디프를 위한 최적화된 툴을 제공함

- Graphite는 스택 디프 워크플로우를 손쉽게 적용할 수 있도록 설계된 플랫폼임
- GitHub CLI와 유사한 커맨드라인 도구를 제공, 리포지토리 내에서 손쉽게 스택 디프를 생성 및 관리할 수 있도록 지원함
- 대시보드를 통해 생성된 모든 스택, PR 상태, 병합 과정을 한눈에 살펴볼 수 있음

### Graphite의 온보딩 데모는 직관적으로 최초 스택 디프 생성을 안내함

- Graphite에 처음 가입하면, Graphite 데모가 자동으로 사용자에게 스택 디프 생성 방법을 안내함
- 최초 변경사항을 생성하고 첫 번째 PR을 만드는 과정을 따라가도록 인터페이스가 안내함
- Git 또는 GitHub CLI와 유사한 사용법으로 운영되어 Git 친숙자에게도 진입 장벽이 낮음

### 연속된 PR 작성 및 관리가 커맨드라인에서 간단하게 이루어짐

- 첫 PR을 만든 후, 추가 변경을 연이어 PR로 올릴 수 있음
- 커맨드라인에서 세 개의 연속 PR을 순차적으로 생성하는 모습을 실제로 시연함
- 각 PR이 이전 PR 위에 쌓여 전체 스택을 형성하며, 대시보드에서 연쇄 구조와 상태를 시각적으로 파악 가능함

### Graphite 대시보드에서 전체 스택과 각 PR을 한눈에 관리할 수 있음

- 모든 스택 디프 및 관련 PR을 대시보드에서 관리 가능함
- 각 PR에 대해 Graphite Agent(내장 AI 어시스턴트)를 활용해 개별적으로 코드 리뷰 수행 가능
- 대시보드 상에서 병합(Merge) 조작도 바로 실행할 수 있어 GitHub 이동 없이 통합 관리 가능함

### 풀 리퀘스트 구조는 표준 방식과 동일하나, 활용법이 훨씬 효율적임

- 각 변경사항은 별도의 기능 브랜치에서 생성되며, 기존 Git 사용법과 비교해 특별한 차이가 없음
- 각 브랜치들끼리 병합해나가다가, 전체 스택이 완성되면 main 브랜치로 통합함
- 기존의 "한 번에 큰 변경을 한 번에 리뷰"하는 방식 대신, "작은 PR을 여러 번에 걸쳐 리뷰"하는 식으로 변경됨

### AI 도구(Claude Code)로 각 스택 PR을 개별 리뷰하면 결과가 비약적으로 향상됨

- 영상에서 Claude Code와 같은 AI 도구를 활용, 한 스택 내의 세 가지 PR 각각을 AI가 리뷰하도록 시연
- 대규모 PR 하나를 AI로 리뷰할 때보다, 세 개의 작은 PR에 AI가 개별적으로 피드백을 주는 경우 훨씬 더 유의미하고 구체적인 검토가 가능함
- "The results that you get from this compared to reviewing one massive pull request is just insane"라는 표현으로 효과의 극대화를 강조함

### 스택 디프와 AI 코드 리뷰 결합이 개발 프로세스 혁신을 실현함

- 스택 디프 도입 후, 코드 리뷰 및 병합이 훨씬 효율적으로 이루어지며, 전체 개발 흐름이 자연스럽게 빨라짐
- Graphite와 AI 리뷰의 결합을 통해, 개발자 경험과 코드 품질이 비약적으로 개선됨을 제작자가 직접 경험담으로 설명함
- 영상 마무리에서 별도 참고용 영상 링크를 제공함(실제 구현법 시연)
