## *MCP는 매우 취약함 – 그 이유와 해결법* 핵심 요약

- 영상은 MCP(Machine Control Panel) 서버의 주요 보안 취약점 3가지를 중심으로 문제를 제시함
- 첫째, MCP 서버는 민감한 데이터(예: 액세스 토큰, 키)를 일반 텍스트로 저장하여 유출 위험이 큼
- 둘째, 사용자에게 실제 필요 이상으로 과도한 권한을 부여하는 것이 일반적임
- 셋째, MCP 서버 사용 내역에 대한 감사(audit) 또는 추적(trace) 기능이 없다는 점이 심각한 문제임
- 예시로 GitHub MCP 사용 상황을 들어, 개인 액세스 토큰을 콘텐츠에서 직접 가려야 할 만큼 안전하지 않음을 시연함
- OAuth 인증으로도 문제는 완전히 해결되지 않으며, 여전히 과도하게 관대한 장기 토큰 저장이 필요함
- MCP 클라이언트에서는 별도의 권한 분리가 불가하고, 특정 동료에게 읽기 전용 권한만 부여하는 것이 불가능함
- 강력한 엔터프라이즈급 래핑 플랫폼이 없이 MCP를 안전하게 운용하는 것은 현실적으로 불가능함을 강조함
- 이 문제의 해결책으로 Teleport란 플랫폼을 소개하며, 세부 기능(감사로그, 권한관리, 시크릿 없는 환경 구성 등)을 상세히 설명함
- Teleport는 기존 MCP 기능과 호환되면서 보안, 감사, 세분 권한제어가 가능한 프록시/플랫폼을 제공함
- 최종적으로 Teleport의 감사로그, 권한 관리, Zero Trust 기능 등으로 MCP의 기존 보안 취약점을 상당 부분 보완할 수 있음을 시연

## 세부 요약 - 주제별 정리

### MCP 서버는 민감한 데이터 노출, 과도 권한 부여, 감사 부재로 심각한 보안 위험을 가짐

- MCP(Machine Control Panel) 서버가 뛰어난 기능에도 불구하고, 보안 문제로 인해 사용에 주의가 필요함을 지적하며 영상이 시작됨
- 주요 보안 취약점 3가지: (1) 민감 정보(키, 토큰 등) 노출, (2) 필요 이상 과도한 접근 권한, (3) 서버 사용내역의 감사 및 추적 불가
- 실제 기업 환경에서는 이러한 취약점 때문에 MCP 서버 사용에 늘 위험이 뒤따름

### GitHub MCP 예시에서 평문으로 액세스 토큰을 저장하는 위협이 시연됨

- 영상자는 GitHub MCP의 설정 화면을 보여주며 자신의 Personal Access Token을 마스킹(가림)해야 했음
- 이는 곧 보안상 "토큰이 평문(text)으로 저장되고 있다는 점”이 위험 요소임을 대변함
- 실제로 일반 사용자 환경에선 본인뿐 아니라 타인까지도 이 토큰에 쉽게 접근할 수 있어 정보 유출 가능성이 큼

### OAuth 인증을 써도 과도하게 긴 범용 토큰 저장 문제는 남아 있음

- 일부에선 OAuth 인증 도입이 해결책이라 생각하나, 실제론 여전히 장기(long-live) 토큰을 로컬 환경에 보관함
- 이 장기 토큰 또한 모든 기능에 접근 가능한 범용적 권한을 가짐
- 사용자 기기에 보안 위협이 존재할 시, 이 토큰 탈취로 심각한 정보 유출/권한 남용이 발생할 수 있음

### MCP 클라이언트는 권한 분배가 불가능하여 읽기 전용 접근도 지정할 수 없음

- MCP 클라이언트를 활용함에 있어, 모든 작업(읽기, 쓰기 등) 권한이 일괄적으로 부여됨
- 예를 들어, 동료에게 MCP 사용 권한을 공유하되 ‘읽기 전용’ 등 제한적 권한만 주고 싶어도 그러한 설정 자체가 없음
- 영상자는 직접 "이 문제의 해법을 찾기 위해 광범위하게 검색했으나" 근본적으로 구조적 한계가 있다고 언급

### 보안 및 권한 문제는 엔터프라이즈급 래핑 솔루션 없이는 사실상 해결 불가함

- MCP 자체만으로는 설정 보안, 감사(audit), 세분화한 권한관리 등이 불가함
- 이러한 이유로 엔터프라이즈 등급의 외부 플랫폼(랩퍼)이 필요함을 강조
- 보안적으로 안전한 MCP 운용을 위해선 외부의 강력한 인프라가 필요하다는 결론을 도출함

### Teleport 등장: 보안 구성, 감사, 역할 기반 권한관리를 한번에 제공

- 소개되는 솔루션: Teleport 플랫폼 (영상에서 직접 데모 시연)
- Teleport는 각 MCP 서버를 "보호되고 관측 가능한 자원"으로 래핑함
- 접속 시 Teleport에 먼저 로그인하면, ‘해당 계정에 맞는 권한만 내재한 단기(Just-In-Time) 토큰’이 발급됨
- 이 방식으로 MCP 클라이언트 설정에는 더 이상 어떠한 시크릿(토큰, 키 등)도 필요 없음
- 기존의 GitHub MCP 서버 기능들과 연동이 그대로 가능함

### Teleport 프록시가 모든 보안·감사·권한 분리의 핵심 인프라 역할을 함

- Teleport 플랫폼은 MCP 서버와 애플리케이션 사이에 프록시 역할로 중간자 역할을 수행함
- 모든 MCP 요청, 트래픽은 Teleport 프록시를 통해 이동하므로 각종 정책 적용, 감사, 관리가 가능해짐
- Teleport의 감사 로그 기능은 누가 언제 어떤 MCP 액션을 했는지 상세 기록이 가능
- 영상 예시에서는 Claw Desktop에서 MCP 호출 내역(서버, 툴, 파라미터 등)이 상세히 기록되는 화면을 직접 시연함

### 감사 로그, 이상행위 탐지, 세분화된 권한설정을 Teleport에서 지원함

- Teleport에서는 모든 MCP 호출 내역이 audit log에 저장되어 상세히 추적 가능
- 특정 호출 내역을 서버별, 툴별, 파라미터별로 세부 검색 분석이 가능함
- 플랫폼 내부의 에이전트를 활용하여 비정상 행위(outlier) 탐지도 자동화 가능
- 영상자는 방금 자신이 Claw Desktop에서 호출한 기록을 실제로 확인하며 시연함

### Zero Trust 원칙 하에서 MCP 서버 접근 권한을 세밀하게 조정 가능함

- Teleport의 ‘Zero Trust’를 활용해 MCP 서버 권한을 역할 기반(role-based)으로 세밀히 통제 가능
- MCP 서버 별로, 사용자 그룹별로 “어떤 툴을 사용할 수 있는지”, “누가 접근 가능한지” 등 고도화된 설정 가능
- 이는 기존 MCP의 "권한 일괄 부여"와 달리, 실제 업무 환경 요구에 맞는 보안 대책을 제공함

### Teleport를 통한 기존 MCP 활용성과 보안성을 모두 만족시키며 문제를 해결함

- MCP 사용 환경이 Teleport 도입으로 보안, 감사, 권한 관리 측면에서 완전히 개선됨을 시연
- 기존의 GitHub MCP 서버도 Teleport 내부에서 그대로 활용 가능함을 강조
- Teleport는 기존 MCP 환경의 생산성을 저해하지 않으면서, 필수적 보안·관리 기능을 자연스럽게 제공함

### 영상 마무리: Teleport팀과의 협업 및 정보 제공 링크 안내

- 영상 하단 설명란에 Teleport 관련 공식링크 제공을 안내
- Teleport팀이 영상 기획 및 정보 정리에 도움을 줬음을 언급하며 마무리
- 영상자는 본인의 오랜 MCP 문제 해결 경험을 바탕으로 내용이 구성되었음을 밝힘
