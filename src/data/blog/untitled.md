## *RAG 2.0: 에이전틱 RAG와 지식 그래프 결합(무료 템플릿)* 핵심 요약

- 최근 몇 달 동안 다양한 RAG(Retrieval-Augmented Generation) 전략을 심도 있게 탐구한 결과, 에이전틱 RAG(Agentic RAG)와 지식 그래프(Knowledge Graphs)가 가장 돋보였음
- 두 방식의 장점을 결합한 강력한 AI 에이전트를 직접 구축함
- 이 에이전트는 질문에 가장 적합한 답을 도출하기 위해 어디서 정보를 조회할지 스스로 추론하여 결정함
- 벡터 데이터베이스로는 Postgres와 PG vector 확장을, 서버리스 플랫폼은 Neon을 사용하여 구현함
- 문서의 임베딩을 생성해 벡터 데이터베이스에 저장하고, 동일 문서를 Graffiti와 Neo4j를 활용해 지식 그래프 형식으로도 저장함
- 벡터 데이터베이스는 '특정 회사의 AI 이니셔티브'처럼 단일 정보를 조회할 때 사용되고, 지식 그래프는 '두 회사 간 관계' 등 네트워크적 질문에 활용됨
- 시스템 프롬프트나 명시적 요청으로 데이터 소스(벡터, 그래프, 혹은 둘 다)를 선택적으로 활용할 수 있음
- 사용자의 요청 방식에 따라 에이전트가 벡터DB, 지식 그래프 또는 두 소스를 조합하여 최적의 결과를 제공함
- 이 에이전트의 구현 템플릿을 무료로 제공하며, Cloud Code로 개발 및 배포 방법도 별도 영상에서 안내함

## 세부 요약 - 주제별 정리

### 다양한 RAG 전략 분석 끝에 에이전틱 RAG와 지식 그래프가 탁월함을 발견함

- 최근 몇 달간 거의 모든 RAG(검색 기반 생성) 전략을 직접 실험하고 검토함
- 실전에서 반복적으로 가장 우수하게 작동한 방법은 Agentic RAG와 Knowledge Graphs였음
- 이 두 가지 전략의 장점을 모두 활용하기로 결정함

### 두 방식의 결합으로 자율적으로 정보 소스를 선택하는 AI 에이전트를 구현함

- Agentic RAG(에이전틱 RAG)는 질문에 따라 가장 적합한 정보원과 검색 방식을 스스로 추론함
- Knowledge Graphs(지식 그래프)는 관계형 데이터와 네트워크 구조를 효과적으로 조회할 수 있음
- 두 방식을 결합하면, 에이전트가 질문의 맥락과 성격에 따라 알맞은 데이터 소스를 지능적으로 선택함
- "이 에이전트는 어디서 정보를 찾아야 할지 스스로 합리적으로 판단합니다"라고 설명함

### Postgres 및 Neon을 활용해 벡터 데이터베이스를 구축함

- 벡터 데이터베이스: Postgres를 사용하며 PG vector 확장 기능을 추가함
- 서버리스 환경에서는 Neon이라는 플랫폼을 선택하여 효율성과 확장성을 더함
- 문서는 여러 조각으로 분할(chunk)하여 각각 임베딩을 생성한 뒤 데이터베이스에 저장함
- 임베딩은 주로 대규모 기업의 AI 이니셔티브 및 파트너십 정보를 포함

### Graffiti와 Neo4j로 동일 문서를 지식 그래프에 저장함

- 동일한 텍스트 문서를, 구조화된 관계형 데이터로 재구성해 지식 그래프에도 저장함
- 지식 그래프 구축 도구로 Graffiti와 Neo4j를 사용함
- 예를 들어 "Amazon은 Anthropic이 모델을 호스팅하는 장소"라는 식의 관계형 구조가 저장됨
- 데이터의 네트워크 구조가 복합적인 질문에 효과적으로 대응할 수 있도록 함

### 벡터 데이터베이스는 단일 개체 질의에 적합하게 활용됨

- "Google의 AI 이니셔티브는 무엇인가?" 같은 단일 회사 정보 질의 시, 에이전트는 벡터 데이터베이스를 선택함
- 벡터검색이 이루어진 뒤, 해당 답변을 반환함
- 에이전트는 어떤 도구(벡터 DB, 그래프 DB 등)를 사용했는지 결과에 명확히 표시함
- 이 경우 도구 사용 내역에 '벡터 데이터베이스 검색'이 명시됨

### 지식 그래프는 관계형 질의 처리에 강점을 보임

- 두 회사 또는 객체 간 관계를 묻는 복합 질문 시, 에이전트는 Knowledge Graph를 활용함
- 예시: "Amazon과 Anthropic 사이의 파트너십 구조" 등
- 시스템 프롬프트에 따라, 관계형 데이터를 다루기 위해 그래프 검색을 선택함
- 결과창에는 '그래프 검색 도구 사용'이 명확히 드러남

### 시스템 프롬프트 수정 혹은 명시적 명령으로 복합 소스 활용이 가능함을 시연함

- 사용자가 시스템 프롬프트를 직접 조정하여 두 소스 모두를 활용하도록 지시할 수 있음
- 또는 명령어에서 '벡터 DB와 그래프 모두 함께 사용'이라고 명시적으로 요청할 수도 있음
- 에이전트는 두 가지 데이터베이스 모두에서 정보를 취합해 포괄적이고 신뢰도 높은 답변을 제공함

### 질의 유연성과 도구 선택 자동화가 에이전트의 큰 강점임을 실증함

- 단일 정보, 관계형 정보, 복합 정보 등 질의 유형에 따라 에이전트의 검색 경로와 전략이 달라짐
- 각 검색 결과에는 사용된 도구(벡터/그래프)가 표시되어 투명성과 신뢰도를 높임
- 사용자는 질의 형태만으로 데이터 접근 방식을 자동으로 정할 수 있어 편리함

### 완성된 에이전트 구조와 개발 방법을 무료 템플릿으로 공개함

- 해당 AI 에이전트의 개발을 위한 템플릿은 무료로 제공됨
- 전체 제작 과정, Cloud Code 활용법, 배포 방법 등을 풀버전 영상에서 자세히 설명함
- 영상 하단 링크를 통해 상세 가이드를 확인 가능함
