## *오픈AI의 숀 그로브: 코드만이 당신의 일의 전부는 아니다* 핵심 요약

- **오픈AI의 숀 그로브는 개발자의 업무 가치를 코드 작성에만 국한시키는 것은 과소평가라고 강조함**
- **코드 작성은 전체 업무 가치의 10~20%에 불과하다고 주장함**
- **나머지 80~90%의 가치는 '구조화된 커뮤니케이션'에서 나온다고 설명함**
- **구조화된 커뮤니케이션이란 사용자의 문제를 이해하고, 이를 팀원과 소통하며 해결책을 모색하는 일련의 과정을 의미함**
- **작업 프로세스는 사용자와의 대화로 시작해, 문제를 정제하고, 해결 방안 구상, 목표 설정, 실행 계획 수립, 동료들과의 공유로 이어짐**
- **이러한 과정은 사람마다 다를 수 있지만, 의사소통 능력이 핵심임을 강조함**
- **"코드를 짜는 것" 이외의 업무도 개발자의 가치에 매우 중요하게 기여함을 재차 강조함**
- **실제 사용자 경험, 문제 정의 능력, 동료와의 협업이 코드 이상으로 중요한 역할을 수행함**

## 세부 요약 - 주제별 정리

### 숀 그로브는 개발자의 가치를 코드에만 국한하는 것은 실제 역할을 제한한다고 강조함

- 오픈AI의 숀 그로브는 "이 일이 직접적이고 실제적으로 느껴지지만, 여러분이 하는 일의 본질을 과소평가하고 있다"고 발언함
- 개발 업무에서 코드 작성 자체가 전체 업무의 일부일 뿐임을 언급
- 코딩이 직관적으로 드러나는 결과이기 때문에 실제 업무의 가치를 과소평가할 수 있음을 지적

### 코드 작성이 전체 업무 가치의 약 10~20%만 차지한다고 주장함

- 코드 작성이 업무의 10~20% 정도의 가치만을 차지한다고 명확하게 수치로 제시
- 나머지 80~90%는 코드 외적인 업무에서 비롯됨을 설명

### 업무의 대부분은 '구조화된 커뮤니케이션'에 있다고 밝힘

- 구조화된 커뮤니케이션이란 문제를 이해하고, 정제하며, 효과적으로 소통하는 과정을 가리킴
- 이 부분이 업무 가치의 아주 큰 비중을 차지한다고 명확히 표현함

### '구조화된 커뮤니케이션'의 구체적 프로세스를 단계별로 설명함

- 1단계: 사용자의 문제를 이해하기 위해 직접 대화함
- 2단계: 사용자 이야기를 구체적이고 명확한 문제로 정제함
- 3단계: 문제를 바탕으로 어떻게 해결할지 고민(아이디에이션)함
- 4단계: 달성할 목표(goal)를 명확하게 정의함
- 5단계: 목표에 도달하기 위한 구체적 실행 방안을 계획함
- 6단계: 이 모든 계획을 동료들과 명확히 공유함(`share those plans with your colleagues`)
- 각 단계가 구조화된 의사소통의 연속임을 지적

### 문제 이해, 목표 도출, 실행 계획 수립 등 다양한 역량이 중요함을 예시로 강조함

- 단순한 코드 구현이 아니라, 사용자 니즈 파악, 비즈니스 목표 설정 등 다양한 역량이 요구됨
- "문제의 본질을 파악하는 일"이 중요한 단계임을 사례로 설명
- 실질적인 문제 해결력과 팀 내외 커뮤니케이션이 경쟁력임을 암시

### 이런 과정과 역량은 사람/팀마다 다양하게 나타날 수 있음을 강조함

- 커뮤니케이션, 업무 프로세스, 문제 접근법 등은 각 개발자 및 조직의 특성에 따라 상이함
- 절대적으로 고정된 방식보다는 유연한 업무 프로세스가 중요함

### 개발자의 가치는 코딩 실력만으로 평가할 수 없음을 단호히 강조함

- 코드 외의 기여도가 실제 프로젝트 성패에 큰 영향을 줌을 지적
- 큰 가치는 반복적 소통, 문제 정의, 해결책 구상 등 '코드 외적 업무'임을 다시 한 번 강조

### 동료와의 협업과 계획의 공유가 성공적 개발의 필수 요소임을 재차 강조함

- 계획을 세운 후, 동료들과 내용을 적극적으로 공유하는 것이 업무의 마무리 단계임을 언급
- 팀 간/내부 커뮤니케이션의 중요성을 실무 예시로 연결

### 코드 작성 업무를 넘어, 사용자를 위한 근본적 문제 해결이 본질임을 강조함

- 단순히 "코드만 짜는 것"이 아니라, 사용자의 현실적 문제를 진정으로 해결하는 것이 중요하다고 명확히 함
- 개발자가 실질적으로 만들어내는 가치의 본질을 다시 상기시킴

### 요약: 개발자란 코드 작성자로만 머무르지 않으며, '커뮤니케이션과 문제해결'이 핵심 업무임을 밝힘

- 개발자의 실질적 역할과 가치는 코드만이 아니라, 사용자와의 소통·문제 파악·해결·협업 등 전체 과정에 있음
- 숀 그로브는 이를 통해 개발자의 직업적 자존감과 업무 인식을 확장할 것을 촉구함
