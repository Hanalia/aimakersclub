## *RAG 2.0: 에이전트 기반 RAG와 지식 그래프의 결합 (무료 템플릿)* 핵심 요약

- 영상 제목은 "RAG 2.0: 에이전트 기반 RAG + 지식 그래프 소개 (무료 템플릿)"으로, 최신 RAG(검색 증강 생성) 전략을 다루고 있음
- 제작자는 최근 몇 달간 다양한 RAG 전략을 깊이 탐구한 결과, 'Agentic RAG'와 'Knowledge Graphs'의 결합이 가장 강력하다고 평가
- 해당 AI 에이전트는 정보 검색 전략을 스스로 판단해, 질문에 가장 적합한 데이터 소스를 선택해 답변함
- 벡터 데이터베이스로는 Postgres와 PG vector 익스텐션, 그리고 서버리스 플랫폼인 Neon을 활용함
- 문서는 먼저 청킹(분할) 및 임베딩 처리되어 벡터 데이터베이스에 저장됨
- 주요 내용은 빅테크 기업, 이들의 AI 이니셔티브, 그리고 기업들간의 파트너십 정보임
- 동일한 문서를 지식 그래프(Graffiti 및 Neo4j 이용)에도 관계형 방식으로 저장함
- AI 에이전트는 단일 기업 정보 질문에는 벡터DB 검색을, 기업 간 관계나 연결성 질문에는 지식 그래프를 쿼리함
- 두 데이터 소스 모두가 필요한 경우, 시스템 프롬프트 수정 또는 명시적 지시로 두 소스를 모두 활용 가능
- 이 에이전트는 무료 템플릿으로 배포되며, 보다 상세한 빌드 과정과 Cloud Code 활용법은 본영상 링크에서 확인 가능

## 세부 요약 - 주제별 정리

### 제작자는 Agentic RAG와 Knowledge Graphs 전략을 반복적으로 검토한 끝에 최적의 결합 방식을 제안함

- 최근 몇 달간 다양한 RAG(검색 증강 생성) 전략을 직접 실험 및 분석함
- 특히 '에이전트 기반 RAG'와 '지식 그래프'가 반복적으로 우수한 성능을 보였다고 강조
- 두 방식의 장점을 결합한 새로운 형태의 AI 에이전트를 개발함

### 새로운 에이전트는 질문에 따라 최적의 정보 소스를 선택하는 역량을 가짐

- 에이전트는 사용자의 질문 의도와 속성을 해석하여, 벡터DB나 지식 그래프 중 어디를 검색할지 스스로 결정함
- 정보 탐색 경로 설정과 이유에 대한 추론 능력이 탁월함
- "가장 좋은 답변을 제공할 수 있는 경로"를 선정해 탐색함

### 벡터 데이터베이스는 Postgres, PG vector, Neon 서버리스 플랫폼으로 구현됨

- 벡터 데이터베이스로 Postgres를, 임베딩 저장에는 PG vector 익스텐션을 활용함
- 플랫폼으로는 서버리스 환경을 지원하는 Neon을 사용
- 문서는 여러 조각으로 청킹(chunking) 처리 후, 각 조각에 임베딩(embedding) 적용해 저장함

### 활용 데이터는 빅테크 기업의 AI 이니셔티브 및 파트너십 정보에 중점

- 데이터셋은 주요 테크기업들의 AI 관련 전략, 추진 이니셔티브, 그리고 기업 간 파트너십 정보를 포함
- 예: "Amazon은 Anthropic이 AI 모델을 호스팅하는 곳" 등 구체적 회사 관계 포함
- 정보가 다양하고 연결성이 많은 형태임

### 동일한 문서를 Graffiti와 Neo4j로 지식 그래프에 관계형 저장함

- 같은 정보를 지식 그래프에도 저장함으로써, 문서 기반과 관계형 데이터의 강점을 동시에 사용
- Graffiti와 Neo4j를 활용하여 지식의 노드, 관계, 쿼리 가능성 등을 극대화함
- 데이터 설계시 관계와 맥락 정보를 따로 저장하여, 다양한 정보 탐색 시 활용도 높임

### 에이전트는 질문의 성격에 따라 벡터DB와 그래프DB 중 올바른 쿼리 경로를 선택함

- 예를 들어 특정 기업(AI 이니셔티브 등)만 묻는 경우 벡터 데이터베이스만 검색함
- "구글의 AI 이니셔티브는?"이라는 질문 입력 시 벡터 DB에서만 쿼리하게 의도적으로 설계됨
- 에이전트가 어떤 도구(검색 소스)를 사용했는지도 명확하게 로그로 출력함

### 관계성 질문에는 지식 그래프 쿼리로 자동 전환되어 답변함

- 두 기업 간의 관계나 파트너십, 협력 네트워크 등 "연결성 탐색" 질문 상황에는 그래프DB 쿼리를 이용
- 시스템 프롬프트의 지침에 따라 에이전트가 이를 자동 인식함
- 예시 질문: 두 회사간 모델 호스팅 관계 등 구조적인 질문에 그래프DB 사용

### 명시적 지시나 시스템 프롬프트를 조정하면 두 소스를 동시 활용할 수 있음

- 사용자는 프롬프트 지시를 통해 "벡터DB와 그래프DB 모두 사용"을 요청할 수 있음
- 에이전트는 요구에 맞춰 두 소스에서 데이터를 동시에 조회해 더욱 정확하거나 종합적인 답변 제공
- 이로써 각각의 검색 방식이 갖는 한계 극복 가능함

### 개발 및 사용 방법은 Cloud Code와 전체 소스 공개 영상으로 안내됨

- 영상 하단 링크를 통해 전체 구현 방법과 템플릿 소스 확인 가능
- 개발 과정에서 Cloud Code 활용 법, 배포 세팅 등 실습 가능 정보도 포함
- 해당 에이전트를 직접 세팅하여 사용할 수 있도록 무료로 템플릿 배포

### 에이전트는 실용적이며 누구나 무료로 사용하고 확장할 수 있음

- 영상 제작자가 '무료 템플릿' 형태로 해당 에이전트를 제공한다고 밝힘
- 빌드 절차, 실례, 배포 환경 등이 구체적으로 소개되어 접근성과 확장성이 뛰어남
- 영상 전체가 실습 중심으로 구성되어 엔지니어 및 AI 활용자 모두 직접 활용 가능
