## *MCP 서버의 근본적인 문제와 강력한 해결책* 핵심 요약

- MCP(Multi-Component Protocol)가 널리 사용되면서, 심각한 핵심 결함이 드러나고 있음
- 주요 문제는 토큰 소모와 문맥 붕괴(context rot) 현상으로, 현재 AI 커뮤니티에서 큰 화두임
- MCP의 취약점을 예시를 통해 상세히 설명하며, 문제가 얼마나 빠르게 심각해질 수 있는지 경고함
- 영상에서는 예시로 매우 표준적인 5가지 MCP AI 코딩 서버를 동시에 사용하는 상황을 가정함
- 각 서버는 자신의 기능을 AI 에이전트에게 설명하는 데만 수천 개의 토큰을 소비함
- 이 때문에 여러 MCP 서버를 한 세션 내에서 활용할 경우, 토큰 중복 사용과 문맥 붕괴 문제가 너무 심각해 실제 사용이 어려워짐
- 이런 문제를 극복하기 위한 강력한 해결책으로 Anthropic의 ‘Agent Skills’ 기능을 자세히 소개함
- Agent Skills는 AI 에이전트가 API 엔드포인트에 상호작용 가능하게 스크립트를 직접 생성하고, 그 스크립트 사용법 매뉴얼 및 코드까지 즉석에서 만들어내게 함
- 이로써 코드 실행과 자동화 수준을 한층 강화하며, MCP의 토큰 및 맥락 문제를 실질적으로 해결할 수 있음을 강조함
- 마지막으로, 이런 혁신이 MCP의 기존 구조에 종말을 가져오는 것인지에 대한 의문으로 영상을 마무리함

## 세부 요약 - 주제별 정리

### MCP 프로토콜의 인기와 동시에 드러난 근본적 한계가 주요 이슈로 떠오름

- MCP(Multi-Component Protocol)는 최근 각광받으며 다양한 AI 및 서버 응용 서비스에서 활용되고 있음
- 기술의 확장과 동시에, MCP의 설계상 취약점이 명확하게 부각되기 시작함
- 제작자는 현재 가장 큰 이슈로 ‘토큰 소비(token consumption)’와 ‘문맥 붕괴(context rot)’를 지적함
- 이 두 가지 문제가 최근 AI 프로토콜과 인프라 분야에서 뜨거운 논쟁 지점임을 명확히 언급함

### 표준적인 MCP AI 코딩 서버 5개만으로도 심각한 토큰 낭비와 맥락 문제를 야기함

- 실제 제작자가 많이 사용해온 5개의 표준 MCP AI 코딩 서버를 예시로 듦
- 동일한 세션에서 이들 서버를 전부 사용하려 하면 문제가 급격히 심각해짐을 실례로 경고함
- 여러 서버가 동시에 입력될 경우 각각 수천 개의 토큰을 소비, 단순하게 서버 능력을 설명하는 데에만 상당한 리소스가 낭비됨
- “나는 절대로, 그리고 다시 한번, 절대로 이러한 서버들을 내 코딩 어시스턴트와 동일 세션에서 함께 쓰지 않는다”는 강조된 언급 포함
- 이로 인해 실제로는 여러 MCP 서버를 병용하는 것이 매우 비효율적이고 제한적임을 시사함

### MCP의 토큰 및 맥락 문제는 실사용에서 엄청난 비효율과 제약을 초래함

- 각 MCP 서버가 자신의 기능 설명을 위해 수차례 중복된 정보를 AI에게 제공해야 함
- 세션 내 토큰 한계로 인해 서버가 많아질수록 핵심 콘텍스트가 빠르게 소모·붕괴됨
- 결과적으로 중요한 맥락 정보 전달이 불가능해지고, 성능 및 사용자 경험이 하락함
- 제작자는 이러한 한계를 “진짜 최악”이라며 직접적으로 표현함

### Anthropic의 ‘Agent Skills’ 기능이 MCP의 구조적 문제를 본질적으로 해결함

- 최근 Anthropic에서 공개한 ‘Agent Skills’(에이전트 스킬) 기능을 해결책으로 소개
- Agent Skills는 AI 에이전트가 API 엔드포인트와 상호작용할 수 있는 스크립트를 직접 생성하도록 지원함
- 에이전트가 스스로 필요한 스크립트 작성, 사용법 설명, 즉석 코드 생성 및 실행까지 가능하게 만듦
- 이로써 각 서버의 기능 설명에 수천 개의 토큰을 반복 소모할 필요가 없어짐
- 코드 실행과 자동화 범위가 근본적으로 확대됨

### Agent Skills의 도입으로 토큰 낭비와 문맥 붕괴 문제를 효과적으로 극복할 수 있음

- MCP에서 발생하던 토큰 중복 소비와 맥락 붕괴 현상 해결에 직결됨
- 스크립트와 사용법을 동적으로 자동 생성해, 단일 세션 내에서 다양한 기능 확장이 용이해짐
- 코드 생성 및 실행을 실시간으로 처리함으로써, 협업 및 AI 어시스턴트의 효율성이 대폭 향상됨
- “코드 실행의 정수”라는 표현을 사용하며, Agent Skills의 실용성과 파괴적 가능성을 보여줌

### MCP 문제와 Agent Skills 해법의 핵심을 상세히 정리하며 결론에 도달함

- 영상 전반에서 MCP 프로토콜의 구조적 한계와 실질적 난점을 예시로 짚고 넘어감
- Agent Skills와 같은 혁신적 솔루션만이 토큰 및 맥락 이슈를 조기에 극복할 수 있다고 강조
- MCP 환경에서의 AI 및 서버 시스템 개발과 운용 패러다임이 급격히 변화할 가능성 시사

### 혁신적 대안의 등장으로 ‘기존 MCP 구조의 종말’ 가능성에 질문을 던지며 마무리함

- 마지막 부분에서 “이것이 지금껏 우리가 알던 MCP의 끝인가?”라는 의문 제시
- MCP와 유사한 프로토콜 환경의 근본적 전환점임을 암시함
- 전체적으로 Agent Skills의 도입이 MCP 시장 내부에 대변혁을 예고한다는 맥락으로 영상 종료
