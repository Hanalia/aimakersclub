## *나는 Claude에게 24시간 동안 쉬지 않고 코딩을 시켰다, 그 결과는?* 핵심 요약

- 영상은 인공지능 Claude Opus 4.5에게 단 한 번의 휴식도 주지 않고 24시간 동안 연속으로 코딩을 시키는 실험 과정을 다룸
- 실험 결과, 크롤닷에이아이(claw.ai)와 유사한 기능의 클론(복제) 프로그램이 거의 완벽하게 구현됨
- 대화 기록을 일관성 있게 남기며, 검증(loop) 단계마다 생성된 내용을 추적·확인할 수 있었음
- 마크다운 포맷을 깔끔하게 지원하고, 다양한 HTML 페이지 자동 생성 및 코드 작성·실행이 가능해짐
- 테마와 기본 모델을 바꿀 수 있는 설정 창, 최대 토큰 개수 조절 슬라이더 등 풍부한 인터페이스 구성 적용
- 인간의 개입 없이도 고도화된 기능을 빠른 시간 내에 구현했다는 점이 놀라움의 핵심으로 부각됨
- 추가로 ‘human-in-the-loop’ 기능(인간 검수 절차)을 계획하고 있으나, 현 상태만으로도 고도화·자동화된 생산성 구현이 가능함을 확인
- 비슷한 조건에서 하루만 더 작업한다면 사실상 ‘완벽한’ 애플리케이션까지 만들 수 있을 것이라 언급함
- Claude Opus 4.5 기반의 하네스(harness) 구조 덕분에 반복적 검증 및 자동화된 개선·구현이 자유로움이 강조됨
- ‘인공지능 코딩 에이전트’가 소프트웨어 개발의 새로운 패러다임 가능성을 보여준 사례

## 세부 요약 - 주제별 정리

### 인공지능 Claude Opus 4.5가 24시간 동안 코딩을 하면서 놀라운 개발 결과를 만들어냄

- Claude Opus 4.5를 활용하여 24시간 동안 쉬지 않고 자동화 코딩 실험을 진행함
- 참여자는 실제로 자신이 직접 코드를 작성하지 않았지만, Claude의 자동화 코딩만으로 많은 기능이 완성됨
- “처음부터 끝까지 손도 대지 않았다”는 언급으로 진정한 자동화의 모습을 부각함
- 이미 주어진 테스트의 절반이 지나기도 전에 ‘완전히 동작하는’ 클론 애플리케이션이 구현됨

### 크롤닷에이아이(claw.ai)와 흡사한 기능을 가진 클론 소프트웨어가 완성됨

- 유명 챗봇 서비스(claw.ai)의 기능을 모방한 클론 프로젝트를 구축함
- 기능적 완성도가 매우 높게 평가됨 (“이미 완벽히 작동하는 claw.ai 복제판처럼 느껴진다”)
- 실용적 수준으로 현존 서비스와 유사한 사용자 경험을 제공함

### 테스트 단계별 반복 검증과 대화 기록 추적이 가능해짐

- 애플리케이션 개발 과정에서 수행된 ‘루프’(loop) 단계별 검증 내역이 모두 기록됨
- 인공지능이 각 단계마다 자체적으로 생성한 대화·로그 기록을 저장하여 추적이 가능
- 과거의 대화 내역을 손쉽게 탐색하고, 검증 흐름을 시각적으로 파악할 수 있도록 지원함

### 깔끔한 마크다운 포맷 및 다양한 HTML 페이지 생성 기능이 구현됨

- 마크다운(markdown) 문서 포맷을 효과적으로 지원
- 컴포넌트별로 다양한 HTML 페이지를 자동 생성할 수 있음
- 자동으로 작성된 코드 또는 안내 내용을 HTML로 즉시 시각화 가능

### 코드 자동 작성 및 실행 환경까지 제공됨

- Claude가 직접 코드를 작성하고, 이를 즉시 실행해서 결과를 확인할 수 있는 환경이 구축됨
- 개발 과정의 반복과 실시간 검증이 완전히 자동화됨
- 코드/결과 피드백 및 디버깅까지 인공지능 자율적으로 수행함

### 사용자 인터페이스 세부 설정(테마, 모델, 토큰 수 등)까지 손쉽게 설계됨

- 다양한 테마(Theme)로 UI를 변경할 수 있는 옵션 제공
- ‘기본 모델’ 선택 기능을 통해 다른 AI 엔진 적용 가능
- 슬라이더(마우스 조정 UI)로 ‘최대 토큰 수’ 제한을 손쉽게 설정함으로써 세밀한 제어 가능

### 인간의 개입 없이도 높은 수준의 소프트웨어 자동화와 생산성이 입증됨

- 실험자는 “손가락 하나 까딱하지 않았다”며 전 과정이 AI 자동화였음을 강조
- Claude Opus 4.5의 엔드-투-엔드(full automation) 소프트웨어 빌드 능력이 드러남
- 빠르고 효율적으로 높은 품질의 애플리케이션을 생산 가능함을 입증함

### ‘human-in-the-loop’ 기능 추가를 통해 더 안전한 프로덕션 환경을 구상 중임

- 현재 시스템에는 아직 인간이 최종 검수하는 단계(‘human-in-the-loop’)가 적용되지 않았음
- 추후 이 기능이 도입된다면 실제 프로덕션(실서비스) 수준에서도 신뢰도와 안전성을 높일 수 있을 것이라 제시함

### 하루 이상의 추가 실행만으로 사실상 완벽한 제품화가 가능하다고 확신함

- 24시간 만에 이만큼의 자동 구현 성과를 낸 데 감탄
- “하루만 더 돌리면 거의 완벽한 앱이 될 것”이라는 개인적 확신이 언급됨
- 극한의 반복 검증 및 self-iteration을 통해 완성도를 비약적으로 높일 수 있음

### 자동화된 검증 하네스(harness)에 힘입어 반복적 개선과 전방위 테스트가 손쉽게 이루어짐

- 자동화 하네스(테스트 프레임워크)를 통해 인공지능이 프로덕트를 반복적으로 검증하고 개선할 수 있었음
- Claude Opus 4.5와 잘 연계된 하네스 구조 덕분에 추가적 인력이나 수동 테스트 없이도 품질을 보장
- 새로운 기능이나 개선 포인트도 반복적인 자체 테스트만으로 검증 가능함

### 이번 사례는 AI 코딩 에이전트가 기존 개발 방식에 혁신적 영향을 줄 가능성을 보여줌

- Claude Opus 4.5와 자동화 하네스 결합으로 개발 효율성과 완성도가 극대화됐음을 입증
- 더 적은 자원, 더 짧은 시간 내에 복잡한 소프트웨어 구현이 현실화될 수 있음을 시연
- 현존 AI 기술의 실질적 생산성과 잠재력을 단적으로 보여준 사례임
