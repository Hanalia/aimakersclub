## *AI 에이전트의 역량을 가장 간단하게 강화하는 방법: Anthropic의 Skills* 핵심 요약

- 본 영상은 Anthropic이 최근 선보인 "Skills(스킬)" 기능의 혁신성과 간결함에 주목하여, AI 에이전트를 효과적으로 강화하는 가장 단순한 방법을 설명함
- Anthropic의 철학은 "더 단순할수록 좋다"이며, 스킬은 불필요한 정보 과다로 인해 에이전트가 혼란스러워지는 문제를 해결함
- 기존의 MCP 서버 방식에서는 수많은 툴을 한 번에 에이전트에 제공하여, 실제 대화 중 쓰이지 않아도 모두 컨텍스트 창에 넣는 비효율이 있음
- Skills 기능의 핵심은 "점진적 공개(progressive disclosure)"로, 필요한 순간에만 능력을 드러내는 구조를 취함
- 시스템 프롬프트 단계에서는 스킬의 "설명(Description)"만 제공하고, 실제 요청이 있을 때에만 구체적 지침(skill.md)을 읽도록 설계됨
- 첫 번째 레이어에서는 해당 스킬의 존재 및 간단한 기능 설명만 전달함
- 두 번째 레이어에서는 사용자 요청에 따라 자세한 사용법 및 단계가 담긴 skill.md를 에이전트가 참조함
- 세 번째 레이어에서는 추가적으로 필요한 문서들을 skill.md가 지시하여 점진적으로 더 많은 정보를 로드함
- 이런 2~3단계 점진적 컨텍스트 주입 방식으로, 컨텍스트 창의 과부하 없이 에이전트 성능을 최대화함
- 영상에서는 PDF 처리(PDF processing) 스킬 구현 예시를 들어 전 과정을 구체적으로 설명함

## 세부 요약 - 주제별 정리

### Anthropic의 Skills는 에이전트 강화에 있어 최근 AI 업계의 가장 단순하고 혁신적인 발전임을 강조함

- Anthropic이 "Skills(스킬)"을 공개한 지 오래 되지 않았음에도 불구하고, AI 분야에서 큰 전환점이 될 만큼 중요한 진보임을 밝힘
- 그 중요성의 핵심 이유는 압도적으로 "단순한 설계"에 있으며, 이는 Anthropic의 철학(더 단순할수록 좋다)과도 일치함
- 복잡하고 방대한 툴셋보다는, 절제된 정보 제공과 체계적 기능 공개 방식이 중요한 혁신임을 강조

### 기존 MCP 서버 방식은 컨텍스트 창 과부하 문제를 야기하여 비효율적이라는 비판이 제기됨

- MCP 서버 방식 등은 시작부터 수많은 툴 및 기능 정보를 에이전트에게 일괄 제공함
- 실제 대화나 작업에서 필요한지와 관계없이 모든 툴 목록이 주어져, 대부분 컨텍스트 창을 낭비하게 됨
- 도구는 필요할 때만 사용되어야 하는데, 불필요한 컨텍스트 적재가 작업 정확성과 효율성을 떨어뜨림
- 따라서, 기존 방식은 에이전트의 “컨텍스트 윈도우(context window)”를 과하게 소모하는 근본적 한계가 있음

### Skills의 근본적 혁신은 ‘점진적 공개(Progressive Disclosure)’ 구조에 있다고 설명함

- 모든 기능을 즉시 노출하는 대신, 필요 시점에 맞춰 단계별로 점차 정보를 공개하는 것이 핵심
- 기본적으로, 에이전트는 각 스킬의 존재와 간단한 설명(Description)만을 시스템 프롬프트에서 전달받음
- 특정 요청이 들어왔을 때만, 그 순간 필요한 기능 설명서(skill.md) 또는 기타 문서를 참조하여 상세히 학습함
- 이로써 초기 컨텍스트 부담 없이 필수역량을 상황별로 "점진적으로" 활성화함

### 시스템 프롬프트에는 오직 스킬의 간단한 설명만 주어진다고 구체적으로 밝힘

- 예를 들어 PDF 처리 스킬이라면 "PDF 문서를 다룰 수 있는 기능이 있음"과 같은 요약 소개만 시스템 프롬프트에 실림
- 해당 스킬의 존재를 알면서도, 미리 상세 매뉴얼이나 복잡한 자료는 에이전트가 읽지 않음
- 이는 정보의 최소주의(minimalism)를 실천, 대화 초반에는 초경량 상태를 유지함

### 사용자가 특정한 요청을 해야만 skill.md라는 상세 설명 문서가 컨텍스트에 로드됨

- 실제로 사용자가 "PDF 문서를 분석해 달라"는 식의 요청을 할 경우에만 에이전트가 skill.md 파일을 참조함
- skill.md에는 해당 스킬의 전체적인 사용법, 구체적 단계, 예시 등이 자세히 기록되어 있음
- 이 과정을 통해 필요한 순간에만 정보를 로드하는 "점진적 정보 주입"이 실행됨

### skill.md 외에도 추가적인 지원 문서는 3차적으로 점진적 공개 방식을 통해 참조됨을 설명함

- skill.md 문서 내에서, 더 상세한 가이드나 참고 자료가 필요할 경우, 추가 문서(예: 첨부파일, 세부 설명서 등)를 연결할 수 있음
- 이렇게 3단계(간단한 설명 → 상세 설명서 → 추가 문서)의 계층적 컨텍스트 확장이 이루어짐
- 각 단계는 필요에 따라 순차적으로 활성화되어, 불필요한 컨텍스트 오염 없이 작업의 정확성과 유연성을 높임

### 점진적 공개 방식은 컨텍스트 윈도우 효율성 및 에이전트의 확장성을 극대화함

- 초기 정보 자원을 최소화하면서, 실제 업무 요구에 맞춰 점차 확장하는 구조로 에이전트 과부하 예방
- 다수의 스킬을 동시에 관리할 수 있고, 에이전트가 실제 필요에 따라만 역량을 발휘함
- 복잡한 시스템 프롬프트 구성 및 맥락 혼선 문제를 근본적으로 방지함

### PDF 처리 스킬 예제를 통해 구체적 적용 방식을 단계별로 시연함

- 사용자 요구에 따라 에이전트가 "PDF 처리 기능 사용 가능" 사실만 인식하는 1단계
- "PDF 분석" 요청을 받았을 때 skill.md에 기록된 구체적 처리 방법을 참조하는 2단계
- 복잡한 PDF 변환, 요약, 추출 등 추가 작업에 대해 별도 추가 문서를 참조하는 3단계 시나리오를 예시로 설명

### Skills가 Anthropic 내 모든 스킬 구현의 표준 파일 구조로 활용되고 있음을 명확히 소개함

- Anthropc의 모든 공식 및 커스텀 스킬 구현은 skill.md 파일을 핵심 운용 파일로 삼음
- 각 skill.md 파일 내에 2차, 3차 확장 기능 참조가 가능하도록 체계적으로 설계됨
- 시스템 프롬프트, skill.md, 기타 문서 간 유기적 연결 방식으로 에이전트 개발의 일관성과 효율성 도모

### Skills의 설계 철학은 AI 에이전트에게 최고의 유연성과 확장성을 부여하면서도, 사용자와 개발자 모두에게 단순하고 명확한 환경을 제공함

- 복잡성 최소화와 단계별 기능 제공의 조합이 실질적 생산성과 정확도 향상에 기여
- 명확하고 직관적인 기능 확장 메커니즘으로, AI 에이전트의 역량 활용을 극대화함
- Anthropic의 Skills(스킬)는 이후 에이전트 아키텍처 설계의 새로운 표준으로 주목될 가능성이 있음을 시사
