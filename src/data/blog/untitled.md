## *Claude 코드를 활용해 더욱 안전한 OpenClaw 대안을 직접 만들어봄* 핵심 요약

- OpenClaw 프로젝트는 현존하는 대표 오픈소스 AI 비서로, 폭발적인 인기를 끌고 있음
- OpenClaw는 사용자의 작업 상황과 선호도를 장기적으로 학습해 맞춤형 지원을 제공함
- 프로젝트는 완전한 오픈소스이며, 깃허브에서 185,000개의 스타를 기록함
- 이 수치는 n8n 등 대형 오픈소스 프로젝트들까지 제치고 역사적으로도 드문 기록임
- 하지만 OpenClaw는 제어권을 많이 부여하는 구조와 코드 자체의 결함으로 인해 심각한 보안 문제가 존재함
- 보안 문제를 해결하고 나만의 맞춤형 도구를 만들고자 직접 OpenClaw의 핵심 기능을 복제함
- 핵심 기능 복제는 수천 줄의 파이썬 코드와 마크다운만으로 불과 몇 일 만에 완료됨
- 누구나 주요 개념을 참고해, 조금만 노력하면 자신만의 커스텀 AI 비서를 쉽게 구축할 수 있음을 강조함
- 오픈소스 AI 어시스턴트를 학습의 재료로 삼고, 스스로 구축·관리함으로써 통제권과 맞춤화 이점을 누릴 것을 권장

## 세부 요약 - 주제별 정리

### OpenClaw는 사용자의 맥락과 선호도를 장기적으로 학습하는 첫 AI 어시스턴트임

- OpenClaw는 기존의 AI 어시스턴트들과 달리, 사용자의 최근 작업들과 선호도를 기억함
- 시간이 지날수록 사용자를 더 잘 이해하고, 점점 맞춤형으로 진화함
- "진짜 내 마음을 읽는 첫 AI 에이전트 같다"는 평가가 많음
- 개인 맞춤형 지원이 가능하다는 점에서 대중적으로 주목받음

### OpenClaw 깃허브 프로젝트는 이례적인 속도로 성장해 185,000 스타를 기록함

- 깃허브 저장소가 순식간에 185,000개의 스타를 획득
- 영상 제작자는 "이 정도의 별 개수는 본 적이 없다"고 언급함
- 같은 노코드 워크플로우 오픈소스 프로젝트인 n8n도 초월하는 인기로, 개발자 커뮤니티에서 반향을 일으킴
- OpenClaw는 완전히 오픈소스임을 강조함

### OpenClaw는 높은 수준의 에이전트 제어권과 코드 결함으로 보안 문제가 심각함

- OpenClaw는 내부 에이전트에게 많은 액세스 권한을 부여
- 이처럼 높은 제어권은 사용 편의성을 높이지만, 동시에 보안을 심각하게 저해함
- 코드베이스 내에도 구조적 결함 등이 있어 여러 보안 이슈가 지적됨
- "이렇게 많은 제어권을 쉽게 넘겨도 되는가?"라는 우려가 영상 속에서 언급됨

### 보안 우려와 커스터마이징 욕구로 인해 직접 OpenClaw 대안 솔루션 개발을 결심함

- 영상 제작자는 OpenClaw의 인기와 강력한 기능에 매력을 느끼면서도, 보안문제가 걸림돌로 작용했다고 밝힘
- 자신만의 니즈에 맞게 맞춤화할 수 있는 솔루션 개발에 동기를 얻게 됨
- 개방된 아이디어와 소스코드를 참고해 자신만의 오픈소스 AI 비서를 개발하는 것이 타협점임을 시사함

### OpenClaw의 강력한 핵심 기능은 직접 구현하기 어렵지 않음을 경험으로 증명함

- OpenClaw가 제공하는 주요 기능의 재현을 며칠 만에 완료했다고 설명
- 개발에는 몇 천 줄의 파이썬 코드와 마크다운 문서 작성만 활용했다고 구체적 수단을 제시함
- 본인처럼 다른 누구라도 조금만 공부하면 실현할 수 있다고 발표함

### 개방형 오픈소스 프로젝트에서 아이디어를 배우고, 직접 개발함으로써 이해도와 통제권을 강화할 수 있음

- OpenClaw 같은 강력한 오픈소스 AI 어시스턴트에서 구조 및 핵심 아이디어를 학습할 것을 권장함
- 다만, 직접 구축하면 보안과 커스터마이징에서 더 큰 장점이 있다고 강조함
- 소스 내부를 직접 관리함으로써 자신이 사용하는 도구를 깊이 이해하고, 위험도 직접 통제 가능함

### 누구나 손쉽게 자신만의 맞춤형, 안전한 AI 에이전트를 만들 수 있음을 다시 한 번 강조함

- OpenClaw, Claude Code 등 공개된 코드를 참고하면 복잡하지 않게 제작이 가능함
- 며칠 내에 기본 기능 재현이 가능했음을 본인 경험으로 구체적으로 밝힘
- 공동체의 지식·도구를 참고하되, 개별 커스터마이징의 중요성을 반복적으로 언급함
