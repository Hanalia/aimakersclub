## *RAG 2.0: 에이전틱 RAG와 지식 그래프의 결합 (무료 템플릿 제공)* 핵심 요약

- 영상 제목은 "*RAG 2.0: 에이전틱 RAG와 지식 그래프의 결합 (무료 템플릿 제공)*"임
- 제작자는 최근 여러 RAG(정보 검색 증강 생성) 전략을 연구했으며, 그중 에이전틱 RAG와 지식 그래프 전략의 시너지를 강조함
- 두 전략을 결합해, 질문에 가장 적합한 데이터 소스(벡터 데이터베이스 또는 지식 그래프)로 유연하게 접근할 수 있는 강력한 AI 에이전트를 구축함
- 벡터 데이터베이스는 Postgres(Neon의 서버리스 환경+PG vector 확장) 기반으로 구현, 텍스트 문서를 임베딩 후 저장
- 동일 정보를 Graffiti + Neo4j 기반 지식 그래프에도 저장, 엔티티 및 관계 기반 질의 지원
- 사용 예시에서, "Google의 AI 이니셔티브"와 같이 단일 회사 정보는 벡터 데이터베이스만 탐색하여 응답
- 반면, "두 회사 간의 관계" 등 관계 기반 질문에는 에이전트가 자동으로 지식 그래프를 활용함
- 사용자는 프롬프트에서 원하는 경우 벡터 DB와 그래프 모두를 명시적으로 활용하도록 지정 가능
- 실제 동작 예시들을 통해, 에이전트가 어떤 도구를 사용했는지 명확하게 결과와 함께 설명함
- 해당 AI 에이전트의 전체 구축 방법과 실행법을 보여주는 전체 영상도 무료로 제공, 템플릿 링크 안내

## 세부 요약 - 주제별 정리

### 제작자는 다양한 RAG 전략을 실험한 끝에 에이전틱 RAG와 지식 그래프의 장점을 결합함을 발견함

- 지난 몇 달간 영상 제작자는 거의 모든 RAG(정보 검색 증강 생성) 전략을 테스트함
- 특히 반복적으로 가장 효과적이라고 판단한 전략이 Agentic RAG(에이전트 기반 전략)와 Knowledge Graphs(지식 그래프)임
- 이 두 가지 전략이 서로 장점을 보완하며 효과를 극대화함을 강조

### 제작자는 두 RAG 전략을 결합한 AI 에이전트(Agent)를 구축함

- 제작자는 위 두 전략을 모두 결합한 강력한 에이전트형 AI 시스템을 직접 구축함
- 이 에이전트는 질문에 대해 “어느 데이터 소스(벡터 데이터베이스, 지식 그래프)에 질의해야 최적의 답을 얻을지” 스스로 판단하여 동적으로 결정
- “이 에이전트는 정말 멋지다(so cool)”고 표현하며 데모를 시연함

### 벡터 데이터베이스는 Postgres+PG Vector 확장 그리고 Neon 서버리스 플랫폼으로 구현됨

- 벡터 데이터베이스로는 Postgres를 채택하고 PG Vector 확장을 사용함
- 이를 Neon이라는 “슈퍼 멋진(serverless)” 서버리스 Postgres 플랫폼에서 구동
- 문서(예: 빅테크 기업 정보, AI 이니셔티브, 기업간 파트너십 등 포함됨)를 청킹(분할)해서 임베딩(vector representation) 생성 후, DB에 저장

### 동일한 정보를 지식 그래프에도 저장해 관계형 탐색 가능성을 높임

- 동일한 문서는 관계형 데이터 저장에 특화된 “지식 그래프”에도 삽입
- 지식 그래프에는 Graffiti와 Neo4j를 사용
- 지식 그래프는 인물, 조직, 파트너십 등 엔티티 및 그 관계 저장에 효과적
- 예시: “Amazon은 Anthropic이 모델을 호스팅하는 곳”이라는 관계도 저장 가능

### 에이전트는 질문의 종류에 따라 데이터 소스를 자동으로 판단해 탐색함

- 에이전트에 “Google의 AI 이니셔티브는 무엇인가?”처럼 단일 회사 관련 질문을 입력
- 이 경우, 에이전트는 벡터 데이터베이스가 더 적합하다고 판단하여 해당 경로로 정확히 정보를 검색
- 검색 결과와 함께, 실제 사용한 도구(이 경우, 벡터 DB가 사용되었음)까지 투명하게 알려줌

### 관계 추론이 필요한 질문에는 지식 그래프를 자동 선택함

- 질문이 기업 간의 관계 등 relational 성격이면(예: 두 회사 간 협력 내용), 에이전트는 시스템 프롬프트에 기반해 지식 그래프를 탐색
- 데모에서는 관련 질문 시, 벡터 DB 대신 그래프 데이터 소스(Neo4j 등)가 활용됨
- 응답 시, 역시 어떤 도구/데이터 소스를 썼는지 명확히 안내

### 벡터 DB와 그래프 DB를 동시에 활용하려면 프롬프트를 조정하거나 명시적으로 지정할 수 있음

- 사용자는 시스템 프롬프트 수정 또는 명령문에서 직접 “벡터 DB와 지식 그래프 모두 사용”을 지시할 수 있음
- 이때 에이전트는 두 경로 모두를 질의해 보다 종합적이고 정확한 답을 생성
- 해당 결과 역시 어떤 도구들이 함께 사용되었는지 명확히 제공

### 실제 예시를 통해 에이전트의 동작 및 도구 활용 내역이 투명하게 제시됨

- 각 예시에서 에이전트가 “어느 소스를 사용했고, 어떤 경로로 답을 도출했는지” 결과와 함께 상세히 나타냄
- 벡터 검색과 그래프 검색이 어떻게 구분되어 사용되는지 실제 동작 모습에서 확인 가능

### 개발에 ‘Cloud Code’가 활용되었으며, 무료 템플릿과 전체 구축 영상도 제공됨

- 이 AI 에이전트 구축에는 개발 지원용 서비스로 ‘Cloud Code’도 사용됨
- 영상에서는 ‘이 에이전트를 직접 구축하고 실행해볼 수 있는 무료 템플릿’의 링크 제공
- 영상 하단의 전체 영상 링크를 통해, 단계별 구현법 및 설정법을 상세히 확인할 수 있음
