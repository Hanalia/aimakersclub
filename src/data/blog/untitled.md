## *클로드에게 24시간 동안 코딩을 시킨 결과* 핵심 요약

- 영상 제작자는 Claude Opus 4.5 AI에게 24시간 동안 쉬지 않고 코딩을 수행하도록 맡겼음
- 제작 과정에서 claw.ai의 완전한 클론(복제 애플리케이션) 기능이 실제로 구현됨
- Claude가 콘벌루션(반복 검증 루프) 단계별로 생성했던 대화 기록을 모두 확인 가능
- 자동화된 코딩 결과물은 우수한 마크다운 포매팅 기능과 HTML 페이지 생성, 코드 작성 및 실행 능력 등 다양한 기능 포함
- 설정(Settings) 메뉴에서는 테마 변경, 기본 모델 선택, 최대 토큰 조절(슬라이더) 등 커스터마이즈 가능
- 별도의 인간 개입 없이 많은 기능이 추가되어, 제작자는 Claude의 성능에 놀라움을 표함
- 아직 인간의 확인(human in the loop) 기능이 남아있지만, 전반적으로 AI의 자동화 개발 성과가 인상적임
- 추가적인 반복 작업을 거치면, 더 완벽한 애플리케이션 개발도 가능할 것으로 기대됨
- Claude Opus 4.5와 같은 AI 모델과 자동 검증 툴(harness)의 조합이 검증 및 개선에 큰 도움이 됨
- 이번 실험은 최신 AI 코딩 도구의 강력한 자동화‧생산성 혁신을 간접적으로 보여줌

## 세부 요약 - 주제별 정리

### Claude Opus 4.5에게 24시간 연속 코딩을 맡겨 개발 실험을 진행함

- 영상 제작자는 Claude Opus 4.5 AI에게 24시간 동안 멈추지 않고 자동 코딩을 진행하게 함
- 별도의 직접적 개입이나 사람이 손대지 않고 AI가 개발 과정을 주도
- 주된 목표는 claw.ai 웹사이트와 기능이 동일한 클론 애플리케이션의 완전 구현
- 실험 목표와 과정 전반에 대한 짧은 리뷰와 소감이 영상 초반부에 언급됨

### 완전히 동작하는 claw.ai 클론 기능이 구현됨

- 제작자는 복제 프로그램이 사실상 완전히 동작하는 claw.ai 클론임을 강조
- 개발 테스트의 절반 이상이 끝난 시점에서도 이미 모든 주요 기능이 정상 작동 중임을 확인
- 일부 남은 테스트의 용도를 오히려 모르겠다고 언급할 정도로 결과물 완성도가 높음
- AI가 코드 뿐 아니라 실제 서비스 수준의 앱까지 자동 개발할 수 있음을 시사

### AI가 생성한 대화 기록과 코드 검증 과정을 모두 확인할 수 있음

- Claude가 반복적인 검증 루프를 돌며 생성했던 모든 이전 대화 내용을 열람할 수 있음
- 각 단계의 코드 생성 및 검증 내역이 남아 있어 개발 흐름 파악이 쉬움
- 이전 기록을 통해 문제 발생 시 롤백이나 원인 분석이 용이함

### 마크다운 문서화, HTML 페이지 생성 등 다양한 출력 형식을 지원함

- 코딩 결과물은 시각적으로 우수한 마크다운 포매팅(Markdown formatting) 기능이 포함됨
- AI가 다양한 HTML 페이지를 자동으로 생성하는 능력도 갖춤
- 소스 코드 초안부터 결과물까지 일관된 문서화가 자동 수행되어 생산성 향상에 기여

### 코드 작성 및 실행이 가능한 인터랙티브 환경이 구현됨

- 사용자 또는 검증자가 실제로 코드를 작성하고 바로 실행해볼 수 있는 기능이 추가됨
- 개발과 테스트 과정을 하나의 패키지에서 즉각 진행할 수 있어 편리함
- 코드 오류 검출 및 수정이 최소한의 수작업으로 가능함

### 설정 변경, 모델 선택 등 커스터마이제이션 지원이 강화됨

- 애플리케이션 내에서 테마(theme) 변경, 기본 AI 모델 선택 등 다양한 설정이 가능함
- 최대 토큰(token) 수를 조절할 수 있는 슬라이더(조절바)도 사용자 인터페이스에 포함
- 사용자는 자신의 취향과 용도에 맞춰 앱 환경을 맞출 수 있음

### 별도의 인간 개입 없이도 다양한 기능이 자동화되어 제작자는 AI의 성능에 깊은 인상을 받음

- 인간이 직접 코드를 작성하거나 UI를 제작하지 않아도 높은 완성도 실현
- 제작자는 “아무것도 손댈 필요가 없다”고 강조하며 AI 자동화 성능에 놀라움을 표함
- 인간의 검증(Human in the loop) 기능은 아직 추가하지 않았으나, 이미 다양한 고급 기능이 탑재됨

### 인간 검증자의 개입이 추가되면 완성도가 한층 더 높아질 것으로 기대됨

- 추후 개발 로드맵상 ‘Human in the loop’(사람이 확인하는 검증 절차)가 추가 예정임
- 현재 상태만으로도 매우 많은 기능을 구현했으나, 이 기능이 추가되면 신뢰도와 기능 완성도가 상승할 전망

### 추가 반복 및 기능 확장을 거치면 완벽에 가깝게 발전할 수 있음

- 제작자는 “하루만 더 이런 식으로 돌리면 완전히 완벽한 앱이 될 것 같다”고 평가
- 반복 실행과 피드백 루프를 강화하면서 성능이 계속 좋아질 것을 기대

### Claude와 자동 검증 도구(harness)의 결합이 효율적인 개발과 반복 검증에 필수적임

- Claude Opus 4.5와 같은 대형 언어 모델의 고성능 코딩·생성 능력이 핵심
- 별도의 검증·반복 확인 루프(=harness)를 붙여 실시간 코드 테스트와 개선이 자동화됨
- AI의 코드 품질 확인, 오류 즉시 수정 등의 반복 피드백 구조가 가능해져, 개발 생산성이 획기적으로 증대됨
