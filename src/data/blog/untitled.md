## *클로드 코드를 10배 더 강력하게 만드는 무료 도구* 핵심 요약

- 본 영상은 클로드 코드(Claude Code) 사용 경험자 관점에서 클라우드 코드의 강점과 주요 한계, 그리고 이를 탁월하게 보완하는 무료 도구 'Kinugi(키누기)'를 자세히 소개함
- 클로드 코드는 강력한 코드 생성 및 변경 능력을 갖췄으나, 여러 인스턴스를 병렬로 사용할 경우 AI가 출력한 모든 결과물을 직접 검토해야 한다는 점이 큰 단점으로 지적됨
- 병렬로 돌리는 여러 세션의 AI 코드 아웃풋을 한 눈에 파악하고, 효율적으로 리뷰·관리하는 과정이 기존에는 비효율적이었음
- 'Kinugi'는 클로드 코드 위에 시각적 레이어(Visual Layer)를 얹은 '에이전틱 개발 환경'(Agentic Development Environment)으로, 기존 워크플로우를 그대로 유지하면서도 AI 결과를 쉽고 빠르게 관리·검토할 수 있음
- 사용자는 하나의 화면에서 모든 세션의 진행 상태, 리뷰 대기 요소, 실행 중인 작업 등을 한 번에 확인·처리할 수 있으며, 에이전트의 변경 계획(Plan of Attack)에 실시간으로 코멘트하고 즉시 반영 가능
- 구현 결과는 기존 파일 로그를 하나씩 뒤지는 방식 대신 ‘풀 리퀘스트’ 형태로 깔끔하게 받아볼 수 있어 사용자 경험이 대폭 향상됨
- 보안 리뷰 등 별도의 코드 리뷰 레이어를 추가 적용하는 것이 가능하며, 이는 개발사인 Sonar(소나)가 보안 전문성을 갖췄기 때문에 실현이 가능해짐
- 키누기는 현재 맥OS에서만 지원, 실험적(Experimental) 성격이 강하지만 무료로 제공되고 있어 클로드 코드 유저에게 매우 실용적인 툴임
- Sonar가 이 도구를 개발했으며, 공식적으로 콘텐트 크리에이터에게 소개 요청 및 시연 기회를 제공함
- 영상 마지막에는 Sonar에 감사의 뜻을 전하며, 링크를 통해 직접 도구를 사용해볼 것을 권장함

## 세부 요약 - 주제별 정리

### 클로드 코드의 강력함과 현재 사용 현황이 제작자 중심으로 소개됨

- 클로드 코드(Claude Code)는 매우 강력한 AI 기반 코드 생성, 변환, 변경 도구로 인정받고 있음
- 제작자는 사내 모든 개발 작업에 클로드 코드를 전적으로 활용 중이라고 밝힘
- AI 코드를 통한 빠른 작업 처리, 여러 변경 사항을 신속하게 제안·적용 가능함
- 영상 예시: 클로드 코드가 병렬적으로 다양한 코드 변경을 한 번에 처리하는 장면이 시연됨

### 클로드 코드의 가장 큰 한계는 코드 리뷰 및 병렬 세션 관리의 불편함으로 드러남

- 클로드 코드가 여러 인스턴스(세션)로 병렬 작업을 진행할 때, AI가 생성한 결과물을 사용자 본인이 모두 직접 검토해야 함
- 이 과정을 "드래그(Drag)" 즉, 시간이 오래 걸리고 불편함이 크다고 표현
- 여러 세션의 코드 변경 사항을 파일 로그로 하나씩 들어가서 확인하는 기존 방식이 비효율적임
- 변경 계획, 실행 상태, 리뷰 필요 요소 등을 통합적으로 관리·파악하기 어려움

### Kinugi(키누기)는 클로드 코드의 워크플로우를 보존하면서 최적의 코드 리뷰 환경을 제공함

- Kinugi는 무료로 제공되는 실험적 도구로, 클로드 코드의 상위 시각화 레이어 역할을 함
- 클로드 코드를 대체하지 않고, 기존 도구 위에 시각적(Visual) 인터페이스를 더해줌
- 사용자는 기존 에이전틱(Agentic) 워크플로우(여러 AI 에이전트의 독립적·동시 작업)를 그대로 유지할 수 있음
- 클로드 코드와 완전히 통합되어 기존 워크플로우를 기반으로 움직임을 강조

### 모든 병렬 세션·에이전트의 실행 상태와 코드 변경점을 한 화면에서 손쉽게 관리할 수 있음

- 제작자는 Kinugi의 가장 큰 장점으로 '모든 병렬 세션을 하나의 화면에서 확인 가능'하다는 점을 꼽음
- 각 세션이 실행 중인지, 리뷰가 필요한지, 완료됐는지 등을 한눈에 파악할 수 있음
- “내가 전체 세션 현황을 보면서, 무엇이 내가 검토해야 할 사항인지, 무엇이 아직 실행 중인지 빠르게 알 수 있다”라고 구체적 기능을 설명

### 자동 생성된 'Plan of Attack'과 실시간 코드 리뷰, 피드백 적용이 가능함

- 각 에이전트는 코드 변경 계획(Plan of Attack)을 자동 생성하여 사용자에게 제시
- 사용자는 이 계획 단계에서 직접 코멘트를 남기고 즉시 피드백을 반영할 수 있음(실시간 인터랙션)
- 변경 사항이 실제로 구현(implementation)된 후에는 사용자에게 다시 검토 요청이 넘어옴

### 기존에는 개별 파일 로그를 수동으로 검토해야 했으나, 이제는 '풀 리퀘스트' 방식으로 혁신됨

- 기존 클로드 코드 로그/파일을 하나하나 다 확인하는 번거로운 방식이 풀 리퀘스트(Pull Request) 방식으로 대체됨
- 이로 인해 검토 속도 및 품질, 사용자 경험이 현저히 향상
- 사용자는 변경된 전체 내용을 깔끔하게, 하나의 정책적 뷰로 받아볼 수 있음

### 별도의 코드 리뷰·보안 리뷰 레이어 추가가 가능하며, 이는 Sonar팀의 전문성이 뒷받침됨

- Kinugi는 AI가 생성한 코드에 추가적으로 다양한 코드 리뷰 레이어를 적용할 수 있음
    - 보안 리뷰(Security review) 기능 내장
- Sonar(소나)팀이 개발한 도구로, Sonar는 본래 보안·코드 품질 검사 플랫폼으로 유명
- “보안 리뷰 기능이 들어간 것은 Sonar팀이 개발했기 때문에 자연스러운 일”이라고 언급
- 영상 제작자는 별도 보안 코드 리뷰 기능을 “정말 좋아한다”고 강조

### Kinugi(키누기)는 현재 맥OS에서만 지원되며, 실험적이지만 매우 강력함

- Sonar가 밝힌 공식 정보: Kinugi는 Mac OS에서만 사용 가능
- 현재 시점에서는 실험적(very experimental) 단계로, 개발 및 피드백 수렴 중임
- 이러한 실험성을 고려한 무료 제공임을 덧붙임
- 베타임에도 불구하고, 실제 사용에서 강력하고 편리함을 다수 체감

### 제작자가 Sonar와의 협업 배경을 밝혔으며 독자에게 직접 사용을 권장함

- Sonar팀이 제작자에게 Kinugi 도구 사용 및 영상 소개를 공식적으로 요청함
- 제작자는 “정말 멋진 도구이기 때문에 직접 경험해볼 수 있는 기회를 잡았다”고 언급
- 영상 후반부에서 Sonar사에 감사 인사, 그리고 Kinugi 사용을 위한 링크 안내
- 실제 도구 사용에 대한 높은 기대와 만족도를 직접적으로 표현하며 영상이 마무리됨
