## *나의 AI 기억이 이제 모든 도구를 따라다닌다!* 핵심 요약

- 영상 제목은 Cole Medin의 *"나의 AI 기억이 이제 모든 도구를 따라다닌다!"* 임
- 제작자는 ‘Claude Code’를 중심으로 한 자기만의 AI 프로젝트 관리와 기억 시스템(Second Brain)을 사용해왔다고 언급
- 다양한 AI 도구(Claude Code, Codex, Pi)를 상황별로 병행 사용하면서, 한 도구에 묶인 이전 기억 시스템에 한계를 느낌
- 이러한 문제를 해결하기 위해, 도구에 종속되지 않고 탈중앙화된 암호화 기억 시스템인 *Walrus and Memory*를 발견
- Walrus는 특정 도구에 얽매이지 않고, 본인의 데이터와 기억을 본인이 소유할 수 있도록 해 주는 외부 기억 시스템
- Walrus는 ‘MCP 서버’와 SDK를 통해 다양한 AI 코딩 도우미 및 앱에 쉽게 연동 가능
- 사용법이 매우 간단해, 홈페이지에서 프롬프트 한 줄만 복사해 붙이면 바로 사용할 수 있고 무료로 시작할 수 있음
- 실제 사용 예시에서는 대량의 텍스트에서 핵심 정보 20개를 추출·암호화·저장하고, 이를 Claude Code와 Pi 등 다양한 AI 도구에서 바로 검색·활용하는 모습을 시연
- Walrus 메모리 시스템을 통해 각각의 AI 에이전트에서 언제든 내 데이터를 암호화된 상태로 안전하게 검색·활용 가능함
- 사용자별 접근 제어와 위임키 기능 등 다양한 활용 옵션 제공하며, 빠른 세팅과 데이터 주권 보장을 장점으로 강조

## 세부 요약 - 주제별 정리

### ‘Claude Code’ 기반의 나만의 세컨드 브레인 메모리 시스템 경험과 한계

- Cole Medin은 그동안 Claude Code를 자신의 ‘세컨드 브레인’(두번째 두뇌)으로 만들어서, 자신과 프로젝트에 관한 모든 메모리와 정보를 쌓아옴
- 이 메모리 시스템 덕분에 생산성이 크게 향상됨
- 하지만 최근 Claude Code 이외에도 Codex, Pi 등 다양한 AI 코딩 도구를 병행 사용해야 하는 상황이 생김
- 각 도구별로 장점이 달라, 상황에 따라 적절한 툴을 선택해 사용함
- 그러나 Claude Code 속 메모리 시스템은 그 도구에 종속되어 다른 도구에서는 과거 기억 활용이 불가함
- 이로 인해, 여러 툴을 오갈 때마다 기억·데이터의 연속성 단절이라는 비효율적 문제가 생김

### 여러 AI 코딩 도구 간 기억·데이터 통합 필요성 인식

- Claude Code에 저장된 세컨드 브레인 정보 활용이 다른 AI 도구에서는 불가능하여 불편함을 느낌
- 여러 AI 코딩 툴 간 정보와 업무 맥락을 자유롭게 이동·공유할 수 있어야 생산성이 대폭 향상됨
- 이러한 필요성으로, 모든 도구에서 ‘같은 기억’을 활용할 방법을 모색하기 시작함

### 탈중앙화 암호화 메모리 시스템 ‘Walrus and Memory’의 발견

- 오랜 시간 해결책을 찾던 중, ‘Walrus and Memory’라는 도구를 발견
- Walrus는 특정 AI 서비스·회사에 얽매이지 않고 ‘외부’에서 작동하는 메모리 시스템임
- 모든 기억은 탈중앙화된 구조로 본인 소유(본인 데이터 주권)
- 모든 데이터는 암호화되어 저장되며, 회사의 인프라에 의존하지 않음

### Walrus의 설계 방식과 주요 특징

- Walrus는 AI 도구 외부에서 기억이 동작하여, 어느 플랫폼에서든 동일한 메모리에 접근이 가능
- 사용자의 프라이버시 보장 및 자기 데이터 소유(암호화·탈중앙화 구조)이 차별점
- ‘MCP 서버’와 연동하면 다양한 AI 도구에서 바로 메모리를 사용할 수 있다
- Walrus용 SDK를 제공, 개발자는 어떤 앱에서도 오픈 형태로 손쉽게 통합 가능
- 예시: 홈 페이지에서 단 1줄의 프롬프트만 복사해 붙이면 설치, 즉시 무료로 시작 가능

### Walrus 메모리 추가, 추출, 검색 실제 시연

- 대용량 텍스트(예: 세컨드 브레인 작업 관련 텍스트)를 붙여넣고 “Walrus로 기억해달라”는 지시만 전달
- MCP 서버가 자동으로 텍스트에서 주요 ‘메모리’ 20개 항목을 추출해 냄
- 추출된 항목은 모두 암호화된 상태로 저장 및 관리됨
- 사용자는 이 정보를 Claude Code 세션에서 수시로 검색·재활용 가능

### Claude Code와 Walrus 연동을 통한 실제 검색 예시

- Claude Code를 통해, 예를 들어 “Dark Factory 프로젝트의 기술 스택이 뭐였는지 내 Walrus 기억에서 찾아달라”고 질의 가능
- Walrus의 MCP 서버 불러와서, 저장된 정보 중 관련 내용을 찾아 정확한 기술 스택 정보를 즉각 제공
- 답변은 저장된 대량 텍스트에서 추출된 중요한 정보임을 확인

### ‘Pi’ 등 다른 AI 코딩 에이전트와도 손쉽게 연동 가능

- Pi 등의 AI 에이전트에서도 Walrus MCP 서버 연결만 하면 동일 메모리 접근 가능
- 별도의 복잡한 세팅 없이 동일한 질의를 할 수 있고, Claude Code와 동일 정보 제공
- Pi에서 받은 답변은 Claude Code 대비 다소 간결하지만, 내용적 일치 확인
- 이렇게 Walrus 시스템을 이용하면, 도구마다 별개로 정보·기억을 관리할 필요가 없어짐

### 접근 제어 및 위임키 등 다양한 메모리 활용 옵션 제공

- 모든 Walrus 기억은 사용자가 직접 소유·제어(탈중앙화, 암호화)
- 접근 제어, 위임키(delegation keys) 기능도 지원: 특정 시스템 혹은 다른 사람에게도 제한적 접근 허용 가능
- 다양한 수준의 보안 및 협업 시나리오를 유연하게 만족할 수 있음

### AI 도구용 범용 메모리 레이어로서의 Walrus의 가치와 사용 편의성

- Walrus는 AI 도구 전반에 걸쳐 쓸 수 있는 ‘기억 레이어’가 됨
- 시작이 매우 쉽고 빠름: 한 줄 프롬프트와 MCP 서버 세팅만으로 바로 적용
- 누구나 무료로 도입할 수 있으며, 영상 설명란 링크도 안내함
- 본인 데이터의 소유·보안·확장성을 모두 만족하면서도 개발과 사용이 간편함을 영상에서 강조
