## *Claude에게 24시간 내내 코딩을 시킨 결과* 핵심 요약

- 영상 제목은 “Claude에게 24시간 내내 코딩을 시킨 결과”로, Claude Opus 4.5를 활용하여 자체적으로 애플리케이션을 개발한 경험을 중심으로 진행된다.
- 사용자는 Claude가 진행한 작업 결과에 깊은 인상을 받으며 현재 상태에서 이미 완전한 claw.ai 복제 앱이 완성된 것 같다고 언급한다.
- 각 루프를 거치며 검증 과정을 자동으로 수행한 다양한 대화 기록을 모두 확인할 수 있다는 점이 강조된다.
- 마크다운(markdown) 형식의 보기 좋은 포매팅이 지원되어 문서화와 출력이 뛰어남을 언급한다.
- 여러 개의 HTML 페이지를 자동으로 생성하고, 코드의 작성과 실행까지 모두 Claude가 처리하였다.
- 설정(settings)에서는 테마(theme) 변경, 기본 사용 모델 선택, 최대 토큰 설정 슬라이더와 같은 다양한 기능이 구현됨을 설명한다.
- 코드 작성부터 검증, 문서화까지 굉장히 다양한 기능이 포함된 매우 feature-rich(기능 풍부)한 애플리케이션임을 강조한다.
- 아직 사람이 개입하는 ‘human in the loop’ 기능은 추가할 예정이나, 이미 아무런 손을 대지 않고 여기까지 구현한 것에 대해 놀라움을 표한다.
- Claude Opus 4.5와 같은 LLM, 그리고 반복적 검증 및 개발이 가능한 장치(harness)의 결합으로 엄청난 수준의 애플리케이션을 제작할 수 있음을 시연한다.
- 만약 하루 더 Claude에게 작업을 맡긴다면, 모든 기능이 완벽히 갖춰진 수준의 앱도 완성될 것이라고 기대감을 드러낸다.

## 세부 요약 - 주제별 정리

### Claude Opus 4.5만으로 claw.ai 복제 앱이 빠르게 완성됨을 보여줌

- 현재까지 개발된 것을 보면 이미 claw.ai의 완전한 복제본에 가깝다고 평가함
- 사용자는 전체 테스트의 후반부 목적조차 불분명할 만큼 기능이 거의 완성 단계에 도달했다고 밝힘
- Claude가 24시간 내내 코딩하면서 별도의 수작업 없이 앱이 완성됨

### 자동화된 반복 검증 및 개발 프로세스로 대화 기록이 남음

- 각 루프(loop)마다 기능 검증을 위해 Claude가 자동으로 대화(프롬프트 및 피드백)를 생성
- 이 대화 기록들은 누적되어 개발 진행 상황과 검증 결과를 쉽게 추적 가능
- 개발자의 개입 없이 자동화 검증이 이루어지고 있다는 점 강조

### 강화된 마크다운 포매팅이 문서화와 결과 출력을 용이하게 함

- 생성된 문서 및 결과물은 보기 좋은 마크다운(markdown) 형식으로 자동 포맷팅됨
- 이를 통해 코드, 결과, 설명문이 명확하게 구분되어 사용성 향상

### HTML 페이지 생성부터 코드 작성, 실행까지 전 과정이 자동화로 이뤄짐

- Claude가 다양한 HTML 페이지를 직접 생성·배포
- 코드의 작성뿐만 아니라 즉시 실행까지 자동화됨
- 사용자는 결과만 확인하면 됨

### 사용자 맞춤형 '설정' 기능도 자동으로 구현됨

- 테마(theme) 변경 등 외관 선택 기능 포함
- 기본 모델(default model) 변경 옵션 제공
- ‘최대 토큰’(max tokens) 양을 조절하는 슬라이더까지 설정에 탑재됨
- 일반적인 SaaS 애플리케이션 수준의 다양한 설정 기능 구현

### 기능이 매우 풍부하며 추가 개발 의지도 명확히 언급함

- 개발자는 “so so feature-rich”(기능이 너무 많다)라고 직접 언급
- 사람이 개입하는 검증 단계(human in the loop)는 추후에 추가할 계획임을 밝힘
- 현재까지 구현된 것만으로도 높은 완성도와 자동화 수준에 만족함

### 사용자의 수작업 없이 LLM이 고도화된 SaaS 앱을 구현하는 능력에 놀라움을 표함

- 개발자는 “손가락 하나 까딱하지 않고 여기까지 만들었다”고 표현
- 추가로 하루 더 작업하면 완벽에 가까운 애플리케이션도 가능하다고 말함

### Claude Opus 4.5와 검증 장치(harness)의 결합이 강력함을 시연함

- LLM(CLAUDE)의 자동화 능력과 다양한 검증, 반복 기능이 결합됨
- 검증 및 이터레이션 과정도 모두 시스템 내에서 자동 발생
- 개발 프로세스가 완전히 새로워질 가능성을 보여주는 사례임

### 마무리에서 LLM의 잠재력과 향후 확장 가능성을 암시하며 영상이 종료됨

- Claude와 같은 LLM이 실제 SaaS 수준의 애플리케이션 제작에 점점 더 적합해지고 있음을 시연
- 반복적 개발 및 자동 검증으로 수작업 최소화
- 앞으로 더 많은 기능이 추가될 수 있음을 예고하며 영상이 마무리됨
