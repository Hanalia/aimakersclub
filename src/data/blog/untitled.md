## *클로드에게 24시간 동안 쉬지 않고 코딩을 시켜본 결과* 핵심 요약

- 영상 제작자는 클로드(Claude) 모델을 이용해 24시간 동안 자동 코딩을 진행함
- 그 결과, **claw.ai**의 거의 완벽한 복제 기능을 갖춘 애플리케이션이 구축됨
- 테스트의 후반부는 필요성을 의심할 정도로, 이미 충분히 완성도 높은 기능들이 구현됨
- 과거에 생성된 대화 내역을 모두 탐색하고 검증 과정에서 생성된 변화를 추적할 수 있었음
- 마크다운 형식의 포멧팅, 다양한 HTML 페이지 생성, 코드 작성과 실행 등 여러 복잡한 기능이 포함됨
- 테마, 기본 모델, 최대 토큰을 조정하는 슬라이더 등 다양한 설정 기능도 구현됨
- 대부분의 작업은 인간의 개입 없이 자동으로 이루어졌으며, “손가락 하나 까딱하지 않았다”고 강조함
- 인간의 검토(human in the loop)를 추가하는 것이 남은 과제임을 언급
- 클로드 오푸스 4.5(Claude Opus 4.5)의 강력한 검증 및 반복(Validation & Iteration) 능력이 크게 기여함
- 만약 하루만 더 작업을 진행했다면 완벽에 가까운 애플리케이션이 될 것이라 언급
- 인공지능을 활용한 자동화·검증 프레임워크의 잠재력을 실질적으로 체험한 경험을 소개함

## 세부 요약 - 주제별 정리

### 클로드를 활용해 24시간 동안 자동 코딩을 시도한 과정이 매우 인상적으로 전개됨

- 영상 제작자는 인공지능 모델인 클로드(Claude)를 활용해 24시간 내내 코딩 작업을 자동화함
- 과정을 '인상적'이라 표현하며, 현재까지 만들어진 결과에 만족감을 드러냄
- 휴식 없이 진행된 이 실험은 인공지능의 지속적 작업 가능성, 높은 작업 속도를 보여줌

### 완전히 작동하는 claw.ai 클론이 자동화만으로 구현됨

- 자동화 과정을 거쳐 claw.ai의 “완전히 기능하는 복제판”이 만들어짐을 강조
- 기능 완성도가 높아 테스트 후반부의 필요성에 의문을 가질 정도였다고 밝힘
- 실제 claw.ai의 핵심 기능들을 모두 포함할 만큼 고도화된 결과물을 얻음

### 생성된 대화 내역 및 검증 과정을 완전히 추적할 수 있음

- 인공지능이 각 루프(반복 검증) 단계에서 실제로 생성한 대화들을 전부 되짚어볼 수 있음
- 과거 대화 기록과 애플리케이션 진화 과정을 명확히 살펴볼 수 있어, 변화 추적 및 검토가 용이함

### 마크다운 포맷팅, HTML 페이지 생성, 코드 실행 등 다층적 기능이 구현됨

- 결과물에는 미려한 마크다운 형식 포맷팅 기능이 포함됨
- 다양한 HTML 페이지를 생성하고, 그 안에서 코드를 직접 작성·실행하는 복합적인 기능도 갖춤
- 단순 대화형 인공지능을 넘어 실제 소프트웨어 생산 환경을 구현함

### 사용자 설정 환경, 테마, 모델 선택, 토큰 제어 등 세밀한 옵션 추가됨

- 사용자는 테마 변경, 기본 사용 모델 변경, 최대 토큰(답변 길이) 슬라이더 설정 등 다양한 옵션을 조정할 수 있음
- 실제 서비스 같은 풍부한 사용자 설정 메뉴가 빠짐없이 구현됨

### 기능의 다양성과 완성도에서 매우 풍부한 결과가 도출됨

- “그야말로 기능이 풍부하다(so so featurerich)”고 강조
- 단순한 복사(copy)가 아닌, 다층적 기능의 집대성으로 진일보한 프로덕트를 체험함

### 인간의 개입 없이 대부분의 코딩이 자동 진행됨

- 개발자는 “손가락 하나 까딱하지 않았다”며 실제 코드를 손으로 작성한 바가 거의 없음을 밝힘
- 자동화 수준이 매우 높아, 인공지능이 주도적으로 설계·개발·검증을 반복함

### 'Human in the Loop'(사람의 검증 역할) 추가가 남은 과제로 언급됨

- 현 상태에서도 자동화가 상당히 성공적으로 동작했으나, 향후 인간의 개입 기능을 추가하려 함
- 사람이 직접 검토하거나 승인·조정할 수 있는 구조가 필요하다는 점을 시사

### 하루만 더 반복 작업을 하면 사실상 완벽한 결과물을 기대할 수 있음을 암시

- 영상 마지막에서, 하루만 더 클로드에게 자동 코딩을 시킨다면 “절대적으로 완벽한 어플리케이션”이 나올 수 있다고 언급
- 인공지능의 꾸준한 반복 개선(V&V: validation & iteration) 능력에 신뢰를 표함

### 클로드 오푸스 4.5 및 검증·검토 구조가 혁신의 핵심 역할을 함

- Claude Opus 4.5의 강력한 검증과 반복 기능이 전체 프로젝트의 완성도에 결정적 역할을 했음
- “이런 하네스(harness, 프레임워크 구조)”를 통해 AI가 실제 애플리케이션을 반복 검증·개선해가는 경험을 실제로 목격함

### 실시간 검증과 반복 구조가 인공지능 자동화의 혁신적 잠재력을 드러냄

- 개발자는 결과물의 완성도에서 AI 기반 자동화의 한계를 다시 생각하게 됐다고 암시
- 실제로 복잡도·다양성 면에서 높은 수준의 제품을 인간 개입 최소화로 구현할 수 있음을 입증함
