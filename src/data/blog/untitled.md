## *AI 에이전트를 초강력으로 만드는 가장 간단한 방법(스킬 기능)* 핵심 요약

- **Anthropic이 최근 선보인 "스킬(Skills)" 기능은 AI 발전에 있어 가장 중요한 혁신 중 하나로 평가됨**
- **Anthropic의 철학은 "더 간단할수록 좋다(The simpler the better)"임**
- **AI 에이전트에게 여러 도구와 기능을 한 번에 주면 대화 컨텍스트 윈도우가 쉽게 과부하될 수 있음**
- **기존 MCP 서버 등은 사용하지 않을 도구까지 모두 사전 제공해 AI가 불필요하게 번잡해지는 단점이 있음**
- **스킬의 핵심은 '점진적 공개(progressive disclosure)'로, 에이전트가 실제 필요할 때만 관련 정보를 받아들이게 함**
- **초기에 시스템 프롬프트에는 기능(스킬)의 '설명'만 제공하며, 자세한 안내는 나중에 요청이 들어오면 읽게 됨**
- **예시로 PDF 처리 스킬의 경우, 시스템에는 "PDF 처리 스킬이 있음"만 전달되고 사용 요청 시 skill.md 파일을 로드함**
- **skill.md가 2차 점진 공개 계층에 해당하며, 추가 상세문서는 3차 계층으로 필요 시에만 읽힘**
- **이 점진적 계층 설계로 불필요한 정보 제공을 줄이고 에이전트의 효율성과 집중도를 극대화함**

## 세부 요약 - 주제별 정리

### Anthropic의 스킬 기능은 AI 발전에 있어 최근 가장 중요한 혁신 중 하나임

- Anthropic이 "스킬(Skills)" 기능을 전 세계에 공개한 시점은 그리 오래되지 않음
- 이 기능은 최근 AI 영역에서 이루어진 진보 중 가장 중요하다고 평가됨
- 발표자의 관점에서 이 기능이 중요한 이유는 "극도의 단순함" 또는 "아름답게 단순한 설계"에 있음
- Anthropic의 회사 철학 자체가 "더 단순할수록 더 좋다"임
- 스킬 도입으로 AI 에이전트의 실질 역량이 대폭 확장될 수 있음

### 기존 도구 제공 방식(MCP 서버 등)은 에이전트 컨텍스트를 과도하게 혼잡하게 만듦

- AI 에이전트를 강화하려면 다양한 도구와 기능을 제공하는 것이 이상적임
- 그러나 한 번에 너무 많은 정보를 주면 에이전트 컨텍스트 윈도우가 쉽게 과부하됨
    - 대화 히스토리, 도구 목록, 시스템 프롬프트 등 모든 정보가 컨텍스트에 들어가기 때문
- MCP 서버 등의 기존 방식은 불필요한 도구까지 에이전트에게 무조건 제공하는 문제가 있음
    - 실제로 특정 대화에서 필요하지 않은 도구까지 에이전트가 모두 인식함
- 이런 과잉 정보 제공 방식은 오히려 AI 대화의 품질을 해치고 비효율적으로 만듦

### Anthropic 스킬의 핵심은 '점진적 공개(progressive disclosure)','실제 필요할 때만 정보가 단계적으로 공개됨'에 있음

- 스킬의 가장 큰 차별점은 정보(도구, 기능, 설명)가 "필요할 때만 단계적으로" AI에게 제공된다는 점임
- 에이전트가 대화 초반에 모든 도구 정보를 받는 게 아니라, 실제 사용이 요청되는 순간 필요한 만큼 정보를 받음
- 이 구조가 과부하 위험을 크게 줄여주며, 에이전트의 맥락 집중력도 향상시킴

### 스킬 도입 방식은 정보 공개를 세 단계(계층)로 나눔

- **1차 계층(초기):** 시스템 프롬프트에는 각 스킬의 간단한 '설명'만 넣음(예: "PDF 처리 기능이 있습니다")
- **2차 계층(사용 요청 시):** 사용자가 관련 요청을 할 경우, 에이전트는 skill.md 파일을 읽어 구체적인 작업 방법을 학습함
- **3차 계층(추가 자료):** 필요에 따라 skill.md에서 참조하는 부가 문서들도 단계적으로 로드됨
- 이런 단계적 구조로 인해 불필요한 정보 제공 없이, 실제 필요한 순간에만 적정 정보가 주어짐

### 예시: PDF 파일 처리 스킬의 적용 사례

- 예시로 "PDF 처리 스킬"이 소개됨
- 에이전트에게는 먼저 "PDF를 다룰 수 있는 능력이 있다"는 설명만 시스템 프롬프트로 전달됨
- 사용자가 "PDF 파일에 대해 ○○ 해줘"라고 요청할 때, 그때서야 skill.md가 불러와짐
    - skill.md 안에는 실제 구현 방법, 지침, 입력/출력 방식 등 구체적 정보가 명시됨
- skill.md가 2차 점진 공개 계층을 담당하며, 추가로 참조할 문서들이 3차 계층이 되어 필요할 때만 열람됨

### 스킬.md 파일은 각 스킬의 중심 지침 역할을 하며, 실제 AI의 즉각적 역량 확장에 기여함

- skill.md는 모든 Anthropic 스킬의 핵심 파일로 기능
- 각 스킬의 구체적 실행 방식, 요구사항, 예시 등이 담김
- 커스텀 스킬 구성 시 skill.md 내용만 바꿔주면 즉각적인 역량 확장이 가능함
- 필요할 때만 불러오므로 그때그때 컨텍스트에 최적화된 정보만 제공됨

### 점진적 공개 계층 구조로 에이전트 컨텍스트의 효율성과 집중도가 극대화됨

- 정보가 단계적으로 제공됨에 따라, 에이전트는 오로지 '현재 필요한 정보'만 사용하게 됨
- 과정보다 결과 중심적이고 효율적인 수행이 가능해짐
- 결과적으로 불필요한 정보, 시스템 리소스 낭비, 맥락 혼란을 대폭 줄임

### 스킬 기능의 구조와 작동 원리는 타 방식 대비 명확하게 효율적임

- MCP 서버 등은 사전에 무조건 다수 도구 정보를 주므로, 낭비와 과부하가 빈번함
- Anthropic 스킬은 진짜 필요한 순간에만 실제 정보와 실행 방식을 로딩함
- 사용 가능 기능/도구의 존재는 에이전트가 이해하지만, 구체 방식은 나중에 전달됨
- 이러한 구조는 '에이전트 신뢰도와 집중력' 측면에서 혁신적임

### 결론적으로, Anthropic의 스킬(점진적 공개 툴)은 AI 시스템 설계에 있어 단순성과 효율성을 모두 달성함

- 스킬 기능은 복잡성 증가 없이 에이전트 역량을 대폭 확장할 수 있게 해 줌
- 점진적 공개 개념이자 실행 체계인 스킬 구조는, 미래 AI 설계의 기준이될 수 있음
- 필요 정보만 그때그때 제공하는 이 방식은 실제 현장에서 탁월한 효율성을 보여줌
