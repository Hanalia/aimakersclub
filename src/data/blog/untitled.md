## *MCP 서버의 큰 문제점과 그 해결책!* 핵심 요약

- **영상은 MCP(Multi-Component Protocol) 서버가 점점 더 인기 있어지고 있으나, 심각한 구조적인 결함을 갖고 있다고 지적함**
- **가장 큰 문제는 토큰 소비량 증가와 '컨텍스트 붕괴(context rot)'로, 최근 AI 분야에서 뜨거운 이슈임**
- **영상은 구체적인 MCP 서버 사용 예시를 통해 문제의 심각성을 설명함**
- **AI 코딩을 위한 일반적인 MCP 서버 5개만 사용해도 각 서버마다 수천 개의 토큰이 필요함**
- **특히, 각 MCP 서버의 기능 설명만으로 엄청난 토큰이 소모되어 실제 세션에서 여러 서버 병용이 비효율적임**
- **이 문제의 해결책으로 Anthropic에서 최근 출시한 '에이전트 스킬(agent skills)' 기능을 소개함**
- **'에이전트 스킬'은 AI 에이전트가 스스로 API 엔드포인트와 상호작용할 수 있는 스크립트를 생성하고, 실시간으로 코드 생성 및 실행 가능하게 함**
- **영상은 이 기능이 MCP의 토큰 및 컨텍스트 문제를 본질적으로 해결할 수 있는 '강력한 솔루션'이라고 평가함**
- **마지막으로, 이러한 변화가 MCP의 기존 개념에 종말을 가져올 수 있는지에 대한 의문을 제시함**

## 세부 요약 - 주제별 정리

### MCP 프로토콜의 인기가 높아지며 구조적 문제가 명확히 드러남

- MCP 프로토콜은 최근 많은 사용자를 확보하며 AI 코딩 등 다양한 분야에서 활용되고 있음
- 사용량과 응용 사례가 증가할수록 프로토콜의 핵심적인 설계 결함이 더욱 뚜렷하게 노출되고 있음
- 영상의 출연자는 "우리가 MCP에서 안고 있던 문제"를 명확히 집어 설명할 것임을 예고함

### MCP의 주요 결함은 과도한 토큰 소비와 컨텍스트 붕괴로 요약됨

- MCP의 가장 치명적인 문제는 토큰 소비량이 지나치게 많고, 이로 인해 대화 맥락(컨텍스트)이 잘려 나가는 '컨텍스트 붕괴' 현상으로 정리됨
- 토큰 과소비 및 컨텍스트 붕괴 문제는 현재 AI 기술 발전 논의에서 가장 핵심적인 이슈로 꼽힘
- MCP 서버들은 동작을 위해 각자 고유의 설명(프롬프트 등)이 필요하므로, 쓰임새가 확장될수록 소모 토큰이 기하급수적으로 늘어남

### 코딩 보조 시 여러 MCP 서버를 병행하면 토큰 사용량이 비효율적으로 커짐

- 실제 예시로, 영상 제작자는 "아주 표준적인 MCP 코딩 서버" 5개를 나열함
- 과거 이 서버들을 각각 별도로 자주 사용했으나, "서로 조합해 같은 세션에서" 쓰는 일은 엄두도 내지 못했다고 강조함
- 핵심 이유는 "각 서버가 AI에게 자신의 기능을 설명하는 데만 수천 개의 토큰을 활용한다"는 점임
- 이런 구조에서는 서버가 많아질수록 서버 설명용 토큰만으로 요청 컨텍스트가 소모되어, 실제 코딩 지원이 불가능해짐

### MCP의 구조적 한계를 극복하기 위한 솔루션으로 '에이전트 스킬'이 제시됨

- Anthropic사가 최근 공개한 '에이전트 스킬(agent skills)' 기능이 MCP의 근본적인 결함에 대한 해결책으로 언급됨
- '에이전트 스킬'은 AI 에이전트가 필요에 따라 실시간으로 스크립트를 생성할 수 있게 해줌
- 스크립트는 API 엔드포인트와 상호작용하거나, 새로운 기능 구현 코드를 즉시 만들어낼 수 있음
- 즉, 기존 MCP와 달리 서버 기능 설명에 불필요한 토큰을 반복해서 소모하지 않아도 됨

### '에이전트 스킬'은 코드 생성과 실행 측면에서 매우 강력한 기능임

- AI 에이전트가 상황에 따라 API 사용법을 자동으로 생성하고, 지침에 따라 즉석에서 코드를 편성 및 실행함
- 이 능력이 토큰 사용량 절감은 물론, 에이전트의 실질적 활용성을 극대화하는 데 기여함
- 영상 제작자는 '에이전트 스킬'이 제공하는 확장성과 파워를 강조하며, MCP 구조 한계의 '본질적'인 개선으로 평가함

### MCP가 지금의 모습으로 유지될 수 있는지에 대해 근원적 질문을 던짐

- 영상 제작자는 "MCP가 지금과 같은 방식으로 존재할 수 있느냐"는 의문을 시청자에게 남김
- 최근의 기능 혁신과 기술 변화가 MCP 생태계의 방향성과 존재 이유 자체를 위협할 수 있음을 암시
- 이처럼 영상은 MCP 문제 제기에서 시작해, 대안의 제시, 근원적 변화 가능성 탐색까지 단계를 밟으며 내용을 매듭지음
