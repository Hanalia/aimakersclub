## *RAG 2.0: 에이전틱 RAG와 지식 그래프의 결합(무료 템플릿 소개)* 핵심 요약

- 영상 제목은 “RAG 2.0: 에이전틱 RAG와 지식 그래프의 결합(무료 템플릿 소개)”로, 두 가지 최신 RAG 방식을 결합한 강력한 AI 에이전트를 소개함
- 제작자는 수개월간 다양한 RAG(검색 증강 생성) 전략을 연구한 결과, ‘에이전틱 RAG’와 ‘지식 그래프’의 조합이 가장 효과적임을 발견함
- 이 에이전트는 주어진 질문에 가장 적합한 정보를 찾기 위해 어디를 탐색할지 스스로 판단하고 추론함
- 벡터 데이터베이스로 Postgres + PG Vector, 서버리스 플랫폼 Neon을 사용해 문서를 청크 단위로 저장 및 임베딩 처리함
- 저장한 문서는 빅테크 기업들의 AI 이니셔티브 및 파트너십 정보를 담고 있음
- 동일 문서를 Graffiti와 Neo4j를 활용해 관계형 방식(지식 그래프)으로도 저장하여 두 가지 데이터 접근 방식을 제공함
- 에이전트는 질문의 유형에 따라 벡터 DB를 조회하거나, 관계성 탐색이 필요할 때는 지식 그래프를 검색함
- 사용자가 원하는 경우, 시스템 프롬프트나 명령을 통해 두 데이터 소스를 동시에 이용할 수도 있음
- 에이전트가 어떤 도구(벡터 검색, 그래프 검색)를 사용했는지 답변과 함께 명시함
- 이 에이전트의 전체 구현 방법과 상세 설명은 별도의 전체 영상 및 무료 템플릿을 통해 제공함

## 세부 요약 - 주제별 정리

### 다양한 RAG 전략을 연구한 끝에 에이전틱 RAG와 지식 그래프의 결합이 최적임을 확신함

- 제작자는 최근 몇 개월간 거의 모든 유형의 RAG(검색 증강 생성) 전략을 심층적으로 분석 및 실험함
- 반복적으로 효용성이 입증된 방식 두 가지는 ‘에이전틱 RAG(Agentic RAG)’와 ‘Knowledge Graphs(지식 그래프)’임
- 이 두 방식을 결합함으로써 단순한 정보 검색을 넘어, 정보의 맥락을 이해하고 다양한 유형의 질문에도 유연하게 대응 가능해짐

### 에이전트는 질문의 맥락에 따라 정보를 찾는 전략을 스스로 결정함

- AI 에이전트는 사용자의 질문에 가장 적합한 대답을 위한 최적의 정보 출처(벡터 DB, 지식 그래프 등)를 자체적으로 판단함
- 사용자의 질문을 받아 필요한 분석 및 추론을 거쳐, 기존 고정형 검색 방식과 달리 동적으로 검색 전략을 선택함
- 예시: 단일 회사의 정보가 궁금할 때는 벡터 데이터베이스만 검색, 두 회사의 관계를 물어보면 지식 그래프를 주로 사용함

### 벡터 데이터베이스는 Postgres + PG Vector, 서버리스 플랫폼 Neon을 사용함

- 벡터 데이터베이스로 Postgres(Relational DB)에 PG Vector 확장 기능을 추가하여 고차원 임베딩을 저장하고 검색함
- 저장된 데이터는 클라우드 기반 서버리스 Postgres 플랫폼 ‘Neon’을 사용해 운용됨
- 문서 데이터를 ‘청크’ 단위로 분할 후 임베딩을 생성하여 데이터베이스에 삽입
- 예시 문서에는 대형 IT 기업들의 AI 이니셔티브 및 상호 파트너십 정보가 포함됨

### 동일 정보를 지식 그래프에 Relational 방식으로 저장하여 관계 탐색 효율을 높임

- 동일한 문서를 Graffiti와 Neo4j 기술을 이용해 지식 그래프(관계형 데이터 구조)로도 저장함
- 지식 그래프는 에이전트가 기업 간의 다양한 관계(예: "Amazon은 Anthropic의 모델 호스팅 위치"와 같은 정보)를 효과적으로 탐색·출력하게 해줌
- 이중 저장으로 질의 타입별 맞춤형 데이터 검색이 가능해짐

### 질문 예시에 따라 벡터 검색과 그래프 검색이 구분되어 사용됨

- 에이전트에게 “Google의 AI 이니셔티브는 무엇인가?”와 같은 단일 정보 요청에는 벡터 검색을 사용
- 실제 시연에서, 에이전트가 벡터 데이터베이스를 검색한 뒤 답변을 제공함(검색 도구 및 경로 명시)
- 두 기업간의 관계나 협업을 묻는 요청에는 자동으로 지식 그래프 검색으로 전환
- 예시: “Amazon과 Anthropic의 관계”를 질문하면 지식 그래프를 검색해 답변함(사용 도구 명확 표기)

### 시스템 프롬프트나 명시적 요청을 통해 두 데이터 소스를 모두 활용 가능함

- 사용자(또는 시스템 프롬프트)가 필요 시 ‘벡터 검색’과 ‘그래프 검색’ 두 가지 모두를 활용하라고 명령 가능
- 시연에서는 “벡터 데이터베이스와 그래프 모두 사용하라”는 명령을 내리고, 에이전트가 두 데이터 소스에서 각각 검색해 통합 답변 제공
- 이 방식은 단일 정보만 제공하는 것보다 신뢰성 있고 포괄적인 응답을 가능하게 함

### 사용자가 에이전트가 활용한 도구와 검색 방식을 투명하게 확인할 수 있도록 설계됨

- 에이전트는 매 응답마다 “방금 어떤 툴(벡터 DB 혹은 그래프 DB)을 사용했는지”, ‘Tools Used’란을 통해 명확하게 고지
- 이는 사용자로 하여금 각 답변의 신뢰도, 데이터 소스, 작동 로직 등을 이해하는 데 도움을 줌

### 영상의 전체 소스코드와 구현 과정, 템플릿은 무료로 제공되며, 자세한 가이드에 별도 링크가 있음

- 이 AI 에이전트의 모든 구현 과정은 무료 템플릿 형태로 별도 영상과 링크에서 제공
- 구체적 구성 방법, Cloud Code 활용법, 개발 지원 기능 등 상세 구축 가이드가 별도 풀영상에서 안내됨
- 템플릿 사용 시, 사용자도 동일한 AI 에이전트를 직접 구축해 활용 가능함
