## *포스트맨의 Git 내장화로 API 개발 방식이 완전히 혁신됨* 핵심 요약

- 기존 API 작업은 엔드포인트, 컬렉션, 테스트, CI 작업이 각각 다른 도구·장소에서 분리 관리되어 비효율적임
- 특히 AI 에이전트 개발 시 다양한 API 엔드포인트를 자주 다루지만, 스펙과 테스트, 코드가 분산되어 업무 흐름이 복잡했음
- 최신 포스트맨은 Git과 직접 연동되어 API 개발 생애주기 전반을 하나로 통합하고 버전 관리 기능을 강화함
- 컬렉션, 스펙, 목(Mock), 테스트 등 모든 Postman 리소스를 로컬 코드베이스 내부의 YAML 파일로 간단히 관리 가능
- 포스트맨 데스크톱 앱과 로컬 파일간 양방향 동기화 지원, 클라우드와도 실시간 연동
- 변경 사항은 git diff로 즉시 확인 가능하며, 코드처럼 명확히 버전 제어 및 협업 리뷰가 가능해짐
- 에이전트 모드에서 "자연어"로 요구사항 입력 시 코드베이스 자동 스캔하여 라우트, 스펙, 테스트, 목까지 생성·동기화
- 생성된 모든 변경 사항은 git 커밋으로 남아 코드리뷰 및 이력 관리가 손쉽게 이루어짐
- Claude, GPT, 셀프호스팅 등 다양한 AI 모델 선택 가능, GitHub·Sentry·Linear 등 주요 개발 도구들과 컨텍스트 연동 지원
- 로컬에서 실행한 API 컬렉션과 테스트를 동일하게 CI 파이프라인 및 GitHub Actions, pre-commit에서 손쉽게 재사용 가능
- 그 결과, 환경 별 테스트 재작성 없이 일관되고 자동화된 API 개발·검증·배포 환경을 제공하며, 개발자가 즉시 사용할 수 있음

## 세부 요약 - 주제별 정리

### 기존 API 워크플로우는 도구와 작업이 분산되어 비효율적임

- API 작업 시 엔드포인트 구현은 보통 IDE에서 하게 되며, 테스트 및 컬렉션 관리는 다른 툴(Postman 등)에서 별도로 이루어짐
- continuous integration(CI) 환경에 테스트를 연동하려면 다시 별도 작업과 세팅 필요
- API 스펙, 컬렉션, 테스트 코드, 실제 구현 코드가 모두 서로 다른 위치에 존재하여 작업 흐름이 단절됨
- 버전 관리가 통합되지 않아 협력 개발 및 변경 추적이 복잡하고 오류 가능성 높음
- 특히 AI 에이전트 개발 환경에서 자주 발생하는 문제임

### 포스트맨은 Git 연동으로 이 모든 불편함을 한 번에 해결함

- 포스트맨의 최근 대규모 업데이트로 API 개발의 모든 리소스를 코드베이스 내부로 통합 가능해짐
- 컬렉션, 스펙(Specs), 목(Mock), 테스트(test) 등 모든 자원을 로컬 저장소에서 YAML 파일 형태로 관리할 수 있음
- YAML 파일은 읽기 쉽고 유지보수가 간편하여 팀 작업이 쉬워짐
- 로컬 파일 관리이므로, 코드와 동일한 방식으로 분기 관리 및 이력 추적이 가능해짐

### 데스크톱 앱과 클라우드 기반 양방향 동기화가 즉각적으로 제공됨

- 로컬 코드베이스의 Postman 리소스와 데스크톱 앱에서 작업한 내용이 자동으로 동기화됨
- 필요시 Postman 클라우드에 연결하여 실시간 양방향 동기화 기능 제공
- 로컬에서 컬렉션·엔드포인트 등을 실시간으로 추가, 변경하면 클라우드 역시 동기화

### Git을 통한 명확한 변경 이력 추적 및 협업 워크플로우 구현임

- 테스트 파일 등 리소스를 수정하면 git diff로 변경점을 즉시 시각적으로 확인 가능
- 모든 변경 사항은 git commit으로 남아 코드리뷰, 롤백, 분기 생성 등 깃베이스 협업 가능
- API 스펙 및 테스트 파일을 코드베이스와 함께 정확히 버전체크·관리할 수 있음

### 모든 포스트맨 리소스가 로컬의 평문 파일로써 직접 관리 가능함

- Postman 리소스가 더 이상 서비스 내부 DB에만 한정되지 않고, .yaml 등 파일로 노출됨
- 이를 통해 개발자가 코드처럼 언제든 수정·추적이 가능하고, 자동화 파이프라인에 쉽게 통합할 수 있음

### 포스트맨 에이전트 모드는 자연어 입력으로 전체 API 생애주기 자동 관리 가능

- 사용자가 자연어(plain English)로 "Flask 앱 컬렉션 생성" 등 필요를 작성하면 자동으로 동작
- 에이전트가 전체 코드베이스를 스캔, 라우트(route) 발견, 관련 컬렉션 및 스펙 생성
- API 테스트, 목(mock) 세팅까지 자동화 처리
- 모든 산출물은 코드베이스의 파일·Git 커밋으로 기록됨

### 생성 및 동기화된 모든 API 관련 변경은 깃 커밋으로 코드리뷰가 가능해짐

- Postman의 모든 자동화 작업 결과물이 git 커밋에 고스란히 남아, 코드리뷰·이력 관리가 쉬어짐
- API 정의, 테스트, 목 등이 협업의 표준 프로세스를 그대로 따를 수 있음

### 다양한 AI 모델 및 주요 개발 도구와의 연동성도 강화됨

- 에이전트에서 Claude, GPT, 셀프 호스팅 모델 등 다양한 AI 모델 선택 가능
- GitHub, Sentry, Linear 등 이미 사용중인 개발 도구에서 필요한 컨텍스트를 끌어와 활용 가능

### CI 파이프라인에서도 로컬과 완전 동일한 환경을 재사용할 수 있음

- 로컬 PC에서 생성 및 테스트한 컬렉션 그대로 CI(continuous integration) 환경에서 실행 가능
- pre-commit hook, GitHub Actions 등 다양한 자동화 환경에서 별도 변환 없이 코드를 재활용할 수 있음
- 환경에 따라 테스트 재작성 등 추가 작업이 필요 없으므로 유지보수 효율이 대폭 향상됨

### 최신 포스트맨은 즉시 실무 개발 적용 및 최신 워크플로우 도입에 용이함

- AI native API 플랫폼으로 격상되어, 최신 AI 및 API 개발에 맞는 통합 워크플로우 제공
- 영상 제작자가 "10년 간 써온 포스트맨이 최근 혁신적으로 개선됐다"고 평가할 정도로 큰 변화임
- 실제 적용 사례(GitHub 리포지토리 예시)와 함께 바로 사용할 수 있도록 구현되어 있음
