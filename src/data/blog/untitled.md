## *유튜브 콘텐츠용 풀스택 RAG 에이전트 최초 라이브 구축* 핵심 요약

- Cole Medin 채널에서 AI 기반 개인 코치(에이전트) 개발 과정을 최초로 라이브 스트림에서 시연함
- 목표는 채팅 인터페이스 구축으로, 사용자가 질문을 입력하면 유튜브 콘텐츠 기반 고급 RAG(검색 증강 생성) 에이전트가 답변하도록 설계됨
- 영상 내용 전체와 AI 코딩 관련 질문에 대응하는 에이전트를 개발하는 것이 궁극적인 목적임
- 데이터 전처리 단계에서는 최신 ‘하이브리드 청킹(hybrid chunking)’ 전략을 사용하여 데이터 분할의 정확도와 효율을 높임
- Dockling 라이브러리를 활용해 하이브리드 청킹을 구현하며, 기존 커스텀 청킹 전략들보다 월등한 성능을 확인했다고 강조함
- 청킹된 데이터는 Supabase 데이터베이스에 저장하여 검색 효율을 극대화함
- 전체 파이프라인 구성은 ‘RAG 파이프라인 구축 → 데이터베이스 저장 → 에이전트 개발’ 순서로 진행됨
- Pideantic AI 툴을 활용해 최종 에이전트 구현 예정임
- 영상 초반에 데이터 처리 및 RAG 파이프라인 구축, 이어서 에이전트 인터페이스 제작 순서로 진행됨
- 개발 과정 전반에 걸쳐 실제 예시와 실험 결과에 기반한 설명이 제공됨

## 세부 요약 - 주제별 정리

### AI 코칭용 개인 에이전트 개발이 채널의 오랜 목표였음을 소개함

- Cole Medin은 자신의 유튜브 채널에 적합한 AI 코치(에이전트) 개발을 오래 계획해왔음을 밝힘
- 해당 에이전트는 AI 코딩과 AI 에이전트 관련 질문들을 전문적으로 처리할 수 있도록 설계
- 영상에서 이 목표의 실현 과정을 라이브로 공개하는 점을 강조

### 최종 목표는 유튜브 콘텐츠 기반 고급 RAG 챗봇 구축임을 밝힘

- 시연의 궁극적 결과물은 사용자가 질문을 받으면 유튜브 내 모든 콘텐츠를 바탕으로 대답하는 채팅 인터페이스임
- 챗GPT와 유사한 질의응답 방식이지만, 답변의 기반 데이터는 자신의 유튜브 데이터임
- 이 구조를 통해 기존 챗봇보다 더욱 개인화되고 전문화된 지식을 제공 가능

### 하이브리드 청킹이 RAG 파이프라인에서 핵심 역할을 함을 설명함

- 데이터 chunking 단계에서 최근에 도입한 "하이브리드 청킹(hybrid chunking)" 방식을 중점적으로 활용
- 기존에 실험했던 다양한 커스텀 청킹 방식들보다 확연히 우수한 결과를 제공함을 체험적 사례로 설명
- "최근 내 RAG 성능을 극적으로 끌어올린 요인"임을 강조
- 하이브리드 청킹 적용을 위해 Dockling 패키지를 적극 활용함
- 실험적으로 얼마나 탁월한 차이를 보였는지 'insane'이라는 표현을 사용해 효과를 강조

### Dockling을 사용하여 데이터 전처리 효율성을 높임을 강조함

- Dockling 도구는 하이브리드 청킹 전략 구현에 최적화된 오픈 소스 라이브러리임
- Dockling의 채택 이유와 실제 적용 경험에 대해 ‘특히 흥분된다’고 표현
- 다양한 청킹 전략을 Dockling으로 실험해본 결과, 기존 방법을 압도하는 효율을 확인

### Supabase에 청킹 데이터 저장으로 빠른 검색을 구현함

- 전처리 및 청킹된 데이터는 Supabase 데이터베이스에 저장함을 명시
- Supabase를 선택한 이유로 데이터 신속 검색 및 관리 편의성을 암시
- 전체 RAG 시스템 파이프라인에서 중요한 데이터 저장소 역할을 함

### 개발 단계는 RAG 파이프라인 구축부터 시작됨을 명확히 함

- 라이브 스트림 초반부는 데이터 처리 및 RAG 파이프라인 구축에 집중됨
- "스트림의 시작 부분에서 바로 이 부분을 다룬다"고 구체적으로 밝힘
- Chunking과 데이터베이스 구축이 에이전트 개발의 선행 작업임을 명확히 함

### Pideantic AI로 최종 에이전트 제작을 예고함

- RAG 파이프라인 구축과 데이터 저장 이후, 실제 에이전트 구축 단계가 진행됨
- 이 때 Pideantic AI라는 도구를 사용할 계획임을 언급
- Pideantic AI의 구체적 기능이나 역할에 대해선 짧게 언급되어 있으나, 전체 에이전트 프레임워크의 핵심으로 등장

### 전체 개발 구조는 단계별 순서로 직관적으로 설계됨을 밝힘

- “RAG 파이프라인 구축 → Dockling 활용 청킹 → Supabase 저장 → 에이전트 개발”이 핵심 개발 흐름임을 명확히 제시
- 각 단계가 실제 라이브 스트림의 진행 순서와 일치함

### 이번 에이전트 개발 시연은 실험적 경험과 실질적 효과를 기반으로 함

- 개발 과정 중 자신이 직접 테스트하며 경험한 다양한 chunking 전략들과 결과를 투명하게 공유
- Dockling 하이브리드 청킹이 얼마나 드라마틱한 향상을 가져왔는지 구체적으로 강조
- 수년간의 RAG 관련 실습 노하우가 집약된 사례임을 간접적으로 드러냄

### 영상의 전개는 구체적인 실습 예시와 단계별 설명이 포함되어 있음

- 실제 데이터 적용 및 처리 과정, 도구 사용 예시 등을 순차적으로 삽입
- 각 단계별로 왜 해당 전략을 선택했는지, 시행착오는 무엇이었는지 언급
- 영상을 통해 AI 기반 개발, 데이터 전처리, 에이전트 구축의 실전 프로세스를 독자적으로 따라할 수 있게 설명 제공
