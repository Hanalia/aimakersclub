## *내 GitHub 이슈를 내 컴퓨터 터치 없이 AI가 자동으로 해결해주는 무료 도구* 핵심 요약

- 영상은 코딩 에이전트가 사용자의 로컬 컴퓨터를 전혀 사용하지 않고 GitHub 이슈를 자동으로 해결해주는 과정을 소개함
- 데모에서는 사용자가 직접 코드를 작성하지 않고 오로지 GitHub 이슈에 "coder" 태그만 부여하면 모든 작업이 시작됨
- 이 프로세스 구현에는 완전 무료이자 오픈소스 도구인 "Coder 플랫폼의 Workspaces"가 활용됨
- Workspaces는 AI 코딩 에이전트를 위한 오케스트레이션(통합 관리 및 자동화) 인프라로, 다양한 에이전트 세션을 동시에 관리 가능하게 함
- 실제 인프라는 사용자의 VPS(가상 프라이빗 서버)에 자체 호스팅되어 원격에서 작동함
- 각 이슈마다 별도의 격리된 환경(워크스페이스)이 생성되어, 여러 작업을 병렬로 처리 가능함
- 운영 과정에서 이슈에 태그만 달면, 클라우드상의 에이전트가 환경을 띄우고 자동으로 GitHub 이슈에 작업 링크 및 상태를 댓글로 남김
- 모든 작업 완료 후 자동으로 Pull Request가 생성되고, 사용자는 최종 결과만 검토/머지하면 됨
- Workspaces 기반 구조 덕분에 대규모 병렬 처리와 팀 단위 협업도 손쉽게 실현 가능함
- 영상 제작자는 이 도구를 "무료 체험 가능하며 매우 추천할 만하다"고 소개함

## 세부 요약 - 주제별 정리

### 코딩 에이전트가 로컬 컴퓨터를 완전히 대체해 GitHub 이슈를 자동 처리함

- 영상 시작에서, 사용자는 실제로 "한 줄의 코드도 작성하지 않았다"고 밝힘
- 코딩 에이전트가 자신만의 컴퓨터(=원격 서버)에서 모든 작업을 수행함
- 사용자는 GitHub 이슈를 오픈하고 "coder" 태그만 붙이면 과정이 자동적으로 시작됨

### 오픈소스 무료 도구인 "Workspaces" 덕분에 전 과정이 원격/자동화됨

- 전체 흐름의 핵심은 "Workspaces"라는 무료 오픈소스 도구임을 강조
- 이 도구는 Coder 플랫폼에서 개발, 누구나 자유롭게 사용할 수 있음
- Workspaces가 존재함으로써, 별도의 인프라 구축 없이 리모트 작업이 가능해짐

### Workspaces는 코딩 에이전트의 대규모 병렬 오케스트레이션을 가능하게 함

- Workspaces 도입 시 여러 AI 코딩 에이전트 세션을 한 번에 관리할 수 있음
- 각각의 세션(작업)은 격리된 환경에서 병렬적으로 실행됨
- 실제 사용 케이스로서 "여러 GitHub 이슈를 동시에 처리"하는 예시 소개

### 사용자 인프라(VPS)에 Workspaces를 셀프 호스팅하여 완전한 원격 개발 환경을 구현함

- 영상에서는 사용자의 VPS에 직접 Workspaces를 설치하여 활용하는 모습을 보임
- 로컬 컴퓨터에는 아무것도 설치할 필요가 없음
- 모든 코딩 작업과 환경 조성, 에이전트 실행이 VPS에서 진행됨

### GitHub 이슈에 "coder" 태그만 추가하면 전자동으로 작업이 시작됨

- GitHub에서 해당 이슈를 열고 "coder" 태그만 붙이면 트리거가 작동
- 사용자는 별도의 명령어나 코딩 없이 간단한 태깅으로 모든 프로세스를 운영함
- 자동화된 태스크 등록 과정을 실제 화면으로 보여줌

### 태그 추가 후 Workspaces UI에서 자동으로 신규 작업(Task)이 등록되고 실행됨

- Workspaces 관리 UI(대시보드)에서 새 태그에 의해 생성된 태스크가 실시간으로 표시됨
- 각 Task에 대해 진행 상황, 초기화 상태 등 시각적으로 확인 가능
- 환경 초기화 및 클라우드에서 코드 실행이 단계별로 모두 자동 처리됨

### AI 에이전트가 GitHub 이슈에 상태 업데이트 및 작업 추적 링크를 코멘트로 남김

- 원격 환경에서 작업이 시작되면 AI가 이슈에 직접 실시간 코멘트를 남김
- 코멘트에는 실제 작업 추적 링크가 첨부되어 있음
- 사용자는 이 링크 통해 진행 상황을 실시간으로 모니터링할 수 있음

### 클라우드 코드가 환경 생성과 코드 구현을 처리하고 자동으로 새 풀 리퀘스트를 만듦

- 환경(컨테이너) 생성 → 코드 생성 및 테스트 → 자동 Pull Request 생성의 순서로 작업이 진행됨
- 사용자는 최종적으로 Pull Request 화면으로 이동해 결과만 검토 및 Merge하면 됨
- 모든 코딩과정은 실제로 AI와 원격 환경에서만 이뤄짐

### 동시에 여러 개의 GitHub 이슈 해결이 가능해 대규모 프로젝트 및 팀 협업에 적합함

- 각 이슈는 별도의 격리된 워크스페이스에서 실행되어 서로 간섭을 받지 않음
- 병렬 처리 덕분에 '수십 개의 이슈'도 동시에 처리 가능함을 강조
- 팀 전체가 인프라를 공유하여 효율적으로 활용 가능

### 무료 및 오픈소스로 누구나 시작할 수 있으며 도구 링크도 영상 설명에 포함됨

- Workspaces는 비용 부담 없이 체험 및 상시 사용 가능
- 실무적인 자동화 개발에 관심 있는 사람에게 적극적으로 추천
- 영상 하단에 공식 다운로드 및 설명 링크가 첨부됨
