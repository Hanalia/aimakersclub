## *Claude에게 24시간 내내 코딩을 시킨 결과* 핵심 요약

- 영상은 Claude Opus 4.5에 24시간 동안 쉬지 않고 코딩을 시키며 프로젝트를 진행한 후 결과를 공유함
- 현재까지 구축된 기능에 대해 제작자는 “매우 인상적”이라고 평가함
- claw.ai와 거의 동일한 기능을 가진 완벽에 가까운 클론이 생성된 것으로 느낄 정도로 진척도가 높았음
- 과거 모든 대화 기록을 살펴볼 수 있고, 각 루프마다 검증 과정을 거친 내역을 확인 가능함
- 아웃풋은 읽기 쉽고 보기 좋은 마크다운 포맷으로 잘 정리되어 있음
- HTML 페이지를 다양하게 생성하고, 코드 작성 및 코드 실행 기능도 포함됨
- 테마, 기본 모델 선택, 토큰 최대값 조절 슬라이더 등 세부 설정도 지원함
- 아직 human-in-the-loop(중간에 사람의 검증/개입)이 필요하지만, 전반적으로 매우 강력하고 풍부한 기능들이 구현됨
- 사람이 아무 작업도 하지 않고 Claude에게 모든 걸 맡긴 채 여기까지 만들 수 있었던 점에 놀라움을 표함
- 기능 전체를 며칠 더 돌리면 거의 완벽한 애플리케이션이 될 수 있을 것이라 언급하며, 반복 검증과 피드백 가능한 ‘하니스(harness)’의 강점을 강조함

## 세부 요약 - 주제별 정리

### Claude가 24시간 동안 코딩해 낸 결과물에 제작자는 완성도와 자동화 수준에 큰 만족을 느낌

- 24시간 내내 Claude Opus 4.5가 코딩에 집중하도록 한 후, 제작자가 직접 개입하지 않은 결과물을 평가함
- “지금까지 구축된 모든 것이 꽤 인상적”이라고 밝힘
- 현재 점검 중인 테스트의 후반부가 어떤 역할인지조차 궁금해질 정도로, 이미 기본 기능이 모두 갖춰진 느낌을 받음
- claw.ai와 흡사하거나 거의 똑같은 클론 애플리케이션이 만들어졌음을 직접 언급함

### 자동 검증 루프와 대화 내역 기록 시스템이 투명한 개발 과정을 보장함

- Claude가 각 기능을 구현할 때마다 자동으로 검증(verification) 루프를 수행함
- 각 루프에서 생성된 대화와 검증 과정이 모두 저장, 사용자는 이를 나중에 열람할 수 있음
- 이 과정을 통해 버그나 구현상의 실수가 쉽게 추적 및 수정 가능함

### 마크다운 포맷 출력으로 코드 및 결과물 가독성이 크게 향상됨

- Claude의 출력물은 읽기 쉽고 정돈된 마크다운(markdown) 형식으로 정렬됨
- 복잡한 내용도 손쉽게 파악할 수 있어 결과물 공유와 후속 조치에 효과적임

### 다양한 HTML 페이지 생성, 코드 작성/실행 등 광범위한 기능 지원이 구현됨

- 애플리케이션 내에서 여러 타입의 HTML 페이지를 자동으로 생성 가능함
- 단순 생성뿐 아니라, 코드 작성과 실행까지 Claude가 자동화하여 지원함
- 이를 통해 입출력, 인터랙티브 기능 테스트 등 다양한 확장이 가능함

### 상세한 사용자 설정 기능(테마/모델/토큰 슬라이더)도 문제없이 작동함

- 설정 창에서 테마(다크모드 등)를 변경할 수 있는 기능이 구현됨
- 기본 탑재 모델을 사용자가 직접 선택할 수 있도록 지원
- 슬라이더 UI로 max tokens(최대 토큰 갯수)를 자유롭게 조절 가능함

### human-in-the-loop 기능 추가가 남았으나, 이미 거의 완벽에 가까운 자동화 결과물이 나옴

- 앞으로 개발자가 직접 검증 및 개입할 수 있는 human-in-the-loop 시스템 추가를 계획 중임
- 그럼에도 불구하고, 기존에 구현된 자동화 시스템만으로도 “정말 멋진” 결과물이 나왔다고 재차 강조함

### 사람이 아무것도 하지 않고도 기능 많은 앱이 만들어지는 점에 제작자는 놀라움을 표현함

- 제작자는 “아무것도 손대지 않고 여기까지 만들었다”고 언급하며, 완전히 자동화된 개발 방식에 감탄함
- 추가적으로 하루 이틀 더 Claude를 돌리면 전 기능이 완벽히 구현될 수 있다고 내다봄

### Claude Opus 4.5와 검증 반복 구조(harness)가 애플리케이션 품질에 큰 영향을 미침

- Claude Opus 4.5 모델의 고도화된 자동생성 및 검증-반복(Validation & Iteration) 구조가 핵심 역할을 함
- 하니스(harness; 반복 및 검증 자동화 시스템)를 통해 기능 구현 후 반복적으로 체크, 오류나 미비점 자동 개선
- 이 구조가 없었더라면 수동 테스트 및 에러 수정이 훨씬 더 많이 필요했을 것임

### 반복 검증이 가능한 시스템 덕분에 “완벽한 애플리케이션” 실현이 가까워졌음을 시사함

- Claude가 다양한 검증 루프를 스스로 거쳐가며 기능 구현의 완성도를 높였음
- 모든 기능을 충분히 반복적으로 점검했다면, 거의 완벽에 가까운 애플리케이션이 될 수 있음을 강조함
