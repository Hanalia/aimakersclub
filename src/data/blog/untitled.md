## *Claude Code로 아름다운 다이어그램을 만드는 전체 워크플로우* 핵심 요약

- Claude Code와 같은 코딩 에이전트는 시각적(비주얼) 작업에 취약하며, 적절한 프롬프트와 능력 부여 없이는 시각적 설명에 한계가 있음
- 이에 대한 해결책으로, 개인 전 과정(workflow)을 '스킬(skill)'로 패키징하여 코딩 에이전트가 실용적이고 아름다운 다이어그램을 만드는 데 도움을 주도록 구성함
- 이 스킬은 Claude Code 등 코딩 에이전트에게 시각적으로 논증하는(visual argument) 법을 알려줌
- Excal 다이어그램(Excalidraw 형식)을 JSON 파일로 자동 생성하고, 시각적으로 다이어그램의 품질까지 검증함
- 생성된 다이어그램을 렌더링 후 PNG로 스크린샷을 찍고, 이를 비교·분석해 불완전함이 있으면 자동으로 반복 수정(iteration)함
- 코딩 에이전트는 Excal 다이어그램 스킬을 셋업(설치)할 때 readme 파일을 참조해 기본 설정 및 매뉴얼 단계를 자동 수행함
- 사용자는 간단한 요청만으로 자신만의 프랜차이즈 색상이나 브랜드 스타일에 맞는 다이어그램을 만들 수 있음
- 핵심 프롬프트 설계는 '블록 나열'을 넘어서, 구조와 라벨이 전체 개념을 시각적으로 명확히 설명하는 방법을 중점적으로 다룸
- Excal 다이어그램 스킬은 PDF, 코드베이스 등 다양한 입력에서도 높은 품질의 다이어그램 초안을 빠르게 생성, 매주 수 시간의 작업 시간을 단축시킴
- 리포지터리(코드 저장소)를 클론하고 스킬을 적용하면 누구나 간편하게 확장·활용 가능

## 세부 요약 - 주제별 정리

### 코딩 에이전트는 원래 시각적 작업에 약점이 있지만 적절한 스킬 부여로 극복할 수 있음

- Claude Code 등 코딩 에이전트는 텍스트 기반은 뛰어나지만 시각 자료(다이어그램, 설명 그림) 생성에는 취약점을 보임
- 사용자가 별도 프롬프트나 능력을 주지 않을 경우, 시각적 설명에서 만족스러운 결과를 내기 어렵다
- 반복적인 프롬프트 입력이나 복잡한 작업 지시 없이는 시각적 논증(visual argumentation)이 미흡함
- 영상 제작자는 이 약점을 보완하는 커스텀 워크플로우를 직접 설계하여 ‘스킬(skill)’ 형태로 패키징함

### 개인 워크플로우를 스킬로 패키지하여 복잡한 다이어그램 생성 과정을 자동화함

- 자신의 전 과정을 하나의 워크플로우로 정리해 Claude Code에서 항상 불러와 사용할 수 있도록 함
- 이 스킬은 특정한 명령어와 자료 형식(JSON 등)으로 다이어그램 설계에 최적화 되어 있음
- 사용자는 별도 번거로운 프롬프트 없이도 직관적인 요청만으로 원하는 다이어그램 형성을 주문할 수 있음

### 코딩 에이전트에 ‘시각적으로 논증하는’ 능력을 부여하기 위해 특별한 인스트럭션 세트를 적용함

- 인스트럭션 세트는 Claude Code가 ‘시각 논증’을 학습하는 데 핵심 역할을 함
- 이는 단순히 다이어그램 요소(블록, 노드)를 늘어놓는 수준이 아니라, 구조와 라벨로 개념 자체를 명확하게 설명할 수 있도록 설계되었음
- 기존의 ‘블록 나열식’ 자동 생성물은 시각적 설득력과 정보 전달력이 떨어짐
- ‘시각적으로 논증한다’는 프롬프트 구조를 통해 더 쓸모 있고 설득력 높은 결과물을 만듦

### Excal 다이어그램 자동 생성과 시각적 검증 프로세스를 하나의 워크플로우로 결합함

- 작성된 다이어그램은 Excal 형식의 JSON 파일로 우선 생성됨
- 그 후, 실제 다이어그램(Excalidraw 이미지)을 자동 렌더링하여 PNG로 스크린샷을 생성함
- 생성된 PNG 이미지를 Claude Code가 직접 확인 및 비교함
- 다이어그램 내 불완전한 부분이나 미흡한 점이 있을 경우, 이를 감지하고 자동으로 수정을 반복함 (iterative workflow)
- 이러한 ‘셀프 밸리데이션(self-validation)’ 프로세스 덕분에 매우 높은 완성도의 다이어그램 산출이 가능해짐

### Excal 다이어그램 스킬 설치 및 세팅도 코딩 에이전트가 대부분 자동으로 처리함

- 사용자는 코딩 에이전트에게 Excal 다이어그램 스킬을 “설정해 달라”고 요청만 하면 됨
- Claude Code 등 코딩 에이전트는 스킬 내 readme 문서를 읽고, 필요한 매뉴얼 단계를 알아서 진행함
- 사용자는 복잡한 설치, 환경 구성, 설치 옵션 선택 등의 번거로움 없이 바로 다이어그램 작성에 들어갈 수 있음

### 원하는 색상·브랜드 스타일 적용도 간편하게 지원함

- 사용자는 다이어그램 생성 시 자신이 선호하는 색상 팔레트, 브랜드 스타일, ‘온 브랜드(on-brand)’ 디자인을 지정할 수 있음
- 스킬은 이런 요구 사항을 반영해 자동으로 스타일을 적용하여 일관성 있는 결과물을 산출함
- 브랜드 특화 다이어그램이 필요할 때, 복잡한 수작업 없이 빠르게 처리 가능

### PDF, 코드베이스 등 다양한 입력 데이터에 대해 폭넓게 활용 가능함

- Excal 다이어그램 스킬은 단순 설명뿐 아니라 PDF 문서, 코드베이스 등 다양한 데이터 소스에도 적용 가능
- 복잡한 시스템 구조, 데이터 흐름, 코드 연관도 등을 명확하게 시각화할 수 있음
- 예시로 영상에서는 코드베이스 또는 일반 PDF 파일을 입력하여 다이어그램을 자동으로 생성할 수 있다고 언급
- 시각적 초안을 매우 신속하게 생성해 복잡한 데이터도 빠르게 이해 및 소통 가능

### 반복적인 셀프 밸리데이션 덕분에 ‘시간 단축’과 ‘품질 확보’ 모두 가능함

- 스킬의 핵심인 자동 렌더 및 이미지 검증 루프 덕분에 오류 수정이 수작업보다 훨씬 빠르게 진행됨
- 기존 수동 작업 대비 매주 수 시간씩 작업 효율을 향상시킴
- 반복적 수정 및 최적화가 자동으로 이뤄지기 때문에 초안 품질 역시 크게 향상됨

### 누구나 리포지터리를 클론하여 즉시 동일한 프로세스를 경험할 수 있음

- 영상에서 소개된 Excal 다이어그램 스킬은 깃허브(또는 연동 저장소)에서 클론(복제)하여 쉽게 가져올 수 있음
- 클론 후 간단한 설정만으로 현재 워크플로우 전체를 사용할 수 있음을 강조
- 즉시 PDF, 코드 등 다양한 작업에서 활용 가능하며, 사용자에 따라 새로운 실험과 확장도 가능
