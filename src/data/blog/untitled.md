## *Postman이 Git-native로 진화하며 API 구축 방식에 혁신을 가져옴* 핵심 요약

- API 작업 방식은 기존에 매우 복잡하고 비효율적이었으나, 특히 AI 에이전트 등 최신 개발에서는 API 엔드포인트 작업이 필수적임
- 일반적으로 API 엔드포인트 생성, 테스트 코드 작성, 컬렉션 관리 등이 별도의 도구 및 환경에서 분리되어 있어, 워크플로우와 버전 관리가 불편했음
- Postman은 이러한 문제점들을 최근 대규모 업데이트를 통해 해결함
- Postman의 주요 업데이트 핵심은 API 리소스(컬렉션, 스펙, 목, 테스트 등)을 로컬 코드베이스에 YAML 파일 형태로 저장·관리할 수 있게 된 점임
- 로컬에서 작업한 리소스는 Postman 데스크톱 앱이나 클라우드와 양방향 동기화 가능
- 코드, 테스트, 스펙 수정 시 git diff 등 버전 관리 도구와 완전하게 통합되어 각 변경 사항을 명확하게 추적할 수 있음
- Postman의 에이전트 모드는 평문 영어로 API 요구사항을 설명하기만 해도, 코드베이스 분석 후 라우트 발견, 스펙‧테스트‧목 생성, 동기화까지 자동 실행
- AI 모델(Claude, GPT, 자체 호스팅 등)과 연동되며, GitHub, Sentry, Linear 등과의 컨텍스트 연동이 지원됨
- 사전에 작성한 컬렉션과 테스트는 환경에 관계없이(로컬/CI파이프라인) 동일하게 작동하며, pre-commit hook 및 GitHub Actions와도 연동됨
- 영상 제작자는 본인이 직접 Postman의 변화를 경험하고 추천하며, 실제 예시 코드가 포함된 GitHub 리포지토리를 설명란에 제공함

## 세부 요약 - 주제별 정리

### 기존 API 개발의 워크플로우는 단절되고 비효율적임을 지적함

- API 작업은 기존에 개발자가 IDE나 코딩 에이전트로 엔드포인트를 만들고 코드 작성
- 테스트, 컬렉션, 스펙 등은 별도의 툴에서 따로 관리되어 서로 연결이 어려움
- 이로 인해 코드, 테스트, 스펙 등이 분산 저장되고, 서로 다른 시스템에서 관리됨
- 버전 관리는 각각 따로 이루어져 변경 사항 추적이 힘들고, 표준화된 워크플로우가 없음
- 특히 AI 에이전트 구축 등에서 많은 API 엔드포인트 처리와 테스트 관리가 필수이나, 효율적인 통합 방법 부족

### Postman의 최신 업데이트가 모든 문제를 한꺼번에 해결함

- 최근 Postman의 대대적인 업데이트 내용 소개
- API 플랫폼으로서 ‘AI 네이티브’와 함께 ‘Git-native’ 특성이 강조됨
- 영상 제작자(콜 메딘)가 거의 10년 간 Postman을 써왔으나 최근 업데이트를 극찬
- 모든 API 리소스(컬렉션, 스펙, 모크, 테스트)들을 코드베이스 내 YAML형태 파일로 저장
- YAML 파일 기반이므로 관리와 유지보수가 용이해짐
- Postman 데스크톱 앱에서도 직접 파일을 열고 편집할 수 있음

### API 리소스를 로컬 코드베이스에 통합·관리할 수 있게 됨

- 모든 컬렉션, 엔드포인트, 각종 API 리소스가 개발자 로컬 환경(코드베이스)에 저장
- Postman 데스크톱 앱에서 컬렉션 및 엔드포인트 확인 가능
- 클라우드 기능과 연동되어 Postman 클라우드에서 동일 자원 확인 및 양방향 동기화 제공
- 개발자가 테스트 등 일부 리소스에 변경사항을 만들고, 바로 에디터로 이동하여 변경 diff(차이점) 확인
- git diff, 커밋 등의 버전관리 시스템과 자연스럽게 통합됨

### 버전 관리와 변경 추적이 획기적으로 개선됨

- 모든 테스트, 스펙, 컬렉션 변경 내역이 git diff를 통해 명확히 확인 가능
- 코드 변경과 API 자원 파일이 같은 버전관리 시스템 안에서 관리됨
- 이전에는 API 스펙과 테스트 변경의 추적이 분산되었으나, 이제는 한눈에 볼 수 있음
- 리뷰 가능한 git 커밋으로 모든 변경사항이 남음

### 에이전트 모드를 통한 전체 API 라이프사이클 자동화가 가능해짐

- Postman의 에이전트 모드 활성화로, 단순히 영어로 API 요구사항 기술 가능 (“Flask 앱의 컬렉션 생성” 등)
- Postman이 코드베이스를 스캔하여 라우트 직접 발견
- 자동으로 스펙 생성, 테스트 작성, 목(mock) 설정까지 일괄적으로 수행
- 모든 과정이 동기화되고, 각 자원이 파일 형태로 관리되어 추적 및 분산 협업이 편리함

### AI 모델 및 서드파티 툴 연동을 통해 확장성이 대폭 증가함

- AI 모델 선택 가능: Claude, GPT, 자체 호스팅 모델 등 선택적 긴밀 연동
- GitHub, Sentry, Linear 등 개발 생산성 도구에서 맥락 정보를 자동으로 참조
- 기존에 사용하던 툴들의 데이터, 이슈 트래킹, 코드 컨텍스트 등을 Postman 안에 연계 활용 가능

### 환경과 무관하게 CI/CD 과정에서 동일한 테스트/컬렉션을 실행할 수 있게 됨

- 로컬에서 작성한 컬렉션 및 테스트가 CI 파이프라인(예: GitHub Actions, pre-commit hook 등)에서 별도 변경 없이 바로 실행
- 개발, 테스트, 배포 환경이 달라도 동일한 스펙/테스트 코드 활용 가능
- 추가적인 변환 과정이나 작성 변경이 불필요
- 일관된 환경 간 테스트, QA, 배포 프로세스가 가능해짐

### 제작자는 실 사용 기반으로 Postman의 진화된 기능을 강력히 추천함

- 약 10년 간 Postman을 사용해온 경험자 입장에서 변화의 혁신성을 직접적으로 강조
- GitHub 저장소에 실제 사용 예시 및 샘플 코드 제공
- Postman 공식 사이트 및 샘플 저장소 링크 정보도 영상 설명란에 명시
- 혁신적인 업데이트로 인해 Postman이 한층 강력하고 실전적인 API 플랫폼이 되었음을 강조
