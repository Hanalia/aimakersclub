## *나는 Claude에게 24시간 내내 코딩을 시켜봤더니 이런 일이 벌어졌다* 핵심 요약

- 영상 제목은 "나는 Claude에게 24시간 내내 코딩을 시켜봤더니 이런 일이 벌어졌다(I Forced Claude to Code for 24 Hours NONSTOP, Here's What Happened)"임
- 제작자는 Claude Opus 4.5 모델을 활용해, 클로(Claw.ai)의 완벽한 복제판(clone)을 거의 완성함
- 테스트의 절반 즈음부터는 이미 기능적으로 완성된 어플리케이션 느낌을 받았다고 언급함
- 진행 중 생성된 모든 과거 대화 기록을, 각 루프(코드 검증 과정)마다 확인 가능함을 강조
- 읽기 좋은 마크다운(markdown) 포매팅, 다양한 HTML 페이지 생성, 코드 작성 및 실행 등 풍부한 기능을 구현
- 사용자 설정에서 테마(theme) 변경, 기본 모델 선택, 최대 토큰 수 조정 슬라이더 등 상세 옵션 제공
- "휴먼 인 더 루프(human in the loop)" 기능 추가는 향후 과제로 남아 있으나, '손 하나 까딱하지 않고' 이 정도를 만들어낸 것을 인상적으로 평가
- 하루만 더 돌리면 거의 완벽한 애플리케이션이 될 수 있을 정도로 Claude의 발전 수준이 "미쳤다"고 표현
- 이런 자동화 툴을 활용한 검증 및 반복(iteration) 과정이, 애플리케이션 품질과 개발 생산성을 극적으로 높일 수 있다는 점을 직접 경험함

## 세부 요약 - 주제별 정리

### Claude Opus 4.5를 이용해 클로.ai 완전 복제품을 거의 자동 생성함

- Claude Opus 4.5로 클로.ai(Claw.ai)의 기능을 대부분 자동으로 복제하는 데 성공함
- 코딩 작업이 거의 자동 진행되어, 개발자가 직접 손을 대지 않아도 클론 코드가 작성됨
- 반복 테스트를 거치며 애플리케이션의 완성도가 올라감
- 화자는 테스트의 절반이 지난 시점에서 이미 "완전히 작동하는 클론"이 만들어졌다고 느낌
- 이런 자동화 덕분에 개발 효율성과 품질 모두에서 인상적인 성과가 나왔음을 언급

### 자동 생성된 애플리케이션에서 다양한 대화 내역 및 작업 상태를 손쉽게 추적할 수 있음

- Claude가 생성한 모든 과거 대화를 루프별로 손쉽게 확인 가능
- 각 반복 검증 과정에서 어떤 결과와 대화가 오갔는지 이력 추적이 용이함
- 개발 및 디버깅 혹은 기능 검증의 투명성이 크게 높아진 결과임
- 반복 결과와 대화 내용의 기록이 남아, 코드 품질 관리에도 도움됨
- 이처럼 각 프로세스를 잘 드러내는 인터페이스는 자동생성된 애플리케이션의 강점이라고 소개

### 읽기 쉬운 마크다운, HTML 페이지 생성, 코드 실행 등 풍부한 개발 도구와 기능들이 구현됨

- 애플리케이션(클론) 내에서 마크다운(markdown)으로 결과물을 효과적으로 포맷팅하여 가독성 확보
- HTML 페이지를 여러 형태로 생성할 수 있는 기능이 구현됨
- 코드 작성 및 실행 기능도 자동 세팅되어 다양한 테스트 및 실험이 가능함
- 복잡한 개발 도구나 편집기 없이도 Claude가 자체적으로 결과물을 생성·실행하는 구조
- 풍부한 개발 지원 기능 덕분에 생산성과 사용자 경험 모두 뛰어남

### 사용자 설정에서 테마/모델/토큰 등 세부 옵션을 조정할 수 있음

- 애플리케이션에는 각종 설정 화면이 구현되어 있음
- 테마(theme) 변경을 지원, 사용자 인터페이스의 개인화 가능
- 특정 "기본 모델"을 선택할 수 있어 여러 모델 대응성 확보
- 최대 토큰 수를 설정할 수 있는 슬라이더(slider) 기능이 내장됨
- 이러한 유연한 세팅 옵션은 실제 주요 서비스 플랫폼과 유사한 품질을 제공함

### 인간의 개입(Human in the loop)은 아직 추가되지 않았으나, 자동 생성 수준에 경탄함

- 화자는 "human in the loop" 등 휴먼 피드백 기능은 아직 추가하지 못했다고 언급
- 그럼에도 손 하나 까딱하지 않고 이 선택지들(기능들)이 구현된 결과에 스스로 놀람
- 추가 작업을 통해 인간 검수, 피드백, 개입 등 기능도 적용할 계획임을 밝힘
- Claude의 자동화 능력이 이미 실사용 수준에 도달했다고 평가

### 하루만 더 자동 코딩을 이어간다면, 완전히 완벽한 어플리케이션이 될 것으로 기대함

- 현재 상태도 거의 완벽에 근접한 애플리케이션이지만, 테스트를 모두 마치지 않은 상태임
- 추가 하루(24시간) 자동 반복 작업으로 남은 기능들도 충실히 개발할 수 있다고 전망
- Claude Opus 4.5의 기능적 잠재력에 대해 "정말 미친 수준"이라고 직접적으로 표현
- AI가 스스로 기능 검증과 반복을 진행하는 "harness"가 개발 자동화에 크게 기여함

### 자동화와 반복 검증 프레임워크(harness)가 어플리케이션 품질과 생산성을 극적으로 끌어올림을 실감함

- Claude와 같은 대형언어모델(Large Language Model)에 "harness(자동 반복 검증 프레임워크)"를 결합
- 애플리케이션 내 각 기능을 자동 검증하고, 결과를 기반으로 즉시 개선·반복하는 구조
- 이때문에 코드 완성도와 서비스 품질이 놀랍게 향상
- 검증-반복 루프가 개발 주기 전체를 자동화하여, 사람의 수동 개입 없이 빠른 발전이 가능
- AI 활용 개발 방식을 체감하고, 전통적 코딩 방식과의 현격한 차이를 경험함
