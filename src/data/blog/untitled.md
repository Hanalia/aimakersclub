## *RAG 2.0: 에이전틱 RAG와 지식 그래프 통합 (무료 템플릿 공개)* 핵심 요약

- 영상 제목은 "*RAG 2.0: 에이전틱 RAG와 지식 그래프 통합 (무료 템플릿 공개)*"로, 차세대 RAG 전략의 결합과 무료 제공 템플릿을 소개함
- 제작자는 최근 몇 달간 다양한 RAG(검색 증강 생성) 전략을 탐구한 결과, "Agentic RAG"와 "Knowledge Graphs" 두 가지에 높은 효율성과 활용성을 느낌
- 영상에서는 위 두 전략이 결합된 강력한 AI 에이전트를 직접 제작, 시연함
- 이 에이전트는 사용자의 질문 내용에 따라 정보를 어디에서 찾아야 할지 스스로 판단하여 최적의 답을 도출
- 데이터 저장에 "Postgres + PG vector 확장"을 사용, 네온(Neon)이라는 서버리스 Postgres 플랫폼 위에서 벡터 데이터베이스 구축
- 문서를 벡터 임베딩 방식과 지식 그래프(Graffiti, Neo4j 활용) 양쪽 모두에 저장함 — 동일한 정보지만 서로 다른 질의 방식 지원
- 벡터 DB는 단일 회사의 AI 전략 등 간단 사실 질의에, 그래프 DB는 여러 기업 간의 관계 질의에 각각 활용됨
- 시스템 프롬프트 조정이나 직접 명령을 통해, 두 데이터 소스를 동시에 활용한 풍부한 답변 생성 가능
- 에이전트는 답변과 더불어 어떤 툴(데이터베이스, 그래프)을 사용했는지 투명하게 안내함
- 영상 마지막에는 에이전트 제작법과 활용법, 클라우드 코드 사용법 등을 다룬 본편 영상 링크 및 무료 템플릿을 안내함

## 세부 요약 - 주제별 정리

### 제작자는 다양한 RAG 전략 중 에이전틱 RAG와 지식 그래프에 특별한 효과를 경험함

- 최근 몇 달간 제작자는 거의 모든 주요 RAG(검색 증강 생성) 전략들을 심층적으로 실험 및 분석함
- 그 과정에서 "에이전틱 RAG(Agentic Rag)" 전략과 "지식 그래프 활용(Knowledge Graphs)" 전략에 반복적으로 주목하게 됨
- 이 두 가지는 각각 독립적으로 높은 활용도와 효율성을 보였음
- 최종적으로는 이 두 전략을 통합하는 방안을 고민하게 됨

### 두 전략을 결합한 강력한 AI 에이전트가 직접 개발됨

- 제작자는 위에서 선별한 두 가지 RAG 전략을 결합해 AI 에이전트를 구현함
- 이 에이전트는 "어떤 질문에 대해서 정보를 어디서 찾을지" 스스로 사고하는 기능(Reasoning)을 가짐
- 질문 유형에 따라 벡터 DB, 지식 그래프 중 최적의 정보를 선택적으로 검색
- 결과적으로 더 정확하고 관련성 높은 답변 생성이 가능해짐

### 벡터 데이터베이스는 Postgres + PG vector를 네온 플랫폼 위에서 구현함

- 데이터는 "Postgres" 데이터베이스에 저장되며, "PG vector" 확장을 통해 벡터 검색이 가능하게 함
- 서버리스 방식의 "Neon" 플랫폼을 사용하여 데이터베이스를 운영함
- 벡터 DB에는 대형 IT 기업과 이들의 AI 전략, 주요 제휴 정보 등 관련 문서를 저장함
- 문서는 자연어 임베딩 과정을 통해 벡터화되어, 빠르고 관련성 높은 질의가 가능

### 동일한 정보를 Graffiti와 Neo4j로 지식 그래프에 중복 저장함

- 문서상의 동일한 정보 세트를 지식 그래프 형태로 다시 저장함
- "Graffiti"와 "Neo4j"를 활용해 관계형 데이터베이스를 구축함
- 이 구조를 통해 기업 간 관계, 예를 들어 "Amazon은 Enthropic의 AI 모델 호스팅을 제공한다"와 같은 정보가 명확히 표현됨
- 지식 그래프 저장 덕분에 복잡한 관계 질의 및 추론도 지원함

### 에이전트는 질문 유형별로 최적의 데이터 소스를 선택해 검색함

- 예시1) "구글의 AI 전략은 무엇인가?"와 같이 단일 회사에 대한 단편적 정보는 벡터 DB를 이용
    - 문장 검색 후 즉시 관련 정보와 함께 답변 제공
- 예시2) "두 기업 간의 파트너십 관계"와 같이 복합적 관계를 묻는 질문에는 지식 그래프를 검색하도록 설계
    - 에이전트는 시스템 프롬프트 및 질문 분석을 토대로 도구를 선택함

### 사용된 도구 정보도 실시간으로 정보를 제공하여 투명성을 높임

- 사용자가 질문하면, 에이전트는 각각 어떤 도구(벡터 검색/그래프 검색)를 사용했는지 답변과 함께 명시함
- 각 질의 처리 후 "Tools Used" 항목으로 명확히 안내
- 이렇게 함으로써 사용자는 처리 과정을 쉽게 파악할 수 있음

### 두 데이터 소스를 동시에 활용해 더 풍부한 답을 얻는 것도 가능함

- 시스템 프롬프트를 별도로 조정하거나, "벡터 DB와 그래프 DB 모두 활용" 명시적 지시가 가능함
- 이때 에이전트는 두 데이터 소스에 각각 질의해, 종합적으로 신뢰도 높은 답변을 반환함
- 간단 명령어 조작만으로 사용자가 원하는 복합적 질의도 지원

### AI 에이전트 개발법과 클라우드 코드 활용이 본편 영상에서 상세히 안내됨

- 영상 말미에는 "무료 RAG 2.0 에이전트 템플릿"을 누구나 사용할 수 있음을 안내
- 본편 전체 영상에서는, 제작자가 에이전트를 직접 만드는 방법을 단계별로 시연함
- "클라우드 코드"를 통한 개발 효율화와 실질적 구현 노하우 제공
- 시청자는 링크를 통해 템플릿 및 구체적 활용법을 모두 확인 가능
