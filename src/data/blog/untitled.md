## *Claude Code로 아름다운 다이어그램을 만드는 전체 워크플로우* 핵심 요약

- Claude Code와 같은 코딩 에이전트는 시각적 작업에 한계가 있어, 시각적으로 설명하거나 타인에게 설명을 돕기에는 적합하지 않음
- 효과적인 다이어그램 생성을 위해 별도의 스킬(workflow)을 직접 개발하여 Claude Code 등 코딩 에이전트에 적용
- 코딩 에이전트에게 시각적으로 ‘주장하는 방법(visual arguing)’을 학습시킬 수 있는 주요 명령어셋(instruction set)을 구성
- 이 스킬은 Excal(Excalidraw를 지칭) 다이어그램을 만들기 위한 JSON 파일을 자동 생성하도록 함
- 다이어그램이 완성되면 렌더링 후 PNG 스크린샷을 확인하고, 불완전한 부분은 반복적으로 수정(셀프 밸리데이션 루프)
- Excal 다이어그램 스킬로 인해 화려하고 실용적인 시각적 자료를 신속하게 생성 가능, 매주 수 시간 절약 효과
- 코딩 에이전트가 README 파일을 분석하여 Excalra(Excalidraw 연동) 다이어그램 스킬을 스스로 설정할 수 있음
- 색상이나 브랜드 스타일 등 본인에게 적합한 디자인을 생성하는 등의 맞춤화도 쉽게 가능
- 스킬에 포함된 프롬프트 구조는 단순 블록형 도식이 아니라 구조와 라벨로 개념 전체를 효과적으로 설명하게 만듦
- Excalra 다이어그램 스킬 레포지토리를 복제(clone)해 바로 PDF, 코드베이스 등 다양한 곳에 활용 가능

## 세부 요약 - 주제별 정리

### 코딩 에이전트는 시각적 설명에 한계가 있으며 프롬프트와 스킬로 보완이 가능함

- Claude Code와 같은 코딩 에이전트는 기본적으로 시각화에 적합하지 않음
- 단순 시각 자료 요청 시 미흡하거나 비효율적이며, 많은 프롬프트 조작과 세밀한 스킬 설정이 필요함
- 이런 문제점을 해결하기 위해 작업자가 자신의 전체 워크플로우를 별도의 스킬로 패키징함
- 이 스킬을 적용하면 Claude Code뿐 아니라 다른 코딩 에이전트에도 확장 활용 가능

### 직접 제작한 워크플로우를 통해 실용적이고 아름다운 다이어그램 생성을 실현함

- 사용자가 개발한 워크플로우는 ‘Excal 다이어그램’을 신속하고 효과적으로 생성할 수 있도록 설계됨
- 코딩 에이전트가 단순 명령 대신, 시각적 표현(다이어그램)으로 정보를 정교하게 전달하도록 유도
- 복잡한 논리나 개념을 시각적으로 설득력 있게 표현하는 데 초점을 맞춤

### 시각적으로 주장하는 능력을 부여한 주요 명령어셋이 스킬의 핵심 역할을 수행함

- “argue visually(시각적으로 주장하다)”라는 개념을 코딩 에이전트에 명확히 학습시킴
- 단순한 상자(블록) 형태의 도식이 아니라, 명확한 구조와 직관적 라벨링을 강조
- 이를 위해 skill.mmd 등 프롬프트 파일 내에 구체적인 지침을 포함
- 다이어그램에 정보의 계층, 관계성, 강조점이 잘 반영됨

### Excal 다이어그램 스킬은 JSON 기반으로 자동화된 도식화를 제공함

- Claude Code는 Excal(Excalidraw) 도식 작업용 JSON 파일을 자동 생성
- 사용자는 별도의 작업 없이 결과물을 쉽게 받을 수 있음
- JSON의 구조와 다이어그램의 내용이 논리적으로 일치함

### 자동 렌더링과 셀프 밸리데이션 기능으로 다이어그램 퀄리티가 비약적으로 상승함

- 워크플로우는 생성된 다이어그램을 바로 시각적으로 렌더링
- PNG 스크린샷을 직접 분석하여 도식의 미흡한 점을 자동 검토
- 이후 반복적으로 수정(Iterate)하여 품질을 높임
- 코딩 에이전트가 ‘스스로 자신의 결과를 검증(check their own work)’하는 루프가 구현됨

### Excalra 다이어그램 스킬은 손쉬운 셋업과 효율적 자동화를 제공함

- 코딩 에이전트에게 ‘Excalra 다이어그램 스킬 셋업을 도와줘’라고 요청하면 자동으로 README 파일을 읽고 분석함
- 이 과정을 통해 수동 설정 과정을 상당 부분 자동화함
- 사용자는 바로 다이어그램 생성 명령을 내릴 수 있으며, 워크플로우가 문제없이 작동함

### 맞춤화가 가능하여 사용자의 브랜드, 컬러 스킴 등에 적합하게 다이어그램을 생성함

- 가장 중요한 기능 중 하나로, 브랜드 정체성이나 개인적 취향에 맞는 색상, 스타일을 쉽게 적용 가능
- 구조뿐 아니라 시각적 스타일의 일관성도 챙길 수 있음
- 사용자는 간단히 원하는 컬러 스킴을 지시하면 코딩 에이전트가 이를 적용함

### 스킬 프롬프트 철학은 블록 배열이 아닌 구조와 라벨 중심 설명을 강조함

- 단순 박스 나열(블록) 위주의 다이어그램은 정보를 효과적으로 전달하지 못함
- 개념의 연관성, 논리적 흐름을 ‘구조+라벨’ 중심으로 명확하게 그리는 것이 지침의 핵심임
- 실제로 스킬 없이 생성할 경우 단순 블록 배열로 끝나지만, 프롬프트 활용 시 설득력 있고 이해하기 쉬운 도식이 만들어짐

### PDF, 코드베이스 등 다양한 소스에 다이어그램 워크플로우를 즉시 적용할 수 있음

- Excalra 다이어그램 스킬 레포지토리를 클론해 실제로 바로 사용할 수 있음
- PDF 문서나 소스 코드, 다양한 지식 구조화 상황에 적용 가능
- 시작점이 되는 시각 자료를 신속하게 생성해, 매주 여러 시간을 절약함

### 다이어그램 워크플로우의 적용으로 반복적 작업 시간을 획기적으로 단축할 수 있음

- 자동화, 검증, 맞춤화가 결합된 다이어그램 생성으로 생산성을 극대화함
- 매주 반복적인 다이어그램 제작에 투입되는 시간과 노력을 절약
- 영상의 데모와 설명을 통해 워크플로우 전반의 이점이 실제로 입증됨
