## *RAG 2.0: 에이전틱 RAG와 지식 그래프의 결합 (무료 템플릿 소개)* 핵심 요약

- 영상 제목은 'RAG 2.0: 에이전틱 RAG와 지식 그래프의 결합 (무료 템플릿 소개)'임
- 제작자는 수개월 간 다양한 RAG(Retrieval-Augmented Generation) 방식들을 심층적으로 실험
- 최종적으로 '에이전틱 RAG(Agentic RAG)'와 '지식 그래프(Knowledge Graph)'의 조합이 가장 효과적임을 강조
- 본인이 개발한 AI 에이전트는 질문 내용에 따라 가장 적합한 정보원(벡터DB/지식그래프)을 스스로 선택해 활용함
- 벡터 데이터베이스는 Postgres + PG vector 확장, 서버리스 Postgres 플랫폼인 Neon 사용
- 문서는 AI 및 빅테크 기업 정보, 이들의 AI 이니셔티브, 파트너십 등 다양하게 포함
- 동일한 문서를 지식 그래프(Graffiti와 Neo4j 활용)로도 저장하여, 기업 사이의 관계 질의도 처리 가능
- 시스템 프롬프트나 명령어로 벡터DB, 지식그래프, 또는 양쪽 모두 활용하도록 제어 가능
- 실제 질문 예시: 단일 기업 AI 이니셔티브 질의 시 벡터DB 조회, 기업 간 관계 질의 시 지식그래프 검색, 양쪽 동시 사용도 가능
- 무료로 제공되는 템플릿 링크와, Cloud Code 기반 개발 및 실전 사용 방법이 담긴 전체 영상 안내

## 세부 요약 - 주제별 정리

### 제작자는 다양한 RAG 전략을 시험한 결과 에이전틱 RAG와 지식 그래프 조합이 가장 효과적임을 발견함

- 수개월 동안 가능한 모든 RAG(Retrieval-Augmented Generation) 전략을 실험했다고 언급
- 여러 전략 중에서 반복적으로 뛰어난 효과를 보인 것은 'Agentic RAG'와 'Knowledge Graphs'임을 강조
- 두 방법의 결합이 개별적 방법보다 더 강력한 성능을 보임
- 영상 전반에서 이 조합의 실제 구현 과정을 소개함

### AI 에이전트는 스스로 정보원 선택 및 검색 경로를 결정할 수 있음

- 에이전트는 주어진 질문을 분석해, 최적의 정보원(벡터 데이터베이스, 지식 그래프 등)을 자율적으로 선택
- 정보 획득 경로를 '추론(reasoning)'하여, 어떤 데이터베이스가 정확한 답을 제공할 수 있을지 판단
- 웹사이트나 명시적 도구 호출 없이도, 시스템 프롬프트에 기반해 자동으로 탐색 경로를 취함
- 이런 방식이 사용자의 다양한 질의에 유연하게 대응하는 데 큰 도움을 줌

### 벡터 데이터베이스는 Postgres와 PG vector 확장 그리고 서버리스 Postgres 플랫폼 Neon으로 구성됨

- 본인 프로젝트의 벡터 데이터베이스는 Postgres를 기반으로 함
- 필요에 따라 PG vector 확장을 활용해 벡터 임베딩 검색이 가능하도록 만듦
- Neon이라는 서버리스(serverless) Postgres 플랫폼 위에서 운영함
- 서버리스 환경의 장점을 살려 빠르고 효율적인 데이터베이스 구성이 가능함

### 문서 정보는 빅테크 기업의 AI 이니셔티브와 상호 파트너십을 풍부하게 포함함

- 입력 문서에는 빅테크 기업들의 주요 정보와, 각각의 AI 이니셔티브에 관한 내용이 포함되어 있음
- 각 기업이 어떤 AI 프로젝트를 추진하는지, 어떤 기업 및 서비스와 파트너십을 맺었는지 등 실질적 데이터 수록
- 이러한 구조로 실제 비즈니스 관계와 시장 내 역학 구도가 데이터 상에 구현됨
- 예로, Amazon이 Anthropic의 AI 모델 호스팅을 담당한다는 사실을 문서 내에서 명시함

### 동일한 문서를 Graffiti와 Neo4j를 활용해 지식 그래프로도 저장함

- 벡터DB 외에, 같은 정보를 Graffiti 및 Neo4j 기반 지식 그래프에도 별도로 저장
- 지식 그래프 형태는 데이터 간 복잡한 관계성을 표현하는 데 강점이 있음
- 기업들 간의 관계성 질의, 네트워크 분석 등 응용이 쉬움
- 실시간 관계 탐색에 최적화된 데이터 구조를 제공

### AI 에이전트는 문장 내 질의 유형에 따라 벡터DB 또는 지식 그래프를 자동 선택하여 검색함

- 예시 질문1: "구글의 AI 이니셔티브는 무엇인가?" → 벡터 데이터베이스로 검색 수행
- 결과값과 더불어, 어느 툴(Vector DB, Graph Search 등)을 사용했는지 명확히 표시
- 예시 질문2: "아마존과 앤트로픽의 파트너십 구조는?" → 지식 그래프에서 관계성 검색
- 시스템 프롬프트에 따라 검색 경로가 제어됨을 확인할 수 있음

### 시스템 프롬프트나 명령문 수정으로 도구 활용 방식을 자유롭게 조정할 수 있음

- 사용자는 시스템 프롬프트 내에서 어느 도구를 쓸지 명확히 지정 가능
- 명령문을 통해 "벡터DB와 그래프DB 둘 다 검색하라"고 호출할 수도 있음
- 엔진이 동시에 두 데이터 소스에서 모두 정보를 취합해 통합된 답변을 제공함
- 이런 조정 가능성은 다양한 현장 요구와 실험에 최적화된 유연성을 부여함

### 실제 질의별 에이전트의 정보 검색 예시가 구체적으로 제공됨

- 그림 보여주기를 통해 실제 데모 과정을 제시
- AI 에이전트가 사용한 도구, 경로 및 답변 결과를 실시간으로 화면에 노출
- "이것 좀 봐라", "결과를 보자"와 같은 자연스러운 데모 형식 진행
- 각 검색 케이스마다 툴의 사용 및 데이터 흐름을 명확하게 안내함

### 무료로 활용 가능한 전체 템플릿 및 구현 방식을 추가 리소스로 안내함

- 영상 후반부에 해당 AI 에이전트의 무료 템플릿을 배포 중임을 알림
- 템플릿 다운로드 및 전체 코드 개요, Cloud Code를 활용한 개발 방법 링크 안내
- 직접 에이전트를 구현, 운영할 수 있도록 실전 가이드 영상도 별도로 제공
- 모든 리소스와 추가 정보는 영상 속 링크에서 확인 가능함
