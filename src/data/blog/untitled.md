## *AI 에이전트 능력을 극대화하는 가장 단순한 방법(스킬)* 핵심 요약

- Anthropic이 최근 공개한 "스킬(skills)" 기능은 최근 AI 발전 중 가장 중요한 혁신 중 하나임
- 이 기능의 가치는 "단순함"에 있으며, Anthropic의 철학인 "더 단순할수록 좋다"를 구현함
- AI 에이전트에게 다양한 기능을 더하고 싶으나, 정보 과부하는 피해야 함
- 기존 방법(MCP 서버 등)은 시스템 프롬프트, 도구, 대화 내역 등 모든 정보를 한 번에 에이전트에 주어 컨텍스트 윈도우를 쉽게 넘치게 함
- MCP 방식은 에이전트가 필요하지도 않은 도구를 사전에 다 받기 때문에 비효율적임
- 스킬의 혁신적 단순함은 "점진적 공개(progressive disclosure)" 방식에 있음
- 최초에는 오직 스킬의 "기술 요약(설명)"만이 시스템 프롬프트에 포함됨
- 실제로 사용자가 해당 기능(예: PDF 처리)을 요구하면, 에이전트가 skill.md와 추가 문서 및 계층화된 정보를 순차적으로 불러와 기능을 수행함
- skill.md는 해당 스킬의 전체 사용 지침이 포함된 핵심 파일로, 추가 맥락이 필요할 때 더 깊은 정보를 순차적으로 로드
- 이러한 다단계 구조로 에이전트는 필요한 순간에만 필요한 정보를 불러오며 과부하를 방지함

## 세부 요약 - 주제별 정리

### Anthropic의 "스킬"은 최근 AI 혁신 중 가장 단순하면서도 중요한 발전임

- Anthropic이 최근 출시한 "스킬(skills)"은 출시 이후 단기간 만에 AI 분야에서 큰 주목을 받음
- 단순함의 원칙을 바탕으로 제작되었으며, Anthropic 내부 모토 역시 "더 단순할수록 좋다"임
- 스킬 기능은 실제 AI 에이전트를 실무 환경에서 훨씬 강력하고 실용적으로 만들 수 있는 변화임

### 에이전트는 다양한 기능 확장을 필요로 하지만 컨텍스트 윈도우 과부하 문제에 직면함

- AI 에이전트에게 더 많은 능력을 추가하면 생산성은 향상되지만, 과도한 정보 입력은 컨텍스트 처리 한계를 초래함
- 기존에는 시스템 프롬프트, 도구 리스트, 대화 이력 등 모든 정보를 한 번에 에이전트에게 입력함
- 이런 접근은 에이전트가 맥락을 이해하지 못하거나 다운되는 현상을 잦게 유발함

### 기존 MCP 서버와 같은 방식은 비효율적으로 많은 도구를 미리 제공해 문제를 일으킴

- MCP 서버 등 기존 도구 통합 방식은 모든 가능성 있는 도구들을 미리 에이전트에 제공함
- 실제 대화에서 해당 도구가 필요하지 않아도 모든 정보가 시스템에 전달되며, 이는 리소스 낭비이자 딥러닝 모델 과부하 원인임
- 이런 방식은 에이전트가 꼭 필요로 하는 순간에 맞춰 정보를 제공하는 데 실패함

### 스킬의 핵심은 "점진적 공개" 방식으로 필요한 순간에만 정보를 로드하는 것임

- Anthropic 스킬 구조의 혁신성은 progressive disclosure(점진적 공개)에 있음
- 에이전트는 작업 진행 중 진짜로 필요할 때만 단계별로 도구 관련 정보를 추가로 불러옴
- 이로 인해 리소스를 아끼고, 과부하 없이 기능 확장이 가능함

### 첫 번째 정보 계층: 시스템 프롬프트에는 스킬의 요약 설명만 포함됨

- 에이전트 초기 구동 시에는 해당 스킬이 어떤 능력(예: PDF 처리 등)을 가지고 있다는 설명만 시스템 프롬프트에 제공됨
- 사용자가 관련 요청을 하지 않으면 다음 단계 정보는 호출되지 않음
- 이 방식을 통해 에이전트가 처음부터 불필요한 정보를 짊어지지 않음

### 두 번째 정보 계층: 사용자가 관련 요청 시 skill.md 파일이 로드되어 상세 지침을 제공함

- 사용자가 "PDF 파일에 관한 작업" 등 명시적 요청을 하면, 에이전트가 skill.md 파일을 읽음
- skill.md는 해당 스킬(기능)의 전체 사용 방법 및 조작 안내가 담긴 핵심 문서임
- 이를 통해 에이전트가 구체적으로 어떤 단계를 밟아야 할지, 어떤 포맷을 써야 하는지를 학습함

### 세 번째 정보 계층: 필요한 경우 skill.md에서 추가 참조 자료를 순차적으로 탐색함

- skill.md 파일에는 종종 추가 문서(예: 레퍼런스 문서, 예시, 세부 매뉴얼)가 링크됨
- 에이전트는 이들 추가 문서를 실제 작업 수행 중 필요시에만 더 깊이 탐색하여 활용할 수 있음
- 이 계층적 구조로 인해 정보 제공량이 실제 필요와 정확히 맞춰짐

### 다단계 정보 로딩 구조로 에이전트의 효율성과 성능이 극대화됨

- 스킬의 progressive disclosure는 첫 단계(설명) → 둘째 단계(skill.md) → 셋째 단계(추가 문서)로 이어지는 계층 구조
- 각 단계에서 실제 상황에 맞춰서만 정보가 순차 로딩됨으로써, 에이전트는 복잡해진 작업 환경에서도 안정적으로 대응
- Anthropic의 스킬 설정 방식은 복잡한 요구도 가볍게 처리할 수 있는 에이전트 제작이 가능함을 보여줌

### 실제 예시: PDF 처리 스킬의 단계별 적용 과정

- 영상에서는 PDF 파일을 처리하는 가상의 스킬을 예시로 제시함
- 에이전트가 기본 시스템 프롬프트에서 "PDF 작업 가능"이라는 설명만 받고 대기
- 사용자가 "이 PDF 파일을 요약해달라" 같은 요청 입력 시, 에이전트가 skill.md를 읽어 내부 지침에 따라 행동
- 추가 설명이나 예시 파일, 세부 매뉴얼이 필요하면 관련 문서를 추가적으로 점진적으로 로드

### Anthropic 스킬 프레임워크는 복잡성은 줄이고 확장성 있는 에이전트 구현을 가능하게 함

- 전체 구조는 에이전트의 초기 과부하 문제를 해결하면서도, 필요에 따라 점진적으로 복잡한 기능이나 지식을 추가할 수 있게 설계됨
- Anthropic 및 영상 제작자는 이런 구조가 사용자 경험과 AI 에이전트의 효율 양쪽을 모두 크게 끌어올릴 수 있다고 평가함
