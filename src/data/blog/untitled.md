## *대부분의 AI 에이전트와 MCP 서버는 개인 단일 계정만 지원해 확장성이 떨어지나, OAuth 연동과 툴 자동화를 통해 수천, 수백만 유저에게 맞춤형 에이전트를 제공할 수 있음* 핵심 요약

- 대부분의 AI 에이전트와 MCP 서버는 단일 사용자만을 대상으로 설계되어, 여러 사용자에게 서비스 확장이 어렵다.
- 일반적으로는 각 툴(예: Gmail, Slack, Asana)의 자격 증명(credential)이 하드코딩되어 있어, 하나의 사용자 계정만 사용할 수 있다.
- 여러 사용자가 자신의 계정으로 동일한 에이전트를 활용하려면, 계정별 개별 인증이 필요해 복잡성이 크게 증가한다.
- 'Arcade'라는 플랫폼이 이 문제를 해결하며, 에이전트가 OAuth를 통해 동적으로 사용자 계정 접근 권한을 요청·획득할 수 있게 해준다.
- Arcade는 사용자 인증만 완료되면 자격 증명을 안전하게 저장해 이후 동일한 인증 요청이 필요 없도록 한다.
- Arcade는 LangGraph 등과 통합되어, 에이전트와 툴 간 인증 및 워크플로우를 시각적으로 관리할 수 있도록 지원한다.
- 실제 데모에서는 사용자가 Asana 계정에 대해 최초 인증(authorization flow)만 마치면, 에이전트가 자동으로 Asana 프로젝트의 태스크 정보를 대신 불러올 수 있다.
- 이러한 방식은 사용자별 계정 접근, 장기 메모리, 다양한 툴 통합을 융합하여, 확장성 높은 초개인화 에이전트 구축을 실현한다.

## 세부 요약 - 주제별 정리

### 대부분의 AI 에이전트 구조는 사용자 1명용으로 설계되어 대규모 확장이 어렵다

- 전통적으로 개발된 AI 에이전트와 MCP 서버(멀티채널 플랫폼 서버)는 한 명의 사용자만 지원하도록 만들어졌으며, 이 점이 보통 간과됨
- 예를 들어 Gmail, Slack, Asana 등 각 툴에 대한 자격 증명이 코드 내에 직접 입력(하드코딩)되는 방식이 일반적임
- 이러한 구조는 단일 사용자의 계정에서는 잘 동작하나, 여러 명의 사용자 각자가 자신의 계정으로 에이전트를 사용하고 싶을 때는 큰 한계가 발생

### 여러 사용자의 계정 지원을 위한 에이전트 구조에는 추가 인증 및 관리가 필요하다

- 다수의 사용자가 같은 에이전트를 본인 계정으로 쓰려면, 각 사용자마다 개별적인 인증 과정이 필요
- 예를 들어, 사용자가 여러 명이라면 각각의 Gmail/Slack/Asana 인증 정보를 별도로 처리해야 함
- 개발 시 이 과정을 직접 구현하는 것은 복잡하고 에러가 발생하기 쉬움

### Arcade 플랫폼은 OAuth 동적 연동으로 이 문제를 간편하게 해결한다

- Arcade는 AI 에이전트가 사용자 계정 접근 시마다 OAuth를 이용해 권한을 동적으로 요청·획득할 수 있는 환경을 제공함
- OAuth 프로세스를 활용해 사용자의 보안과 권한 범위를 명확히 관리하며, 에이전트가 필요한 권한만 받아올 수 있도록 설계
- 각 사용자로부터 한 번 권한이 부여되고 나면, Arcade가 해당 자격 증명을 안전하게 저장하여 재요청이 필요하지 않음

### LangGraph와의 통합으로 인증·툴 사용 흐름을 워크플로우로 시각화할 수 있다

- Arcade는 LangGraph와 긴밀히 통합되어 있어, 인증 플로우와 에이전트의 행동(툴 실행 등)을 하나의 워크플로우로 그려볼 수 있음
- 이를 통해 개발자는 각 단계(계정 접근 확인 → 인증 필요 여부 판단 → 인증 플로우 → 툴 실행)의 전체 과정을 직관적으로 이해·관리할 수 있음

### 에이전트는 툴 사용 전 사용자 계정 접근 권한을 실제로 확인하고 요청한다

- 에이전트가 처음으로 특정 사용자의 예를 들어 Asana 계정에 접근하려 하면, 툴 실행 전 계정 연결 여부를 반드시 확인
- 만약 연결되어 있지 않거나 기존 인증 정보가 없으면, OAuth 인증 프로세스로 유저의 허가를 요청함
- 실제 데모에서는 "Asana의 'SaaS Ideas Projects'에 어떤 태스크가 있지?"라는 질문 실행 전, 계정 사용 권한 허가 플로우가 자동으로 작동함

### 사용자 계정 인증 완료 후 에이전트가 자동으로 워크플로우를 이어서 실행한다

- 사용자는 OAuth 인증 플로우(Arcade 제공)에 따라 자신의 Asana 계정 접근을 에이전트에 한 번 허가함
- 인증이 완료되면 에이전트 워크플로우가 자동 재개되어, 즉시 Asana 계정의 태스크 정보를 가져옴
- 추가 인증이나 중단 없이 에이전트가 원하는 모든 작업을 사용자의 실제 계정에서 수행함

### 이 방식은 수천~수백만 사용자까지 에이전트를 확장하는 기반이 된다

- 이제 동일한 에이전트를 수많은 사용자가 본인 계정으로 작동시켜 초개인화된 서비스가 가능함
- 각 사용자별로 계정 연동 및 인증 플로우가 자동화되어 추가 개발 부담 없이 확장성이 극대화됨
- 관리자는 새로운 툴이나 서비스를 통합할 때도 인증 구조를 별도로 수정할 필요 없이 빠르게 지원 가능

### 장기 기억 및 다양한 툴 통합과 결합해 진정한 초개인화 에이전트 실현이 가능하다

- 개별 사용자별 인증 기반을 장기 기억(예: LangGraph와의 연동) 알고리즘과 접목 가능
- 유저 이력, 선호, 맥락 등을 AI 에이전트가 기억·활용하면 더욱 그래뉼러하고 '나만을 위한' 맞춤형 서비스 제공 가능
- OAuth 인증, Arcade 플랫폼, LangGraph 워크플로우 등 현대 툴을 조합해 초대규모 맞춤형 AI 시스템 구축이 쉬워짐
