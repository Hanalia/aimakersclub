## *Glue.ai는 Slack이 지향했어야 할 미래형 AI/MCP 기반 채팅 플랫폼* 핵심 요약

- **영상 제목:** Glue.ai는 Slack이 지향했어야 할 미래형 AI/MCP 기반 채팅 플랫폼
- 영상 제작자는 매일 Slack을 사용하지만, 알림 문제와 비효율적인 워크플로우로 인해 점점 불만이 커지고 있다고 밝힘
- 중요한 버그 수정 스레드를 3일 늦게 발견하는 등 Slack의 조직력 결여로 업무가 불편함을 경험
- Slack에서는 실제 업무를 위해 여러 앱을 오가야 하며, 이는 큰 번거로움으로 작용함
- 최근 Glue 플랫폼을 직접 테스트하고 있음; 이 플랫폼은 Slack이 진화했어야 할 방향을 제시한다고 평가
- Glue는 최초의 멀티플레이어 MCP(Multiplayer Command Platform) 기반 채팅 플랫폼으로, 팀 전체가 효율적으로 업무를 진행할 수 있게 설계됨
- Glue AI는 대화 스레드 내에 직접 존재하며, 팀원들이 함께 업무를 처리하고 맥락을 공유할 수 있게 지원
- MCP 기반으로, Asana의 업무 업데이트, Notion 문서 읽기, GitHub 이슈 확인 등 35가지 주요 도구와 쉽고 신속하게 연동 가능
- 추가로, 커스텀 MCP 서버를 통해 수천 개의 툴과 연결할 수 있어 유연성이 매우 높음
- Glue는 단순한 AI챗봇이 탑재된 Slack 대체제가 아니라, 업무 효율화와 맥락 전환 최소화를 지향하는 ‘AI 네이티브’ 협업 플랫폼임
- 영상 마지막에서 최신 AI 툴을 추구하는 팀에게 Glue AI 사용을 강력히 권장

## 세부 요약 - 주제별 정리

### Slack 사용의 불편함과 조직력 문제로 인한 업무 비효율이 커지고 있음

- 영상 제작자는 자신의 비즈니스를 운영하면서 매일 Slack을 사용한다고 언급함
- 시간이 지날수록 업무 내 스레드가 점점 산만해지고, 정리가 되지 않아 불편함을 느꼈다고 고백
- 특히 자신이 해결 중이던 버그 수정 프로젝트의 중요한 스레드를 3일이나 늦게 확인하는 사고가 발생함
- 이러한 문제는 '알림(notification) 시스템 문제'로 규정하며, 본인의 스레드 관리 탓일 수도 있음을 인정
- 그러나 동시에 Slack 내 워크플로우에서 원하는 정보를 신속히 찾기 어렵고 체계화되지 않아 실질적 업무 효율이 떨어진다고 지적

### Slack에서의 작업은 여러 앱 전환과 맥락 전환 문제로 인해 효율이 크게 저하됨

- 업무를 실행하려면 Slack 대화 내용을 참고한 후, Asana, Notion, GitHub 등 다양한 협업툴로 이동해야 함
- 한 가지 작업을 완료하기 위해 수차례 앱을 전환하는 번거로운 과정을 반복해야 함
- 단순한 커뮤니케이션 툴의 한계 때문에 실제 업무 집행과 대화가 분리되어 생산성이 낮아지는 현상이 발생
- 영상 제작자는 '맥락 전환(context switching)'이 업무 스트레스의 큰 요인 중 하나임을 지적함

### Glue는 Slack의 한계를 극복한 최초의 MCP 기반 업무 협업 플랫폼이라 평가됨

- 영상 제작자는 최근 Glue 플랫폼을 테스트 중이라고 소개함
- Glue를 'Slack이 진화했어야 할 궁극의 모습'으로 명확히 규정
- Glue는 단순 메신저가 아니라, 실질적으로 팀이 협업하며 업무를 완수하는 데 최적화된 플랫폼임을 강조
- '최초의 멀티플레이어 MCP(Multiplayer Command Platform) 기반 채팅 플랫폼'이라는 점을 차별점으로 내세움
- MCP가 결합된 플랫폼 덕분에 사용자는 여러 도구, 앱, 서비스를 한 채팅 내에서 바로 컨트롤할 수 있음

### Glue AI는 대화 맥락 전체를 이해하고 팀이 함께 사용할 수 있는 업무용 AI로 구현됨

- Glue AI는 모든 스레드(대화) 내에 직접 상주하며, 대화 전체 맥락을 파악함
- 한 명이 아니라 팀 전체가 Glue AI의 기능을 함께 활용하여 업무 논의 및 실질적 실행까지 지원받을 수 있음
- 기존 Slack의 외부 AI 플러그인 방식이 아니라, AI가 플랫폼 핵심에 내장된 구조임

### Glue의 MCP 연동 기능은 35개 이상의 툴을 원클릭으로 연결 가능하고, 커스텀 서버를 통한 확장성도 뛰어남

- MCP의 강력한 통합 기능 덕분에 Asana의 업무 업데이트, Notion의 문서 확인, GitHub의 이슈 조회 등 주요 협업 기능을 채팅 내에서 직접 실행할 수 있음
- 35가지 인기 협업 도구를 사용자가 클릭 한 번으로 연동할 수 있도록 설계됨
- MCP 커스텀 서버를 활용하면, 수천 가지 도구와도 손쉽게 연결 및 명령 전송이 가능해 업무 자동화 수준이 매우 높음
- Glue는 '가장 대규모의 인앱 MCP 디렉터리'를 구축했다고 밝힘

### Glue는 AI가 부가적으로 탑재된 것이 아니라, 완전히 AI 중심적으로 재설계된 차세대 업무 채팅 플랫폼임

- Glue의 본질은 'Slack에 AI 기능을 나중에 붙인 것'이 아니라, 아예 설계 단계부터 'AI 네이티브 플랫폼'으로 구현되었다는 점임
- 팀 채팅, 업무 집행, 정보 조회, 협업 명령어 실행이 모두 하나의 채팅내에서 무리 없이 합쳐짐
- 맥락 전환 없이 신속하게 여러 작업을 처리하고, 팀 전체가 효율적인 협업을 이어갈 수 있도록 지원
- 영상에서는 구체적으로 “Glue는 업무를 신속히 처리하며 맥락 전환을 피하고 싶은 팀에게 적합하다”고 강조

### Glue AI는 최신 AI 툴을 만든 팀이지만 팀 내 커뮤니케이션이 구식이라 느끼는 이들에게 최적임

- 영상 마무리에서는 AI 관련 개발, 도구 생산에는 앞서 있지만, 팀 채팅만은 Slack 등 기존 도구에 머물러 있다고 느끼는 팀을 겨냥함
- 그런 팀에 Glue AI 검토를 강력히 권장하며, 영상의 요점을 마무리함
