## *RAG 2.0: 에이전트형 RAG와 지식 그래프의 결합(무료 템플릿 공개)* 핵심 요약

- 영상 제목은 “RAG 2.0: 에이전트형 RAG + 지식 그래프(무료 템플릿)”임
- 최근 여러 RAG(Retrieval-Augmented Generation) 전략을 분석한 결과, 에이전트형 RAG와 지식 그래프 두 방식에 주목했음
- 영상 제작자는 이 두 가지 방식을 동시에 사용하는 강력한 AI 에이전트를 직접 제작함
- 이 에이전트는 질문의 성격에 따라 가장 효율적인 정보 검색 경로(벡터 DB/지식그래프)를 스스로 판단함
- 벡터 데이터베이스로는 Postgress와 PG vector 확장, Neon(서버리스 플랫폼)을 사용하여 문서 임베딩 및 청킹 진행
- 동일 문서를 Graffiti와 Neoforj를 이용해 지식 그래프 형태로도 저장, 관계 질의까지 구현
- 특정 회사 단일 정보는 벡터 DB를, 회사 간 관계성 질의는 지식 그래프를 자동 질의함
- 프롬프트를 조정하거나 명시적으로 두 소스를 모두 사용하도록 지정 가능
- 이 AI 에이전트의 템플릿은 무료로 제공되며, Cloud Code로 개발 과정을 자세히 설명한 전체 버전 영상 링크 안내
- 실제 데모 과정을 통해, 질문 의도에 따라 AI가 적절한 도구(벡터DB, 그래프 등)를 선택하여 답변함을 시연함

## 세부 요약 - 주제별 정리

### 최근 다양한 RAG 전략 분석 결과 에이전트형 RAG와 지식 그래프 방식이 핵심임을 제시함
- 영상 제작자는 최근 몇 달간 모든 RAG 전략을 심층적으로 검토했다고 언급
- 본인이 반복적으로 돌아오게 된 두 가지 전략은 “에이전트형 RAG(Agentic Rag)”와 “지식 그래프(Knowledge Graphs)”임을 강조
- 두 방식을 조합하면 기존 RAG 방식보다 유연하고 강력한 정보 검색 및 답변이 가능해짐

### 에이전트가 벡터 DB와 지식 그래프를 결합해 최적의 정보 검색 경로를 스스로 결정함
- 제작자는 두 방식을 통합한 AI 에이전트를 직접 구축, ‘매우 강력하다’고 설명
- 에이전트가 어떤 소스로부터 정보를 검색해야 최적의 답변이 나올지 스스로 “추론(reason)”하는 구조임
- 이는 사용자의 질문 의도를 파악하여 벡터 DB, 지식 그래프 중 적절한 데이터 소스를 자동 선택하는 방식임

### 벡터 데이터베이스 구축에는 Postgress와 PG vector, Neon을 활용함
- 벡터 데이터베이스는 Postgress에 PG vector 확장을 설치해 구현
- 서버리스 방식(Postgress 기반)으로는 Neon 플랫폼을 사용함을 밝힘
- 문서는 ‘청킹(chunked)’되어 벡터 임베딩 생성, 효율적 검색이 가능하도록 준비함
- 테스트용 문서는 ‘빅테크 기업들의 AI 프로젝트 및 파트너십 정보를 담은 내용’임을 구체적으로 명시

### 동일 문서를 Graffiti와 Neoforj를 활용하여 지식 그래프 구조로도 저장함
- 같은 문서를 지식 그래프(relationship-based DB)로도 변환·저장
- 이를 위해 Graffiti 및 Neoforj(Neo4j의 오타로 추정) 툴을 활용함
- 지식 그래프 방식은 “회사 간의 관계성” 등 복잡한 관계 질의 검색에 뛰어난 성능을 보임
- 예시: “Amazon은 Enthropic의 모델 호스팅 장소이다”처럼 기업 간 관계 DB화

### 단일 회사 정보 질의 시, 에이전트가 벡터 데이터베이스 사용을 자동 선택함
- 실제 데모: “Google의 AI 프로젝트 현황”과 같이 한 기업에 대한 정보를 물어봄
- 에이전트는 벡터 DB에서 답을 검색, 사용한 도구까지 결과에 명시함
- 출력 결과: 벡터 DB를 탐색하여 얻은 정보를 사용자에게 반환

### 두 회사 간 관계성 질문 시, 에이전트는 자동으로 지식 그래프를 탐색함
- 관계성 질의(예: 두 기업간 파트너십 등)에 대한 데모 진행
- 이 때 에이전트는 프롬프트 설계에 따라 지식 그래프를 자동 탐색함
- 답변에는 “그래프 서치 사용”과 같이, 실제 어떤 도구를 사용했는지도 명시됨

### 시스템 프롬프트 조정 및 명시적 명령을 통해 정보 출처를 수동 지정할 수도 있음
- 기본 프롬프트 외에도, 사용자가 벡터 DB와 그래프 DB 모두 사용을 명시할 수 있음
- 실제 예시: “벡터 DB와 그래프 DB 둘 다 사용하라”는 식으로 직접 지시
- 에이전트는 이 경우 두 데이터 소스를 모두 검색해 보다 정확하고 풍부한 답변 제공

### AI 에이전트 전반 구조와 무료 템플릿, 상세 구축 과정 소개
- 제작자는 본 에이전트의 소스 및 템플릿을 누구나 사용·활용 가능하도록 무료로 공개함을 강조
- 전체 에이전트 제작과정(Cloud Code 활용 등)은 별도 전체 영상 링크에서 자세히 다룸을 안내
- 이번 영상은 요약 및 데모, 다음 영상에서 상세 제작과정 안내

### 데모를 통해 도구 선택과 결과 제시 방식을 명확히 시연함
- AI 에이전트가 실제로 질문 의도에 따라 벡터 DB, 그래프 DB를 구분해 검색을 수행
- 답변 출력시, 사용한 도구(예: "Tools used: vector database" 또는 "Tools used: graph search") 명시
- 질문 종류에 따라 다르게 동작하는 프롬프트-기반 에이전트의 작동 원리를 실시간으로 보여줌

### 누구나 활용 가능한 오픈 무료 템플릿 배포 및 후속 콘텐츠 안내로 영상 마무리
- 본 AI 에이전트 템플릿은 무료로 제공되고 있음
- 전체 제작법 및 코드 안내를 담은 완성 영상 링크를 자막과 내용 내에서 제공
- 개발자는 Cloud Code를 활용해 개발이 용이하도록 했음을 언급
- 실제 영상에서는 구현 과정을 빠르게 안내하고, 이후 영상에서 상세 개발법을 공유할 것임을 안내
