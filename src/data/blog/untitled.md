## *RAG 2.0: 에이전틱 RAG와 지식 그래프의 결합(무료 템플릿 소개)* 핵심 요약

- 영상 제목은 "RAG 2.0: 에이전틱 RAG와 지식 그래프의 결합(무료 템플릿 소개)"임
- 제작자는 최근 다양한 RAG(검색 증강 생성) 전략을 연구한 끝에 ‘에이전틱 RAG’와 ‘지식 그래프’ 방식에 집중하게 되었음
- 이 두 가지 접근을 조합해, 상황에 따라 적합한 정보원(벡터 데이터베이스, 지식 그래프)을 자동으로 선택해 탐색하는 강력한 AI 에이전트를 개발함
- 벡터 데이터베이스에는 Postgres와 PG vector extension을 적용했고, Neon이라는 서버리스 Postgres 플랫폼을 활용함
- 문서의 임베딩을 생성해 벡터 데이터베이스에 저장하는 한편, 동일 내용을 Graffiti와 Neo4j 기반의 지식 그래프에도 관계형 형태로 저장함
- 사용자의 질문 유형에 따라 AI 에이전트는 해당 질문이 단일 정보 탐색(예: "구글의 AI 이니셔티브는?")이면 벡터 DB, 두 회사 사이의 관계(예: "아마존과 앤트로픽의 협력 관계")라면 지식 그래프를 활용함
- 시스템 프롬프트나 명시적 지시를 통해 양쪽 정보원을 모두 사용하여 보다 신뢰도 높은 답변도 제공 가능함
- 이 에이전트는 무료 템플릿으로 배포 중이며, 전 과정 구현 및 Cloud Code 활용법은 본 영상의 링크에서 확인 가능함

## 세부 요약 - 주제별 정리

### ‘에이전틱 RAG’와 ‘지식 그래프’의 결합이 AI 정보 탐색력의 핵심 동력으로 작동함

- 최근 몇 달 동안 거의 모든 종류의 RAG(검색 증강 생성) 전략을 직접 실험하고 연구함
- 두 가지 전략에 주목: 에이전틱 RAG(Agentic RAG)와 지식 그래프(Knowledge Graphs)
- 에이전틱 RAG는 AI 에이전트가 자체적으로 검색 전략을 세우고, 관련 도구(정보원)를 선택해 활용하는 방식임
- 지식 그래프는 데이터 간의 복잡한 관계를 표현하고 탐색하는 데 특화된 구조임
- 이 두 가지 기술을 통합하면 질문 유형이나 맥락에 맞는 최적의 탐색 전략을 AI가 자동으로 설계·수행함

### 벡터 데이터베이스에는 Postgres, PG-vector, Neon 플랫폼을 활용해 임베딩 기반 검색을 구현함

- 벡터 데이터베이스로 PostgreSQL에 PG vector extension을 붙여 사용함
- 이 과정에서 Neon이라는 서버리스 Postgres 플랫폼을 선택해 효율적으로 환경을 구축함
- 실험 대상으로 사용한 문서는 빅테크 기업들의 AI 이니셔티브, 상호 파트너십 등 다양한 정보를 포함함
- 해당 문서를 청크 단위로 쪼개어, 각 청크에 대해 임베딩을 생성함
- 생성된 임베딩을 PostgreSQL 벡터 DB에 저장해 질의 시 벡터 유사도 기반 정보를 빠르게 찾도록 함

### Graffiti와 Neo4j로 동일 문서를 관계형 데이터로 변환해 지식 그래프에 저장함

- 동일 데이터(문서)를 또 한번 변환해 그래피티(Graffiti) 및 네오포제이(Neo4j) 기반 지식 그래프에 저장함
- 지식 그래프는 각 회사, 주요 AI 이니셔티브, 협력 관계 등 데이터 간의 관계성을 구조적으로 표현함
- 예: ‘앤트로픽(Anthropic)이 아마존의 AWS에 모델을 호스팅함’ 같은 관계 정보가 그래프의 형태로 저장됨
- 정보 간 연결고리가 필요한 복합적 질의에 매우 유용함
- 한 문서를 두 가지 포맷(벡터 DB, 지식 그래프)으로 모두 저장해, 탐색 방식의 유연성을 극대화함

### AI 에이전트는 질문 유형을 자동 분석해 어떤 정보원을 쓸지 스스로 판단함

- AI 에이전트는 ‘최적의 답변을 얻기 위해 어느 정보원(벡터 DB, 지식 그래프)에 접근할지 스스로 연산함
- 제작자가 작성한 system prompt에 이 판단 로직이 반영되어 있음
- 예시: "구글의 AI 이니셔티브는?" 같이 단일 엔터티 정보는 벡터 DB 질의가 효율적임
- 복수 개체 간 관계를 묻는 질문(예: “Amazon과 Anthropic의 파트너십은?”)은 지식 그래프 질의가 적합
- 모든 질의 후, 에이전트가 사용한 도구(벡터 검색/그래프 검색)가 결과에 표시됨

### 사용자는 system prompt 조정 또는 직접 명령으로 정보원 활용을 세밀하게 제어 가능함

- 사용자는 프롬프트 세팅에 따라 기본 AI 판단 방식을 바꿀 수 있음
- “두 정보원을 모두 이용해서 답변하라” 등 명시적 요청도 가능
- 이런 경우 에이전트는 벡터 DB와 지식 그래프 양쪽에서 정보 탐색 후, 신뢰도 높은 복합 답변 생성
- 도구 사용 현황(검색 경로 등)까지 투명하게 알려줌
- 질의 패턴과 정보원 선택의 상호작용을 직접 확인할 수 있음

### 무료로 제공되는 AI 에이전트 템플릿과 실습 영상 안내

- 본 AI 에이전트는 누구나 사용할 수 있는 무료 템플릿으로 제작됨
- 영상 끝 부분에서 전체 구현 과정과 코드 배포 링크를 별도 안내함
- Cloud Code를 활용한 에이전트 개발 방법, 직접 설치·운영하는 방법 등에 대한 추가 영상 존재
- 무료 템플릿이니 영상 설명란의 링크를 통해 실습 및 테스트 가능함
