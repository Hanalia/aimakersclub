## *RAG 2.0: 에이전틱 RAG와 지식 그래프의 결합 (무료 템플릿 소개)* 핵심 요약

- 영상 제목은 "RAG 2.0: 에이전틱 RAG와 지식 그래프의 결합 (무료 템플릿 소개)"로, 두 가지 최신 RAG(검색증강생성) 전략을 결합한 AI 에이전트 구현 과정을 다룸
- 제작자는 최근 몇 달간 다양한 RAG 전략을 실험한 결과, 'Agentic RAG'와 'Knowledge Graphs' 두 가지 방법에 집중하게 됨
- 이 두 방법을 결합한 결과, 질문에 가장 적합한 정보원을 스스로 판단하여 검색하는 고성능 AI 에이전트를 구축
- 벡터 데이터베이스로는 PG vector 익스텐션이 설치된 PostgreSQL과 서버리스 플랫폼 'Neon'을 사용하여 임베딩 및 검색을 처리
- 지식 그래프는 Graffiti와 Neo4j를 활용해, 동일한 정보를 관계형 데이터 구조로 저장해, 기업 간 관계까지 질의할 수 있도록 설계
- 도큐먼트 및 임베딩 자료에는 빅테크 기업들의 AI 사업과 네트워킹(파트너십) 정보가 상세히 포함됨
- 에이전트는 질문의 성격에 따라 벡터 데이터베이스 또는 지식 그래프 중 적합한 도구를 자동 결정하여 검색
- 사용자가 원한다면 벡터 DB와 그래프를 동시에 명시적으로 활용해 더욱 신뢰도 높은 답변 산출 가능
- 에이전트가 실제 어떤 검색도구를 사용했는지 출력 결과에 명확히 표시
- 영상 말미에서 무료 에이전트 템플릿, 구축 방법 전체 영상, Cloud Code 활용법, 셋업 방법에 대한 안내 링크를 제공

## 세부 요약 - 주제별 정리

### 제작자는 에이전틱 RAG와 지식 그래프를 깊이 실험한 끝에 두 전략의 결합을 선택함

- 최근 몇 달간 거의 모든 최신 RAG(검색증강생성) 전략을 폭넓게 실험
- 'Agentic RAG'(에이전틱 RAG)와 'Knowledge Graphs'(지식 그래프) 두 방식을 여러 차례 반복적으로 검토
- 결국 이 두 전략이 가장 강력함을 직접 확인하고, 통합 에이전트 구현을 시도함

### 제작된 AI 에이전트는 질문의 특성에 따라 적합한 정보원 선택과 검색을 수행함

- 새로 만든 AI 에이전트는 복수의 데이터 소스를 동시에 활용할 수 있음
- 사용자의 질문 의도와 내용에 따라, 어느 정보원이 더 효과적으로 답변을 줄지 스스로 추론
- 예시: '한 회사의 AI 전략' → 벡터 DB 검색 / '회사 간의 파트너 관계' → 지식 그래프 질의를 자동 선택

### 벡터 데이터베이스는 PG vector와 Neon(서버리스 PostgreSQL) 조합을 사용함

- 문서(도큐먼트)는 여러 '덩어리(chunk)'로 분할되어 저장
- 각 문서 조각에 대해 임베딩(embedding) 정보 생성 및 저장
- 사용된 플랫폼:
    - PostgreSQL(포스트그레스)에서 PG vector 확장 모듈 활용
    - ‘Neon’이라는 서버리스(PostgreSQL 기반) 플랫폼 채택
- 이 DB에는 빅테크 기업들과 그들의 AI 전략, 상호 파트너십 관련 데이터가 포함

### 동일 문서를 지식 그래프로 재구성해 복잡한 관계형 검색까지 가능하게 함

- 위의 동일한 데이터셋을 Graffiti, Neo4j 엔진을 활용해 지식 그래프로 정형화
- 그래프 구조는 엔터티(회사, 프로젝트 등) 간의 다양한 관계(누가 누구를 파트너로 삼는지 등)가 표현됨
- 예시:
    - "Amazon은 Anthropic이 자기 모델을 호스팅하는 플랫폼"임을 그래프 쿼리로 검색 가능

### 에이전트는 사용자의 질문 유형에 따라 자동으로 적합 도구를 선택해 답변함

- "구글의 AI 전략이 무엇인가?"처럼 단일 기업 특성 질문 → 벡터 데이터베이스로 질의
- "Amazon과 Anthropic의 파트너십 구조는 어떻게 되는가?"처럼 관계 기반 질문 → 지식 그래프로 질의
- 실시간으로 어떤 데이터 소스(벡터/그래프)를 사용했는지 결과와 함께 표시

### 사용자가 원할 경우 벡터 DB와 그래프를 동시에 활용하여 통합 답변도 생성 가능함

- 시스템 프롬프트를 조정하거나, 직접 "벡터 DB와 그래프 모두 사용"이라고 명령할 수 있음
- 에이전트는 두 데이터 소스를 동시에 질의해, 더욱 완비되고 신뢰성 높은 답변을 생성
- 실제 동작 시, 두 소스에서 가져온 데이터를 종합해 최종 결과 제공

### 에이전트의 동작 이력과 사용 도구는 사용자에게 투명하게 제공됨

- 에이전트가 검색 시 어떤 소스(벡터 DB/그래프)와 도구를 사용했는지 명시적으로 결과에 표시됨
- 답변 근거와 써머리, 사용 데이터 소스 모두 확인 가능

### 무료 템플릿 및 전체 구축 영상 제공 등 실질적 활용 안내로 마무리됨

- 본 영상에서는 기능 시연 위주로 다루며, 실제 실습과 구축법은 전체 영상에서 설명하는 링크 안내
- 무료 AI 에이전트 템플릿을 공개해 직접 사용 및 커스터마이즈가 가능함을 강조
- 전체 플로우 및 코드 구현에는 Cloud Code를 활용했음을 추가 언급
- 시청자는 안내된 링크를 따라가면, 해당 에이전트 설치 및 운영법을 실습할 수 있다고 안내
