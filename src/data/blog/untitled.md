## *AI 에이전트를 획기적으로 강화하는 가장 단순한 방법(스킬)* 핵심 요약

- Anthropic이 최근 공개한 ‘스킬(Skills)’ 기능은 최근 AI 발전 중 가장 중요한 혁신 중 하나로 평가됨
- Anthropic의 개발 철학은 “더 단순할수록 더 좋다(The simpler the better)”에 기초함
- 에이전트의 능력을 대폭 강화하고 싶지만, 너무 많은 정보로 컨텍스트 윈도우(즉, 입력 용량)를 과부하시키는 문제를 피해야 함
- 기존의 MCP 서버 방식은 각 에이전트에게 여러 도구 정보를 한 번에 제공하여, 특정 대화에서 필요하지 않은 도구까지 과도하게 넣는 문제가 있음
- ‘스킬’은 이런 문제를 해결하기 위해 ‘점진적 공개(progressive disclosure)’라는 개념을 활용함
- 에이전트는 처음에 각 스킬의 간단한 설명만 시스템 프롬프트로 받아들이고, 실제 필요 시에만 자세한 설명과 문서가 단계적으로 추가됨
- 예시로, PDF 처리 스킬의 경우 “PDF를 처리할 능력이 있다”는 사실만 우선 시스템 프롬프트로 주입함
- 사용자가 PDF 관련 요청을 하면 그때 에이전트가 skill.md 파일 등 구체적 스킬 지침을 실제로 불러와 활용함
- skill.md가 스킬 작동의 핵심 문서이며, 필요 시 추가 문서 참조까지 이어짐(단계별 맥락 확장)
- 이로써 필요할 때만 관련 정보를 온디맨드로 불러오며, 에이전트의 맥락 과부하 없이 다양한 능력을 담을 수 있음

## 세부 요약 - 주제별 정리

### Anthropic이 ‘스킬’ 기능을 도입하며 AI 발전의 중요한 이정표를 세움

- Anthropic은 최근 AI 도구 개발 역사상 중요한 진전에 해당하는 ‘스킬(Skills)’ 기능을 공개함
- 이 기능은 AI 에이전트의 기능 확장과 프로그래밍 측면에서 혁신적인 전환점을 제공함
- 영상 제작자는 Anthropic의 스킬이 “최근 AI 분야에서 가장 중요한 발전 중 하나”라고 강조함

### 단순함이 Anthropic의 AI 설계 핵심 원칙으로 작용함

- Anthropic이 추구하는 기본 모토는 “더 단순할수록 더 좋다(The simpler the better)”임
- 복잡한 시스템보다 최대한 단순한 구조가 에이전트 효과성과 유지 관리, 확장성에 유리하다는 점을 지적함
- 스킬 기능도 단순성을 전제로 설계되어 있음

### 에이전트의 능력 확장이 콘텍스트 윈도우의 과부하로 이어지는 문제를 지적함

- 에이전트에 다양한 기능을 부여하고 싶지만, 모든 툴 정보와 대화 히스토리, 시스템 프롬프트 등을 한 번에 주입하면 콘텍스트 윈도우가 과도하게 사용됨
- AI 에이전트는 입력 정보가 너무 많아질 때 쉽게 혼란스러워지고, 의도한 대로 작동하지 않게 될 수 있음
- ‘도구(tool)’, ‘대화 기록(conversation history)’, ‘시스템 프롬프트’ 모두 동일한 콘텍스트 윈도우(즉, 제한된 입력 칸)에 포함됨

### MCP 서버 등 기존 에이전트 도구 부여 방식의 단점이 지적됨

- MCP 서버 방식처럼 처음부터 수많은 툴 정보를 에이전트에 전송하는 것은 비효율적임
- 어떤 대화에서는 해당 도구들을 전혀 사용하지 않더라도 모든 정보를 맥락에 불필요하게 포함하게 됨
- 이런 접근 방식은 AI 성능 저하와 리소스 낭비 문제로 이어짐

### 스킬(Skills)의 ‘점진적 공개’(progressive disclosure) 설계가 핵심 혁신임

- 스킬은 ‘점진적 공개’라는 설계 패러다임이 적용된 것이 가장 큰 특징임
- 에이전트가 실시간으로 필요할 때만 관련된 추가 정보를 받아서 맥락 내에 끼워넣는 방식을 취함
- 모든 스킬의 세부 설명과 지침을 처음부터 제공하지 않고, 매우 간단한 설명만 초기 시스템 프롬프트로 제공

### 세부 정보 노출이 필요 시점에 단계별로 이뤄지도록 설계됨

- ‘점진적 공개’는 단계별로 정보가 에이전트에게 주어지는 구조임
- 1단계: 각 스킬의 간단한 설명이 시스템 프롬프트 초기에 제공됨
- 2단계: 실제로 스킬이 필요해졌을 때 에이전트가 ‘skill.md’(스킬 설명 및 사용법 문서)를 읽음
- 3단계: skill.md가 추가 문서를 참조할 경우, 그때 해당 문서 정보가 더해짐
- 이렇게 점진적으로 맥락(Context)을 끌어오며, 과도한 정보로 시작하지 않고, 필요한 순간에만 온디맨드로 활용

### PDF 처리 예시를 들어 스킬 적용 원리를 구체적으로 설명함

- 예를 들어 PDF 처리 능력이 필요한 경우, 시스템 프롬프트는 initially(처음)에 “PDF를 처리할 수 있다”는 사실만 기술
- 사용자가 PDF 관련 요청을 할 때, 그때 에이전트가 skill.md(해당 스킬의 설명 전문) 내용을 맥락에 추가함
- 이후 필요할 경우 skill.md가 참조하는 추가적 문서(3단계)를 불러옴으로써, 사용 요청 흐름에 따라 점차 상황에 맞는 정보가 추가됨

### skill.md가 Anthropic 스킬 구조 설계의 중심적 역할을 수행함

- 모든 스킬의 본문 역할을 하는 문서가 skill.md 파일임
- skill.md에는 해당 스킬의 전체적 동작 지침 및 사용 방법이 구체적으로 기록됨
- Anthropic의 공식 스킬 또는 커스텀(사용자 정의) 스킬 모두 이 구조를 따름

### 단계별 스킬 적용으로 에이전트 능력은 강화하면서도 콘텍스트 효율성이 향상됨

- 스킬의 점진적 정보 제공 방식 덕분에, 에이전트는 많고 다양한 능력을 부담 없이 탑재할 수 있음
- 동시에 시스템의 콘텍스트 윈도우가 과부하되지 않도록 효율을 극대화함
- 즉, 유연성과 확장성, 성능 안정성 모두 유지하는 최적 해법이 됨

### 점진적 공개로 에이전트의 작업 흐름과 정보 이용 방식이 근본적으로 변화함

- 스킬 정보가 단계적으로 추가되어, 에이전트가 필요할 때 필수 정보를 실시간으로 즉시 접근할 수 있음
- 초기 설정 시 부담이 적고, 작업 중 필요한 맥락만 정확히 불러올 수 있어 작업 정확도 및 효율 증대
- 기존 일괄 도구 등록 대비, 실사용 관점에서 에이전트 사용자 경험이 크게 개선됨
