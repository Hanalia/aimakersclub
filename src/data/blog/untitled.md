## *MCP 서버의 큰 문제점과 해결책* 핵심 요약

- MCP 프로토콜이 널리 사용되면서 중대한 결함이 드러났음을 영상 초반에서 강조함
- 핵심 문제는 AI 활용에서의 토큰 소비량 과다와 ‘컨텍스트 소실(context rot)’로 요약되며, 이는 현재 매우 논쟁적인 주제임
- 영상에서는 MCP가 실제로 어떻게 문제가 되는지 구체적인 예시를 들어 설명하기로 함
- 예시에서는 AI 코딩을 지원하는 표준 MCP 서버 5개를 사용하는 상황을 제시함
- 각 서버는 AI 에이전트에게 기능을 설명하는 데만도 수천 개의 토큰이 필요하여, 여러 서버를 동시에 사용할 경우 토큰 낭비가 극심함
- 이러한 비효율로 인해 한 세션에서 여러 MCP 서버의 동시 사용이 사실상 불가능함을 강조
- 영상은 이 문제의 해결책으로 상당히 강력한 ‘agent skills’를 소개함
- 엔트로픽(Anthropic)에서 최근 출시한 agent skills가 스크립트 생성, 코드 실행, API 연동 등의 자동화를 지원하여 MCP의 문제를 근본적으로 해결한다고 설명함
- 영상 후반에는 '기존 MCP 방식이 끝난 것인가?'라는 질문을 던짐
- 결론적으로 'agent skills'와 코드 실행 기능이 MCP의 오랜 문제를 해소할 혁신적 솔루션임을 강조하며 영상을 마무리함

## 세부 요약 - 주제별 정리

### MCP의 인기가 높아지면서 원래의 구조적 단점이 두드러지게 드러남

- MCP(Multi-Component Protocol)는 점차 대중화되고 다양한 AI 작업에 활용되고 있음
- 하지만 사용자가 늘고 프로토콜이 복잡해지면서 바탕에 깔린 구조적 결함이 더 명확해졌음을 언급함
- 영상 초반, 현재 개발자 커뮤니티 내에서 가장 ‘핫’한 두 가지 이슈로 토큰 소비와 컨텍스트 소실(context rot)을 꼽음
- “이 문제를 완전한 예시와 함께 설명하겠다”고 예고하여 실제적인 사례 분석을 예고함

### 토큰 소비와 컨텍스트 소실이 MCP 사용에서 치명적인 문제임을 사례로 설명함

- 영상 제작자는 직접 겪은 문제를 예로 들며 설명을 전개함
- 예시: AI 코딩 보조에 특화된 대표적인 MCP 서버 5개를 나열
- 각 서버마다 AI에게 “내가 누구고, 뭘 할 수 있는지” 설명하는 데만 각각 ‘수천 개’의 토큰을 사용
- 여러 서버를 동일한 세션에서 활용하려면 토큰 요구량이 기하급수적으로 증가하여 금세 한계에 부딪힘
- “나는 절대, 진짜 절대 한 세션에서 이 MCP 서버 5개를 동시에 쓰지 않는다”고 강조
- 토큰 제한으로 인해 실제 업무에서 여러 MCP 서버 병행 활용이 매우 비효율적이라는 것을 강조함

### 기존 MCP 방식에서는 서버 간 기능 결합이 사실상 불가능함을 지적함

- 현재 구조에서는 각 MCP 서버의 기능을 AI 에이전트에게 한 번에 설명할 때 토큰 문제로 인해 제약이 큼
- MCP 서버 각각이 ‘독립적 능력 집합’으로 존재할 뿐, 이들을 조합하여 고차원 기능을 생성하기 어렵다고 지적
- 실제 워크플로우에서 서버 간 상호운용성 부족이 실무 방해 요소가 됨을 강조함

### 엔트로픽의 agent skills가 MCP의 핵심적 한계를 근본적으로 해결함

- 영상의 해결책으로 ‘agent skills’라는 신규 기능을 상세히 소개함
- 엔트로픽(Anthropic)에서 최근 릴리즈한 것으로, AI 에이전트가 동적으로 스크립트를 생성할 수 있게 하는 기능임을 밝힘
- 대표적 활용: API 엔드포인트와 상호작용, 자동화된 명령어/스크립트 생성, 실시간 코드 생성 등
- “code execution at its finest(최고 수준의 코드 실행)”이라고 언급하며, 자동화와 즉시 실행성이 뛰어남을 강조

### agent skills의 도입이 AI 워크플로우 및 MCP 서버 활용 방식을 혁신적으로 변화시킴

- agent skills의 도입으로 개발자가 더 이상 ‘토큰 경제’ 문제에 제약받지 않고 다양한 기능 조합이 가능함을 시사
- 에이전트는 각 서버의 기능을 사전에 전달받지 않아도, 필요할 때마다 스크립트를 동적으로 만들어 활용
- 이로 인해, 컨텍스트 소실이나 토큰 한계로 인한 서버 연동의 비효율성이 사라짐
- 신규 방식은 기존 MCP 프로토콜의 구조적 문제를 기술적으로 근본 해소한다고 평가

### code execution 및 스크립트 생성의 확장성이 엔지니어링 생산성을 향상시킴

- agent skills를 통해 개발자가 API, 데이터베이스, 기타 프로그래밍 대상에 쉽게 접근하고 조작 가능
- 자동 코딩 및 즉각적 자동화가 가능해짐에 따라 엔지니어는 반복작업을 현저히 줄일 수 있음
- 복잡한 워크플로우 자동화도, MPC 서버 기능 제약 없이 손쉽게 설계할 수 있음을 시사

### ‘MCP의 끝인가?’라는 중대한 질문을 던지며 기술 전환의 전조를 보여줌

- 영상 후반부에서 “Is this the end of MCP as we know it?”(우리가 아는 MCP의 끝인가?)라는 의문을 던짐
- 이는 long-context 관리와 토큰 비용이 필요 없는 새로운 agent skills 패러다임이 기존 MCP 방식의 마침표가 될 수 있음을 내포
- 기술 생태계의 패러다임 전환 및 주요 전환점임을 시사하면서도 구체적 미래 전망 언급은 피함

### 영상 결론에서는 agent skills와 코드 실행 능력이 MCP의 기존 문제를 완전히 해결한다고 밝힘

- 영상 마지막에는 “이것이 바로 MCP의 문제와 solution(해결책)이다”라고 직접적으로 정리
- ‘skills’ 및 ‘코드 실행(code execution)’이 결합됨으로써, MCP 서버와 AI 에이전트의 협업 구조가 실제적으로 혁신 가능함을 강조
- 단순 결함 지적을 넘어서, 실제 대안과 근거, 그리고 기술 발전 방향까지 간명하게 전달함
