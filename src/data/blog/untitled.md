## *Postman이 Git-네이티브로 진화하며 API 빌드 방식을 근본적으로 혁신함* 핵심 요약

- 기존의 API 작업은 API 엔드포인트, 컬렉션, 테스트, 스펙이 도구와 위치별로 분산되어 있어 워크플로우가 복잡하고 비효율적이었음
- AI 에이전트 개발 등 최신 프로젝트에서 API 엔드포인트와 연동 작업은 필수적이지만 관리와 버전관리 체계가 통합되지 않아 불편함 피할 수 없었음
- Postman이 최근 'AI 네이티브 API 플랫폼'으로 대대적인 업데이트를 단행, Git-네이티브를 본격 지원하기 시작함
- 컬렉션, 스펙, 모의(Mock), 테스트 등 Postman 리소스를 단순한 YAML 파일로 로컬 코드베이스에 직접 저장·관리 가능해짐
- Postman 데스크탑 앱에서 직접 파일을 관리하거나 클라우드와 2-way 동기화해 실시간 협업 및 관리 가능
- 모든 변경사항이 git diff 등으로 즉시 추적·버전관리 가능, 코드와 동일하게 사양/테스트 버전 통일화 실현
- 모든 API 관련 리소스가 디스크 파일로 통합된 덕분에 Postman의 에이전트 모드로 전체 API 라이프사이클을 자동화 가능
- 사용자는 평문 영어로 요청을 기술하면 코드를 분석해서 자동으로 컬렉션 생성, 라우트 추론, 스펙/테스트 생성, Mock 환경 구축까지 일괄 처리
- AI 모델(GPT, Claude, 자체 호스트 등) 선택 가능하며, GitHub/Sentry/Linear 등 다양한 툴에서 컨텍스트를 자유롭게 가져올 수 있음
- 동일한 컬렉션이 로컬·CI 파이프라인 모두에서 재사용되어 환경별 테스트 재작성 없이 Pre-commit, GitHub Actions 등 자동화 파이프라인과 통합됨

## 세부 요약 - 주제별 정리

### 기존 API 개발 워크플로우는 분산과 복잡성으로 인해 반복적 비효율이 발생함

- API 엔드포인트는 주로 코드 내(IDE 또는 AI 코딩 에이전트)에서 별도로 구현해야 함
- Postman 등 별도의 도구를 통해 테스트 컬렉션과 API 사양을 추가 생성해야 했음
- 컬렉션, 테스트, 스펙, 실제 코드가 각기 다른 위치와 도구에 흩어져 통합 관리가 어려움
- CI(Continuous Integration)에 테스트 연동을 또 별도로 처리해야 하며, 전체 워크플로우가 단일화되지 않음
- 각각의 리소스가 버전관리 시스템(Git 등)에서 분리되어 있어 변경 내역 추적 및 협업이 비효율적
- AI 에이전트나 최신 개발 프로젝트에서 API 엔드포인트와의 상호작용이 빈번하나, 관리와 자동화의 한계 명확

### Postman의 대규모 업데이트로 하나의 AI 네이티브 API 플랫폼이 등장함

- Postman은 최근 'AI 네이티브 API 플랫폼'으로 대규모 업데이트를 단행함
- 사용자(AI 에이전트 등)가 API와 관련된 전 과정을 Postman 내에서 일관된 경험으로 처리 가능해짐
- Git-네이티브 기능을 전면에 내세워 API 리소스 관리 패러다임 전환 유도

### Postman의 Git-네이티브 기능이 개발자 경험을 완전히 변화시킴

- Postman 컬렉션, 스펙, 모의(Mock), 테스트 등의 리소스를 YAML 파일로 코드베이스(로컬 폴더)에 직접 병합 가능
- 각 리소스는 읽기 쉬운 YAML 포맷으로 관리되어 유지·보수가 쉬워짐
- 데스크탑 Postman 앱에서 이 파일들을 직접 불러와 수정하거나, 클라우드로 2-way 동기화해 실시간 협업 지원
- 어떤 변경이든 git diff를 통해 바로 확인·추적할 수 있고, 기존 코드와 동일하게 버전관리 적용

### 파일 통합으로 인해 API 사양·테스트의 버전 추적이 가능해져 협업 및 품질관리가 강화됨

- API 컬렉션, 엔드포인트, 테스트 등 모든 변경사항이 git commit 이력으로 남음
- 팀원 간의 리뷰, 변경 승인, 롤백 등 개발자의 일반 협업 플로우에 자연스럽게 합류
- 분산·중복 없이 코드 수정과 동기화가 가능해 전체 API 라이프사이클 관리 효율 증가

### Postman 에이전트 모드가 전체 API 라이프사이클 자동화를 현실화함

- 모든 리소스가 단순 파일로 존재하므로 Postman의 에이전트 모드가 자동화와 통합관리 핵심이 됨
- 사용자가 자연어(Plain English)로 API 요청·생성을 기술하면, 에이전트가 코드베이스 분석 → 라우트 검색 → 스펙·테스트·모의 생성까지 자동 처리
- 예시: Flask 앱에 대해 "컬렉션 생성"을 평문으로 입력하면 라우트와 테스트, 사양, 모의까지 한 번에 생성
- 모든 작업 결과가 리뷰 가능한 git 커밋으로 기록됨

### AI 모델 및 외부 도구 연동을 통한 문맥파악 및 자동화 수준을 높임

- 컬렉션 생성 방식에서 Claude, GPT, 자체 호스트 모델 등 다양한 AI 엔진을 선택 가능
- GitHub, Sentry, Linear 등 외부 협업·모니터링 툴의 컨텍스트를 가져와 보다 정확한 API 관리 지원
- 인프라나 툴체인에 종속되지 않고 유연한 확장성 제공

### 동일한 API 컬렉션이 개발·테스트·CI 환경에서 완전히 호환되어 환경별 중복작업이 사라짐

- 로컬에서 돌아가는 API 컬렉션/테스트가 그대로 CI 파이프라인(Pre-commit, GitHub Actions 등)에서 실행 가능
- 환경에 따라 테스트 또는 사양을 따로 변환하거나 중복 생성할 필요 없음
- 일관된 테스트 환경 및 자동화 파이프라인 구현이 쉬워짐

### Postman의 업데이트가 개발자 생산성과 협업 효율을 대폭 향상시킴

- Git-네이티브 통합, 자연어·AI 기반 자동화, 리소스 통합 관리로 근본적인 생산성 개선
- 테스트, 스펙, 모의 환경 구축·유지에 드는 시간과 반복노력을 최소로 단축
- 허브 링크를 통해 Postman과 실제 GitHub 예제 저장소 참조 방법도 안내됨

### 영상에서 Git-네이티브 Postman 사용 예시가 단계별로 시연됨

- 데스크탑 앱에서 컬렉션·엔드포인트 파일을 직접 관리하고, 클라우드 연동으로 동기화 과정을 보여줌
- 한 엔드포인트에 대한 테스트를 수정하고 곧바로 Git에서 diff를 실행함으로써 버전 추적 예시 제시
- 자연어로 컬렉션 생성 요청 후, 에이전트가 Flask 앱 라우트 스캔 → 스펙·테스트·모의 파일 생성 과정을 실시간으로 시연
- 생성된 모든 변경사항이 git commit으로 남아 리뷰 가능함을 확인

### Postman의 혁신적 기능으로 API 개발·관리가 코드와 동일 수준으로 체계화됨

- API 관련 모든 리소스가 코드처럼 파일로 관리되고, git을 통한 협업과 버전관리가 기본 워크플로우가 됨
- 반복적인 수동작업이나 환경별 중복처리 등 기존의 비효율이 자연스럽게 해소됨
- AI와 자동화 기능 접목, 리소스 통합, 파일 중심 철학으로 개발자 경험이 크게 진화함
