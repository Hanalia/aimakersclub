## *RAG 2.0: 에이전틱 RAG와 지식 그래프의 결합 (무료 템플릿 소개)* 핵심 요약

- 영상 제목은 "RAG 2.0: 에이전틱 RAG와 지식 그래프의 결합 (무료 템플릿 소개)"임
- 최근 RAG(검색 증강 생성) 전략 중 가장 강력했던 두 가지, 즉 에이전틱 RAG와 지식 그래프 방식을 결합한 새로운 AI 에이전트를 만든 과정을 소개함
- 이 에이전트는 주어진 질문이 어떤 정보를 필요로 하는지 스스로 판단해 최적의 데이터 소스(벡터 DB 또는 지식 그래프)를 선택하여 검색함
- 백엔드에는 Postgres DB와 PG Vector 확장 기능을 활용하고, 서버리스 플랫폼 네온(Neon)을 기반으로 벡터 데이터베이스를 구현
- 문서는 AI 산업 내 빅테크 기업들의 이니셔티브와 상호협력 정보를 포함하며, 임베딩되어 벡터 DB와 동시에 지식 그래프(Graffitti, Neo4j 활용) 형식으로 저장
- 질문 내용에 따라 단일 기업에 관한 것은 벡터 DB에서, 기업 간 관계성은 지식 그래프에서 답을 찾음
- 에이전트는 어떤 도구를 활용해 답변했는지도 알려줌(예: vector search 또는 graph search)
- 필요시 시스템 프롬프트 조정 또는 명시적으로 벡터 DB와 지식 그래프를 모두 활용하게 할 수도 있음
- 이 에이전트의 구조와 개발 과정, 직접 적용법을 본인의 전체 영상(Cloud Code 이용)에서 무료 템플릿과 함께 안내함

## 세부 요약 - 주제별 정리

### 다양한 RAG 전략 중 에이전틱 RAG와 지식 그래프가 가장 효과적임을 발견함

- 최근 몇 개월간 다양한 RAG(검색 증강 생성) 전략을 깊이 연구하고 실험함
- 경험적으로 두 가지 전략, 즉 에이전틱 RAG(Agentic RAG)와 지식 그래프(Knowledge Graphs)가 반복적으로 탁월한 결과를 보임
- 이 두 전략을 결합할 때 정보 검색 및 추론 능력이 크게 향상됨
- 반복되는 실험 끝에 가장 강력한 AI 에이전트 프로토타입을 구축함

### 에이전트가 질문의 성격에 따라 정보원 선택과 추론을 자동으로 수행함

- 새롭게 개발한 AI 에이전트는 질문에 따라 어디에서 정보를 찾아야 하는지 스스로 판단함
- 질문에 최적화된 데이터 소스를 선택함으로써 답변의 정확도와 효율성을 높임
- 인간의 개입 없이도 사용자의 의도에 맞게 벡터 DB 또는 지식 그래프를 활용할 수 있음
- “이 에이전트가 정말 멋지다. 한 번 보라”며 데모에 대한 기대감을 표시함

### 벡터 데이터베이스 구현은 Postgres와 PG Vector, Neon 서버리스로 이루어짐

- 벡터 DB로는 Postgres와 PG Vector 확장 기능을 사용함
- 데이터 저장 플랫폼은 서버리스 Postgres 플랫폼인 Neon 기반임
- 문서들은 적절한 크기로 나누어 벡터 임베딩 처리가 완료됨
- “Super awesome serverless PostgreSQL platform called Neon”이라고 직접 언급함

### 문서는 AI 산업 내 빅테크 기업 정보 및 협업관계에 초점을 맞춤

- 샘플 문서에는 구글, 아마존, 앤쓰로픽 등 주요 빅테크 AI 이니셔티브 정보가 포함됨
- 각 기업의 AI 사업 및 상호 협력 현황에 대한 데이터가 다수 존재함
- 임베딩 작업을 거친 후 이 문서를 그대로 데이터베이스에 저장하여 활용함
- 실제 정보 예시: “Amazon is where Anthropic hosts their models.”

### 문서는 지식 그래프 형식으로도 Graffiti와 Neo4j에서 저장, 관리됨

- 동일한 정보는 Graffiti와 Neo4j로 지식 그래프에도 저장됨
- 정보 간의 관계성을 명확히 구조화(예: “A 기업과 B 기업의 파트너십”)
- 에이전트가 관계형 질의를 수행할 수 있도록 설계됨
- 이렇게 함으로써 복잡한 상호작용을 단순 검색보다 더 효과적으로 처리함

### 질문 유형에 따라 에이전트가 벡터 DB와 지식 그래프를 스스로 오가며 검색함

- 단일 기업의 정보(예: “구글의 AI 이니셔티브는?”)는 벡터 데이터베이스에서 검색
- 실제 예시로 구글 관련 정보를 묻자 벡터 DB에서 답변이 반환됨
- 에이전트는 답변과 동시에 어떤 도구(벡터 DB)를 사용했는지 명시적으로 기록함
- 반면 기업 간의 관계(예: “Amazon과 Anthropic의 파트너십”)에 관한 질문에는 지식 그래프에서 검색
- 이 경우에도 ‘graph search’를 사용했다고 에이전트가 직접 안내함

### 시스템 프롬프트나 명시적 요청을 통해 양쪽 데이터 소스 동시 활용도 가능함

- 필요하다면 시스템 프롬프트를 조정해 두 소스를 모두 동원할 수 있음
- 또는 직접 “벡터 DB와 지식 그래프를 모두 사용해 검색하라”는 식으로 에이전트에 명령 입력 가능
- 에이전트가 두 데이터 소스에서 정보를 가져와 통합된 답변을 출력함
- 이를 통해 복합적이고 신뢰도 높은 답변도 제공 가능함

### 모든 과정이 무료 템플릿으로 제공되어 누구나 활용 가능함을 강조함

- 영상에서 데모한 에이전트는 무료 템플릿 형태로 바로 사용 가능함
- 전체 제작 과정과 클라우드 코드 활용법을 소개하는 전체 영상 링크를 별도로 안내함
- 누구든 해당 과정을 따라 하며 에이전트를 구축하고 실습할 수 있도록 유도함
- Cloud Code와 같은 개발 지원 툴 정보를 추가적으로 안내함

### 개발의 핵심 도구와 구축법을 깊이 다루는 별도의 전체 영상이 있음을 고지함

- 본 요약 영상은 기능과 핵심 구조의 데모에 집중함
- 실제 개발 과정과 구체적 구축법은 전체 영상을 통해 세부적으로 안내한다고 명시함
- 전체 영상에서는 템플릿 사용법, Cloud Code 환경에서의 세팅, 실습 예시 등이 포함됨

### 에이전틱 RAG와 지식 그래프 융합이 차세대 AI 정보 탐색의 혁신임을 시사함

- 두 접근방식의 결합으로 정보 검색·추론 능력이 극대화됨을 시각적 데모와 예시로 입증함
- 사용성, 확장성, 답변의 신뢰도 측면에서 매우 유용한 템플릿임을 강조함
- 제공된 예시와 연동 도구, 데이터 구조가 차세대 AI 에이전트 설계에 중요한 사례임을 드러냄
