## *MCP는 안전하지 않다 - 그 이유와 해결 방법* 핵심 요약

- MCP 서버는 많은 이점이 있지만, 심각한 보안 취약점도 함께 가지고 있음
- 주요 보안 위협 세 가지는 민감한 데이터 노출, 과도한 접근 권한, 감사 및 추적 불가임
- 대부분의 MCP 서버는 실제로 필요한 것보다 훨씬 많은 접근 권한을 사용자에게 부여함
- GitHub MCP 서버 예시에서, 개인 액세스 토큰을 평문으로 저장해야 해서 보안에 취약함
- OAuth 인증을 도입해도 문제 해결이 안 되는 이유는, 여전히 권한이 과도한 토큰을 장기간 로컬에 저장해야 하기 때문
- MCP 서버 사용 시 권한을 세분화해서 설정하거나 팀원에게 필요한 권한만 부여하는 것이 불가능함
- 기존 MCP 서버의 보안 문제를 해결하려면 안전한 구성, 감사 로그, 세분화된 권한 설정이 가능한 엔터프라이즈급 플랫폼 필요
- Teleport는 MCP 서버 각각을 보호 및 관찰 가능한 리소스로 만들고, 연결 시 사용자의 권한만 담긴 JIT(Just-In-Time) 토큰 발급
- Teleport에서는 MCP 클라이언트 설정에 비밀 키를 저장하지 않아도 되고, 서버 연결 및 사용도 매우 간편함
- Teleport의 감사 로그 기능으로 MCP 서버의 모든 호출 내역을 추적·분석할 수 있음
- 역할 기반 권한 관리(RBAC)를 통해 서버별로 세부 권한을 부여·관리 가능함
- Teleport 팀과 협업을 통해 이 문제를 깊이 연구하고 실제 솔루션을 제시함

## 세부 요약 - 주제별 정리

### MCP 서버는 편리하지만 심각한 보안 위험을 내포하고 있음

- MCP 서버(Managed Control Plane Servers) 사용이 점점 늘고 있으며, 많은 개발자와 팀이 생산성 향상을 위해 채택 중임
- 그러나 업계에서는 MCP 서버가 도움이 되는 동시에 심각한 보안 위험을 갖는다는 우려가 큼
- 영상에서는 이 위험이 실체적으로 무엇이며, 어떤 대안이 있는지 구체적으로 설명함

### MCP 서버의 주요 보안 위협 세 가지는 민감 정보 노출, 과도한 권한, 감사 부재임

- 첫 번째 위험: 민감한 데이터(예: 접근 토큰)가 서버나 로컬 환경에 노출됨
- 두 번째 위험: 서버가 실제 필요 이상으로 과도한 접근 권한을 제공함
- 세 번째 위험: MCP 서버 사용 내역에 대한 감사(audit) 및 추적이 불가하여 문제가 발생해도 원인 파악이 힘듦
- 모든 위험 요소가 실제 운영 환경에서 심각한 보안 이슈로 연결될 수 있음

### GitHub MCP 예시에서 평문 토큰 저장의 치명적 문제가 드러남

- 영상에서는 GitHub MCP 서버를 예로 들어 위험성을 설명함
- 영상 제작자는 개인 접근 토큰을 평문으로 콘피그 파일에 저장한 뒤, 이 내용을 보여주기 위해 일부를 검열함
- 실무에서는 키나 토큰이 평문으로 저장되는 것은 매우 위험함
- 이는 제3자가 파일 접근 시 모든 시스템 권한이 유출될 수 있음을 의미함

### OAuth 인증도 과도한 권한 및 저장 문제를 해결하지 못함

- 일반적으로 보안 강화를 위해 OAuth 같은 인증 방식을 도입함
- 그러나 OAuth를 적용해도 MCP 클라리언트 환경에 권한이 과도한 'Long-lived token'이 그대로 저장됨
- 토큰 만료 전까지 노출 위험이 지속되며, 실제로는 보안에 큰 개선 효과가 없음

### MCP 서버 사용 시 권한 세분화 및 팀 내 권한 분배가 사실상 불가함

- MCP 서버를 팀원과 공유할 때 읽기 권한만 주는 등의 세분화된 권한 설정이 없는 점이 문제
- "내가 만든 MCP를 팀원에게 공유하되, 특정 제한만 주고 싶다"는 시나리오가 불가능
- 이는 잘못된 접근·남용으로 이어질 수 있음
- 영상 제작자가 공개적으로 다양한 해결책을 찾으려 했으나, 현실적으로 마땅한 솔루션이 없음을 언급

### 엔터프라이즈급 플랫폼이 필요한 이유와 Teleport 도입 배경

- 근본적인 해결책은 보안 구성을 관리하고, 감사 추적, 세밀한 권한 관리를 지원하는 엔터프라이즈급 플랫폼이 필요함
- 영상에서 언급한 Teleport는 이러한 요구사항을 충족함
- MCP 서버 연결을 Teleport라는 플랫폼을 통해서 중계하는 방식으로 보안 취약점 해결

### Teleport는 MCP 서버를 안전하게 보호하고 사용 과정을 완전히 감사·추적 가능하게 함

- Teleport 내에서 MCP 서버 각각을 '보호되고 가시적인 리소스'로 관리함
- 사용자가 MCP에 연결할 때, Teleport 인증 절차를 선행하고 해당 계정 권한에 맞는 JIT(Just-In-Time) 토큰 발급
- 결과적으로 MCP 클라이언트 설정에 더 이상 비밀 키나 토큰을 저장할 필요가 없음

### Teleport 도입으로 설치 및 연결 과정이 오히려 더 쉽고 간편해짐

- Teleport 플랫폼을 통하면, 서버를 애플리케이션에 연결하는 과정이 기존 대비 복잡하지 않음
- 오히려 설정 과정이 단순화되어 관리 효율성이 증가함
- 기존 GitHub MCP 서버도 Teleport 하에서 정상적으로 동작하며, 모든 트래픽과 쿼리가 Teleport를 통해 중계됨

### 감사 로그, 권한 분석, 이상 탐지를 Teleport가 지원함

- Teleport의 감사 로그 기능을 통해 MCP 서버의 모든 호출 내역 추적 가능
- 사용된 MCP 서버, 도구, 각 호출의 개별 파라미터까지 확인할 수 있음
- 플랫폼 내 에이전트가 데이터 분석 및 이상 탐지(Outlier Detection)도 지원
- 실제 예시로, 자신이 호출한 MCP call을 Claw Desktop에서 로그로 조회할 수 있음을 시연

### 역할 기반 접근 제어(RBAC)로 서버별·툴별 세부 권한 부여가 자유로움

- Teleport는 역할 기반 권한 관리(RBAC) 기능을 지원
- 관리자 페이지에서 MCP 서버별로 다양한 도구와 권한을 사용자 그룹별로 편하게 지정할 수 있음
- 예: 특정 팀원은 읽기만 허용, 특정 도구만 사용할 수 있도록 설정 가능

### Teleport와의 협업을 통해 MCP 보안 문제 솔루션을 실제로 구현하고 검증함

- 영상 제작자가 Teleport 팀과 직접 협업하여 여러 보안 이슈의 근본 원인과 개선책을 연구함
- Teleport 솔루션을 실제로 사용하며 그 효과와 편리성을 사례로 제시
- 영상 마지막에는 Teleport 웹사이트 링크를 공유함
