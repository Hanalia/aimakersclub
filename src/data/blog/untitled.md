## *AI 에이전트를 능률적으로 강화하는 가장 단순한 방법(스킬)* 핵심 요약

- Anthropic이 최근에 공개한 '스킬(skills)' 기능은 AI 분야에서 매우 중요한 진보로 간주됨
- 스킬의 핵심 강점은 사용 방식이 매우 직관적이고 단순하다는 점에 있음
- 복잡한 도구와 기능을 한 번에 주입하는 기존 방식(MCP 서버 등)은 에이전트의 컨텍스트 윈도우를 쉽게 과부하 상태로 만들어 비효율을 초래함
- Anthropic의 스킬은 '점진적 공개(progressive disclosure)' 원리를 적용, 필요 시점에만 필요한 정보를 단계적으로 제공함
- 초기 시스템 프롬프트에는 각 스킬의 '설명(description)'만 간단히 포함되어 에이전트가 능력을 인지할 수 있도록 유도함
- 예시로, PDF 처리 스킬의 경우 "PDF 관련 작업을 할 수 있음" 정도만 알려주고, 실제 요청이 있을 때만 필요한 세부 정보를 가져옴
- 사용자가 실제로 특정 스킬을 요청할 때 에이전트는 skill.md 파일을 불러와 해당 스킬의 전체 사용법과 지침을 파악함
- skill.md는 스킬의 두 번째 컨텍스트 레이어로, 필요한 경우 추가 문서를 참조하는 등 2~3단계에 걸쳐 정보를 점진적으로 확장함
- 이러한 구조는 불필요한 리소스 낭비를 피하면서, 에이전트가 실제 필요한 순간에만 기능을 효율적으로 습득하고 수행하게 함
- Anthropic의 '더 단순할수록 더 좋다'는 철학이 AI 체계의 실질적 성능 향상에 기여함이 강조됨

## 세부 요약 - 주제별 정리

### Anthropic의 스킬 기능 공개는 AI 발전의 결정적 진전을 의미함

- 최근 Anthropic이 '스킬(skills)' 기능을 AI 에이전트에 적용하여 새롭게 공개
- 해당 기능은 최근 AI 진화에서 가장 중요한 변화 중 하나로 소개됨
- 스킬의 혁신성은 도구 추가 방식을 '단순화'하는 데 초점을 맞춤
- Anthropic이 추구하는 "더 단순할수록 더 좋다(The simpler the better)"라는 모토를 잘 반영함
- 복잡하고 불필요한 정보 주입을 지양하고, 필수적인 핵심만 제공하여 AI의 생산성과 효율성 극대화 추구

### 기존 방법(MCP 서버 등)으로는 에이전트 컨텍스트 과부하 문제가 컸음

- 전통적으로 AI 에이전트에게 수십 개의 도구(tool), 대화 내역, 시스템 프롬프트 등 다양한 정보를 한 번에 투입
- MCP(Multi-tool Context Provision) 서버 방식은 사용하지 않을 도구까지 무조건 적재하는 문제가 있음
- 이런 방식은 에이전트 컨텍스트 윈도우를 쉽게 포화상태로 만들어, 실제 대화와 추론에 필요한 정보 제공을 어렵게 함
- 정보 과잉으로 에이전트가 혼란을 느끼거나 성능이 저하되는 현상 빈번
- 근본적으로는 "필요 없는 도구까지 항상 포함하는 것이 비효율적"이라는 문제를 내포

### 스킬 기능의 핵심 원리는 '점진적 공개(progressive disclosure)'에 있다

- 스킬 방식의 본질은 필요한 순간에만 그 기능의 세부 정보를 에이전트가 점진적으로 습득하게 하는 데 있음
- 모든 기능을 미리 알려주지 않고, 필요 시 능력을 '발견(discover)'하도록 설계
- 이로써 컨텍스트 윈도우 사용량과 에이전트 혼란을 대폭 줄임
- 점진적 공개로 인해 실제 활용이 있을 때만 관련 설명, 매뉴얼, 문서를 단계적으로 추가함
- 전체 시스템의 반응성과 유연성 증가, 실제 사용성 개선에 도움

### 에이전트는 시스템 프롬프트의 '스킬 설명'만을 처음에 인식함

- 스킬 구현 시, 시작 단계에서 시스템 프롬프트에는 오로지 각 스킬의 간단한 설명(desription)만 포함
- 예시: "PDF를 처리하는 능력이 존재함" 정도의 간단한 안내만을 에이전트에게 제공
- 에이전트는 이런 설명을 통해 자신이 어떤 능력을 가질 수 있는지만 먼저 파악
- 이 방식은 실제 사용 안 되는 정보를 미리 context에 쌓지 않게 해서 효율성 극대화

### 사용자가 실제로 요청하면 skill.md 등 구체 파일로 관련 컨텍스트가 추가됨

- 사용자가 예를 들어 "PDF를 요약해줘"라고 요구할 때, 에이전트는 그제야 해당 스킬을 활성화
- skill.md 파일이 불러와져 해당 스킬 작동 방식, 사용 규칙 등 구체적인 실행 지침을 세부적으로 습득
- skill.md는 각 스킬별 상세 설명서 역할을 하며, Anthropic이 제공하는 모든 공식 스킬에서 공통적으로 사용됨
- 커스텀 스킬 구현 시에도 동일한 구조 및 접근법을 적용 가능

### 컨텍스트 추가는 단계별(레이어별)로 이루어져 불필요한 정보 낭비를 막음

- 1단계: 시스템 프롬프트에 스킬 기본 설명만 포함(최소한의 정보)
- 2단계: 실제 요청이 있을 때 skill.md 등의 파일을 추가(중간 정보)
- 3단계: skill.md가 더 많은 세부 문서나 자료를 참조하면 이에 따른 추가 문서도 점진적으로 context에 적재
- 이런 2~3단계 레이어 시스템이 스킬별 필요한 정보만 순차적으로 공개, context 인식 효율을 최적화함

### Anthropic의 스킬 시스템 철학은 AI의 실제 문제 해결력을 강화함

- 근본적으로 '단순함'을 강조한 Anthropic 철학과 개발 전략이 AI 실행 시스템에 직접 적용됨
- 효율적인 컨텍스트 관리로, 실제 작업 수행 시 에이전트가 더 빠르고 명확하게 의도를 파악하고 실행 가능
- 점진적 정보 공개는 사용성과 품질에 큰 영향을 주는 핵심 원리로 기능함
- Anthropic식 스킬 구조와 철학은 커스텀 AI 에이전트 구축에도 활용할 수 있음을 시사
