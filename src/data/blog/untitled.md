## *내 AI 메모리가 모든 툴을 따라다닌다!* 핵심 요약

- 영상 제목은 “내 AI 메모리가 모든 툴을 따라다닌다!”로, 사용자가 다양한 AI 툴에서 일관된 메모리 시스템을 사용할 수 있는 방법을 소개함
- Cole Medin은 기존에 Claude Code의 ‘second brain’에서 개인 및 프로젝트 관련 모든 정보를 저장·활용하는 메모리 시스템을 구축해 생산성을 높였다고 밝힘
- 하지만 Claude Code 외에도 Codex, Pi와 같은 다양한 AI 코딩 도구를 활용하게 되면서, 단일 툴에 국한된 메모리 시스템에 불편을 느낌
- 이런 문제를 해결하기 위해 ‘Walrus and Memory’라는 새로운 솔루션을 사용함
- Walrus는 한 툴에 종속되지 않고, 탈중앙화된 방식으로 사용자 본인이 소유하고 암호화된 형태의 메모리시스템을 제공함
- 시작이 매우 쉽고, 홈페이지에서 제공하는 프롬프트 한 줄만 복사해 AI 코딩 도구에 붙여넣으면 곧바로 메모리 시스템이 세팅됨
- SDK를 통해 원하는 앱에 메모리 시스템 연동도 가능함
- 저장·분석·검색 등 주요 기능을 시연하며, Claude Code와 Pi 등 여러 에이전트에서 동일한 정보에 접근 및 활용 가능함을 데모로 입증함
- 각 툴에서 Walrus를 통해 대량 텍스트에서 주요 정보 추출·저장, 검색 등 메모리 관리 원활히 수행
- 저장된 메모리는 사용자 소유이므로 접근 권한 및 위임키 생성 등 다양한 옵션 활용 가능

## 세부 요약 - 주제별 정리

### Claude Code 기반 ‘Second Brain’ 메모리 시스템은 생산성을 높이지만 타 툴 사용에 제약을 줌

- Cole Medin은 Claude Code를 활용해 자신과 진행 중인 프로젝트에 관한 모든 정보를 ‘second brain’에 저장하고 있음
- 이 메모리 시스템은 메모리화된 정보 덕분에 높은 생산성을 경험하게 해줌
- 하지만 최근 Codex, Pi처럼 다양한 AI 코딩 보조도구들을 병행 사용하기 시작하며, 각각 강점을 살리기 위해 작업을 나눠 맡김
- 문제는 현재의 메모리 시스템이 Claude Code에 종속되어, 다른 툴 사용시 동일한 정보를 공유·활용할 수 없다는 점임
- 이로 인해 여러 도구 간 정보 일관성 및 접근성 저하 발생

### AI 도구 상관없이 메모리를 자유롭게 넘길 수 있는 솔루션이 필요했음

- 다양한 AI 코드 어시스턴트(Claude Code, Codex, Pi 등)에서 동일한 정보를 이어가며 작업하려면, 메모리 시스템의 ‘도구 독립성’이 절실함
- Cole은 장기간 이런 니즈에 부합하는 솔루션을 찾아왔음

### ‘Walrus and Memory’는 툴 종속적이지 않은 탈중앙화·암호화 메모리 시스템을 제공함

- ‘Walrus and Memory’라는 툴을 최근 발견, 요구사항에 완벽히 부합한다고 소개
- Walrus 메모리 시스템은 특정 서비스나 회사 인프라가 아닌, 완전한 탈중앙화 구조로 동작함
- 사용자의 메모리는 암호화되어 보관되며, 실제로 해당 정보의 소유권과 통제권은 전적으로 본인에게 있음
- 데이터가 특정 회사의 서버에 저장되지 않기에 프라이버시와 보안이 강화됨

### Walrus를 시작하는 방법은 매우 간단하며, 한 줄 프롬프트로 즉시 연동 가능함

- Walrus 도입 과정을 시연: 홈페이지 링크에서 프롬프트 한 줄 복사 후, AI 코딩 어시스턴트(Claude, Codex 등)에 입력만 하면 됨
- 별도 복잡한 설치나 사전 환경설정 없이, 단 한 번의 프롬프트로 바로 메모리 서버가 연동됨
- 개발자를 위한 SDK도 제공되어, 다양한 앱이나 워크플로에 자유롭게 통합 가능

### MCP 서버가 Walrus 메모리의 핵심으로 동작하며, 다양한 기능을 제공함

- 프롬프트 입력 후 Walrus가 제대로 설치되었는지 확인 위해 MCP 서버 연결 여부를 챗 인터페이스(예: Claude Code)에서 확인 가능
- 주요 기능: 기억시킬 정보 저장, 긴 텍스트에서 주요 항목 추출(분석 저장), 메모리 내 정보 검색 등 지원
- 예를 들어, 사전 작성된 메모리에서 특정 프로젝트의 기술 스택 등 핵심정보를 즉시 조회 가능

### Walrus를 통해 ‘자연스러운 대화 흐름’ 중에도 필요시 자유롭게 메모리 사용이 가능함

- 실제 사용 예시로, 긴 텍스트 블록을 붙여넣고 ‘Walrus로 이 내용을 기억하라’고 명령
- Walrus가 MCP 서버를 통해 텍스트에서 주요 기억 요소(20가지 팩트 등) 추출 후, 암호화해 저장
- 서로 다른 AI 도구(Claude Code, Pi 등)에서 같은 질문(특정 프로젝트의 tech stack 등) 요청 시, Walrus 메모리에서 정확한 정보 반환됨
- Claude에서는 상세히, Pi에서는 좀 더 간결하나 본질적으로 동일한 답변을 확인할 수 있음

### 분산형 메모리는 사용자 소유·암호화·접근권한 제어 등 다양한 활용성을 보장함

- 저장된 모든 기억 정보 자체가 암호화 및 사용자 소유임을 강조
- 각 프로젝트나 상황별로 delegate key 등을 생성, 시스템 또는 타인에게 다른 접근 권한 부여 가능
- 실질적으로 어떤 AI 툴이든 메모리 레이어로 Walrus를 연결해 쓸 수 있어 활용성이 높음

### Walrus는 빠르고 쉽고, 무료로 체험 가능하여 바로 써볼 가치가 있음

- 홈페이지 링크 및 시작 프롬프트 등 즉시 체험 가능한 구성을 제공
- 빠른 설치, 별도 비용 없이 누구나 쉽고 안전하게 AI 기억 시스템 구축이 가능함
- 영상 전체적으로 실제 데모를 통해 ‘도구 경계 없는 AI 기억 서비스’의 효율성과 편의성을 시각적으로 전달함
