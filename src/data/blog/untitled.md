## *AI 에이전트를 가장 간단하게 고도화하는 방법 (Skills)* 핵심 요약

- Anthropic의 ‘Skills’ 기능은 최근 AI 분야에서 가장 중요한 발전 중 하나로 평가됨
- ‘Simple is better’(단순함이 최선)이라는 Anthropic의 철학을 체현하는 기능임
- 기존 에이전트는 도구, 대화 이력, 시스템 프롬프트 등 많은 정보를 한꺼번에 전달받아 컨텍스트 윈도우(문맥 창)를 쉽게 초과할 수 있음
- MCP 서버 방식은 수많은 도구를 대화 초기에 전부 에이전트에게 제공하는데, 이는 꼭 필요하지 않은 도구까지 불필요하게 포함되어 효율성이 떨어짐
- Skills의 가장 큰 특징은 ‘점진적 공개(Progressive Disclosure)’ 메커니즘으로, 에이전트가 필요한 순간에만 해당 기능(스킬)에 접근하도록 함
- 대화 초기에 에이전트에 전달되는 것은 각 기능(스킬)의 ‘설명(description)’이 전부임
- 사용자가 특정 작업(예시: PDF 처리)을 요청하면 그때서야 에이전트는 그 스킬의 구체 지침이 담긴 ‘skill.md’ 파일을 참고하여 실제 실행을 준비함
- ‘skill.md’ 파일은 각 스킬의 조작 방법 및 세부지침이 담긴 메인 문서로, 상황에 따라 그 안에서 추가 참고문서까지 점진적으로 참조 가능
- 이처럼 Skills 방식은 에이전트의 컨텍스트 오염 없이 원하는 순간에만 정보와 도구를 적시에 제공, 성능과 확장성을 높임
- Anthropic 뿐 아니라 실제 커스텀 프로젝트 구현 사례들도 이 방식을 적극 채택함

## 세부 요약 - 주제별 정리

### Anthropic의 Skills 기능은 단순함을 극대화한 AI 핵심 발전으로 평가됨

- Anthropic이 최근 공개한 Skills 기능은 AI 분야에서 매우 중요한 발전으로 언급됨
- Cole Medin은 “Skills 자체가 얼마나 아름답게 단순한지, 그리고 이것이 왜 중요한가”를 영상 핵심 테마로 삼음
- Anthropic은 제품 개발 철학 자체가 ‘단순함이 최고의 설계’임을 강조함

### 기존 에이전트 도구 활용 방안은 컨텍스트 윈도우 과부하 문제를 야기함

- 에이전트에게 대화 이력, 시스템 프롬프트, 다양한 도구 정보 등을 미리 입력하면 문맥 창(context window)이 쉽게 과포화됨
- 정보가 많을수록 에이전트는 혼란스럽거나 처리 성능이 저하될 수 있음
- 특히 여러 도구/기능을 한번에 주입하는 MCP 서버 방식은 이 문제의 전형적인 예시로 제시됨
- Cole “도구를 다 쓰지 않더라도 모두 제공하는 건 비효율적이고 ‘나쁜’ 접근법”이라고 직접 언급

### Skills의 ‘점진적 공개’ 원리가 핵심이며 기존 방식과 근본적으로 다름

- 점진적 공개(Progressive Disclosure)는 한 번에 모든 정보를 제공하지 않고, 필요할 때만 차근차근 노출시킴
- 기존 MCP 방식과 달리 처음에는 간단한 설명만 에이전트에게 전달 (도구의 정체, 할 수 있는 일 정도)
- 사용자가 관련 요청을 해야만 이후 단계로 넘어감

### 기능(스킬) 설명만으로 초반 컨텍스트 오염 없이 에이전트에 능력을 부여함

- 시스템 프롬프트 단계에서는 각 스킬의 ‘설명(description)’만 짧게 전달
- 예시: “PDF 처리 능력을 가지고 있다”고만 명시
- 필요하지 않은 방대한 지침이나 실행 예시 등은 이 단계에 포함되지 않음

### 실제 필요 요청이 있을 때 에이전트는 ‘skill.md’ 파일로 본격 작동에 돌입함

- 예를 들어 사용자가 PDF 관련 작업을 요청하면, 그 시점에만 에이전트가 skill.md 파일을 열람
- skill.md는 해당 기능을 실제로 수행하는 데 필요한 상세 절차, 명령어, 옵션 등을 구체적으로 안내

### skill.md가 스킬 구동의 메인 엔트리 포인트로 작동함

- Anthropic의 모든 Skills 구조에서 skill.md 파일이 ‘주문서’ 같은 역할을 담당
- 커스텀 구현 사례에서도 같은 구조가 적용됨
- skill.md를 통해 2단계 컨텍스트가 열리고, 실제 작업 흐름이 이곳을 따라간다

### 점진적 공개의 3단계 설계로 정보 유입과 사용이 단계별로 조절됨

- 1단계: 스킬 설명(description)만 에이전트에 할당 (최소 정보)
- 2단계: 실제 요청 시 skill.md 파일이 로딩되어 상세 지침 제공
- 3단계: skill.md에서 추가로 필요시 외부 문서(참고 자료 등) 참조 (심화/확장 정보)
- 이렇게 함으로써, 점진적으로 문맥에 더 많은 정보를 유동적으로 주입 가능

### 점진적 정보 주입은 컨텍스트를 효율적으로 유지하며, 확장성도 동시에 달성함

- 단순한 설명 → 세부 지침 → 추가 문서 등 단계별 정보 확대 방식
- 각 스킬은 실제 사용 수요가 있을 때에만 전체 데이터를 에이전트가 참고
- 이로 인해 컨텍스트 윈도우 낭비 및 혼란 최소화, 시스템 전체의 성능도 확보 가능

### 실제 구현 사례에서도 커스텀 스킬 구조와 Anthropic 모델이 동일하게 접목됨

- 영상 내 커스텀 구현 사례는 Anthropic 공식 Skills와 동일하며, 누구나 확장해 응용할 수 있음
- 다양한 스킬을 점진적으로 추가/관리하면서도 컨텍스트 오염 없이 안전, 효율적으로 조작 가능함

### Skills는 ‘필요할 때만 정보 제공’ 원칙으로 AI 에이전트 확장성 및 안정성 문제를 동시에 해결함

- 모든 도구를 미리 주입하는 방식 대신, 사용 시점에 ‘능력’을 점진적으로 개방하는 것이 핵심
- 이로써 AI 에이전트의 과부하 문제 없이 다양한 기능을 유연하게 제공 가능
- Anthropic의 철학적 모토와 실질적 기술 설계가 완전히 일치하는 중요한 사례로 제시됨
