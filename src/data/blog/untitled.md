## *스택드 디프(차곡차곡 변경) 방식은 AI 코드 리뷰의 효율과 정확성을 극대화함* 핵심 요약

- AI 코드 리뷰는 현대 코드 검증에 필수 요소로, 매우 강력한 도구임을 강조함
- 기존 방식의 풀 리퀘스트(PR)는 하나의 코드 변경마다 승인을 기다리며 반복하는 비효율이 있음
- 스택드 디프(workflow)는 작은 코드 변경들을 차례대로 쌓아서 큰 기능을 완성하는 방식임
- 각 작은 변경이 서로 위에 쌓이면, 승인 대기 없이 다음 단계로 바로 진행 가능해 개발 속도가 빨라짐
- 이 방식은 각 단계별로 AI가 정확하고 집중적으로 코드를 리뷰할 수 있어, 리뷰의 질이 높아짐
- Graphite라는 플랫폼이 스택드 디프에 최적화된 CLI와 AI 코딩 어시스턴트, 대시보드를 제공함
- Graphite의 직관적인 온보딩(가입 시 데모)은 스택드 디프 생성 전 과정을 단계별로 안내함
- 커맨드라인에서 3개의 풀 리퀘스트를 차례로 생성하고, 각 PR을 대시보드로 개별 관리 및 AI 리뷰 가능
- 각 PR은 개별 브랜치에서 병합되며, 최종적으로 전체 변경이 메인 브랜치(main)로 병합됨
- Claude(클로드) AI로 각 PR을 리뷰한 결과, 하나의 대규모 PR을 리뷰할 때보다 훨씬 세밀하고 효과적임
- Git 운영방식 자체는 매우 표준적이나, 스택드 디프 + AI 리뷰의 결합이 혁신적인 생산성 향상을 가져옴

## 세부 요약 - 주제별 정리

### AI 코드 리뷰는 검증 프로세스에서 필수적이며, 그 효능이 입증됨

- AI를 활용한 코드 리뷰가 단순 옵션이 아니라, 본인의 검증 프로세스에서 반드시 포함시키는 필수 단계임을 언급함
- AI 코드 리뷰 도구의 강력함을 강조하며, 개발자의 코드 검증 부담을 크게 줄여줌
- AI의 도움으로 실수나 보안 취약점, 코드 스타일 위배 등 다양한 문제를 빠르게 발견 가능함

### 기존 풀 리퀘스트 방식은 승인 대기와 반복의 비효율성 문제를 가짐

- 일반적인 풀 리퀘스트 작업 흐름은 다음과 같음: 변경 사항 생성 → 승인 대기 → 작업 반복
- 각 단위 변경마다 반드시 팀원(혹은 리더)의 승인을 받아야 하므로 병목 현상 발생
- 코드 병합과 리뷰 속도가 늦어지고, 개발 프로세스 전체의 효율성이 저하됨

### 스택드 디프 방식은 코드 변경을 작은 단위로 분할하여 연속적으로 추가하는 구조임을 설명함

- 하나의 큰 변경을 여러 개의 작은 변경(디프)으로 쪼개어, 차곡차곡 위에 쌓아올리는 방식이 스택드 디프임
- 각 작은 변경(디프, diff)은 그 이전 작업 위에 쌓이며, 전체적으로 하나의 큰 기능이나 피처를 이루게 됨
- 승인 대기 없이 연속적으로 각 단계의 코드를 작성하고 커밋할 수 있어 작업이 중단되지 않고 연결됨

### 스택드 디프는 AI 코드 리뷰와 결합 시 각 변경에 대한 집중적인 리뷰 결과를 가져옴

- 모든 작은 변경을 독립적으로 AI가 검토함으로써, 각 단계별로 초점을 맞춘 상세 피드백을 얻을 수 있음
- 대규모 풀 리퀘스트 하나를 리뷰하는 것보다, 여러 단계로 나누어 세밀하게 AI 피드백을 받을 수 있음
- 버그나 스타일 위배, 논리적 결함이 어느 단계에서 유입되었는지 추적이 용이함

### Graphite 플랫폼은 스택드 디프 적용과 관리를 대폭 간소화함

- Graphite는 스택드 디프 사용을 위한 CLI(Command Line Interface), 대시보드, AI 코드 리뷰 기능을 통합 제공함
- GitHub의 공식 CLI 도구처럼, 익숙한 명령어 스타일을 유지하면서도 스택드 디프 생성 및 관리에 특화됨
- 대시보드를 통해 전체 스택의 흐름, 개별 풀 리퀘스트의 상태, AI 리뷰 결과 등을 직관적으로 관리할 수 있음

### Graphite의 온보딩 데모는 스택드 디프 도입 과정을 단계별로 안내하여 초보자도 쉽게 활용 가능함을 보여줌

- 가입 시 제공되는 데모는 사용자의 첫 번째 스택드 디프를 만드는 전 과정을 따라가며 안내
- 커맨드라인에서 최초 변경(디프) 생성 및 풀 리퀘스트 작성 → 두 번째, 세 번째 풀 리퀘스트를 이어서 생성하는 과정을 체험하게 함
- 실제로 3개의 풀 리퀘스트가 서로 위에 쌓이며, 실습을 통해 스택 구조 전체를 쉽게 이해할 수 있도록 돕는다

### 실전 예시에서 여러 풀 리퀘스트가 스택 구조로 연결되며 관리의 편의성이 강조됨

- 커맨드라인에서 3개의 풀 리퀘스트를 연속적으로 생성하여 보여줌
- 대시보드에서 각 PR(풀 리퀘스트)를 개별적으로 살피고 부가 기능을 활용하는 예시를 제공함
- 각 풀 리퀘스트는 별도의 기능 브랜치에서 준비되어, 서로를 기반으로 위에 쌓임
- 언제든지 하나의 대시보드에서 전체, 개별 PR을 확인 및 병합(Merge) 가능함

### AI 코드 리뷰는 각 디프별로 독립적이고 집중적인 검토를 제공함

- Claude(클로드) AI를 활용하여 각 풀 리퀘스트를 개별 리뷰함
- 모든 변경 사항이 작은 단위로 쪼개져 있기 때문에, AI가 특정 변경의 영향과 오류를 더 정확히 포착함
- 대규모 PR을 한 번에 리뷰한 결과와 비교해, 스택드 디프 방식에서 훨씬 세밀하고 피드백의 질이 높아짐

### 최종적으로 표준 Git 워크플로우를 영리하게 활용하여 생산성과 관리 효율을 극대화함

- 핵심 워크플로우: 각 인크리멘털(점진적) 변경을 독립 브랜치에 올려 차례로 머지(Merge)
- 모든 변경을 준비되었을 때 메인 브랜치(main)로 한 번에 병합함
- 코드는 결국 표준 Git 브랜치/풀 리퀘스트로 관리되므로, 기존 Git 사용 경험이 충분히 적용됨
- GitHub 리포지토리 접근 없이 모든 관리가 Graphite 내에서 이뤄질 수 있음

### 스택드 디프와 AI 리뷰 결합은 코드 품질 및 팀 전체 워크플로우에 혁신적 변화를 유도함

- 스택드 디프 + AI 조합은 복잡하고 방대한 코드 리뷰 과정을 세밀하고 체계적으로 분산함
- 개발 공정의 속도, 코드 품질, 수정 용이성이 모두 크게 개선됨
- Graphite 플랫폼처럼 워크플로우 최적화 도구를 함께 사용할 때 효과가 극대화됨
- 영상 마지막에서 상세한 튜토리얼 영상 안내 링크를 추가로 공유함
