## *스택드 디프(누적 변경) 방식이 AI 코드 리뷰 효율을 극대화함* 핵심 요약

- AI 기반 코드 리뷰는 매우 강력하며, 코드 검증의 필수 단계로 간주됨
- 효율성과 집중도가 높은 코드 리뷰를 위해 ‘스택드 디프(stacked diffs)’ 워크플로우가 효과적임
- 기존의 풀리퀘스트는 변경 후 승인 대기 시간을 반복적으로 요구함
- 스택드 디프는 작은 단위의 변경을 누적시켜 하나의 대규모 피처를 완성하고, 각 단계별 승인 대기를 줄임
- 각 작은 변경이 쌓여 전체 기능을 완성하며, 각 레이어별로 AI 리뷰가 가능해 집중적인 피드백이 제공됨
- 스택드 디프를 위해 Graphite 플랫폼을 사용, GitHub CLI와 유사한 커맨드라인 툴과 대시보드 제공
- Graphite에는 AI 코드 어시스턴트가 내장되어, 풀리퀘스트 리뷰를 손쉽게 진행할 수 있음
- 공식 데모 과정을 통해 사용자에게 직관적으로 스택드 디프 생성 및 관리 방법을 안내함
- 각 단계별 풀리퀘스트가 서로를 기반으로 쌓여가며, 대시보드에서 쉽게 병합 및 관리 가능
- AI가 각 단계별 작은 변경에 집중해 리뷰할 때, 대규모 풀리퀘스트 한 번에 리뷰하는 것보다 훨씬 뛰어난 결과(정확성, 효율) 도출


## 세부 요약 - 주제별 정리

### AI 리뷰는 집중적이고 체계적인 검증 단계로 간주됨

- AI를 활용한 코드 리뷰 기능은 코드 검증 과정에서 매우 중요하게 여겨짐
- AI 리뷰가 코드의 오류, 개선사항을 빠르게 발견하는 데 큰 도움을 줌
- 영상 제작자는 AI 리뷰를 “필수(validation 중 must)”라 강조함
- 일반적인 코드 리뷰보다 AI 도입 시 리뷰 품질과 속도가 상승함

### 스택드 디프(누적 변경) 워크플로우는 작은 단위 변경으로 큰 기능을 만듦

- 기존 방식: 한 번에 중요한 변경을 하고, 풀리퀘스트(Pull Request) 승인을 기다렸다가 더 작업하는 방식 반복
- 스택드 디프 방식: 작은 코드 변경을 여러 번 진행, 각 변경은 이전 변경 위에 쌓여 전체 기능을 구성함
- 작은 incremental changes(점진적 변경) 각각이 유기적으로 결합됨
- 각 단계 사이에 승인 대기 시간이 필요 없어 개발 속도 향상
- 스택드 디프가 누적될수록, 각 변경사항의 맥락과 상호 영향 관계가 명확해짐

### AI는 스택드 디프별로 세밀한 피드백을 제공하며 리뷰 품질을 높임

- 각 단위 변경(디프, diff)은 AI가 개별적으로 집중해 리뷰함
- 소규모 incremental diff에 대한 리뷰이므로, 대규모 풀리퀘스트보다 AI가 맥락 이해 및 오류 탐지가 쉬움
- 레이어별 피드백을 수렴, 전체 코드 흐름의 문제를 미리 발견하는 데 유리함

### Graphite 플랫폼은 스택드 디프 적용 및 AI 리뷰를 직관적으로 지원함

- Graphite는 스택드 디프 기반 개발 워크플로우를 위한 전용 플랫폼임
- GitHub CLI와 유사한 커맨드라인 툴을 제공, 스택드 디프 생성 및 관리에 특화됨
- 별도의 대시보드에서 전체 변경 사항과 풀리퀘스트를 일목요연하게 관리 가능
- AI 코딩 어시스턴트(agent)가 내장되어, 사용자는 AI와 함께 각 변경사항을 리뷰할 수 있음

### Graphite의 입문 데모는 스택드 디프 생성 과정을 명확하게 안내함

- Graphite 신규 가입자는 데모 과정을 통해 스택드 디프 실습이 가능함
- 첫 번째 단위 변경(디프) 생성 → 풀리퀘스트 생성 → 이후 단계별 추가 PR 작성 흐름을 자연스럽게 안내
- 해당 Demo는 커맨드라인 중심이며, 각 단계별 커맨드 및 실행 예시가 명확함

### 실제 PR 생성 및 관리가 Git & GitHub CLI 워크플로우와 거의 동일함

- 각 incremental change는 피처 브랜치에서 관리, 병합은 상위 브랜치로 누적
- 완성된 전체 기능은 main 브랜치로 최종 합쳐짐
- 모든 변경사항은 표준 Git 흐름에 기반해 관리됨 (PR 쌓기, 머지, 리뷰 반복)
- 각 단위별 PR은 “표준적인 풀리퀘스트”처럼 보임

### Graphite 대시보드에서 풀리퀘스트 진행상황 전체를 쉽고 빠르게 파악·제어 가능

- 전체 PR 현황, 머지 여부, 리뷰 상태 한눈에 확인 가능
- 대시보드 내에서 모든 PR 머지 및 관리가 가능, GitHub 레포지토리에 직접 들어가지 않아도 됨
- 각 풀리퀘스트 상태 전환이 쉽고 직관적임

### AI 리뷰를 단계별로 적용할 때와 대규모 PR 한 번에 적용할 때 결과가 극명하게 다름

- 각 단위(3개의 작은 PR)마다 Claude 코딩 모델을 적용해 리뷰를 진행
- 모든 작은 PR에 집중된 피드백을 받으며, 전체 코드를 한꺼번에 보는 것보다 오류 탐지 및 리뷰 품질이 월등함
- “결과가 정말로 다르다(insane)”고 강조, 스택드 디프 활용의 큰 장점을 시사

### 예시 실습 화면과 Graphite 기능을 영상 내에서 실제로 시연함

- 커맨드라인에서 3개의 PR을 차례로 쌓아가는 과정을 직접 시연
- Graphite 대시보드에서 각 PR의 상태 및 AI 리뷰 활용 모습을 구체적으로 보여줌
- PR별 병합, 리뷰, 관리가 쉽다는 것을 실제 화면 예시를 통해 설명

### 스택드 디프 방식과 Graphite 활용법 안내 자료를 영상 설명란에 추가 공유함

- 영상 내에서 “아래에 링크된 영상을 참고하라”고 안내
- Graphite와 스택드 디프 활용법을 담은 추가 자료를 영상 외에서 안내함
- 해당 링크를 통해 스택드 디프 적용 및 AI 리뷰 상세 방법을 학습할 수 있도록 안내함
