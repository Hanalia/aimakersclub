## *MCP는 안전하지 않다 - 그 이유와 해결 방법* 핵심 요약

- MCP 서버는 많은 사용자들이 편리하다고 평가하지만 심각한 보안 위험을 안고 있음.
- 주요 위험은 민감 정보 노출, 과도한 접근 권한 부여, 그리고 사용 내역의 추적 불가 세 가지임.
- 실제 예시로 GitHub MCP를 설정할 때 개인 엑세스 토큰을 평문으로 저장해야 하며, 이는 보안상 매우 취약함.
- OAuth 방식도 문제를 해결하지 못하며, 여전히 기기에 과도한 권한을 가진 토큰들을 저장하는 구조임.
- MCP 클라이언트에서도 세밀한 권한 부여가 불가능하여, 특정 동료에게 읽기 전용 접근만 줄 수도 없음.
- 취약점 해결의 실제적 방법은 안전한 구성, 기록, 세분화된 권한부여를 제공하는 엔터프라이즈급 플랫폼이 필요함.
- Teleport 플랫폼은 MCP 서버를 보호된 자원으로 관리하며, 사용자별로 적시에 필요한 최소 권한만 부여하는 토큰 시스템을 적용함.
- Teleport 사용 시 클라이언트 구성에 비밀번호, 토큰 등 비밀 정보가 저장될 필요 없고 연결 절차도 간단함.
- 모든 MCP 요청은 Teleport를 프록시로 거쳐 보안, 감사 추적이 가능해지며, 서버, 도구, 사용 파라미터까지 모두 기록·분석 가능함.
- 역할 기반의 접근 제어 및 보안 설정 변환 등도 플랫폼 내에서 직관적으로 설정할 수 있음.

## 세부 요약 - 주제별 정리

### MCP 서버는 편의성 대비 심각한 보안 취약점이 존재함을 지적함

- 많은 사용자가 MCP(Machine Control Protocol) 서버를 활용하며 그 효용성을 높이 평가함.
- 그러나 대다수 사용자가 동시에 보안 위험에 대한 우려도 언급함.
- 실제로 MCP의 보안 문제는 단순 오해가 아니라 실질적 사례와 구조적인 한계에서 비롯됨.

### MCP 서버가 노출하는 주요 위험 세 가지를 구체적으로 설명함

- 첫 번째 위험은 민감 데이터(예: 접근 토큰, 키)의 노출임.
- 두 번째는 사용자에게 실제 필요 이상으로 과도한 서버 접근 권한이 부여되는 것임.
- 세 번째(및 연쇄적 위험)는 서버 사용에 대한 감사(추적) 및 가시성이 전혀 없다는 점임.
- 이 세 가지 위험은 서로 연관되어 심각한 보안 위협으로 작용함.

### GitHub MCP 예시를 통해 평문 토큰 저장의 문제점을 직접 시연함

- 실제 설정 화면에서 개인 접근 토큰을 검은색 박스로 가린 뒤 예시를 보여줌.
- "이런 작업 자체가 필요 없었어야 한다"라며, 토큰/키를 평문으로 저장해야만 하는 구조를 지적.
- OAuth 인증도 구조적 한계는 동일하며, 긴 수명의 토큰이 너무 높은 권한을 가진 채로 기기에 남는다는 점 강조.
- 결과적으로 키/토큰 보관방식 자체가 근본적인 취약점임.

### MCP 클라이언트는 반드시 세밀한 권한 제어 기능이 필요함을 강조함

- MCP 클라이언트 이용 시, 실질적으로 GitHub의 모든 기능에 접근 가능한 권한이 부여됨.
- 특정 동료에게 일부(예: 읽기 전용) 권한만 줄 수 있는 제어 기능이 부재함.
- 제한적 권한 부여(least privilege)가 어렵고, 실제 팀 협업 환경에서는 치명적 결함임을 설명.

### 기존 솔루션으로는 근본적인 MCP 보안 문제 해결이 불가함을 언급함

- 여러 방법을 전 세계적으로 직접 찾고 시도했으나 뚜렷한 대안이 없었음을 언급.
- 단순 설정, 권한설정, 감사추적의 부재 문제는 엔터프라이즈급 플랫폼의 개입 없이는 해결될 수 없다고 결론지음.

### Teleport 플랫폼이 MCP 보안 문제에 대한 실질적 해답임을 제안함

- Teleport는 MCP 서버를 개별적으로 보호된·관찰 가능한 자원으로 취급함.
- Teleport에 로그인 시, 계정 권한만큼의 just-in-time 토큰이 발급되어 과도한 권한 부여 우려를 해소함.
- 클라이언트 측에는 토큰이나 비밀정보를 저장할 필요 없이 안전하게 동작함.

### Teleport의 동작 방식과 MCP 통합 예시를 구체적으로 보여줌

- 기존과 동일하게 GitHub MCP 서버를 백엔드로 사용하지만, 모든 요청이 Teleport를 프록시로 통과함.
- 사용자는 이전보다 더 쉽고 안전하게 서버와 애플리케이션을 연동할 수 있음.
- MCP 클라이언트 설정에서 민감 정보가 필요 없으므로 오히려 더 안전하고 관리가 단순해짐.

### Teleport의 감시·분석 및 역할 기반 접근통제가 가능함을 상세하게 설명함

- Teleport 플랫폼의 감사 로그(audit log)에서 MCP를 통한 모든 호출 내역을 실시간으로 조회할 수 있음.
- 호출한 서버, 사용 도구, 전달된 인수(파라미터)까지 세밀하게 기록되고, 분석 에이전트를 통해 이상 징후 탐지가 가능함.
- 역할 중심의 제어 패널에서 각 MCP 서버별 상세 권한 구성, 툴 접근 범위 설정, 사용자별 정책 부여가 인터페이스를 통해 바로 가능함.

### MCP 보안 문제 해결을 위해 엔터프라이즈 플랫폼 도입 및 Teleport 활용을 적극 권장함

- 영상 제작자 입장에서 Teleport의 실질적 효과에 만족감을 표명하며, 링크 공유 및 협업 감사 인사로 마무리함.
- 엔터프라이즈 환경에서만이 아니라, 민감 데이터 취급이 필요한 다양한 상황에서 보안적 대안임을 구체적으로 시사함.
