## *MCP는 안전하지 않다 - 그 이유와 해결 방법* 핵심 요약

- MCP(Machine Control Protocol) 서버는 뛰어난 기능으로 인식되지만, 동시에 심각한 보안 위험 요소를 가지고 있음
- 주요 보안 문제는 민감한 데이터 노출, 과도한 접근 권한 부여, 그리고 서버 사용 기록의 부재임
- Github MCP 예시에서 패스워드나 토큰 등 중요 키를 평문으로 저장해야 하는 상황이 발생하며, 이는 취약점으로 작용함
- OAuth 역시 루트 문제가 해결되지 않아, 여전히 지나치게 많은 권한을 가진 장기 토큰이 저장됨
- MCP 서버는 사용자의 개별 권한 설정이 불가능해, 모든 사용자에게 동일하게 광범위한 접근 권한이 부여됨
- 실질적인 권한 관리, 감사로그 제공, 세분화된 접근 제어를 하기 위해서는 엔터프라이즈급 플랫폼이 필요함
- Teleport 플랫폼은 MCP를 보호·감시 가능한 리소스로 취급하고, 연결 시 계정별로 적합한 임시 토큰을 발급해 보안을 강화함
- 구성 파일에 비밀정보 저장이 필요 없으며, 기존 MCP 서버 기능은 그대로 유지됨
- 모든 요청이 Teleport 프록시를 거치도록 하여, 접근 기록과 실시간 모니터링·감사 기능 제공
- Teleport의 감사 로그에서 각 요청의 대상 서버와 툴, 개별 파라미터까지 추적·분석 가능
- 이상행위 탐지 에이전트와 역할 기반 권한 관리가 Teleport에서 바로 구현 가능

## 세부 요약 - 주제별 정리

### MCP 서버는 뛰어난 기능 뒤에 심각한 보안 취약점을 내포함

- MCP 서버는 편리성 및 접근성으로 개발·운영 환경에서 자주 선택됨
- 하지만 민감한 데이터가 쉽게 노출되는 구조를 기본적으로 가지고 있음
- 실무 환경에서는 누구나 필요 이상으로 많은 권한을 부여받는 경우가 많음
- 서버 사용 기록을 추적하거나 감사할 수 없는 상황이 보편적임

### 민감한 데이터가 평문으로 저장되고 있으며 권한 범위가 과도하게 넓음

- Github MCP 예시에서, 사용자는 개인 접근 토큰(PAT) 값을 반드시 평문으로 저장해야 함
- 영상 내에서 직접 PAT 값을 마스킹한 예시를 보여주며 이것이 위험하다고 강조함
- "이 정도 접근성을 제공하면서도, 평문 토큰을 저장하지 않아도 되는 방법이 필요하다"고 지적
- OAuth도 궁극적인 해결책이 되지 못하며, 장기 유지되는 과도한 권한의 토큰 문제를 남김

### MCP 서버는 사용자별로 권한을 나눌 방법이 사실상 없음

- MCP 서버는 사용자별로 접근 권한(예: 읽기 전용, 쓰기 제한 등)을 세분화하거나 제한할 수 없음
- 예시로, 팀원에게 MCP 서버 접근 권한을 일부만 주려 해도 방법이 없어 모두에게 모든 권한을 주게 됨
- 영상 제작자는 "관련 솔루션을 찾기 위해 광범위하게 조사했으나 실용적 대안이 없었다"라고 언급

### 실질적인 해법은 엔터프라이즈급 플랫폼에서만 제공됨

- 보안 설정, 감사 추적, 권한 세분화를 포괄적으로 다루려면 엔터프라이즈급 별도 플랫폼이 필요함
- 이 조건을 충족하는 플랫폼 사례로 Teleport를 직접 데모와 함께 소개함

### Teleport는 MCP 서버를 보호되고 추적 가능한 리소스로 감싸 보안을 대폭 강화함

- Teleport 내에서는 각 MCP 서버가 보호 및 감시 가능한 리소스로 등록됨
- 사용자는 Teleport에 로그인함으로써 임시로 할당된 ‘Just-in-time’ 토큰을 통해 접속
- 이 토큰은 계정별 권한만을 부여해 불필요한 접근을 방지함

### Teleport 도입 시 MCP 서버의 구성 파일에 비밀정보가 남지 않음

- MCP 클라이언트의 설정에서 더 이상 키나 토큰 등 비밀정보를 저장할 필요가 없음
- 서버 연결 및 애플리케이션 통합 방식 또한 복잡하지 않으며, 기존 MCP 서버의 기능은 유지됨
- Teleport가 중계(proxy) 역할을 하여 모든 요청을 보호 관찰하게 됨

### 모든 요청과 접근 행위는 Teleport 감사 로그에서 상세 추적 가능

- Teleport 플랫폼 내 ‘Audit Log’ 기능을 통해 모든 MCP 호출 내역을 확인할 수 있음
- 실제로 영상에서는 ‘Claw Desktop’에서 발생한 MCP 호출을 Audit Log에서 찾아 세부 내역을 확인
- 호출한 서버, 사용된 툴, 전달 파라미터 등 모든 정보를 체크할 수 있음

### 이상행위 감지와 세밀한 접근 권한 관리는 Teleport에서 손쉽게 구현됨

- Teleport에는 각종 요청 기록을 분석해 이상치(outlier) 행위를 탐지해주는 에이전트가 내장됨
- MCP 서버 관련 ‘롤(Role)’을 지정해, 각 롤이 어떤 MCP 서버와 툴에 접근할 수 있는지를 세밀하게 조정 가능
- 역할 기반 설정 화면에서 직관적으로 사용자 접근권한을 관리할 수 있음

### 실제 사용 경험과 협업을 통해 Teleport의 보안성을 실증함

- 영상 내에서 직접 Teleport의 설정, 감사로그, 권한 관리 화면을 시연함
- Teleport 팀과 협업해 영상에 등장한 보안 문제들을 실제로 재현 및 해결하여 신뢰성을 강조함

### 마무리 안내: Teleport 관련 추가 정보는 영상 설명란에서 확인 가능

- Teleport에 대한 자세한 내용은 영상 설명란의 링크를 통해 접속할 수 있다고 안내
- Teleport와의 협업에 특별히 감사를 표하며 영상이 마무리됨
