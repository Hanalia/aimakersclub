## *MCP는 안전하지 않다 - 그 이유와 해결 방법* 핵심 요약

- MCP(Managed Control Plane) 서버는 많은 사람들이 훌륭하다고 평가하지만, 동시에 심각한 보안 위험을 수반함
- MCP의 주요 보안 문제는 ① 민감한 데이터 노출, ② 필요 이상으로 많은 권한 부여, ③ 감사 및 추적 불가능이라는 세 가지로 요약됨
- 예시로 GitHub MCP에서는 설정 파일 내에 개인 액세스 토큰과 같은 중요 키를 평문으로 저장해야 하는데, 이는 매우 위험함
- OAUTH 인증 방식 등도 장기 토큰 저장 문제를 근본적으로 해결하지 못하며, 여전히 너무 많은 권한을 부여하게 됨
- MCP 클라이언트에서는 사용자별로 세분화된 권한 설정 기능이 없어, 모든 사용자가 과도한 권한을 갖게 됨
- 문제를 근본적으로 해결하려면, 엔터프라이즈급 플랫폼에서 보안 설정, 감사 기록, 권한 제어를 MCP에 추가적으로 적용해야 함
- Teleport라는 플랫폼은 MCP 서버를 보호 가능한 자원으로 만들고, 접속 시 계정별로 필요한 권한만 가지는 즉시 생성 토큰(Just-In-Time Token)을 부여함
- 이 방식으로 MCP 설정에서 비밀번호 등 민감 정보 노출이 사라지고, 모든 요청과 활동을 감사 로그로 추적할 수 있음
- Teleport의 대시보드에서 서버, 도구, 파라미터 수준까지 모든 MCP API 호출을 세밀하게 확인 및 분석 가능
- Zero Trust 기반의 접근 제어와 역할 기반 권한 설정을 플랫폼 내에서 손쉽게 구성할 수 있음

## 세부 요약 - 주제별 정리

### MCP 서버는 뛰어나지만 본질적인 보안 위험을 동반함

- MCP 서버는 강력하고 유용하다 평가받지만, 동시에 심각한 보안 위협을 내포하고 있다는 의견이 많음
- 특히 광범위하게 받아들여진 MCP 서버일수록 사용자 및 조직의 민감한 데이터 보호에 취약할 수 있음
- 영상의 주요 질문: "실제 위험이 무엇이며, 이를 어떻게 해결할 수 있는가?"에 초점을 맞춤

### MCP 환경에서 가장 큰 세 가지 보안 문제는 명확하게 존재함

- MCP 관련 대표적인 위험 3가지는 다음과 같음
  - 민감한 데이터(Access Token 등) 노출
  - 사용자가 실제 필요한 범위를 초과하는 범용적 권한 부여
  - 서버 사용 내역의 미감사 및 관리 미비로 인한 추적 불가성
- 이러한 문제는 단순한 설정 변경으로 해결되지 않는 구조적 문제임을 강조

### GitHub MCP 예제로 민감한 키의 평문 저장 위험성이 드러남

- 실제 데모에서 GitHub MCP configuration 파일 내에 토큰 정보가 평문으로 저장되어 노출 요구 발생
- 영상 제작자는 프라이버시 보호를 위해 자신의 액세스 토큰을 가렸으나, 일반적으로 평문 저장이 불가피함을 지적
- 이는 해킹, 내부자 위험, 단순 실수 등 다양한 경로로 정보가 유출될 수 있음을 의미
- OAUTH 인증 절차를 사용하더라도, 실제로는 여전히 장기적 토큰(롱리브 토큰)을 머신에 저장하는 방식이므로 근본 해결책이 될 수 없음

### MCP 클라이언트는 권한 관리의 허점으로 모든 기능이 노출되는 문제가 있음

- 예시: GitHub MCP 클라이언트에 접속하면 사용자는 상상할 수 있는 거의 모든 GitHub 작업 권한을 얻게 됨
- 팀원에게 MCP 서버를 공유해 줄 때도, 읽기 권한(Read)만 제한적으로 부여할 수 있는 수단이 없음
- 영상 제작자는 이 문제의 해법을 찾기 위해 여러 솔루션을 광범위하게 조사했다고 언급

### 엔터프라이즈 플랫폼(예: Teleport)만이 실제로 MCP 보안 환경을 구축할 수 있음

- 발견 결과, MCP 보안 문제의 실질적 해결은 엔터프라이즈급 보안 관리를 덧입혀야만 가능함
- 구체적으로 요구되는 해결책:
  - 안전한 구성 관리 (secure configuration)
  - 세밀한 권한 제어 (granular permissioning)
  - 사용 내역 감사(auditability)와 추적(traceability) 기능
- Teleport 플랫폼은 이 모든 기능을 기본 제공함

### Teleport는 MCP 서버를 보호 가능한 자원으로 만들고, 연결 프로세스를 완전히 바꿈

- Teleport에 등록된 MCP 서버는 보호·관찰 가능한 자원(resource)으로 관리됨
- 접속 시 Teleport에 로그인하면, 계정 권한에 맞는 JIT(Just-In-Time) 토큰만 발급되어 불필요한 권한 부여가 차단됨
- 사용자는 더 이상 MCP 설정에 민감 정보(토큰/키 등)를 저장할 필요가 없으며, 연결 절차도 동일하거나 오히려 더 쉬움
- 모든 MCP 요청은 Teleport 프록시를 통해 처리되어 보안성이 극대화됨

### Teleport를 통한 MCP 요청은 완전한 실시간 추적과 감사를 가능하게 함

- Teleport 플랫폼 내 Audit Log(감사 로그) 기능에서 모든 MCP API 호출 기록을 조회 가능
- 예시: 클로 데스크톱(Claw Desktop)에서 발생한 개별 요청까지 즉시 확인 가능
- 서버, 도구, 각 호출 파라미터까지 세부적으로 기록
- Teleport 내 ‘에이전트’ 기능을 이용하여 비정상 패턴(Outlier) 탐색 등의 세밀한 보안 분석 가능

### Zero Trust 접근 제어와 역할 기반 권한 관리가 직관적으로 제공됨

- Teleport의 Zero Trust Access 메뉴를 활용하면, 역할(Role) 단위로 MCP 서버 접근 권한을 설정/관리할 수 있음
- 역할별로 서버 내에서 사용할 수 있는 도구(tool)를 직관적으로 지정하고 조정 가능
- 팀 내 사용자 별로 필요한 리소스와 기능만 부여할 수 있도록 설정 가능

### Teleport 활용이 MCP 보안 위험을 실질적으로 해결할 수 있음을 강조함

- 영상 마지막에서 Teleport 플랫폼 링크를 공유하며, MCP 관련 보안 문제 해결 수단으로 강력하게 추천함
- Teleport팀과 협업하여 실제 이슈를 깊이 있게 분석하고 영상에 담았음을 명시

### MCP 보안 관리는 단순 설정이나 인증 수준이 아닌, 플랫폼 차원에서 접근해야 함을 시사

- 기존의 인증 방식(OAUTH 등)이나 권한 관리 없이 운영하는 방식의 한계를 명확하게 설명
- MCP를 안전하게 운영하려면 반드시 보안, 감사, 권한 관리가 통합된 플랫폼을 기반으로 접근해야만 위험을 방지할 수 있음을 설득력 있게 전달
