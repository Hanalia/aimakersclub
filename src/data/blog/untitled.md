## *MCP 서버의 큰 문제점과 그 해결책* 핵심 요약

- MCP(Multi-Component Protocol) 프로토콜이 AI 코딩 분야에서 인기가 증가하면서, 토큰 소비와 맥락 소실(context rot)이라는 심각한 결함이 드러나기 시작함
- 영상에서는 MCP의 문제를 실제 예시를 통해 자세히 설명함
- 주요 문제는 여러 MCP 서버를 동시에 사용할 때 각 서버가 AI 에이전트에게 자신의 기능을 설명하기 위해 수천 개의 토큰을 소모한다는 점임
- 토큰 소비 문제 때문에 한 세션에서 여러 MCP 서버를 함께 쓰는 것이 현실적으로 매우 비효율적임
- 이는 실제 AI 코딩 작업에서 빠르게 심각한 문제로 발전할 수 있음을 예제와 함께 강조함
- 이러한 문제의 해결책으로 Anthropic에서 최근 공개한 '에이전트 스킬(agent skills)'이라는 기능을 소개함
- '에이전트 스킬'은 AI 에이전트가 API 엔드포인트와 상호작용할 스크립트를 직접 생성하고, 해당 스크립트 사용법이나 즉석 코드 작성 등 자동화된 기능을 활용할 수 있게 해줌
- 이 접근 방식이 MCP 프로토콜의 토큰 소모 및 맥락 소실 문제를 효율적으로 극복하는 핵심적인 해결책이라고 제시함
- 영상의 결론은 "이제까지 우리가 알던 MCP의 시대가 끝나는 것인가?"라는 질문으로 마무리됨
- 영상 전체에서 문제 제기와 솔루션 소개를 명확하게 구분하여, 기술적 세부사항을 강조함

## 세부 요약 - 주제별 정리

### MCP 프로토콜의 인기 증가로 인한 문제점이 명확해짐

- MCP는 AI 코딩 작업에서 널리 사용되는 프로토콜로, 최근 사용 빈도와 인기가 급증함
- 그러나 프로토콜 활용이 본격화되며 내재적인 결함이 더욱 뚜렷하게 드러남
- 대표적인 결함으로 토큰 소비 과다와 맥락 소실(context rot) 현상을 지적함
- 이 두 가지 문제는 현재 AI 및 LLM 관련 커뮤니티에서 매우 뜨거운 이슈로 부상 중임

### 토큰 소비와 맥락 소실이 MCP 서버 사용의 가장 큰 장애임을 예시로 제시함

- 영상 속 화자는 MCP 서버의 문제를 설명하기 위해 실제 환경 예시를 제시
- 예시는 ‘AI 코딩’에 사용되는 5개의 표준 MCP 서버를 동시에 사용하는 상황을 가정
- 각 MCP 서버는 자신의 능력과 제공 기능을 AI 에이전트에게 설명할 때 수천 개의 토큰을 요구함
- 이로 인해 여러 MCP 서버를 단일 세션에서 동시에 사용하는 것은 토큰 한도 내에서 실질적으로 불가능에 가까움
- ‘코딩 어시스턴트’와 협업 시 MCP 서버를 다중으로 조합하는 것은 매우 비효율적임을 강조
- 화자는 “나는 절대, 진짜로 절대 이렇게 다섯 서버를 한 세션에서 동시에 쓰지 않는다”며 한계점을 직접 언급

### MCP 문제의 근본적인 심각성을 구체적으로 시각화함

- 토큰 낭비와 맥락 소실 문제가 얼마나 신속하게 통제불능으로 이어질 수 있는지 ‘무서운 예시’를 통해 부각
- 여러 서버가 각자 자신을 설명하기 위한 반복적 토큰 소모가 만성적으로 발생함
- 이를 경험적으로 겪어본 이들은 실제 시스템 사용에서 큰 불편함을 체감함
- 현 시점에서 이 문제는 아직 일반적으로 널리 해결되지 않은 ‘치명적 결함’임을 영상 내 여러 부분에서 강조

### '에이전트 스킬'이 MCP의 문제를 근본적으로 해결할 대안으로 제시됨

- Anthropic이 최근 발표한 ‘에이전트 스킬(Agent Skills)’이 해결책으로 언급됨
- 이 기능은 AI 에이전트에게 아래와 같은 능력을 부여함:
    - API 엔드포인트와 상호작용하는 스크립트 생성
    - 자동화된 작업 지시 생성(instructions)
    - AI 스스로 즉시 코드 생성 및 실행
- MCP의 방대한 토큰 설명 부담을 한 번의 스크립트 및 명령어로 간결화할 수 있음
- 기존 방식과 달리 AI가 동적으로 필요한 코드만 생성·실행하므로 토큰 소모 문제를 크게 완화
- “코드 실행(code execution)의 정점”이라며 해당 기능의 강력함을 다시 한 번 강조

### MCP 프로토콜의 미래에 대한 의문과 전환점을 제시함

- 영상은 ‘MCP가 이제까지처럼 계속 사용될 수 있을 것인가?’라는 도전적 질문으로 마무리
- ‘에이전트 스킬’ 기반의 접근이 MCP 프로토콜의 종결점이 될 수 있음을 암시
- MCP 고유의 결함(토큰 소모, 맥락 소실)에 대응하는 ‘새로운 패러다임’의 필요성 제기
- 구체적인 결론을 내리지는 않지만, 업계와 개발자들에게 기술적 방향 전환의 시그널을 줌

### 영상 흐름이 문제 제기-구체 예시-솔루션 제안-미래 전망의 구조로 진행됨

- 서두에서 MCP의 결함을 명확히 식별하고, 직접적인 예시를 통해 체감시킴
- 기술적 문제와 사용자 입장(불편함/비효율성)을 모두 강조
- Anthropic의 신기술 ‘에이전트 스킬’을 현실적 대안으로 구체적으로 설명
- 마지막에는 기술적 혁신이 산업 전반에 미칠 잠재적 영향을 언급하며, 문제 해결에 그치지 않는 미래지향적 시사점까지 전달
