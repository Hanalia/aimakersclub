## *Postman이 Git 네이티브로 진화하며 API 개발 방식에 혁신을 가져옴* 핵심 요약

- 기존 API 작업 방식은 API 엔드포인트, 컬렉션, 테스트 작성 및 CI 연동이 각기 다른 도구와 위치에서 관리되어 매우 혼란스러움
- 코드, 명세(spec), 컬렉션, 테스트 파일이 서로 분리되어 버전 관리가 어려웠음
- Postman에서 최근 대규모 업데이트를 통해 모든 API 리소스 관리가 코드를 중심으로 통합 가능해짐
- YAML 파일 기반으로 컬렉션, 명세, 목(mocks), 테스트 등 모든 리소스를 코드베이스 내에 보관 가능
- Postman 데스크톱 앱, 로컬 코드베이스, Postman 클라우드 간에 양방향 동기화가 자동 지원됨
- 버전 관리(git diff) 및 변경 이력 추적이 코드베이스와 동일하게 가능해짐
- AI Agent 모드에서는 "영문 설명"만으로도 컬렉션, 명세, 테스트, 목 자동 생성·동기화 가능
- 다양한 AI 모델(Claude, GPT, 셀프 호스팅 모델) 및 외부 도구(GitHub, Sentry, Linear 등)와 연동 지원
- 동일한 컬렉션/테스트가 로컬, CI 파이프라인(Pre-commit, GitHub Actions 등)에서 환경 구분 없이 실행 가능
- 이러한 변화로 Postman은 AI 네이티브 API 플랫폼으로 거듭나 API 개발 전체 수명 주기 관리가 압도적으로 쉬워졌음

## 세부 요약 - 주제별 정리

### 기존 API 개발은 분산된 도구와 복잡한 워크플로우로 인해 비효율적이었음

- API 작업 시 개발자는 IDE(코드 에디터)에서 엔드포인트를 만들고, Postman 등 별도의 도구에서 컬렉션 및 테스트를 따로 생성했음
- 테스트들을 CI 환경에 적용하려면 또 추가적인 설정과 연동 작업이 필요했음
- API 명세(spec), 컬렉션, 테스트, 실제 코드가 위치와 관리 방식이 서로 달라 버전 통제가 어려움
- 여러 도구를 오가며 정보 동기화와 협업, 변경 이력 추적이 번거롭고 실수에 취약한 구조였음

### Postman 최신 업데이트로 모든 API 리소스를 하나의 코드베이스에서 관리할 수 있게 됨

- Postman은 최근 대규모 업데이트로 "Git 네이티브" 기능 도입을 발표함
- 컬렉션, 스펙(명세), 목(mock), 테스트 등 모든 리소스를 로컬 코드베이스 내 YAML 파일로 단순하게 관리 가능
- YAML 포맷의 파일이므로 읽기 쉽고, 유지보수와 버전 컨트롤 측면에서 매우 용이함
- Postman 데스크톱 앱에서도 바로 이 파일들을 불러와서 작업할 수 있도록 연동됨

### Postman Cloud와 로컬 환경 간의 실시간 양방향 동기화가 가능해짐

- 로컬 코드베이스와 Postman Cloud 간의 2-way sync(양방향 동기화)를 공식적으로 지원
- 개발자가 로컬에서 컬렉션 혹은 테스트 등 리소스를 수정하면, Postman 데스크톱 앱 혹은 클라우드 환경에도 자동 반영
- 반대로, Postman Cloud에서 진행한 변경도 다시 로컬 환경에 동기화되어 일관성이 보장됨
- 이로 인해 협업시 충돌 및 관리 이슈가 대폭 줄어듦

### git diff를 활용한 버전 관리 및 변경 이력 추적이 API 리소스에도 동일하게 적용됨

- 모든 API 리소스가 코드처럼 관리되어 git diff, 커밋, 브랜치, 머지 등 git의 모든 기능을 그대로 활용 가능
- 예를 들어, 테스트 케이스 파일이나 명세에서 변경이 발생하면 git diff로 어느 부분이 바뀌었는지 즉각 확인 가능
- 코드베이스의 나머지 부분과 테스트/스펙이 동시에 버전 컨트롤되어 이력 관리가 투명해짐
- 협업 중 변경 사항 리뷰, 롤백, Merge 등이 API 작업에도 손쉽게 적용됨

### “Agent Mode”에서는 영어 자연어로 API 컬렉션 및 테스트 등의 자동 생성을 할 수 있음

- Postman의 "Agent Mode" 활성화 시, 개발자는 영어로 필요한 API 리소스를 서술하기만 하면 됨
- 예: 플라스크(Flask) 앱에 대한 컬렉션을 생성하려고 할 때, "이 App에 맞는 컬렉션을 만들어줘" 등 영어 문장 입력
- Postman이 전체 코드베이스를 스캔하여 라우트(route)를 자동 탐색하고, 명세, 테스트, 목까지 한 번에 생성
- 생성된 모든 리소스는 자동으로 서로 연결되고, 일관성을 유지함

### AI 모델 선택과 다양한 외부 도구와의 연동 기능이 가능해짐

- AI 모델은 Claude, GPT, 셀프-호스팅 모델 등 여러 선택지 중에서 지정 가능
- MCP 서버 지원 등 엔터프라이즈급 연동도 내장됨
- Github, Sentry, Linear 등 이미 사용하는 생산성 도구와 연결시켜 더 많은 맥락(context) 정보를 불러올 수 있음
- API 리소스 생성·동기화·관리가 AI 및 외부 데이터와 긴밀히 통합

### CI 파이프라인, Pre-commit Hook, GitHub Actions 등 전 개발 단계에서 일관성 있게 테스트를 실행할 수 있음

- 로컬 환경에서 작성한 컬렉션 및 테스트들이 CI 파이프라인에서도 별도의 재작성 없이 그대로 동작
- Pre-commit hook, GitHub Actions 등 다양한 자동화 흐름에서 동일 컬렉션/테스트가 실행됨
- 환경 별 스크립트 분리, 코드 중복 없이 일괄적으로 QA가 가능해짐
- 테스트의 재작성, 환경별 수정 등이 제거되어 생산성이 크게 높아짐

### 변경사항은 리뷰 가능한 Git 커밋으로 남으며, 협업 효율성이 크게 개선됨

- Postman의 모든 API 리소스 변경은 일반 코드 커밋과 같이 git 커밋으로 남음
- 팀원들은 새로운 스펙/컬렉션/테스트 변경 내용을 리뷰, 코멘트, 승인, 롤백할 수 있음
- API 변경 히스토리도 소스코드와 동일한 수준의 추적 및 이력이 남음

### 실제 플라스크 앱 예시와 자동 리소스 생성 과정이 시연됨

- 예제로 Flask 웹앱에 대해 Postman이 코드를 분석해, 라우트 경로 자동 인식 후 컬렉션을 자동 생성
- Postman이: 경로 분석 → 스펙 파일 생성 → 테스트 자동 작성 → 목(mock) 설정까지 일괄 진행
- 모든 작업이 개발자 개입 최소화 및 자연어 설명 기반으로 이루어짐을 보여줌

### 사용자는 실제 작업에 집중하고, 나머지는 Postman이 자동화하며 AI 네이티브 API 플랫폼으로 진화했음을 강조함

- 반복적이고 번거로운 설정, 동기화, 재작성 등의 과정을 Postman이 대폭 자동화
- 사용자는 오직 "원하는 API 리소스를 자연어로 설명"하는 데만 신경 쓰면 되는 환경이 구현됨
- 코딩, 배포, 테스트, 협업이 모두 하나의 연속적 파이프라인에 통합됨

### 영상 제작자는 Postman의 혁신적 변화에 강한 만족감을 표하며, 자세한 예제 및 링크를 소개함

- 10년 이상 Postman을 사용했으나, 최근 업데이트에 대해 역대급으로 "게임 체인저"임을 강조
- 데모 영상에서 실제로 Postman 및 GitHub repository 예시를 활용하여 기능을 직접 시연함
- 영상 설명란에 실습 가능한 예제와 추가 정보를 제공한다고 언급함
