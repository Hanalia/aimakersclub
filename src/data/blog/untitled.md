## *당신의 AI 에이전트를 초강화하는 가장 단순한 방법 (스킬)* 핵심 요약

- 이 영상은 Anthropic이 최근 공개한 ‘스킬(Skills)’ 기능이 AI 분야에서 매우 중요한 발전임을 설명함
- 스킬의 핵심 가치는 **매우 단순하게 AI의 역량을 강화**한다는 점에 있음 — Anthropic의 모토는 "단순할수록 좋다"
- 기존 에이전트에게 많은 기능(도구, 대화기록, 시스템 프롬프트 등)을 한 번에 제공하면 **컨텍스트 윈도우가 과부하**될 위험이 있음
- MCP 서버 등 기존 방식은 AI가 필요하지 않은 도구까지 모두 한 번에 받아 문제를 일으킴
- **스킬은 ‘점진적 공개(progressive disclosure)’ 방식을 도입함**—필요할 때만 점차적으로 정보를 세부적으로 제공
- 시스템 프롬프트 초기에 에이전트에게 **‘기능 설명’**만 알려주고, 실제 사용 시 구체적 지시사항(skill.md)이 제공됨
- 예시로, ‘PDF 처리 스킬’은 처음엔 존재만 안내하고, 필요 시에만 skill.md의 전체 지침을 읽어 들임
- 추가적으로 skill.md가 다른 문서(3단계 정보공개)를 참조해, 작업에 따라 필요한 정보를 단계별로 로딩함
- 이 방식은 에이전트의 컨텍스트 과부하를 막으면서도 **다양한 기능을 유연하게 사용할 수 있게 함**
- 결과적으로, 스킬을 통해 AI 에이전트가 상황에 맞게 적절한 역량 확장과 정보 활용이 가능해짐

## 세부 요약 - 주제별 정리

### Anthropic의 스킬은 최근 AI 업계에서 가장 중요한 단순화 혁신으로 부상함

- Anthropic이 최근 ‘스킬’을 도입하면서 AI 에이전트 운용 방식에 큰 변화를 줌
- 영상 제작자는 ‘스킬’이 최근 AI에서 가장 중요한 발전 중 하나라고 평가함
- 그 이유는 ‘스킬’ 시스템이 **매우 심플하면서도 강력한 기능 강화** 방식을 제공하기 때문임
- Anthropic의 개발 방향성 자체가 “단순함이 최고(Simple is better)”임을 강조함

### 기존 도구 제공 방식은 에이전트에게 과도한 정보를 한꺼번에 주는 단점이 있음

- 에이전트에게 여러가지 툴, 대화 이력, 시스템 프롬프트 등 모든 정보를 컨텍스트 윈도우에 삽입하는 기존 방식이 많았음
- 이런 접근은 AI 에이전트가 ‘과부하(overwhelmed)’ 상태에 빠지기 쉬움
- 특히 MCP 서버와 같은 방법은, 실제로 사용하지도 않는 여러 도구가 초반에 모두 로딩됨
- 이는 컨텍스트 윈도우의 효율성과 확장성 모두를 저해함

### 스킬은 ‘점진적 정보공개’로 필요한 순간에만 정보를 세부적으로 제공함

- 스킬의 핵심 메커니즘은 **progressive disclosure(점진적 정보공개)** 임
- 모든 기능/도구를 초기부터 공개하지 않고, 실제 니즈가 있을 때만 구체적으로 공개함
- 이로 인해 필요하지 않은 정보가 컨텍스트 윈도우를 차지하지 않음
- 결과적으로 에이전트가 보다 다양한 기능을 효율적으로 사용할 수 있게 됨

### 에이전트에 초기 제공되는 정보는 간단한 ‘기능 설명’뿐임

- 시스템 프롬프트에선 오로지 기능(스킬)의 ‘짧은 설명’ 한 가지만 우선적으로 제공됨
- 예를 들어 “너는 PDF를 처리할 수 있음”과 같은 식으로 안내함
- 이로써 불필요하게 긴 설명, 사용법, 도구 목록 등이 처음부터 노출되지 않음

### 사용자가 기능 요구 시 skill.md를 통해 상세 지시를 추가로 제공함

- 사용자가 실제로 스킬 활용이 필요한 요청(예: “이 PDF를 처리해줘”)을 하면,
- 에이전트는 그때서야 skill.md(스킬 설명서)를 로딩함
- skill.md는 해당 스킬의 상세한 사용법, 절차, 입력값 등 전체 지침을 포함함
- 이 시점에 컨텍스트 윈도우에 구체적인 정보가 추가됨 (점진적 공개의 두 번째 레이어)

### skill.md 이외 추가 문서가 또 필요할 땐 3단계 정보공개가 이루어짐

- 어떤 스킬들은 skill.md 외에 서브 문서(매뉴얼, 참고자료 등)를 추가로 참조함
- 이들 문서는 필요한 상황에서만 접근하여 단계적으로 정보를 더 로딩함
- 결과적으로, 정보 공개는 1단계(설명)→2단계(skill.md)→3단계(추가 문서)로 점진적으로 진행됨

### PDF 처리 스킬 예시로 점진적 정보공개의 실제 과정을 설명함

- PDF 처리 기능을 가진 에이전트 예시가 소개됨
- 첫 번째 단계: 에이전트에게 “PDF 관련 기능이 있음”만 안내
- 두 번째 단계: 사용자가 PDF 작업 요청 시 skill.md로부터 상세 지시정보 로딩
- 세 번째 단계: 필요시 skill.md가 다른 문서(예: 특정 라이브러리 매뉴얼 등)를 참조
- 이러한 프로세스를 통해, 에이전트가 상황에 따라 정보를 확장함을 보여줌

### 점진적 정보공개는 에이전트의 컨텍스트 관리 효율성과 확장성을 극대화함

- 모든 정보와 도구가 한 번에 제공될 때 생기는 ‘컨텍스트 윈도우 과부하’ 문제를 근본적으로 해결
- 에이전트는 작업 상황에 따라 필요한 만큼만 정보를 받아들이게 됨
- 수십 개, 수백 개의 기능이 있어도 실제 필요한 순간에만 컨텍스트에 반영됨
- 에이전트가 더 다양한 일(Task)에 적응하며, 최적의 정보로 해답을 도출할 수 있게 됨

### Anthropic의 스킬 구조와 작동 방식은 사용자 커스텀 구현에도 적용 가능함

- 영상에서 소개된 점진적 구조는 Anthropic 공식 스킬뿐만 아니라, 커스텀 제작 시에도 동일하게 적용할 수 있음
- skill.md 등의 구조적 문서 체계를 활용해 누구나 확장과 관리를 쉽게 할 수 있음을 시사함
- 영상 제작자는 앞으로 더 많은 커스텀 스킬 예제를 소개할 계획임을 암시함

### 에이전트의 역량은 단순함과 효율성을 결합한 스킬 기반 구조로 최적화됨

- 기존의 복잡한 도구 일괄 제공 방식보다, 필요한 시점에 정보가 켜지는 스킬 구조가 훨씬 효과적임을 강조
- Anthropic의 스킬 시스템은 **최대한 단순하게, 필요한 만큼만 정보를 제공해 에이전트의 능력을 극대화**함
- 이 원칙은 향후 AI 에이전트가 더 다양하고 복잡한 업무에 대응하는 데도 필수적임

### 영상 전체 구조는 Anthropic의 스킬 개혁 설명 → 기존 방식을 단점 지적 → 점진적 정보공개 원리 소개 → 실제 예시 설명 → 최종 장점 요약 순서로 전개됨

- 영상은 도입부에서 Anthropic 스킬의 의의와 단순성을 강조함
- 이어 기존 MCP 방식 등 문제점을 구체적으로 지적
- ‘점진적 공개’ 원리와 실제 동작 과정을 단계별로 상세히 풀이
- PDF 처리 스킬 등 구체적 예시에서 실제 동작 과정을 재확인
- 마지막엔 점진적 정보공개 방식의 확장성과 효율성을 재차 강조하며 마무리됨
