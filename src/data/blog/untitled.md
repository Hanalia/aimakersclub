## *AI 에이전트의 능력을 극대화하는 가장 간단한 방법: 스킬(Skills)의 도입* 핵심 요약

- Anthropic이 최근 공개한 ‘스킬(Skills)’은 AI 발전에서 가장 중요한 성과 중 하나로, 간결함이 핵심임
- Anthropic의 철학인 "더 단순할수록 더 낫다(The simpler the better)"가 스킬 설계에 반영됨
- 다양한 능력을 에이전트에 부여하되, 너무 많은 정보를 한 번에 제공하면 컨텍스트 윈도(context window)가 과부하됨
- 기존의 MCP 서버 방식 등은 초기에 모든 도구를 에이전트에게 주기 때문에 실제 대화에서 필요하지 않은 도구까지 불필요하게 노출됨
- 스킬은 '점진적 공개(progressive disclosure)' 방식을 활용하여, 필요한 순간에만 관련 정보를 에이전트에 노출함
- 시스템 프롬프트에는 오직 해당 능력(스킬)의 간략한 설명만 포함되어, 바로 정보 과부하를 피함
- 사용 예시로 PDF 처리 스킬을 들어, 유저가 PDF 관련 요청을 할 때에만 자세한 지침(skill.md)이 주어짐
- skill.md 파일이 각 스킬의 실행 주체로, 단계별(최대 3단계)로 점진적으로 관련 문서와 정보를 로딩함
- 이 접근법은 과도한 도구 제공으로 인한 에이전트의 혼란을 막고, 실제 필요한 순간에만 맥락을 늘려 효율성을 높임

## 세부 요약 - 주제별 정리

### Anthropic의 스킬(Skills) 도입이 최근 AI 발전의 핵심 성과임을 강조함

- Anthropic이 최근 ‘스킬’을 공개한 건 AI 분야에서 매우 중대한 진전임
- 발표 이후 시장과 업계에서 주목하며, 이 기능에 대한 평가가 높음
- 영상 제작자는 “최근 AI 발전에서 가장 중요한 것 중 하나”라고 언급
- Anthropic이 추구하는 ‘단순함’ 기조와 스킬 기능이 깊은 관계가 있음
- 복잡함 대신 단순함을 추구한 설계 덕분에 실제 적용성이 높아짐

### ‘최대한 단순화’라는 Anthropic의 철학이 스킬 설계에 반영됨

- Anthropic의 모토는 ‘더 단순할수록 더 낫다(The simpler the better)’임을 명확하게 밝힘
- 스킬 기능의 작동 방식 역시 이 모토에 충실하게 구성되어 있음
- 불필요한 복잡성 없이, 에이전트의 능력을 '핵심'에 집중시킬 수 있게 도와줌
- 단순성을 통해 에이전트 혼란이나 오류 확률을 낮춤

### 에이전트에 많은 정보를 한 번에 주면 컨텍스트 윈도 과부하 문제가 발생함

- AI 에이전트는 시스템 프롬프트, 대화 기록, 각종 툴 정보 등 다양한 정보를 한 번에 받아들이는데 약함
- 정보가 많아질수록 에이전트가 혼란스러워질 수 있고, 이로 인해 결과의 질이 저하됨
- 영상에서는 "Agents are very prone to being overwhelmed..."라는 표현으로 이 위험성을 직접 강조
- 표준적인 접근 방식(모든 도구를 미리 제공)이 실제로는 문제를 악화시킴

### 기존 MCP 서버 방식은 비효율적이며 불필요한 도구들을 초기에 과다 제공함

- MCP 서버 방식(예시로 언급됨)은 대화에 필요 없는 수많은 툴을 에이전트가 항상 인지하게 함
- 문제점은 실제로 사용되지 않을 도구까지 모두 에이전트 컨텍스트에 포함되는 것임
- 이는 시스템 자원의 낭비이자, AI 응답 품질 저하의 원인임
- 영상 제작자는 이를 “That is bad.”라고 단호하게 평가

### 스킬의 '점진적 공개(progressive disclosure)'가 불필요한 정보 과다 지급을 막음

- 스킬은 Progressive Disclosure(점진적 공개) 원리를 도입하여, 필요한 순간에만 관련 정보를 노출함
- 초기에는 도구 전체를 보여주지 않고, 필요한 스킬만 '발견'해나갈 수 있도록 설계됨
- Progressive Disclosure는 정보 설계 분야에서 검증된 전략으로, 여기서도 효과적으로 적용됨
- 예를 들어, PDF 처리 능력은 해당 요청이 있을 때만 상세 정보가 드러남

### 시스템 프롬프트에는 스킬의 간단한 설명만 포함해 에이전트에 부담을 최소화함

- 초기 시스템 프롬프트에는 각 스킬 별로 “이런 능력이 있다”고 ‘설명’만 전달
- 실제로 스킬을 쓸 때까지 구체적 실행 정보와 코드는 전달하지 않음
- 이렇게 함으로써 컨텍스트 윈도의 용량을 효율적으로 배분할 수 있음

### 스킬 활성화 시점에 skill.md 파일을 불러 새로운 맥락을 추가함

- 유저가 특정 능력을 요청할 때 해당 스킬의 상세 설명(skill.md 파일)이 컨텍스트에 로딩됨
- skill.md는 각 스킬의 메인 문서로, 능력 발현을 위한 실제 실행 지침 포함
- 이 문서는 스킬마다 별도로 존재하며, Anthropic의 스킬 시스템 핵심 역할을 함
- 커스텀 구현 시에도 이 방식 사용 가능

### 오직 실제 요청이 들어올 때만 단계별로 추가 문서와 맥락을 제공함

- 1단계: 시스템 프롬프트에는 스킬의 간단한 설명(Description)만 전달됨
- 2단계: 사용자가 해당 스킬을 요청하면 skill.md(전체 지침 포함)가 로딩됨
- 3단계: 추가적으로 필요한 경우, skill.md가 참조하는 연관 문서들이 추가적으로 컨텍스트에 들어옴
- 점진적으로 맥락이 확장되어, 정보의 적재 적소 활용이 가능해짐

### 스킬의 단계별 점진적 설계가 정보혼란과 응답 품질 저하를 방지함

- 스킬 구조 덕분에 에이전트는 오직 필요할 때마다 새로운 능력을 습득하며 불필요한 정보폭주를 피함
- 모든 능력이 처음부터 로딩되는 것이 아니므로 속도와 품질을 동시에 확보
- “discovering more and more context over time”이라는 직접적인 언급 포함
- 실질적으로 AI 에이전트 개발과 활용에서 유의미한 개선이 됨

### 실용 예시로 PDF 처리 스킬을 들어 실제 구현 상황을 설명함

- 영상에서는 PDF 파일을 처리하는 에이전트를 예시로 제시함
- 시스템 프롬프트: “PDF 처리 스킬을 쓸 수 있다”는 설명만 우선 제공
- 사용자가 실제로 “PDF를 처리해줘”라고 요청하면, skill.md 문서가 시스템에 포함됨
- 필요에 따라 추가적인 문서 참조를 통해 단계별로 정보가 확장됨
- 이 구조가 실제 사용자 요구에 따라 유연히 대응하는 방법임
