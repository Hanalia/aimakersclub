## *Claude Code로 아름다운 다이어그램 만들기: 전체 워크플로우* 핵심 요약

- 코딩 에이전트(Claude Code 등)는 기본적으로 시각적 작업에 취약하여, 시각적 설명이 필요할 때 많은 프롬프트와 추가 기능이 요구됨
- 제작자는 자신의 전체 다이어그램 제작 워크플로우를 하나의 '스킬'로 패키징하여 Claude Code 등 어떤 코딩 에이전트도 활용 가능하게 만듦
- 이 스킬은 코딩 에이전트가 Excal 다이어그램 용 JSON 파일을 작성할 수 있도록 주요 명령어 집합을 제공함
- 다이어그램을 만드는 것뿐만 아니라, 실제로 다이어그램을 렌더링하고 스크린샷을 찍어 PNG를 검토함으로써 시각적으로 자체 검증(셀프 밸리데이션)을 수행
- 이러한 셀프 밸리데이션 덕분에 다이어그램 생성 결과물이 크게 향상됨
- Excal 다이어그램 스킬을 설치하는 가장 쉬운 방법은 코딩 에이전트에게 직접 설치 도움을 요청하는 것(코딩 에이전트가 readme를 읽고 자동화)
- 사용자는 자신만의 브랜드 컬러나 선호하는 색상으로 다이어그램을 맞춤 설정 가능
- 프롬프트 설계의 핵심은 '시각적으로 논증하도록' 코딩 에이전트를 훈련시키는 것에 있음
- 이 스킬이 없다면 다이어그램은 단순하고 직사각형 블록만 나열되는 경향(비주얼적 논증 부족)
- 제작자는 이 스킬을 사용하면 매주 수 시간의 작업을 절약할 수 있다고 설명하며, 누구나 레포를 클론(복제)하고 PDF, 코드베이스 등 다양한 주제로 바로 시도할 수 있음을 강조

## 세부 요약 - 주제별 정리

### 코딩 에이전트는 시각적 설명에 약점을 보임

- Claude Code와 같은 코딩 에이전트는 기본적으로 시각화 도구로서 기능이 제한적임
- 시각적으로 무언가를 설명하거나, 다른 사람에게 설명하도록 도와줄 때 프롬프트 설계가 복잡해짐
- 충분한 시각적 설명을 원한다면 많은 설정과 능력 부여가 필요함
- 제작자는 이런 한계점 때문에 개선 방안을 모색하게 됨

### 워크플로우 전체를 '스킬'로 패키징하여 코딩 에이전트 활용 범위를 확장함

- 자신의 다이어그램 제작 과정을 하나의 통합된 '스킬'로 정제하여 여러 코딩 에이전트가 손쉽게 사용하도록 패키징함
- Claude Code뿐 아니라 기타 에이전트들도 활용 가능함을 언급
- 다양한 실용적(혹은 미적인) 다이어그램을 누구나 쉽게 만들 수 있도록 지원

### Excal 다이어그램 생성 방식을 코딩 에이전트에게 학습시켜 시각적 논증이 가능해짐

- 스킬의 핵심은 코딩 에이전트에 '시각적으로 논증한다'는 개념을 학습시키는 명령어 집합을 부여하는 것
- 코딩 에이전트가 Excal 다이어그램 용 JSON 파일을 만들 수 있도록 유도
- 구조와 레이블 자체가 개념을 제대로 설명하게끔, 단순 블록 나열이 아닌 '논증적 도식'을 생성하는 것이 목표

### 셀프 밸리데이션 기능으로 다이어그램 품질을 대폭 향상함

- 단순히 다이어그램을 만드는 것에서 그치지 않고, 에이전트에게 다이어그램을 실제로 렌더링(시각화)하도록 지시
- 생성된 다이어그램의 스크린샷을 찍어 PNG 파일로 저장한 뒤, 직접 그 결과물을 시각적으로 검토
- 검토과정에서 미흡하거나 어색한 부분을 반복적으로 수정(이터레이션)
- 이 셀프 밸리데이션이 결합되면서, 기존 에이전트가 가진 시각적 작업의 한계를 극복함
- "저는 이 워크플로우로 정말로 탁월한 결과를 얻고 있어요"라는 제작자의 경험이 언급됨

### Excal 다이어그램 스킬 설치는 에이전트에게 직접 요청하는 것이 가장 간편함

- 실제로 스킬을 세팅할 때 가장 쉬운 방법은 코딩 에이전트에게 "Excalra diagram skill을 설치해줘"라고 요청하는 것
- 에이전트가 직접 레포의 readme 파일을 읽고, 수동으로 할 수 있는 설치 과정을 자동적으로 진행함
- 사용자는 복잡한 과정을 거치지 않고 바로 스킬을 쓸 수 있음

### 맞춤 컬러와 스타일로 개인 맞춤형 다이어그램을 제작할 수 있음

- 다이어그램을 자신의 브랜드, 회사, 혹은 개인 취향(컬러 스킴 등)에 맞춰 커스터마이즈할 수 있음
- 단순 기능 제공을 넘어서, 브랜딩 요소를 반영할 수 있다고 소개

### 프롬프트 설계의 핵심은 ‘시각적 논증’ 훈련에 있음

- skill.mmd 파일 등에서 가장 중요한 부분은 '시각적으로 설명하고 논증하는 법'을 에이전트가 익히도록 만든 프롬프트임
- 단순히 박스 여러 개를 나열하거나, 틀에 박힌 다이어그램이 아니라, 전체 구조와 텍스트 레이블 만으로도 개념을 드러내는 것이 원칙

### 별도의 스킬 없이 생성된 다이어그램은 지나치게 단순하거나 박스 위주로 구성됨

- Excal 다이어그램 스킬이 없는 상태에서 생성하게 되면, 다이어그램이 "매우 블록 위주"로 나오게 됨
- 설명력이 떨어지고, 시각화의 목적을 잘 달성하지 못한다는 지적

### 누구나 레포를 복제해서 다양한 활용이 즉시 가능함

- 제작자는 "이 레포를 클론해서 skill을 가져오고 PDF나 코드베이스 등 다양한 주제로 바로 실험해보라"고 제안
- 실제로 자신이 매주 수시간씩 작업 시간을 아꼈음을 공유

### Excal 스킬은 다이어그램 제작에서 시작점을 빠르게 만들어주며 무한한 활용 가능성이 있음

- 단순히 미적인 다이어그램 제작을 넘어서 실용적 쓰임새(문서, 코드, 프레젠테이션 등)에 활용 가능함을 강조
- 시작하기 가장 좋은 방법은 스스로 사용해보는 것이며, 응용 범위가 매우 넓다고 밝혔다
