## *RAG 2.0: 에이전틱 RAG와 지식 그래프를 결합한 강력한 AI 에이전트 소개 (무료 템플릿)* 핵심 요약

- 영상은 최신 RAG(리트리벌 어그멘티드 제네레이션) 전략들을 탐구한 경험을 토대로, 두 가지 핵심 전략인 **에이전틱 RAG(Agentic RAG)**와 **지식 그래프(Knowledge Graphs)**의 결합 방식을 소개함
- 제작자는 이 두 기법의 장점을 융합하여, **정보를 어떤 방식으로 조회하는 것이 질문의 성격에 가장 적합한지 스스로 판단하는 AI 에이전트**를 직접 구축함
- 구현 환경으로는 **PostgreSQL 데이터베이스와 PG Vector 확장 프로그램**을 활용하였으며, 서버리스 PostgreSQL 플랫폼 *Neon*을 사용함
- 실험 데이터는 빅테크 기업들의 AI 전략 정보와 기업 간 파트너십 등 다양한 내용을 담은 문서임
- 동일한 문서를 **벡터 데이터베이스**(임베딩 기반)와 **지식 그래프**(Graffiti, Neoforj 활용)에 각각 저장하여, 정보 유형별로 둘 다 조회할 수 있도록 설계
- 에이전트는 **질문이 단일 기업에 관한 경우 벡터 데이터베이스만**, **기업 간 관계 등 연관성 질문은 지식 그래프만** 조회함
- 사용자가 직접 양쪽 모두를 활용하도록 요청할 수도 있고, 시스템 프롬프트를 조정해 동작 방식을 통제할 수 있음
- 데이터 검색 과정에서 **어떤 도구를 사용했는지까지 사용자에게 명시적으로 안내**하여 투명성 제공
- 영상 마지막에는 이 에이전트를 직접 구현해보거나 활용할 수 있는 **무료 템플릿 제공 및 전체 튜토리얼 링크 안내**
- 전체적으로 RAG 2.0의 탄탄한 아키텍처, 실제 동작 예시 및 실무 적용 방법까지 구체적으로 다룸

## 세부 요약 - 주제별 정리

### 제작자는 최신 RAG 전략 가운데 에이전틱 RAG와 지식 그래프의 강점을 지속적으로 확인함

- 최근 몇 달간 다양한 RAG(리트리벌 어그멘티드 제네레이션) 전략을 직접 실험함
- 그 중 **Agentic RAG**와 **Knowledge Graphs**라는 두 접근이 실제로 가장 유용하다고 반복적으로 판단
- 이 두 기법을 결합하는 방향으로 프로젝트를 진행함

### 에이전트는 입력 질문의 특성에 따라 정보 탐색 경로를 스스로 결정함

- 직접 구축한 AI 에이전트는 들어온 질문을 분석하여 **정확한 답변을 위해 어디서 정보를 찾을지 스스로 판단**
- “Reasoning about where it goes to look for information”이라는 프레이즈로 자율적 검색 경로 설정을 강조
- 이 방식은 단일형 정보(예: 기업 하나의 AI 전략)와 관계형 정보(예: 기업 간 파트너십) 모두에 적합

### 벡터 데이터베이스와 지식 그래프가 모두 활용될 수 있도록 데이터 구조를 이중으로 설계함

- 실험 데이터는 주로 “빅테크 기업들의 AI 이니셔티브 및 파트너 협력 정보”로 구성되어 있음
- 문서 데이터를 **“청킹(chunking)”** 후, **임베딩을 생성**하여 **PostgreSQL+PG Vector** 기반 벡터 데이터베이스에 저장
- 같은 데이터를 **Graffiti 및 Neoforj**를 활용한 **지식 그래프**로도 구현
- 데이터의 내용은 동일하나, 벡터DB는 임베딩 기반 유사도 탐색, 그래프DB는 관계형 질의가 가능함

### 서버리스 벡터 데이터베이스 플랫폼으로 Neon을 선택하였음

- DB 구현에는 **PostgreSQL**을, 벡터 인덱싱에는 **PG Vector 확장**을 활용
- 서버리스 방식의 **Neon** 플랫폼을 이용, 인프라 운영 부담을 감소
- 이 덕분에 프로젝트 실행 및 관리가 용이함을 시연

### Graffiti와 Neoforj로 구축된 지식 그래프에서 기업 간 관계성 탐색이 가능함

- **Graffiti**와 **Neoforj**를 조합하여 지식 그래프를 구축
- 예시 문장: “Amazon is where Anthropic hosts their models”처럼, 기업 간의 구체적 관계성 (예: Anthropic의 모델은 Amazon에서 호스팅됨)을 그래프로 저장해 탐색
- 네트워크 기반 데이터 질의의 강점을 확실하게 활용

### 에이전트는 질문 유형에 따라 벡터검색 또는 그래프검색 중 가장 적합한 경로를 자동으로 선택함

- 질문 사례: “구글의 AI 이니셔티브는 무엇입니까?” → 벡터 데이터베이스만을 조회
- 검색 후, 응답과 함께 “사용된 도구”도 명확히 표기하여 유저가 경로를 투명하게 확인 가능
- 관계성 중심 질문(예: 두 기업 사이의 협업 구조)에는 지식 그래프만을 검색
- 시스템 프롬프트를 통해 검색 경로 지정을 유도 (“now it's using the graph search instead of the vector search”)

### 사용자가 두 데이터 소스를 모두 활용해 질의할 수 있도록 명확하게 옵션이 제시됨

- 필요시 시스템 프롬프트를 수정하거나, 질의시 “벡터DB와 그래프DB 모두 사용”을 명시할 수 있음
- 이 경우 에이전트는 양쪽 데이터를 모두 탐색하여 더 신뢰도 높은 답변 도출

### 에이전트의 데이터 탐색 과정과 사용 도구가 결과와 함께 투명하게 제공됨

- 각 질의 결과에 대해 “어떤 툴이 사용되었는지” 명시적으로 안내
- 벡터DB 기반 검색 혹은 지식 그래프 기반 검색 여부를 명확히 유저가 인지 가능
- 복수 경로 활용시는 두 가지 데이터 소스 모두에 질의를 수행하여 결과 집합을 제공

### 에이전트 구현을 위한 무료 템플릿 및 전체 제작법 영상이 함께 제공됨

- 해당 AI 에이전트의 **무료 템플릿**을 공개
- 템플릿 사용 및 세부 구현 방법은 영상 내 **전체 튜토리얼 링크**로 안내
- 개발과정에서는 **Cloud Code**를 활용하여 제작을 더 효율적으로 지원

### 실무에 즉시 적용 가능한 RAG 2.0 구조, 실제 예시, 개발 환경까지 구체적으로 시연함

- 슬라이드, 실습, 인터페이스 시연 등으로 실제 시스템 동작 모습을 실시간으로 확인 가능
- 벡터DB/그래프DB/프롬프트 조정 등 각종 실전 노하우가 포함됨
- 누구든 무료 템플릿과 전체 가이드를 통해 직접 동일한 에이전트 구축 가능
