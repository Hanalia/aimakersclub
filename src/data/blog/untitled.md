## *이제 내 컴퓨터 손대지 않고, Claude 코드 에이전트가 GitHub 이슈를 자동 해결한다 (무료 툴 소개)* 핵심 요약

- 영상에서는 개인 컴퓨터를 전혀 사용하지 않고, AI 코딩 에이전트가 클라우드에서 GitHub 이슈를 자동으로 해결하는 과정을 직접 시연함
- 이 자동화는 오픈소스 무료 툴인 **Workspaces (by coder)** 덕분에 가능하며, Workspaces는 여러 코딩 에이전트 세션을 병렬로 관리하는 오케스트레이션 레이어를 제공함
- 사용자는 GitHub 이슈에 단순히 “coder” 태그를 추가하는 것만으로, Workspaces 인프라에서 원격 코드 에이전트가 즉시 새로운 인스턴스를 생성하여 이슈를 풀기 시작함
- 모든 코드 작성 및 처리는 사용자의 로컬 환경이 아닌 클라우드(VPS)에 구축된 Workspaces 인프라에서 이루어짐
- 에이전트가 작업을 시작하면, GitHub 이슈에 작업 추적용 링크가 자동으로 댓글로 달리며 진행 상황을 실시간으로 확인 가능함
- 처리 과정에서 원하는 만큼 여러 이슈를 동시에 병렬로 처리할 수 있으며, 팀 전체와 리소스를 공유하여 확장성이 뛰어남
- 처리 결과인 풀 리퀘스트(PR)은 코드 작성자 개입 없이 자동으로 생성되어 검토 및 병합만 하면 됨
- Workspaces는 코딩 에이전트 그 자체가 아닌, 에이전트를 대규모로 원격 관리할 수 있도록 인프라 환경을 제공하는 점이 핵심임
- 무료로 사용 가능하며, 특히 병렬 개발 및 자동화에 관심있는 개발자/팀에 유용함을 강조함
- 영상 마지막에 실제 워크플로우를 직접 데모로 보여주고, 사용 링크를 설명란에 제공함

## 세부 요약 - 주제별 정리

### 사용자는 GitHub 이슈에 ‘coder’ 태그만 추가하면 자동화가 시작됨

- 개인 사용자가 직접 코드를 작성하지 않고, 클라우드 기반의 코딩 에이전트가 모든 작업을 대체함
- GitHub 이슈에 ‘coder’라는 태그를 지정하는 것만으로 자동화된 에이전트가 구동되기 시작함
- 별도의 복잡한 설정이나 매뉴얼 입력 없이 단순 태깅만으로 전체 플로우가 작동한다는 점을 강조함
- 프로그래밍이나 추가적인 명령 입력 없이, 태그 추가 한 번만으로 인프라가 자동 반응함

### Workspaces는 AI 코딩 에이전트 오케스트레이션 인프라를 원격에서 제공함

- “Workspaces by coder”는 사용자의 컴퓨터에 아무것도 설치하지 않고, 원격(VPS 등)에 구축하여 사용함
- 실제 코딩 에이전트가 실행되는 인스턴스와 관리(오케스트레이션)는 모두 클라우드 인프라에서 이뤄짐
- 여러 개의 코딩 에이전트 세션을 병렬적으로 관리할 수 있게 해주는 것이 핵심 역할
- 영상에서는 본인이 VPS에 직접 Workspaces를 셀프호스트했다고 언급함

### Workspaces는 코딩 에이전트가 아니라, 코딩 에이전트를 대규모 관리하는 인프라임을 명확히 함

- Workspaces 자체는 문제를 해결해주는 AI 에이전트가 아니라, 에이전트 오케스트레이션 프레임워크임을 명확히 밝힘
- 코딩 에이전트 각각에 대해 독립된 격리(격리된 환경)가 제공되도록 인프라를 셋업함
- 사용자는 어떤 에이전트든 Workspaces 인프라 안에서 손쉽게 스케일할 수 있음

### 태그 추가 후, Workspaces가 문제 해결을 자동으로 시작하는 과정을 실제로 시연함

- 사용자는 GitHub 이슈 페이지에서 ‘coder’ 태그를 추가함으로써 트리거를 직접 보여줌
- 태그가 적용되면, Workspaces UI에서 새로운 작업(task)이 자동으로 등록됨을 확인 가능
- 대기 후 ‘tasks’ 섹션에서 신규 작업이 초기화되는 모습을 UI로 보여줌
- 자동으로 환경이 세팅되고, 에이전트가 GitHub 이슈에 댓글을 남겨 추적 링크를 제공함
- 사용자는 별도 개입 없이 진행상황을 실시간으로 확인 가능

### 모든 작업이 사용자의 컴퓨터 바깥, 클라우드 인프라에서 실행됨

- 사용자의 로컬 컴퓨터에서는 아무것도 실행되지 않음
- 모든 빌드, 테스트, 코드 작성, PR 생성 등은 Workspaces가 설치된 VPS/클라우드 서버에서 처리됨
- 이로 인해 사용자는 환경 오염이나 시스템 부하 없이 자동화 혜택을 볼 수 있음

### 병렬성 및 팀 단위 개발 효율성이 크게 강화됨을 강조함

- Workspaces를 통해 원하는 만큼 많은 이슈/작업을 한꺼번에 병렬로 처리할 수 있음
- 팀 전체가 동일 인프라를 공유할 수 있어, 공동 작업이나 여러 프로젝트 자동화에 이상적임
- 각각의 작업(Task)는 서로 완전히 격리된 환경에서 실행되어 충돌이 없음

### 코딩 에이전트는 GitHub에 PR을 자동 생성하고, 사용자는 검수만 하면 됨

- 작업이 완료되면, 에이전트가 자동으로 풀 리퀘스트(PR)을 생성함
- 사용자는 생성된 PR을 확인하고 리뷰 및 병합(Merge)하면 절차가 끝남
- 데모에서는 실제 PR 생성 과정을 화면상으로 보여줌으로써 신뢰성 부각

### Workspaces는 무료이며, 누구나 쉽게 시작할 수 있음을 언급함

- Workspaces가 오픈소스이자 무료 배포임을 강조
- 영상에서는 직접 경험을 바탕으로 "시도해볼 것을 적극 추천"하며, 다운로드 및 레퍼런스 링크를 설명란에 안내하였다

### 작업 진행 과정 전반을 실시간 UI와 GitHub 댓글 알림으로 투명하게 확인 가능함

- 각 작업 진행 상태를 Workspaces UI 및 GitHub 이슈 페이지에서 실시간 확인 가능
- 에이전트가 이슈에 남기는 트래킹 링크를 통해 진행 상황을 외부에서도 쉽게 추적할 수 있음

### 영상 전체 흐름에 실제 사용 데모가 포함되어 있어, 실용성 및 신뢰도를 보여줌

- 단순 설명이 아니라, 전 과정을 직접 본인의 프로젝트와 VPS 인프라에서 실시간 시연하여 신뢰성 부여
- 실사용 워크플로우(태깅 → 자동 인스턴스 생성 → 작업 진행 → PR 생성) 전체를 빠짐없이 보여줌
- 사용자가 접할 수 있는 실제 인터페이스(Workspaces, GitHub)를 화면으로 공개함
