## *모든 AI 도구에서 나를 따르는 AI 메모리!* 핵심 요약

- 영상은 개인의 프로젝트와 정보에 대한 기억(메모리)을 여러 AI 도구(Claude Code, Codex, Pi)에서 통합적으로 활용하는 방법을 다룸
- 기존에는 Claude Code에 ‘세컨드 브레인’ 형태로 메모리 시스템을 구축해 높은 생산성을 누렸으나, Claude Code에 종속되는 한계 발생
- 사용자는 각 AI 도구의 장점에 따라 적합한 도구를 선택하고 싶지만, 메모리가 Claude Code에만 묶여 자유로운 활용이 어려움
- 이 문제의 해결책으로 ‘Walrus and Memory’라는 탈중앙화, 암호화 기반의 외부 메모리 솔루션을 소개함
- Walrus는 특정 AI 서비스에 종속되지 않고, 사용자가 소유하는 분산형 메모리 시스템을 제공
- Walrus 메모리는 단 하나의 프롬프트로 무료로 설치 시작이 가능하며, 다양한 AI 코딩 어시스턴트와 SDK를 통한 앱 연동을 지원
- MCP 서버 연동 후에는 여러 도구(Claude, Pi 등)에서 동일한 기억을 저장·검색·분석할 수 있음
- 예시 데모에서 대량의 텍스트에서 20개 주요 정보를 추출·암호화하고, 다양한 AI 세션에서 이를 검색하는 실제 활용 과정을 보여줌
- Walrus 메모리는 소유권과 접근 권한의 유연한 관리(대표키 발급 등)를 제공하며, 데이터 보안과 확장성을 강조
- 영상 마지막에서 Walrus 설치와 사용의 간편함, 무료체험, 빠른 설정을 안내함

## 세부 요약 - 주제별 정리

### Claude Code에 구축된 세컨드 브레인 메모리 시스템은 생산적이나 도구 종속성의 한계가 발생함

- Claude Code를 활용해 개인의 프로젝트와 관련된 모든 정보와 기억을 ‘세컨드 브레인’ 메모리 시스템에 저장함
- 이 시스템 덕분에 생산성이 매우 높아졌다고 밝힘
- 그러나 최근 Claude Code 외에 Codex, Pi 등 다양한 AI 코딩 도구들을 병행해 사용하면서 문제 발생
- 각 AI 도구별로 장‧단점이 있어, 개별 작업에 적합한 도구를 자유롭게 쓰고자 함
- 기존 메모리 시스템이 Claude Code 내부에 한정되어 있어 다른 도구 사용 시 정보 접근이 제한됨

### 다양한 AI 코딩 어시스턴트 사이에서 메모리 연속성이 필요한 문제를 인식함

- 사용자는 Claude Code 외에도 Codex, Pi 등 여러 AI 기반 코딩 도구를 사용하며 작업함
- 각 도구의 강점을 활용하려면 동일한 ‘나의 기억(프로젝트 정보 등)’이 서로 공유되어야 함
- 기존 메모리 시스템이 특정 도구에 묶여 있어 작업 연속성 및 정보 활용에 큰 제약이 있었음을 강조

### 외부 메모리 솔루션인 Walrus and Memory가 분산 메모리의 해법이 됨

- 오랜 기간 이 문제에 대한 해결책을 모색해왔음을 밝힘
- 최근에 Walrus and Memory라는 도구를 발견했고, 자신이 찾던 ‘외부 메모리 계층’에 적합하다고 평가함
- Walrus는 특정 AI 서비스에 종속되지 않고, 사용자가 직접 소유‧통제하는 탈중앙화·암호화 기반 메모리 시스템 제공
- 모든 데이터가 사용자 본인에게 귀속되며, 기업 인프라가 아닌 분산 영역에 저장
- 단 하나의 프롬프트만으로 설치를 시작할 수 있으며, 자세한 과정은 설명란에 링크 제공

### Walrus는 간편한 설치 및 다양한 AI 도구 연동을 지원하며 SDK도 제공함

- Walrus 설치 과정은 매우 간단하게 설계됨
- 공식 홈페이지에서 복사한 프롬프트 한 줄만 AI 코딩 어시스턴트(Claude, Codex, Pi 등) 창에 입력하면 설치됨
- MCP(Memory Control Point) 서버가 연결되어 작동 현황을 GUI로 확인 가능
- Walrus SDK를 활용하면 별도의 앱에 직접 메모리 시스템을 구축·연동할 수 있음

### 대량 텍스트에서 주요 정보 추출 및 암호화 저장 기능이 데모로 시연됨

- 데모에서는 대량의 텍스트(기존에 진행 중인 세컨드 브레인 관련 작업물 등)를 입력함
- “Use Walrus to remember this” 명령을 통해 MCP 서버가 해당 텍스트에서 핵심 정보(메모리) 추출 시작
- 복잡한 텍스트에서 20개의 주요 사실을 자동 추출해서 암호화 및 저장 완료
- 데이터는 분산 저장 방식으로 보안 유지되며, 특정 도구가 아닌 Walrus 메모리 전체에서 활용 가능

### Claude Code와 Pi 등 다양한 AI 세션에서 동일한 Walrus 메모리 검색이 즉시 가능함

- 저장된 메모리 정보에 기반해 추가적인 검색 요청을 테스트
- 예: Claude Code에서 “Search my Walrus memories to tell me about the tech stack for Dark Factory”(Dark Factory의 기술 스택을 알려달라) 요청
- Walrus MCP 서버가 즉시 해당 정보를 찾아 제공, 완성도 높은 답변 반환됨
- 동일 쿼리를 Pi 도구에서도 바로 사용할 수 있음(단순한 MCP 서버 연결만으로 별도 추가 설정 불필요)
- 각 도구별로 미묘하게 다른 답변이지만 핵심 정보는 동일하게 제공됨

### 암호화·소유권·권한관리를 통한 보안성과 확장성 체계를 강조함

- Walrus 메모리는 모든 데이터가 암호화되어 저장됨을 강조
- 사용자가 모든 메모리의 소유권을 가지며, 어느 기업 서버에 귀속되지 않음
- 각기 다른 시스템이나 사용자에게 접근 권한 위임(delegation key) 발급 가능
- 다양한 접근 방식과 권한 관리를 통해 기억에 대한 유연한 활용이 가능함

### Walrus Memory는 AI 도구와 독립적인 ‘메모리 계층’으로서 누구나 쉽게 사용할 수 있음을 안내함

- Walrus를 설치하면, 이제부터 모든 AI 어시스턴트에서 언제든 기억을 불러오고 저장·검색·분석할 수 있음
- 모든 작업 흐름에 통일된 메모리 계층이 제공되므로 생산성과 정보 연속성 향상

### 무료 체험과 빠른 설정 과정을 마지막으로 재차 안내함

- Walrus는 무료로 시작할 수 있다는 점을 강조
- 공식 홈페이지 링크와 간단한 설치 프롬프트 제공을 알림
- 빠른 구성, 쉬운 연동 등 실질적인 사용자 경험의 편리성을 마지막으로 짚어줌
