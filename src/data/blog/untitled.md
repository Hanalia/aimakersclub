## *에이전트 하네스가 Vibe Coding을 다시 불러오는가?* 핵심 요약

- 최근 LLM(대규모 언어 모델)의 본질적 성능 향상 속도가 과거 몇 년 전처럼 폭발적이지 않다는 점을 지적함
- Gemini 3, Claude Opus 4.5 등에서 벤치마크 수치는 개선되고 있으나, 진정한 혁신은 LLM 위에 구축되는 ‘레이어’와 에이전트 상단의 구체적 구현에 있다고 주장
- 이 '레이어'란 LLM을 기반으로 한 에이전트에 더해 새로운 기능, 구조, 최적화를 추가하는 영역을 의미
- 현 시점에서 각 개발자가 이 레이어를 어떻게 설계하고 최적화할지 직접 고민하고 적용할 수 있는 기회가 펼쳐짐을 강조
- 대표 예시로 ‘월드 모델(world model)’이나 LLM 래퍼(wrapper), 즉 에이전트 하네스(agent harness) 등이 언급됨
- 에이전트 하네스는 에이전트의 새로운 역량 해방(언락)의 핵심이며, 앞으로 에이전트가 훨씬 더 많은 업무(특히 코딩)를 대체할 수 있게 해줌
- 상상력이 아직 필요하지만, 이러한 변화로 인해 개발자는 대부분의 코드를 직접 작성하지 않아도 될 미래를 예상
- 2026년 즈음에는 시스템과 하네스 설계에 집중하고, 코딩 자체는 에이전트에게 맡기는 'Vibe Coding'이 현실화될 것으로 전망함

## 세부 요약 - 주제별 정리

### LLM 자체의 발전 속도는 이제 더 이상 과거처럼 가파르지 않음

- 최근 2~3년간 LLM의 기술적 성장, 즉 원초적 성능 향상(예: 언어 이해, 생성 능력)이 이전처럼 급격하지 않음을 언급
- 과거와 달리 LLM이 새로운 패러다임적 전진보다는 점진적 개선 단계에 접어든 것으로 진단
- “Harsh truth”(냉혹한 진실)으로, 혁신의 방향이 변화했음을 시청자에게 인식시킴

### 최신 LLM 예시와 벤치마크는 여전히 향상되고 있지만 한계도 존재함

- Google의 Gemini 3, Anthropic의 Claude Opus 4.5 등 최신 LLM 모델들이 벤치마크(성능지표) 상으로는 계속 발전함을 언급
- 벤치마크 수치상의 인상적 결과에도 불구하고, 본질적 혁신은 더 이상 LLM 자체에서만 나오지 않는다는 점을 강조

### 진정한 혁신은 LLM을 둘러싼 ‘레이어’와 상위 구조에서 나온다는 주장

- 현시점 가장 주목할 변곡점은 LLM 자체가 아니라 LLM을 감싸는 ‘레이어’와 그 활용법임을 명확히 함
- “What we build on top of our agents”(우리가 에이전트 위에 무엇을 구축하는가)가 혁신의 핵심임을 반복
- 이 레이어는 각 개발자가 자유롭게 설계·최적화할 수 있다는 점에서 가치와 기회를 지님

### 개발자 각자가 LLM 레이어와 하네스를 직접 설계하고 최적화할 수 있는 기회를 맞이함

- LLM 기반 에이전트의 외부 구조(레이어)는 모두가 직접 실험하고 최적화할 수 있는 영역임을 강조
- 레이어 설계를 통해 자신만의 에이전트 최적화 방안(효율성, 기능 확장 등)을 발견할 수 있음을 시사
- 이는 일종의 ‘창조적 공간’으로, 새로운 시도와 도전이 가능한 상태임을 내포

### 월드 모델이나 래퍼와 같은 상위 구조 설계가 혁신의 구체적 예시로 제시됨

- “월드 모델(world model)”과 “래퍼(wrapper)”라는 두 가지 구체적 구성을 잠깐 언급
- 월드 모델: LLM이 단순 언어 이해 외에 더 광범위한 맥락/상태/환경 모델링까지 수행하게 만드는 고차원적 프레임워크의 예시
- 래퍼: LLM을 감싸 특정 기능이나 제약을 추가한 구조로, 다양한 분야의 적용 및 맞춤화가 가능함

### 에이전트 하네스가 차세대 역량 해방을 이끌어낼 열쇠임을 강조

- 에이전트 하네스(agent harness, LLM과 외부 프로세스 간의 ‘연결부’)가 바로 다음 단계의 혁신적 역량 발휘(언락)를 이끈다고 규정
- 하네스는 다양한 업무를 오랜 시간(롱 러닝) 동안 자동 수행할 수 있게 만드는 핵심 장치
- 에이전트 하네스의 도입이 코딩 에이전트의 잠재력 활성화에 결정적 역할을 함을 반복

### 하네스를 통해 코딩 업무 대부분을 에이전트에게 위임할 수 있는 미래가 예상됨

- 하네스가 진정한 ‘잠재력 개방(unlock)’의 수단이 될 것임을 확신하는 어조로 서술
- “99% 또는 그 이상”의 코딩이 코딩 에이전트에 의해 수행될 수 있음을 구체적으로 언급
- 이러한 변화의 적용 대상은 코딩뿐 아니라, 다른 장기 지속형(롱러닝) 에이전트 역할까지 확장됨

### 상상력이 필요한 단계이지만, 하네스 설계 중심의 프로그래밍 시대가 도래함

- 아직은 몇 년 앞선 비전이긴 하나, 하네스가 얼마나 발전할 수 있을지 상상력이 필요함을 인정
- 그럼에도 에이전트 활용 및 하네스 설계가 컴퓨팅 환경의 핵심이 될 것을 전망
- 직접 코드를 작성하지 않고 시스템 구조와 하네스 방식만을 설계하는 형태로 직무가 변화할 것임을 암시

### 2026년에는 개발자가 코드 대부분을 작성하지 않는 ‘Vibe Coding’이 현실이 될 것으로 전망

- 미래에는 ‘Vibe Coding’, 즉 직접 코딩 없이 시스템과 하네스 설계만으로 작업이 진행되는 환경이 도래함을 예견
- 이때 개발자는 코드를 세부적으로 작성하는 역할보다 시스템 엔지니어링, 하네스 설계에 더 집중하게 될 것임
- “2026년”이라는 구체적 목표 연도를 언급하며, 변화의 도래가 머지않았음을 강조

### Vibe Coding은 기존과 크게 다른 모양새로 재정의될 예정임

- Vibe Coding이 가능해지나, 과거와 비교해 많이 달라질 것임을 시사
- 앞으로는 전체 코드의 ‘구조 설계’와 시스템 ‘엔지니어링’이 주된 업무가 되며, 실제 코딩 자체는 에이전트에게 맡겨질 것임
- 개발 업무의 패러다임이 AI 협력 구조 중심으로 근본적으로 전환될 것으로 예견
