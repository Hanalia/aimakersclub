## *MCP 서버의 심각한 문제점과 그 해결책* 핵심 요약

- MCP(멀티 컴포넌트 프로토콜) 서버가 널리 사용되면서 심각한 결함이 드러나고 있음
- 대표적인 문제는 토큰 소비량 급증과 문맥 붕괴(context rot) 현상임
- 이 문제는 특히 여러 MCP 서버를 한 세션에서 사용할 때 심각해짐
- 예를 들어, 일반적으로 쓰이는 MCP AI 코딩 서버 5개만 사용해도 각 서버가 AI 에이전트에게 자신을 설명하는 데 수천 개의 토큰을 소비함
- 이로 인해 세션 내에서 빠르게 토큰이 고갈되어 효율적 사용이 매우 어려워짐
- 이러한 한계를 근본적으로 해결하는 방안으로 Anthropic의 '에이전트 스킬(agent skills)' 기능이 제시됨
- 에이전트 스킬은 AI 에이전트가 직접 API 엔드포인트와 상호작용할 수 있는 스크립트 생성, 동적 코드 생성 및 실행 등의 능력을 제공함
- 이 기능은 토큰 소비와 문맥 문제를 획기적으로 줄이고 높은 유연성과 실행력을 제공함
- 영상은 MCP 생태계의 종말 가능성에 대한 의문으로 마무리됨
- 결론적으로, 에이전트 스킬과 코드 실행 능력이 MCP의 주요 문제를 해결하는 대안임을 강조

## 세부 요약 - 주제별 정리

### MCP 서버는 토큰 소비와 문맥 붕괴 문제를 필연적으로 야기함

- MCP는 다양한 AI 서버를 조합해 고도화된 작업을 수행하는 프로토콜로 인기를 얻음
- 그러나 서버마다 AI에게 자신의 기능과 상태를 길게 설명해야 하기 때문에, 대량의 토큰이 소모됨
- 여러 MCP 서버를 한 번에 사용하면, 각 서버의 설명만으로 문맥 창(window)이 거의 소진됨
- 토큰이 빨리 고갈되면서, 문맥 붕괴(context rot) 문제가 심화되어 효율성 저하가 발생

### 실제 예시를 통해 MCP 프로토콜의 비효율성을 드러냄

- 영상에서는 "AI 코딩 작업에 많이 사용하는 평범한 MCP 서버 5개"를 예시로 제시
- 각 서버는 AI 에이전트에게 자신의 기능을 이해시키기 위해 수천 개의 토큰을 사용
- 다섯 개 서버만 동시 활용해도, 토큰 잔량이 급격히 줄어들어서 실질적 업무처리가 비효율적임
- 영상 제작자는 "코딩 어시스턴트와의 한 세션에서 이 서버들을 동시에 절대 쓰지 않는다"고 여러 번 강조

### 에이전트 스킬(agent skills) 도입이 MCP의 구조적 결함을 해결함

- Anthropic(앤트로픽)이 최근 공개한 '에이전트 스킬' 기능이 MCP 한계를 근본적으로 해소함
- 이 기능은 AI와 사용자가 별도의 복잡한 서버 설명 없이, 스크립트로 각 기능을 구현해 작업 수행 가능하게 함
- AI 에이전트가 API 엔드포인트에 직접 접근 및 상호작용할 수 있도록 스크립트 생성 및 활용 지원

### AI 에이전트가 스스로 코드 생성 및 실행을 통해 유연하게 문제를 해결함

- 에이전트 스킬이 적용된 환경에서는 AI 에이전트가 작업 목표에 맞는 스크립트를 즉시 생성함
- 이 스크립트들은 API와의 상호작용, 외부 작업 지시, 실시간 코드 작성·실행 등에 사용될 수 있음
- 그 결과, 고정된 서버 설명을 반복적으로 입력하는 기존 MCP 방식보다 훨씬 적은 토큰 사용으로 높은 유연성을 달성

### MCP의 존속 가능성에 대한 의문이 제기됨

- 영상의 마지막 질문: "지금의 MCP는 이 발전을 계기로 사라질 것인가?"
- 영상 제작자는 명확한 결론을 내리지는 않지만, 기존 MCP 프로토콜의 구조적 한계와 새로운 대안을 강조함
- 기존 MCP 생태계가 에이전트 스킬 등 신기술로 대체될 수 있음을 강하게 시사함

### 결론적으로, MCP의 토큰·문맥 문제는 에이전트 스킬과 동적 코드 실행으로 극복될 수 있음

- 핵심 해결책은 ‘에이전트가 필요한 순간에 스크립트·코드를 직접 생성해서 즉시 실행’하는 것임
- 코딩 어시스턴트, 멀티 서버 연동 등 다중 에이전트 환경에서의 토큰 효율화와 문맥 보존에 탁월함
- Anthropic의 에이전트 스킬이 앞으로 AI 활용 패러다임을 바꿀 수 있는 강력한 도구임을 시사하며 영상이 마무리됨
