## *AI 에이전트를 극적으로 향상시키는 가장 간단한 방법: 스킬(능력) 활용* 핵심 요약

- 이 영상은 Anthropic이 최근 공개한 '스킬(Skills)' 기능이 AI 발전에서 중요한 전환점임을 강조함
- Anthropic의 핵심 철학 "더 간단할수록 더 좋다"를 바탕으로, 스킬은 매우 단순하게 구현되어 있음
- AI 에이전트에게 다양한 기능(능력)을 부여하고 싶지만, 한 번에 너무 많은 정보를 주면 에이전트의 '컨텍스트 윈도우(context window)'가 과부하된다는 점을 지적함
- 기존 방법(MCP 서버 등)은 초기 단계부터 많은 도구들을 한꺼번에 AI에게 노출시켜, 특정 대화에 필요하지 않은 도구까지 불필요하게 포함시키는 문제가 있음
- '스킬'의 강점은 '점진적 공개(progressive disclosure)'에 있음: 에이전트가 실제로 필요할 때만 각 능력의 세부 내용을 단계적으로 습득함
- 초기에는 에이전트에게 능력의 설명(예: PDF 처리 스킬)만 주어 기능이 있다는 사실만 인지하게 함
- 사용자가 해당 능력을 필요로 하는 요청을 하면 에이전트는 skill.md 파일의 상세 지침을 로드하여 실제 사용법을 호출함
- skill.md는 각 스킬의 핵심 지침서를 담당하며, 이에 따라 추가적인 문서(서브 매뉴얼 등)를 필요시 3단계로 점진적으로 읽음
- 전반적으로, 필요할 때마다 필요한 정보만 단계적으로 제공하는 구조로 컨텍스트 과부하를 방지함
- 이 점진적 공개 방식을 통해 AI 에이전트의 기능 확장과 효율성을 혁신적으로 증진할 수 있음을 시연함

## 세부 요약 - 주제별 정리

### Anthropic이 출시한 ‘스킬(Skills)’은 최근 AI 발전의 핵심 전환점임을 단언함
- Anthropic이 최근 AI 업계에 ‘스킬’ 기능을 출시하였으며, 이는 “최근 등장한 AI 발전 가운데 가장 중요한 혁신 중 하나”로 소개됨
- 이 기능은 여러 AI 개발자 및 사용자들이 기존보다 훨씬 효율적으로 에이전트를 확장할 수 있게 도와줌
- 영상 제작자는 스킬의 혁신성이 ‘간결함’에서 비롯된다고 강조

### “더 간단할수록 더 좋다”라는 Anthropic의 개발 철학이 스킬 설계에 반영됨
- Anthropic의 제품 철학은 “The simpler, the better.(더 간단할수록 더 좋다)”임을 명확히 밝힘
- 스킬 시스템 역시 불필요한 복잡성을 배제하고 핵심만 남기는 방식으로 의도적으로 설계됨
- 이로 인해 AI를 사용하거나 개발하는 입장에서 기능 도입과 확장에 부담이 줄어듦

### 에이전트의 컨텍스트 윈도우 과부하 문제가 기존 방법의 가장 큰 한계로 지적됨
- AI 에이전트는 다양한 정보(도구 집합, 대화 내역, 시스템 프롬프트 등)가 ‘컨텍스트 윈도우’에 한꺼번에 투입될 때 쉽게 과부하에 걸릴 위험이 있음
- 개발자들은 기능(툴, 히스토리 등)이 많아질수록 메모리 한계 및 정확도 저하 같은 문제에 직면함
- 이로 인해 ‘능력을 더한다 = 성능 저하 초래’라는 딜레마가 발생함

### 기존 방식(MCP 서버)은 모든 도구와 정보를 초기에 제공해 비효율과 부작용을 초래함
- MCP(멀티툴 제공) 서버 등의 이전 기술에서는 AI에게 필요 여부와 무관하게 모든 기능과 도구 정보를 일괄로 제공함
- 예를 들어, 사용자가 이미 특정 기능을 요구하지 않더라도 그에 대한 상세 정보를 AI가 읽게 됨
- 이는 “불필요한 도구가 에이전트의 컨텍스트 윈도우를 불필요하게 차지한다”는 문제로 이어짐

### 스킬의 핵심은 ‘점진적 공개’ 전략으로 에이전트의 컨텍스트 부담을 최소화하는 데 있음
- 스킬은 Progressive Disclosure(점진적 공개) 개념을 적극 활용함
- AI에게 모든 도구를 한꺼번에 ‘던져주는’ 대신, 상황에 따라 필요한 도구만 ‘하나씩’ 드러내는 구조
- 이로써 불필요한 정보는 숨기고, 진짜 필요한 상황에서만 지침과 데이터를 제시할 수 있음

### 초기에는 “이런 능력이 있다”라는 간단한 설명만 에이전트에 사전 주입되어 있음
- 예시로 PDF 처리 기능을 스킬로 제공하는 경우, 시스템 프롬프트상 “너는 PDF 작업을 할 수 있다”는 정도의 짧은 설명만 표시됨
- 이 단계에서는 기능의 실질적 사용법, 구문, 입력/출력 표준 등 상세 지침은 전달되지 않음

### 사용자가 실제로 스킬의 능력을 활용하고자 요청할 때만 상세 지침이 제공됨
- 만약 사용자가 “PDF 파일을 변환해줘”와 같이 기능 활용을 요구하면, 그때서야 에이전트는 skill.md 파일을 읽음
- skill.md에는 해당 스킬(예: PDF 처리)의 상세 작동 매뉴얼이 담겨 있음
- 이 구조는 필요할 때만 필요한 정보를 컨텍스트에 올릴 수 있도록 설계됨

### skill.md 파일과 연관 문서를 통해 단계적으로 심층 정보를 추가하는 계층화 전략을 사용함
- 1단계: 처음엔 ‘이런 스킬이 있습니다’라는 짧은 설명만 에이전트가 인지
- 2단계: 사용 요청이 감지되면 skill.md에서 상세 지침을 컨텍스트에 로드
- 3단계: skill.md 내부에서 다른 부속 문서(서브 메뉴얼 등)가 필요하면 추가로 단계별 로드
- 이렇게 3단계로 점진적으로 정보를 노출, 늘 상황별로 최적의 컨텍스트 확보가 가능함

### ‘점진적 공개’는 에이전트 확장성과 기능 효율성을 동시에 만족시키는 혁신적 구조임을 강조함
- 컨텍스트 윈도우를 무리하게 소모하지 않고, 필요한 때마다 필요한 정보만 주는 구조
- 다양한 기능 확장이 요청될 때도 시스템 전반의 부담이 크게 줄어듦
- PDF 처리 기능 외에도 Anthropic이 제공하는 모든 커스텀 스킬에 이 구조가 적용됨

### 전반적으로 필요한 순간에만 정보를 노출하는 방식이 AI 에이전트 성능 극대화에 필수임을 시사함
- 스킬 시스템 덕분에, 처음부터 모든 정보를 던지지 않아도 에이전트가 업무를 처리할 수 있음
- '스킬.md'와 연동 문서 구조는 이후 커스텀 기능 개발에도 동일하게 적용 가능
- 영상 제작자는 이 점진적 공개 방식이 에이전트 구조의 차세대 트렌드임을 언급하며 마무리함
