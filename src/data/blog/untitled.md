## *Slack이 되었어야 할 Glue.ai: AI/MCP 네이티브 채팅 플랫폼* 핵심 요약

- 영상 제목은 'Glue.ai는 Slack이 되었어야 할 모습 (AI/MCP 네이티브 채팅 플랫폼)'임
- 영상 제작자는 비즈니스 운영에 Slack을 매일 사용하지만, 점차 비효율과 조직화 문제로 인해 불만이 커졌다고 고백함
- 중요한 버그 수정 스레드 알림을 놓쳐 3일이나 해결이 지연되는 등 알림 관리가 어렵다고 밝힘
- Slack 내 작업의 연속성이 떨어져 다양한 앱을 오가야 하는 번거로움을 경험했다고 구체적으로 언급함
- 이런 비효율을 해소하고자 Glue라는 새로운 플랫폼을 테스트 중임을 밝힘
- Glue는 세계 최초의 멀티플레이어 MCP(Multiplayer Computing Platform) 기반 채팅 플랫폼으로, 실제로 일 처리를 위해 설계됨
- Glue.AI는 모든 스레드에 완전한 맥락을 이해한 상태로 항상 함께하며, 팀원 모두가 직접 활용할 수 있음
- MCP 기반이기에 Asana의 작업 업데이트, Notion 문서 읽기, GitHub 이슈 확인 등 다양한 툴 통합이 채팅 내에서 바로 가능함
- Glue는 35가지 주요 툴을 원클릭으로 연결할 수 있는 대규모 인앱 MCP 디렉토리를 구축했으며, 커스텀 MCP 서버를 통해 수천 개의 추가 툴 연결 지원
- Slack에 AI를 단순 추가하는 것과 달리, Glue는 AI 네이티브 팀 채팅 플랫폼으로 맥락 전환 없는 빠른 협업을 강조함
- AI 도구를 개발하지만 팀 채팅 경험이 '2015년' 수준에 머문 듯 느껴진다면 Glue.AI 테스트를 적극 추천함

## 세부 요약 - 주제별 정리

### Slack의 점증하는 비효율로 인해 작업 누락과 번거로움이 발생함

- 영상 제작자는 본인의 회사 운영에 Slack을 매일 사용하고 있다고 밝힘
- 최근 Slack이 점점 산만하고 체계적이지 못하게 느껴진다는 불만을 토로함
- 직접 경험한 문제로, 중요한 버그 수정 처리 스레드 알림을 3일 간 확인하지 못해 작업이 늦어진 사례를 언급함
- 자신만의 '스레드 위생(thread hygiene)' 문제도 있겠지만, 슬랙의 알림 관리나 정보 집중도가 떨어진다는 문제점을 짚음

### 실제 작업 진행 과정에서 다양한 앱을 전환하는 번거로움이 Slack의 한계로 드러남

- Slack에서 논의된 사항을 바탕으로 실제로 업무를 진행하려면 여러 다른 앱을 동시에 활용해야 하는 현실을 설명함
- 예시로, 채팅 내에서 바로 업무 처리가 되지 않아 Asana, Notion, GitHub 등 다양한 앱을 번갈아 이동하는 번거로움을 제시함
- 이는 생산성 저하로 직결되며, 협업의 효율을 해친다고 말함

### Glue는 Slack이 지향했어야 할 팀 협업 플랫폼으로 등장함

- 제작자는 최근 Glue라는 새로운 플랫폼을 테스트 중임을 밝힘
- Glue를 'Slack이 되었어야 할 모습'이라고 표현하며, 기존 채팅 도구의 한계를 극복한다고 평가함
- Glue는 단순히 대화 및 알림 중심이 아닌 실제로 '일을 완수'하는 데 초점을 맞춘 플랫폼임을 강조함

### Glue는 최초의 멀티플레이어 MCP 기반 채팅 플랫폼으로 협업을 혁신함

- Glue는 세계 최초의 'Multiplayer MCP(Multiplayer Computing Platform)' 기반 채팅 플랫폼임을 내세움
- MCP는 여러 사용자가 동시에 여러 툴과 완전히 통합된 상태에서 실시간 협업이 가능하도록 설계된 구조임
- 팀원 모두가 Glue.AI와 상호작용하며, 단일 환경에서 프로젝트 관련 정보를 공유하고 업무를 처리함

### Glue.AI는 모든 스레드에 맥락을 유지한 채 실시간 지원을 제공함

- Glue.AI는 각 채팅 스레드(Threads)와 항상 함께하며, 대화 맥락과 정보를 완전히 이해함
- 팀의 모든 멤버가 Glue.AI의 도움을 실시간으로 받을 수 있음
- 단순한 챗봇이 아닌, 대화 흐름 전체와 업무 맥락을 반영한 지능적인 지원이 가능함

### MCP 기반 통합으로 업무에 필요한 다양한 툴을 한 플랫폼에서 연결함

- Glue는 MCP 기반으로 개발되어, Asana의 작업(Task) 업데이트, Notion의 문서 확인, GitHub의 이슈 확인 등이 Glue 대화 내에서 바로 가능함
- 각 툴을 오갈 필요 없이 Glue 창에서 대부분의 업무를 처리할 수 있음
- 따라서 맥락 전환(context switching) 없이 효율적인 업무 처리가 가능해짐

### Glue는 35종의 주요 툴을 클릭 한 번에 연결하는 최대 규모의 인앱 MCP 디렉토리를 제공함

- Glue는 전 세계 최대 규모의 '인앱 MCP 디렉토리'를 구축했음을 강조함
- Asana, Notion, GitHub 등 35가지 툴을 클릭 한 번으로 바로 연동 가능
- 별도의 커스텀 MCP 서버를 연결하면 수천 개 이상의 추가 외부 툴 통합도 지원함

### Glue는 기존 Slack과 달리 'AI 네이티브 팀 협업 환경'을 지향함

- Slack과 달리 Glue는 처음부터 AI 통합을 전제로 설계된 플랫폼임
- 기존 Slack에 AI를 외부 서드파티 도구로 "붙이는" 방식이 아님을 분명히 함
- 본질적으로 "빠른 업무 처리와 맥락 전환 없는 협업"을 목표로 한 AI 네이티브 채팅 플랫폼임

### AI 도구를 배포 중이지만 팀 협업 도구가 시대의 흐름을 못 따라간다면 Glue.AI가 해법임을 제안함

- 영상은 "최신 AI 도구를 배포하지만 팀 채팅 경험이 2015년에 머무른 느낌이라면 Glue.AI를 꼭 사용해보라"고 조언함
- 신속하고 효율적이며, 팀 단위로 실제 '작업'을 마칠 수 있는 협업 환경이 중요하다는 점을 강조함

### 영상은 Glue.AI가 팀 업무 처리와 툴 통합 면에서 Slack에 근본적인 대안을 제시한다고 결론지음

- 전체적으로 Slack의 기존 한계를 짚으며, Glue.AI가 실질적이고 혁신적인 대안임을 언급
- 영상 내내 Slack과 Glue의 구체적 차별점을 예시, 실제 경험을 바탕으로 설명함
- 다양한 툴의 통합, AI 기반 실시간 지원, 팀원 간 동시 협업의 장점 등을 제시함
