## *RAG 2.0: 에이전틱 RAG와 지식 그래프 결합 소개 (무료 템플릿)* 핵심 요약

- 영상은 최근 몇 달간 다양한 RAG(Retrieval Augmented Generation, 검색 기반 생성) 전략을 실험한 결과, **Agentic RAG(에이전트형 RAG)와 지식 그래프** 두 방식을 결합한 강력한 AI 에이전트를 소개함
- 이 에이전트는 질문에 가장 적합한 답변을 얻기 위해 어디에서 정보를 찾을지 스스로 추론함
- **벡터 데이터베이스**로는 Neon의 서버리스 Postgres 및 pgvector 확장 기능을 활용
- **지식 그래프**는 Graffiti와 Neo4j를 사용하여 동일 문서를 관계형 데이터로 별도 저장함
- 실제 데이터로는 여러 빅테크 기업들의 AI 이니셔티브와 파트너십 정보를 활용
- 에이전트는 단일 회사처럼 개별 엔티티에 대한 질문에는 벡터 DB 검색을, 두 기업 간 관계를 묻는 질문엔 지식 그래프 탐색을 선택적으로 실행함
- 사용자가 원하면 프롬프트 혹은 명시적 요구를 통해 두 소스를 동시 활용하는 것도 가능
- 모든 동작에서는 에이전트가 사용한 도구(벡터 DB, 그래프 등)를 명확히 피드백함
- 해당 AI 에이전트는 **무료 템플릿**으로 제공되며, 자세한 구축법은 전체 영상 링크로 안내
- 개발에는 Cloud Code 플러그인을 활용하여 적용 및 배포 절차도 설명함

## 세부 요약 - 주제별 정리

### Agentic RAG와 지식 그래프 결합이 다양한 RAG 전략 중에서 가장 실용적임을 실험으로 입증함

- 최근 몇 달간 다양한 RAG 전략을 직접 실험·비교
- 단순 벡터 검색, 그래프 기반, 기타 조합 등 모든 주요 방식 다룸
- 반복적으로 "Agentic RAG"와 "Knowledge Graphs" 방식이 유용함을 실감
- 이 두 가지를 결합하면 챗봇 또는 AI 에이전트의 정보 검색 능력이 비약적으로 향상됨을 강조
- 이러한 결합이 검색 전략의 진화된 모습임을 언급함

### 에이전트는 질문의 성격에 따라 최적화된 정보원(벡터 DB, 그래프 등)을 스스로 선택함

- 에이전트는 입력된 질문을 분석해 가장 적합한 정보 접근 방식을 선택함
- 정보 반출 경로나 방법을 사람이 아닌 AI 에이전트가 직접 결정
- "질문에 대한 최선의 답을 구하기 위해 어디서 정보를 찾을지 스스로 사고(Reasoning)"함
- 이를 통해 효율적이고 정확한 정보 검색 가능

### 벡터 데이터베이스는 Neon 기반 Postgres와 pgvector 확장 모듈로 구현됨

- 데이터베이스로는 "Postgres" 사용, 여기에 "pgvector" 확장 기능 적용
- 서버리스 환경에서는 "Neon" 플랫폼을 활용하여 부담 없이 구성 가능함
- 임베딩 생성된 문서를 벡터 DB에 저장, 유사도 기반 빠른 검색 가능
- 실제 시연용 데이터로는 빅테크 기업 AI 이니셔티브 및 파트너십 문서가 활용됨

### 동일한 정보를 Graffiti와 Neo4j 기반의 지식 그래프에 관계형 데이터로 저장함

- 위와 동일한 문서 데이터(빅테크 기업들의 AI 이니셔티브와 파트너십 상황 등)를 지식 그래프 형태로 저장
- 지식 그래프 구축에는 "Graffiti"와 "Neo4j" 솔루션 사용
- 동일 정보라도 그래프 구조에서는 "관계" 자체에 초점을 맞춤
- 예시: "Amazon은 Enthropic이 모델을 호스팅하는 곳"과 같은 엔티티 간 관계를 저장·질의 가능

### 단일 회사 관련 질문은 벡터 DB 검색으로 자동 처리됨을 시연함

- "Google의 AI 이니셔티브가 무엇이냐?"와 같은 단일 엔티티 질문 예시 제시
- 이 경우, 에이전트는 벡터 데이터베이스에서 임베딩 기반 유사성 검색으로 답변
- 실제 검색 동작, 검색 결과 및 사용된 도구(벡터 DB)가 명확하게 표시됨

### 복수 기업 간 관계성 질문에는 지식 그래프 탐색이 자동 실행됨

- "두 회사 간의 관계"를 묻는 질문을 입력
- 에이전트는 시스템 프롬프트 내 규칙에 따라 벡터 DB가 아니라 지식 그래프를 사용함
- 예시: Amazon과 Enthropic의 제휴관계 탐색 가능
- 결과와 더불어 이번에는 graph search가 시행된 점을 출력에서 확인 가능

### 시스템 프롬프트나 명시적 요구로 벡터 DB와 그래프를 동시에 활용할 수도 있음

- 사용자가 시스템 프롬프트를 수정하거나, 직접 명령문에 두 소스 활용을 요구할 수 있음
- 에이전트는 동시에 벡터 데이터베이스 검색과 그래프 탐색을 병행 실행함
- 양측 데이터 소스를 종합한 더욱 풍부한 답변 제공 가능
- 실제 예시로 벡터 DB와 지식 그래프를 모두 사용하는 출력 화면 시연

### 에이전트 동작 시 검색에 활용한 툴 종류를 항상 투명하게 안내함

- 매 번 답변 출력 시 “The tools used”로 실제 사용한 데이터 소스를 표시
- 벡터 DB와 그래프 검색 각각의 케이스에서 차이점이 명확하게 드러남
- 사용자는 내부 검색 경로와 중간 결과를 쉽게 확인 가능

### 모든 기능은 무료 템플릿 제공 및 Cloud Code 활용을 통해 누구나 쉽게 구현 가능함

- 본 Agentic RAG + Knowledge Graph AI 에이전트는 무료 템플릿 형태로 공개됨
- 전체 구현 과정 및 상세 사용법은 영상 내 별도 링크된 전체 영상에서 안내
- Cloud Code 도구를 활용해 에이전트 개발과 관리 효율화
- 템플릿 활용법, 실제 배포 방법 등도 영상에서 추가 설명

### 빅테크 기업 AI 정보와 파트너십이 사례 데이터로 활용되어 실제 활용 시나리오를 구체화함

- 데이터셋 구성에 구글, 아마존, 엔트로픽 등 빅테크 기업들의 AI 이니셔티브 정보 활용
- 각 기업 간 상호 파트너십, 모델 호스팅 위치 등 사내외 실제 협력관계도 포함
- AI/Tech 업계 실무 적용 가능성이 높은 정보 구조 예시로 활용됨
