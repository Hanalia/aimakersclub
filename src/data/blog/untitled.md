## *MCP 서버의 중대한 문제점과 그 해결책* 핵심 요약

- 영상은 AI 코딩 영역에서 많이 사용되는 MCP 서버 프로토콜의 주요 한계점을 집중적으로 다룸
- MCP 프로토콜의 현저한 문제로 ‘토큰 소비’와 ‘컨텍스트 손실(context rot)’을 꼽음
- 토큰 소비 문제는, 서버마다 AI 에이전트에게 서버의 기능을 설명하기 위해 수천 개의 토큰이 소모된다는 점에서 발생
- 예시로 AI 코딩에 자주 사용되는 표준 MCP 서버 5개를 한번에 세션에 연결하면, 비효율적이고 심각할 정도로 토큰 소모가 커짐
- 이로 인해 여러 MCP 서버를 동시에 하나의 코딩 어시스턴트 세션에서 사용하는 것은 매우 비효율적임
- 이러한 문제를 해결할 방안으로 ‘agent skills’를 소개하며, 이는 최근 Anthropic에서 공개한 솔루션임
- ‘agent skills’는 AI가 API 엔드포인트와 상호작용하며 스크립트를 생성·실행하고, 사용법도 스스로 작성할 수 있도록 지원함
- 이를 통해 MCP에서 발생하는 토큰 소비 및 컨텍스트 손실 문제를 효과적으로 극복할 수 있음을 강조
- 영상 말미에 MCP의 근본적인 변화를 시사하며 “MCP의 종말인가?”라는 질문을 남김

## 세부 요약 - 주제별 정리

### MCP 프로토콜의 인기와 그에 따른 근본적 결함이 드러나고 있음

- MCP(멀티 서버 프로토콜)는 AI 코딩과 관련된 분야에서 최근 점점 더 인기를 얻고 있음
- 프로토콜이 널리 사용됨에 따라, 치명적인 구조적 단점을 더 명확히 드러내고 있음
- 영상 제작자는 이 결함들이 단순한 불편함이 아니라, AI 개발 환경에서 매우 큰 영향을 미친다고 지적함
- ‘토큰 소비(token consumption)’와 ‘컨텍스트 손실(context rot)’이 특별히 논의되는 주요 이슈임

### 토큰 소비와 컨텍스트 손실 문제는 현존 MCP 활용의 최대 걸림돌임

- 여러 MCP 서버를 한 세션에서 사용할 때, 각 서버의 기능을 AI 에이전트에게 설명하는 데 수천 개의 토큰이 필요함
- 토큰은 AI 시스템에서 허용하는 입력 컨텍스트의 용량(예: 프롬프트 길이 등)을 의미하며, 한도 내에서 사용해야 함
- 토큰 낭비가 누적되면 AI의 컨텍스트 창(window)을 금세 가득 채워, 이전 정보나 대화 맥락이 무시되는 ‘컨텍스트 손실’ 현상이 발생함
- 이로 인해 복수의 MCP 서버를 동일 세션에서 병행 활용하는 것이 현실적으로 비효율적임

### 실사례를 통한 MCP 서버의 토큰 사용 문제점 설명

- 영상에서는 실제로 AI 코딩 보조에 사용하는 5개의 표준 MCP 서버를 예로 듦
- 각 서버는 기능 설명만으로 ‘수천 개’의 토큰을 소모하여, 5개만 동시에 연결해도 토큰 사용량이 극심해짐
- 영상 제작자는 “이 서버들을 전부 한 세션에서 사용할 일은 없으며, 절대 추천하지 않는다”고 명백히 강조함
- 이처럼 MCP 환경에서는 확장성 및 자원 효율성이 심각하게 저하된다는 사실이 강조됨

### Anthropic의 ‘agent skills’가 MCP의 구조적 문제의 해결책으로 제시됨

- 이 문제를 해결할 신기술로 ‘agent skills’(에이전트 스킬)라는 방식을 소개함
- ‘agent skills’는 Anthropic(엔트로픽)에서 최근에 출시한 기능임
- 핵심 원리는 AI 에이전트가 코딩 작업에 필요한 스크립트를 직접 생성하거나, 새로운 API 엔드포인트와 자동 상호작용할 수 있게 해주는 것임
- 기존의 길게 나열된 서버 설명(token-heavy description) 대신, 효율적으로 기능을 불러오고 실행할 수 있도록 함

### ‘agent skills’는 AI의 코드 생성, 스크립트 작성 및 실행 부문에서 혁신적임

- AI는 단순 API 호출뿐 아니라, 스스로 실행 가능한 ‘스크립트’와 해당 사용법(usage instructions)까지 동적으로 만들어냄
- 필요시 순간적으로 코드를 생성해 실제 환경에서 실행하는 ‘코드 실행(code execution)’ 기능도 포함됨
- 이로써 AI 코딩에 있어 많은 서버의 기능 설명 반복이나 불필요한 토큰 소모 없이 작업 수행이 가능해짐
- 제작자는 “이 기술은 막강한 힘을 지녔다”고 언급, 변화의 실질성을 시사함

### 최신 솔루션 도입으로 MCP의 기존 한계가 극복될 수 있음을 강조함

- ‘agent skills’는 MCP 프로토콜의 토큰 효율성 문제에 대한 현실적 대안임을 다시 한번 상기시킴
- AI 에이전트가 스킬/스크립트 기반으로 API, 코드, 자동화 기능을 유연하게 활용할 수 있음
- 전통적 MCP 서버 기반 방식에서는 불가능했던 대규모 확장과 정보 전달의 효율성이 보장됨
- 기존 시스템의 구조적 한계를 넘는 패러다임 전환 가능성이 언급됨

### 영상 말미에서 MCP의 미래에 근본적 변화가 있음을 암시함

- 제작자는 “이것이 우리가 알던 MCP의 종말인가?”라는 도발적인 화두로 마무리함
- 의미는 명확히 언급하지 않았으나, 등장한 솔루션(‘agent skills’ 기반)이 MCP 프로토콜에 커다란 변혁을 가져올 것임을 시사함
- 전체적으로 MCP 서버에 대한 기존 접근법과 향후 발전 방향을 비판적이면서도 혁신적으로 짚고 있음
