## *AI 에이전트의 능력을 극대화하는 가장 단순한 방법(스킬)* 핵심 요약

- 이 영상은 Anthropic의 ‘스킬(skills)’ 기능 도입이 최근 AI 발전에 있어 매우 중요한 혁신임을 강조함
- Anthropic의 모토인 “더 단순할수록 더 낫다”는 원칙에 따라, 스킬은 복잡한 도구 제공 방식을 피함
- 기존 방식(MCP 서버 등)은 에이전트에 다수의 도구와 정보를 한 번에 던져주며, 이는 문맥(Context) 윈도우를 쉽게 과부하 시킴
- 스킬의 핵심은 ‘점진적 공개(progressive disclosure)’로, 필요할 때 필요한 기능만 단계적으로 노출함
- 시스템 프롬프트에는 각 기능(스킬)의 간략한 설명만 담아, 에이전트가 필요 시 그 설명을 바탕으로 추가 정보를 탐색함
- 예시로, PDF 처리 스킬의 경우 처음에는 “PDF 처리 가능” 설명만 제공되고, 실제 PDF 요청이 오면 자세한 사용법(skill.md)를 추가로 불러옴
- skill.md 파일은 해당 스킬의 구체적이고 전체적인 사용 지침을 포함하며, 필요에 따라 추가 문서(3단계)로 이어질 수 있음
- 점진적 공개를 통해 컨텍스트 과부하 없이, 필요한 시점에 적절한 정보만 제공함으로써 에이전트의 효율성과 유연성을 높임
- Anthropic의 여러 커스텀 스킬 구현 사례에 대해 소개하며, 이러한 구조가 다양한 작업에 효과적임을 시사함
- 영상 전체를 관통하는 주장은 “에이전트에게 필요한 순간에 필요한 정보만 제공하는 방식이 가장 강력하다”는 점임

## 세부 요약 - 주제별 정리

### Anthropic의 스킬 기능은 최근 AI 발전에서 가장 중요한 이정표임

- Anthropic가 ‘스킬(skills)’ 기능을 공개한 시점이 그리 오래되지 않았음을 강조
- 발표 이후 업계에서 큰 변화와 관심을 이끈 대표적 발전임을 언급
- 영상 내내 스킬의 혁신성과 중요성에 초점을 맞춤

### ‘단순함’은 Anthropic이 추구하는 제품 철학의 핵심 원칙임

- Anthropic가 “더 단순할수록 더 낫다(The simpler the better)”라는 모토를 고수함
- 복잡한 기능을 최소화하고, 직관적으로 쓸 수 있는 설계를 우선함
- 이러한 접근이 스킬 기능 설계에도 반영됨

### 기존 AI 에이전트 도구 구조는 컨텍스트 과부하 문제를 야기함

- 대부분의 에이전트 구조는 대화 기록, 툴, 시스템 프롬프트 등 매우 많은 정보를 한 번에 윈도우에 전달함
- MCP 서버 같은 기존 방식은 에이전트가 한 번에 다수의 도구와 기능을 알게 됨
- 실제 필요한 도구만 쓰이는 것이 아니라 모든 정보가 초기부터 주어져, 컨텍스트 윈도우를 순식간에 소모함
- “Agent overwhelm” 현상이 쉽게 일어남

### 스킬 구조의 핵심은 ‘점진적 공개(progressive disclosure)’임

- 스킬은 필요한 도구와 정보를 한꺼번에 에이전트에 제공하지 않음
- 첫 단계는 시스템 프롬프트에 스킬의 “설명”만 제공(레이어 1)
- 이후 사용자가 특정 작업(PDF 처리 등)을 요청할 때, 두 번째 단계로 skill.md 파일(운영 지침 및 상세 설명)을 보충(레이어 2)
- 추가적으로, skill.md가 다른 참고 문서나 서브파일을 참조하는 경우도 있어(레이어 3), 점차 컨텍스트 정보가 세분화되어 추가됨
- “Progressive disclosure”는 상황 필요에 따라 필요한 만큼만 정보를 드러낸다는 설계 사상임

### 스킬 적용 예시: PDF 처리를 위한 능력 부여 방식 설명

- 예시로 “PDF 처리(PDF processing) 스킬”을 설명함
- 시스템 프롬프트에 “PDF를 처리하는 기능이 있음”이라고만 기술
- 사용자가 PDF 관련 작업을 실제로 요청하면, 해당 시점에 skill.md 내의 구체적 사용법・지침을 참조함(전체 맥락은 이때 추가됨)
- skill.md 파일이 스킬의 주 사용 설명서 역할을 하며, 필요시 추가 문서로 확장 가능

### 단계형 문맥 추가로 에이전트의 컨텍스트 윈도우 효율성을 극대화함

- 최초에는 매우 요약된 정보(스킬 설명)만 컨텍스트에 포함
- 실제로 스킬이 필요한 순간에만 전체 가이드(skill.md) 및 추가 자료를 컨텍스트에 적재
- 불필요하게 많은 정보를 윈도우에 넣지 않으므로 과부하 없이 정확한 정보 활용이 가능

### Anthropic의 커스텀 스킬 구현 방식은 다양한 도메인에 유효함

- 영상에서 “Anthropic이 제공하는 모든 스킬”, “커스텀 구현 버전” 등 실제 활용 사례가 언급됨
- skill.md를 중심으로 하는 스킬 아키텍처가 확장성과 다양한 도메인 적용에 적합함을 시사

### 점진적 공개 방식이 에이전트의 효율성, 유연성, 맥락 이해도를 동시에 개선함

- 모든 툴보다는 필요한 순간에, 필요한 정보만 보여줘서 에이전트가 더 “똑똑하게” 동작
- 복잡한 환경에서도 정보와 기능 노출을 적절하게 조절 가능
- 컨텍스트 윈도우의 낭비를 최소화하며, 다양한 작업 수행에서 장점 제공

### 기존 MCP 서버 방식과 비교해, 스킬 도입이 가지는 구조적 차별점을 강조함

- MCP 서버 등 기존 방식의 “모든 기능 upfront 제공” 패턴의 단점(과부하, 활용도 저하)을 비판
- 스킬의 “진입 장벽 낮춤과 피로도 최소화” 효과를 대조적으로 부각

### ‘에이전트는 필요한 순간에 적정한 정보를 점진적으로 받아야 최대한의 성능을 낸다’는 결론에 도달함

- 영상 마지막까지 이 구조의 효율성, 단순성, 실용성에 대한 강조가 이어짐
- “에이전트에게 너무 많은 도구를 한꺼번에 주지 마라”, “점진적으로 정보를 제공하라”는 핵심 메시지가 반복됨
