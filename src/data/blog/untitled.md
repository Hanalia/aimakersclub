## *AI 에이전트의 능력을 극대화하는 가장 단순한 방법(스킬)* 핵심 요약

- **Anthropic이 최근 공개한 "스킬(Skills)"은 AI 분야에서 가장 중요한 혁신 중 하나로 꼽힘**
- **스킬의 핵심 가치는 '아름답게 단순하다'는 점에 있음: 단순성은 Anthropic의 모토임**
- **AI 에이전트에게 많은 능력을 부여하고 싶지만, 너무 많은 정보를 한꺼번에 제공하면 context window(문맥 창)가 과부하됨**
- **다른 방식(MCP 서버 등)은 에이전트에게 대량의 툴을 미리 모두 건네주는 문제가 있으며, 이 경우 사용하지 않는 도구까지 모두 인식하게 되어 비효율적임**
- **스킬의 방식은 '점진적 공개(Progressive Disclosure)'로, 필요한 순간에만 관련 정보를 공개하는 구조임**
- **초기에는 시스템 프롬프트에 해당 능력의 간단한 설명만 제공함(예: PDF 처리 스킬이 있음을 알리는 설명)**
- **이후 실제로 해당 능력이 필요한 요청이 들어올 때, 구체적 지침이 담긴 skill.md 등 추가 자료를 단계적으로 에이전트에 제공**
- **skill.md 파일에는 해당 스킬의 사용법과 상세 내용이 담겨있고, 그 이후 3단계로 추가 참조자료가 로드되는 구조(총 3단계 progressive disclosure)**
- **이 방식을 통해 에이전트는 실제로 필요한 순간에만 관련 정보를 받아들임으로써 맥락 과부하 없이 다양한 능력을 갖출 수 있음**

## 세부 요약 - 주제별 정리

### Anthropic의 스킬(Skills)은 AI 발전에 있어 단순성과 효율성을 동시에 해결함

- Anthropic이 "스킬" 기능을 공개한 것은 최근의 중요한 AI 진전으로 평가받고 있음
- Anthropic의 철학은 "더 단순할수록 더 좋다(The simpler the better)"임
- 복잡한 도구나 과도한 정보보다, 필요한 능력을 필요할 때만 부여하는 접근이 강조됨

### 에이전트의 context window는 과부하 문제가 심각할 수 있음

- 에이전트에게 툴, 대화 이력, 시스템 프롬프트 등 수많은 정보가 context window에 들어감
- context window가 너무 많은 정보로 채워질 경우, 에이전트의 처리 효율과 정확성이 떨어짐
- 기존 방식대로 모든 도구와 정보를 미리 전달하는 것은 context window 오염의 주요 원인임

### MCP 서버 방식은 도구 과잉 제공 문제를 낳음

- 기존 MCP 서버 등에서는 초기부터 에이전트에게 모든 툴을 일괄 제공함
- 이러한 경우 실제 대화에서는 거의 사용되지 않을 도구까지 포함되어 context window를 불필요하게 소모함
- 이로 인해 불필요한 복잡성 및 비효율성이 발생함

### 스킬은 '점진적 공개(Progressive Disclosure)'를 통해 필요할 때만 정보가 주어짐

- 스킬의 핵심은 progressive disclosure, 즉 정보와 도구를 점점 필요한 만큼만 공개하는 방식임
- 모든 툴을 미리 공개하지 않고, 상황에 따라 단계적으로 필요한 정보만 제공함
- 사용자가 특정 기능(예: PDF 처리)을 요청하기 전에는 상세 지침을 제공하지 않음

### 시스템 프롬프트에는 능력의 설명만 1차적으로 노출됨

- 에이전트가 초기 시스템 프롬프트에서 받는 것은 "이런 스킬이 있다"라는 간단한 설명임
- 예시) "PDF를 처리할 수 있음"이라는 문장만 시스템 프롬프트에 들어감
- 이는 에이전트가 자신에게 해당 능력이 있다는 사실만 인식하도록 설계됨

### 사용자가 실제로 해당 능력을 요청할 때 두 번째 정보 공개가 이뤄짐

- 사용자가 PDF 관련 요청 등 특정 능력을 필요로 하는 상황이 발생하야만 구체적 파일(skill.md)을 읽게 됨
- skill.md 파일 안에는 해당 스킬을 실제로 어떻게 작동시키고 사용하는지에 대한 상세 설명이 들어있음
- 이 단계는 progressive disclosure의 "두 번째 층"에 해당함

### skill.md 파일을 통해 스킬 사용의 세부 지침 및 추가 자료가 제공됨

- skill.md 파일은 모든 Anthropic 스킬의 핵심 문서로 작동함
- 이 파일에는 기능별 완전한 사용법, 조건, 예시 등이 기재되어 있음
- skill.md에서 추가적으로 다른 문서를 참조하면, 그때 3단계로 해당 문서들까지 로드됨

### 3단계 point: 점진적 맥락 추가로 효율성 극대화

- 1단계: 에이전트가 단순 설명으로 기능 존재를 인식
- 2단계: 사용자 요청 시 skill.md로 구체적 기능을 학습
- 3단계: 필요하다면 skill.md에서 언급된 추가 문서(참고자료)를 읽어들이며 상황별로 점차 맥락을 확장
- 이 구현을 통해 에이전트가 다양한 도구를 보유하면서도 context window 오염 없이 대응 가능

### Anthropic 스킬 방식은 확장성과 실용성을 모두 확보함

- 이 점진적 공개 방식을 활용하면, 커스텀(사용자 정의) 구현도 매우 용이해짐
- 필요 없는 순간에는 정보를 숨겨두고, 실제 필요 시에만 단계적으로 추가
- 따라서 다양한 스킬 추가 및 확장 또한 맥락의 부담 없이 진행할 수 있음

### 예시: PDF 처리 스킬로 progressive disclosure 구조 설명

- 예로 든 PDF 처리 스킬에서, 에이전트는 PDF 작업 요청 이전까지는 상세 정보를 참조하지 않음
- 요청이 들어오면 skill.md 파일에 기초해 정확한 절차와 담당 기능을 수행
- 필요 시 skill.md가 참조하는 외부 문서까지 활용함
- 실제 작업에 필요한 정보만 context에 로드되므로, 불필요한 정보가 에이전트에 영향을 주지 않음

### 결론적으로 스킬 구조는 현대 에이전트 설계에서 정보 효율성과 기능 확장의 최적 해법임

- 점진적 공개 덕분에 에이전트가 맥락 창을 효율적으로 사용하며, 과부하될 걱정 없이 많은 능력을 잠재적으로 보유할 수 있음
- Anthropic의 스킬 구조는 커스텀 구현에서도 매우 유연하게 활용 가능
- 이는 AI 에이전트 설계와 운영에서 앞으로의 표준적인 접근이 될 수 있음을 시사함
