## *RAG 2.0: 에이전틱 RAG와 지식 그래프 결합(무료 템플릿 소개)* 핵심 요약

- 영상 제목은 "RAG 2.0: 에이전틱 RAG + 지식 그래프(무료 템플릿)"이며, 최신 RAG(Retrieval-Augmented Generation) 전략 두 가지를 결합한 강력한 AI 에이전트를 실습 중심으로 소개함
- 주인공이 수개월 동안 다양한 RAG 전략을 탐구한 결과, '에이전틱 RAG'와 '지식 그래프'가 핵심적임을 강조
- 제안된 에이전트는 정보 탐색 위치를 스스로 판단하여 요청에 가장 적합한 데이터 소스를 선택함
- 백터 데이터베이스로 Postgres에 PG Vector 확장 사용, 서버리스 Postgres 플랫폼인 Neon에 구현
- 문서(빅테크 기업의 AI 이니셔티브 및 파트너십 정보)를 청크 단위로 분할, 임베딩 후 벡터DB에 저장
- 동일 문서를 Graffiti와 Neo4j를 활용하여 지식 그래프로도 저장, 관계형 검색을 가능하게 만듦
- 단일 기업 정보처럼 단일 엔티티 질의는 벡터 DB를, 두 기업 간 관계 질의는 지식 그래프를 활용해 각각 최적화된 결과 제공
- 에이전트의 도구 사용 내역도 포함, 어느 데이터 소스를 활용했는지 투명하게 알려줌
- 시스템 프롬프트 수정 또는 명시적 호출로 두 소스의 동시 사용도 가능, 최적의 답변 품질 목표
- 영상 내 제공된 무료 템플릿 링크와 전체 구현 방법에 대한 상세 영상 안내
- Cloud Code 활용 개발법, 누구나 빠르게 같은 에이전트 구축 가능한 실습 가이드 제공

## 세부 요약 - 주제별 정리

### 저자는 다양한 RAG 전략을 심층적으로 탐구한 끝에 두 가지 핵심 방식을 발견함

- 최근 몇 달 동안 영상 제작자는 거의 모든 RAG 전략을 실험적으로 검토해봄
- 반복적으로 떠오르는 접근 방식은 '에이전틱 RAG'(Agentic RAG)와 '지식 그래프'(Knowledge Graph)임을 강조
- 두 방식을 연결하면 더 강력하고 유연한 정보 검색형 AI 에이전트를 만들 수 있음을 설명

### 에이전틱 RAG와 지식 그래프를 결합한 강력한 AI 에이전트를 구현함

- 해당 영상에서는 두 방식의 장점을 결합한 AI 에이전트를 직접 구축해 시연함
- 해당 에이전트는 입력 질의의 의도와 정보 유형을 해석하여, 어떤 데이터 소스(벡터DB 또는 지식 그래프)가 가장 적합한지 자동 판단함
- 스스로 추론(Reasoning)을 통해 최적의 정보 획득 경로를 선택

### 벡터 데이터베이스는 Postgres와 PG Vector 확장, 서버리스 플랫폼 Neon으로 구현함

- 문서 임베딩 저장을 위한 벡터 데이터베이스로 Postgres 선택
- 효율적인 벡터 검색을 위해 PG Vector 확장 기능 연동
- Neon이라는 서버리스 Postgres 플랫폼을 통해 클라우드 환경에서 구축 및 운영

### 문서 데이터는 AI 이니셔티브 및 빅테크 기업 파트너십 정보로 구성됨

- 사용된 예시 문서는 주요 빅테크 기업(구글, 아마존 등)의 AI 프로젝트, 전략, 그리고 상호 협력 관계 등 다양한 정보 포함
- 문서는 AI 기반 RAG 에이전트의 정보 검색 시나리오에 꼭 맞는 실제적이고 풍부한 데이터로 활용됨

### 데이터는 벡터DB 임베딩과 동시, Graffiti-기반 Neo4j 지식 그래프로도 저장함

- 동일한 문서 데이터를 두 가지 방식으로 동시 저장: 1) 임베딩 후 벡터DB, 2) Graffiti 및 Neo4j 활용 지식 그래프
- 텍스트 벡터 DB와 관계형 데이터 구조(지식 그래프)의 강점을 모두 활용
- 지식 그래프에서는 "Amazon은 Anthropic의 모델 호스팅 파트너" 같은 기업 간 관계질의도 자연스럽게 처리 가능

### 질의 유형에 따라 에이전트는 적합한 도구(벡터DB/지식그래프)를 선택적으로 사용함

- "구글의 AI 이니셔티브는 무엇인가?"와 같이 단일 엔티티 기반 질의시 에이전트는 벡터DB 검색 활용
- 답변 이후 에이전트가 어떤 도구(데이터 소스)를 사용했는지 투명하게 리포팅함
- 두 기업 관계("Amazon과 Anthropic의 관계") 등을 묻는 질의는 지식 그래프를 탐색하여 명확한 구조적 결과 제공

### 에이전트는 시스템 프롬프트 수정이나 명시적 요청으로 두 소스를 동시에 호출할 수 있음

- 시스템 프롬프트(operational prompt)를 적절히 조정하여 검색 논리를 사용자가 조율 가능
- 명시적으로 "벡터 DB와 그래프를 모두 써서 답하라"고 지정도 가능
- 복합질문 등에서 벡터DB·지식 그래프 동시 검색을 통해 답변의 정합성과 정보의 풍부함을 극대화

### 에이전트의 작동 과정과 도구 데이터 소스 사용 내역 역시 결과에 포함됨

- 에이전트가 질의별 어떤 데이터 소스를 택했는지, 처리 경로까지 답변 내 명확히 기재
- 투명한 결과 제공은 신뢰성과 분석용으로도 유용

### 무료로 제공되는 에이전트 템플릿과 전체 실습 영상을 추가 안내함

- 영상 내 직접 체험 가능한 무료 템플릿 제공 링크 안내
- 전체 영상에서는 이 에이전트를 실제로 어떻게 만들고 배포하는지 단계를 상세 설명

### Cloud Code를 활용해 누구나 쉽게 이 RAG 2.0 에이전트를 구축할 수 있도록 안내함

- 영상과 템플릿 내 Cloud Code(클라우드 개발 코드) 적용법을 친절히 설명
- 개발 경험이 적은 사람도 실습을 따라 동일한 에이전트 구축 가능
- RAG 2.0을 활용한 차세대 AI 검색 에이전트 구축 방법을 손쉽게 익힐 수 있는 기회 제공
