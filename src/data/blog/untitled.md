## *Claude에게 24시간 내내 코딩을 시킨 결과, 완벽에 가까운 클론 앱을 자동 구축함* 핵심 요약

- 영상의 주제는 Claude Opus 4.5를 사용해 24시간 동안 자동으로 코딩을 진행시켜 claw.ai의 클론 앱을 만드는 실험임
- 제작자는 작업 중간 오차나 개입 없이, Claude가 스스로 대부분의 기능을 구현했다고 평가함
- 과거 대화와 검증 과정을 확인할 수 있어 개발 과정의 반복과정을 명확하게 추적 가능함
- Markdown 형식의 문서화, 여러 HTML 페이지 생성, 코드 작성 및 실행 등 다양한 기능이 완성됨
- 앱에는 테마/모델 설정 변경 기능, 토큰 한도 조절용 슬라이더 등 세부적인 설정도 포함됨
- 상당히 풍부하고 완성도 높은 기능을 거의 인간의 개입 없이 자동으로 구축한 점을 강조함
- 추가 목표로 ‘Human-in-the-loop(인간 개입) 기능’을 넣고 싶다는 계획을 언급함
- 만약 하루 더 실행하면 거의 완벽한 애플리케이션이 될 수 있을 정도로 발전 가능성을 시사함
- Claude Opus 4.5와 이 자동화 환경(harness)이 얼마나 빠르고 체계적 검증·개발이 가능한지 보여줌
- 인상적인 자동화 성능에 대해 ‘정말 미쳤다’며 최신 LLM의 개발 역량을 재확인함

## 세부 요약 - 주제별 정리

### Claude Opus 4.5를 활용해 claw.ai 클론을 완전히 자동으로 개발함

- Claude Opus 4.5 모델을 활용해 claw.ai 웹앱의 클론을 만들기 위한 실험을 진행함
- 24시간 동안 Claude에게 코딩 작업을 ‘논스톱’으로 맡김
- 사람의 직접적인 작업 개입 없이 Claude가 기능 구현 및 테스트 자동화 작업을 계속함
- 제작자는 결과물을 보고 ‘이미 거의 완벽하게 작동하는 claw.ai 클론’이라고 평가함
- 중간중간 기능의 완성도를 확인하는 추가 테스트가 필요할 정도로 결과물이 완성됨

### 과거 대화와 자동 검증 과정을 저장 및 재확인할 수 있도록 구현됨

- Claude가 각 반복(루프)마다 코드와 기능을 검증하는 대화 내역을 자동으로 저장함
- 제작자는 앱 내에서 이 대화 이력 전체를 확인 및 추적할 수 있다고 설명함
- 개발 진행 상황과 검증 루틴을 시간 순서대로 투명하게 파악할 수 있는 구조임

### Markdown 형식 문서화 및 다양한 HTML 페이지 자동 생성 기능이 구현됨

- 앱에서 Markdown 포맷으로 결과를 표현하는 등 문서화 수준이 매우 높음
- HTML 페이지를 자동으로 여러 개 생성하여 기능별로 나눌 수 있도록 설계됨
- 이처럼 고도화된 문서 및 UI 출력도 Claude가 알아서 처리함

### 코드 작성 및 코드 실행 기능까지 자동 구현됨

- Claude가 앱 내에서 코드를 직접 작성할 뿐 아니라 이를 즉시 실행하는 기능도 구현됨
- 실시간으로 코드의 동작 결과를 확인할 수 있어 개발-테스트 루프가 자동화됨
- 개발자가 손을 대지 않고 코드 전체를 작성하고 실행까지 일괄처리 가능해짐

### 테마, 모델 변경, 토큰 슬라이더 등 세부 설정 기능이 풍부하게 지원됨

- 앱 내에서 테마(색상 등 UI 스타일)와 기본 LLM(모델)을 자유롭게 변경 가능함
- 최대 토큰 수를 슬라이더를 통해 실시간 조정하는 등 미세 설정을 지원함
- 사용자 편의 및 실험적 세부 조작까지 자동으로 반영된 점이 강조됨

### 전반적인 기능의 풍부함과 완성도를 인간 개입 없이 달성함

- 제작자는 ‘이렇게까지 다양한 기능을, 거의 손 하나 대지 않고 구현했다’는 점에 매우 놀라움
- 스스로 “이 정도면 정말 기능이 풍부하다(so so featurerich)”며 감탄함
- 인간의 역할이 최소화된 상황에서도 완제품 수준 결과가 도출됨

### 추가로 ‘Human-in-the-loop’ 기능을 도입하고자 하는 계획이 언급됨

- 앞으로 사용자의 피드백과 개입이 가능한 ‘Human-in-the-loop’ 기능을 추가하고 싶다고 밝힘
- 아직 완전히 인간 제어가 없는 ‘100% 자동’에 머물러 있지만, 협업구조에 대한 발전 방향 도출

### 하루만 더 작업해도 거의 완벽한 앱이 될 가능성을 시사함

- Claude가 이미 많은 기능을 구현한 상태에서, 하루만 더 자동 운용하면 ‘완벽한 애플리케이션’이 될 수 있다고 언급함
- 폭발적으로 빠르게 발전하는 LLM + 자동화 조합의 가능성을 실험으로 보여줌

### Claude Opus 4.5와 자동화 환경의 결합이 검증–개발 루프를 극단적으로 효율화함

- Claude가 자동화 도구(harness)와 연계돼 애플리케이션을 검증하고 반복 개발함
- 개발→검증→수정의 종단간 사이클을 전자동화함으로써 생산성 및 품질을 극대화함
- 최신 LLM의 실용 개발 프로세스 혁신성을 시연함

### 최신 LLM의 자동화 개발 능력에 대한 인상과 감탄을 재차 강조함

- “정말 미쳤다(crazy)”는 표현을 사용하며, Claude와 자동화 툴의 역량에 깊은 인상을 받음
- 기존의 앱 개발과는 차원이 다른 ‘자동 기능 확장력’을 실제 실험으로 증명함
- Claude Opus 4.5가 얼마나 놀라운 수준까지 고도화됐는지 직접 체감한 경험을 공유함
