## *RAG 2.0: 에이전틱 RAG와 지식 그래프를 결합한 무료 AI 에이전트 템플릿 소개* 핵심 요약

- 영상 제목은 "*RAG 2.0: 에이전틱 RAG와 지식 그래프를 결합한 무료 AI 에이전트 템플릿 소개*"로, 최신 RAG(검색 기반 생성) 전략의 결합 방식을 다룸
- 제작자는 최근 다양한 RAG 전략을 연구한 끝에 ‘에이전틱 RAG’와 ‘지식 그래프’를 반복적으로 활용하게 되었음을 강조
- 두 전략을 결합한 강력한 AI 에이전트를 직접 구현함
- 이 에이전트는 질문의 맥락에 따라 정보를 가장 잘 가져올 수 있는 데이터베이스를 스스로 판단해 검색함
- 벡터 데이터베이스로는 Neon의 서버리스 Postgres에 PG vector extension을 적용해 사용
- 문서는 대형 기술 기업들의 AI 이니셔티브와 파트너십 정보로 구성, 이를 벡터 데이터베이스에 저장하고 임베딩 생성
- 동일 문서를 Graffiti와 Neoforj(아마 Neo4j 오타)의 지식 그래프 형태로도 저장, 기업 간 관계 등 컨텍스트가 중요할 때 활용
- 에이전트는 질문 유형에 따라 벡터 검색, 그래프 검색, 또는 두 방식을 조합하여 답을 제공하며, 사용한 도구까지 명시적으로 보고함
- 사용자가 시스템 프롬프트를 조정하거나 직접 명령하여 양쪽 데이터베이스를 모두 사용할 수 있음
- 전체 템플릿은 무료로 배포하며 풀 버전 영상 및 Cloud Code 기반 구현 가이드를 추가로 제공

## 세부 요약 - 주제별 정리

### 다양한 RAG 전략을 검토한 끝에 Agentic RAG와 Knowledge Graphs의 결합이 핵심임을 파악함

- 영상 제작자는 최근 수개월간 거의 모든 RAG(검색 기반 생성) 전략을 깊이 있게 분석함
- 여러 접근법 중에서도 특히 ‘Agentic RAG’(에이전트가 상황에 맞게 정보를 탐색)와 ‘Knowledge Graphs’(지식 그래프) 두 가지 방식이 반복적으로 효과적임을 체감함
- 결국 이 두 전략을 하나의 에이전트 내에서 결합하도록 설계하게 됨

### 새로운 AI 에이전트는 질문 특성에 따라 최적의 정보원을 골라 검색함

- 제작된 에이전트는 질문의 유형에 따라 어디에서 정보를 가져올지 자체적으로 추론함
- 단순 사실 조회 질문, 관계성 탐색 질문 등 다양한 유형을 스스로 분별
- 이를 통해 항상 가장 적합한 데이터베이스와 검색 방식을 선택해 답변

### 벡터 데이터베이스에는 Postgres + Neon + PG Vector 조합을 적용함

- 벡터 데이터베이스로는 오픈소스 데이터베이스 Postgres에 PG Vector 익스텐션을 추가해 임베딩 검색이 가능하도록 구성
- Postgres는 Neon라는 서버리스(postgreSQL 클라우드 플랫폼) 상에서 운영되어 손쉽고 관리가 용이함
- 예시 문서는 대형 기술 기업들의 AI 이니셔티브와 각사 간 파트너십(협력 정보)을 포함

### 문서는 사전에 ‘청킹’(문서 쪼개기) 및 임베딩 과정을 거쳐 벡터 데이터베이스에 저장됨

- 하나의 문서를 여러 단위로 청킹(chunking)하여 적절한 임베딩(embedding) 생성
- 이 벡터화된 정보가 Postgres 데이터베이스에 저장되어 단일 기업 관련 질문 등에서 활용 가능

### 지식 그래프에는 같은 데이터를 Graffiti, Neo4j 기반으로 관계형 구조로 저장함

- 동일 문서를 지식 그래프 형태로도 별도로 구축함
- 사용된 그래프 시스템은 Graffiti와 Neoforj(오타, 실제로는 Neo4j)
- 지식 그래프는 기업 간 관계, 예를 들어 “Amazon은 Anthropic이 모델을 호스팅하는 곳” 등 컨텍스트가 중요한 정보에 효과적

### 에이전트는 질문 유형에 따라 벡터 검색, 그래프 검색, 또는 둘을 조합함

- 예시 1: “Google의 AI 이니셔티브가 무엇인가?”처럼 단일 기업에 대한 질문은 벡터 데이터베이스에서 정보 추출
  - 실제로 Postgres 벡터 DB만을 검색하고 도구 선택 내역을 반환함
- 예시 2: “두 기업 간의 관계” 등 연관성이 필요한 질문에는 지식 그래프 쿼리로 전환
  - 시스템 프롬프트에 따라 자동으로 그래프 검색으로 전환, 이 역시 도구 선택 내역 리턴
- 예시 3: 사용자가 직접 명령하거나 프롬프트를 조정할 경우, 벡터 DB와 그래프 모두에 동시 질의 가능
  - 두 자료원을 함께 활용하여 더 신뢰도 높은 답변 생성

### 에이전트는 답변뿐만 아니라 검색에 사용한 도구의 종류까지 명확히 보고함

- 에이전트는 어떤 DB(벡터/그래프)를 사용했는지 답변과 함께 명시
- 투명한 도구 사용 보고로 신뢰성과 활용도를 높임

### 시스템 프롬프트나 명령어 조정으로 에이전트의 데이터 검색 방식을 유연하게 변경할 수 있음

- 사용자는 시스템 프롬프트에서 의도를 명확히 지정 가능(예: 두 DB 모두 사용 지시)
- 특별한 코드 수정 없이도 자연어 명령이나 지시로 기능 확장

### 무료 에이전트 템플릿을 제공하며, Cloud Code 기반 구현 가이드가 포함됨

- 이 에이전트는 무료 템플릿으로 배포됨
- 영상 더보기 링크에서 전체 영상 및 Cloud Code를 활용한 구축 방법 제공
- 누구나 이 템플릿을 손쉽게 활용 가능

### 결론적으로 에이전트 RAG와 Knowledge Graph 결합은 다양한 정보 검색 시나리오에서 동적으로 최적화된 결과를 제공함

- 단일 소스에 한정되지 않고, 다양한 데이터 유형 및 질문 목적에 따라 검색 전략을 유연하게 설정 가능
- 복잡한 관계 추론, 단순 사실 검색 등 전천후 RAG 솔루션의 구체적 구현 예시를 제공함
