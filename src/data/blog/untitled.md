## *스택드 디프(스택형 변경 이력) 방식이 AI 코드 리뷰의 효율을 극대화함* 핵심 요약

- 영상 제목은 "스택드 디프가 AI를 활용한 코드 리뷰 방식을 어떻게 바꾸었는가"임
- 기존의 일반적인 풀리퀘스트(PR)는 한 번에 큰 변경을 진행하고, 승인 대기 후 다음 변경을 반복함
- 스택드 디프 방식에서는 작은 변경들을 여러 개의 PR로 쪼개어 연속적으로 쌓으며, 각 변경이 이전 변경 위에 쌓이는 방식
- 각 스택 PR은 독립적으로 AI 코드 리뷰 피드백을 받을 수 있어, 보다 집중적이고 효과적인 검토가 가능함
- 이 워크플로우를 최적화하여 지원하는 플랫폼으로 Graphite가 소개됨
- Graphite는 CLI(명령줄 도구)와 대시보드를 제공, 스택드 디프 생성·관리·AI 코드 리뷰를 한 곳에서 처리 가능
- 플랫포옴 가입 시 직관적인 데모를 제공하여, 첫 스택드 디프 PR 생성 과정을 단계별로 안내함
- GitHub의 워크플로우와 유사하나, 각 기능별 브랜치가 서로 위에 쌓이도록 하여, 메인 브랜치로의 머지까지 효율적으로 관리함
- 생성된 각 PR에 대해 Claude와 같은 AI 도구가 개별적으로 리뷰하여, 전체 대규모 변경보다 세부적이고 유의미한 리뷰 결과 도출
- Graphite의 대시보드를 통해 PR 생성, 리뷰, 병합까지 GitHub에 직접 접속하지 않고도 한 번에 처리할 수 있음

## 세부 요약 - 주제별 정리

### AI 코드 리뷰가 워크플로우에서 필수 도구로 자리잡았음을 강조함

- AI를 활용한 코드 리뷰는 매우 강력하며, 본인에게 있어서 검증 과정의 필수 도구로 간주됨
- AI 검토가 일반화된 일상 업무가 되어가고 있음을 시사
- 하지만 AI 코드 리뷰의 효율을 극대화하기 위해서는 적합한 워크플로우가 필요하다고 강조

### 전통적인 풀리퀘스트 기반 워크플로우의 한계를 설명함

- 기존 풀리퀘스트(PR)의 일반적인 진행 방식 소개
- 한 번의 변경 사항을 만들고, 승인 대기 후 다음 변경을 반복하는 절차를 거침
- 승인 대기 때문에 생산성이 저하될 수 있고, 리뷰 범위가 크면 AI의 피드백의 질도 저하될 수 있음

### 스택드 디프는 작은 단위의 변경을 연속적으로 쌓는 방식을 지향함

- 스택드 디프(workflow)는 대규모 변경을 쪼개어 여러 개의 소규모 PR로 분할함
- 각 단위 PR은 이전 PR 위에 쌓이며, 단계별로 독립적이면서도 체계적으로 기능 추가를 확장
- 단일 PR별로 승인 대기를 하지 않고, 연속적으로 작업 진행 가능
- 개발 프로세스의 병목을 줄이고, 신속한 피드백을 유도

### 스택드 디프가 AI 코드 리뷰 효율을 크게 향상시킴

- 잘게 쪼개진 각 변경(PR)마다 AI가 “매우 집중적”이고 “정확한” 피드백을 제공할 수 있음
- 한꺼번에 하나의 커다란 PR을 리뷰하는 데 비해 세밀하고 체계적인 검토가 가능
- 각 변경 사항별 개선점이나 오류를 빠르게 탐지하여 품질 향상

### Graphite 플랫폼이 스택드 디프 작업을 극대화해서 지원함

- Graphite라는 서비스를 활용하여 스택드 디프 방식의 작업 최적화를 소개
- GitHub CLI와 유사한 명령줄 인터페이스와 함께, 스택드 디프에 특화된 대시보드 제공
- 저장소 내 스택드 디프 PR 생성, 관리, 병합 과정을 모두 지원함
- 대시보드는 PR별로 상태 및 작업 내역, 리뷰 결과 등을 관리할 수 있음
- AI 코딩 어시스턴트(예: Graphite 에이전트)와 연동 가능

### 가입 직후 Graphite의 직관적 데모가 스택드 디프 적응을 용이하게 함

- Graphite 플랫폼 가입 시, 매우 직관적이고 단계별로 안내하는 데모 제공
- 첫 번째 스택드 디프 생성에서 풀리퀘스트 생성, 다음 PR 누적으로 진행하는 방법 설명
- 명령줄에서 직접 3개의 연속된 PR을 만들고 쌓는 것을 시연함
- Git 및 GitHub CLI 이용 경험이 있다면 쉽게 적응 가능함을 시사

### Graphite 대시보드에서 직관적인 스택드 디프 상태 관리 가능

- 최종적으로 PR 3개가 쌓인 형태(스택드 디프)를 Graphite 대시보드에서 확인 가능
- 각 PR의 병합(Merge), 리뷰 진행, PR 관리(prediction)를 대시보드 내에서 직접 처리 가능
- GitHub 저장소로 이동하지 않고도, 대시보드에서 모든 워크플로우를 마무리할 수 있음

### 각 풀리퀘스트는 깃 브랜치를 활용해 기능 단위로 쌓아 올려집니다

- 각 스택드 디프 PR은 기능별 브랜치에서 생성되어, 상위 PR에 쌓이는 구조
- 최종적으로 각 브랜치를 메인 브랜치에 병합하여 전체 기능을 통합 관리
- Git의 브랜치, 머지 원리를 “더 지능적으로” 적용한다는 점을 강조

### Claude와 같은 AI 툴을 통해 각 변경 이력을 독립적으로 효과적으로 검토함

- 각 스택드 디프 PR마다 Claude(Code) 등의 AI 도구로 개별 리뷰를 요청
- 한 번에 하나의 큰 PR을 검토할 때보다, 세 개의 포커스된 PR로 검토할 때의 결과가 “획기적으로 다르다”고 언급
- 결과적으로, 더 깊이 있고 실질적인 품질 개선을 실현 가능

### 영상의 결론은 스택드 디프 방식과 Graphite의 결합이 AI 코드 리뷰의 생산성과 품질을 혁신한다고 밝힘

- 모든 변화 기록을 스택드 디프로 쪼개 작업하면서 AI가 각 변경을 꼼꼼히 리뷰하도록 유도
- Graphite와 AI Code Review의 결합이 기존 대형 PR 위주의 방식 대비 획기적인 효율과 효과를 제공함
- 더 자세한 사용법에 대한 영상은 하단 설명란에 별도 링크로 제공됨
