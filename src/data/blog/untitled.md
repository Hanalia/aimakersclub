## *RAG 2.0: 에이전틱 RAG와 지식 그래프 통합(무료 템플릿 소개)* 핵심 요약

- 영상 제목: **RAG 2.0: 에이전틱 RAG와 지식 그래프 통합(무료 템플릿 소개)**
- 제작자는 최근 몇 달간 다양한 RAG(검색 증강 생성, Retrieval Augmented Generation) 전략을 연구해 왔으며, 그중 에이전틱 RAG(Agentic RAG)와 지식 그래프를 특히 유용하게 평가함
- 두 전략을 융합한 강력한 AI 에이전트를 직접 구현하여 소개
- 이 에이전트는 질문의 성격에 따라 최적의 정보 소스를 스스로 판단하여 탐색 방법을 결정
- 벡터 데이터베이스로는 Postgres 및 PG vector 확장, 그리고 서버리스 Postgres 플랫폼인 Neon을 활용
- AI 및 빅테크 기업의 정보, 이들의 AI 이니셔티브, 상호 파트너십 데이터 등을 포함한 문서를 임베딩화하여 벡터 DB에 저장
- 동일한 문서를 Graffiti와 Neo4j 기반 지식 그래프에도 관계형 데이터로 저장, 기업 간 연관 질의에 최적화함
- 에이전트는 단일 기업 정보를 묻는 질문에는 벡터 DB를, 두 개 이상의 기업 간 관계 질문에는 지식 그래프를 자동 선택해 검색
- 질문에 따라 어느 도구(벡터 DB/그래프 DB)가 사용되었는지 사용자에게 명확히 설명
- 시스템 프롬프트 조정이나 명시적 지시를 통해 두 데이터 소스를 동시에 활용도 가능
- 코드는 무료 템플릿으로 제공되며, Cloud Code를 활용한 빌드 방식 및 세팅법은 전체 강좌에서 자세히 확인 가능

## 세부 요약 - 주제별 정리

### 에이전틱 RAG와 지식 그래프는 RAG 실전 전략에서 반복적으로 선택되는 핵심 방식임

- 제작자는 다양한 RAG(검색 증강 생성) 전략을 수개월간 실험 및 연구함
- 그 결과, 지속적으로 손이 가는 두 가지 접근법이 '에이전틱 RAG'와 '지식 그래프' 형식임을 강조
- RAG 내에서 정보 탐색 효율성과 관계형 질의의 필요성을 두 방식으로 보완한다고 판단
- 새로운 시도로, 이 두 방식을 하나의 강력한 AI 에이전트에 통합하기로 결정함

### 에이전트는 질문 별로 최적의 정보탐색 경로를 자율적으로 판단함

- 구축된 AI 에이전트는 사용자가 입력한 질문과 시스템 프롬프트를 바탕으로 적절한 데이터 소스를 선택함
- 정보 탐색의 목적(예: 단일 정보 vs 관계 분석)에 따라 벡터 DB 또는 지식 그래프 중 하나 혹은 두 개를 활용하도록 설계
- 에이전트의 이러한 의사결정은 reasoning(추론) 절차에 기반함

### 벡터 데이터베이스는 Postgres + PG vector + Neon으로 구축됨

- 문서 임베딩과 검색을 위한 벡터 데이터베이스로 Postgres를 사용
- PG vector 확장(extension)을 통해 Postgres에서 벡터 검색이 가능하도록 함
- 서버리스(Postgres as a Service) 플랫폼인 'Neon'을 선택해서 구축 및 확장성 확보
- 문서를 chunk(조각)로 나누고, 텍스트 임베딩을 생성해 벡터DB에 저장함

### 저장되는 데이터는 빅테크 기업 AI 이니셔티브 및 파트너십 정보를 포함함

- 예시 문서는 빅테크 기업의 정보 및 각 사의 AI 관련 이니셔티브, 그리고 이들 기업 간 다양한 협력 구조(파트너십 등)를 세세하게 담고 있음
- 문서에는 구체적으로 Google, Amazon, Anthropic 등과 같은 대기업의 AI 사업 및 상호 관계에 대한 정보가 포함됨

### 동일 데이터를 Graffiti와 Neo4j로 지식 그래프 형태로도 저장함

- 같은 내용을 Graffiti와 Neo4j를 활용하여 지식 그래프에 저장
- 이 때 데이터는 '관계형'(relational) 구조로 설계되어 기업 간의 역할, 협력, 소유구조 등 관계 질의에 최적화됨
- 예: "Amazon은 Anthropic의 모델 호스팅처"와 같은 관계 정보를 질의 가능

### 질문 유형에 따라 벡터 DB와 지식 그래프를 자동 전환하며 검색함

- 단순한 기업 정보(예: "Google의 AI 이니셔티브는?")를 묻는 경우, 에이전트는 벡터 데이터베이스만 검색하도록 설정됨
- 사용자는 실제 화면에서 벡터 DB만 사용되어 답변이 도출되는 과정을 확인함
- 그 반면, "Amazon과 Anthropic의 관계는?"과 같이 관계성이 필요한 질문에는 시스템 프롬프트에 의해 에이전트가 지식 그래프를 탐색함

### 답변과 함께 사용된 도구(탐색 경로)를 명확히 추적/알려줌

- 답변 도출 후, 시스템은 이번 질의에 '어떤 도구(벡터 db, 그래프 db 등)'를 활용했는지 투명하게 표시
- 각 단계별로 실제로 벡터 DB를 썼는지, 그래프 검색을 했는지 사용자 확인 가능
- 특정 질문에서는 두 소스를 동시에 활용해 보다 심층적이고 정확한 답을 도출함

### 프롬프트와 명시적 명령을 통해 원하는 검색 방식을 자유롭게 선택 가능함

- 시스템 프롬프트를 조정해 에이전트가 기본적으로 어느 소스를 우선적으로 사용하는지 설정 가능
- 또는, 사용자가 "벡터DB와 그래프DB를 모두 활용해 답하라"고 명시적으로 요청해서 병행 검색이 가능
- 이렇게 하면 두 오리지날 데이터 소스에서 모두 검색해 종합 답변을 도출하게 됨

### 개발과정은 Cloud Code 등 자동화/생산성 도구로 효율화함

- 전체 코드는 Cloud Code 플러그인을 이용해 에이전트 개발, 배포 과정을 간소화
- 소스코드는 무료 템플릿으로 직접 제공하여, 누구나 쉽게 자신의 환경에 도입 가능하도록 배려함

### 전체 데모 및 구체 구축 방법은 전체 영상에서 자세히 확인 가능함

- 영상 말미에서, 전체 에이전트 빌드/직접 구축법은 본 영상에 첨부된 링크와 전체 강좌에서 자세하게 설명함을 안내
- 누구나 무료 템플릿과 안내 영상을 통해 유사 솔루션을 직접 구현해 볼 수 있도록 구조 제공
