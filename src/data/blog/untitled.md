## *MCP 서버의 큰 문제점과 그 해결책* 핵심 요약

- MCP(Multi-Component Protocol)은 인기를 얻으면서도 분명한 한계가 드러남
- 핵심 문제는 '토큰 소모'와 '컨텍스트 왜곡(context rot)'으로, 현재 AI 분야의 주요 이슈임
- 여러 MCP 서버를 동시에 사용할 때, 각 서버의 기능을 설명하기 위해 AI에 수천 개의 토큰이 소비됨
- 영상 제작자는 일반적으로 5개의 표준 MCP AI 코딩 서버를 활용하지만, 이들을 한 세션에서 동시에 쓰지 않는다고 강조
- 동일 세션에서 여러 MCP 서버를 사용하면 토큰 소모가 급격히 심각해짐
- 이 문제를 해결할 강력한 방안으로 '에이전트 스킬(agent skills)' 개념이 등장
- 최근 Anthropic이 공개한 'agent skills'는, AI 에이전트가 기능 수행용 스크립트를 자동 생성·실행할 수 있도록 해줌
- 이를 통해 API 엔드포인트와 상호작용하거나 즉석에서 코드 생성이 가능해져 토큰 소모 문제 해소 기대
- 영상은 MCP 시스템의 문제점과 'agent skills' 기반 코드 실행 솔루션을 구체 예시와 함께 세부적으로 설명함
- 끝으로 "우리가 알고 있던 MCP의 시대는 끝났는가?"라는 근본적 질문을 던짐

## 세부 요약 - 주제별 정리

### MCP는 인기를 얻었지만 토큰 소모와 컨텍스트 왜곡이라는 치명적 문제가 동시에 발생함

- MCP(Multi-Component Protocol)는 AI 코딩 인프라 등 여러 분야에서 활용도가 빠르게 높아지고 있음
- 그러나 프로토콜이 널리 쓰일수록 '토큰 소비량 증가'와 '컨텍스트 왜곡(context rot)' 문제가 명확히 부각됨
- 이 이슈들은 현재 AI 시스템 개발 커뮤니티에서 가장 뜨거운 화두 중 하나임

### 여러 MCP 서버를 동시에 사용할 경우 토큰 사용량이 급격히 늘어난다는 구체적인 사례가 영상에서 제시됨

- 영상 속에서 제작자는 실제로 5개의 표준 MCP AI 코딩 서버를 예시로 제시함
    - 과거 이 서버들을 각각 다양하게 활용했으나, '동일 세션에서 동시에 사용하지 않는다'고 강조함
- 각각의 MCP 서버는 자신의 기능(역할 및 API 등)을 AI 에이전트에게 전달하는데 수천 개의 토큰이 필요
- 서버가 여러 개일수록 토큰 사용량이 기하급수적으로 증가해 시스템 비효율이 심각해짐

### 토큰 소모와 컨텍스트 왜곡 문제는 AI 에이전트 활용에 실질적 장애물이 됨

- 각 MCP 서버의 기능 설명(프롬프트 등)만으로도 이미 방대한 토큰이 소비됨
- 전체 토큰 한도가 빠르게 소진돼 중요한 맥락 정보가 누락되거나 덮어씌워지는 '컨텍스트 왜곡' 현상이 심화
- 이로 인해 실제 서비스를 설계 및 운영할 때 원활한 AI 협업 및 연동이 어려운 구조가 됨

### MCP 문제의 본질을 극복하기 위해 '에이전트 스킬'이 새로운 해결책으로 등장함

- 이 문제에 대해 Anthropic사가 최근 'agent skills' 기능을 출시함(구체적 출시 시점은 영상상 미언급)
- '에이전트 스킬'은 본질적으로 AI가 필요한 기능 스크립트를 세션 내에서 즉시 생성하고, 필요할 때마다 실행하도록 함
- 프롬프트에 상세 기능을 매번 입력하는 방식이 아닌, 필요 시점에 자유롭게 기능을 호출할 수 있는 구조임

### 'Agent skills' 솔루션은 API 사용, 스크립트 생성, 코드 실행 문제를 한 번에 해결할 수 있음

- 'Agent skills'를 사용하면, AI 에이전트가 API 엔드포인트와 직접 상호작용이 가능
- 또한 API 사용법에 대한 명령문(script 또는 instruction)을 즉석에서 생성
- 기존 MCP 방식의 '서버별 프롬프트 사전 전달'을 대체하여 토큰 소모를 획기적으로 줄여줌
- 실시간 코드 생성 및 실행(Code execution on the fly)도 이 기술의 대표적 강점으로 언급됨

### 영상은 MCP 문제의 위험성을 '빠르게 심각해질 수 있다'는 무서운 예시로 직관적으로 보여줌

- 5개 MCP 서버의 각 기능을 일일이 AI에게 설명하는 데만 이미 수천 토큰 소모
- 만약 여러 MCP 서버를 동시에 세션에 투입하면, 토큰 한도가 금새 소진되어 작업 전체가 비효율적으로 변모할 수 있음
- 이러한 상황이 "얼마나 빠르게(How quickly) 통제 불능이 되는가"를 실제로 보여주기 위한 의도적인 예시임

### Anthropic의 'agent skills'는 MCP의 구조적 한계를 근본적으로 뒤집을 잠재력이 있음을 강조함

- 기존 MCP 시스템 구조에서는 확장성과 효율성 모두에서 제약이 많았음
- 'Agent skills'는 필요 기능만을 즉석 호출·생성함으로써 시스템 전체의 효율성을 크게 제고할 수 있음
- 영상 제작자는 이 방식을 "정말 강력한(stuff)"이라고 표현하며 혁신적임을 시사

### 영상 말미에 "우리가 알고 있던 MCP의 시대는 끝난 것인가?"라는 근본적 의문을 제기함

- 'agent skills' 솔루션의 도입으로 MCP 기반 시스템 전체의 패러다임 변화 가능성 언급
- 기존 MCP 방식은 신규 기술에 의해 점차 대체될 수 있음을 '최종 질문(final question)'으로 제시
- 이 테마를 끝으로 영상은 마무리됨

### 영상 전체는 MCP 구조 문제 → 구체적 예시 → 혁신적 솔루션 → 시대적 변화 가능성이라는 순서로 구성되어 있음

- 문제 제기: MCP의 인기와 함께 드러나는 토큰 소모-컨텍스트 왜곡 이슈
- 실전 예시: 5개 AI 코딩 MCP 서버를 한 세션 내 사용 시 불합리성
- 혁신적 제안: Anthropic의 'agent skills'와 코드 실행 기반 솔루션
- 근본적 질문 제시: "MCP의 시대는 끝인가?"
