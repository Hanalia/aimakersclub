## *RAG 2.0: 에이전틱 RAG와 지식 그래프 결합 템플릿 소개* 핵심 요약

- 최근 몇 달간 다양한 RAG(Retrieval Augmented Generation) 전략을 연구해 온 결과, 에이전틱 RAG(Agentic RAG)와 지식 그래프(Knowledge Graphs)가 특히 유효함을 확인함
- 영상에서는 두 가지 전략을 결합한 강력한 AI 에이전트를 직접 구현하여 선보임
- 이 AI 에이전트는 사용자의 질문에 최적의 답을 도출하기 위해 어디에서 정보를 검색할지 스스로 추론함
- 벡터 데이터베이스는 Postgres + PG Vector 확장과 Neon 서버리스 플랫폼을 활용해 구축됨
- 문서는 청킹(chunking) 및 임베딩(embedding)을 거쳐 벡터 데이터베이스에 저장됨
- 동일한 문서를 Graffiti와 Neo4j를 이용해 관계형 형태로 지식 그래프에 병행 저장함
- 벡터 DB는 "구글의 AI 이니셔티브는 무엇인가?"처럼 단일 회사 중심의 질문에 효과적임
- 두 기업 간의 관계와 같이 복합적인 질의에는 시스템 프롬프트에 따라 지식 그래프를 쿼리하여 정확한 답변 제공
- 사용자가 벡터 DB와 그래프 모두를 활용하도록 명시할 수도 있으며, 에이전트가 두 소스를 모두 사용하여 근거 있는 답변 생성 가능
- 이 AI 에이전트는 무료 템플릿으로 제공되며, 추가 영상을 통해 Cloud Code를 이용한 구축 방법도 안내됨

## 세부 요약 - 주제별 정리

### 에이전틱 RAG와 지식 그래프의 결합이 RAG 전략 중에서 가장 효과적임을 발견함

- 영상 제작자는 다양한 RAG 전략에 대해 수개월간 심도 있게 연구해 옴
- 그 중에서도 에이전틱 RAG와 지식 그래프 기반 접근법이 반복적으로 좋은 결과를 보여줌
- 이 두 가지 방식을 결합하면 각각의 한계를 상호 보완할 수 있다는 점이 관찰됨

### AI 에이전트가 다양한 데이터 소스를 자동으로 활용하며 최적의 답을 추론함

- 새로 만든 AI 에이전트는 질문의 특성에 따라 정보 검색 경로를 스스로 판단함
- "어떤 소스에서 정보를 찾는 게 질문에 더 적합할지" 자율적으로 reasoning(추론) 가능
- 이로써 단순 검색 이상의 응답 품질과 정확도를 기대할 수 있음

### 벡터 데이터베이스는 Postgres와 Neon, PG Vector로 손쉽고 강력하게 구현됨

- 벡터 데이터베이스의 기반은 Postgres 데이터베이스와 PG Vector 확장 활용
- 서버리스 환경에서는 Neon이라는 플랫폼을 사용함
- 문서 데이터는 여러 청크로 분할(chunked) 후, 각각 임베딩(embedding)되어 저장됨
- 문서에는 빅테크 기업들 및 이들의 AI 이니셔티브, 그리고 기업 간 파트너십 등 다양한 정보가 포함됨

### 지식 그래프는 Graffiti와 Neo4j를 활용해 관계 중심으로 동일 정보를 구축함

- 앞서 벡터 DB에 저장된 동일 정보를 바탕으로 지식 그래프도 별도로 구축
- Graffiti 및 Neo4j가 지식 그래프 구축과 저장에 사용됨
- 텍스트 정보가 노드와 관계(relationship)형태로 저장돼, 예컨대 "Amazon은 Anthropic의 모델 호스팅 기업"과 같은 복잡한 관계 질의가 가능함

### AI 에이전트가 질문의 의도에 따라 자동으로 벡터 검색 또는 그래프 쿼리를 선택함

- 단일 기업에 대한 정보 질의(예: "Google의 AI 이니셔티브는?")에는 벡터 데이터베이스를 선호적으로 검색함
- 실제로 Google 관련 질문을 하자 에이전트는 벡터 DB만을 활용하고, 사용된 도구와 결과를 명확히 알려줌
- 질의가 두 기업 간 관계(예: "Amazon과 Anthropic의 관계") 등이라면, 에이전트가 지식 그래프 쿼리를 이용해 결과를 도출함
- 시스템 프롬프트에 따라 어느 도구를 이용할지 자동 결정됨

### 필요에 따라 사용자가 양쪽 데이터 소스를 모두 활용하도록 제어할 수 있음

- 사용자는 시스템 프롬프트를 조절해 에이전트에게 "벡터와 그래프 둘 다 사용하라"고 명시할 수 있음
- 혹은 명령어에서 직접적으로 양쪽 소스를 호출토록 지정 가능
- 이런 경우 에이전트는 두 소스 모두에서 정보를 찾아, 더욱 신뢰성 있는 답변과 근거(output)를 생성

### 실제 동작 예시를 통해 각 도구 사용 현황과 결과를 명확히 검증할 수 있음

- 각 질의마다 에이전트가 사용한 데이터 소스(벡터 검색, 그래프 검색 등)를 반환
- 벡터 DB만 사용한 경우, 그래프만 사용한 경우, 양쪽 모두를 사용한 경우 결과를 한 눈에 확인 가능
- 이런 명확한 피드백은 시스템 신뢰성 및 투명성 확보에 기여함

### 이 에이전트는 무료 템플릿으로 제공되고 Cloud Code 활용법 등 추가 안내를 함

- 영상 내 무료로 쓸 수 있는 에이전트 템플릿 링크를 안내함
- 추가 영상에서는 Cloud Code를 이용해 본 에이전트를 구축하는 상세 방법도 시연함
- 누구나 직접 실행 및 개발이 가능하도록 실질적인 리소스를 함께 제공

### 복합적 쿼리와 동적 소스 선택이 가능한 에이전트 구조가 RAG와 지식 그래프의 한계를 해결함을 보여줌

- 벡터 DB와 지식 그래프 각각의 장점을 통합하여 복합 쿼리에 대응할 수 있음
- 사용자가 질의 형태에 따라 자료구조의 장점을 극대화하기 쉽도록 설계
- AI 시스템이 자체적으로 reasoning을 수행해 여러 데이터 소스 중 적합한 곳을 선택함으로써, 기존 RAG 방식 대비 효율성과 신뢰도를 높임

### 영상은 실전 구축, 예시 시연, 그리고 무료 배포 템플릿 안내 정보를 모두 아우름

- 영상 내 실제 예시, 세팅법, 시스템 동작 방식 등을 자세히 소개
- 개발 도구 명칭, 구축 스택, 문서 관리 방법 등 실무적 세부사항도 빠짐없이 설명함
- 최종적으로 시청자가 바로 따라 구현할 수 있도록 무료 템플릿 및 추가 안내 리소스를 제공함
