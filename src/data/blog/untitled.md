## *모두가 놓치고 있는 AI 에이전트의 대규모 확장: 메모리와 도구 통합* 핵심 요약

- 본 영상은 대부분의 AI 에이전트 및 MCP 서버(멀티채널 플랫폼 서버)가 오직 한 명의 사용자만을 위해 설계되었다는 점을 지적함
- 단일 사용자에 한정된 설계는 일부 상황에선 괜찮지만, 수천~수백만 명의 사용자에게 맞춤형 에이전트를 제공하려면 문제 발생
- 일반적으로 에이전트나 MCP 서버를 구축할 때, 각 툴(예: Gmail, Slack, Asana 등)에 자격증명을 하드코딩하는 방식을 사용함
- 이 방식은 오직 한 명(예: 특정 Gmail 계정)만 사용할 수 있고, 여러 사용자에 확장 적용이 불가능함
- 각기 다른 사용자가 개별 계정에 에이전트를 연결하려면 인증 및 권한 부여가 복잡해짐
- Arcade 플랫폼은 OAUTH 방식을 활용해, AI 에이전트가 동적으로 사용자 계정 접근 권한을 취득하게 해줌
- Arcade는 자격증명을 안전하게 저장해 이후 추가 요청 필요 없이 에이전트가 각 사용자 계정에서 작업 수행 가능하게 함
- Langgraph 연동 지원으로 에이전트 인증 패턴 시각화 및 워크플로우 자동 처리가 가능함
- 데모에서는 Asana 계정 연동 예시를 통해, 사용자가 최초로 권한 승인을 완료하면 Langgraph 워크플로우가 자동으로 재개되고, 해당 계정의 Asana 작업들이 정상적으로 출력됨
- 이러한 구조는 사용자별 맞춤화 및 장기적 메모리와 결합해, 수많은 사용자를 대상으로 확장 가능한 AI 에이전트 제작 가능성을 제시함

## 세부 요약 - 주제별 정리

### 대부분의 AI 에이전트는 한 명의 사용자만 위한 구조로 설계되고 있음

- 현재 사용되는 AI 에이전트 및 MCP 서버(Multi Channel Platform Server)는 대부분 단일 사용자만을 위해 만들어진다.
- 개발자는 대부분 본인 자신만을 위한 에이전트를 만드는 데 그치고 있다.
- 이 방식은 개인용 혹은 소규모 테스트에 유효하지만, 대규모 서비스에는 제한적이다.
- 영상에서는 더 많은 사용자, 즉 수천 또는 수백만 명이 동시에 사용할 수 있는 맞춤형 에이전트의 필요성을 제기함

### 단일 계정 하드코딩 방식은 확장성을 가로막는 근본적 문제로 지적됨

- 일반적으로 개발자가 AI 에이전트나 MCP 서버를 만들 때, 예시로 N8N과 같은 워크플로 툴을 사용한다.
- 각종 툴(Gmail, Slack, Asana 등)과 에이전트를 연결할 때 계정 자격증명을 하드코딩한다.
- 결과적으로 한 명의 Gmail이나 Slack 계정에만 연동이 가능하고, 여러 사용자가 각자 다른 계정으로 사용할 수 없다.
- 실제 서비스로 확장할 때, 각 사용자가 자신의 계정에 AI 에이전트를 연결하기 위해서는 복잡한 인증 로직이 필요하다.

### Arcade 플랫폼은 OAUTH 기반 인증으로 이 문제를 효과적으로 해결함

- Arcade 플랫폼은 AI 에이전트가 OAUTH 인증 방식을 통해 각 사용자의 계정에 접근 권한을 동적으로 요청하도록 해준다.
- 사용자는 자신의 계정에 대해 AI 에이전트의 액세스 요청을 승인할 수 있다.
- 승인된 후에는 Arcade가 자격증명을 안전하게 저장하여, 이후 에이전트가 추가적인 인증 절차 없이도 계속 액세스가 가능하다.
- 이로써 모든 사용자가 자신의 계정에서 개별적으로 에이전트를 사용할 수 있게 된다.

### Langgraph 통합으로 설계와 워크플로우가 시각적으로 간편해짐

- Arcade는 Langgraph와 통합되어 있다.
- Langgraph 상에서 각 도구(툴)에 대한 인증 패턴과 워크플로우를 시각적으로 설계할 수 있다.
- 에이전트가 어떤 서비스(예: Gmail, Asana)를 처음 호출하면, 사용자 인증 절차(권한 승인 플로우)가 자동으로 동작한다.
- 인증이 완료되면 다시 Langgraph 워크플로우로 복귀하여 원래 요청한 작업을 이어서 처리한다.

### 실습 예시: Asana 계정 연동 시나리오 시연

- 영상에서는 Asana 연동 예제를 선정해 실습을 보여준다.
- 예시 질문: "SaaS 아이디어 프로젝트에 어떤 할 일이 있지?"와 같이 에이전트에게 Asana 작업 목록을 요청한다.
- 에이전트가 Asana에 접근을 시도하면, 최초에는 사용자 권한 승인이 필요하다는 사실을 인식함
- 사용자는 에이전트에게 본인 Asana 계정 접근권한을 부여하는 OAUTH 인증 프로세스를 따라야 한다.
- 인증이 완료되면, 워크플로우가 자동 재개되고, 에이전트는 실제로 해당 계정에서 작업 목록을 추출하여 반환한다.

### 여러 사용자가 동시에 사용하는 대규모 맞춤화 에이전트 제작이 가능해짐

- Arcade와 Langgraph의 조합으로, 사용자별 계정 접근 및 인증이 자동화되어 개인 맞춤형 동작이 가능해진다.
- 사용자가 몇 명이든(수천~수백만 명), 에이전트가 각자의 계정에 적절히 연결되어 독립적으로 동작할 수 있다.
- 계정별 권한 처리와 장기 기억(Long-Term Memory) 기능을 결합하면, 각 사용자별 하이퍼퍼스널라이즈드(초맞춤형) 에이전트 구축이 쉬워짐

### 권한 및 계정 관리의 자동화로 개발 및 보안 부담이 크게 줄어듦

- Arcade가 자격증명의 저장 및 호출을 대신 처리해주므로, 각 개발자는 직접 보안 로직을 다루거나 API 요청마다 사용자 로그인 절차를 구현할 필요가 없다.
- 보안성과 범위(Scopes)를 OAUTH로 관리하므로, 사용자는 본인 데이터에 대한 통제를 할 수 있고, 개발자는 책임 분산의 이점을 얻음.

### 언제든지 새로운 서비스와 워크플로우 추가가 가능함

- 기본 구조가 확장에 용이하도록 설계되어 있어, Gmail 외에 Slack, Asana, 기타 업무 툴 등 다양한 서비스로 손쉽게 확대 가능하다.
- 새로운 도구 추가, 인증 및 권한 승인 로직은 Arcade와 Langgraph 내에서 재사용 및 자동화가 가능하다.

### 영상의 데모와 추가 자료는 실습을 원하는 개발자에게 도움을 제공함

- 실제 데모 영상 링크가 별도로 안내되어 있어, 개발자가 Arcade 및 Langgraph 환경에서 실습 과정을 직접 따라할 수 있다.
- 이 데모를 통해 OAUTH 프로세스, 인증 후 워크플로우 재개, 다수 사용자 지원 구조 등을 구체적으로 확인 가능하다.
