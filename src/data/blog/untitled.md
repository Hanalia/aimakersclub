## *RAG 2.0: 에이전틱 RAG + 지식 그래프 (무료 템플릿)* 핵심 요약

- 영상 제목은 “RAG 2.0: 에이전틱 RAG + 지식 그래프(무료 템플릿)”이며, 최신 RAG 전략 두 가지를 결합한 AI 에이전트를 소개함
- 제작자는 수개월간 다양한 RAG 방식 중 에이전틱 RAG(Agentic Rag)와 지식 그래프 전략에 주목했음을 밝힘
- 두 방식을 결합한 강력한 AI 에이전트를 직접 구현하여 소개함
- 이 에이전트는 질문의 성격에 따라 정보를 검색할 데이터소스(벡터 데이터베이스 또는 지식 그래프)를 스스로 판단함
- 백엔드 인프라로는 Postgres 데이터베이스에 PG Vector 확장기능을 사용하며, 서버리스 플랫폼 Neon을 활용함
- 문서 정보를 벡터 임베딩 형태로 저장하고, 동일 문서를 Graffiti와 Neo4j로 구축한 지식 그래프에도 저장함
- 에이전트는 “Google의 AI 이니셔티브”와 같은 질문에는 벡터 데이터베이스 검색을, 기업 간의 관계를 묻는 질문에는 지식 그래프 검색을 사용함
- 사용자가 원하면 에이전트가 두 소스를 동시에 활용해 답변하도록 명령할 수도 있음
- 데모를 통해 각 경우에 실제 어떤 툴이 사용되었는지 결과를 보여줌
- 이 AI 에이전트의 구현 코드 및 템플릿은 무료로 제공하며, 전체 제작 과정을 설명하는 영상 링크도 안내함

## 세부 요약 - 주제별 정리

### 제작자는 RAG 전략 중 에이전틱 RAG와 지식 그래프 방식에 집중하게 됨

- 최근 수개월간 다양한 RAG(Retrieval-Augmented Generation) 방식과 전략을 실험하며 연구함
- 그 중 에이전틱 RAG(Agentic Rag)와 지식 그래프(Knowledge Graph) 기반 방법론이 반복적으로 가장 효과적이라는 결론에 도달함
- 두 방식 모두 각각의 장점이 있으며, 서로 결합하면 시너지가 크다고 판단함

### 에이전트는 질문 유형에 따라 정보 검색 방식을 스스로 선정함

- 제작된 AI 에이전트는 질문의 성격을 분석해, 어떤 데이터 소스를 이용해야 최적의 답이 나올 지 스스로 추론함
- 구체적으로, 단일 객체/기업에 관한 질문에는 벡터 데이터베이스를 주로 활용함
- 관계성 탐구나 연관성 분석이 필요한 경우에는 지식 그래프를 선택함
- 이로써 더 효율적이고 정확한 정보 검색이 가능해짐

### 벡터 데이터베이스는 Postgres, PG Vector, Neon으로 최적화함

- 벡터 DB로 Postgres를 사용하며, 고차원 벡터 처리를 위해 PG Vector 확장을 활용함
- 서버리스 서버 환경은 Neon 플랫폼을 채택하여 구동함
- 예시 문서가 준비되어있고, 해당 문서를 일정 크기로 나눠(Chunking) 벡터 임베딩을 생성함
- 문서에는 빅테크 기업, 이들의 AI 전략, 그리고 상호 파트너십 정보가 포함되어 있음

### 동일 정보를 지식 그래프로도 구축하여 관계성 탐색이 가능함

- 동일 문서를 기반으로 Graffiti(그래피티)와 Neo4j(네오포제이)를 사용, 지식 그래프 형태로도 데이터를 저장함
- 지식 그래프는 정보의 관계성을 구조적으로 표현할 수 있어 복잡한 기업 간 연관성 탐색에 용이함
- 예시로 “Amazon이 Anthropic의 모델을 호스팅한다”와 같은 관계적 정보를 탐색할 수 있도록 설계됨

### 에이전트의 데이터소스 선택 데모를 구체적으로 시연함

- “Google의 AI 이니셔티브는 무엇인가?”와 같이 단일 기업에 대한 질문을 제시
- 에이전트는 벡터 데이터베이스 검색을 선택하여 답변을 도출함
- 실제 검색 결과와 어떤 툴(벡터 DB)이 사용되었는지 에이전트가 명확히 안내함

### 관계성 질문 시 에이전트는 지식 그래프 사용으로 자동 전환함

- “두 기업 간 협력이나 파트너 관계 등 기업 간 연결성”을 묻는 질문 제시
- 사전 설정한 시스템 프롬프트에 따라 에이전트가 지식 그래프 검색으로 전환함
- 결과적으로 지식 그래프 브라우징 툴을 활용해 답변하고, 사용 툴 정보를 명시함

### 두 데이터소스 동시 활용도 에이전트가 지원함을 설명함

- 사용자가 명시적으로 “두 소스 모두 활용하라”며 프롬프트하거나, 시스템 프롬프트를 조정해 병행 검색이 가능함
- 이때 에이전트는 벡터 DB와 지식 그래프를 모두 탐색하여, 보다 완성도 높은 답변을 제공함
- 실제 예시로 두 경로를 동시 사용한 질의를 시연함

### 앞서 설명한 AI 에이전트는 무료 템플릿으로 공개됨

- 영상 하단 링크를 통해 전체 구현 과정 및 구조를 자세히 보여주는 풀 버전 영상을 별도로 안내함
- 해당 템플릿은 클라우드 코드(Cloud Code) 기반으로 개발되어 신속히 적용 가능함
- 클라우드 코드의 구체적 활용법과 동시에, 시청자가 자신의 프로젝트에 쉽게 적용하는 방법도 안내함

### 에이전트 구현에 활용된 주요 오픈소스 및 툴 구성을 투명하게 소개함

- 벡터 DB : Postgres + PG Vector + Neon
- 지식 그래프 : Graffiti, Neo4j
- 전체 개발 프레임워크에 Cloud Code 사용

### 에이전트의 동작 원리 및 사용자 경험 향상 방안에 대해 시각적으로 시연함

- 각 질의에 따른 에이전트의 행동, 선택된 검색 툴, 최종 답변 과정을 시각적 데모 형식으로 보여줌
- 질의 유형, 백엔드 검색경로, 반환 응답 방식 모두 확인 가능
- 이를 통해 에이전트의 자동화된 선택과정과 효율성을 명확하게 전달함
