## *RAG 2.0: 에이전틱 RAG와 지식 그래프 통합(무료 템플릿)* 핵심 요약

- 영상 제목은 "*RAG 2.0: 에이전틱 RAG와 지식 그래프 통합(무료 템플릿)*"임
- 최근 몇 달간 다양한 RAG(Retrieval-Augmented Generation) 전략을 연구하던 중 ‘에이전틱 RAG(Agentic Rag)’와 ‘지식 그래프(Knowledge Graph)’가 가장 뛰어나다고 평가함
- 영상 제작자는 두 전략을 결합한 강력한 AI 에이전트를 직접 개발했으며, 이 에이전트는 질문의 특성에 따라 정보를 탐색할 위치와 방식을 스스로 판단함
- 벡터 데이터베이스로는 Postgres와 PG vector 확장, 그리고 서버리스 플랫폼 Neon을 활용
- 문서는 단일 파일로 생성해 벡터 임베딩을 적용하고, 대형 IT 기업들의 AI 이니셔티브와 상호 파트너십 관련 정보를 담음
- 같은 정보를 Graffiti와 Neo4j로 지식 그래프 형태로도 저장하여 관계 중심 탐색이 가능하도록 함
- 에이전트는 “Google의 AI 이니셔티브는 무엇인가?”와 같은 단일 회사 정보에는 벡터 DB만 활용
- “Amazon과 Anthropic의 관계”처럼 관계 탐색 질문에는 지식 그래프 쿼리를 자동으로 사용
- 사용자가 명령문 시스템 프롬프트를 조정하거나 명시적으로 요청하면 두 데이터 소스를 동시에 활용해 깊이 있는 답변을 제공
- 영상 마지막에서 개발 과정을 상세히 다룬 전체 영상과 무료 템플릿, Cloud Code 기반 구축 방법 안내 링크를 공유함

## 세부 요약 - 주제별 정리

### 에이전틱 RAG와 지식 그래프 전략이 반복적으로 뛰어난 성과를 보임

- 제작자는 여러 가지 RAG(검색 증강 생성) 접근법을 실험함
- 수많은 전략 중에서 ‘에이전틱 RAG’와 ‘지식 그래프’가 반복적으로 성능과 효율성에서 돋보였다고 진단
- 에이전틱 RAG는 정보 제공에 있어 단순 검색 이상의 논리적 추론과 선택 능력을 발휘함
- 지식 그래프는 정보 간의 관계성 탐색 및 통찰 제공에 강점이 있음
- 두 방법 모두 최신 AI 정보 검색 분야에서 주목받는 트렌드임

### 두 전략의 결합으로 정보 탐색 방식이 한층 진화함

- 영상 제작자는 에이전틱 RAG와 지식 그래프를 통합한 AI 에이전트를 직접 개발함
- 핵심 기능은 질문 특성에 따라 “어디서”, “어떻게” 답을 찾을지 에이전트가 스스로 결정하는 점임
- 기존에는 벡터 검색 또는 관계 검색 중 하나만 사용, 이번 통합을 통해 최적의 방향 도출 가능

### 데이터 저장과 임베딩에서는 Postgres, PG vector, Neon 활용

- 벡터 데이터베이스는 Postgres 서버와 PG vector 확장(extension) 사용
- 서버리스 Postgres 솔루션인 ‘Neon’을 적용하였다고 명확히 언급
- 관련 문서를 chunk(덩어리) 단위로 분할 후 벡터 임베딩을 생성
- 데이터에는 빅테크 기업들의 AI 사업 현황, 이니셔티브, 파트너십 등 심층적 정보 포함

### 동일 정보를 Graffiti와 Neo4j로 지식 그래프 형태로도 구축함

- 같은 소스 문서를 지식 관계 기반으로도 저장하여, 두 채널 정보 탐색이 모두 가능하도록 설계
- Graffiti 및 Neo4j 툴을 활용, 관계성을 구조화하여 저장
- 예: “Amazon은 Anthropic의 모델 호스팅 위치다”와 같은 기업 간 관계 정보 모델링 예시 제시

### 에이전트는 질문 유형에 맞는 데이터 소스를 자동 선택함

- 단일 회사 정보(예: “Google의 AI 이니셔티브”)를 물어보면, 벡터 데이터베이스만 검색함
- 답변과 동시에 어떤 도구(tool)가 사용됐는지 결과에 표기해줌
- 예시 질의 시 벡터 DB만을 사용해 적합한 답변을 반환함
- 관계성 기반 질문(예: “두 기업 간 파트너십”)에는 에이전트가 자동으로 지식 그래프를 탐색하도록 설정됨

### 툴 사용 내역을 투명하게 사용자에게 알려줌

- 답변과 함께 “어떤 데이터 소스와 도구를 사용했는지” 명확히 표시
- 앞선 예시에서 벡터 DB 또는 그래프 탐색이 각각 언제 사용되었는지 UI로 바로 알 수 있음

### 사용자는 프롬프트나 직접 명령으로 두 소스 동시 활용도 가능함

- 시스템 프롬프트를 조정하거나, 명령에서 벡터 DB와 그래프 DB 모두 활용을 요청할 수 있음
- 두 데이터 출처를 통합적으로 검색하여 보다 정확하고 풍부한 정보를 제공
- 실제 영상 시연에서는 명확히 “두 소스를 모두 사용”을 명시해 다면적 응답을 확인함

### 개발 과정과 구현 방법을 추가 영상 및 링크에서 상세히 안내함

- 본 영상은 시연과 개요에 집중하며, 전체 개발 과정은 링크된 장문의 영상에서 설명
- 무료 에이전트 템플릿을 배포하며, 누구나 직접 설치 및 이용 가능
- Cloud Code를 활용한 개발 자동화/보조 방식을 활용한 점 강조
- 위 영상 및 소개된 링크를 통해 시청자가 직접 구축하고 체험할 수 있도록 안내
