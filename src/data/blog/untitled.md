## *클로드에게 24시간 동안 코딩을 맡겼더니 벌어진 일* 핵심 요약

- 영상 제작자는 최신 AI 모델인 Claude Opus 4.5를 활용해, 클로드에게 24시간 연속으로 코딩 작업을 시켰음
- 이 실험을 통해 claw.ai의 거의 완벽한 클론(복제품)을 자동으로 구현하는 데 성공했으며, 실제로 대부분의 테스트를 문제 없이 통과함
- AI가 각 기능 구현 과정에서 생성한 과거의 대화 및 검증 과정을 모두 기록해, 개발 역사를 손쉽게 추적 가능함
- 마크다운 형식의 문서화(서식화)가 잘 적용되어 있어, 결과물이 읽기 쉽고 깔끔하게 생성됨
- HTML 페이지 생성, 코드 작성과 실행 등 복잡한 개발 기능까지 AI가 주도적으로 수행함
- 테마, 기본 모델, 최대 토큰 슬라이더 등 다양한 설정 변경 기능도 빠짐없이 포함되어 있음
- 영상 제작자는 자신이 수동으로 작업에 직접 관여하지 않아도 AI 혼자서 매우 기능이 풍부한 애플리케이션을 만들 수 있었음을 강조함
- 아직 "Human in the loop" 등 추가적으로 사람의 개입을 넣고 싶은 부분이 남아 있으나, 현 상태만으로도 완성도가 매우 높다고 평가함
- 하루만 더 AI가 스스로 기능을 보완한다면, 완벽에 가까운 애플리케이션이 탄생할 수 있을 정도의 가능성을 보여줌
- Claude Opus 4.5와 같은 최신 AI 도구와 자동화 프레임워크(harness)를 결합해, 검증과 반복 개발이 놀라운 수준으로 자동화될 수 있음을 실험을 통해 입증함

## 세부 요약 - 주제별 정리

### Claude Opus 4.5를 24시간 동안 활용해 claw.ai 복제 앱을 개발함

- 영상 제작자는 Claude Opus 4.5 모델로 인해 완전히 자동화된 코딩 실험을 진행함
- 실험 목표는 claw.ai라는 웹 애플리케이션의 복제판을, 사람의 개입 없이 24시간 동안 AI에게 맡겨보는 것이었음
- 결과적으로 claw.ai의 주요 기능을 갖춘 완전히 작동 가능한 클론을 만들어냄
- 영상 속에서 "이미 모든 게 완벽히 동작하는 것 같다"고 언급함

### 대부분의 테스트를 손쉽게 통과하며 높은 완성도를 보임

- 실험 말미에, 제작자는 테스트의 절반이 어떤 용도인지조차 의문을 품을 만큼 완성도를 느꼈다고 밝힘
- 기능 구현과 검증이 AI 내부에서 루프(반복) 형태로 효율적으로 이루어짐
- 애플리케이션의 각 기능별 테스트 결과가 긍정적으로 나타나, 추가적인 수정 필요성이 거의 없었음

### AI는 개발 과정을 투명하게 추적할 수 있도록 대화 및 검증 기록을 관리함

- Claude는 각 기능 단계에서 생성한 과거 대화와 검증 과정을 모두 기록함
- 사용자는 애플리케이션에서 이 기록을 손쉽게 열람하면서, AI가 어떤 과정을 거쳤는지 확인 가능함
- 개발 히스토리 파악 및 문제 추적이 용이함

### 마크다운 포맷팅과 HTML 페이지 생성 등 문서화와 출력 기능이 우수함

- 애플리케이션 내부에서 Markdown 형식의 출력이 적용되어 문서화가 매우 깔끔하게 이루어짐
- 사용자는 각종 대화 및 기능 설명을 구조화되고 보기 좋은 문서로 확인 가능
- 다양한 HTML 페이지도 자동으로 생성되어, 웹 상에서 다양한 기능을 확장할 수 있음

### 코드 작성과 실행도 Claude가 직접 자동으로 이행함

- Claude는 실제로 코드 작성은 물론, 즉석에서 코드를 실행해 오류를 확인하는 등 개발의 전 과정을 스스로 처리함
- 코드 생성, 테스트, 검증 및 재수정까지 AI가 주도적으로 반복 실행함
- 개발자의 직접적인 개입 없이, 복잡한 로직과 기능을 구현할 수 있음을 시연함

### 사용자 설정 기능(테마, 모델 선택, 토큰 슬라이더 등)도 완비됨

- 완성된 앱은 테마 변경, 기본 모델 선택, 최대 토큰 개수 조절(슬라이더 방식) 등 다양한 사용자 조정 기능을 포함함
- 현존하는 주요 챗봇 및 웹 기반 AI 서비스와 동등하거나 더 다양한 설정 옵션을 제공함
- 영상 제작자는 기능의 풍부함(feature rich)에 깊은 인상을 받음

### 오직 AI의 힘으로 뛰어난 생산성과 완성도를 경험함

- "손가락 하나 까딱하지 않았다"는 표현에서 알 수 있듯, 제작자는 실제 코딩이나 인터페이스 작업에 전혀 직접 관여하지 않음
- AI가 모든 단계(설계, 코드 작성, 검증, 반복)를 스스로 주도하며, 엄청난 생산성 향상을 보여줌
- 자동화 개발 환경의 현실 가능성과 잠재력을 강조함

### 인간의 개입(Human in the loop) 기능이 향후 추가될 필요가 있다고 언급함

- 제작자는 "향후 인간의 개입(Human in the loop)을 추가"하고 싶다고 언급
- 완전 자동화만으로도 높은 완성도를 달성했으나, 일부 상황에서는 사람이 결과를 확인하고 피드백하는 구조가 필요함을 시사함
- 현 시스템이 향후 더 발전할 수 있는 여지가 있음을 공유

### 하루만 더 AI에게 맡긴다면 완벽에 가까운 앱이 될 수 있음을 기대함

- "하루만 더 돌리면 정말 완벽한 앱이 될 수 있다"는 언급을 통해, Claude의 반복 학습과 자동화 능력에 대한 신뢰를 표현함
- AI의 장기 반복 작업이 품질 개선에 실질적으로 기여함을 소감으로 남김

### Claude Opus 4.5와 자동화 프레임워크의 결합이 개발 검증 및 반복에 혁신을 가져옴

- Claude Opus 4.5와 같은 최신 AI 모델, 그리고 다양한 자동화 검증 프레임워크(harness)의 결합 효과를 실험함
- 애플리케이션의 전체 기능 검증, 코드 에러 수정, 인터페이스 요소까지 AI가 루프를 돌며 자동으로 개선함
- 결과적으로 반복적이고 검증이 중요한 소프트웨어 개발 환경에서 큰 혁신 가능성을 입증함
