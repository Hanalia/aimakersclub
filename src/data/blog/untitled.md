## *RAG 2.0: 에이전틱 RAG와 지식 그래프 결합형 AI 에이전트 소개 (무료 템플릿)* 핵심 요약

- 영상 제목은 "RAG 2.0: Agentic RAG + Knowledge Graphs (FREE Template)"으로, 에이전틱 RAG와 지식 그래프를 결합한 새로운 형태의 AI 에이전트 개발 방식을 소개함
- 제작자는 최근 모든 RAG(Retrieval-Augmented Generation) 전략을 탐구한 결과, 에이전틱 RAG와 지식 그래프 방법이 가장 뛰어나다고 판단함
- 본 영상에서 두 전략을 결합한 강력한 AI 에이전트 구현 사례를 시연함
- 정보 검색 도구로는 PostgreSQL의 PG vector 확장과 서버리스 플랫폼 Neon, 그리고 지식 그래프는 Graffiti와 Neo4j를 사용함
- 벡터 데이터베이스에는 빅테크 기업들의 AI 이니셔티브, 파트너십 등 정보가 입력되어 있으며, 지식 그래프에도 동일 정보를 관계형 구조로 저장함
- 에이전트는 질문의 성격에 따라 벡터 데이터베이스 또는 지식 그래프 중 최적의 검색 도구를 판단해서 사용함
- 예를 들어 단일 기업의 정보는 벡터 데이터베이스에서, 두 기업 관계는 지식 그래프에서 각각 효율적으로 조회함
- 사용자는 시스템 프롬프트 조정이나 명시적 요청을 통해 양쪽 데이터베이스 동시 활용도 가능함
- 에이전트는 질문에 대한 답과 함께 어떤 도구를 사용했는지도 명확히 표시해줌
- 영상 말미에 본 AI 에이전트의 무료 템플릿 제공 및 전체 개발 과정 소개 영상을 별도 링크로 안내함
- 개발 과정에서 Cloud Code 사용법과 직접 구동 방법도 제공한다고 언급함

## 세부 요약 - 주제별 정리

### 제작자는 다양한 RAG 전략을 탐구한 결과, 에이전틱 RAG와 지식 그래프 조합에 집중함

- 최근 수개월간 영상 제작자는 시장에 존재하는 거의 모든 RAG(검색 기반 생성모델) 전략을 심층적으로 실험함
- 그 결과, 반복적으로 가장 뛰어나다고 찾았던 전략이 "Agentic Rag"와 "Knowledge Graphs"임을 강조
- 이를 바탕으로, 두 가지 방법을 결합한 AI 에이전트를 직접 설계하게 됨

### 에이전틱 RAG와 지식 그래프의 결합은 정보 검색 능력을 비약적으로 확장함

- 개발한 AI 에이전트는 질문의 성격에 따라 어떤 데이터베이스에서 정보를 적극적으로 검색할지 사유(reason)함
- 이는 사용자의 질문에 가장 적합한 형태의 정보를 찾아내는 능력을 의미함
- 동적인 검색 전략 채택이 가능하여, 기존 단일 저장소 기반 RAG에 비해 유연성과 파워가 크게 향상됨
- 영상에서 "진짜 쿨하다(so cool)"고 표현하며 혁신성을 강조함

### 벡터 데이터베이스 구축은 PostgreSQL PG Vector와 Neon으로 서버리스 환경에서 구현함

- 벡터 데이터베이스는 PostgreSQL 데이터베이스에 PG vector 익스텐션을 적용해 구축
- 서버리스 환경은 ‘Neon’이라는 플랫폼을 사용해 운용함
- 문서 데이터는 작은 단위(chunk)로 쪼개어 임베딩을 생성하여 저장
- 임베딩 생성까지의 전처리 과정을 상세히 언급함
- 입력된 데이터는 빅테크 기업 정보, AI 프로젝트, 상호 파트너십 등 방대한 내용을 포함

### 동일한 데이터셋을 Graffiti와 Neo4j 기반 지식 그래프에 변환 저장함

- 같은 문서를 지식 그래프에도 입력해, 정보 간의 관계를 구조적으로 활용할 수 있도록 설계
- 지식 그래프 솔루션으로는 Graffiti와 Neo4j를 사용함
- 관계형 구조 속에서 "아마존이 Anthropic 모델의 호스팅 기업"임과 같은 복잡한 연결성 정보를 효과적으로 질의 가능함

### 에이전트는 질문 유형에 따라 최적의 검색 수단(벡터 vs 그래프)을 자동으로 선택함

- 예시1: "구글의 AI 이니셔티브는 무엇인가?"와 같이 단일 기업 중심 질문 → 벡터 데이터베이스 조회
    - 에이전트가 자동적으로 벡터 DB를 선정해 검색 후, 답변과 함께 사용 도구를 알려줌
- 예시2: "두 기업 간의 관계"처럼 복합적 관계 질의 → 지식 그래프 검색 수행
    - 에이전트가 시도한 검색 도구의 종류(이 경우 ‘graph search’)를 명확하게 표시함

### 시스템 프롬프트 조정 혹은 명시적 지시로 양쪽 데이터베이스 동시 활용 가능함

- 사용자는 시스템 프롬프트를 수정하거나, 명령 프롬프트에서 "벡터 DB와 그래프를 모두 사용"하도록 명시 가능
- 이 경우 에이전트는 두 데이터베이스 모두 접근하여 신뢰도 높은 복합 답변을 생성함
- 복수의 데이터 소스를 병렬적으로 활용하는 방식을 실제 예시로 시연함

### 에이전트는 답변과 함께 정보 획득 경로(검색 도구)를 투명하게 공개함

- 각 답변에 대해 "어떤 도구(vector search/graph search)를 사용했는지"를 출력해 신뢰성과 투명성 보장
- 사용자가 결과의 신뢰성을 직접 평가하고, 검색 과정에 대해 명확히 파악할 수 있음

### 무료로 제공되는 템플릿과 전체 개발 과정 영상 링크 안내

- 본 에이전트의 완성된 템플릿을 무료로 제공한다고 안내
- 영상과 연동된 전용 링크를 통해 전체 개발 과정 설명 영상을 볼 수 있음
- 개발 과정 영상에서는 구체적 구현법과, 어떻게 해당 에이전트를 직접 구축·실행할 수 있는지 안내함

### Cloud Code를 활용한 개발 자동화 및 배포 방법도 제공됨

- 영상 내에서 Cloud Code 도구 활용법을 간략히 언급
- 코드 기반 개발 및 운용 자동화, 배포 과정을 지원하여, 사용자가 쉽게 본 에이전트를 실제 환경에 도입할 수 있도록 돕는다고 언급

### 기술적 조합(Neon, PG Vector, Graffiti, Neo4j)이 실제적인 응답 유연성을 구현함

- 각 도구의 결합이 실제 서비스 수준의 에이전트 구축을 가능하게 했음
- 영상에서 설명하는 흐름 속에 구체적인 데이터베이스/그래프 기술 스택이 어떻게 동작하는지 짧은 데모로 시연
- 이를 통해 양질의 정보 검색 정확도와, 복잡한 관계 질의에 대한 빠른 대응 능력을 입증함
