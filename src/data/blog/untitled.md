## *Claude Code로 아름다운 다이어그램을 만드는 전체 워크플로우* 핵심 요약

- 영상에서는 Claude Code와 같은 코딩 에이전트가 시각화(다이어그램) 설명에 한계가 있음을 지적함
- 이를 개선하기 위해 워크플로우 전체를 하나의 “스킬”로 패키징해 시각적 설명 능력을 확장
- Claude Code 등 코딩 에이전트에게 Excal(Excalira) 다이어그램 제작 및 검증 능력을 부여하는 방법을 중점적으로 소개
- 주된 방법은 JSON 파일 기반의 Excal 다이어그램 설계와 자동 시각화 결과 확인(렌더링 후 PNG 검증)
- 워크플로우의 핵심은 코딩 에이전트가 다이어그램을 자체적으로 검증 및 반복 개선하도록 하는 “셀프 밸리데이션” 과정임
- Excal 스킬을 셋업하는 과정은 코딩 에이전트에게 직접 설치 및 안내를 요청할 수 있어 접근성이 높음
- 사용자는 다이어그램의 브랜드 색상, 스타일 등 맞춤 설정이 가능함
- skill.mmd 프롬프트 구성 원칙은 “블록 나열”이 아닌, 구조적 시각화로 전체 개념을 전달하는 법을 코딩 에이전트에 가르침
- Excal 다이어그램 스킬은 PDF, 코드베이스 등 다양한 입력에 활용 가능하며, 반복적인 다이어그램 생성에 큰 시간 절약 효과를 제공함
- 마지막으로 시청자들에게 해당 레포지토리 클론, 스킬 적용, 실제 실습을 권장하며, 활용 가능성이 매우 다양함을 강조함

## 세부 요약 - 주제별 정리

### 코딩 에이전트의 시각적 설명 능력에는 근본적인 한계가 존재함

- Claude Code 등 코딩 에이전트는 주로 텍스트 및 코드 작업에 강점을 가짐
- 사용자가 무언가를 시각적으로 설명하거나, 타인에게 시각적으로 전달할 때 한계가 드러남
- “많은 프롬프트와 별도의 능력 부여 없이” 즉시 좋은 다이어그램을 만들지 못함

### 워크플로우 전체를 하나의 스킬로 패키징하여 다이어그램 생성 과정을 효율화함

- 사용자가 직접 구축한 본인의 전체 작업 플로우를 “스킬” 형태로 패키징
- 이 스킬을 통해 Claude Code, 기타 코딩 에이전트가 실질적으로 유용한, 아름다운 다이어그램을 생성할 수 있게 함
- Excal(Excalira) 다이어그램 툴을 중심적으로 활용

### 코딩 에이전트가 시각적으로 논증(Argue Visually)하는 능력을 갖추도록 프라이머리 인스트럭션 세트를 제공함

- Claude Code가 “시각적으로 논증”한다는 개념에 맞춰, 주요 명령어 집합(인스트럭션 세트)을 제작
- 명령어 집합이 Claude에게 JSON 파일을 작성하여 Excal 다이어그램 생성 절차를 명확하게 안내
- 단순히 상자(block)만 나열하는 형태에서 벗어나, 구조와 레이블(표식) 등으로 전체 내용을 명확하게 시각화하도록 설계

### 다이어그램 생성 과정에 “셀프 밸리데이션(자체 검증)” 기능을 추가하여 결과물을 자동으로 개선함

- 코딩 에이전트가 다이어그램을 만든 후, 자동으로 렌더링하여 PNG 형태로 시각적으로 결과를 검증
- 완성된 이미지를 직접 확인하고, 결함이나 미흡한 부분이 있으면 스스로 반복 개선
- 이 과정 덕분에 “예상치 못한 좋은 결과”가 반복적으로 도출됨을 언급
- 작업자의 추가 개입 없이, 에이전트가 반복적으로 스스로 오류를 점검하며 결과물을 진화시킴

### Excal 다이어그램 스킬 설치 및 셋업은 코딩 에이전트를 통해 매우 손쉽게 진행할 수 있음

- Excalra 다이어그램 스킬 설치를 코딩 에이전트에게 바로 요청 가능
- 에이전트가 스킬 레포지토리의 README 파일을 읽고, 매뉴얼 단계들을 알아서 진행
- 사용자는 손쉽게 준비 완료 상태에서 다이어그램 요청만 하면 됨

### 다이어그램의 색상·스타일 등 맞춤화가 가능하여 브랜드 일관성을 유지할 수 있음

- 사용자의 개인/프로젝트 브랜드, 또는 프레퍼런스(선호하는) 색상 스킴 적용 가능
- 다이어그램 결과물이 “on brand”, 즉 브랜드 일관성 및 선호하는 스타일에 맞게 출력됨

### skill.mmd의 프롬프트 설계 원칙은 “구조적 시각화 설명”을 코딩 에이전트에 학습시키는 데 있음

- 단순히 사각형(블록)만 나열하는 방식은 이해도를 떨어뜨림
- 프롬프트 설계에서 특정 구조와 레이블을 통해 “전체 개념이 명확하게 전달”되도록 유도
- 코딩 에이전트가 시각적인 논리와 스토리라인을 가진 다이어그램을 제작하도록 설계

### Excal 다이어그램 스킬은 다양한 환경 및 데이터에 적용 가능하여 범용성이 높음

- PDF 문서, 코드베이스 등 입력 유형의 제한 없이 다이어그램 생성 가능
- 새로운 프로젝트 시작 시, 기본 다이어그램 생성 단계에서 즉시 활용 가능
- 반복적인 다이어그램 작업에 소요되는 시간을 “매주 수 시간” 단축시켜줌

### 레포지토리 클론 및 스킬 실습을 시청자에게 직접 권장하며, 활용의 폭이 넓음을 강조함

- 영상 제작자는 시청자가 직접 레포지토리를 클론(clone)해서 스킬을 적용할 것을 추천
- 실제로 다양한 입력(문서/코드)에 활용해볼 것을 제안
- “가능성이 무한하다”고 표현하며 실제 노하우 실습을 독려

### 다이어그램 자동화로 인한 시간 절약 효과가 뛰어남을 경험적으로 강조함

- 영상 제작자는 자신이 이 워크플로우로 인해 매주 수시간을 절약한다고 밝힘
- 초기 다이어그램 생성의 질이 높기 때문에 반복 수정 부담도 줄어듦
- 반복적으로 쓰이는 다이어그램이 필요한 누구에게도 높은 생산성 향상임을 시사
