## *클로드에게 24시간 쉬지 않고 코딩을 시켜본 결과* 핵심 요약

- 코드는 이미 완전히 동작하는 claw.ai의 복제본 수준까지 개발되었으며, 많은 테스트의 필요성도 느껴지지 않을 정도임
- 과거 대화 기록도 모두 확인 가능하며, 각각의 루프에서 검증 과정을 거친 생성 결과를 추적할 수 있음
- Markdown 포맷팅이 매우 잘 구현돼 있어 문서 가독성과 관리가 용이함
- 새로운 HTML 페이지를 생성하고, 실제로 코드를 작성 및 실행하는 기능도 구현됨
- 테마와 기본 모델을 설정할 수 있으며, max tokens 값도 슬라이더로 조절 가능함
- 매우 다양한 기능이 포함된, 기능적으로 풍부한 앱이 만들어졌음
- 아직 human-in-the-loop(사람 개입)이 추가되어야 하지만, 직접 코드를 입력하지 않고 대부분의 개발이 자동화된 점이 놀라움
- Claude Opus 4.5 덕분에 하루 만에 이처럼 완성도 높은 애플리케이션을 만들 수 있었음
- 자동화된 검증 및 반복(iteration)이 가능한 시스템 하네스가 전체 개발의 효율성과 품질을 높임
- 만약 하루만 더 진행된다면 거의 완벽에 가까운 애플리케이션이 될 수 있을 것으로 예상됨

## 세부 요약 - 주제별 정리

### Claude Opus 4.5를 이용해 claw.ai 복제본 수준의 애플리케이션을 개발함

- Claude Opus 4.5를 기초로 해서 실제 claw.ai와 거의 동일한 수준으로 동작하는 애플리케이션을 24시간 만에 개발함
- 직접 코드를 작성하지 않고 Claude에게 모든 작업을 맡김
- 남은 테스트의 목적조차 분명치 않을 정도로 애플리케이션이 완성도 있게 작동함
- 기능 구현의 범위와 속도에 대해 “정말 인상적”이라고 언급할 정도임

### 개발 과정에서 과거 대화와 검증 절차가 체계적으로 기록됨

- 앱 내에서 Claude가 각 반복 루프(loop) 마다 생성한 모든 대화와 검증 내용을 확인 가능함
- 이전 대화 기록을 통해 개발 및 검증이 어떻게 이뤄졌는지 추적할 수 있음
- 상세한 로그 및 히스토리 기반 점검이 가능한 것이 장점으로 부각됨

### 마크다운(Markdown) 포맷팅 구현으로 문서의 가독성과 관리가 뛰어남

- 산출물, 설명, 코드 등이 모두 마크다운 형식으로 깔끔하게 정리됨
- 테크니컬한 정보와 개발 노트 등을 Markdown으로 효과적으로 표현함
- 이로 인해 개발 중 결과물이 더 읽기 쉽고 관리하기 쉬워짐

### HTML 페이지 생성과 코드 실행 등의 고급 기능까지 지원함

- 애플리케이션 내에서 다양한 HTML 페이지 생성 기능이 갖춰짐
- Claude가 실제로 코드를 작성하고, 애플리케이션 내에서 이를 바로 실행해 볼 수 있음
- 단순 코드 제너레이션을 넘어 실행까지 자동화되는 개발 흐름을 보여줌

### 사용자 환경 설정(테마, 모델 선택, Max Tokens 슬라이더 등) 기능이 풍부함

- 앱 설정 옵션을 통해 다양한 환경을 조정할 수 있음
- 테마 변경(예: 라이트/다크 모드 등)이 가능하며, 기본 사용 모델(모델 종류)도 선택할 수 있음
- Max tokens(최대 토큰 수) 값을 슬라이더 UI로 직관적으로 조절 가능
- 사용자 맞춤 편의성이 강화됨

### 기능이 매우 많고 복잡함에도 자동화와 효율성이 증명됨

- “so so featurerich(정말 기능이 많다)”는 표현으로 설명할 만큼 다양한 기능 내장
- 복잡한 기능들(세팅, 대화 추적, 자동문서화 등)이 모두 자동으로 구현되어 있음
- 개발자가 직접 코드를 입력하지 않아도 애플리케이션의 큰 틀과 세부가 모두 완성됨

### 향후 추가 개선점으로 사람의 개입(Human-in-the-loop)이 필요함을 언급함

- 현재까지의 구현에서는 human-in-the-loop(사람의 마무리 점검 및 개입) 과정이 생략되어 있음
- 향후 사람이 직접 확인하고 개입할 수 있는 절차를 추가할 계획임을 밝힘

### Claude가 하루 더 작업한다면 완벽에 가까운 결과물도 가능함을 시사함

- 이렇게 빠른 시간 내 많은 기능을 자동으로 개발할 수 있다는 점에 놀라움을 표현
- 만약 Claude에게 하루만 더 자동 개발을 맡기면, 완전히 완벽한 애플리케이션이 될 수 있다고 전망함
- AI 기반 개발이 미래 소프트웨어 개발의 패러다임을 바꿔놓고 있음을 간접적으로 시사

### 자동 검증 및 반복 시스템(하네스)이 효율적 개발과 품질 개선에 기여함

- 애플리케이션 검증, 반복(Iteration), 자동화 테스트가 시스템(harness) 안에서 이뤄짐
- 반복적 검증 및 개선이 자동으로 돌아가며, 개발 품질과 완성도를 높임
- 개발 전체를 지원하는 툴로서의 하네스의 가치가 강조됨

### Claude Opus 4.5의 강력함과 개발 프로세스 자동화의 잠재력이 드러남

- Claude Opus 4.5 버전의 성능과 자동화 능력을 체감할 수 있음
- 사용자는 코드 작성에 직접적으로 개입하지 않아도 복잡한 앱이 자동으로 만들어지는 경험을 얻게 됨
- 최신 AI 기술이 짧은 시간에 최대의 생산성과 품질을 달성할 수 있게 함
