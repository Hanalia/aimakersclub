## *클로드 코드로 더 안전한 OpenClaw 대안을 직접 만든 경험*  핵심 요약

- OpenClaw 프로젝트가 폭발적인 인기를 얻고 있으며, 현 시점에서 가장 주목 받는 개인용 AI 어시스턴트로 자리매김하고 있음
- OpenClaw는 사용자의 작업 내역과 선호도를 지속적으로 학습해 맞춤형 지원을 제공, 그 결과 사용자 경험이 혁신적으로 좋음
- 이 툴은 완전한 오픈소스이며, 누구나 코드에 접근하고 기여할 수 있다는 점도 인기를 끄는 요인임
- GitHub에서 OpenClaw 레포지토리는 185,000개 이상의 스타를 기록, 이는 현재까지 본 적 없는 수준의 수치임
- 이 수치는 인기 오픈소스 워크플로우 자동화툴 N8N을 뛰어넘는 수준으로 개발자 커뮤니티에서 큰 화제를 모음
- 그러나 OpenClaw에는 심각한 보안 취약점들이 존재하며, 이는 사용자가 에이전트에게 부여하는 높은 수준의 권한 행사와 코드베이스 구조의 문제에서 비롯됨
- 본인은 이러한 문제점과 더불어 개인 맞춤화 욕구로, OpenClaw의 아이디어를 바탕으로 직접 대체품을 개발하게 됨
- 불과 몇 천 줄의 Python 코드와 마크다운만으로 OpenClaw의 주요 기능을 며칠 만에 자체 구현 가능했음을 강조함
- 누구나 스스로 이러한 맞춤형 AI 어시스턴트를 쉽게 개발할 수 있으며, 강력한 오픈소스 솔루션을 참고하되 직접 만들어 봐야 자신에게 맞는 최적의 제어와 이해, 커스터마이징이 가능하다고 조언함

## 세부 요약 - 주제별 정리

### OpenClaw는 사용자 맞춤형 기억력과 편의성으로 폭발적 인기를 얻음

- OpenClaw 프로젝트가 이전에 본 적 없을 정도로 빠른 속도로 커뮤니티에서 확산됨
- 현재 많은 사람들이 OpenClaw를 가장 혁신적인 ‘개인 AI 어시스턴트’로 평가
- 사용자가 수행한 작업 이력, 개인의 선호도 등 정보를 장기적으로 기억해, 점점 더 개인화된 지원을 제공
- 다른 AI 어시스턴트 대비, 사용자와의 상호작용 품질이 ‘정말 내 취향, 내 상황을 이해하는 느낌’을 준다는 점에서 특별하게 여겨짐
- 이러한 맞춤형 경험이 뛰어난 사용자 만족도로 이어지고 있음

### 완전한 오픈소스 구조와 커뮤니티 공유가 성장에 기여함

- OpenClaw는 GitHub를 통해 전체 코드가 오픈소스로 공개되어 있음
- 누구나 쉽게 소스코드를 분석, 수정, 기여가 가능하기 때문에 개발자 및 파워유저에게 특히 매력적으로 다가옴
- 오픈소스 구조 덕분에 빠른 기능 개선과 다양한 커스터마이징, 확장이 활발하게 이루어짐
- GitHub 스타 수가 이를 직접적으로 보여주며, 185,000개라는 수치는 역대급 기록임

### GitHub 상의 대기록은 오픈소스 AI 프로젝트의 대중적 인기와 영향력을 증명함

- OpenClaw의 GitHub 리포지토리는 185,000개 이상의 스타를 기록하여 그 위상을 입증
- 이는 유명한 워크플로우 자동화툴인 N8N을 뛰어넘는 기록으로, 개발자 커뮤니티 내에서 큰 파장을 일으킴
- 기존에 본 적 없던 GitHub 내 스타 수치임을 언급, 인기와 파급력이 얼마나 큰지 구체적으로 전달

### OpenClaw에는 보안 취약점이 상당히 많다는 점도 드러남

- OpenClaw가 이처럼 큰 호응을 얻고 있지만, 사용자의 데이터와 시스템 권한을 광범위하게 행사하는 구조적 특성을 가짐
- 내부 코드베이스 자체에도 문제가 발견되는 등, 보안 측면에서 여러 한계가 동시에 존재함
- 영상 제작자는 이러한 보안 우려가 상당히 현실적인 리스크임을 강조

### 개인화와 보안 단점을 모두 고려해 맞춤형 대안을 스스로 개발할 필요성이 제기됨

- 영상 제작자는 보안 문제뿐만 아니라, 본인만의 커스텀 기능에 대한 욕구가 동기로 작용
- OpenClaw의 ‘마법 같은 경험’(ex. 장기 기억, 맞춤형 지원)만을 그대로 가져오고, 단점은 최소화한 자체 버전을 만들기로 결정

### 핵심 기능 복제는 소수의 기술과 노력만 있다면 생각보다 쉽다는 점을 강조

- OpenClaw의 주요 기능(개인화, 작업 이력 학습 등)을 직접 며칠 만에 재현하는 데 성공
- Python 코드와 마크다운만 사용, ‘몇 천 줄’ 정도로 충분히 가능하다고 명시
- 기존 오픈소스 프로젝트의 강점을 참고해 빠르게 프로토타이핑/개발할 수 있다는 점을 구체적으로 제시

### 누구나 직접 제작에 도전하면 최적의 제어와 이해, 확장이 가능함을 제안함

- 영상 제작자는 오픈소스 AI 어시스턴트 프로젝트를 ‘단순히 사용하는 것’에서 나아가, 직접 개발해볼 것을 권장
- 이렇게 해야 각자 원하는 수준의 제어와 기능 커스터마이즈가 가능하고, 솔루션에 대한 이해도도 깊어진다고 강조
- 오픈클로의 공개 성공사례를 참고하되, 충분히 커스텀 가능한 자신만의 AI 어시스턴트를 만들 수 있다고 격려

### 결론적으로, 오픈소스 AI 에이전트를 참고해 직접 만드는 경험이 더 안전하고 자기주도적인 활용을 가능하게 함

- 강력한 오픈소스 프로젝트를 학습 및 참고 대상으로 삼아야 하며, 동시에 자기주도적인 개발을 통해 각자의 요구를 충족하는 솔루션을 만들어야 한다고 피력
- 이렇게 하면 원하는 수준의 보안, 맞춤화, 이해도 모두를 잡을 수 있다는 메시지로 영상이 마무리됨
