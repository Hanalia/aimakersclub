## *스택드 디프(Stacked Diffs)가 AI 코드 리뷰 효율성을 어떻게 극대화하는지* 핵심 요약

- 영상은 "스택드 디프(스택드 변화)" 워크플로우를 도입함으로써 AI 기반 코드 리뷰의 효율성을 극대화할 수 있음을 상세히 설명함
- **전통적인 풀 리퀘스트(pr) 방식**에서는 각 변경사항에 대한 승인을 기다려야 하며, 시간이 지연되는 단점이 있음
- **스택드 디프**는 여러 개의 작은 변화(각각의 diff)가 하나의 큰 기능을 점진적으로 완성하도록 계층적으로 쌓아올리는 방식임
- 각 작은 변화는 독립적으로 AI 기반의 세밀한 리뷰를 받을 수 있어, 코드 품질 관리와 피드백의 초점을 극대화할 수 있음
- 그래파이트(Graphite)라는 플랫폼이 스택드 디프 워크플로우의 구현을 매우 용이하게 해주는 점이 강조됨
- 그래파이트는 **명령줄 도구**(GitHub CLI와 유사)와, 각 스택드 디프 관리용 **대시보드**, 그리고 AI 코드 어시스턴트를 제공함
- 첫 계정 생성 시 체험 가능한 **그래파이트 데모**에서 실제 스택드 디프와 풀 리퀘스트 생성/관리가 매우 직관적으로 안내됨
- 각 인크리멘털(점진적) 변경사항은 독립된 브랜치에서 풀 리퀘스트로 관리되고, 마지막에는 메인 브랜치로 병합되는 구조임
- AI는 각 변경사항별로 피드백을 제공함으로써, 대규모 단일 풀 리퀘스트를 리뷰할 때보다 훨씬 정밀한 코드 품질 관리가 가능함
- 영상에서는 대표적으로 Claude 등의 AI가 3개의 각각의 풀 리퀘스트를 리뷰하는 예시를 제시함

## 세부 요약 - 주제별 정리

### AI 코드 리뷰는 매우 강력하며 반드시 활용해야 하는 검증 도구임을 강조함

- AI 기반 코드 리뷰의 활용도와 필요성을 강하게 주장함
- AI 리뷰 시스템은 코드 품질 확보를 위해 점점 필수 요소로 인식되고 있음
- 하지만 어떤 워크플로우를 사용하는지에 따라 AI 리뷰의 효과성이 달라질 수 있음을 설명함

### 전통적인 풀 리퀘스트 방식의 비효율성과 그 한계를 지적함

- 기존 방식에서는 변경사항 작성 → 승인 대기 → 반복의 형태로 진행
- 코드 변경 승인을 기다리는 동안 개발 프로세스가 중단되거나 지연될 수 있음
- 이러한 구조는 긴 텀과 병목현상이 발생하는 원인이 됨

### 스택드 디프 방식은 작은 변화들을 계층적·점진적으로 쌓아가는 방식임을 소개함

- 스택드 디프는 각 변경사항을 소규모로 쪼개어, 차례대로 쌓아가며 큰 기능 단위를 완성함
- 한 번에 하나의 큰 변경 대신, 여러 작은 변경(diff)이 누적되는 구조
- 각 변화가 독립적이기 때문에, 전 단계 승인 전이라도 다음 변경을 진행할 수 있음

### 스택드 디프는 AI 코드 리뷰에 초점을 극대화하는 최적의 워크플로우임을 설명함

- 각 변경사항마다 AI가 **초점을 맞춘 세밀한 피드백**을 제공할 수 있음
- 단일 대규모 풀 리퀘스트 리뷰에서 자주 발생하는 **복잡도 증가**와 피드백 누락을 방지
- 세분화된 리뷰로 코드 품질, 유지보수성, 협업 효율성이 비약적으로 상승

### 그래파이트(Graphite) 플랫폼이 스택드 디프 실습을 직관적이고 간단하게 해줌을 강조함

- 그래파이트는 스택드 디프 워크플로우에 특화된 CLI 명령어 툴을 지원(GitHub CLI와 유사)
- 레포지토리에서 스택드 디프를 손쉽게 생성·관리 가능
- **대시보드**가 내장되어 모든 풀 리퀘스트 및 변경사항을 한곳에서 시각적/효율적으로 관리할 수 있음
- AI 코드 어시스턴트가 통합되어 골치 아픈 수작업 검토도 자동화

### 그래파이트의 데모 경험은 신규 사용자들도 손쉽게 스택드 디프 방식을 시작할 수 있도록 안내함

- 플랫폼 최초 가입 시 **인터랙티브 데모**를 통해 스택드 디프 생성 및 관리를 친절히 안내함
- 첫 변화를 만들고 풀 리퀘스트 → 그 위에 차례차례 풀 리퀘스트를 추가하는 과정을 실습
- 명령어 기반 UX가 직관적이며, Stack diff 생성 및 병합 과정을 명확히 보여줌

### 모든 풀 리퀘스트를 그래파이트 대시보드에서 자유롭게 관리 및 병합할 수 있음을 시연함

- 영상에서 실제로 **3개의 풀 리퀘스트**를 스택드 디프로 연속 생성
- 각 풀 리퀘스트는 독립적이나, 서로 위에 쌓여 전체 기능을 완성함
- 그래파이트 대시보드에서 각각의 풀 리퀘스트를 편리하게 관리, 리뷰, 병합까지 진행 가능
- GitHub 리포지토리에 직접 접근하지 않고도 완전한 워크플로우 운영 가능

### 각 변경사항이 별도 브랜치에서 관리·병합되는 구조로 Git의 장점을 적극적으로 활용함

- 각 점진적 변화는 모두 개별 feature branch에서 pull request로 등록(메인 브랜치로 곧장 병합 X)
- 각 커밋은 이전 커밋에 쌓이고, 마지막에 메인 브랜치로 병합함
- Git의 브랜치·병합 시스템을 **지능적으로 활용**하는 방식임을 설명

### AI가 각 단계별로 리뷰를 실시하여, 대규모 변화 리뷰 대비 극적인 개선 효과를 제공함

- 각 작은 변화에 대해 AI(Claude 등)가 개별적으로 피드백을 제공
- 세 개의 각기 다른 풀 리퀘스트에 대해 AI가 순차적으로 코드 리뷰를 실행하는 예를 소개
- 단일 대규모 풀 리퀘스트를 한번에 리뷰하는 것보다, **정밀도와 디테일에서 큰 차이가 있음**을 강조

### 영상 말미에 전체 프로세스가 요약 및 그래파이트와의 협업에 대한 감사 표시로 마무리됨

- 전체적으로 **스택드 디프 + AI 리뷰** 조합의 강력함을 재차 요약
- 그래파이트와의 협업 및 지원(facilitation)에 감사 인사를 마지막에 언급하며 영상 종료
