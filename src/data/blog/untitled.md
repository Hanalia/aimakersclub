## *Claude에게 24시간 동안 논스톱 코딩을 시킨 결과* 핵심 요약

- 영상에서는 Claude Opus 4.5를 사용해 24시간 동안 인공지능에게 쉬지 않고 코딩을 맡긴 결과를 상세히 소개함
- 제작자는 거의 손을 대지 않고도 claw.ai의 클론을 완전하게 구축하는 데 성공했다고 밝힘
- 테스트의 후반부 용도가 궁금할 정도로, 이미 주요 기능이 갖춰져 있다고 평가함
- 생성 과정에서 발생한 모든 과거 대화 기록을 쉽게 확인할 수 있도록 설계되었음
- 마크다운 포맷팅이 뛰어나며, 다양한 HTML 페이지를 생성하고 직접 코드 실행도 가능함
- 사용자 설정에서 테마 변경, 기본 모델 선택, 토큰 최대치 조절 슬라이더 등 세부 기능도 탑재됨
- 추후 ‘human in the loop(사람 개입)’ 기능만 추가하면, 거의 완벽한 애플리케이션이 완성된다는 언급이 있음
- Claude의 자동 검증과 반복 개발(루프)을 용이하게 하는 하니스(harness) 활용법이 인상적으로 소개됨
- 24시간 만에 이처럼 기능이 풍부한 소프트웨어를 AI만으로 만든다는 사실에 놀라움을 표함
- 추가적인 하루만 있으면, "완벽"에 가까운 앱도 가능하리라 기대함

## 세부 요약 - 주제별 정리

### Claude가 구현한 claw.ai 클론은 주요 기능을 완벽히 탑재함

- 제작자는 Claude Opus 4.5로 claw.ai와 거의 동일한 클론 서비스를 개발했다고 설명함
- 테스트의 절반 정도만 진행된 상황에서도 이미 ‘완전한’ 수준의 기능성이 확보됨
- 기능 완성도에 감탄하며, 테스트의 후반 기능이 무엇인지조차 의문을 가질 정도였음
- “완전히 기능이 작동하는 claw.ai 클론”이라는 직접 평가를 영상에서 밝힘

### 개발 과정을 AI의 자동 반복 루프와 대화 기록이 투명하게 지원함

- 개발 과정에서 Claude가 각 검증 루프마다 생성한 대화 기록을 모두 열람할 수 있게 설계됨
- 과거에 거친 모든 대화와 검증 과정을 하나씩 확인할 수 있는 점을 강조함
- 반복적인 루프를 통해 기능 검증과 코드 개선이 자동적으로 이루어짐
- 이 기록들은 개발 투명성과 추적성에 도움을 주는 형태임

### 다양한 부가 기능과 설정 옵션이 즉시 제공됨

- 생산된 웹 애플리케이션은 우수한 마크다운(markdown) 포맷팅 기능을 기본 탑재함
- HTML 페이지 생성 및 코드 작성-실행 기능을 손쉽게 구현할 수 있음
- 사용자 설정 페이지에서 테마(theme) 변경, 기본 언어 모델 선택, “최대 토큰” 슬라이더 등 여러 옵션을 지원함
- 단순 복제 수준을 넘어, 실제 서비스급 제품에 가까운 세밀함을 제공함

### 거의 손대지 않고 AI가 모든 코드를 생성했다는 점이 핵심 성과임

- 제작자는 “내가 손가락 하나 대지 않고 이 모든 것을 만들었다”고 직접 강조함
- 일반 개발자 개입 없이, Claude가 알아서 전체 코드와 구성요소를 완성함을 반복하여 언급함
- AI 활용을 통해 개발 시간을 극적으로 단축하는 혁신적 사례임을 시사함

### 검증 및 반복 개발을 위한 하니스(harness)의 중요성이 드러남

- application harness를 통해, Claude가 계속해서 애플리케이션을 자동 검증함
- 반복 루프 구조와 하니스를 합쳐 검증과 기능 보완이 자연스럽게 이루어짐
- 하니스 시스템이 개발 품질과 효율성을 크게 높임

### 사용자 개입(human in the loop)만 추가된다면 완벽에 근접할 것이라는 기대

- 현재 상태에서도 매우 뛰어난 완성도를 보여주나, 제작자는 “human in the loop” 추가 계획을 언급함
- 사용자의 직접적인 피드백이나 수동 개입이 들어가면 한층 완벽해질 것으로 예상함
- 이로써 사용성과 신뢰도가 더 높아질 가능성을 내비침

### Claude Opus 4.5의 잠재력에 대한 강한 인상을 남김

- Claude Opus 4.5만으로 다양한 대화, 코드 작성, 실행, 검증 루프가 가능함을 시연함
- AI가 대규모 프로젝트 환경에서도 강력한 성능을 보임을 사례로 드러냄
- “신기하다”, “crazy”같은 직접적 감탄으로 기술 진보에 놀라움을 표함

### 하루만 더 자동 개발을 이어간다면, 절대적으로 완벽한 앱이 만들어질 것으로 전망함

- 현재의 완성도를 토대로, 추가적 AI 반복 개발 시간을 투입하면 “절대적으로 완벽한 애플리케이션”도 현실적으로 가능하다고 언급
- 반복 루프와 검증, 기능 추가가 AI만으로도 충분히 지속될 수 있는 점을 강조함

### 이 개발 방식이 향후 소프트웨어 제작 방식의 혁신을 상징함

- 전통적인 코딩을 대체할 정도로 AI가 복잡한 애플리케이션 전체 제작이 가능함을 사례로 들음
- 사람이 거의 개입하지 않고 실 서비스 수준 소프트웨어 제작이 가능해진 변화를 상징적으로 보여줌
- Claude와 같은 언어모델과 하니스 결합의 실질적인 효용성을 실제로 증명함
