## *RAG 2.0: 에이전틱 RAG와 지식 그래프 통합 무료 템플릿 소개* 핵심 요약

- 영상 제목은 "RAG 2.0: 에이전틱 RAG와 지식 그래프 통합 무료 템플릿 소개"
- 지난 몇 달 동안 다양한 RAG(검색 기반 생성, Retrieval Augmented Generation) 전략을 시도한 끝에, **Agentic RAG**와 **Knowledge Graphs** 두 가지가 특별히 효과적임을 확인
- 두 방식을 결합한 강력한 **AI 에이전트**를 직접 개발하여 소개
- 이 에이전트는 입력된 질문을 분석해 **가장 적합한 데이터 소스(벡터 DB/지식 그래프)를 선택**하여 답변을 생성
- 벡터 데이터베이스는 **Postgres + PG Vector**로 구축, 서버리스 플랫폼인 **Neon** 활용
- 문서를 Chunk로 분할하고 임베딩 생성, 예시 문서는 빅테크 기업들의 AI 이니셔티브 및 파트너십 정보로 구성
- 동일한 정보를 **지식 그래프(Graffiti, Neo4j)**에도 저장, 기업 간 관계 쿼리 가능
- 사용자의 질문 유형(단일 기업 vs 기업 관계)에 따라, 에이전트는 자동으로 최적의 도구(벡터DB 혹은 그래프) 선택
- 벡터DB와 그래프를 **동시에 활용하는 하이브리드 쿼리**도 지원, 시스템 프롬프트 조정 또는 명시적 요청 가능
- 해당 AI 에이전트의 **무료 템플릿 제공**, 전체 제작 과정 및 구현 방법은 링크된 풀 영상을 통해 안내

## 세부 요약 - 주제별 정리

### RAG 전략을 연구한 끝에 Agentic RAG와 Knowledge Graphs의 강점을 확인함

- 영상 제작자는 수개월간 다양한 검색 기반 생성(RAG) 전략을 집중적으로 실험
- 여러 방법을 검토한 결과, **에이전틱 RAG(Agentic RAG)**와 **지식 그래프(Knowledge Graphs)**가 가장 효과적임을 반복적으로 확인
- 각 전략의 장단점을 평가하며, 두 가지 방식을 결합할 때 가장 강력한 결과 도출 가능함을 깨달음

### Agentic RAG와 Knowledge Graphs 통합으로 강력한 AI 에이전트 구현이 가능함

- 두 가지 RAG 전략을 결합하여, **스스로 최적의 검색 경로를 계획하는 에이전트**를 만들어냄
- 이 AI 에이전트는 사용자의 질문 의도와 속성에 따라 **벡터 DB 검색, 그래프 검색, 혹은 둘 모두를 동적으로 활용**
- 정보가 저장된 형태(벡터/그래프)에 따라 에이전트의 접근 방식이 달라짐

### Postgres와 Neon을 이용해 벡터 데이터베이스를 구축함

- 벡터 데이터베이스로 **Postgres**를 사용하며, 벡터 연산을 위한 **PG Vector 확장** 도입
- Postgres 인스턴스는 서버리스 아키텍처를 제공하는 **Neon** 플랫폼에 탑재
- 처리 문서는 여러 chunk로 분할 및 임베딩(embedding) 생성 과정을 거침
- 예시로 사용된 문서는 **빅테크 기업들의 AI 이니셔티브, 협업(파트너십) 현황 등 복합 정보**를 담고 있음

### 동일한 정보를 Graffiti 및 Neo4j 기반 지식 그래프에도 구조적으로 저장함

- 위 벡터 DB에 저장한 동일한 정보를 **지식 그래프 형태(Graffiti, Neo4j 기반)**로도 함께 보관
- 지식 그래프는 데이터 간 **관계/연결성**을 명확히 정의할 수 있어서, 복잡한 네트워크 쿼리에 유리
- 예를 들어 "엔트로픽 모델이 아마존 AWS 인프라에서 호스팅된다"와 같은 기업 간 파트너십 정보도 명확히 표현/검색 가능

### 에이전트가 질문 내용을 바탕으로 적합한 검색 도구를 스스로 선택함

- 사용자의 질문이 특정 기업 하나에 집중된다면, 에이전트는 **벡터 데이터베이스**만 검색
- 예시 질문: "구글의 AI 이니셔티브에 대해 알려줘"
- 이 경우 벡터 DB에서 질문 임베딩을 활용, 관련 정보를 탐색해 신속히 답변
- 답변 결과에는 **어떤 도구(벡터DB, 그래프 등)가 사용됐는지** 명시적으로 안내

### 기업 간 관계 등 복합 질의에서는 지식 그래프를 자동 활용함

- 사용자가 "두 기업 간의 관계" 등 관계 기반 정보를 묻는 경우, 에이전트는 자동으로 **지식 그래프**를 선택
- 예시: "엔트로픽의 AI 모델은 어디에서 호스팅되는가?"
- 이 경우 에이전트는 벡터 DB 대신 그래프 DB에서 관련 관계 쿼리 수행
- 답변 역시 사용된 도구(그래프 DB)가 무엇인지 자동 표기

### 벡터 DB와 지식 그래프를 모두 활용하는 하이브리드 검색도 지원함

- 필요에 따라 **시스템 프롬프트를 수정하거나** 사용자가 명시적으로 요청해 두 데이터 소스를 병행 활용 가능
- 복합적인 정보를 한 번에 제공하기 위해 벡터 DB와 그래프 모두 조회
- 에이전트가 각 데이터 소스에서 정보를 취합, 보다 정교하고 완성도 높은 답변을 생성

### AI 에이전트 무료 템플릿 제공 및 전체 구현 방법 안내

- 위에서 소개한 **Agentic RAG+Knowledge Graphs 기반 AI 에이전트**의 **무료 템플릿**을 제공
- 본 영상은 데모 성격의 하이라이트이며, 실제 구현 및 세부 개발 과정은 전체 영상에서 설명
- **Cloud Code** 툴을 사용한 개발법, 구축 및 실행 방법도 전체 영상에서 확인 가능
- 관련 링크가 영상 내 안내되어 있음 (풀 영상 참조)
