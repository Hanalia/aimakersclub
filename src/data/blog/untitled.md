## *스택드 디프(계단식 변경점) 방식이 AI 코드 리뷰 효율을 극대화함* 핵심 요약

- AI 코드 리뷰는 매우 강력하며, 코드 검증의 필수 단계로 간주됨
- 기존의 풀 리퀘스트 방식은 변경사항 승인 대기 시간이 길어 작업 흐름을 제한함
- 스택드 디프(계단식 변경점)는 작은 단위의 변경을 순차적으로 쌓아 대규모 기능 개발을 유연하게 지원함
- 각 변경은 독립적인 풀 리퀘스트로 생성되며, 하나의 변경이 완료되면 바로 다음 단계 작업이 가능함
- 이러한 방식은 AI 코드 리뷰에 최적화되어, 각 변경(디프)에 대해 집중적이고 세분화된 피드백을 받을 수 있음
- Graphite라는 플랫폼이 스택드 디프 작업을 쉽고 직관적으로 구현해주는 툴과 대시보드를 제공함
- Graphite CLI(명령줄 도구)와 대시보드를 통해 스택드 디프 관리, 리뷰, 병합을 효율적으로 지원
- 플랫폼에는 AI 코드 어시스턴트가 내장되어 각 풀 리퀘스트에 대해 자동 리뷰를 제공함
- 예시 과정에서는 세 개의 풀 리퀘스트를 차례로 쌓아나가며, 각 단계별로 AI와 함께 리뷰하는 워크플로우를 시연함
- 이 방식은 기존의 모든 변경을 한 번에 리뷰하는 것보다 훨씬 상세하고 효과적인 코드 검증 결과를 도출함

## 세부 요약 - 주제별 정리

### AI 코드 리뷰는 더 이상 선택이 아니라 필수로 자리잡고 있음

- AI를 통한 코드 리뷰는 기존 수작업 검증보다 강력하며, 신뢰성 있는 검증 도구로 활용됨
- AI 코드 리뷰 기능이 개발 워크플로우에 기본으로 포함되어야 한다는 점을 강조
- AI가 코드 변경 지점마다 세밀하게 개입할 때 더 큰 효과를 발휘함

### 기존의 풀 리퀘스트 방식은 작업 흐름에 비효율을 초래함

- 전통적인 풀 리퀘스트 프로세스는 다음 단계로 진행하기 전에 검토와 승인을 기다려야 함
- 승인 대기 동안 생산성이 정체되고, 대규모 변경을 한 번에 다루면 리뷰가 어렵고 부담스러움
- 이런 문제점은 빠른 피드백 순환과 세밀한 코드 검증에 장애가 됨

### 스택드 디프는 작은 변경의 연쇄로 대규모 개발을 유연하게 가능하게 함

- 스택드 디프(계단식 변경점)란: 작은 단위의 변경을 점진적으로 쌓아 올려 하나의 큰 기능을 완성하는 방식
- 각 변경 사항이 이전 변경을 기반으로 이어짐(의존적 변경)
- 개별적으로 풀 리퀘스트가 생성되어 병렬적·연속적으로 작업 가능
- 각 단계마다 별도의 코드 리뷰가 이뤄짐으로써 한 단계씩 안정적으로 품질을 확보할 수 있음

### 스택드 디프 방식은 AI 코드 리뷰의 강점을 극대화함

- 각 디프(변경점)에 대해 AI가 집중해서 리뷰를 수행
- 작은 변경 단위별로 명확하고 구체적인 피드백을 받을 수 있어, 누락이나 오인 가능성이 줄어듦
- 방대한 한 번의 풀 리퀘스트보다 분해된 다수의 풀 리퀘스트를 AI가 다루기에 적합함

### Graphite 플랫폼이 스택드 디프 관리의 전 과정을 혁신적으로 바꿈

- Graphite는 스택드 디프 작업에 특화된 플랫폼
- Graphite CLI(명령줄 도구)는 GitHub CLI와 유사하나, 스택드 디프에 최적화되어 있음
- 대시보드를 통해 각 변경점(풀 리퀘스트)을 직관적으로 한눈에 관리 및 병합 가능
- 내장 AI 어시스턴트가 각 풀 리퀘스트의 코드 리뷰를 자동으로 지원
- 사용자 경험이 직관적으로 설계되어, 입문자도 손쉽게 스택드 디프를 적용할 수 있음

### Graphite 데모 및 실제 워크플로우 소개로 적용 방법을 명확히 전달함

- 플랫폼 첫 가입 시 제공되는 데모 튜토리얼이 신속하고 직관적으로 학습을 유도함
- 첫 변경(스택 디프 1)을 만들고, 즉시 풀 리퀘스트를 발생
- 이후 후속 변경을 추가하여 연속해서 독립적인 풀 리퀘스트를 생성(총 3개 예시)
- 각 단계별로 변경이 쌓여가는 과정을 커맨드라인과 대시보드에서 시각적으로 표시

### 각 풀 리퀘스트는 표준 Git 브랜치로 활용되어 유연하게 관리됨

- 각 변경사항은 별도 기능 브랜치(feature branch)에서 작업됨
- 상위 풀 리퀘스트에 변경을 병합한 뒤, 전체를 메인(main) 브랜치로 병합하는 구조
- Git 사용법을 따르기 때문에 쉽게 적응 가능하며 기존 워크플로우와의 호환성이 높음

### AI와 함께 각 풀 리퀘스트를 개별적으로 리뷰해 극대화된 효율을 경험함

- 예시에서는 Claude 코드(유명 AI 코드 리뷰어)가 각 풀 리퀘스트 세 개를 순차적으로 검토
- 개별 PR 리뷰 결과가 누적되며, 분해된 PR마다 세심한 코드 평가가 진행됨
- 이 과정에서 얻는 피드백과 검증의 질이 한꺼번에 커다란 PR을 리뷰하는 것보다 월등히 높음
- 모든 변경사항이 검토·수정·병합된 후, 대시보드에서 전체 스택을 한 번에 병합함

### 최종적으로 기존의 Github 저장소 접근 없이 모든 관리가 대시보드에서 완료됨

- Graphite 대시보드에서 각 PR 등록, 리뷰, 병합까지 전체 관리가 가능
- Github 저장소에 직접 들어가지 않고도 쉽고 빠르게 워크플로우를 진행할 수 있음
- 물론 필요하다면 GitHub로 전환하여 표준 워크플로우와 연계도 가능

### 계단식 디프와 AI 코드 리뷰의 결합이 코드 검증의 패러다임을 진화시킴

- 긴밀한 분업, 빠른 피드백 루프, 품질 제고라는 개발 팀의 핵심 니즈를 충족시킴
- 기존 대형 PR 리뷰 대비, 코드 변경의 맥락 파악과 결함 검출이 훨씬 용이함
- 결론적으로, 스택드 디프와 AI 코드 리뷰의 조합이 코드리뷰 방식 자체를 획기적으로 개선함
