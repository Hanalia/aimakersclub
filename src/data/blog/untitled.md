## *스택 디프(차곡차곡 쌓는 코드 변경)이 AI 코드 리뷰 효율을 극대화한 방법* 핵심 요약

- AI 코드 리뷰는 매우 강력하며 검증 과정에서 필수 도구로 간주됨
- 기존 풀 리퀘스트 방식은 한번의 변경 후 승인 대기, 반복의 비효율이 있음
- 스택 디프(Stacked Diffs) 방식은 큰 변경을 여러 단계의 작은 변경으로 나누어, 단계별로 변경을 차곡차곡 쌓아 진행함
- 각 단계의 디프(diff)는 이전 단계의 변경 위에 쌓는 방식이며, 승인 대기 없이 연속 작업 가능
- 이 방식은 AI 코드 리뷰의 집중도를 높여, 각 단계별로 구체적이고 포커스된 피드백을 받을 수 있음
- 대표 플랫폼 Graphite는 스택 디프 관리를 위한 전용 CLI(명령어 도구)와 대시보드, 그리고 AI 코드 어시스턴트 기능을 제공
- 사용자는 직관적인 데모를 통해 첫 스택 디프 생성~적용~검토 과정을 실습 가능
- 실제 사례로, 3개의 풀 리퀘스트를 스택 방식으로 만들고 관리했으며, AI(Claude 등)로 각 단계별 리뷰를 실시함
- 이 결과, 하나의 거대한 풀 리퀘스트를 한 번에 리뷰하는 방식보다 훨씬 뛰어난 결과와 관리 효과를 확인
- Git의 브랜치 병합과 유사한 방식으로 각 변경 사항을 관리, 최종적으로 메인 브랜치에 반영
- Graphite 대시보드에서 모든 단계별 풀 리퀘스트를 관리 및 병합 가능, GitHub 진입 없이도 작업 가능함
- 영상은 Stack Diff 방식을 직접 시연하고, 추가 설명 영상 링크를 안내함

## 세부 요약 - 주제별 정리

### AI 코드 리뷰는 매우 강력하고 필수적인 개발 검증 도구로 간주됨

- AI 기반의 코드 리뷰 도구가 개발자 업무의 큰 변혁을 가져옴을 강조
- AI 코드 리뷰를 실제 작업 검증(param validation) 단계에서 항상 활용한다고 언급
- 한 번의 수작업 리뷰보다 빠르고 체계적, 오류 가능성이 현저히 낮음

### 기존 풀 리퀘스트 중심의 개발 방식은 승인 대기로 인한 비효율성이 존재함

- 전통적 풀 리퀘스트(merge request)는 코드 변경마다 리뷰 및 승인을 끝까지 기다려야 함
- 승인 대기 시간이 누적되면서 전체 개발 사이클이 느려짐
- 이러한 방식은 작은 단위의 빠른 변경, 피드백 반복에 한계가 존재함

### 스택 디프는 작은 단위로 변경 사항을 쌓음으로써 효율적 피드백 구조를 만듦

- Stack Diffs는 하나의 커다란 코드 변경이 아니라 ‘여러 소단계’로 나누어 각 단계별 작은 차이(diff)로 구현
- 각 단계는 이전 단계 위에 누적되어 올라가므로, 완성 시 전체 기능 구현이 가능함
- 각 소단계는 독립적으로 리뷰와 테스트가 가능하여, 병렬적 작업 및 빠른 피드백이 가능함

### 스택 디프 방식은 AI 코드 리뷰의 정확도와 집약성을 극대화함

- 작은 단계별 변경마다 ‘초점 집중 피드백’을 받을 수 있어 오류 검출과 개선점 파악이 쉬움
- 불필요하게 거대한 리뷰에서 누락될 수 있는 세부사항도 각 단계에서 포착 가능
- AI가 작은 변화에 최적화된 리뷰를 할 수 있으므로, 오버로드 현상 없이 품질 보장

### Graphite 플랫폼은 스택 디프 전용 환경을 제공하여 접근과 활용성을 높임

- Graphite는 스택 디프에 최적화된 CLI(Command Line Interface)를 제공하며, 사용 경험이 GitHub CLI와 유사
- 전용 대시보드에서 스택 디프의 모든 단계와 흐름을 한눈에 확인 및 관리 가능
- AI 코딩 어시스턴트가 기본 내장되어 풀 리퀘스트 단계별 자동 리뷰를 구현
- 사용자 친화적 인터페이스와 실습 데모를 제공, 첫 사용 시 단계별 안내가 직관적

### 실습 예시를 통해 스택 디프와 Graphite의 실제 활용 과정을 소개함

- 가입 후 첫 데모 과정에서 이용자는 코드 변경 → 첫 풀 리퀘스트 생성 → 추가 풀 리퀘스트 스택화 과정을 겪음
- CLI 상에서 3개의 연속된 변경사항을 풀 리퀘스트로 관리하고, 단계별 쌓아가는 과정이 명확히 드러남
- 각 단계별 리뷰, 스택 전체의 병합 과정이 대시보드 상에서 모두 가능함
- 각 변경사항을 기능별 브랜치로 운영, 최종적으로 메인 브랜치로 병합하는 통합 과정 안내

### 각 풀 리퀘스트에 대해 AI(Claude 등)로 단계별 개별 리뷰를 진행하여 더 나은 결과를 확인함

- 3개의 차례차례 쌓인 풀 리퀘스트 각각에 Claude 코드 리뷰어 활용
- 개별 리뷰를 통해 커다란 PR을 한 번에 리뷰할 때보다 명확, 효과적인 피드백과 수정점 확인 가능
- 작은 단위의 변경과 그 위에 쌓인 변경 각각에서 세밀한 점검이 이루어짐

### 스택 디프의 모든 관리 및 병합은 Graphite 대시보드에서 편리하게 이루어짐

- Graphite 대시보드에서는 각 단위 풀 리퀘스트 별 진행 상태, 병합, AI 리뷰 등 전체 관리 가능
- GitHub 리포지토리에 들어가지 않고도 통합 관리 및 최종 병합까지 수행 가능
- Git의 브랜칭 전략을 지능적으로 조합해 각 변경 사항(브랜치)을 서로 병합시키고, 마지막으로 메인으로 머지

### 기존 방식 대비 스택 디프 방식은 코드 품질과 생산성 모두에서 뚜렷한 개선 효과를 보여줌

- 거대한 PR을 한 번에 리뷰할 때보다 단계별로 나누어 집중적으로 관리할 때 품질이 현저히 올라감
- 각 변경 단계에 대한 빠른 피드백, 오류 발견, 개선점 반영이 반복되어 전반적인 개발 효율 증대
- 개발팀간 협업, 코드 검토, 승인 과정 전체의 속도와 신뢰도도 높아짐

### 영상은 직접적인 실습과 시연, 추가 학습 영상을 안내하여 실무 활용성을 증대함

- 영상 내 실제 Graphite CLI 사용 화면, 스택화 과정, 대시보드 활용 모습을 직접 보여줌
- 설명 및 시연 내용을 통해, 실무에서 스택 디프 관리와 AI 리뷰가 어떻게 연계되는지 구체적으로 시각화
- 후속 영상 링크를 영상 설명란에 제공하여, 이 방식을 직접 따라해볼 수 있도록 안내함
