## *스택 디프(차등 변경)로 AI 코드 리뷰가 더 효율적으로 가능해진 과정* 핵심 요약

- AI를 활용한 코드 리뷰의 강점을 강조하며, 효과 극대화를 위해 적합한 워크플로우로 ‘스택 디프(차등 변경)’ 방법을 소개
- 기존 PR(풀 리퀘스트) 방식은 변경사항 승인마다 대기 시간이 발생하는 단점이 존재
- 스택 디프는 작고 점진적인 변경을 쌓아 한 기능을 완성함으로써, 변경마다 승인 대기 없이 연속적으로 작업 가능
- 각 변경점(diffs)에 대해 AI로 초집중 피드백을 받을 수 있어 리뷰 품질이 대폭 향상
- 이러한 스택 디프 워크플로우에 최적화된 툴로 ‘Graphite’ 플랫폼을 시연
- Graphite는 GitHub CLI와 유사한 명령줄 툴과 함께 대시보드, AI 코딩 어시스턴트 탑재
- 플랫폼 시연에서는 연속적인 3개의 풀 리퀘스트를 계층적으로 생성하며 스택 디프 과정을 안내
- Graphite 대시보드에서 각 PR을 손쉽게 관리, AI 리뷰 및 머지까지 일원화하여 처리 가능
- 세 개의 작은 PR을 개별적으로 AI가 리뷰하는 것이 대규모 단일 PR을 한 번에 리뷰하는 것보다 월등한 결과를 제공
- 실습 영상 및 추가적인 안내 자료는 본문 링크를 통해 제공 예정

## 세부 요약 - 주제별 정리

### AI는 코드 리뷰 과정에서 반드시 활용해야 할 필수 도구로 자리잡고 있음

- AI를 이용한 코드 리뷰는 점차 필수 검증 과정의 일부로 자리잡고 있음
- AI가 코드의 정확성, 잠재적 버그, 개선 제안 등을 실시간으로 점검할 수 있음
- 코딩 생산성과 코드 품질 모두를 높여주는 역할을 함

### 스택 디프(차등 변경) 워크플로우는 전통적 풀 리퀘스트와 확연히 다름을 경험함

- 기존 풀 리퀘스트 방식은 각 변경사항마다 PR을 작성하고, 승인 대기 이후 다음 변경을 반복하는 구조임
- 이런 방식은 단계마다 대기가 필요해 작업 속도가 느려지고, 대규모 기능 추가에는 비효율적임
- 스택 디프는 각 변경을 작은 단위로 쪼개 단계적으로 쌓아 한 기능을 완성하는 방식을 채택함
- 모든 변경사항이 순차적으로 연결되므로 개별 단계의 승인 후 대기 없이 연속 작업 가능
- 각 단계 변경(diff)에 대해 즉각적 코드 리뷰가 가능해 오류를 조기에 발견하고 개선 가능

### 각 단계별 변경점마다 AI 피드백을 개별적으로 적용함으로써 코드 품질이 극대화됨

- 기존에는 대규모 단일 PR에 AI 리뷰를 요청할 때 세밀한 검토가 어렵고, 리뷰 결과 해석도 복잡했음
- 스택 디프 방법에서는 각각의 작은 변경마다 AI가 별도 리뷰를 수행, 초점이 명확해짐
- 각 변경 단계에 맞춤화된 권고와 분석이 가능해 실질적인 코드 개선에 효과적임
- 세 개의 개별 PR을 처리했을 때, 각각 독립적으로 Claude 같은 AI가 리뷰에 참여

### Graphite는 스택 디프를 가장 쉽게 실현할 수 있는 워크플로우 및 도구를 제공함

- Graphite는 스택 디프 작업에 특화된 명령줄(command line) 툴을 제공
- GitHub CLI와 유사한 사용성을 제공하지만, 스택 디프 전용 기능이 추가됨
- 각 변경 단계를 CLI를 통해 곧바로 PR로 만들 수 있도록 하는 등 워크플로우가 단순화됨
- 대시보드에서는 전체 변경 스택을 한눈에 보고 관리할 수 있음

### Graphite 데모 과정을 통해 스택 디프 생성 및 리뷰 흐름을 쉽게 익힐 수 있었음

- Graphite 신규 가입자를 위한 데모 세션에서 스택 디프 제작 흐름을 친절히 안내
- 첫 번째 변경을 생성하고 바로 PR을 만들 수 있음
- 이후 추가 변경을 새롭게 만든 뒤, 기존 PR 위에 차곡차곡 쌓여서 스택을 형성
- 최종적으로 3개의 PR이 서로 연결되어 점진적 기능 개발이 구현됨

### 각 단계 PR 생성 및 관리가 기존 Git, GitHub와 큰 차이 없이 익숙하게 이루어짐

- 각 변경은 feature 브랜치에서 작업 후, 상위 PR(부모 PR)로 머지되는 구조
- 최종적으로 여러 차등 변경을 모두 검증한 뒤 ‘main’ 브랜치로 병합하여 완결
- 기존 GitHub 리포지토리를 그대로 활용하면서도, 추가 툴로 편의성 향상

### Graphite 대시보드는 각 단계 변경사항, AI 리뷰 및 머지 작업을 통합적으로 지원함

- 각각의 PR(총 3단계)을 대시보드에서 손쉽게 확인·관리 가능
- Graphite 내장 에이전트가 AI를 활용해 PR 코드 리뷰를 자동화
- 모든 변경사항을 한 번에 머지할 수 있으며, 따로 GitHub 인터페이스에 진입할 필요 없음

### 개별 PR 단위의 AI 리뷰 결과가 대형 PR 단일 리뷰보다 압도적으로 유익함을 직접 경험함

- 개별 변경 PR 3개에 대해 Claude 등 AI 리뷰어가 독립적으로 리뷰를 수행
- 각 PR마다 초집중 피드백과 수정 포인트 제시 → 코드 품질 및 명확성 현저히 향상
- 대형 PR을 일괄 리뷰했을 때보다 오류 탐지와 보완이 훨씬 용이

### 누구나 Graphite 및 AI 코드 리뷰 도구 활용 방법을 추가 영상에서 확인 가능함

- 영상 본문에 지원 영상(튜토리얼/시연 영상)을 추가로 링크할 예정
- 누구나 따라하기 쉬운 실습 영상과 함께 워크플로우 도입 전략을 안내

### AI 코드 리뷰 및 스택 디프 워크플로우 도입으로 개발팀의 검증 및 협업 효율이 크게 강화됨

- 한 개의 대형 기능보다, 여러 개의 세밀한 변경에 AI 코드 리뷰를 적용할 경우, 전체 프로젝트의 안전성과 신뢰도가 증가함
- Graphite와 같은 도구가 실제 워크플로우 내에서 충분히 실용적임을 시연함
