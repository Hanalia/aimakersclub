## *AI 에이전트를 극적으로 강화하는 가장 간단한 방법 (스킬)* 핵심 요약

- Anthropic이 최근 공개한 '스킬(Skills)' 기능은 AI 에이전트 발전에서 매우 중요한 혁신으로 평가받음
- 이 기능의 핵심 가치는 놀랍도록 '단순한' 구조에 있으며, "더 단순할수록 더 낫다"는 Anthropic의 모토를 충실히 반영함
- 기존에는 에이전트에게 너무 많은 도구, 대화 기록, 시스템 프롬프트 등 많은 정보를 한 번에 context window에 넣어 과부하되는 문제가 있었음
- MCP 서버 등 기존 방식은 모든 도구를 에이전트에게 미리 제공해 실제 대화에서 필요 없는 도구까지 부담을 주는 단점이 있음
- 스킬의 가장 큰 장점은 '점진적 공개(Progressive Disclosure)' 방식으로, 에이전트가 필요할 때만 필요한 정보를 점차적으로 제공함
- 시스템 프롬프트에는 스킬 또는 도구의 '설명'만을 우선 제공하여 최초 부담을 최소화함
- 예를 들어, "PDF 처리 스킬"은 우선 해당 기능이 있음을 에이전트에게 알려주고, 사용자가 PDF 관련 요청을 해야 실제 상세 지침이 제공됨
- skill.md 파일이 각 스킬의 핵심 지침을 담고 있으며, 추가적으로 필요할 경우 더 심도 깊은 문서(3번째 레이어)로 확장됨
- 이 3단계 접근법을 통해 AI 에이전트는 과부하 없이 확장성과 유연성을 확보할 수 있음
- Anthropic의 스킬 접근법은 에이전트의 능력을 크게 높이면서도 사용 환경을 단순하게 유지하는 데 크게 기여함

## 세부 요약 - 주제별 정리

### Anthropic의 '스킬'은 최근 AI 발전에서 가장 인상적인 혁신으로 평가됨

- Anthropic이 스킬 기능을 공개한 시기는 길지 않지만, 그 영향력과 중요성은 매우 큼
- 영상 제작자는 "최근 AI 발전에서 가장 중요한 진전 중 하나"로 스킬을 꼽음
- 이 기능이 혁신적인 이유 중 하나는 구조의 명확한 단순함 때문임
- Anthropic의 모토는 "더 단순할수록 더 낫다(The simpler, the better)"임

### 기존 도구 방식의 한계는 에이전트에 과도한 정보를 미리 주는 것이었음

- 기존에는 에이전트가 다양한 도구, 대화 내역, 시스템 프롬프트 등 많은 데이터를 한 번에 문맥 창(context window)에 넣었음
- 이로 인해 에이전트가 쉽게 과부하되어 효율과 성능이 저하되는 문제가 반복적으로 나타남
- 특히 MCP 서버 방식은, 각 세션에서 실제 사용 여부와 무관하게 모든 도구를 에이전트에게 미리 제공함
- 실제 대화에서 필요 없는 도구까지 포함됨으로서 불필요한 부담과 정보 과잉 현상이 심화됨

### 스킬의 '점진적 공개(Progressive Disclosure)'가 이 문제를 해결함

- 스킬의 파워는 바로 'progressive disclosure', 즉 점진적으로 정보를 드러내는 방식에 있음
- 모든 도구 정보를 일시적으로 제공하는 게 아니라, 실제 필요 시마다 단계적으로 해당 정보를 공개함
- 이는 단순 명료함을 지키면서도 에이전트에 부담을 줄임

### 시스템 프롬프트에는 우선 스킬의 '간략 설명'만 담김

- 첫 번째 레이어에서, 에이전트가 가진 스킬 또는 도구가 무엇인지에 대한 '설명'만 우선 시스템 프롬프트에 주입됨
- 예를 들어, "PDF 처리 기능이 있다"는 정보만 에이전트가 출발점에서 인지함
- 실제 사용자의 요청(예: "PDF 관련 작업을 해줘")이 들어오기 전까지는 상세한 내부 지침은 에이전트가 알지 못함

### 에이전트가 실제 요청을 받으면 skill.md 파일로 단계적으로 기능을 습득함

- 사용자가 에이전트에 특정 기능(PDF 처리 등)을 요청하면 그때서야 skill.md 파일이 불러와짐
- skill.md는 해당 스킬의 "전체 지침(Full Instructions)"을 담은 파일로, 실질적 명령과 절차가 제공됨
- 이 단계를 '두 번째 레이어'라고 영상은 설명함
- 첫 번째 레이어는 단순 설명, 두 번째 레이어는 실질 지침의 불러오기임

### 추가적으로 필요한 경우, skill.md가 다른 상세 문서를 참조하며 정보가 확장됨

- skill.md에는 특징적으로 또 다른 참조 문서들이 연결될 수 있음(세 번째 레이어)
- 예를 들어 더 복잡한 처리나 내부 로직 등이 필요할 경우, skill.md가 추가 문서를 참조하여 필요한 단계만 순차적으로 문맥에 주입함
- 각 레이어별로 필요한 시점에 문서가 차례로 공개되므로, 불필요한 정보 과부하를 효과적으로 막음

### 전체적으로 '스킬'은 AI 에이전트에 유연성·확장성·단순성 모두를 제공함

- 첫 단계에서 스킬 존재만 알리고, 실제 요청 직후 필요한 부분만 문맥에 주입됨으로써 확장성과 효율성이 동시에 강화됨
- 점진적 정보 공개 방식을 통해, 상황 맞춤형 대응과 메모리 자원 절약을 동시에 달성
- 영상에서 강조하듯, Anthropic의 스킬 시스템은 에이전트에 "과부하 없는 강화된 능력"을 부여함

### '스킬' 구조는 맞춤 구현(custom implementation)에도 적용 가능함을 언급함

- 영상에서는 Anthropic 스킬 시스템을 직접 구현하여 확장할 수도 있음을 시사
- 실제 Anthropic 스킬뿐만 아니라, 영상 제작자가 자체적으로 만든 맞춤형(커스텀) 스킬에도 같은 구조가 적용됨을 예로 설명
- skill.md 파일과 연속 참조되는 다층 레이어 구조는 다양한 활용이 가능함

### '점진적 공개' 개념은 향후 다양한 에이전트 설계 방식에 영향 미칠 수 있음

- 모든 도구와 정보를 한 번에 제공하는 방식의 한계를 지적하며, 맞춤형 단계 공개가 현행 한계를 극복하는 키로 작용함을 시사
- 이 점진적 공개 패턴은 앞으로 에이전트 설계의 표준적 접근법이 될 가능성을 영상에서 암시
- 현재 Anthropic 스킬 시스템은 이러한 변화를 대표하는 사례임

### 영상 내용은 스킬의 Workflow를 3단계로 구체적으로 제시함

- 1단계: 시스템 프롬프트에 요약 설명만 주입
- 2단계: 사용자 요청 시 skill.md 전체 지침 주입
- 3단계: 필요시 추가 문서(레퍼런스 문서 등) 참조 및 추가 정보 주입
- 각 단계별로 "과부하 최소화, 상황 맞춤 정보 제공, 문맥 관리"가 달성됨

### 이러한 방식은 실제로 에이전트의 과부하 문제를 해소하면서 고도화된 기능을 손쉽게 사용할 수 있게 해줌

- 최종적으로, Anthropic의 '스킬' 시스템은 사용성과 성능의 균형을 가장 단순한 로직으로 이룬 대표적 사례임을 영상에서 보여줌
