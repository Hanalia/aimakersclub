## *스택드 디프(누적 변경 사항) 방식이 AI 코드 리뷰의 효율성을 극대화함* 핵심 요약

- AI 코드 리뷰는 검증 과정에서 필수적이며 매우 강력한 도구임을 처음에 명확히 강조
- 전통적인 풀 리퀘스트(코드 검토) 워크플로우와 달리, 스택드 디프(누적 변경 사항) 방식은 점진적이고 순차적인 변경을 쌓아 개발
- 스택드 디프는 각 변경 사항이 서로 위에 쌓이며, 다음 변경을 위해 반드시 승인을 기다릴 필요가 없음
- 이 방식은 각 디프(변경)의 범위가 작아 AI가 매우 집중된 피드백을 줄 수 있도록 최적화됨
- Graphite라는 플랫폼이 스택드 디프 작업을 간편하게 할 수 있게 지원함(명령줄 도구 및 대시보드 제공)
- Graphite 명령줄(CLI)은 GitHub CLI와 유사하지만, 오직 스택드 디프 업무에 특화됨
- Graphite 대시보드는 모든 풀 리퀘스트와 스택을 한눈에 관리할 수 있고, 내장 AI 코드 어시스턴트가 리뷰 작업을 도와줌
- 실제로 여러 개의 연속된 풀 리퀘스트를 만들며, 각각이 독립적으로 AI 리뷰를 받으며 쌓여 나감
- Claude Code 같은 AI를 이용해 각 풀 리퀘스트마다 세분화된 검토가 이루어짐
- 대형 풀 리퀘스트 하나로 리뷰하는 것과 비교할 때 훨씬 뛰어난 결과와 관리 편의성을 체감
- Graphite 플랫폼의 직관적 데모와 가이드로 사용이 매우 쉬웠음을 언급
- 종합적으로 이 방식이 효율적 코드 개발 및 리뷰에 확실한 개선을 가져옴을 체험적으로 공유

## 세부 요약 - 주제별 정리

### AI 코드 리뷰는 현대 소프트웨어 검증에서 강력하고 필수적인 도구로 자리잡음

- AI 코드 리뷰가 매우 강력하며, 본인의 코드 검증 과정에서 반드시 필요한 단계라고 언급
- 검토 과정을 자동화·지능화함으로써 오류를 줄이고, 리뷰 효율을 대폭 향상시킴
- 풀 리퀘스트(PR) 작업 시, AI 리뷰가 반드시 포함될 정도로 신뢰하고 활용하는 사례 제시

### 전통적 풀 리퀘스트 방식에서는 피드백과 개발 속도가 제한됨

- 일반적인 풀 리퀘스트 워크플로우에서는 한 번 변경 후 반드시 리뷰와 승인을 기다려야 함
- 승인 이후에 다음 변경사항을 준비하면서 ‘기다리는 시간’이 생기고 개발과 리뷰 프로세스가 단발성임
- 개발자가 기능 하나를 완성하기 위해 여러 번 이 과정을 반복해야 함
- 리뷰 범위가 넓어질수록 집중도도 떨어지고, 피드백도 덜 명확해지는 문제를 내포

### 스택드 디프는 작은 변경을 차곡차곡 쌓아 하나의 큰 기능으로 확장함

- Stack Diff(스택드 디프)란 작은 PR(변경)을 여러 개 순차적으로 쌓아 전체 기능을 구현하는 워크플로우임
- 각 변경(PR)은 독립적으로 다음 변경의 기반 위에 쌓여 ‘스택 구조’를 이룸
- 각 변경 사항이 작고 명확하여 검토가 용이하고, 전체 기능은 이 조각들이 차곡차곡 합쳐짐으로써 완성됨
- 기능 추가, 버그 픽스 등에서 점진적으로 진행할 때 효과적임

### 순차적 승인·병합이 필요 없으므로 생산성과 효율이 극대화됨

- 스택드 디프 방식에선 모든 변경을 한 번에 제출하고, 각 디프를 병렬적으로 리뷰·관리 가능
- 이전 변경이 승인되기까지 다음 작업을 대기할 필요가 없어 개발 속도가 빨라짐
- 개발자는 각 변경에 신경 쓰면서도 전체 기능 흐름을 놓치지 않음

### AI 코드 리뷰가 각 디프별로 초집중 피드백을 제공함

- 각 스택 디프는 범위가 작기 때문에 AI 리뷰는 해당 변경에 초점을 맞춰 정밀 피드백 제공
- 큰 단일 PR에서 리뷰보다, 여러 개의 작은 PR에서 AI가 개별 리뷰를 해주는 것이 효과적임
- 코딩 오류, 논리 버그 등도 세분화된 단계에서 더 확실하게 포착 가능함
- 각 디프별로 AI 리뷰 상태를 명확히 체크하며 진행

### Graphite 플랫폼이 스택드 디프 적용 및 관리를 강력하게 지원함

- Graphite는 스택드 디프 방식의 코딩 워크플로우에 최적화된 플랫폼임
- GitHub CLI와 유사한 커맨드라인 툴을 통해 Repo 관리와 스택드 디프 생성을 지원
- 대시보드에서 각 PR, 스택 전체 상태, 병합·관리를 시각적으로 조작 가능
- 내장 AI 코드 어시스턴트(예: Graphite Agent)가 PR 검토를 지원
- 각각의 스택 변경, 리뷰, 병합을 GUI에서 간편하게 처리하도록 설계됨

### Graphite 플랫폼의 온보딩 및 실제 사용 예시가 직관적이고 간편함

- Graphite 플랫폼에 처음 가입 시, 직관적 데모 과정을 통해 첫 스택 디프 생성 과정을 따라함
- Git 또는 GitHub CLI를 사용해 봤다면 동일한 원리로 쉽게 적응할 수 있음
- 첫 PR 생성 후, 연달아 두 번째·세 번째 PR을 차례로 쌓는 흐름을 명확히 보여줌
- 각 PR이 체인처럼 순차적으로 연결(스택)되는 구조가 커맨드라인과 대시보드에 시각적으로 표시됨

### Graphite 대시보드에서 모든 PR 및 리뷰 진행 상황을 쉽게 관리 가능함

- 대시보드에서 각 스택별 PR 상태·승인·병합까지 한눈에 확인 가능
- Graphite Agent를 통해 각 PR에 대한 AI 리뷰를 바로 요청하고, 결과를 보기 용이함
- GitHub 저장소로 직접 가지 않더라도 모든 작업을 Graphite 대시보드 내에서 처리 가능함
- 실제로 각 PR은 Git의 기능 브랜치에 매핑되며, 각 변경(branch)이 서로에게 머지됨

### 각 단계별로 AI 코드 리뷰 진행 및 실제 리뷰 파이프라인을 시연함

- 세 개의 PR이 순차적으로 쌓인 예시를 기반으로 각 단계마다 Claude Code AI를 이용해 리뷰를 시킴
- 각 Pull Request마다 AI가 별도로 검토하고, 문제와 개선점을 제시
- 작은 변경마다 개별 피드백을 받으므로 누락이나 오판 가능성이 줄어듦
- 전체적으로 AI와의 협업 구조가 실무적으로 어떻게 돌아가는지를 실제로 시연하여 보여줌

### 대형 풀 리퀘스트 하나에서 얻는 리뷰 결과 대비, 다중 스택 방식에서 결과가 월등함

- 전통적으로 대규모 PR 하나에서 검토받는 것과 비교해, 여러 개의 스택 디프 PR에서 AI 리뷰를 받으면 결과물이 ‘insane(압도적)’할 만큼 뛰어남을 강조
- 각 단계별 변경을 명확히 파악하고 문제 분석이 쉬움
- PR 각 단계를 언제든 합치거나 분리하는 것도 손쉽게 처리 가능함

### 전체 흐름을 요약 및 Graphite와의 협업을 통해 실제 활용법을 공유함

- 스택드 디프와 Graphite 플랫폼의 조합이 실무에서 효율적임을 실천 사례로 소개
- 본인이 모든 과정을 직접 체험하며, 얻은 장점과 실제 리뷰 결과를 구체적으로 공유
- 끝으로, 본 영상 아래에 더 상세한 실제 사용법 비디오 링크를 첨부했음을 안내
