## *스택드 디프(누적 변경 사항)가 AI 코드 리뷰 효율을 극대화함* 핵심 요약

- 영상 제목은 "스택드 디프(누적 변경 사항)가 AI를 활용한 코드 리뷰 방식을 어떻게 혁신시켰는가"로 번역됨
- AI 코드 리뷰는 매우 효과적이며, 코딩 작업 검증 시 필수 요소로 여겨짐
- 기존 Pull Request(풀 리퀘스트) 방식은 승인 대기 과정이 반복되어 비효율적인 면이 있음
- 스택드 디프는 작은 단위의 변경 사항을 단계별로 누적하여 빌드업하며, 각 변경 사항을 신속히 추가 가능
- 각 단계별 디프(변경 내역)에 대해 AI가 개별적으로 집중 리뷰를 제공하므로, 피드백의 질과 초점이 크게 향상됨
- Graphite라는 플랫폼이 스택드 디프 작업을 극도로 간단하게 만들어 주며, CLI(명령줄 도구)와 대시보드, AI 코딩 어시스턴트를 제공함
- 가입 후 제공되는 데모를 통해 처음부터 스택드 디프 생성 및 관리 과정을 직관적으로 안내받을 수 있음
- 각 변경 사항이 별도의 feature 브랜치에 쌓여 최종적으로 main 브랜치에 병합됨
- Graphite 대시보드에서 각각의 PR을 쉽게 관리하고, AI 에이전트가 리뷰를 돕는 등 효율성이 극대화됨
- 대규모 PR 하나를 일괄 리뷰하는 것보다, 세분화된 PR을 개별적으로 AI가 리뷰한 결과가 매우 뛰어남
- 영상 마지막에는 각각의 스택 PR을 Claude(클로드) 등 AI로 리뷰하는 방법과 이를 통한 이점을 강조함

## 세부 요약 - 주제별 정리

### AI 코드 리뷰 도구는 검증 과정에서 필수적이며 점점 더 강력해지고 있음

- AI를 활용한 코드 리뷰는 프로젝트의 검증 단계에서 반드시 포함해야 할 정도로 강력한 도구임을 강조
- AI 코드는 규모와 복잡성에 관계없이 코드 품질 향상과 버그 발견에 큰 도움을 줄 수 있음
- 영상 속 화자는 기존에도 AI 코드 리뷰를 항상 필수로 이용해왔음을 언급함

### 기존의 Pull Request 방식엔 대기 시간이 필연적으로 따라옴

- 일반적으로 개발자는 한 번에 변경사항을 생성(Pull Request 생성)하고, 승인(Diff Review)을 기다린 후 반복 작업을 이어감
- 이 과정은 개발 생산성 저하 및 대기 시간 증가라는 비효율성을 내포하고 있음
- 병렬 작업이 어렵고, 대규모 기능 개발 시 진행이 느려진다는 문제가 존재함

### 스택드 디프(workflow)는 작은 변경의 누적으로 최종 피처를 완성함

- 스택드 디프란, 여러 개의 '작고 점진적인 변경 사항(Diff)'을 순차적으로 쌓아가며 개발하는 방식을 의미함
- 각 디프가 전 단계 위에 누적되어 전체 기능을 완성하는 구조로, 대형 기능 개발에도 활용 가능
- 각 변경 사항이 상호 독립적이면서도 순차적으로 빌드되므로, 개별 PR 검토 후 바로 다음 단계 개발이 가능
- 승인 대기 시간이 대폭 줄어 효율적임

### 이 방식은 AI 코드 리뷰와의 궁합이 매우 뛰어남

- 작고 단순화된 각 변경 사항(스택 디프)에 대해 AI가 집중적이고 세밀하게 피드백을 제공할 수 있음
- 각 단계별로 문제가 생기면 해당 스택의 변경만 추적하여 리뷰 및 수정 가능
- 분기별 검토 및 피드백이 빠르므로, 전체 코드 품질 및 개발 속도가 모두 향상됨
- 대형 PR 하나를 통째로 리뷰할 때보다, 파악과 문제 수정이 훨씬 용이하다고 설명

### Graphite는 스택드 디프 작업 환경을 혁신적으로 개선해줌

- Graphite 플랫폼은 스택드 디프 작업에 최적화된 툴임
- GitHub CLI와 유사한 Graphite 전용 CLI를 제공하며, 저장소 내에서 스택드 디프 수정을 지원함
- 통합 대시보드에서 전체 변경사항, PR 관리 및 AI 코딩 어시스턴트를 활용 가능
- 모든 스택의 개별 변경 사항을 한 눈에 파악, 관리할 수 있도록 설계됨

### Graphite 가입 후 제공되는 데모는 스택드 디프의 첫 시작을 아주 직관적으로 안내함

- 가입 시 초보자를 대상으로 하는 데모 과정이 자동 실행됨
- 사용자는 첫 번째 변경 사항을 만들고 PR을 생성하는 과정을 안내받음
- Git 또는 GitHub CLI 사용 방식과 유사해 친숙하게 접근 가능함
- 첫 PR 후에는 연속적으로 새로운 PR을 쌓아 올리는 스택드 디프 실습이 제공됨
- 각 단계별 스택 PR 생성 과정을 직접 커맨드라인에서 확인할 수 있음

### 실제 스택드 디프 PR 관리와 AI 코드 리뷰 과정이 대시보드 내에서 간편하게 이루어짐

- 최종적으로(Stacked Diff 완성시) 모든 PR이 Graphite 대시보드에 집적 표시됨
- 하나의 대시보드에서 각 Pull Request를 선택 관리, AI(그래파이트 에이전트)를 통한 리뷰, 병합(merge)까지 모두 실행 가능
- 별도 GitHub 저장소에 들어가지 않고도 전 과정을 관리 가능하지만, GitHub 연동 역시 자유로움
- 각 풀 리퀘스트는 표준 Git PR 방식과 동일해, 익숙한 워크플로 유지 가능

### 누적 변경사항별(feature branch) 분리 및 최종 메인 브랜치 병합 절차가 명확함

- 각 Incremental Change(점진적 변경)는 각각 독립된 feature branch에서 작업되어 상호 병합됨
- 마지막으로 모든 변경을 메인(main) 브랜치에 통합함으로써, 전체 기능 배포가 완료됨
- 작업 이력을 명확히 분리 및 추적할 수 있도록 설계되어 관리가 용이함

### 세분화된 개별 PR 리뷰에서는 AI가 각 변경마다 초점을 맞춘 분석과 수정 제안을 제공함

- Claude 등 최신 AI 코드 리뷰어로 각 스택 PR을 리뷰 진행
- 각 PR 별로 AI가 세밀하게 코멘트, 수정 제안 등을 제공하므로, 한 번에 대량 리뷰할 때 놓칠 수 있는 문제도 빠르게 포착
- "큰 PR 하나를 대충 리뷰하는 것과 비교하면, 이 방식의 결과물은 정말 엄청나다(insane)"고 직접 언급
- 세분화된 피드백 및 빠른 반영 덕분에 전체적인 코드 품질이 상승함

### 영상 마무리에서는 실제 워크플로우 데모와 Claude 코드 리뷰 예시를 강조하며, Graphite와 콜라보를 언급함

- 영상 마지막에는 실제 워크플로우 실행 예시와 함께, 각각의 PR을 Claude(AI)로 리뷰한 과정 및 결과를 보여줌
- 영상 설명란에는 스택드 디프 사용법, 워크플로우 소개 영상 링크를 별도 안내
- "Graphite와 협업해 이 모든 과정을 시청자들에게 공유하게 되어 고맙다"고 마무리함
