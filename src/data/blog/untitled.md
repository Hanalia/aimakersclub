## *Postman이 Git-네이티브로 전환되어 API 구축 환경이 완전히 바뀌다* 핵심 요약

- 영상은 Postman의 최신 업데이트가 API 개발 환경에 미치는 혁신적 변화를 소개함
- 기존에는 API 엔드포인트, 테스트, 스펙, 컬렉션 등이 서로 다른 도구와 환경에서 관리되어 비효율적이고 복잡한 workflow가 문제였음
- Postman은 이번 대규모 업데이트를 통해 모든 Postman 리소스를 로컬 코드베이스에 YAML 파일로 관리할 수 있게 함
- 이 변경으로 컬렉션, 스펙, 모크, 테스트 등이 코드와 함께 Git으로 버전 관리 및 동기화 가능해짐
- Postman 데스크톱 앱과 클라우드 간에 양방향 싱크를 지원하여 어디서든 동일한 자원 접근 및 관리가 가능함
- 코드베이스 내 변경 사항은 git diff로 명확히 추적할 수 있으며, spec과 테스트의 변경 이력을 코드 변경과 동일하게 관리할 수 있음
- 에이전트 모드에서는 자연어로 요구 사항을 작성하면, 코드베이스를 스캔하여 엔드포인트, 스펙, 테스트, 모크 등을 자동 생성 및 동기화할 수 있음
- Claude, GPT, 자가호스팅 모델 등 다양한 AI 모델 및 MCP 서버까지 지원하며, Github, Sentry, Linear 등 외부 도구와도 연동 가능함
- 동일한 컬렉션이 로컬 환경과 CI 파이프라인(예: pre-commit hook, Github action 등)에서 재작성 없이 일관되게 실행됨
- 영상 제작자는 Postman의 기능 진화를 강력히 추천하며, 예시 코드와 리소스 링크를 영상 설명란에 안내함

## 세부 요약 - 주제별 정리

### 기존의 API 개발은 여러 툴과 환경에 분산되어 비효율적이고 혼란스러움을 초래함

- 지금까지의 API 개발 과정은 한 곳에서만 작업이 이뤄지지 않음
- 엔드포인트 생성은 IDE 내에서 수동으로 하거나, 코딩 에이전트를 활용해 개발자가 직접 작성해야 함
- 컬렉션, 테스트, API 스펙은 다시 별도의 툴(Postman 등)에서 관리되어, 작업을 옮겨다녀야 하는 번거로움이 있었음
- CI 파이프라인에서 테스트 구현이 별도로 필요해 작업 흐름이 분산됨
- 명세, 컬렉션, 코드 등의 파일이 각각 다른 위치에 존재해 통합된 버전 관리가 어려웠음
- 이로 인해 API 개발, 테스트, 배포 단계에서 일관성 유지와 협업이 매우 비효율적이었음

### Postman의 대규모 업데이트가 API 개발 워크플로우를 근본적으로 혁신함

- Postman이 최근 Git-네이티브 기능을 대폭 강화한 대규모 업데이트를 발표함
- 모든 Postman 리소스(컬렉션, 스펙, 모크, 테스트 등)를 코드베이스 로컬 디렉터리에 간단한 YAML 파일 형태로 저장 가능
- YAML 파일들은 유지 관리가 용이하고, 기존 코드와 동일한 환경에서 다뤄짐
- 이제 분리된 도구 없이 하나의 코드 저장소에서 모든 작업을 처리할 수 있음
- Postman 데스크톱 앱과도 이 파일들을 직접 연동 가능함

### Postman의 Git 네이티브 기능으로 코드, 스펙, 테스트를 통합적으로 버전 관리할 수 있음

- Postman 리소스들이 코드와 함께 로컬 저장소에 존재하여 git을 통한 버전 관리가 자연스럽게 이루어짐
- 컬렉션, 엔드포인트, 테스트, 스펙 등의 변경사항을 git diff 명령어로 쉽게 확인 가능
- 각 변경사항이 명확하게 드러나 코드 변경 히스토리와 같이 추적 및 검토가 용이함
- 코드, 문서, 테스트의 일관성과 통합관리가 실질적으로 가능해짐

### 데스크톱 앱과 클라우드 사이의 양방향 싱크로 어디서나 동일한 리소스를 관리할 수 있음

- 로컬 코드베이스와 Postman 클라우드 간에 진정한 양방향 동기화가 지원됨
- 로컬에서 작업한 내용이 즉시 클라우드에 반영되며, 클라우드의 변경도 로컬로 동기화됨
- 개발자들은 클라우드 또는 로컬 환경 어디서든 동일한 자원을 활용하며 협업 가능

### Postman 에이전트 모드는 자연어 지시만으로 API 전체 생애주기 관리가 가능하게 함

- Postman의 에이전트 모드에서는 일반 영어 등 자연어로 API 요구 사항을 입력할 수 있음
- 예시로 Flask 앱의 컬렉션 생성을 지시하자, 에이전트가 코드베이스 전체를 스캔함
- 코드 내 라우트 자동 파악, 엔드포인트 명세 생성, 테스트 케이스 및 모크 설정 등이 자동으로 진행됨
- 코드, 테스트, 스펙, 모크 등이 항상 동기화되어 관리 효율성이 극대화됨
- 이렇게 생성된 결과물은 모두 reviewable한 git 커밋 형태로 남음

### 다양한 AI 모델 및 서버, 외부 협업 툴과의 연동 지원으로 확장성이 높음

- Postman은 Claude, GPT, 자가 호스팅 모델 등 다양한 AI 모델을 선택해 활용할 수 있음
- MCP 서버와의 연동도 지원되어 사내 인프라 활용에 유용함
- Github, Sentry, Linear 등 이미 사용 중인 협업/DevOps 툴의 컨텍스트를 컬렉션에 바로 불러올 수 있음
- 외부 데이터를 기반으로 더욱 실질적인 테스트 및 관리가 가능해짐

### Postman 컬렉션이 동일하게 로컬과 CI/CD 환경에서 동작하여 일관성이 보장됨

- 로컬 머신에서 실행되는 컬렉션 및 테스트가 그대로 CI 파이프라인(예: pre-commit hook, Github action 등)에도 적용됨
- 환경에 따라 별도로 테스트 스크립트를 재작성할 필요가 없음
- 프로덕션, 개발, 배포 모든 단계에서 동일한 테스트가 반복 실행되어 신뢰성 확보

### Postman의 새로운 기능으로 API 개발의 번거로움이 한 단계 해소됨

- 영상 제작자는 10년 가까이 Postman을 사용해 왔으며, 이번 업데이트에 대해 큰 만족감을 표현함
- 특히 코드와 테스트, 명세 등이 단일 워크플로우에서 통합적으로 관리되는 점을 높이 평가함
- 단순한 파일 기반 구조, 깃 기반 버전 관리, 자동화된 워크플로우의 결합으로 API 개발 생산성이 크게 향상됨

### 예시 코드 및 자세한 자료는 영상 설명란에서 확인 가능함

- 영상 하단 설명란에 Postman 공식 사이트 링크와 Cole Medin의 예시 Github 저장소가 첨부되어 있음
- 실제 Git 네이티브 API 관리 방법을 참고할 수 있는 구체적 예시 제공
