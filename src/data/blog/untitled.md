## *Claude에게 24시간 쉬지 않고 코딩을 시킨 결과* 핵심 요약

- 이 영상은 인공지능 Claude Opus 4.5를 이용해 24시간 동안 논스톱으로 코딩시키며 실제로 어떤 결과물이 나오는지 보여준다.
- 제시자는 Claude로 claw.ai의 완전한 클론(복제 서비스)을 개발할 만큼 높은 완성도의 결과물을 얻었다고 평가한다.
- 생성된 애플리케이션은 이전 대화 이력과 검증 절차를 모두 볼 수 있게 설계되어 있다.
- 마크다운 포맷팅이 매우 잘 구현되어 문서화가 뛰어나다고 언급한다.
- HTML 페이지 생성, 코드 작성 및 실행 기능 등 다양한 개발 툴킷이 포함되어 있다.
- UI에는 테마 및 기본 모델 변경, 최대 토큰 슬라이더 등 다양한 설정 옵션이 포함된다.
- 아직 'human-in-the-loop(인간 검증 절차)' 단계만 추가하면 더욱 완벽해진다고 언급하지만, 이미 현재 기능만으로도 매우 풍부함을 강조한다.
- 제시자는 자신이 직접 코드에 손을 대지도 않고 이 정도 수준의 결과가 나온 것에 놀라워한다.
- 만약 하루만 더 Claude를 작업시키면 완벽한 애플리케이션이 될 것 같다고 전망한다.
- Claude Opus 4.5와 같은 도구가 검증 및 반복 과정을 자동화하는 강력한 ‘하네스(harness)’ 역할을 한다고 설명한다.

## 세부 요약 - 주제별 정리

### Claude 오퍼스 4.5로 claw.ai 클론을 단기간에 완성한 경험이 강한 인상을 남김

- 영상 제작자는 Claude Opus 4.5에게 24시간 연속적으로 코딩을 시켜볼 것을 실험했다.
- 결과적으로 claw.ai(클로.ai)와 동일한 수준 또는 유사한 웹 애플리케이션을 거의 완성했다는 점을 강조한다.
- 개발이 완료된 후 테스트의 후반부가 어떤 역할인지조차 알기 어렵다고 할 정도로 높은 완성도를 실감한다.
- 제시자는 이미 충분히 동작하는 서비스가 만들어졌기에 남은 테스트가 필요없을 만큼 기능이 충실하다고 판단한다.

### 이전 대화 내역 및 코드 검증 루프를 모두 확인할 수 있어 투명성이 확보됨

- Claude가 생성한 모든 과거 대화 내역을 시스템에서 쉽게 검토할 수 있다.
- 각 기능이 실행되고 검증되는 과정을 루프별로 추적할 수 있어, 개발의 투명성을 높였다.
- 이 구조는 서비스 개발 및 디버깅을 원하는 사용자들에게 실시간 피드백과 검증을 제공한다.

### 뛰어난 마크다운 포맷팅으로 가독성 및 정리가 탁월함

- 애플리케이션 내에서 생성된 문서나 대화 결과들이 매우 깔끔한 마크다운 형식으로 정리된다.
- 코드 블록, 리스트, 링크, 표 등 다양한 마크다운 문법을 적극 활용해 결과를 시각적으로 명확하게 제공한다.
- 이로써 코드 리뷰나 결과물 문서화 시 가독성·편의성이 크게 향상되었다고 평가한다.

### HTML 페이지 생성과 코드의 작성 및 실행이 자동화되어 있음

- Claude가 필요에 따라 다양한 HTML 페이지를 자동으로 생성한다.
- 웹 인터페이스 기반에서 즉시 코드를 작성하고 실행할 수 있는 기능도 제공된다.
- 프론트엔드와 백엔드 개발 전반을 AI가 스스로 구축할 수 있는 점이 주요 특징으로 언급된다.

### 테마, 모델 설정 및 토큰 크기 조절 등 다양한 설정 옵션이 지원됨

- UI 세팅에는 테마(theme) 변경 기능이 내장되어 사용자 취향에 따라 인터페이스 분위기를 조정할 수 있다.
- 기본 AI 모델을 선택할 수 있는 옵션이 존재하여 용도별로 모델 변경이 가능하다.
- 'max tokens'(최대 토큰) 값을 슬라이더 방식으로 조절할 수 있어 결과물의 길이와 성능을 사용자가 직접 통제할 수 있다.

### 기능의 풍부함과 남은 과제로 ‘human-in-the-loop’ 추가가 필요함을 언급

- AI만으로 생성된 시스템이지만, 현 단계에서 이미 매우 다양하고 풍부한 기능을 제공한다고 언급한다.
- 향후 완성도를 높이기 위해 ‘human in the loop’(인간 검증 개입) 프로세스를 추가할 예정임을 밝힌다.
- 인간의 최종 검증이 더해지면 실전 서비스로 바로 사용할 수 있음을 암시한다.

### 실제 손을 대지 않고도 이 정도 결과물이 나오는데 놀라움을 표함

- 영상 제작자는 직접 코딩하지 않고 AI에게만 맡긴 결과물의 수준에 강한 놀라움을 표시한다.
- 단 하루(24시간) 만에 이뤄진 개발이라는 점이 인상적으로 부각된다.
- 자신의 개입 없이 단지 명령과 프롬프트, 검증 루프만으로 프로토타입이 완성되었다고 반복해 강조한다.

### 추가 하루만 더 작업한다면 완전히 완벽한 앱까지 가능할 것으로 기대

- 실험자가 “하루만 더 Claude에게 맡긴다면 완전 무결점의 애플리케이션이 나올 것 같다”며 기대감을 드러낸다.
- 현재의 기능성도 충분히 만족스럽지만, 남은 기능까지 자동화에 성공할 수 있음을 시사한다.

### Claude Opus 4.5와 같은 AI 하네스가 대규모 검증·반복을 자동화할 수 있음을 강조함

- Claude Opus 4.5를 애플리케이션의 ‘검증 및 반복’을 자동화하는 강력한 하네스(harness)로 본다.
- 오류 검증, 반복 실행, 테스트 루프 등 복잡한 개발 프로세스를 AI가 통합적으로 수행함을 직접 확인했다.
- AI 시대의 소프트웨어 개발 패러다임 변화 가능성을 직접 체험한 셈임을 결론적으로 보여준다.
