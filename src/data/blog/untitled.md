## *Claude에게 24시간 내내 코딩을 시킨 결과, 완성도 높은 클론 앱이 만들어짐* 핵심 요약

- 영상에서 저자는 Claude Opus 4.5 모델에게 24시간 동안 코딩을 시키는 실험을 진행함
- Claude가 claw.ai와 유사한 완전히 동작하는 클론 애플리케이션을 단독으로 만들어냄
- 테스트의 후반부에 들어서면서도, 이미 실질적으로 완전한 기능을 갖춘 상태에 도달함
- 생성된 모든 대화 내용을 저장 및 추적할 수 있고, 각 검증 루프마다 결과 확인이 가능함
- MarkDown 형식의 보기 좋은 포맷팅 지원, 다양한 HTML 페이지 생성, 코드 작성 및 실행 기능을 갖춤
- 테마, 기본 모델, 최대 토큰 값을 조정하는 등 다양한 세팅 옵션이 포함됨
- 기능적으로 매우 풍부한(Feature-rich) 애플리케이션이 단기간에 자동 생성됨
- 아직 ‘human in the loop’(사람의 직접 개입) 요소가 추가될 여지가 남아있으나, 거의 손대지 않고도 복잡한 앱을 완성함
- 애플리케이션 검증과 반복 과정을 자동화하는 하니스(테스트 프레임워크)를 통해 품질 확보가 가능해짐
- Claude Opus 4.5와 같은 AI가 개발 효율성 및 애플리케이션 완성도를 크게 향상시킬 수 있음을 직접 실증함

## 세부 요약 - 주제별 정리

### Claude Opus 4.5는 24시간 만에 claw.ai와 유사한 완전한 클론 앱을 혼자서 개발함

- 저자는 Claude Opus 4.5에게 쉬지 않고 24시간 동안 코딩과 개발 지시를 내리는 실험을 진행함
- Claude는 사용자의 직접적인 손길 없이 claw.ai와 거의 유사한 수준의 기능을 갖춘 애플리케이션을 자동으로 제작함
- 영상 내에서 “실제로 기능적으로 완성된 claw.ai 클론이 생겼다”라고 직접 언급함
- 전체 개발 프로세스가 AI에 의해 진행되었고, 인간의 개입은 극히 제한적이었음

### 검증 루프와 대화 추적 시스템을 통해 진행 상황과 결과를 확인할 수 있었음

- Claude가 애플리케이션 기능 구현 중 각 단계마다 대화와 검증 과정을 자동으로 기록함
- 이전의 모든 생성 대화를 쉽게 확인할 수 있음
- 각 개발/검증 루프별로 어떤 사항이 처리되었는지 추적 및 검증 가능

### 마크다운 포매팅, HTML 페이지 생성 등 다양한 산출물이 자동으로 생성됨

- 생성된 결과물은 보기 좋은 마크다운(Markdown) 포맷을 통해 정리됨
- Claude는 여러 종류의 HTML 페이지를 자동으로 생성해냄
- 코드 작성 및 실행 환경도 기본적으로 포함되어 있음

### 사용자는 테마, 기본 모델, 최대 토큰값 등 다양한 설정을 직접 조정할 수 있음

- 세팅 메뉴에서 전반적인 테마(예: 다크/라이트 모드 등) 교체 가능
- 애플리케이션에서 사용할 기본 LLM(언어모델) 지정 기능 제공
- 슬라이더를 통한 ‘Max tokens’(LLM이 1회에 출력할 최대 토큰 수) 조절 가능
- 유저 중심적인 세팅 기능이 다수 포함되어 있어 실제 사용에 용이

### 기능 풍부함(Feature-rich)는 실제 작업 없이도 빠르게 구현됨을 보여줌

- “so so featurerich”라는 표현을 쓸 만큼, 단기간에 다양한 고급 기능이 자동 탑재됨
- 사용자는 코드에 직접 손대지 않고 원하는 수준의 완성도를 달성할 수 있었음
- 애플리케이션의 조작성, 시각적 완성도, 커스텀 옵션 등 많은 요소가 이미 포함됨

### 아직 인간의 개입(‘Human in the loop’) 요소가 최적화될 여지가 남아 있음

- 저자는 “인간이 중간에 개입해 검증하는 부분을 추가하고 싶다”고 언급
- 완성도가 매우 높지만, 사람의 직접 확인 및 개입이 들어가면 품질이 더 높아질 수 있다는 여지를 남김

### 자동화된 하니스와 반복적 검증 구조로 품질 확보 가능성이 높아짐

- 애플리케이션의 모든 기능과 코드 검증을 Claude가 루프를 돌며 자동 테스트
- 하니스(harness) 구조를 통해, 지속적인 기능 추가와 검증이 자동 반복됨
- 개발자 개입 없이도 지속적인 버그 수정, 기능 개선을 가능하게 함

### Claude Opus 4.5의 개발 파워와 실험 결과는 생성형 AI 개발 생산성의 새 지평을 보임

- 사용자는 거의 아무런 수고 없이, 고도의 복잡성과 실용성을 갖춘 앱을 완성할 수 있었음
- 저자는 만약 24시간이 아닌 며칠 더 돌린다면 “완벽에 가까운 애플리케이션이 될 것”이라고 언급함
- Claude Opus 4.5와 같이, 고성능 언어 AI의 활용이 소프트웨어 개발 프로세스를 혁신하고 있음을 직접 입증함
