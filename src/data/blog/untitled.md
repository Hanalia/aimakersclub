## *스택 디프 방식이 AI 코드 리뷰 효율을 극대화함* 핵심 요약

- 영상에서는 AI 코드 리뷰의 강점을 소개하며, 특히 “스택 디프(Stacked Diffs)”라는 워크플로우가 AI 코드 리뷰의 효율과 효과를 크게 높여준다고 강조함
- 전통적 풀리퀘스트(Pull Request) 방식은 기능 수정 마다 승인을 얻고, 다시 변경을 시작하는 반복적이고 느린 과정을 따름
- 스택 디프는 하나의 기능을 작은 단위의 인크리멘탈 변경(incremental change)들로 쪼개, 각 변경이 이전 변경 위에 쌓이도록 만듦
- 이러한 구조 덕분에 각 변경에 대한 빠르고 집중적인 피드백이 가능하고, 변경들 간 대기 시간이 대폭 줄어듦
- 스택 디프를 쉽게 적용할 수 있게 하는 툴로 ‘그래파이트(Graphite)’가 소개되며, GitHub CLI와 유사한 커맨드라인 도구와 명확한 대시보드 제공
- 그래파이트는 AI 코드 리뷰 기능, 각 풀리퀘스트에 대한 리뷰 지원, 통합 병합 기능 등 다양한 자동화 도구를 지원
- 데모에서는 사용자가 작은 변경을 추가하고, 각각의 풀리퀘스트를 순차적으로 만드는 과정을 안내하며, 각 단계별로 AI의 리뷰가 진행됨
- 모든 변경 사항은 주요 브랜치(main)에 병합하기 전에 단계별 관리를 하며, AI가 각각의 인크리멘탈 변경을 심도있게 검토함
- Claude와 같은 AI 코드 리뷰어를 3개의 풀리퀘스트에 각각 적용, 하나의 거대한 풀리퀘스트를 단일 검토하는 것에 비해 훨씬 세밀하고, 효율적이며, 높은 품질의 결과를 얻었음을 언급
- 영상 마지막에 추가적인 튜토리얼 영상 링크와 함께, 그래파이트 팀에 감사 인사를 전함

## 세부 요약 - 주제별 정리

### 스택 디프 방식은 AI 코드 리뷰의 집중성과 효율을 극대화함

- 기존의 AI 코드 리뷰만으로도 효율성이 높지만, 스택 디프 방식이 이를 더욱 강화한다고 설명
- 스택 디프는 하나의 큰 기능 개발을 각각의 작은 변화(인크리멘탈 디프)로 나누고, 이 변화들이 계층적으로 쌓이도록 만드는 프로세스
- 각 작은 변경 단위마다 AI의 집중 리뷰가 가능해지며, 각 변경 사항에 대해 더 명확하고 정확한 피드백이 이루어짐
- 변경 간 승인 대기 없이 다음 단계를 진행할 수 있어 생산성이 크게 향상

### 전통적인 풀리퀘스트 방식은 승인 대기 때문에 속도가 느림

- 일반 풀리퀘스트(Pull Request, PR) 프로세스는 코드 변경 후 승인을 받아야 다음 변경을 진행할 수 있으므로 작업 효율이 떨어짐
- 각각의 기능이나 수정사항을 덩어리째 리뷰하는 경우, AI나 사람의 검토가 비효율적이고 누락이 발생하기 쉬움
- 반복적인 승인-변경 사이클로 인해 전체 개발 주기가 늘어남

### 스택 디프는 작은 변경의 누적으로 전체 기능을 완성하도록 설계됨

- 각 변경 사항이 선행 변경 위에 적층되며, 개별 PR로 관리됨
- 각 PR이 독립적이면서도 전체 기능의 일부로 연계됨
- 중간에 승인이 지연되더라도 다음 단계 변경 작성이 가능
- 빠른 피드백과 병렬 작업이 가능해져 워크플로우가 유연해짐

### 그래파이트(Graphite)는 스택 디프 관리에 특화된 플랫폼임

- 그래파이트에서는 스택 디프 방식으로 작업할 수 있도록 별도의 CLI(Command Line Interface) 도구를 제공
- GitHub CLI와 비슷하지만, 저장소 내에서 스택 디프 전용 작업에 초점을 맞춤
- 대시보드에서 전체 변경 사항과 AI 리뷰 도구를 쉽게 관리할 수 있음
- PR 생성, 리뷰, 병합까지 모든 과정을 그래파이트 내부에서 통합 관리 가능

### 그래파이트 데모는 입문자를 위한 단계별 안내로 구성되어 있음

- 플랫폼 가입 시 최초로 진행되는 데모에서 스택 디프 생성부터 PR 작성, 병합까지 전체 과정을 시각적으로 안내
- 사용자는 첫 번째 변경을 만들고, PR을 제출하며, 이후 변경을 더해가면서 스택을 구성함
- 커맨드라인에서 여러 개 PR을 계층적으로 추가하는 실전 과정을 보여줌
- 각 단계별로 AI 리뷰어가 PR을 검토

### 그래파이트 대시보드는 모든 스택 디프의 관리를 직관적으로 지원함

- 각 풀리퀘스트 별로 리뷰, 병합, 상태 관리를 할 수 있음
- 그래파이트에 내장된 에이전트(Agent)가 각 PR의 코드 리뷰를 지원
- 모든 PR을 대시보드에서 직접 병합 가능
- 필요 시 GitHub 저장소와 연동해 PR을 확인할 수도 있어 유연함

### 기존 Git/GitHub 워크플로우와 호환되면서도 더 지능적으로 활용됨

- 각 인크리멘탈 변경은 독립적인 기능 브랜치에서 작업됨
- 점진적으로 브랜치를 서로 merge하여 메인(main) 브랜치로 최종 반영
- 기존 Git 사용법과 크게 다르지 않아 진입장벽이 낮음

### AI 코드 리뷰어(Claude 등)를 각 단계별로 적용해 검토 품질을 높임

- 영상 속에서는 Claude 코드 리뷰어를 각기 다른 3개 풀리퀘스트에 별도로 투입함
- 대형 풀리퀘스트 하나를 한 번에 검토하는 것에 비해, 각 인크리멘탈 변경마다 AI의 집중적이고 세밀한 검토가 이루어짐
- 세분화된 리뷰가 오류, 개선점 등 발견에 효과적임

### 대형 PR 한 번 검토와 달리 스택 디프는 리뷰 품질과 속도가 압도적으로 높아짐

- 세 개의 작은 PR 각각을 AI가 리뷰한 결과, 전체 품질 및 피드백의 깊이가 매우 커졌다고 설명
- 단일, 큰 풀리퀘스트는 종종 중요한 부분이 묻히거나 실수 발견이 어렵지만, 스택 디프 방식을 쓰면 각 변경별 집중 검토가 가능
- 전체 리뷰 대역폭을 효율적으로 사용하게 됨

### 영상 마지막에 추가 튜토리얼 영상 안내 및 그래파이트 팀에 감사 표시

- 영상의 마지막 부분에 스택 디프 적용 튜토리얼 영상 링크를 설명문에 안내
- 그래파이트와 협업하여 이 과정을 영상으로 보여주게 된 것에 감사 인사
- 사용자가 실제로 따라해 볼 수 있도록 관련 자료 제공을 언급
