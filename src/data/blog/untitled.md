## *내 Claude 코드가 내 컴퓨터를 건드리지 않고 GitHub 이슈를 자동으로 해결함 (무료 도구 소개)* 핵심 요약

- 이 영상은 사용자가 직접 코딩하지 않고, AI 코딩 에이전트가 원격 환경에서 GitHub 이슈를 자동으로 해결하는 과정을 시연함
- 무료 오픈소스 도구인 **Workspaces(by coder 플랫폼)**를 활용하여, 전체 AI 코딩 지원 인프라를 구축하고 운영할 수 있음을 보여줌
- 사용자는 GitHub 이슈에 **“coder” 태그**만 추가하면, 원격 인프라에서 자동으로 코딩 에이전트 인스턴스가 시작되어 이슈를 처리하게 됨
- 모든 작업은 **본인 컴퓨터가 아닌 클라우드, VPS 등 원격 환경**에서 진행되기 때문에 로컬 리소스 소모가 없음
- Workspaces는 코딩 에이전트들의 세션을 병렬로 여러 개 관리할 수 있는 **오케스트레이션 레이어** 역할을 함
- GitHub 이슈에 태그만 추가로, 에이전트가 자동으로 작업을 시작하고, 상태 추적 링크를 댓글로 즉시 남김
- 에이전트가 이슈를 처리해서 **풀 리퀘스트(PR)**까지 자동 생성하며, 사용자는 이를 직접 검토하고 병합할 수 있음
- Workspaces 덕분에 팀 단위 협업이나 대량 이슈 처리가 가능하며, 각 세션이 격리된 환경에서 실행되어 안정성이 높아짐
- 모든 과정이 간단한 트리거와 몇 번의 클릭만으로 이루어져 **효율성과 자동화 수준**이 대폭 향상됨
- 영상 말미에는 이 도구가 무료이며, 사용을 강력 추천한다는 메시지와 함께 설명란에 링크를 남김

## 세부 요약 - 주제별 정리

### 사용자는 한 줄의 코드도 작성하지 않고 AI 코딩 에이전트가 모든 GitHub 이슈를 해결함

- 발표자는 단 한 줄의 코드도 직접 작성하지 않은 채 GitHub에 Pull Request(PR)가 생성되었음을 강조
- 모든 코딩 작업은 AI 코딩 에이전트가 원격에서 자동으로 처리함
- 사용자는 단순히 GitHub 이슈를 생성하고, 특정 태그(“coder”)만 추가하는 것만으로 요청이 이루어짐

### 무료 오픈소스 도구 Workspaces를 통해 AI 코딩 에이전트 인프라가 구축됨

- ‘Workspaces’는 coder라는 플랫폼에서 제공하는 무료 오픈소스 도구임
- 해당 도구를 통해 AI 코딩 보조 시스템의 전체 오케스트레이션(제어, 배치 등)이 가능해짐
- Workspaces는 코딩 에이전트 자체가 아니라, 에이전트 실행과 관리 인프라 역할을 담당함

### 태그 추가만으로 원격 환경에서 코딩 에이전트가 자동으로 활성화됨

- GitHub 이슈에 “coder” 태그를 붙이면 자동으로 코딩 에이전트 인스턴스가 생성됨
- 에이전트 인스턴스는 자동으로 클라우드 상의 인프라(영상에서는 VPS 사용)에서 실행됨
- 사용자는 로컬에서 아무것도 실행하지 않으며, 모두 원격에서 처리됨

### Workspaces 덕분에 여러 코딩 에이전트 세션을 병렬로 관리하고 실행할 수 있음

- Workspaces는 여러 코딩 에이전트 세션(작업 인스턴스)를 동시에 병렬로 실행 및 관리할 수 있도록 해줌
- 다수의 GitHub 이슈를 동시에 처리할 수 있어, 대규모 협업이나 복수 작업이 용이함
- 각 세션은 격리된 환경에서 독립적으로 운영되어 충돌 위험이 없음

### 실제 구현 예시를 통해 자동화 작업의 프로세스를 상세히 시연함

- 사용자는 GitHub 이슈 페이지에서 “coder” 태그만 추가함
- 이후 Workspaces의 UI에서 신규 작업(Task)이 등록되는 과정을 직접 보여줌
- 새로운 작업이 초기화되면, GitHub 이슈에 실시간 작업 추적 링크가 자동 댓글로 남겨짐

### 모든 코딩 및 PR 생성 과정이 클라우드 기반으로 자동 처리됨

- 에이전트가 작업 시작 후 클라우드 환경에서 코딩 작업을 실행
- 문제가 해결되면, 바로 Pull Request가 생성되어 GitHub에서 확인 가능
- 풀 리퀘스트는 사용자가 직접 리뷰 및 병합할 수 있도록 완료된 상태로 제출됨

### 에이전트별 세션 분리 및 고립 환경 제공으로 대규모 병렬 처리 가능

- Workspaces의 가장 큰 장점 중 하나는 각 작업 세션이 독립적, 격리된 환경에서 실행된다는 점임
- 이 덕분에 한 번에 수십 건의 이슈를 각기 다른 환경에서 충돌 없이 동시에 처리 가능
- 팀 전체가 하나의 인프라를 공유하여 효율적으로 활용할 수 있음을 강조

### 로컬 리소스 사용 없이 누구나 손쉽고 효율적으로 대량의 이슈를 처리할 수 있음

- 본인 PC의 자원을 전혀 사용하지 않으므로 로컬 PC의 사양과 무관하게 대용량 작업이 가능
- 도구 사용법이 간단해, 복잡한 세팅 없이 태그 추가와 클릭 몇 번만으로 전체 프로세스 완성
- 기능 전체가 무료로 제공되어 높은 접근성을 지님

### 팀 협업 환경이나 다양한 개발 요구에 유용한 인프라로 추천됨

- 여러 명이 동시에 코딩 에이전트 인프라를 공유할 수 있어 팀별 협업에 최적화
- 각각의 사용자는 각기 다른 이슈를, 서로 격리된 환경에서 별개로 처리 가능함

### 영상 마지막에서 Workspaces의 무료성과 적극적인 사용 권장이 언급됨

- Workspaces가 오픈소스이며 무료로 누구나 사용 가능함을 재차 강조
- 영상 설명란에 사용법 및 다운로드 링크가 제공될 예정임
- 발표자는 “강력 추천한다”고 밝히며, 도구의 실질적 효율성에 높은 만족을 표함
