## *포스트맨의 깃(Git) 통합으로 API 개발 방식이 완전히 달라짐* 핵심 요약

- 영상은 지금까지 API 개발 workflow가 불필요하게 복잡하고, 여러 도구와 환경에서 분절적으로 이루어졌음을 지적하며 시작함
- 최근 포스트맨(Postman)이 대규모 업데이트로 Git 네이티브(Git-native) 기능을 추가, 모든 API 리소스(컬렉션, 명세, 모킹, 테스트 등)를 코드베이스 내 YAML 파일로 관리 가능하게 되었음
- 이로써 개발자는 API 스펙, 테스트, 컬렉션, 코드 등을 동일 저장소 내에서 같이 버전 관리하며 동기화 가능
- 포스트맨 데스크탑 앱과 클라우드 양방향 동기화 지원, 언제든지 로컬/클라우드 환경 간 자유로운 전환 제공
- 변경 사항 확인 및 추적이 git diff로 간단하게 가능, 개발팀 협업 및 리뷰 효율성 대폭 향상
- 파일 기반 API 리소스 구조로 인해 포스트맨 agent 모드가 전체 API 개발 라이프사이클을 조정 가능
- 자연어로 원하는 API 작업을 기술하면, 포스트맨이 코드베이스 분석→ 라우트/스펙/테스트/모킹 자동생성→모두 git 커밋 형태로 정리
- 클로드(Claude), GBT, 자체 호스팅 모델 등 AI 모델 선택 가능
- GitHub, Sentry, Linear 등 다양한 개발 도구와 연동해 풍부한 맥락/context 활용
- 동일한 컬렉션/test 세트가 개발 환경과 CI 파이프라인 모두에서 일관되게 동작, 추가 환경설정 필요 없음
- 업데이트 이후 포스트맨 활용도와 생산성이 크게 높아졌으며, 유튜버는 자신의 예시 코드 레포지터리 공개 예정

## 세부 요약 - 주제별 정리

### 기존의 API 작업 방식은 생산성이 낮고 불필요하게 복잡했음

- 지금까지 API 엔드포인트를 다루는 작업은 불필요하게 복잡하고, 생산성 저하의 주요 원인 중 하나였음
- API를 활용하는 모든 개발자(특히 AI 에이전트 개발자)는 수시로 다양한 엔드포인트를 다루어야 함
- 기존 워크플로우는 API 엔드포인트 생성은 IDE나 코드 에이전트에서, 컬렉션 및 테스트 생성은 또 다른 도구에서 따로 진행해야 했음
- CI 파이프라인에서 동작시키려면 테스트를 별도 연결해야 하는 등 작업 분산과 연동 문제 지속
- 스펙, 컬렉션, 코드가 서로 분리되어 존재하고, 버전 연동이 제대로 되지 않는 문제가 있었음

### 포스트맨의 깃 통합 업데이트가 등장하며 모든 문제가 해결됨

- 포스트맨(Postman)이 최근 출시한 대규모 업데이트에서 Git-native(깃 기반) 기능 도입
- 영상 제작자는 “포스트맨을 거의 10년간 사용해왔으며, 금번 업데이트를 매우 긍정적으로 평가”한다고 언급
- 이번 업데이트로 포스트맨은 AI Native API 플랫폼으로 진화함 동시에, 깃 기반 API 워크플로우를 완성함

### API 컬렉션, 스펙, 모킹, 테스트 등 모든 리소스가 YAML 파일로 코드베이스 내에서 관리 가능함

- 포스트맨의 모든 API 관련 리소스(컬렉션, 스펙, 모킹, 테스트 등)를 코드베이스(로컬 디렉토리) 내 YAML 파일로 손쉽게 저장 가능
- 이 파일들은 관리와 유지보수가 매우 용이하며, 버전관리 시스템과 통합되어 협업에 유리함
- YAML 포맷은 사람이 읽기 쉬워 빠른 수정, 추적 등이 편리함

### 데스크탑 앱과 클라우드 간 양방향 동기화로 로컬/원격 환경을 자유롭게 오갈 수 있음

- 포스트맨 데스크탑 앱과 포스트맨 클라우드를 연동해 양방향으로 동기화 지원
- 개발자는 로컬 환경과 클라우드를 자유롭게 오가며, 모든 컬렉션, 엔드포인트, 테스트를 동일하게 관리
- 환경 차이에 구애받지 않고 모든 리소스 일관된 유지가 가능함

### git diff 활용으로 API 스펙 및 테스트 변경 사항을 한눈에 추적할 수 있음

- 깃 통합으로 모든 변경 사항을 git diff 명령어로 손쉽게 확인 가능
- 예를 들어, 엔드포인트 테스트에 사소한 수정 시 그 변경이 곧바로 git diff로 추적됨
- 이런 방식은 스펙/테스트/코드의 동시 버전 관리를 실현하고, 개발 리뷰 및 PR 과정의 효율을 극대화함

### 파일 기반 구조로 모든 API 작업이 하나의 깃 저장소에 통합되고, agent 모드의 API 라이프사이클 전체 관리가 가능함

- 모든 API 리소스가 파일(디스크) 기반으로 존재하므로, 포스트맨 agent 모드가 전체 API 라이프사이클을 조정할 수 있게 됨
- agent 모드에서, 사용자는 자연어로 작업을 기술(예: "이 Flask 앱의 collection을 생성해줘")하면 됨
- 포스트맨이 자동으로 코드베이스 라우트 분석, 스펙 및 테스트 생성, 모킹 설정 및 전체 동기화 기능을 제공

### 자연어 질의로 컬렉션∙스펙∙모킹∙테스트 자동 생성 및 모두 깃 커밋으로 관리 가능

- 사용자는 자신이 원하는 API 작업을 자연어(plain English)로 기술
- 예시로 Flask 앱을 대상으로 collection 자동 생성 요청 시
    - 포스트맨이 코드베이스 전체를 스캔, 라우트(documented path) 자동 추출
    - API 사양 문서, 테스트 케이스, 모킹 리소스를 자동 생성
    - 생성 결과가 자동으로 깃 커밋에 병합되어 검토 가능
- 모든 변경사항은 reviewable git commit(리뷰 가능한 커밋) 형태로 정리됨

### AI 모델 연동, 자체 호스팅 및 외부 개발 도구들과의 강력한 인터페이스 제공

- 포스트맨에서 사용할 AI 모델 종류를 선택 가능(클로드 Claude, GBT, 자체 호스팅 등)
- MCP servers(멀티 컴퓨트 파워 서버)까지 지원해 유연한 인프라 호환
- GitHub, Sentry, Linear 등 기존 개발자들이 즐겨 사용하는 도구들과 연동하여 다양한 context 참고 가능

### 동일 컬렉션과 테스트가 개발 환경, CI 파이프라인 모두에서 일관되게 동작

- 로컬 환경에서 실행하던 컬렉션/테스트파일이 CI(지속적 통합) 파이프라인에서도 그대로 동작
- pre-commit hooks, GitHub Actions 등 자동화된 빌드/테스트 환경에서 별도의 설정 또는 변환 필요 없음
- 환경에 따라 test를 다시 작성하거나 재구성할 필요 없이 동일하게 활용 가능

### 포스트맨의 최근 변화로 API 기반 개발의 생산성과 효율성 모두 크게 강화됨

- 포스트맨의 Git native 전환으로 API 개발 및 테스트 협업의 전 과정이 대폭 간소화되고, 품질 관리가 극적으로 향상됨
- 영상 제작자는 자신이 직접 만든 예시 API 코드와 포스트맨 리소스 레포지터리를 설명란에 추가할 예정이라고 언급

### 결론: 포스트맨의 깃 통합 업데이트는 API 개발 방식에 구조적 전환을 가져왔음

- 이번 업데이트는 API 개발, 테스트, 협업 전 과정을 완전히 새롭게 만들었다는 데 영상의 모든 논리가 집중됨
- 실 무개발자 관점에서 강력히 추천하는데, 영상 설명란에 포스트맨과 예시 깃허브 링크가 제공됨
