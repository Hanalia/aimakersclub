## *Anthropic, Claude Sonnet 5를 공개하며 뛰어난 코드 작성 성능과 혁신적 가격으로 시장에 도전* 핵심 요약

- Anthropic이 Claude Sonnet 5를 새롭게 출시하였으며, 중급 모델임에도 SweBench Verified 코딩 점수 85.2점을 기록하여 상위 모델인 Opus 4.8과 성능 격차를 크게 좁힘
- Sonnet 5의 가장 큰 변화는 속도가 아닌, 합리적인 가격에 거의 최고급 모델에 가까운 성능을 제공한다는 점임
- Sonnet 5는 ‘가장 에이전트적(Agentic)인’ Sonnet 버전으로, 코드 저장소를 지정하고 목표를 주면 계획부터 터미널 실행, 멀티스텝 작업까지 자동 완료
- 기존 Sonnet 4.6 대비 SweBench Verified 점수는 85, Terminal Bench는 67에서 80, Browse Comp는 76에서 86으로 큰 폭 상승
- Frontier Code 벤치마크에서 Sonnet 5는 15점에서 38점으로 두 배 이상 개선, “가장 큰 도약”은 순수 코드 작성 역량
- 모델 업그레이드는 model ID 하나만 Sonnet 4.6에서 5로 교체하면 되어, 별도 재작성 과정 없이 손쉽게 적용 가능
- Opus 4.8이 벤치마크에서는 아직 Sonnet 5보다 앞서지만 Sonnet 5의 접근성과 가성비가 강점임
- Sonnet 5는 Claude 서비스의 전체 요금제(Free, Pro, Max, Team, Enterprise, Claude Code, 플랫폼)에서 사용할 수 있음
- 2024년 8월 31일까지 입력 100만 토큰당 $2, 출력 100만 토큰당 $10로 제공되어 ‘Opus급 코드 작성이 Sonnet 가격’으로 가능
- 영상을 제작한 Cole Medin은 에이전트 빌더를 위한 워크숍(Dynamis)도 운영하며 Sonnet 5의 변화가 세대를 뛰어넘는 혁신인지, 저렴해진 Opus에 불과한지 의견을 청중에게 질문함

## 세부 요약 - 주제별 정리

### Sonnet 5 출시와 중점은 속도가 아닌 가격과 성능의 밸런스임을 강조함
- Anthropic이 Claude Sonnet 5(model) 출시 소식을 알림
- 이번 Sonnet 5는 ‘속도’가 주제가 아닌, 중급 모델에서 ‘최상위급 코딩 성능’을 보는 것이 진정한 헤드라인임을 강조
- 중간 가격대의 모델임에도 SweBench Verified 코딩 벤치마크 점수가 85.2로, Opus 4.8 모델과 거의 동등한 수준에 도달함을 지적
- 기존보다 훨씬 ‘가성비’에 초점을 둠

### Sonnet 5는 이전 버전과 비교해 ‘가장 에이전트적’으로 진화함
- Cole Medin은 Sonnet 5가 지금까지 나온 중 ‘가장 에이전트적(Agentic)’인 Sonnet임을 언급
- 저장소(Repository)를 지목해 주고, 목표(Goal)를 제시하면 자체적으로 행동 계획을 수립함
- 터미널에서의 명령 실행과 멀티스텝 작업까지 직접 처리 가능
- 과거 Sonnet은 중간에 멈췄던, 복잡하고 여러 단계가 필요한 작업들도 스스로 끝까지 완수하는 성능을 보임

### Sonnet 4.6 대비 주요 벤치마크 수치에서 비약적으로 도약함
- SweBench Verified 점수: Sonnet 4.6에서 85점으로 크게 상승
- Terminal Bench: 기존 67점에서 80점까지 개선
- Browse Comp: 76점에서 86점으로 향상됨
- Opus 4.8이 여전히 약간 앞서지만, Sonnet 5가 상당히 근접했다고 평가

### 모델 업그레이드는 매우 손쉬워 단일 모델 ID 변경만으로 가능함
- 기존 Sonnet 4.6 기반 코드・시스템에서 Sonnet 5로의 이행은 간단
- ‘모델 ID’만 Sonnet 4.6에서 5로 바꾸면 모든 기능을 즉시 사용할 수 있음
- 복잡한 마이그레이션이나 코드 수정이 필요하지 않아, 실무 적용이 편리함

### Frontier Code 벤치마크에서 Sonnet 5가 두 배 이상 높은 점수를 기록함
- 가장 눈에 띄는 도약은 ‘순수 코딩 성능(Frontier Code 시험)’ 분야임
- Sonnet 4.6이 15점을 기록했던 테스트에서 Sonnet 5는 무려 38점으로 향상
- “이전 Sonnet의 두 배 이상 수준”이라 직설적으로 강조
- 코딩 작업 자동화, 에이전트 빌딩 등 실제 활용도에서 큰 기대를 모음

### Opus 4.8이 일부 벤치마크에서 여전히 우세하지만, 초점은 Sonnet 5의 접근성임을 강조함
- Opus 4.8이 일부 성능 벤치마크(특히 Frontier Code 등)에서 아직 Sonnet 5를 앞섬
- 그러나 Cole Medin은 “이 한 벤치마크에만 매몰되지 말라”고 제언
- 실제로는 Opus와 비슷한 수준의 코딩 능력을 ‘누구나, 다양한 요금제에서’ 사용할 수 있게 되었다는 데 초점을 두어야 함

### Sonnet 5는 Claude의 다양한 요금제와 플랫폼에서 동일한 성능 이용이 가능함
- Sonnet 5는 Claude Free, Pro, Max, Team, Enterprise, Claude Code, Claude 플랫폼 등 모든 요금제에서 이용할 수 있음
- 별도의 구분이나 제한 없이, ‘Opus급’ 혹은 이에 근접한 코딩 능력을 손쉽게 적용가능
- 이는 팀 또는 기업단위 에이전트 구축 및 상업용 자동화 프로젝트에 특히 유리함

### 2024년 8월 31일까지 파격적인 토큰당 가격 혜택이 제공됨
- 현재(영상 기준) 프로모션 요금 적용 중: 입력 100만 토큰당 $2, 출력 100만 토큰당 $10
- 이 가격은 2024년 8월 31일까지 한정 제공됨
- Opus 수준 코딩을 Sonnet 가격에 경험할 수 있다는 점을 ‘진정한 반전(the twist)’으로 소개
- 가격 대비 제공 성능이 업계 최고 수준임을 암시

### 에이전트 및 에이전트 코딩 변화에 특화된 실용적 워크숍도 영상 제작자가 운영함
- Cole Medin은 Sonnet 5로 구현되는 ‘에이전트 중심 코딩’ 전환을 주제로 Dynamis라는 워크숍을 매주 운영 중임을 언급
- 실제 현장에서 Sonnet 5와 같은 발전이 어떻게 활용되는지, 실습 중심 교육 기회를 안내
- Sonnet 5의 실전 적용 사례·노하우 제공

### Sonnet 5의 의의와 한계에 대한 시청자 의견을 질문하며 영상을 마무리함
- 최종적으로, Sonnet 5가 진정한 세대 교체(generational leap)인지, 아니면 단순히 Opus의 저렴한 버전인지 청중에게 의견을 물음
- 의사결정의 근거가 될 현황 데이터와 구조적 변화점을 영상을 통해 정리함
