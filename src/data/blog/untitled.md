## *Postman이 Git 기반으로 진화하며 API 구축 방식에 혁신을 가져옴* 핵심 요약

- 영상은 Postman이 최근 대규모 업데이트를 통해 Git-native(깃 기반) API 플랫폼으로 진화했음을 설명함
- 이전에는 API 엔드포인트, 테스트, 컬렉션 등이 분산된 곳에서 별도로 관리되어 워크플로우가 매우 복잡했음
- Postman 업데이트로 모든 리소스를 간단한 YAML 파일 형태로 코드베이스에 직접 저장하고, 로컬과 클라우드 간 양방향 동기화가 가능해짐
- 사양, 테스트, 컬렉션 등을 Git으로 버전 관리할 수 있어 코드와 동일하게 변경 추적 및 관리가 쉬워짐
- 파일 기반 저장으로 인해 Postman의 에이전트 모드가 전체 API 라이프사이클을 자동으로 조율하는 것이 가능해짐
- 영어로 간단히 지시하면 코드베이스를 스캔해 라우트, 스펙, 테스트, 목(mock)까지 자동으로 생성 및 동기화함
- 모든 변경 내용은 리뷰가 가능한 최신 Git 커밋으로 나타나 개발자가 쉽게 확인 가능함
- 지원 AI 모델로는 Claude, GPT, Self-hosted 등이 있으며, MCP 서버까지 포함해 유연하게 활용 가능
- GitHub, Sentry, Linear 등 개발팀이 쓰는 다양한 도구와 연동해 맥락 정보를 활용할 수 있음
- 동일한 컬렉션, 사양, 테스트가 개발 환경과 CI 파이프라인(Pre-commit hook, Github Actions 등)에서 재작성 없이 그대로 실행됨

## 세부 요약 - 주제별 정리

### API 워크플로우의 기존 문제점은 관리 분산과 복잡성에 있었음을 강조함

- 이전까지 API 개발 시 엔드포인트, 테스트, 컬렉션 생성 및 관리가 하나의 통합 환경에서 이뤄지지 않았음
- IDE(코드 에디터)에서 API 엔드포인트를 개발하지만, 컬렉션/테스트 작성은 Postman 같은 별도의 툴에서 작업해야 했음
- CI(지속적 통합) 환경에서 동작하도록 테스트 과정을 별도로 세팅해야 하는 번거로움이 존재함
- 이러한 분산 구조로 인해 스펙, 컬렉션, 테스트 등 핵심 리소스들이 각각 다른 위치에 존재하며, 이들 간 연동 및 버전 관리가 어려웠음
- 결과적으로 변경 추적, 동기화, 협업 및 유지보수가 매우 까다로웠다는 점을 사용자 입장에서 짚어줌

### Postman의 Git-native 업데이트로 전 과정이 하나의 코드베이스로 통합됨

- Postman이 최근 대규모 업데이트를 통해 모든 주요 리소스를 Git 기반으로 관리할 수 있도록 플랫폼을 업그레이드함
- 컬렉션, 스펙, 목(mock), 테스트 등 Postman 자원 전체를 코드베이스 내에 간단한 YAML 파일로 저장할 수 있음
- 이로써 개발자는 기존 코드와 함께 API 사양 및 테스트를 동일한 저장소에서 관리 가능
- YAML 파일 기반이기 때문에 읽고 쓰기가 쉽고, 구조 변경도 간단해짐

### 로컬 코드베이스와 Postman 클라우드 간 양방향 동기화가 간편하게 구현됨

- Postman 데스크톱 앱을 통해 로컬에서 작업한 컬렉션 및 엔드포인트를 직접 확인 및 수정 가능
- 로컬 환경과 Postman 클라우드 간의 실시간 ‘투웨이 동기화(양방향 싱크)’를 지원해, 어느 쪽에서든 변경 사항이 자동 반영됨
- 예시로, 엔드포인트 테스트 문서를 수정하면 에디터에서 곧바로 Git diff 명령을 통해 해당 변경 내역을 확인할 수 있음
- 이런 구조로 인해 개발 단계별 변경 추적 및 버전 관리가 용이해짐

### Git을 통한 API 리소스의 버전 관리가 쉽고 강력하게 가능해짐

- 모두 파일로 떨어지다 보니 코드 변경과 동일하게 스펙, 테스트, 컬렉션 등에도 버전 관리가 적용 가능
- 각 변경 내역은 커밋 단위로 기록되며, 팀원 간 변경점 검토 및 관리가 수월해짐
- Git diff, 커밋 이력 활용이 가능해져, 코드와 테스트, API 사양 간의 일관성을 손쉽게 유지할 수 있음

### 파일 기반 구현으로 Postman Agent가 전체 API 라이프사이클 자동 관리에 특화됨

- Postman Agent Mode를 활용하면 API 라이프사이클 전 과정을 자동화할 수 있음
- 플레인 잉글리시(영어 지시문)로 원하는 목적을 설명하면, 코드를 분석해 엔드포인트 라우트, 스펙, 테스트, 목까지 자동 생성 및 갱신 가능
- Flask 앱 예시에서, 코드베이스를 스캔해 모든 API 라우트 발견, 관련 스펙 작성, 자동 테스트 구축 및 목 세팅까지 자동 지원
- 이 모든 과정이 최신 커밋 형태로 저장되어 리뷰/수정 가능함

### 주요 AI 모델 및 서버 지원으로 유연한 API 자동화가 가능함

- 작업 과정에서 사용할 AI 모델을 선택할 수 있음: Claude, OpenAI GPT, 셀프 호스팅 AI 등 선택적 활용
- MCP(Multi-Cloud Platform) 서버까지 지원하므로 다양한 팀 환경에서 적용 가능

### 개발자가 이미 활용 중인 다양한 협업 도구와 자연스럽게 연동됨

- GitHub, Sentry, Linear 등 자동화/협업 도구에서 맥락 데이터를 불러와 Postman에서 활용 가능
- API 문서화, 오류 추적, 이슈 관리 등 팀 협업 워크플로우와 매끄럽게 결합 가능

### 동일 리소스가 개발 환경과 CI 파이프라인에서 일관성 있게 동작함

- 로컬에서 만든 컬렉션 및 테스트가 동일하게 CI 파이프라인(프리커밋 훅, GitHub Actions 등)에서 실행됨
- 환경 별로 테스트 케이스를 재작성할 필요 없이 한 번 작성한 리소스를 모든 개발 환경에서 재활용 가능
- 테스트 실패 등 이슈가 있을 때 빠르게 피드백 받고 일관된 코드 품질 유지가 가능함

### Postman은 이번 업데이트로 API 개발 생산성과 협업 효율을 크게 증진함을 강조함

- 영상 제작자는 10년 가까이 Postman을 사용해 왔다고 직접 내세우며, 이 업데이트가 단순 버전업이 아니라 패러다임 전환임을 강조
- 기존에 느꼈던 불편함(분산, 동기화, 반복 작업)을 이 한 번의 업데이트로 모두 해소했다고 평가
- 모든 개발자와 팀에게 이 새로운 Postman을 경험해 볼 것을 추천하며, 영상 설명란에 예시 리포지터리 및 Postman 공식 링크를 안내함
