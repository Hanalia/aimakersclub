## *RAG 2.0: 에이전틱 RAG와 지식 그래프의 결합 (무료 템플릿 소개)* 핵심 요약

- 영상 제목은 "RAG 2.0: 에이전틱 RAG + 지식 그래프의 결합 (무료 템플릿)"으로, 두 가지 최신 RAG(검색증강생성) 전략의 융합을 소개함
- 제안된 AI 에이전트는 에이전틱 RAG(Agentic RAG)와 지식 그래프(Knowledge Graphs)를 결합하여, 질의에 따라 최적의 정보를 찾아 답변할 수 있도록 설계됨
- 백터 데이터베이스에는 Neon의 서버리스 PostgreSQL과 PG Vector 확장 모듈을 활용하여 문서 임베딩 및 저장을 처리함
- 예시 문서는 빅테크 기업들의 AI 프로젝트, 각 사의 협업 관계 등 다양한 정보로 구성되어 있음
- 동일 문서를 Graffiti 및 Neo4j로 지식 그래프화하여, 기업 간 관계 탐색 등 관계형 질의에 특화된 저장방식을 병행
- AI 에이전트는 사용자가 특정 기업의 AI 이니셔티브 등 단일 주제 질의 시 백터DB에서 검색하고, 기업 간 관계 등 복합 질의시 지식 그래프 검색을 자동 선택함
- 각 질의마다 어떤 툴(벡터DB, 그래프DB)을 사용했는지 반환 내역에 명확히 표기함
- 사용자가 원한다면 시스템 프롬프트 조정 혹은 직접 명시를 통해 두 DB를 동시에 활용할 수도 있음
- 이 AI 에이전트 및 구축방법을 담은 무료 템플릿과 상세 영상을 별도 링크로 제공함
- 영상 후반부에서는 Cloud Code를 사용한 개발 방법과 개인 구축 가이드까지 안내하며 실습 확장성을 강조함

## 세부 요약 - 주제별 정리

### 에이전틱 RAG와 지식 그래프는 복합 정보탐색에 최적임을 시연함

- 영상 제작자는 수개월 동안 다양한 RAG(검색증강생성) 방식을 탐구해왔음을 언급함
- Agentic RAG와 Knowledge Graphs 두 가지 전략에 주목하게 되었다고 밝힘
- 두 방식을 결합해 한층 강력한 AI 에이전트를 개발했다고 소개함
- 이 에이전트는 사용자의 질문에 따라 가장 적합한 정보 소스를 스스로 판단해서 검색함
- 예시로 제시한 AI 에이전트는 질의의 성격에 따라 벡터DB와 그래프DB를 자동 또는 혼합 활용함
- 이 시스템은 복합적이고 다단계적인 질문에 유연하게 대응하는 것이 특징임

### 벡터 데이터베이스 환경은 Neon 기반 서버리스 PostgreSQL 및 PG Vector로 구현됨

- 벡터 데이터베이스는 PostgreSQL과 PG Vector 확장(extension)을 사용하여 구현함
- 서버리스 데이터베이스 환경으로 Neon 플랫폼을 활용함
- 문서를 적절한 크기로 분할(chunking)하고, 각 청크마다 임베딩을 생성하여 저장 처리함
- 예시 문서는 빅테크 기업들의 다양한 정보와 AI 프로젝트, 기업 간 파트너십 데이터 등 광범위한 정보를 포함함
- 이러한 설정은 벡터DB 환경에서 유사문장 검색 및 빠른 정보 추출에 최적화되어 있음

### 동일 정보를 Graffiti와 Neo4j로 지식 그래프화하여 관계형 검색을 지원함

- 동일한 원본 문서를 지식 그래프로 변환해 별도로 저장함
- 지식 그래프 구축에는 Graffiti와 Neo4j 도구를 사용함이 강조됨
- 지식 그래프에서는 각종 엔티티(예: 기업명), 관계(예: 협력, 호스팅) 등이 명확히 연결되어 있음
- 예시로, "Amazon은 Anthropic이 자체 AI 모델을 호스팅하는 플랫폼이다"와 같은 관계형 정보 쿼리가 가능함
- 지식 그래프는 복잡한 관계 탐색 질의에 보다 적합함

### 에이전트는 질의의 유형을 판별해 백터DB 또는 그래프DB를 자동 선택함

- 사용자가 "Google의 AI 프로젝트에는 무엇이 있나?"와 같이 특정 엔티티 위주 질문 시 벡터DB 검색을 선택
- 에이전트는 벡터 데이터베이스 검색을 거쳐 구체적인 답변을 반환함
- "Amazon과 Anthropic 사이의 관계"나 "두 회사가 어떤 식으로 협력하나?"와 같이 관계성 탐색 질문 시, 에이전트가 자동으로 지식 그래프 검색을 시행
- 시스템 프롬프트 조건에 따라 에이전트의 질의 소스 선택 행위가 유도됨

### 매 요청마다 사용한 데이터 소스(툴)를 반환 결과에 포함시킴

- 에이전트는 답변 시 사용된 툴(벡터DB, 그래프DB, 혹은 둘 다)을 명확히 결과 내에 표기
- 예: Google에 대한 AI 질문 시 "Vector database를 사용함"이라고 안내
- 관계형 정보 질의 시에는 "Graph search를 사용함"으로 결과에 명시
- 유저는 결과를 통해 응답의 근거가 된 데이터 소스를 파악할 수 있음

### 사용자 지정으로 두 종류의 데이터 소스를 동시에 활용 가능함

- 사용자가 시스템 프롬프트를 조정하거나, "두 DB 모두 사용"을 명시할 경우, 에이전트가 벡터DB와 그래프DB를 병행해서 질의 처리함
- 실제 예시에서는 두 소스를 동시에 활용해 더욱 심도 있는 답변을 제공하도록 시연함
- 이 방법은 다층 정보 수집, 질의의 복잡도가 높을 때 유용함

### AI 에이전트는 무료 템플릿으로 공개되어 직접 실습이 가능함을 알림

- 영상 마지막에 AI 에이전트의 무료 템플릿이 제공됨을 안내
- 별도의 링크를 통해 전체 구축 과정과 코드, 실습 방법을 숙지할 수 있다고 설명
- 클라우드 코드(Cloud Code)를 활용해 에이전트 개발을 쉽게 접근 가능하다고 부연함
- 시청자 스스로 해당 에이전트를 구축, 활용할 수 있도록 지원함

### Cloud Code 활용법 및 개인 구축 프로세스가 별도 영상에서 상세히 안내됨

- 영상 본문은 짧게 시스템과 원리를 소개함
- 전체 에이전트 구축, Cloud Code를 이용한 구현, 설치 및 실전 배포 방법은 별도의 본편 영상과 템플릿에서 추가 안내함
- 영상 내 링크를 통해 구체적 개발 절차와 실습 가이드 확보 가능
- 누구나 무료로 시도 가능한 점을 다시 한번 강조함
