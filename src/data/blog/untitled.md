## *스택 디프를 활용한 AI 코드 리뷰 혁신* 핵심 요약

- 영상 제목은 "스택 디프가 AI 코드 리뷰 방식을 어떻게 바꿨는가"로 번역됨
- AI 코드 리뷰는 매우 강력하며, 검증 과정에서 필수 도구로 여겨지고 있음
- 전통적 풀 리퀘스트 방식은 각 변경마다 승인을 기다려야 하며 비효율적임
- 스택 디프(Stacked Diffs)를 활용하면 작은 변화들을 차례로 쌓아 전체 기능을 점진적으로 개발할 수 있음
- 스택 디프 방식을 사용하면 각 변경사항에 즉시 피드백이 가능하여 효율적임
- 이 방식은 AI 코드 리뷰에 최적화되어, 각 단계별로 세밀한 피드백을 받을 수 있음
- '그래파이트(Graphite)'라는 플랫폼이 스택 디프 워크플로우를 쉽게 구현할 수 있도록 명령줄 도구와 대시보드, AI 코드 어시스턴트를 제공
- 사용자는 명령줄에서 쉽게 여러 개의 풀 리퀘스트를 계층 구조로 생성하고, 대시보드에서 각 PR을 확인 및 관리할 수 있음
- 그래파이트 대시보드와 에이전트를 활용해 PR별로 AI 리뷰와 병합 등 모든 작업을 한 곳에서 가능
- 세 개의 PR이 순차적으로 쌓인 스택 디프 예시를 직접 보여주며, 대형 PR 하나를 AI로 리뷰하는 것보다 훨씬 뛰어난 결과를 체험함
- 동영상 설명에 실제 활용하는 자세한 가이드 영상 링크도 존재

## 세부 요약 - 주제별 정리

### AI 코드 리뷰는 반드시 활용해야 할 강력한 검증 도구임

- AI를 통한 코드 리뷰는 최근 개발 프로세스에서 큰 변화를 가져오고 있음
- 영상 제작자는 AI 코드 리뷰를 개인적으로 검증 과정에서 필수적인 단계로 간주함
- AI가 코드의 결함, 논리적 오류, 스타일 문제 등의 다양한 측면을 진단하는 데 매우 효과적임
- 기존의 코드 리뷰 프로세스와 병행하여 사용하면 높은 신뢰성과 생산성을 동시에 확보할 수 있음

### 전통적 풀 리퀘스트 방식은 비효율적임을 지적함

- Git 및 GitHub 기반의 전통적 풀 리퀘스트 워크플로우는 다음의 단점이 있음
    - 한 번에 하나의 변경사항만 처리 가능
    - 각 변경마다 동료의 승인을 기다려야만 다음 단계 진행이 가능
- 이러한 방식은 대규모 기능 개발 시 전체 개발 기간이 길어짐
- 리뷰어와 개발자 모두에게 반복 작업과 대기 시간이 증가함

### 스택 디프는 작은 변화들을 위계적으로 쌓아 나가며 작업을 혁신적으로 개선함

- 스택 디프(또는 스택드 디프)는 "작은 증분적 변화"를 순차적으로 계층구조로 쌓는 방식
- 각 변화(디프)는 이전 변경사항을 바탕으로 추가됨
- 전체 기능이나 대규모 PR을 한 번에 올리는 대신, 세부 기능별로 작은 PR을 생성하고 관리함
- 각 단계별로 독립적으로 리뷰 받고 즉시 반영할 수 있어 빠른 개발 주기가 가능함

### AI 코드 리뷰는 스택 디프 방식을 통해 더욱 정밀하게 구현될 수 있음

- 작은 단위의 PR을 생성하면, AI가 각 단계에 보다 집중해서 리뷰 가능
- 각 디프별로 상세한 피드백을 받아 코드의 품질과 정확성을 높일 수 있음
- 대규모 PR 한 개를 AI에게 맡겼을 때보다, 분할된 PR 각각을 리뷰할 때 결과물이 더욱 우수함
- “insane(엄청나다)”라는 표현으로 그 차이를 강조

### 그래파이트(Graphite)는 스택 디프 워크플로우를 위한 최적화된 도구임

- 그래파이트는 스택 디프 방식의 작업에 특화된 플랫폼
- 명령줄 도구 제공: GitHub CLI와 유사하나, 스택 디프 관리에 맞춰졌다 설명
- 그래파이트 대시보드를 통해 전체 PR과 스택 구조를 시각적으로 관리 가능
- AI 코드 어시스턴트(그래파이트 에이전트)를 통해 각 PR 리뷰에도 AI가 직접적으로 개입함

### 그래파이트의 가입 및 데모 사용 경험은 매우 직관적으로 설계되어 있음

- 그래파이트 플랫폼 가입 후 제공되는 데모가 첫 스택 디프 생성 과정을 친절하게 안내함
- 사용자는
    - 첫 번째 변화(디프)를 만들고
    - 첫 풀 리퀘스트(PR) 생성
    - 이후 변화를 추가하며, 두 번째와 세 번째 PR을 해당 순서대로 쌓음
- 명령줄 상에서 직관적으로 3개의 PR을 계층적으로 만들 수 있음
- 각 PR은 기존 Git, GitHub CLI와 유사한 방식으로 관리됨

### 그래파이트 대시보드를 통해 스택 디프의 전체 구조와 병합을 한눈에 관리할 수 있음

- 대시보드 내에서 각 PR의 상태를 시각적으로 확인할 수 있음
- 그래파이트 에이전트(코드 어시스턴트)를 활용해 PR별 리뷰 가능
- 전체 PR을 대시보드 내에서 개별적으로 머지(Merge)하거나, 모두 병합해서 메인(Main) 브랜치로 반영 가능
- GitHub 저장소에 직접 들어가지 않고도 모든 관리가 가능하다고 언급

### 실제 예시에서는 3개의 풀 리퀘스트가 연쇄적으로 쌓인 스택 디프 구조를 시연함

- 첫 번째 PR 이후, 그 위에 두 개의 PR을 추가로 생성하여 총 3개 PR로 계층을 만듦
- 각 PR은 기능 브랜치(feature branch)로 관리되고, 서로 차례로 병합되는 구조
- 최종적으로 준비가 끝나면 모든 변경사항을 메인 브랜치에 병합

### 각 변경사항마다 AI를 활용해 개별적으로 리뷰를 진행함

- 제작자는 각 스택 디프(3개의 PR)마다 Claude 코드(Claude code, AI)를 Reviewer로 지정함
- 각 PR이 GitHub 또는 그래파이트 대시보드 내에서 AI로 개별적으로 검증됨
- “한 번에 거대한 PR을 리뷰하는 것과 비교하면” 결과 차이가 뚜렷함을 경험적으로 강조

### 영상 설명란에 스택 디프 활용법을 자세히 다룬 다른 영상을 별도 제공함

- 실제 스택 디프 생성 및 AI 리뷰 과정 방법을 보여주는 추가 영상 링크가 설명란에 포함됨
- 본 영상은 개념 설명과 실제 워크플로우 체험 중심임
- 그래파이트와의 협업으로 이 워크플로우 전체를 시연함

### 마지막으로 스택 디프와 그래파이트의 결합이 AI 기반 코드 리뷰 효율성을 극대화함을 강조함

- 그래파이트를 통한 스택 디프 구조는 개발자 생산성과 리뷰 품질, 코드 관리 편의성을 모두 향상시킴
- 작은 변화 단위에 맞춰 AI의 리뷰 효과가 극대화됨을 실제 체험을 통해 보여줌
- “Git을 매우 지능적으로 사용하는 것”이라고 그래파이트-스택 디프 방식을 평가함
