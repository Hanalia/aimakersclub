## *MCP 서버의 큰 문제점과 그 해결책* 핵심 요약

- 이 영상은 MCP(Multi-Component Protocol) 서버가 겪고 있는 핵심 문제와 이를 해결할 수 있는 최신 방법을 다룸
- MCP 프로토콜이 점점 인기를 얻으면서, 토큰 소모와 컨텍스트 붕괴(context rot) 문제가 명확하게 드러남
- 영상은 MCP의 문제를 실사용 예시로 구체적으로 설명함
- 5개의 일반적인 AI 코딩용 MCP 서버 각각이 수천 개의 토큰을 사용하여 AI 에이전트에게 자신의 기능을 설명해야 하므로, 동일 세션에서 여러 MCP 서버를 함께 쓰는 것은 극히 비효율적임
- 이로 인해 개발자나 사용자가 쉽게 토큰 할당량을 초과하게 됨
- 이 문제의 혁신적 해결책으로 Anthropic이 개발한 '에이전트 스킬(agent skills)'이 등장함
- 에이전트 스킬을 활용하면 AI가 API 엔드포인트에 상호작용하는 스크립트를 동적으로 생성하거나, 코드 작성 및 실행도 실시간으로 가능해짐
- 영상은 MCP 문제의 원인, 구체적 사례, 그리고 기술적 대안을 순서대로 다루며, 결론적으로 '스킬과 코드 실행'이 게임체인저가 될 수 있음을 강조함
- 마지막으로 "이로 인해 MCP 시대가 종말을 맞이하는가"라는 근본적 질문을 남김

## 세부 요약 - 주제별 정리

### MCP 프로토콜의 인기와 심각한 결점이 동시에 부상함

- MCP 프로토콜은 다양한 인공지능 AI 서버들이 통신할 수 있는 구조로, 점차 많은 주목을 받고 있음
- 프로토콜이 널리 쓰이면서, 두 가지 핵심 문제가 커지고 있음: 토큰 과다 소모와 문맥 붕괴(문맥 정보가 금세 사라짐)
- 최근 AI 개발자 커뮤니티에서 이 두 가지 이슈(특히 'token consumption'과 'context rot')가 매우 뜨거운 논쟁 거리로 떠오름
- 영상은 이 문제를 실제 예시와 함께 조목조목 설명하겠다고 밝힘

### 여러 MCP 서버 사용 시 토큰 소모와 컨텍스트 붕괴 문제가 급격히 심각해짐

- 영상에서는 AI 코딩 지원을 위해 많이 사용되는 MCP 서버 5개가 등장함
- 각 서버는 자신이 가능한 일을 AI 에이전트에게 설명할 때만도 수천 개의 토큰을 필요로 함
- 여러 MCP 서버를 한 세션에서 AI 코딩 에이전트에 물릴 경우, 초기 세팅만으로 급격히 토큰 소모가 폭증함
- 토큰 예산이 빠르게 소진되어 실제로는 여러 서버를 조합하기가 비현실적임
- "나는 내 코딩 어시스턴트와 세션에서 이 다섯 MCP 서버를 절대 동시에 사용하지 않는다"는 말을 반복 강조함
- 이런 병목 때문에 MCP 기반 워크플로우의 실질적 확장성에 심각한 한계가 있다 언급

### Anthropic의 '에이전트 스킬(agent skills)'이 결정적 해결책으로 등장함

- 최근 Anthropic(앤트로픽)에서 공개한 '에이전트 스킬'이 MCP의 구조적 문제에 본질적 해답을 제시함
- 에이전트 스킬로 인해 AI 및 개발자가 필요할 때마다 동적으로 기능(스크립트)을 정의하고 제공받을 수 있게 됨
- API 엔드포인트와 연동하는 스크립트 생성을 자동화할 수 있어, 굳이 여러 서버의 기능을 미리 모두 설명하며 토큰을 낭비하지 않아도 됨
- 사용 예시로 "코드 생성, 실행, 인스트럭션 자동 생성" 등 실제 프로그래밍 워크플로우가 가능해짐

### 에이전트 스킬과 실시간 코드 실행이 MCP 한계를 완전히 극복함

- 스킬 기반 접근은 AI가 필요할 때만 필요한 작업 정의·실행을 능동적으로 수행하게 함
- 즉, 불필요한 맥락 정보 전달(설명)이나 토큰 소모가 없으므로 확장성과 효율이 극적으로 향상됨
- "코드 생성 및 실행의 정점(code execution at its finest)"이라는 표현으로 변화의 혁신적 성격을 강조함

### 영상은 MCP의 문제, 혁신적 대안, 향후 전망 순서로 구조화함

- 영상은 MCP 프로토콜의 구조적 약점 설명 → 실사례 제시(5개 서버 예시) → Anthropic의 솔루션 안내 → 예시적 이점 강조 순서로 진행됨
- 결론적으로, '스킬(Skill)'과 '코드 실행' 방식이 이제 AI 워크플로우의 필수적 트렌드가 될 수 있음을 제시함
- 마지막으로 "MCP의 시대가 끝나는 것인가?"라는 질문으로 남기며, 기술적 전환기에 놓여 있음을 암시함
