## *AI 에이전트를 극대화하는 가장 간단한 방법 (스킬)* 핵심 요약

- 이 영상은 Anthropic에서 최근 선보인 "스킬(Skills)" 기능이 왜 AI 분야에서 중요한 발전인지 설명함
- 스킬은 단순함을 핵심 모토로 하여 복잡한 기능 관리의 어려움을 극적으로 해결함
- AI 에이전트에게 많은 능력을 부여할 수 있으나, 과도한 정보가 컨텍스트 윈도우(context window)를 쉽게 초과시킴
- 기존 MCP 서버 방식처럼 초기부터 모든 도구를 일괄 제공하면, 실제로 필요하지 않은 정보도 에이전트에 전가되어 비효율 초래
- Anthropic의 스킬은 "점진적 공개(progressive disclosure)" 방식을 도입해, 필요 시에만 필요한 정보를 단계적으로 노출
- 시스템 프롬프트에는 스킬 또는 기능의 "설명"만 간단 증여(1단계)
- 사용자가 실제로 해당 기능(예: PDF 처리 요청)을 필요로 하면, 에이전트가 자세한 설명이 담긴 skill.md 파일을 추가로 읽어들임(2단계)
- skill.md에 명시적으로 참조된 다른 문서 등이 있다면, 에이전트는 점진적으로 그들 역시 로드하여(3단계), 작업에 필요한 맥락만 취득
- 이러한 단계적 정보 노출 방식 덕분에 컨텍스트 윈도우를 효율적으로 사용하고, 에이전트 성능도 극대화할 수 있음
- 영상은 PDF 처리 스킬을 예시로, 스킬 작성 파일 구조와 단계별 정보 공급의 구체적 흐름을 설명함

## 세부 요약 - 주제별 정리

### Anthropic의 스킬 도입은 최신 AI 발전에서 가장 중요한 변화를 이끌고 있음

- Anthropic은 최근 "스킬"이라는 기능을 발표했고, 이는 AI 업계에서 주목받는 진전임
- 발표된 지 오랜 시간이 지나지 않았지만 벌써부터 실질적 영향력이 강조됨
- 영상 제작자는 "AI의 최근 발전 중 가장 중요한 것 중 하나"라고 평가

### 스킬의 핵심은 '단순함'과 '점진적 정보 공개'임

- Anthropic의 철학은 "단순할수록 좋다(The simpler the better)"에 기반함
- 에이전트에게 다양한 기능을 부여하되, 정보의 과다 주입을 피함
- 복잡한 도구 관리 없이, 필요한 시점에 필요한 정보만 단계적으로 제공함

### 기존 방식(MCP 서버 등)은 도구 과다 노출로 오히려 에이전트의 과부하를 초래함

- MCP 서버 등 전통적 접근법에서는 시스템 프롬프트에 너무 많은 도구를 미리 제공
- 이로 인해, 실제 대화 흐름에서 필요하지 않은 도구까지 컨텍스트 윈도우에 포함됨
- 모든 정보가 한 번에 제공될 경우, 에이전트가 혼란을 겪고 효율이 저하됨

### 스킬의 점진적 공개는 에이전트의 컨텍스트 관리 문제를 혁신적으로 해결함

- 스킬 방식은 "progressive disclosure"(점진적 공개) 원칙을 적용
- 일괄 공개 대신, 단계적으로 정보를 노출해 컨텍스트 윈도우의 부담을 줄임
- 실제 사용자 요청이 들어오기 전까지, 자세한 정보와 문서는 노출되지 않음

### 1단계: 시스템 프롬프트 내에 기능 설명만 간결히 전달함

- 가장 먼저, 시스템 프롬프트에는 단순히 스킬(예: PDF 처리)의 '설명'만 포함됨
- 예시: "이 에이전트는 PDF 처리 기능(capability)을 보유하고 있음" 정도의 정보만 제공
- 에이전트는 해당 기능 보유 여부만 인지하고 세부 구현 방법은 아직 알지 못함

### 2단계: 실제 요청이 들어올 때 skill.md 파일을 통해 명확한 지침을 획득함

- 사용자가 실제로 해당 스킬을 활용해달라고 요청하면, 에이전트는 skill.md 파일을 로드
- skill.md에는 기능 구현을 위한 세부 지시사항이 명문화되어 있음
- 실질적인 사용 방법, 단계, 옵션 등이 상세히 기재되어 있음
- 이 단계에서 필요한 추가 데이터를 컨텍스트에 불러옴

### 3단계: 필요할 때 skill.md가 참조하는 추가 문서를 단계적으로 불러옴

- skill.md 파일에는 종종 추가 문서(참고자료, 상세 설명 등)에 대한 참조가 포함됨
- 실제로 해당 기능의 복잡한 사용이 요구될 경우, 이들 문서를 3차적으로 로드
- 각 작업에 필요한 맥락만 컨텍스트 윈도우로 들어와, 전체 정보량을 최소화

### PDF 처리 스킬 예시로 단계별 작동 방식을 구체적으로 안내함

- 예시: PDF 처리 능력을 가진 에이전트
    - 1단계: 시스템 프롬프트에 "PDF 처리 기능 보유" 설명만 제공
    - 2단계: 사용자가 PDF 작업 요청 → 에이전트가 skill.md로 상세 지침 확인
    - 3단계: 필요 시 skill.md가 참조하는 추가 문서 로드
- 이런 흐름을 통해, 사용 목적에 부합하는 모든 정보를 시점별로 즉시 획득

### 세분화된 단계적 정보공유로 컨텍스트 윈도우 사용량을 최소화함

- 필요한 정보를 "지금-여기"에서만 노출, 전체 컨텍스트 윈도우가 낭비되지 않음
- 복잡한 시스템 설계 없이도 에이전트의 역량을 확장 가능
- '점진적 정보 공개'가 에이전트의 과부하 방지와 실제적 효율화에 크게 기여

### Anthropic의 스킬 구조가 맞춤형 에이전트 구축에 이상적으로 설계됨

- 모든 스킬(Anthropic에서 제공 또는 커스텀 제작)은 동일한 progressive disclosure 구조를 따름
- 개발자는 skill.md 등 문서 기반으로 각 스킬을 유연하게 설계할 수 있음
- 여러 문서/단계로 나누어 설계 가능하여, 다양한 커스텀 에이전트 시나리오에 적용 용이

### 영상은 점진적 정보 공개 전략의 논리와 장점을 명확하고 실용적으로 설명함

- 구체적 예시(도구 목록 과부하 vs 점진적 공개 결과)로 효과 차이 제시
- Anthropic 스킬 구조 그 자체가 에이전트의 기능성과 단순함을 양립시키는 핵심임을 강조
- 궁극적으로, "스킬"이라는 매우 단순한 아이디어가 AI 에이전트의 커스텀화·확장성 문제를 격차 있게 해소함을 시사
