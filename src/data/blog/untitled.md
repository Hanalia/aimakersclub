## *Granola가 Zoom 및 Otter와 같은 거대 기업을 제치고 AI 노트테이킹 시장에서 성공한 이유* 핵심 요약

- **"Granola"라는 제품은 기존 AI 기반 노트테이킹 서비스와는 다른 고객의 요구를 발견함**
- **많은 고객들이 "모든 노트를 자동으로 받아쓰기" 해주는 것이 아닌 "더 좋은 노트 작성에 도움을 받는" 방식을 원함**
- **Granola는 사용자가 스스로 더 효율적으로 노트를 작성할 수 있도록 지원하는 방향으로 제품을 개발함**
- **Granola는 초기 기능(‘레고 브릭’ 3개 수준)이 완성된 뒤에도 반복적으로 새로운 조합과 기능 세트를 개발함**
- **기본 기능 위에 프로젝트 및 팀 워크스페이스, 협업 기능 등 새로운 레이어를 차례대로 쌓아감**
- **Granola는 단순히 기능 나열에 그치지 않고, 특정 사용자 및 특정 작업 중심으로 지속적으로 집중함**
- **시장에서 기존 대형 솔루션(Zoom, Otter 등)이 채우지 못한 틈새 수요를 정확히 분석하여 제품을 설계함**
- **이러한 연속적인 창의적 조합과 고객 중심 접근이 Granola의 차별화와 성공의 원동력임**

## 세부 요약 - 주제별 정리

### Granola는 대형 경쟁사와 차별화된 고객 니즈를 발굴함

- Granola는 Zoom, Otter 같은 기존 AI 노트테이킹 툴과 달리 다른 사용자의 요구에 집중함
- “자동으로 모든 노트를 받아준다”는 기능이 기존 대형 제품의 핵심 판매 포인트였음
- Granola는 다수 사용자들이 “내가 더 좋은 노트를 직접 만들고 싶다”, “AI가 보조해주길 원한다”는 새로운 수요를 발견함
- 이러한 미충족 요구를 제품에 반영하여 방향성을 설정함

### Granola는 사용자가 주체가 되는 노트 작성 지원에 제품 개발의 초점을 맞춤

- Granola의 차별점은 사용자가 능동적으로 더 좋은 노트를 작성할 수 있도록 돕는다는 것임
- 기존 제품들이 자동화에 의존했던 반면, Granola는 사용자의 작업 경험과 직접적인 개입을 강조함
- “특정 작업 및 사용자”에 최적화된 지원 기능을 개발함

### Granola는 초기 기능 이후에도 지속적인 조합과 혁신을 시도함

- 초기에는 몇 가지 주요 ‘레고 브릭(기능 조각)’ 조합으로 제품을 구성함
- 그러나 여기에서 멈추지 않고, 기존 기능을 기반으로 반복적으로 새로운 조합을 만들어냄
- “초기 3개의 레고 브릭을, 또 다른 독특한 세트로 연결하는 작업”이 계속됨

### Granola의 진화: 기능 확장과 팀/프로젝트 협업 지원

- Granola는 제품이 자리를 잡은 뒤, 프로젝트 워크스페이스와 팀 협업 기능 등 새로운 레이어를 추가함
- 점차 팀 단위의 협력, 프로젝트 기반 공동 작업 관리 등으로 범위를 넓혀감
- 단순 개인 노트툴에서 협업 도구로 성장하는 전략적 방향성을 보여줌

### Granola의 반복적 조합 방식이 차별화와 성장을 이룸

- Granola 제품 개발은 “처음 레고 조립”에 그치지 않고 계속해서 새로운 기능 세트를 조합·추가함
- 각각의 새로운 기능 추가는 실제 사용자 요구와 업무 시나리오에 근거함
- 이런 반복적 ‘기능 조합-확장’이 Granola의 차별성과 경쟁력을 만들어냄

### Granola는 고객 문제와 과업 중심 접근으로 시장의 틈새를 공략함

- Granola의 성공에는 “일반적 기능 나열”이 아니라 “특정 고객 문제/작업”에 대한 집중적 해결이 핵심적임
- 경쟁사 대비 Granola는 ‘미충족 수요’를 정확하게 파악하여 더 작은 단위의 문제의식에서 출발
- 고객의 실제 작업 환경, 과업, 문제를 인식하고 이를 매번 제품 진화에 반영함

### 영상은 실전 사례로서 Granola의 전략적 실행력을 강조함

- 모든 설명은 Granola를 구체적 사례로 들어 “이론→실천”의 과정을 보여줌
- 단순히 전략이나 이론만이 아닌, 실제 시장에서 ‘작은 제품’이 ‘거대 솔루션’을 이기는 실제 과정을 제시함
- Granola의 지속적 변화와 반복적 혁신이 시장에서의 실질적 성과로 나타남
