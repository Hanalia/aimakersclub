## *스택 디프(분할 변경)를 활용한 AI 코드 리뷰 혁신* 핵심 요약

- 영상 제목이 "Stacked Diffs Changed How I Reviewed Code with AI"로, AI 코드 리뷰의 효율성을 극대화하는 스택 디프(분할 변경) 방식의 워크플로우를 소개함
- 전통적인 풀 리퀘스트(PR) 방식에서는 변경 후 리뷰 및 승인까지 대기해야 하는 단점을 지님
- 스택 디프 방식은 작은 변경 사항들을 순차적으로 쌓으며, 각 단계별로 AI로부터 집중적이고 정밀한 피드백을 받을 수 있음
- 그래파이트(Graphite)라는 플랫폼이 이러한 스택 디프 방식 적용을 손쉽게 만들어주는 기능(명령줄 툴, 대시보드, AI 비서)을 제공함
- 그래파이트 CLI는 GitHub CLI와 유사하지만 스택 디프 작업에 특화되어 있음
- 데모 과정을 통해 첫 스택 디프를 직관적으로 생성하고, 추가적으로 여러 PR을 겹겹이 쌓는 방법을 안내받음
- 최종 대시보드에서 각 PR을 관리하고, AI 에이전트로 각각을 개별적으로 리뷰할 수 있으며, 모든 변경을 한 번에 메인 브랜치로 병합 가능함
- 거대한 단일 PR을 리뷰하는 것과 달리, 분할된 변경 사항을 AI가 각각 검토하도록 하여 더욱 세밀하고 효과적인 코드 리뷰 결과를 얻을 수 있다고 강조함

## 세부 요약 - 주제별 정리

### AI 코드 리뷰는 필수적이며 더 효과적인 워크플로우가 존재함을 알림

- AI를 활용한 코드 리뷰는 검증 과정의 핵심 요소로 자리잡았다고 주장
- AI 코드 리뷰는 효율성, 정확성 측면에서 개발자의 필수 도구로 활용됨
- 기존의 단순 PR 형태만으로는 AI 리뷰의 잠재력을 충분히 이끌어내기 어려움
- 더욱 효과적인 AI 코드 리뷰를 위해 새로운 워크플로우 필요성을 강조함

### 전통적 풀 리퀘스트(풀리퀘) 방식의 한계를 설명함

- 일반적인 PR 프로세스에서는 한 가지 변경을 만든 후 승인 대기 → 반복 과정을 거침
- 각 변경 단계마다 승인 완료까지 개발자의 작업이 지연됨
- 대량의 변경을 한 번에 PR로 제출하는 것은 리뷰 효율과 집중도를 떨어뜨림
- 변경 내용의 복잡성이 커질수록 리뷰의 정확성과 속도가 저하됨

### 스택 디프는 작은 변경의 누적으로 대기 시간 없이 작업을 촉진함

- 스택 디프 방식에서는 각 변경(디프)을 작은 단위로 쪼개어 연속적으로 진행함
- 각 단계는 이전 단계 위에 누적되어 전체 기능의 완성을 지향
- 변경 사항이 작고 명확하므로, 승인 대기 없이 다음 단계를 계속 진행할 수 있음
- 결과적으로 전체 개발 및 리뷰 사이클이 현저히 단축됨

### 스택 디프 방식은 AI 코드 리뷰에 최적화되어 있음을 강조함

- 각 변경(디프) 단위가 작아 AI가 집중적으로 리뷰할 수 있음
- 문제 발생시, 어느 변경에서 문제가 발생했는지 빠르게 추적 가능
- AI의 피드백이 구체적이고 명확하여 개발자가 즉각 반영 가능
- 전체 기능 개발이 아닌, 각 미세 변경 단계를 독립적으로 리뷰함으로써 코드 품질 향상

### 그래파이트(Graphite) 플랫폼이 스택 디프 적용을 간편하게 지원함

- 그래파이트는 스택 디프 워크플로우를 위한 특화된 플랫폼임
- GitHub CLI처럼 명령줄 도구(CLI)를 제공하여 편리한 명령어 기반 PR 관리 가능
- 각 변경을 스택(겹침) 형태로 쉽게 연속 생성할 수 있음
- 종합 대시보드 제공으로 여러 PR과 상태를 한눈에 확인 가능

### 그래파이트 플랫폼은 AI 기반 코드 리뷰 및 PR 관리를 완비함

- 대시보드 내에 AI 코딩 어시스턴트 기능 내장
- 각 단계별 PR을 별도로 AI가 리뷰 가능
- 그래파이트 에이전트가 코드 리뷰를 지원하며 평가 피드백 제공
- 별도의 GitHub 인터페이스 접근 없이도 모든 관리를 그래파이트 플랫폼 내에서 가능

### 데모 과정을 통해 신규 사용자도 쉽게 스택 디프를 적용할 수 있음을 시연함

- 가입 시 첫 화면에서 직관적 데모 진행
- "첫 번째 변경(PR) 생성 → 풀 리퀘스트 생성 → 연속적인 추가 PR 생성"의 프로세스 실습
- 명령줄에서 3개의 PR을 연속으로 쌓아가며, 각 PR은 이전 단계 위에 누적됨
- 각 PR이 피처 브랜치에 생성되고, 단계적으로 메인 브랜치에 병합됨

### 최종 스택 디프 대시보드에서 PR별 리뷰, 병합 및 관리가 가능함

- 그래파이트 대시보드에서 각 PR을 별도로 관리할 수 있음
- 그래파이트 에이전트로 각각의 PR을 독립적으로 리뷰
- 모든 변경 사항을 한 번에 대시보드에서 메인 브랜치로 병합 가능
- GitHub 저장소 접속 없이도, 모든 변경 관리를 플랫폼 내에서 처리 가능

### 분할(스택)된 PR 리뷰는 대규모 PR 리뷰 대비 혁신적으로 나은 결과를 산출함

- 각각의 세분된 변경(pr)에 대해 AI가 독립 리뷰를 진행함
- 예로, 3개의 스택 PR 각각에 Claude 코드 AI를 적용해 리뷰 받음
- 거대한 단일 PR에서 봐야 하는 리뷰 분량과 비교했을 때, 집중력과 정확도 현저히 개선
- "이 방식으로 얻어진 결과가 전통 방식을 압도할 만큼 우수하다"고 강조

### 그래파이트와의 협업을 통해 본 워크플로우를 선보였으며 후속 자료도 안내함

- 본 영상 내용은 그래파이트와 협업을 기반으로 실제 사용 사례를 보여주기 위한 목적임을 밝힘
- 추가적으로 스택 디프 실습 영상 링크를 영상 설명란에서 제공함
- 본 영상은 전체적인 스택 디프 워크플로우 요약 및 실제 활용법 소개에 초점을 맞춤
