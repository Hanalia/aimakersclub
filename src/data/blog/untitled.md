## *Postman이 Git 기반으로 진화하며 API 구축 방식을 혁신함* 핵심 요약

- 기존의 API 작업 워크플로우는 엔드포인트, 스펙, 테스트, 컬렉션 등이 여러 도구와 경로에 분산되어 비효율적임
- 특히 AI 에이전트 등 다양한 프로젝트를 만드는 과정에서 API 관리가 항상 복잡하고 중복적이었음
- 새로운 Postman 업데이트로 모든 API 관련 리소스를 코드베이스 내에서 YAML 파일 형태로 관리 가능해짐
- Postman 리소스는 이제 로컬 코드베이스와 클라우드 간 양방향 싱크가 가능하여, 모든 수정사항을 git diff로 추적 가능함
- API 컬렉션, 스펙, 목(mock), 테스트 파일이 모두 버전 관리를 통해 코드와 동일한 수준의 변경 추적 및 협업이 가능해짐
- 평문 영어로 요구사항을 설명하면 Postman ‘에이전트 모드’가 코드베이스를 분석해 API 스펙, 테스트, 목 등을 자동 생성
- AI 모델(Claude, GPT, 자체 호스팅 등) 선택 및 GitHub, Sentry, Linear 등 다양한 툴과의 연동 지원
- 머신에서 실행되는 컬렉션과 CI 파이프라인(Pre-commit hooks, GitHub Actions 등)에서 실행되는 테스트가 완전히 동일하게 동작
- 환경 불일치 없이 API 테스트 자동화 및 버전 동기화를 실현함
- 영상 제작자는 Postman의 거대한 업데이트를 "AI 네이티브, Git 네이티브 API 플랫폼의 탄생"으로 평가하며 사용을 적극 추천함

## 세부 요약 - 주제별 정리

### API 개발 워크플로우가 분산되고 비효율적이었던 원인 설명

- 기존에는 API 엔드포인트, 스펙, 컬렉션, 테스트 파일이 서로 분리된 위치에서 관리됨
- 예를 들어 API 엔드포인트는 IDE에서 직접 작성하거나 코딩 에이전트로 구현해야 함
- 컬렉션과 테스트는 별도의 외부 도구(예: Postman 웹앱 등)에서 따로 구성해야 했음
- CI 파이프라인에서의 테스트 실행 설정 역시 별도로 진행되어야 했음
- 이로 인해 스펙, 컬렉션, 코드가 하나로 연결되지 않고 서로 다른 위치에 분산 관리됨
- 각 요소들의 버전 관리와 동기화가 어렵고, 전체 API 라이프사이클이 혼란스러웠음

### Postman의 대규모 업데이트가 개발자 경험을 근본적으로 바꾼 방식

- Postman이 최근 대대적인 기능 업데이트를 발표함
- "AI 네이티브, Git 네이티브 API 플랫폼"으로 거듭났다고 강조
- 10년 넘게 Postman을 사용해 온 영상 제작자 본인이 직접 새로운 변화를 체험했다고 밝힘
- YAML 기반의 단순한 형식으로 모든 Postman 리소스를 코드베이스 내에 저장 가능하게 됨
- 로컬 환경과 Postman 데스크탑 앱, 클라우드 간이 완벽하게 연동됨

### 모든 Postman 리소스가 코드베이스 내 YAML 파일로 통합 관리되는 방식 설명

- 컬렉션, API 스펙, 목(mock), 테스트 등 모든 리소스가 단일 코드베이스 폴더에 로컬 파일로 저장됨
- YAML 포맷으로 관리되어 유지보수 및 협업이 매우 용이해짐
- Postman 데스크탑 앱에서 이 리소스 파일을 직접 불러와 편집 및 관리 가능
- 로컬 머신에서 컬렉션, 엔드포인트 등 모든 API 리소스를 구성하고 실행할 수 있음

### 클라우드와 로컬 코드베이스 간 양방향 실시간 싱크 지원

- Postman 클라우드와 로컬 환경 간에 양방향 동기화(2-way sync)가 제공됨
- 사용자는 로컬에서 파일을 수정하고, Postman 앱/클라우드 양쪽 어디에서도 똑같이 접근 가능함
- 테스트 파일 등 리소스에 변경을 가하면 곧바로 코드 에디터에서 해당 변경사항을 git diff로 추적할 수 있음
- git을 통한 버전 관리와 히스토리 추적이 모든 스펙, 테스트 파일에 동일하게 적용됨

### 버전 컨트롤 시스템(Git)과 Postman의 완전 통합 실현

- 모든 Postman 리소스가 이제 파일 시스템 위에서 코드와 똑같은 git 버전관리 체계 하에 존재함
- 스펙, 테스트, 컬렉션 등이 리뷰 가능한 git 커밋 형태로 일관성 있게 관리 가능
- 변경 이력, 리뷰, 협업 플로우의 투명성이 크게 향상됨
- 코드 베이스와 API 정의, 테스트 간 수정 추적을 동일하게 처리할 수 있음

### Postman 에이전트 모드로 API 라이프사이클 전체 자동화

- 에이전트 모드에서는 "자연어로 요구사항을 설명"할 수 있음
- 예시: Flask 앱을 지정하면 Postman이 코드베이스를 분석하여 routes를 자동으로 탐지
- 이를 바탕으로 API 스펙, 컬렉션, 테스트, 목 등 모든 요소를 자동으로 생성 및 셋업
- 모든 작업 결과가 리뷰 가능한 git 커밋으로 저장되어 팀 협업이 용이함
- AI 에이전트를 활용한 API 관리 및 테스트 자동화가 실현됨

### AI 모델 지원과 주요 개발툴 연동 기능 강조

- AI 모델은 Claude, GPT, 자체 호스팅 모델 등 중에서 선택적 활용 가능
- MCP 서버 역시 지원되어 유연한 모델 배포/운용 가능
- GitHub, Sentry, Linear 등 주요 생산성/개발 툴과의 연동을 통해 문맥(Context) 주입이 가능함
- 기존에 사용하던 툴들과의 자연스러운 연계가 원활하게 이루어짐

### CI 파이프라인과 로컬 테스트의 완벽한 일치 보장

- 머신(로컬)에서 실행되는 컬렉션과 테스트와 CI(Continuous Integration) 환경에서의 실행이 완전히 동일함
- Pre-commit hooks, GitHub Actions 등 다양한 환경에서 별도의 추가 작업 없이 동일 테스트 실행 지원
- 환경에 따라 테스트를 다시 작성하거나 별도로 구성할 필요 없이, 일관성 있는 자동화 실행 달성

### 새로운 Postman 플랫폼이 개발자 생산성과 협업 수준을 혁신적으로 높인 효과

- API 리소스의 중앙 집중화, 자동화, 버전 관리를 통해 기존의 혼란스러웠던 워크플로우를 일원화
- 영어 프롬프트 기반 생성, 파일 기반 관리, git 통합, 클라우드 연동 등 핵심 변화로 협업 효율과 품질 향상 실현
- 코드, API, 테스트 파일, 문서화 등 전체 API 라이프사이클의 변화가 단일 플랫폼에서 이루어짐

### 영상 제작자의 추천과 참고자료 제공 안내

- 영상 제작자는 이번 업데이트를 "Postman의 새로운 차원"이라 객관적으로 호평함
- 직접 Postman과 GitHub 리포지토리 예시를 설명란에 제공한다고 언급
- 적극적으로 새 업데이트를 활용해볼 것을 권장하며, 신규 기능에 만족감을 표현함
