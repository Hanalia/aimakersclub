## *MCP 서버의 큰 문제와 그 해결책* 핵심 요약

- MCP 프로토콜은 인기를 끌었지만, 명백한 주요 결함(토큰 소비와 컨텍스트 손실)이 드러나고 있음
- MCP 서버는 기능을 AI 에이전트에 설명하기 위해 각각 수천 개의 토큰을 필요로 하여 비효율적임
- 여러 MCP 서버(예: 5개)를 한 세션에서 동시에 사용할 경우 토큰 소모가 기하급수적으로 증가하여 현실적으로 사용이 어렵다
- "토큰 소비"와 "컨텍스트 손실(context rot)"은 현재 AI 분야에서 매우 중요한 토픽임을 강조
- 영상에서는 이러한 문제의 심각성을 실례를 들어 강조함: AI 코딩 작업에 5개의 표준 MCP 서버만 사용해도 상황이 과도하게 악화될 수 있음
- 이러한 한계를 해결하기 위해 "에이전트 스킬(Agent Skills)"이 최근에 공개됨(Anthropic사 출시)
- 에이전트 스킬은 AI와 에이전트가 API 엔드포인트와 상호작용할 수 있는 스크립트 생성, 코드 실행, 사용법 지침 생성 등을 지원한다는 점에서 강력함
- 결론적으로 MCP의 문제는 에이전트 스킬로 상당 부분 해결될 수 있으며, 이는 코드 실행 중심 AI 개발에 큰 영향을 미침
- 마지막으로 "이것이 우리가 알던 MCP의 끝인가?"라는 질문으로 영상을 마무리하며 변화의 도래를 시사함

## 세부 요약 - 주제별 정리

### MCP 프로토콜의 인기와 그로 인한 심각한 결함이 부각되고 있음

- MCP(Multimodal Communication Protocol)는 최근 점점 더 많은 주목을 받으며 AI 분야에서 인기를 얻고 있음
- 그러나 MCP 프로토콜이 널리 사용됨에 따라 심각한 결함이 명확히 드러나기 시작함
- 영상 제작자는 "토큰 소비(token consumption)"와 "컨텍스트 손실(context rot)"이 MCP의 핵심 문제임을 강조함
- 특히 이 두 문제는 최근 AI 기술계에서 매우 뜨거운 이슈로 재론되고 있음
- 제작자는 이 문제점을 구체적인 예를 통해 설명할 예정임을 밝힘

### 토큰 소비량과 컨텍스트 손실이 MCP 실사용의 최대 장애물이 됨

- MCP 서버는 사용 시, 각 서버의 기능을 AI 에이전트에게 설명하기 위해 수천 단위의 토큰을 소모함
- MCP 서버 각각이 이러한 토큰 소비를 요구하기 때문에, 여러 서버를 동시에 쓸 경우 문제가 심각해짐
- 컨텍스트 손실(context rot) 문제란, 많은 정보(토큰)를 전달해야 할 때 상대적으로 중요한 정보가 대화 맥락에서 밀리거나 소실되는 현상을 의미함
- 이러한 상황은 AI-에이전트 기반 코딩에서는 더 큰 비효율을 야기함
- MCP 서버 기반 구조에서는 각 서버가 독립적으로 많은 컨텍스트와 토큰을 소모함

### MCP 서버 여러 개를 동시에 사용할 경우 토큰 소모가 기하급수적으로 증가함

- 영상 속 예시: "5개의 매우 표준적인 MCP 서버"를 AI 코딩 보조 세션에 사용한다고 가정
- 영상 제작자는 이 5개 MCP 서버를 이전에도 자주 사용했다고 언급함
- 그러나 "하나의 세션에서 5개 MCP 서버를 동시에 사용하는 일은 절대 없었다"고 단언함
- 그 이유는 각 MCP 서버가 자신의 기능을 설명하는 데만 수천 개의 토큰을 필요로 하기 때문임
- 예를 들어, 각 서버가 2,000 토큰씩 필요하다면 5개만 해도 10,000 토큰이 기본적으로 소모됨
- 이는 코딩 보조 AI의 세션 컨텍스트 한도를 급격히 소진시키는 수치임

### 토큰 소비와 컨텍스트 관리 한계는 현실적 활용에 큰 제약이 됨

- MCP 프로토콜이 아무리 유용하다 해도, 실제 동시 활용에서는 구조적으로 토큰 한도를 초과하기 쉬움
- 토큰 초과와 컨텍스트 손실은 AI 모델이 최신 정보, 명령, 범위, 기능을 제대로 인식하지 못하게 만듦
- 이로 인해 실시간 코딩 보조, 멀티서버 연동 AI 시스템, 복합 작업 처리 등에서 비효율 및 오류가 증폭됨
- 영상에서는 이러한 토큰 비용 문제와 현실적인 제약을 "정말 짜증나는(painful)" 문제로 규정함

### Anthropic의 '에이전트 스킬'이 이 문제에 대한 강력한 해결책으로 등장함

- 최근 Anthropic(앤트로픽)사에서 "에이전트 스킬(Agent Skills)"라는 새로운 기능을 출시함
- 에이전트 스킬은 MCP의 구조적 한계를 극복할 수 있는 강력한 솔루션으로 평가됨
- 이 기능은 AI 및 AI 에이전트가 다음과 같은 작업을 할 수 있도록 지원함:
  - API 엔드포인트와의 자동 상호작용
  - AI가 자체적으로 스크립트를 생성해 업무를 수행
  - 코드 실행(code execution) 기능을 내장
  - 사용법 지침, 스크립트 활용법 등도 AI가 자동으로 생성
- 코드 실행 및 스킬 스크립트 생성 기능을 통해, AI가 동적으로 기능을 확장할 수 있음

### '에이전트 스킬'을 채용하면 토큰 소비 구조를 효율적으로 바꿀 수 있음

- AI 에이전트가 개별 MCP 서버별로 모든 기능설명을 반복해 전달하는 비효율이 사라짐
- 필요시에만 동적으로 API 호출, 코드 실행, 작업 처리 지시를 스킬 스크립트 형태로 생성 및 수행
- 이에 따라 전체 토큰 소비가 대폭 절감되고 컨텍스트 공간을 효율적으로 사용할 수 있음
- 에이전트가 실시간으로 스킬을 생성·수정·확장하기 때문에, AI 기반 협업 환경에서 유연성이 극대화됨

### 제시된 해결책은 코딩 AI 환경에서 매우 강력하고 실질적인 효과를 갖춤

- 에이전트 스킬을 통해 AI가 코딩 도우미 역할을 할 때, 기존 MCP 기반에 비해 훨씬 원활한 협업이 가능해짐
- 실제 API 활용, 스크립트 코드 작성·수행 지능화 등 고도화된 작업도 부담 없이 진행 가능해짐
- 영상 제작자는 "이 기술, 정말 강력하다"라고 직접 강조함(“I really think this is powerful stuff”)
- MCP의 근본적 한계를 보완하는 혁신적 접근법으로 소개

### "이 변화가 우리가 알던 MCP의 끝인가?"라는 의문을 제시하며 영상이 마무리됨

- 영상 마지막에서 "이것이 우리가 알던 MCP의 끝인가?"(Is this the end of MCP as we know it?)라는 질문을 던짐
- 이는 기존 MCP의 시대가 저물고 새로운 '에이전트 스킬' 기반의 패러다임이 도래할 수 있음을 시사함
- 단순한 기능설명 토큰구조가 아닌, AI의 자율적 코드 생성-실행 구조로 진화할 가능성을 예고
- 영상의 결론 및 전체 구조가 이 질문에 대한 탐구로 귀결됨
