## *MCP부터 자동화 스케일까지: 스스로 구축되는 파이프라인* 핵심 요약

- **Bright Data의 MCP(Master Control Program)는 CAPTCHA, 봇 탐지 시스템 등 강력한 방어를 가진 웹사이트에서도 LLM이 접근·데이터 수집을 할 수 있도록 지원함**
- **전통적으로 스크래퍼를 직접 구축·유지해야 했지만, MCP와 Bright Data의 ‘스킬셋’을 활용하면 LLM이 스스로 파이프라인(스크레이퍼) 생성 및 관리가 가능함**
- **웹 데이터 수집 시 HTML 전체를 LLM이 파싱하지 않고, 스크래퍼로 필요한 정보만 추출하여 토큰 사용량을 대폭 절감(3페이지 크롤링에 약 1백만 토큰 절약, 최대 62% 토큰 절감 효과)**
- **Bright Data MCP는 5,000회 무료 요청, 150만개 이상의 IP 인프라, 66여개 웹 자동화 툴을 제공하며, 원격 브라우저, 캡챠 자동 해결, 헤더/쿠키 자동 세팅 등 다양한 기능 제공**
- **실습 데모에서는 Walmart, Very와 같은 봇 차단이 강한 사이트도 LLM 기반 파이프라인으로 손쉽게 데이터 크롤링 및 자동화 가능함을 시연**
- **API 웹 언락커, scrape as markdown, JSON 변환 등 다양한 데이터 출력 옵션으로 토큰 효율화, 후처리 및 활용성 극대화**
- **파이프라인은 자동 self-heal(자체 복구) 기능으로 웹사이트 변경·오류 시 자동 유지보수, 30분 주기로 데이터 검증·업데이트 등 스케줄링 가능**
- **스크래핑 외에도 폼 자동 입력, 버튼 클릭 등 유저 에이전트 시뮬레이션도 가능(로그인 등 비공개 데이터 제외), 실제 사람과 유사한 브라우저 조작으로 탐지 우회**
- **법적 보호 하에 오픈(public) 데이터만 수집·활용하며, Bright Data는 해당 이슈로 Meta, Elon Musk(트위터) 등과의 소송에서 ‘공공 데이터의 수집 자유’ 판결 확보**
- **실제 개인 활용 예시(아파트 매물 알림 등)와 함께, 엔터프라이즈 대규모 데이터 수집부터 개인적 자동화까지 방대한 웹 데이터에 손쉽게 접근하는 방법을 안내함**

## 세부 요약 - 주제별 정리

### LLM 기반의 대규모 웹 데이터 수집은 스크래핑 파이프라인 자동화로 효율이 극대화됨

- 레딧 등에서 "수만 건의 데이터를 LLM으로 파싱하면 토큰 소모가 크다"는 고민이 많음
- HTML 전체를 LLM에 입력하면 수십만~수백만 토큰 소요
- Bright Data의 MCP·스킬셋을 활용하면 LLM이 스스로 스크래퍼를 구축하여 필요한 부분만 추출, 토큰 대폭 절감
- 예시: 3페이지 크롤링에 약 100만 토큰 절약
- 자주 사용하는 검증 루프(30분마다 데이터 점검, 빠진 데이터 자동 복구)로 추가 관리 비용 없이 대규모 수집 가능
- 실시간 및 정기 자동화가 쉬워짐(새 매물, 신제품, 이벤트 감지 등)

### MCP와 Bright Data ‘스킬셋’은 스크레이퍼 생성, 실행, 유지보수까지 전 과정의 자동화를 지원함

- MCP: Master Control Program, Bright Data가 제공하는 웹 자동화·스크래핑 플랫폼
- 웹사이트의 HTML 추출, 셀렉터 정보 인식 및 과제별 맞춤 스크래퍼 자동 생성
- Bright Data GitHub의 ‘스킬셋’에서 스크래퍼 구축 베스트 프랙티스, 코드, 도구 모두 자동 제공
- 클라우드 코드, CloudCrawl 등 클라우드 기반 자동화 엔진과 연동해 사용
- LLM이 신규 스크래퍼를 3분 이내 작성하며, 유지보수 부담 없이 데이터 수집 및 파이프라인 자체 복구 가능

### 복잡하거나 강력히 차단된 사이트도 MCP·LLM 조합으로 손쉽게 크롤링 및 자동화됨

- Walmart, Very 등 강한 웹 보안(CAPTCHA, 봇 탐지) 사이트에서 실습 시연 진행
- MCP의 웹 언락킹, 원격 브라우저, 캡챠 자동 해결 등으로 대부분의 공개 데이터 접근 가능
- 150만개 이상의 Bright Data 프록시 IP 인프라로 차단 우회
- 실제 사례: 영국 유명 쇼핑몰 Very.com에서 헤드폰 검색 결과 자동 스크래핑 및 데이터 구조화 완료

### 스크래핑 토큰 절감 효과는 매우 큼; 아키텍처-API 활용법도 체계적으로 제공됨

- scrape as markdown 기능: HTML 태그 제거 후 텍스트만 LLM에 입력하여 토큰 소모 최소화
- 500여 종의 사전 구축 API(Amazon 등)로 마크다운 대신 JSON 형태 데이터도 바로 받아올 수 있음
- 실습에서 90개 상품 기준, input/output 토큰 합산 시 약 62% 토큰 절감 효과(웹사이트 구조에 따라 변동)
- JSON이 마크다운 등 비정형 데이터보다 LLM 파싱 효율/속도가 월등히 높음

### 스크레이퍼 유지보수 및 오류 대응이 자동화되어, 사람 개입 없이 수명 주기 전과정을 관리함

- 웹사이트 구조 변경, 셀렉터 변경, 데이터 누락 등 문제 발생 시 자체적으로 수정 루프 수행
- 예) 검증 오류 감지 시 5분 내 자동 수리, 수집 파이프라인 장애 복원
- 파이프라인 주기 실행(예: 30분마다 브랜드 신제품 체크 등)이 GUI/스크립트로 매우 손쉽게 가능
- 직접 코드를 실행하면 변수/키 지정 등만으로 반복 실행 및 관리 가능

### 원격 브라우저·캡챠 자동화 등 자체 AI 솔루션으로 현실적 크롤링 한계 극복

- 원격 브라우저: Bright Data 시스템 상에 수천개의 브라우저를 동시에 띄워 동작 가능
- 브라우저 자동 제어로 클릭, 폼 입력, 스크롤 등 실제 유저 행동을 LLM이 수행
- AI 기반 캡챠 솔버는 움직임, 클릭 타이밍, 마우스·키보드 프리셋 등으로 탐지 회피
- 실제 사람 행동의 케이스를 미리 녹화해 유저 행태 기반 크롤링 구현

### 개인정보/비공개 영역은 크롤링하지 않으며, 법적 이슈는 ‘공공 데이터의 자유’ 판례로 뒷받침됨

- 로그인 후 영역, 약관 동의(로봇 금지) 요구 사이트는 Bright Data에서 취급하지 않음
- Meta, Elon Musk(트위터)와의 소송에서 Bright Data는 ‘공공 데이터 활용은 합법’ 판결 확보
- 판례: 공개 웹페이지 데이터 수집 자체는 법적 제한이 없음(단, 약관 위반 여부는 각 서비스별 착오 방지 명시)
- 크롤링 범위: 오픈 데이터(로그인 전, API·페이지 공개 데이터)로 제한

### 활용 사례는 기업 대량 크롤링부터 개인 맞춤 자동화까지 매우 폭넓음

- 실제 발표자의 사례: 이사 희망시 아파트 매물, 가격 조건 등 자동 청취 후 알림받아 입주 성공
- 레스토랑 예약 등 자주 변동되는 ‘희소성 예약’도 자동 감지 및 직접 예약/폼 입력 가능
- 시장조사, 베스트 리뷰 제품 찾기 등 개인 연구·의사결정도 자동화에 적용 가능

### 다양한 인터페이스 및 기술 환경 지원으로 접근성과 확장성이 뛰어남

- Cloud Code, CloudCrawl 등 주요 클라우드 기반 환경과 빠르게 연동
- API/스크립트/GUI 등 사용 목적에 따라 간편한 통합 가능
- 입력값(키워드, 최대 페이지 등) 지정으로 유저 관점에 맞는 맞춤 크롤러 구축 가능

### LLM·크롤러 연계를 통한 후처리 질의 및 자연어 기반 데이터 탐색이 완전히 자동화됨

- 스크래퍼 구축 후 LLM이 추출된 데이터를 토큰 효율적으로 분석·질의 처리
- 대규모 데이터 수집 후 LLM에 “가장 평점 좋은 상품 추천” 등 다양한 자연어 분석 요청 가능
- 전체 파이프라인(수집→분석→질의)이 LLM 하나로 자동화되어 반복성 업무 경감

### 최적 모델·환경 선택을 통한 과금 및 성능 최적화도 안내됨

- 브라우저 기반 탐색에는 고가·고성능 LLM 모델이 불필요(Claude Haiku 등 경량 모델로 충분)
- 전체 파이프라인에서 토큰 소모량 극소화(수천~수만 토큰에서 10분의 1 이하로 절약)
- 운영 비용이 기존 대비 ‘페니’ 수준까지 낮아짐(수만 건 크롤링 시 비용 부담 현저히 감소)
