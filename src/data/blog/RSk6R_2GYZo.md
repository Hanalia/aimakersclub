## *아콘(Archon)으로 다시 10배 빨라진 최신 AI 코딩 워크플로우 LIVE 데모* 핵심 요약

- Cole Medin이 자신의 최신 AI 중심 코딩 환경과, 특히 Archon이라는 오픈소스 하네스 빌더를 활용해 실제 대규모 소프트웨어 엔지니어링을 어떻게 10배(실감은 100배) 빠르게 수행하는지 실시간으로 공개함
- 기존의 일반 코딩이 '1배' 속도였다면, 클라우드 기반 AI 어시스턴트 도입으로 10배, Archon 도입 후에는 병렬 실행과 워크플로우 자동화로 체감상 100배까지 작업 효율이 향상
- Archon을 사용한 브라운필드(기존 코드베이스 확장/버그픽스)와 그린필드(신규 기능·작업 시스템 구축) 개발 모두를 실제 코드베이스를 통해 시연
- Archon은 AI 코딩 과정을 '워크플로우'로 규격화하여 신뢰도와 반복성을 보장하며, 작업 전체의 입력(이슈·계획)과 출력(풀 리퀘스트 생성, 코드 검증 등)을 명확히 설계 가능
- Archon의 워크플로우는 YAML 파일 하나와 명령어·스킬(MD, script 파일 등)로 구성되며, 개별 코드베이스별 맞춤 워크플로우를 직접 쉽게 만들고 확장할 수 있음
- 브라운필드 개발 실습에서는 GitHub 이슈를 AI 코딩 에이전트에게 투입, 여러 이슈(버그, 신규 기능 등)를 병렬로 한 번에 처리하여 각 작업이 격리 가능함을 실증
- 실작용 사례로, ‘워크플로우 마켓플레이스’ 신규 모듈을 Archon의 PIV 반복 루프(계획-구현-검증 게이트) 방식으로 발상~계획~구현~수정까지 시연
- Archon의 워크플로우는 각각의 작업 단계를 에이전트/스크립트/휴먼 인더루프로 나누어 신뢰성을 높이며, 실제로 실패도 복구·재시도 가능(워크플로우 내구성, DB에 상태 저장)
- 작업 구조와 로그 시스템을 통해, 사용자는 모든 실행 과정을 실시간 추적·관리·개입 가능(MLOps·엔터프라이즈 수준)
- 대규모 코드베이스 운영과 개인 ‘세컨드 브레인’, 클라우드 코드·Codeex·Pie 등 다양한 AI 코딩 에이전트, 라우팅, 커스텀 스킬 연동법까지 세밀하게 해설

## 세부 요약 - 주제별 정리

### Archon은 AI 코딩 워크플로우의 반복성과 신뢰성을 극적으로 높여 병렬 대량작업에 최적화됨

- 개발자는 Archon을 이용해 자신만의 AI 코딩 워크플로우를 YAML로 정의하고, 각 작업(이슈 처리, PR 리뷰 등)을 병렬·격리 환경에서 자동·반복적으로 수행 가능함
- Archon 도입 전에는 클라우드 기반 AI 도구(Copilot 등)로 10배 향상, Archon 추가 후에는 10~100배 생산성 증가 체감(주장)
- 병렬 실행: 한 번에 최대 20개 이상 이슈를 고립된 워크트리에서, 각각 독립된 에이전트 세션과 컨텍스트로 동시 처리(코딩 에이전트 간 간섭 방지)
- 워크플로우는 반복·검증·분기·조건·휴먼 개입 지점 등 엔터프라이즈 SW 라이프사이클 전체를 세밀하게 통제할 수 있음
- 입력(이슈, 티켓, 계획) → 처리(연구, 구현, 검증) → 출력(PR, 문서, 배포 등)이 명확히 아티팩트로 귀결됨
- 장시간 실행되는 워크플로우도 '파이어 앤 포켓' 방식: 사용자는 여러 작업을 걸어두고 동시에 다른 일 수행 가능
- 오픈소스 하네스(집) 빌더로서 남의 워크플로우 복제, 프리셋 활용, 코드베이스 맞춤 설계 모두 지원

---

### Archon 워크플로우는 엄격한 단계화, 각 단계별 AI·스킬·검증·인간 개입 분리가 강점임

- 각 워크플로우는 "노드"(단계)로 이루어짐: 예컨대 이슈 컨텍스트 수집 → 조사 → 구현(에이전트) → 검증(별도 에이전트) → PR 생성 → 리뷰 루프 등
- 구현/플랜/검증 등 각 단계별로 다른 LLM, 프롬프트, 스킬, 스크립트(파이썬, 타입스크립트, Bash 등) 지정 가능
- '휴먼 인 더 루프'를 여러 단계(플랜 리뷰, 구현 검수 등)에 직접 삽입 가능하여, AI 에이전트의 실수·맥락 오해 방지
- 계획-검증 루프가 워크플로우 하에 동작: 계획 문서 작성 → 사용자가 확인·수정 반복 → 최종플랜에만 실제 구현 시작
- 검증 단계는 별도 세션에서 수행: 에이전트가 직접 자기 코드를 검증하는 편향 방지, 완전히 신규 컨텍스트에서 리뷰 강제
- 자동·조건부 분기, 반복 루프, 병렬 블럭, 강제 검증 등 복합 제어로 신뢰도와 안정성 대폭 강화

---

### 자신의 '세컨드 브레인'과 Archon을 연동해 코드베이스별 자동화된 AI 개발 환경을 구성함

- Obsidian 등 메모 시스템에 ‘repositories.mmd’ 형태로 자신이 다루는 각 코드베이스의 인덱스를 만들고, Archon 연동 설정을 기록
- 각 프로젝트별 Archon 워크플로우, 작업 선호도, 실행 이력(Dispatch History) 등을 누적 저장해 추천도 자동화
- 세컨드 브레인의 기본 대화에서 "이 이슈 1607, 1610, 1612 처리해줘"만 해도, 적절 워크플로우와 컨텍스트 로딩 자동 수행
- 실시간 로그와 상태를 세컨드 브레인(Cloude Code 등)에서 질의·조작 가능: 명령 센터화
- 워크플로우 사용 히스토리를 누적해, 향후 자동으로 코드베이스/작업 유형에 따라 적합한 처리를 추천·자동화

---

### Archon 워크플로우 기본에서는 GitHub 이슈 등 외부 티켓을 중심으로 브라운필드 개발 전체를 자동화함

- 각 GitHub Issue는 충분한 배경/솔루션 제안/유저 플로우 등 맥락을 담도록 사전에 준비
- Archon 워크플로우에서 이슈 콘텍스트를 기반으로 리서치/플랜/구현/PR/검증 절차를 진행
- '실제 코딩은 vibe coding이 아님'—즉, 충분한 맥락/계획이 준비된 상태만 워크플로우에 투입, 신뢰도 보장
- 워크플로우 실행 후 Pull Request가 생성, PR 검수 및 merge까지 일련의 작업이 자동화됨
- 예시로 "Archon setup wizard에 new AI provider 옵션 추가" 등 이슈를 정확히 기술—AI가 맥락 상실 없이 구현 가능
- 각 브라운필드 작업의 아티팩트(산출물): 대부분 경우는 PR, 경우에 따라 전체 신규 Git repo

---

### 그린필드 개발(신규 모듈)도 Archon PIV 루프 워크플로우로 착상~계획~구현~검증~피드백까지 단계적으로 자동화함

- 실시간 데모로, ‘Archon 워크플로우 마켓플레이스’ 신규 기능을 2회 PIV 루프 구조로 직접 구축
- PIV 루프는 Plan(기획), Implement(구현), Validate(검증)의 반복 구조로, 매 단계별 인간-에이전트 협업
- 착수 전에는 전체 맥락(목표, 기능, 범위, 기술 등)을 세컨드 브레인에서 정리; 존재하지 않는 ‘로드맵’ 등 시스템이 자동으로 맥락화
- 각 단계마다 AI가 명확한 질문 리스트를 생성—사용자(휴먼) 피드백을 수렴, 플랜→피드백 반복 후 확정
- 구현 단계(Implementation)는 계획 MD만을 컨텍스트로 삼아 AI가 모든 코드, 문서, 설정 파일 등 생셩
- 생성된 기능은 바로 Doc 사이트 내 로컬 인스턴스를 실행하여 검수, 실제 브라우저에서 동작 확인
- 만일 설계에 허점(예: 커스텀 커맨드·스킬이 여러 파일인 경우 파일 설치 문제 등)이 발견되면, ‘Resume’ 피드백으로 워크플로우가 해당 사항을 자동으로 보완·재실행

---

### Archon은 각자 자신의 프로세스와 업무 도구에 맞춘 별도 워크플로우를 쉽게 만들고 관리할 수 있게 설계됨

- 워크플로우(Workflow)는 기본적으로 단일 YAML 파일로 구성; 필요한 Skill, Command 등은 별도 MD파일로 동봉
- 기본 제공 워크플로우(예: fix-github-issue, PR 리뷰, piv 루프, R 루프 등)를 참고해, 각자의 코드베이스 맞춤 프로세스 반영 가능(Linear, Jira, monday.com 등 다양한 이슈 트래커와 연동)
- 필요하다면, 코딩 외 조직/업무/컨텐츠 제작(영상, PDF 등)에도 활용 가능—non-coding 워크플로우는 'no-worktree' 플래그로 실행
- 각 노드마다 AI 모델(Claude, Codeex, Pie 등), 프롬프트, 컨텍스트, 인입 스킬, CLI커맨드 등 세분화 지정
- 구조나 프롬프트를 컴팩트하게 관리할 수 있도록, 구조-YAML/프롬프트-MD/스킬-MD 등 파일 분리 지원
- 워크플로우 빌더 내장—별도 생성 워크플로우(Archon Workflow Builder) 혹은 AI와의 대화로 워크플로우 생성
- 러닝 커브 보완을 위해 공식/커뮤니티 영상, 다이나미스 커뮤니티, AI 튜터(chat.dynamus.ai) 등 다수 가이드 제공

---

### Archon은 워크플로우의 내구성·복구력 등 안정성 및 MLOps 수준 관리 체계를 내장함

- 워크플로우 실행 중 실패(노드 충돌 등) 발생 시, 내부 DB(SQLite or Postgres)에 상태 저장—재시도·복구 가능
- 노드(작업단위) 실패 시 재시도, 전체 워크플로우 중단시에도 이어서 복구 가능
- '파이어 앤 포켓' 실행: 사용자는 실행 후 상태/로그 질의, 추가 실행, 취소, 중단, 재개 모두 실시간 제어 가능
- 세분화된 로깅 시스템, 상태 추적, 통합 히스토리, 토큰 효율 모니터링 등 엔터프라이즈 도구 수준의 관리 지원
- 로컬 환경의 리소스 소모는 대체로 매우 경량(수십여 워크플로우 동시 실행 시 1GB 미만, 단, 브라우저 오토메이션 등은 예외)

---

### 커스텀 스킬, 클라우드 코드·Pie·Codeex 등 다양한 모델·에이전트를 병렬·믹스 활용 가능함

- 각 워크플로우, 노드별로 AI 모델 지정(Claude, Codeex, Pie, Kimmy, Minimax 등): Implement엔 Claude, Review엔 Codeex 등 조합 가능
- 아키텍처상 ‘AI에 모든 결정권을 주는 것’ 대신, 에이전트 행위 상위에 워크플로우(하네스)가 통제(규칙, 검증, 반복, 경로선택 등)
- 각 노드는 주입 스킬 지정, 필요한 시점/단계에만 스킬(예: LLM 웹브라우저 등) 컨텍스트에 포함
- 실례: 브라우저 자동화 스킬(agent_browser)·조건부만 적용—설치, 플랜, 검증 단계별로 토큰/스킬 최소화
- 다양한 패키지 마켓플레이스(예: npm, PyPI, skills.sh)에서 얻을 수 있는 방식처럼, 워크플로우·스킬 번들 구상 및 마켓플레이스 구축 사례 시연

---

### Archon 워크플로우 마켓플레이스는 분산형 워크플로우 공유·설치를 쉽게 만들어 커뮤니티 활성화 및 모듈화 확장에 기여함

- 유저 각자가 자신만의 Archon 워크플로우(YAML+명령/스킬 등)를 자신의 깃헙 공개repo에 두고, Archon doc 사이트의 중앙 registry(json)에 PR로 메타데이터만 등록
- 1줄 명령어로 설치 및 시작 가능—npm 패키지, skills.sh/skills registry와 유사한 사용성
- 설치 커맨드는 실제 repo 디렉토리 전체를 받아 커스텀 커맨드/스킬까지 통째로 설치하게 개선
- 초기 버전은 인증/댓글·평가 기능 없이 동작, 향후 버전엔 서드파티 검증, 평가, 코멘트, 랭킹 등 커뮤니티 피드백 기능 추가 예정
- 마켓플레이스 구축 과정도 Archon piv 루프 워크플로우로 전체 시연—문제 발견(참조파일 누락 등)시 중간 반복·보완

### 실시간 데모에서 브라운필드/그린필드 실제 코드 변경 사항과 결함 수정·PR merge까지 전체 자동화 사례를 자가검증함

- 실제 시연: 특정 GitHub 이슈 3개를 병렬 Archon 워크플로우로 처리하며, 각각 독립 워크트리와 브랜치 관리
- 각 이슈 처리 워크플로우는: 컨텍스트 수집→이슈 조사→조건별(버그vs기능) 분기→구현→자기검증→PR생성→PR 리뷰→PR 코멘트→최종 커밋·MR 구조
- PR이 생성되면, 즉시 CI 테스트·PR 리뷰·머지까지 모두 자동화
- 실사용 환경에서 발생한 문제(모델/워크플로우 설정 오류 등)는 워크플로우 durability에 의해 자동 수정·재진행
- 워크플로우 출력물(계획서, 수정코드, PR 등)은 즉시 사용자가 검증/피드백/재요청 가능
