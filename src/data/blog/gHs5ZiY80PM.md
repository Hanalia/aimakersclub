## **"50단계의 디퓨전 스텝이 항상 필요한 것은 아니다 — Ziv Ilan, Nvidia"** 핵심 요약

- Nvidia AI랩의 Ziv Ilan이 디퓨전 기반 이미지 및 비디오 생성 모델의 실용적 사용성 향상을 위한 최적화 기술 세 가지(양자화, 캐싱, 증류)에 대해 발표함
- 전통적으로 20~50번의 디노이징 스텝이 요구되는 디퓨전 모델에서 실시간 생성과 지연 감소가 주요 도전 과제로 부상함
- 양자화(quantization)는 메모리 사용량 감소 및 저사양 GPU에서도 실행 가능한 장점을 제공하며, Black Forest Labs와의 Flux 2 실험에서는 동적 양자화 방식을 통해 실제 효과를 입증
- 캐싱 기술은 각 스텝마다 무의미하게 반복되는 계산을 줄임으로써 딜레이를 단축하며, T-cache 등 기법이 TRT LLM Visual Gen 등 오픈소스에 적용되고 있음
- 증류(distillation)는 주어진 스텝(20~50)을 대폭 줄이면서도 이미지/비디오 품질을 유지하는 핵심 최적화 수단으로 소개됨
- DeepSeek 등의 증류 사례에서 학습 모델 크기가 작아지는 게 아니라 스텝 수 감소에 초점이 맞춰짐
- 증류 방식은 궤적 기반(trajectory-based)과 분포 기반(distribution-based)으로 구분되며, 현재는 분포 기반이 더 일반적이고 품질이 높음
- FastGen 등 최근 오픈소스 라이브러리가 초대형 모델(20~40B 파라미터 이상)까지 지원하며 실제 실시간 영상 생성에 성공 (Blackwell B200 GPU로 실시간 달성)
- 각 최적화 방법은 독립적이지만 누적적으로 적용 가능하여, 원하는 지연·품질·자원 상황에 맞춰 조합할 수 있음
- 증류 작업에는 고가의 GB200 GPU가 꼭 필요한 것은 아니고, H100/H200 등 보급형 클러스터도 사용 가능함
- 실제 사례와 최신 연구(예: How Lab의 Attention FP4, FastGen의 하이브리드 증류법 등)도 소개됨

## 세부 요약 - 주제별 정리

### 디퓨전 모델이 이미지·비디오 생성 분야에서 실용화를 향해 진전하고 있음

- Nvidia AI랩의 Ziv Ilan은 파리 기반으로 영상 생성, 프론티어 모델 빌더 등 다양한 도메인과 협업 중임을 소개하며 발표를 시작함
- 최근 1년 사이 Flux 2, LTX 2.1, Google의 Nano Banana 등 높은 품질의 이미지·비디오 생성 모델이 등장
- 실시간 생성 능력이 로보틱스, 게임, 콘텐츠 제작 등 폭넓은 산업적 활용 가능성을 열어줌
- 하지만 실질적 B2B·B2D(엔터프라이즈, 개발자) 환경에서는 성능·지연(laency)·확장성·성숙도 부족이 가장 큰 장애물로 지적됨
- 기존 LLM(VLM) 생태계에서 검증된 최적화 기법을 디퓨전 세계로 "증류"해 적용하고 있음

### 디퓨전 모델의 생성 과정과 지연의 근본 원인이 설명됨

- 디퓨전 모델은 LLM의 오토리그레시브 구조와 달리 20~50회의 반복 디노이징을 통해 결과물 생성
- 고품질 이미지(1080p, 720p 등)일수록 첫 번째 이미지 생성까지 시간이 오래 걸림
- 실시간(Real-Time) 이미지·비디오 생성이 '성배(Holy Grail)'로, 이를 위한 최적화가 연구의 중심임

### 실시간 생성 성능 개선을 위한 세 가지 핵심 최적화 전략이 제시됨

- 이번 발표에서는 양자화(Quantization), 캐싱(Caching), 증류(Distillation) 3가지 테크닉이 상세히 다뤄짐
- 이 중 대개 증류 → 양자화 → 캐싱 순서로 적용하지만, 개별 현실에 맞게 조합 가능함
- 각 최적화법은 상호 독립적이면서 누적(incremental)적 적용이 가능하며, 상황별로 단독 또는 복합 활용 가능

### 양자화는 GPU 자원 효율 극대화와 성능 향상에 유리함

- 양자화는 파라미터 정밀도를 줄여 메모리 절감 및 낮은 스펙의 GPU에서도 실행 가능하게 해줌
- LLM에서와 마찬가지로 Post-Training Quantization(PTQ), Quantization-Aware Training이 있음
- 디퓨전 모델은 Attention이 무거워 양자화 효과가 상대적으로 제한적이나, 여전히 '저렴한' 최적화 기법
- Black Forest Labs와의 Flux 2 실험에서 동적(dynamic) 양자화 적용
    - 정적 양자화(Static)는 모든 파라미터 범위를 선산정하지만, 동적 양자화는 실행 시점에서 범위 조정
- Nvidia는 TRT LLM Visual Gen 오픈소스에 양자화 예시 코드 및 프리양자화 체크포인트(Pre-Quantized Checkpoint)를 제공
    - Huggingface 등지에서 손쉽게 다운로드 후 바로 실습 가능
- 양자화 적용 시 1) 메모리 절약, 2) 저가형 GPU에서의 실행, 3) 전체 성능 개선이라는 효과 있음
- How Lab의 Attention FP4, 최근 연구 등 최신 Attention 최적화 기법을 적극적으로 트래킹 및 적용 중임

### 캐싱은 반복적·불필요한 계산을 대폭 줄여 지연을 줄임

- LLM 오토리그레시브 모델에서는 KV-Cache가 토큰 생성의 표준 최적화이나, 디퓨전에서는 적용 방식이 상이함
- 디퓨전의 디노이징 스텝 간 변화가 크지 않은 영역에서는 이전 계산 결과의 재사용(Caching)이 가능함
- 예시: T-Cache 기법은 인접 디노이징 스텝 간 변화가 거의 없을 경우, 재계산을 하지 않고 캐시를 활용
    - 전체 픽셀/잠재 공간 전역 캐싱부터, 특정 Chunk(동작이 적은 화면 등) 단위로만 재계산하는 방식도 있음
    - 예: 강연장 청중은 거의 변화 없으니 해당 영역은 캐싱, 발표자(동적)는 별도 계산
- 임계값(Threshold)·범위 등 직접 세팅할 수 있으며, 부적절한 캐싱은 품질 하락 초래 가능
- TRT LLM Visual Gen 라이브러리 등에서 지원하며, VLLM Omni, GLN diffusion 등 서빙 라이브러리에도 적용됨
- 적절한 캐싱 사용 시 품질 유지 및 성능 대폭 향상이 가능함

### 증류(distillation)는 스텝 수를 극적으로 줄이며 '실시간' 수준 성능을 가능케 함

- 기존 증류(Deepseek 등)는 대형 모델을 작은 모델로 압축하는 데 중점, 디퓨전에서는 '스텝 수 단축'에 초점 이동
- 예: 50스텝 대신 4, 8스텝 또는 단 1스텝(one-shot)만으로도 결과물 품질을 크게 손상시키지 않고 생성 가능
    - 성능상 최대 수십~수백 배 가속 실현 (예: 200x 개선)
- 실제로 최근 GTC(San Jose) 데모에서 두 가지 증류법으로 실시간(near real-time) 영상 생성에 성공
    - Blackwell B200 단일 GPU로 실시간 운영 확인

### 증류 방법은 궤적 기반, 분포 기반 두 가지로 대별되며 하이브리드도 출현

- 증류 과정에는 '교사(teacher) 모델'과 '학생(student) 모델'이 존재
- 궤적 기반: 교사 모델의 각 디노이징 스텝 경로를 따라가도록 학습
- 분포 기반: 최종 결과(분포)가 동일해지도록 최적화, 경로는 자유롭게 하도록 함
- 최근에는 분포 기반이 더 우수하며, FastGen 등 최신 연구에서는 두 방법(하이브리드) 결합도 시도
    - 하이브리드는 품질/안정성 모두 향상
- 증류는 사후(post-training) 기법이므로, 자체 데이터 사용시 별도 파인튜닝 및 적합 데이터 준비 필요

### FastGen 등 오픈소스 도구로 초대형·복잡한 증류 프로세스 관리가 쉬워짐

- FastGen(엔비디아 리서치 주도)은 대형 비디오 디퓨전 모델(최대 20~40B 파라미터 이상)에 대한 증류·샤딩·파인튜닝 등 과정을 구조화/자동화함
- 사용자 데이터로 파인튜닝이 가능하고, 오픈소스·공개 데이터만으로도 상당한 품질 달성함
- FastGen 내부에는 특정 데이터 없이도 실험 가능하며, 도메인 특화 데이터(예: 단백질 생성)에선 별도 서브셋 필요
- GTC 사례: Blackwell B200 1장으로 실시간 생성 달성

### 최적화 방법은 누적적으로 조합 적용할 수 있어 다양한 환경에 맞게 선택 가능

- 양자화(가장 쉽고 즉각적)부터 시작하여,
- 멀티 GPU, 캐싱, 증류 등 필요에 따라 조합 및 증설 가능
- "incremental": 한 기법만 써도 되고, 여러 기법을 단계별로 적용해도 됨
- 사용 중인 모델(ONE, Flux 2, LTX 2 등)별로도 자유도 큼

### 최신 연구와 툴(Attention FP4, FastGen 등)이 실제 모델 개발에 바로 적용 중임

- How Lab의 Attention FP4 등 주목받는 최신 양자화/어텐션 경량화 연구가 매주 쏟아지는 상황
- FastGen의 분포·궤적 하이브리드 증류법은 품질과 속도 두 마리 토끼를 동시에 달성
- 오픈소스 체크포인트, 실습 코드 제공 등으로 실무 적용 진입장벽이 낮아짐

### 증류 등 최적화 시 실제 요구 데이터셋과 컴퓨트 자원 규모 안내

- 증류 적용에는 꼭 GB200급 초고가 GPU가 필요한 것은 아님. Hopper, H100, H200, Blackwell B200 등 폭넓게 지원
- 프리트레이닝에 비해 컴퓨트 부담이 적으며, 모델 사이즈가 2B, 4B인 경우 훨씬 간단
- 데이터셋은 범용 데이터로도 괜찮으나, 특수 목적일 땐 도메인 별 맞춤 데이터셋 필요
- 적절한 평가 기준(데이터 간 성능 차이 탐지 등)도 중요하다고 언급

### 디퓨전·영상 생성 모델 최적화는 연구 중심이지만 점차 안정적으로 정착 중

- 디퓨전 영상 생성 분야는 여전히 연구 중심으로, 매번 새로운 테크닉이 등장 중
- 증류, 양자화 등 기존 LLM에서 쓰이던 기법들이 빠르게 접목
- 앞으로도 오토리그레시브+디퓨전(트랜스퓨전) 등 복합형 신기술이 확장 전망
- 실제 영상 생성 품질·속도 개선이 실무에 임팩트를 주고 있으며, 커뮤니티·새 모델 개발자들의 활발한 참여를 독려하며 발표 마무리
