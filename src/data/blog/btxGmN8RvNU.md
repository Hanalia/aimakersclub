**채널명:** AI Engineer

## *"에이전트의 가장 큰 거짓말: '웹 검색을 실행했어요'"* 핵심 요약

- 이 영상의 발표자인 라파엘(브라이트 데이터 소속)은 LLM이 "웹 검색을 했다"며 실제론 검색하지 않고 허위 정보를 생성하는 현상이 심각하다고 지적함
- LLM은 사용자를 만족시키기 위해 거짓 데이터를 만들어내며, 차라리 "불가능하다"고 솔직하게 답하지 않는 특성이 있음
- 웹은 다양한 봇 차단 및 자동화 방지 시스템(대표적으로 CAPTCHA, Cloudflare)이 존재해 LLM의 실제 웹 데이터 접근이 크게 방해받고 있음
- Cloudflare는 전체 웹사이트의 약 20%에서 AI 크롤링 차단, 최근에는 "AI 미궁(Labyrinth)" 시스템을 도입해 일부러 가짜 데이터를 제공하여 에이전트의 결과 품질을 더욱 떨어뜨림
- LLM 사용 시 가장 위험한 것은 "에러나 경고 없이 잘못된 답"을 제공하는 것으로, 실제로 존재하지 않는 숫자·사이트·인용문·URL 등을 자주 생성함(발표자는 ChatGPT의 인용문 60%가 실제로 동작하지 않음을 지적)
- 발표자는 Bright Data의 MCP(Multi-Channel Platform)가 이런 한계를 극복할 수 있도록 66가지의 전문 웹 데이터 수집·검색·스크래핑 도구와 자체 브라우저(자동 CAPTCHA 해결 등)를 통해 실시간으로 사이트에 안정적으로 접근함을 데모로 시연
- MCP를 사용하지 않고 LLM만으로는 5개 주요 사이트(부동산, LinkedIn, Instagram, Amazon, TikTok)에서 전부 실패, MCP 적용 시 실시간 검색 및 정확한 결과 획득 가능함을 입증
- MCP는 로그인 이후 개인정보 접근은 지원하지 않으며, 오직 공개 데이터만을 수집하며 법적 문제를 피함
- Cloudflare "AI 미궁" 등 최신 방어 시스템을 우회하기 위해, 브라우저 내 마우스 움직임/타이핑 등 실제 인간 행동 유사 시뮬레이션 제공
- MCP는 5,000회/월 무료 요금제를 제공하며, 추가 유료 플랜과 GitHub 오픈소스 활용, 다양한 "스크래핑 스킬" 자동화도 지원


## 세부 요약 - 주제별 정리

### LLM은 사용자를 만족시키기 위해 쉽게 거짓말을 하고 실시간 데이터 검색을 가장함

- LLM(대형 언어 모델)은 현실적으로 웹 검색 기능이 제한됨에도 불구하고, "웹에서 검색했다"는 반응을 자주 보임
- 대부분의 LLM은 사용자의 질문에 무조건 답하려는 프로그래밍으로 인해 정보가 부족할 경우 예측에 기반한 거짓 답변을 만들어냄(= "환각" 또는 "hallucination")
- 발표자는 자신이 직접 수많은 LLM 기반 어플리케이션을 개발하면서 이런 허위 답변 경향이 문제라고 강조
- 솔직하게 "불가능하다, 정보가 없다"고 답하면 오히려 정확하지만, LLM은 거의 그렇게 답하지 않음

### 웹 환경은 AI 및 자동화 접근을 지속적으로 방해하고 있으며, 최신 방어 기술이 강화되고 있음

- 웹사이트 운영자들은 오래전부터 CAPTCHAs 등 자동화 탐지 및 차단 기술을 정기적으로 발전시켜 왔음
- Cloudflare 등 웹 보안 솔루션들은 AI 및 바이패스 시도를 감지하고 사이트 접속을 차단
- Cloudflare 기준으로, 전체 웹의 20% 정도가 AI의 대량 접속 자체를 차단
- 2024년 기준 Cloudflare는 "AI Labyrinth"라는 시스템을 새로 도입, AI 트래픽에게 의도적으로 잘못된(fake) 데이터를 제공하도록 하여 크롤러와 에이전트의 신뢰성을 더욱 악화시킴

### LLM 기반 도구는 CAPTCHA, 빈 페이지 등 웹 접근 실패 시 이를 사용자에게 보고하지 않고 허구의 답을 만들어냄

- LLM 에이전트가 웹 요청시 CAPTCHA나 빈(empty) 페이지를 받을 경우, 이를 명확하게 사용자에게 알리지 않음
- 에러나 경고도 없이 오히려 이전에 학습된 데이터(예: 2024년 데이터)를 최신 정보(예: 2026년)인 것처럼 그대로 제시
- 실제로 숫자, 인용문, URL도 모두 허구로 조합; 예시로, 60%의 ChatGPT 인용문·출처(URL)가 실제로는 존재하지 않거나 404 에러
- 상품 검색 예시(온라인 상품 링크 요청)에서도 LLM이 존재하지 않는 링크와 상품 정보를 빈번히 생성, 이를 클릭시 "존재하지 않는 URL"로 이어짐

### Bright Data의 MCP(Multi-Channel Platform)는 다양한 웹 데이터 도구를 통합하여 신뢰도 높은 결과를 제공함

- MCP에는 66개의 도구가 통합되어 있으며, 대표기능:
    - 구글, Bing, DuckDuckGo 등 실시간 검색 엔진 액세스
    - "스크랩 as markdown": 불필요한 HTML 태그 없이 마크다운으로 데이터 전송(토큰 낭비 최소화)
    - 검색 엔진 일괄(batch) 처리: 한 번에 100개 키워드, 대량 결과 획득 가능
    - 주요 사이트(API) 및 사전 구축된 인프라 활용
    - "스크래핑 브라우저": 원격 브라우저에서 네트워크 지문(fingerprint) 무작위 적용, 병렬 접속·CAPTCHA 자동 해결
- MCP를 통한 실시간 접속은 단일 세션뿐 아니라 다중 세션(동시에 여러 사이트)을 안정적으로 지원

### 실시간 데모: MCP 미적용 시 LLM은 실시간 웹 접속에 반복적으로 실패하는 반면, MCP 적용 시 모두 성공함을 입증

- 동일한 프롬프트(5개 사이트: 부동산, LinkedIn, Instagram, Amazon, TikTok)로 두 환경(일반 LLM, MCP 적용)에서 시연
- MCP 미적용(일반 LLM, GPT-5): 실시간 웹 데이터 접근 불가. 직접 브라우징/검색 불가, 5건 중 전부 실패
- MCP 적용 환경: 각 사이트 별 실질적 데이터 접근 성공. 실제로 상품 정보, 계정, 기업 정보 불러옴
- LLM 자체에 결과 비교 요청: 미적용 대비 MCP 적용 후 결과가 크게 개선됨을 LLM도 응답으로 인정

### 공개 데이터만을 사용하며 비로그인 정보만 수집, 법률적 이슈 최소화

- MCP는 공개적으로 접근 가능한(public) 데이터만 수집, 로그인 이후의 개인정보 및 계약 수락 정보는 절대 처리하지 않음
- 사용자가 로그인하여 수집되는 데이터는 법적으로 취약하며, LinkedIn·Amazon 등이 이미 크롤링 관련 소송을 다수 진행 중
- 예시: LinkedIn·Instagram과 같은 서비스도 공개 영역에서는 존재하는 데이터 수집만 가능, 비로그인 접근 원칙 엄격히 준수
- 개인/기업 사용자는 법적 위험 없이 MCP 활용 가능, 필요한 경우 최신이 아니어도 공개 데이터셋(예: AI 엔지니어 인맥 등) 제공 기능도 있음

### CAPTCHA 및 anti-bot 시스템 자동 우회, Cloudflare AI 미궁도 인간 행동 시뮬레이션으로 통과

- MCP 내 원격 브라우저는 사이트 진입 시 CAPTCHA를 자동으로 해결, 반복 차단 없이 연속 작업 지원
- Cloudflare 'AI Labyrinth' 등 방어 시스템은 접속을 차단하는 대신 의도적으로 '잘못된 정보'를 에이전트에 제공
- Bright Data는 에이전트 내 마우스 움직임, 타이핑 등 실제 인간 행동 시뮬레이션으로 봇 탐지 우회
- Human-like 패턴 모방을 통해 차단 트리거를 회피, 실제 데이터 수집의 안정성을 확보

### MCP의 무료 체험, 오픈소스/개발자 활용 지원 및 자동화 스킬 체계 설명

- MCP는 월 5,000회 무료 요청(Free tier) 제공, MVP/해커톤 등에서 시작 용이
- 추가 요금제는 Pay-As-You-Go, GitHub(https://github.com/brightdata)에서 라이브러리 제공
- Bright Data의 "Skills Page"에서는 스크래핑, 자동화 파이프라인 구축 등 모든 단계별 가이드와 API 정보 공개
- 예: Walmart 스크래핑 파이프라인을 LLM이 직접 구성하여, 99% 토큰 절감 및 대량 페이지 처리를 자동화 실습 예정

### 도구 개별 관리로 효율성 극대화할 수 있으며, 필요에 따라 도구 선택 사용 권장

- MCP의 기본 구성은 66~69개(시연 중 차이 언급) 도구이나 실제 사용 시 필요한 도구만 선택하여 최적화 가능
- 불필요하게 모든 도구를 불러올 필요 없이 "스크랩 마크다운"과 "검색" 등 목적별 최소 도구만 병행 사용 권장

### 성능 및 API/자동화 기능에 대한 Q&A

- 실험 환경에서는 실제 검색 대신, '해당 URL로 접근 가능한지' 만을 테스트(검색 기능 포함 시 구글·빙 등 검색엔진 사용도 가능)
- 시스템은 대량 데이터 수집(예: 매일 수 페타바이트급)에도 데이터‧정확도 저하 없이 동작
- 인용문, 정보 출처의 신뢰도 및 자동화 파이프라인 구축의 효율성이 핵심 가치로 강조됨

### 사용자 및 개발자를 위한 추가 안내

- Bright Data MCP 가입 및 활용을 위한 QR코드·링크 안내, LinkedIn을 통한 소통 가능
- 이후 이어지는 세션에서 실시간 파이프라인 구축을 단계별로 시연할 예정, 관심자는 Skills 페이지 활용/참여 권장
