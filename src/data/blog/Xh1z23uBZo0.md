## *Claude Plans, Gemini Designs: 아름다운 프론트엔드를 위한 통합 워크플로의 실험* 핵심 요약

- 본 영상은 Gemini 3.5 Flash와 Claude(Opus, Sonnet), 그리고 Clerk를 활용해 다양한 LLM(Large Language Model) 모델 및 도구를 하나의 워크플로에 통합하여 "아름다운 프론트엔드+실질적인 작업"을 자동화하는 실험을 실시간으로 진행함.
- Gemini 3.5 Flash는 뛰어난 UI/프론트엔드 생성 능력을 보였으나, 실제 컨텐츠와 정확성, 시스템 연동(예: 인증 연결)에는 한계를 드러낸다는 경험적 사례가 공유됨.
- 이를 보완하기 위해 Opus로 정보 플래닝 및 통합 작업을 하고, Gemini로 실제 UI/프론트엔드 개발, Sonnet으로 검증 작업을 각각 담당시켜 각 모델의 장점만 추출하는 전략적 조합을 시도함.
- 실습 프로젝트로는 "여러 LLM 워크플로의 품질, 속도, 비용을 비교 평가할 수 있는 벤치마킹 대시보드" 개발을 설정, 실무 복잡성을 높이기 위해 Clerk 인증 기능도 통합함.
- 이 모든 과정을 자동화, 스마트 분업화하는 오픈소스 워크플로 오케스트레이터인 Archon이 핵심 역할(각 모델/도구 간 맥락 및 결과 전달, 작업 분할, 병렬 처리 등)을 맡음.
- 워크플로 노드는 탐색→계획→UI빌드→통합→검증→수정→배포→마무리 등 8단계로 구분, 각 단계는 범용적 아키텍처와 명확한 입력/출력 구조로 설계됨.
- 각 모델별 API/요금 체계(Kimi의 저렴함, Gemini의 속도와 합리적 가격, Opus의 강점, 비교 데이터 등)가 상세히 공유되어 도구 선택 및 비용 효율성에 대한 깊은 논의가 있음.
- 실시간 실패(예: Gemini 빌드 요약 누락) 및 이를 대비한 리커버리 설계(노드간 아티팩트 교환, 검증/수정 단계 역할, 제한적 오류 허용 등) 사례가 소개됨.
- 실질적인 작업 흐름, UI 결과, 인증 통합, 배포 자동화 등 코드 및 도구 세팅 전 과정이 구체적으로 시연되어, AI/코딩 워크플로 자동화 실전 능력을 보여줌.
- 실무 특화된 인증 도구(Clerk), 오픈소스 워크플로 설계 및 LLM 모델 간 혼합전략 등 최신 AI코딩 트렌드와 이를 통한 생산성 극대화 노하우가 풍부히 전달됨.

## 세부 요약 - 주제별 정리

### Gemini 3.5 Flash는 UI 생성에 뛰어나지만 정확한 정보 생성과 통합에서는 한계를 보임

- Gemini 3.5 Flash로 생성한 프론트엔드는 심미적으로 매우 우수하나, 실제 컨텐츠(예: 제품 설명, 사용법, 코드 연결)에 있어서 사실 왜곡(‘환각’), 기능적 오류가 다수 발생함
- 예시로, Gemini 3.5 Flash만으로 “Archon 사용법” 랜딩페이지를 만들었을 때, GitHub 저장소까지 참조했음에도 과도한 오류가 포함됨(설정법, 기능 안내가 잘못됨)
- 합성된 UI는 “LLM이 생성했다”는 흔적(일관된 테일윈드 CSS나 특유의 디자인 요소, 박스 배치 등)이 적어 실제 감각적으로 우수하지만, 정보 신뢰성은 취약함
- 이런 한계를 보완하려면, 모델이 가장 잘하는 부분만 맡기고 다른 부분은 다른 모델에 분업시키는 복합 워크플로가 필요함

### Opus, Gemini, Sonnet 등 여러 LLM 모델을 단계별로 분업해 단점을 극복하는 전략이 효과적임

- Opus(Anthropic Claude Opus) : 논리적 플래닝·추론에 강함(컨텐츠·정보 설계/통합/전략 담당)
- Gemini 3.5 Flash : UI/프론트엔드 시각적 구현에 특화(컴포넌트 및 디자인 담당)
- Sonnet : 속도·저비용 기반의 탐색, 검증 역할에 최적(간단한 문서 요약·점검 등)
- 워크플로 설계에서 각 노드에 최적의 모델을 할당해 장점만 발휘, 취약점은 보완되도록 함→ 결과적으로 비용·속도·신뢰성, 생산성을 모두 잡을 수 있음
- OpenRouter, Pi, 클라우드/로컬 혼합, 다양한 API플랫폼 활용 방법도 함께 적용함

### 실증 프로젝트로 벤치마킹 대시보드를 선정, 복잡한 인증 통합까지 풀스택으로 자동화함

- 실험 도구: Archon용 벤치마킹 대시보드(여러 개의 LLM 워크플로를 여러 모델 조합으로 실행하고, 결과/속도/비용 등을 비교평가할 수 있는 웹앱)
- 대시보드는 Archon 워크플로 명령, 스펙 입력, 실행 결과 수집, 평가, 모델별 벤치마크 입력-출력 관리 등 상당히 복합적인 요구사항을 처리해야 함
- 단순 랜딩페이지가 아니라, 실제 애플리케이션 통합·인증 처리(예: Clerk로 회원인증 및 로그인을 구현)까지 자동화해야 성공으로 판단
- 손쉬운 작업 완성도를 위해 Clerk의 신규 CLI/Skill 활용(ex: 인증키·도메인 자동 설정, 배포/테스트CLI 통합 등)

### Archon 오픈소스 워크플로 오케스트레이터가 복합 모델 통합 및 단계 분업의 핵심 역할을 함

- Archon: 각 단계별 노드 정의(탐색-계획-UI-통합-검증-보완-배포 등) 및 그 사이 데이터/문맥/출력 전달, 모델 제공자(OpenRouter, Pi, Claude 등) 혼합, 환경변수/아티팩트/로그 관리 등 자동화
- YAML 기반 워크플로 사양 구성: 입력 스펙, 각 단계별 아티팩트(markdown 파일 등) 지정, 스킬/CLI 및 환경 변수 설정 등 구조적/재사용성 높게 설계
- 아키텍처: 단순히 LLM 연속 호출이 아니라, 각 노드별 전문화+맥락·중간결과 분리→모델별 최적화·비용/성능 도달

### 워크플로 각 단계별 상세 설계 및 문서 내 아티팩트 전달 구조를 통해 안정성과 확장성이 보장됨

- 각 워크플로 노드는 입출력 아티팩트 형식(Markdown 등)과 명확한 작업 범위를 가짐→ 후속노드는 앞 단계 산출물을 참조해 령적 일관성 유지
- 예시: 탐색(sonnet)이 context.md 생성→계획(Opus)이 해당 파일을 읽고 계획(플랜) 작성→UI 빌딩(Gemini)이 ‘UI 범위’ 섹션만 전달/참조
- 필요시 일반화/유지보수 편의를 위해 중간 노드를 추가, 플랜 분할, 의존성/컨텍스트 연계성 강화가 용이함
- 에러/실패(특정 파일 누락 등) 대비해 수정, 보완, 재실행(고치기, fixed-validation 등) 단계가 설계되어 있음

### 각 모델/API 별 요금, 선택기준, 성능비교를 구체적 데이터로 공개하며, 경제성 논의가 병행됨

- Gemini 3.5 Flash:
    - 입력 100만 토큰당 $1.50, 출력 당 $9.00
    - 속도·비용은 중간급, Kimi보다 비싸지만 Opus(입력 $5, 출력 $25)보다 현저히 저렴함
- Kimi: 가장 저렴, 충분한 품질을 원할 때 “초저가” 모델(신뢰도의 단점)
- Opus: reasoning power 최강, 비용은 매우 높음, “플래닝”등 반드시 필요한 단계만 제한사용 권장
- 모델 혼합 및 토큰 효율 설계(예: 오직 플래닝만 Opus, 그 외는 Kimi/Haiku 등 저가모델)로 “성능 ⬆, 비용 ⬇” 전략을 제시함
- API 요금 제한, rate limit, 무료/유료 등 실제 써본 경험, 문제점, 대비책 등 현장성 있는 정보도 다룸

### 실시간 워크플로 구축과정과 오류, 복구, 자동화, 검증 등 실전 사례가 상세하게 시연됨

- Build Prompt 작성부터 아키텍처 설계, 노드별 역할/입출력 플로우, CLI 도입까지 구체적으로 소개
- 시작: YAML로 탐색(sonnet), 계획(Opus), UI(Gemini), 통합(Opus+Clerk), 검증(Sonnet), 보완(Opus), 배포(Vercel) 등 8노드 설계
- 각 노드 산출물을 artifacts 폴더에 Markdown 파일로 저장→워크플로 실행 중간중간 문서 확인 가능
- Gemini가 중요한 요약 파일 생성을 누락하는 현실적 실패도 포착, 이 때에도 워크플로 구조 덕분에 전체 프로세스가 멈추지 않도록 설계됨
- Clerk CLI 내장 게임(Flappy Bird) 시연 등 해당 도구 사용법/특징 외 구성도 체험적으로 전달

### 인증(Clerk) 통합 및 배포, 환경변수, DNS, 실제 제품 배포 등 풀스택 연동 과정이 구체적으로 공유됨

- Clerk 인증 통합: CLI skill 및 wizard로 dev/prod 환경 자동화, 도메인 셋업, DKIM 등 이메일 인증용 DNS 직접 등록, 도구가 출력하는 명령어대로 진행
- Vercel 배포: 프론트엔드 서버 구동부터 실제 도메인 배포까지 환경변수 세팅, 인증키 전달, 환경 분리 등 자동화 됨
- DNS 전파 지연/오류 발생 시 수동 조치 및 단계별 확인(Clerk admin UI 등)까지 실전 경험을 통해 설명
- 배포 완료 시 실제 로그인, 기본 동작 확인, 여러 모델의 Archon 워크플로 탑재 및 실행(일부 기능 미구현/오류는 실시간으로 공지하며 반영)

### 동적 워크플로, 하니스 엔지니어링, AI 코드 자동화 등 최신 AI 코딩 트렌드/개념도 체계적으로 소개됨

- 하니스/harness 엔지니어링과 오픈소스 워크플로 관리(예: Archon) 실무 내용을 직접 실습하며 응용법 소개
- Claude Code, Codex, OpenRouter 등 다양한 에이전트, 워크플로/서브에이전트, 평가 자동화 등 실제 동적인 대화형 AI코딩 트렌드 설명
- 동적 워크플로·서브에이전트(예: Dynamic Workflow, GitHub Actions, CodeRabbit) 등과 Archon의 차별성을 구조/철학/유연성/혼합적용 등으로 비교 설명

### LLM 시대의 유튜브 및 교육 문화, AI 창의성, 실시간 라이브 스트리밍 교육 방식에 대한 현실적 논의

- 유튜브·AI코딩계의 컨텐츠 현주소(레포만 읽는 채널, 튜토리얼, 교육의 수준하락)에 대한 자기성찰적 논의 포함
- Live 코딩을 통한 ‘진짜’ 실전 시연의 가치를 강조, 실패/실험/문제해결이 함께 노출되는 학습 효과에 중점
- 클라우드 기반 협업/자동화/기획 노하우, 직접적 업무 적용 방안 등을 강사 경험담과 실제 고민/사례로 풀어냄

### AI 기업의 실적/평가/지속가능성 및 산업 구조 변화에 대한 구체적 금융 논의도 곁들여짐

- Anthropic, OpenAI, Berkshire Hathaway, Walmart, Samsung 등 AI/비AI 산업군별 기업 가치와 수익구조, ARR 대비 밸류에이션 20배 초과 등 팩트&수치 풍부하게 제시
- Anthropic의 연 매출(47억–60억~, 약 1조 미만) 대비 965억 달러(130조 원) 가치 등 AI 버블/산업적 현실도 실제 데이터와 함께 비교 설명

### 현업에서 바로 쓸 수 있는 극실전적 팁·노하우·구현 전략이 대화 및 데모에 자연스럽게 녹아 있음

- 모델 선택, 워크플로 단계별 작업 분리/통합, 아티팩트 관리, 인증 자동화, 환경 구성, 에러 핸들링, 성능 평가 등 실무 활용 전 과정 팁이 풍부함
- 손쉬운 도구/모듈 구조, 오픈소스 연동 및 CLI 자동화, 토큰·가격 효율 극대화 등 업무 자동화와 관련된 현대 AI 실무 역량이 자연스럽게 체득될 만한 내용임

### 끝으로, Gemini와 Opus의 조합 실전 실험을 통해 LLM 기반의 프론트엔드 제작 자동화 방법을 입체적으로 탐구함
