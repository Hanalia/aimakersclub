## *GenMedia와 함께하는 바나나 실습 — Guillaume Vernade, Google DeepMind* 핵심 요약

- **구글 딥마인드의 Guillaume Vernade가 GenMedia 멀티모달 생성 AI의 최근 기능과 실제 활용법을 소개함**
- **영상·음악·이미지 등 미디어 데이터가 일상에 필수적임을 강조하며, 딥마인드는 이 모든 모달리티를 통합하는 월드모델 구축을 지향하고 있음**
- **Gemini의 발전과정(1.0~2.0)과 딥마인드 내 다양한 멀티모달·에이전트·로보틱스 및 오픈·리서치 모델 라인업 설명**
- **GenMedia 하위모델: Nano Banana(이미지), VO(Video), LIA(음악), TTS(음성합성) 및 각각의 최신 기능(예: Nanobanana2의 검색 기반 이미지 그라운딩, VO 3.1 Light의 저렴한 영상 생성, LIA의 실시간·악기지정 음악 생성) 데모**
- **Jupyter/Colab 노트북에서 실제로 오픈북(‘The Wind in the Willows’) 텍스트로 다양한 생성작업(캐릭터·챕터별 이미지, 챕터 영상, 배경음악, 대사 TTS)을 연동하는 구체적 워크플로우 시연**
- **API 활용법(파일 업로드, 구조화 출력, 챗모드, 서비스 우선순위/요금 구조, 상호작용 API 등)의 기술적 차이와 권장 사용법 안내**
- **프롬프트 엔지니어링과 Gemini를 활용한 프롬프트 생성 자동화(특히 긴 컨텍스트·일관성 유지, 음악/영상/이미지를 위한 맞춤 프롬프트 작성)를 강조함**
- **대화 음성합성 시 내레이션/캐릭터별 스타일 조정, 다국어·악센트, 감정 표현 등 세밀한 TTS 활용 트릭 공유**
- **실시간 음악 생성, 이미지→음악 변환, 음악구조(BPM, Verse, Chorus 지정) 등 GenMedia 모델의 고급 활용 예시 다수 데모**
- **유럽 사용자의 모델 접근성 제약, 데이터 소버린티/프라이버시 문제에 대한 현황과 구글의 대응 방향 논의**

## 세부 요약 - 주제별 정리

### 아침 세션 및 발표자의 역할 소개와 개발자 옹호 활동의 실무적 예시 제시

- 발표자 Guillaume Vernade는 전직 게임 프로듀서 출신으로, 6년 전 구글합류 및 마지막 2년간 DeepMind 개발자 옹호자(Developer Advocate)로 활동
- 개발자 옹호자는 새로 출시되는 모델이나 기능을 개발자들이 쉽게 쓸 수 있도록 문서, 코드 샘플, 데모, 가이드(예: 프롬프트 가이드) 등을 제작·배포함
- 내부적으로는 ‘꼭 구글 중심이 아니라 실제 개발자 활용에 실용적인 기능설계’를 사내팀에 피드백/조정하여 양방향 소통 역할
- 모델 별 API가 통일되지 않은(예: Imagine 모델명별 상이한 API) 내부 혼란 사례, 개선을 위해 노력했으나 현실적 한계도 겪음(하지만 일부 타협적 종료)
- 업무의 대표적 예로, Imagine 브랜드·API 통합 이슈, 출시주기 조정, 각 팀 별 모델 운영체계 간소화 등 현장을 반영하는 목소리 전달을 언급

### 실제 세계의 미디어는 다양한 모달리티로 구성되며, 딥마인드는 이를 통합하는 ‘월드모델’을 추구함

- 현재 휴대폰/IT 환경에서 이미지·사운드·영상 등 미디어 데이터가 보편적임을 강조
- DeepMind의 장기 목표는 현실 세계의 모든 감각(소리, 영상, 텍스트, 센서 신호 등)을 아우르는 멀티모달 월드모델 구축
- 기존에는 이미지·비디오·음악 생성 등 모달리티별 특화 모델을 배포했지만, 궁극적으로는 올인원 모델 통합을 지향
- 릴리즈 및 서비스 안정성을 위해 우선 각 모달리티 별 세분화 모델로 출시 후, 점진적 통합중임

### Gemini 모델의 멀티모달 진화와 딥마인드 모델군 라인업 소개

- Gemini 1.0~1.1은 본래 멀티모달 설계이나, 실제 초기 버전에는 이미지 인풋 기능이 일시 제거된 적 있음(테스트/운영상 이유)
- Gemini 1.5부터 본격적으로 멀티모달 입력 지원, 2.0에서는 멀티모달 대응 한계점 개선됨
- DeepMind 전체는 평균 5일마다 신규 모델 혹은 기능을 릴리즈, 실제로는 주간 2~3건, 소규모 기능까지 포함하면 주중 다수 업데이트
- 주 모델군: Gemini(대형멀티모달), Nanobanana(이미지), VO/VO3.1 Light(VIDEO), LIA/LIA Real Time(음악), TTS(음성합성), Gemma(오픈모델), Alpha Evolve/Genome/Weathernext(특수 연구모델), 다양한 Robotics/Agents
- 2024년은 실제 에이전트 개발의 “실행의 해”로 내다봄(2023년은 말만 무성)

### Nanobanana2, VO3.1 Light, LIA 등 GenMedia 최신 주요 생성 모델 기능과 요금 구조 구체 시연

- Nanobanana2: 지원 픽셀(520~4K), 웹 검색 기반 검색/이미지 그라운딩, 건축/동물 이미지 등 실시간 검색 활용(법적 이유로 신축 빌딩 등 제한)
- VO3.1 Light: 1초당 5센트, 1영상 40센트로 매우 저렴하게 비디오 시안 제작 및 후속 업스케일 가능, 빠른 반복작업에 적합
- LIA: 30초 클립(4센트), 3분 곡(8센트), 악기/스타일/BPM/가사 등 프롬프트 기반 제어, LIA Real Time은 실시간 음악 생성·프롬프트 전환 가능
- 최근 도입된 새로운 서비스 티어: Normal(기본), Flex(50% 저가·대기 가능), Priority(2배 가격·최상 우선순위), 모델별 지원상황 다름
- API 호출 과금 예시(영상 파트가 비용의 대부분), 제한있는 무료티어(Nanobanana 1)
- 실시간 음악 생성 데모(실행 타이밍에 따라 곡 분위기 변화, 프롬프트 전환), DJ 믹스·게임 BGM 등 실시간 활용 아이디어 제시

### 오픈북(‘The Wind in the Willows’) 연동을 통한 각 미디어 생성 기능 통합 워크플로우 예제 상세 해설

- 구텐베르크 프로젝트에서 오픈 소스 책 다운로드(‘The Wind in the Willows’), 텍스트 전체 Gemini에 입력
- Gemini의 긴 컨텍스트 윈도우+챗모드 활용: 챕터·캐릭터별 이미지/프롬프트 자동 생성, 각 챕터 상황 묘사 등 연속적 실행
- 챗모드의 히스토리 기능·구조적 출력(JSON 스키마, 프롬프트명 포함) 활용, 스타일(예: "colorful building block") 지정 등 실용적 구현
- 각 캐릭터별 이미지 생성 후 배열에 저장, 추후 챕터 이미지 생성시 등장 캐릭터별 이미지 전달하여 일관성 유지
- 이미지 생성 방식: 챗모드의 히스토리 기반 자동 연계 및, 각 챕터별 등장 인물 리스트-참조 이미지 기반 직접 전달 두 가지 방식 비교 및 성능 차이
- 프롬프트 자동화: 챕터별 상황/스타일/인물 명세 기반 내용 생성, 일관된 일러스트 제공

### Gemini와 GenMedia API/SDK의 다양한 기술 옵션, 운영 차이, 사용상의 팁과 주의사항 비교 안내

- Google AI Studio 앱(소비자용, 쉬운 UX), Vertex AI(엔터프라이즈급 커스텀/세분화 제어, 복잡한 버킷·ACL 관리), Gemini 개발자 API(중간 지점, 빠른 시작)
- 파일 업로드(특수 API), 버킷관리 생략, API 키 관리 유의(보안 이슈)
- 챗모드(컨텍스트 관리), 자원을 아끼기 위한 재시도(auto retry), 서비스 티어 옵션, generate_content/generate_content_stream 등 다양한 호출 패턴 제공
- 신규 Interactions API(상태유지형, 컨텍스트를 서버에서 재활용, 캐싱 자동처리, 포크·파생 채팅에 유용, 세션 저장기간 약 2일), 향후 기본 API 전환 가능성
- 예약어 간, 매개변수/컨텍스트 용량, 추후 가격정책 등 실전 팁 안내

### 영상 생성(VO 활용)시 챕터별 프롬프트, 이미지·텍스트 연계 및 애니메이션 시나리오 생성법 심층 시연

- VO 모델로 챕터별 주요 장면을 이미지+프롬프트 기반 영상으로 생성(예: 프롬프트+참조 이미지=영상 1프레임), 고가 VO3.1 사용시 품질↑
- 프롬프트를 별도로 재작성(Gemini로 ‘다음 몇 초간의 장면 설명’을 새로 생성하여 맥락 일치), 대사/상황 설명을 세밀하게 반영
- 프롬프트 구체성·길이에 따라 영상 퀄리티 및 스토리 일관성 좌우됨, 자동화 개선 아이디어(예: 각 캐릭터별 전신·측면 등 다각도 이미지 추가)
- 실습 중 챗모드 단점(기존 히스토리 과도 반영→비슷한 프롬프트 중복 생성)과 한계 사례 직접 해설

### 음악 생성(LIA 활용) 시 구조화 프롬프트, 가사/악기/BPM/장면 설명 등 고급 활용법과 트릭 제시

- 30초 클립(BPM, 악기 지정, 지시문 내 다양한 섹션/악기 요구), 3분 곡(복잡한 곡구조/가사/번역 등 지원)
- Gemini로 각 챕터 상황 맞춤 프롬프트 생성, 프롬프트가 곧 음악설정(길이, 스케일, BPM 등) 역할
- 챗모드 사용시 곡 테마가 과도 일관화되는 단점(히스토리 중복 영향) 실사례
- 가사 입력시 짧은 시간 내 곡+가사 동시 생성 가능, LIA의 다양한 언어·악센트·노래방 타임태그 활용(가사-음성 출력, 타임스탬프 병렬 생성)
- 가사 직접 입력or자동 생성, 특정파트 반복(코러스), 다중언어/감정표현 등 세밀한 요구문 지원 등 고급 프롬프트 활용법 제시

### TTS(음성합성) 활용시 내레이션/캐릭터별 스타일 차별, 다중 목소리·톤·악센트 활용 트릭 실제로 구현

- Gemini로 책 내 대사 자동 추출→플레이 형식으로 재구성(캐릭터명+지시문), 각 캐릭터별 대화 스타일(예: 내레이터-천천히, 캐릭터1-시적, 캐릭터2-숨가쁨 등) 명령
- 구글 TTS 모델은 동일 음성으로도 프롬프트 상 스타일 지시로 각기 다른 톤·속도·감정·악센트 표현 가능
- 자동화 구성 팁: ‘read this text’ 같은 심플한 명령은 필수, 대사별 음성 조작 및 상황별(흥분, 귓속말 등) 세밀한 제어 가능
- 노트북상에서 실제 두 캐릭터 동일 음성으로 각기 다른 스타일 출력(청자 분별 가능, 대본만 옳게 설계되면 현실적 구현 가능) 시연

### 초대형 컨텍스트 윈도우 활용(책 전체 입력), 멀티모달 입력(텍스트/음성/영상 등) 확장 실제 예시

- 초대형 컨텍스트로 단일 세션에 책 전체/다수 정보 동시 입력 가능, 히스토리 재활용 및 멀티모달 간 맥락 맞춤 실현
- 예시2: Sci-fi 네온 스타일(‘The Adventures of Chatter the Red Squrriel’)로 책 이미지 일괄 생성, 스타일 변화로 참신한 이미지 생성
- 실제로 여러 언어, 다중 스타일 조정, 네온·레트로 게이밍 스타일, 동화·판타지 등 다양한 장르 적용 가능

### AI Studio, 모델 갤러리, 실시간 음악 및 이미지-음악 변환 등 다양한 GenMedia 실전 예시·갤러리 소개

- AI Studio 내 예제 앱(실시간 음악 생성, 이미지→음악, Space DJ·노래방·시뮬레이션 등 창의적인 활용앱 다수)
- LIA Real Time 모델로 이용자 인터랙션에 따라 스타일·장르(BGM·K-pop·포스트펑크 등) 실시간 전환, 게임 등 인터랙티브 환경 데모
- Space DJ: 행성별 프롬프트 지정/이동시 음악 전환, 실시간 DJ 믹스같은 체험 가능
