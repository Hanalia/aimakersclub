- 이 영상은 Google DeepMind의 Thor Schaeff와 Philipp Schmid가 Gemini API 및 구글 AI Studio를 활용해 대화형 에이전트(Conversational Agent)와 코딩 에이전트, 실시간 음성 에이전트를 구축하는 전과정을 현장 실습과 데모 중심으로 안내한 워크숍입니다.
- 세션은 개발자 경험 개선과 API 생태계 변화, Gemini API(특히 Interactions API)의 구조와 사용법, 서버사이드 상태 관리, 도구 결합·함수 호출 등 다양한 개발 패턴을 상세히 설명합니다.
- Free Tier의 Gemini API와 AI Studio를 활용해 API 키 생성부터 프로젝트 생성, 인증까지 실제 실습 과정을 단계적으로 시연합니다.
- Interactions API의 도입과 핵심 기능(통합 모델/에이전트 인터페이스, 서버사이드 상태, 캐싱 최적화, 툴 콤비네이션, 스트리밍 등)이 OpenAI, Anthropic 등 업계 표준에 부합하게 개선됐음을 강조합니다.
- Gemini 기반의 코딩 에이전트(파일 읽기/쓰기, 커맨드 실행 등)를 실제 Python 예제로 구성, 코드 자동생성, 상호 함수 호출/응답 흐름 등 실전적인 예제를 순차적으로 보여줍니다.
- 에이전트의 기능 확장(시스템 프롬프트 활용, 도구 추가, 커맨드 실행 등), 연속 대화 및 히스토리 관리, 툴 체이닝 등 개발 노하우를 구체적으로 다룹니다.
- 서버사이드 상태 관리를 통해 대화 기록과 컨텍스트를 파편화 없이 효율적으로 유지, 입력 토큰 캐싱 효율을 2~3배 높이며 비용을 절감함을 실제 수치와 예시로 설명합니다.
- Gemini 3.1 Flash Life 등 최신 실시간 오디오 모델의 특징, 멀티모달/실시간 처리, 툴 사용, 실시간 구글 검색 그라운딩, 다국어/억양 지원, WebSocket API 활용 등 혁신 기술을 다양한 데모와 함께 시연합니다.
- Viva coding, 실시간 음악 생성 에이전트, 이미지/음성/비디오/텍스트 동시 처리, 매개변수 기반 목소리 조절 등 최신 사례와 활용 팁을 공유합니다.

## 세부 요약 - 주제별 정리

### 다양한 언어와 환경에서 Gemini API 및 구글 AI Studio 실습 환경을 준비함

- 발표자 Thor Schaeff, Philipp Schmid가 참석자 국적/언어를 묻는 소통으로 시작하며 워크숍의 글로벌 성격을 강조.
- AI Studio 활용 시 무료 계정(구글 로그인)만 있으면 어떤 환경(Windows, Mac, Linux 등)에서도 참가 가능함을 안내.
- ai.dev(구매한 도메인)로 접속 후 키 발급, 프로젝트 생성 및 API 키 안전하게 관리하는 팁(공유 금지, 깃허브에 실수로 노출 등 유의)을 반복적으로 강조.
- 실습 중 발생하는 오류 처리는 현장 질의응답으로 즉각 해결.
- 실습에 앞서 각자 소개 및 기대목표(예시: Gemini로 코딩 수업, 고객지원 챗봇, 구글 글래스 연동 등)를 공유하도록 유도해 실전 중심의 실습 방향 설정.

### Gemini Interactions API의 구조와 기존 API 대비 개선된 개발 경험을 설명함

- Interactions API는 2023년 12월 베타 출시, 향후 기존 generateContent API를 대체할 계획임을 알림.
- OpenAI의 Chat Completions API 등과 매우 유사한 통일된 요청/응답 구조 채택, 모델과 에이전트를 자유롭게 구성 가능.
- 딥 리서치(Deep Research), 툴(예: 구글 검색, 사용자 정의 함수) 조합, 호출 후 비동기 처리(폴링/웹훅), SSE 스트리밍 등 새로운 작업 패턴 지원.
- JSON 기반 입출력 타입(함수 호출, 텍스트, 이미지, 오디오, 비디오 등)을 한눈에 쉽게 운용할 수 있게 됨.
- 서버사이드 상태 관리(대화 히스토리, 이전 인터랙션 ID 체인) 지원으로 개발자가 직접 히스토리 관리 불필요, 코드가 단순화됨.
- 기존 generateContent API 대비 Interactions API는 서버 상태, 비동기 실행, 툴 조합, 쉬운 컨텍스트 이어붙이기, 높은 캐싱율(입력 토큰 90% 비용 절감) 등 다수의 이점을 제공.

### Gemini 기반 코딩 에이전트 구축 과정을 단계별로 실제 코드 예제로 시연함

- 기본 에이전트 클래스 생성: Gem AI 클라이언트와 모델ID, 이전 인터랙션ID 필드를 선언하고, run 메서드에 interactions.create 호출을 구현.
- Multi-turn 대화: 예시 프롬프트("My name is Phil", "What is my name?")로 대화형 기억능력 구현.
- 실습 간 발생할 수 있는 오류(예: uv 패키지 충돌, 라이브러리 미설치)를 즉시 수정하는 방법 제시.
- 시스템 프롬프트 추가, Persona(역할 설정), 도구 사용 안내 등 프롬프트 엔지니어링 팁 공유.
- 파일 읽기/쓰기(read/write file), Bash 명령 실행(run command) 등 실제 도구 정의 – Python 함수 및 JSON 스키마 동시 생성으로 함수 호출 흐름 관리.
- 툴 호출 시 함수 실행 → 결과 응답 → 다음 입력 루프 자동화 로직(재귀 호출)을 단계별로 구현.
- 시스템 프롬프트와 예제 응답을 동적으로 조정, '친절한 코딩 에이전트' 등 맞춤형 성격 부여가 API상에서 쉬워진 사례 제시.
- Bash 커맨드 예시(날짜 확인, 파일 생성, 난이도 조절 등)로 기능 확장을 시연.

### Gemini 도구(스킬) 생태계와 실제 개발 워크플로우 자동화 방식을 소개함

- Gemini Agent Skills를 공식 깃허브(google-gemini-gemini-skills)에 공개, npx 등으로 손쉽게 설치 가능함.
- 스킬 설치 위치(글로벌 또는 프로젝트별) 및 폴더 구조(agents/skills/...)에 대한 실무 팁 안내.
- 여러 에이전트 도구(Anti-gravity, Cloud Code, Cursor 등)와의 호환성 언급, 각 툴의 특성과 연동법 소개.
- 스킬 내용은 문서가 링크되어 실시간 fetch가 가능, 기능 업데이트 시 문서만 갱신하면 최신 정보가 반영됨(문서 기반 에이전트 트레이닝).
- 도구 추가 시 실제 코딩 스킬과 도구 정의, 파일 구현, JSON 스키마, 함수 연결 등을 자동화하는 전략 제공.

### Interactions API의 서버사이드 상태관리 및 캐싱 최적화 효과를 구체적으로 분석함

- 서버가 대화 히스토리와 상태(이전 인터랙션ID, 대화 내역 등)를 보존, 클라이언트는 최소 정보만 전송해 반복 작업(토큰 인코딩, 맥락 조립)이 불필요.
- 토큰 캐싱: 서버가 동일 입력의 토큰 인코딩을 재활용 → 최대 90% 비용 절감, 스타트업 기준 캐시 적중률 2~3배 개선 수치 제시.
- 캐싱 구조: 입력 파편(줄바꿈, 공백 등) 변경시 캐싱이 깨질 수 있으므로 서버 상태를 활용하는 것이 오류 가능성 최소화.
- API의 저장 기간(Free Tier: 1일, 유료/엔터프라이즈: 55일 등), 클라이언트에서 인터랙션ID 브랜칭, 상태 복구 등 고급 사용법 설명.
- 맥락 윈도우(최대 백만 토큰) 도달 시 자동 에러 및 압축(context compaction) 기법 논의.

### Gemini 3.1 Flash Life 등 최신 실시간 오디오 모델의 구조와 가능성을 구체적으로 데모로 소개함

- Gemini 3.1 Flash Life: 완전한 실시간 음성 인식∙생성 모델, 음성-텍스트 변환 경유하지 않고 음성 간 토큰 직접 추론(사운드 토큰→사운드 토큰), 네이티브 오디오 지능 기반 설계.
- 97개 언어 지원(실시간 다국어 코드스위칭), 다양한 억양 및 스타일 프롬프트(영어, 아일랜드 억양 등) 가능.
- WebSocket 기반 쌍방향 실시간 통신, 실시간 오디오/비디오/텍스트 입력, 기초 프롬프트(시스템인스트럭션)를 통한 대화 캐릭터 세팅.
- 실시간 툴 사용(날씨 검색, 음성 답변, 다양한 도구 조합)과 자동 Voice Activity Detection(중간 끊기, 인터럽트 대응).
- 구글 공식 데모(실시간 암연결, 음악·텍스트 생성, 이미지/비디오 분석 등) 및 파트너 사례(Shopify Sidekick, HeyAdo, Whimo, Stitch 등) 실무 적용 실제 예제.
- Native Audio의 장점: 반응 속도, 자연스러운 대화, 자유로운 언어 혼합, 실전 활용 시 다양한 파트너 솔루션(음성 데이터 저장·가시화 등) 연동 가능.

### 멀티모달리티(텍스트·음성·비디오 동시) 환경에서의 에이전트 제작 및 구현 예시를 상세히 제시함

- 실시간 음성 대화와 동시에 이미지, 비디오 입력을 통해 문맥적 이해 및 응답을 실행하는 멀티모달 에이전트 구조 시연.
- FastAPI 기반 서버, WebSocket 연결-프록시 구조, 클라이언트-서버 비동기 처리(에피머럴 토큰 활용) 실제 Python 코드 예시.
- 클라이언트에서 직접 WebSocket을 통한 음성/비디오 스트리밍 가능, 시스템 프롬프트 통한 Guardrail 설정/도구 관리.
- 프론트엔드 뷰, 토큰/인증 구조, 실시간 서버 이벤트 모니터링 등 전체 구현 과정을 단계별로 설명.
- Google Search Grounding(실시간 검색 연동) 및 커스텀 툴 연동 데모, 실제 음성 명령(날씨 질의, UI 색상 변경 등) 가능한 구조 안내.

### 실무 적용 관점에서 세션 관리, 컨텍스트 윈도우, 데이터 보안 및 파트너 솔루션 대응방식을 논의함

- 세션 제한: 오디오 전용은 15분, 오디오+비디오는 2분(슬라이딩 윈도우, 컨텍스트 압축 기능 활용 가능).
- PII, 데이터 저장: Store=False 설정 시 서버에 저장하지 않음(단, 서버사이드 상태 이용 불가), 엔터프라이즈 용 Vertex AI의 세부 기능 언급.
- 자동 트랜스크립트 저장 미지원(파트너사 착수 필요), 파트너 툴(Lifekit, Pipecat 등)이 통합 솔루션 및 대시보드 제공.
- 실무 도입시 보안, 법적 요구(HIPPA, SOC2 등) 및 커스텀 파이프라인 구축, 파트너 통합 활용 전략 안내.

### 실제 적용사례(Shopify, HeyAdo, Whimo 등)와 혁신적 서비스 및 한계·실전 팁 공유

- Shopify Sidekick: 실시간 화면 공유를 통한 쇼핑몰 셋업 지원, 화면 프레임 인식 기반 안내.
- HeyAdo(아르헨티나): 노인 복지 영역의 멀티링구얼 음성 동반자 및 보호자 알림 제공.
- Whimo: 자율주행차 대화 UX 적용 등 혁신적 실제 사례 소개.
- 실무 한계: 실시간 음성 모드(네이티브 오디오)는 파이프라인 기반 대비 투명성 및 응답 개입(중간 편집, 가드레일 삽입 등)이 부족함.
- 향후 발전 방향: 향상된 도구 관리, 개별 스피커 구분, 세밀한 콘텍스트 파악 등 보완 지점 안내.

### 컨텍스트 관리, 사용자 수준별 반응(개인화) 등 대화형 에이전트의 실전 최적화 팁을 공유함

- 컨텍스트 윈도우 내에서 실시간으로 사용자 레벨/지식 반영 가능, 스피커 구분은 초기 컨텍스트 세팅 or 함수 호출로 가능.
- 인터랙션 역사를 클라이언트에서 저장하면 브랜칭 및 평행 대화/업무 지원 가능.
- 대화 내에서 이름, 역할 등 기억하며 연속적으로 맞춤 반응 가능(단, 컨텍스트 윈도우를 벗어나면 잊음).
- 다수 사용자 실습 시 스피커 구분(voice print, 자체 인증 등) 보완 요청이 반영되지 않은 한계와 현존 대안(엔비디아 파라킷 등) 언급.

### 실무 환경에서 대화형 에이전트 평가 및 고도화 전략, 베스트프랙티스 안내

- 음성 대화형 앱은 보안·법적 요구(HIPPA, SOC2 등), 함수 호출, 가드레일 등 추가 관리 필요, 실제 서비스 적용 전 파트너 통합(예: Lifekit) 고려 추천.
- 허위 응답(Weather Hallucination 등) 최소화를 위해 시스템 프롬프트/가드레일 정교화 중요, 툴 정의와 지침 강화로 신뢰성 향상.
- 엔터프라이즈 환경(B2B, Vertex AI 등)에서는 전용 베타 기능/높은 쿼터/데이터 주권 설정과 함께 현 솔루션의 한계 명확히 인지 필요.
