## *AI 다크 팩토리는 스스로 코드를 작성하는 코드베이스다 – 실시간 데모* 핵심 요약

- "다크 팩토리"는 AI가 전적으로 코드를 작성·배포하는 오픈 소스 프로젝트로, 인간의 코드 리뷰나 배포 개입 없이 움직임
- 본 영상은 이 "다크 팩토리" 시스템의 실전 동작과정, 이슈 트리아지, 자동 개발·테스트·배포 워크플로우 전체를 시작부터 실제 예시까지 실시간으로 데모함
- 핵심 도구로는 Archon(아콘, 오픈소스 하니스를 설계하는 도구)과 Miniax M2.7 모델(저비용, 빠른 AI LLM), Claude Code, GitHub Issues 등이 쓰임
- 사용자는 발견된 버그나 개선점, 기능 요구사항을 자연어 서술로 GitHub Issue로 등록하고, AI가 이를 직접 트리아지, 계획, 코딩, 테스트, PR 생성·머지 및 배포까지 수행
- 각 이슈는 Archon YAML 워크플로우로 정의되어 자동화되고, 실행은 VPS(4GB RAM 저사양)에서 배치 스케줄 기반으로 진행, 리소스 관리 문제는 시행착오로 적절히 조정
- 실제 예시로 UI/UX 문제(채팅 입력창 높이·오토네이밍·오토스크롤 등)를 그룹화해 여러 이슈로 분류, AI가 이를 병렬 또는 순차로 처리하도록 함
- 자동화 과정엔 실행 로그 모니터링, 워크플로우 상태 점검, Pull Request 자동 검증(브라우저 자동화 포함), 자동 회귀테스트(Weekly Cron)이 내장
- 다양한 LLM(예: Miniax, GLM 5.1, Quen 3.6 등)의 성능/비용 비교 및 즉석 교체 방법 실습, 공공 오픈 소스/서드파티 툴(OhMyCodeex, Open Harness 등)과 비교 논의도 포함
- 빌드 과정에서 prompt engineering, 워크플로우 prompt 수정, 버그 재현용 자동화 등 실시간 problem solving 및 개선 방법을 상세히 시연
- 해당 시스템은 코드 품질 향상과 반복적 자동 개선(자가학습 루프)에 집중하며, 생산성·유연성·확장성을 극대화한 실험적 접근임을 강조

## 세부 요약 - 주제별 정리

### AI가 전적으로 코드를 개발하고 배포하는 ‘다크 팩토리’의 실시간 동작이 공개됨

- "다크 팩토리"는 인간의 코드 작성·리뷰나 배포 개입이 0%인 AI 기반 소프트웨어 공장임
- 운영자는 단 1줄의 코드도 작성하지 않고, 모든 이슈/버그/작업 지시는 GitHub Issue와 자연어로 입력
- AI가 이슈를 감지·분류(트리아지), 코드 작성, Pull Request 생성, 코스 리뷰, 자동 병합, 배포까지 자동화로 순환
- 데이터 파이프라인, 유튜브 멀티 RAG 챗봇 등 실제 서비스(프로덕션)로 적용되며, 강의·워크샵 자산(_Dynamist_ 연동)도 통합 예정
- 실시간 스트리밍으로 초기 인프라부터 완성된 워크플로우까지 2시간 반 가량의 데모를 수행

### 사용된 주요 기술: Archon, Miniax M2.7, Claude Code 등으로 유연한 AI 자동화를 구현함

- Archon(아콘): AI 코딩 프로세스를 YAML 워크플로우로 패키징하는 오픈소스 하니스 빌더. 분리된 워크 트리, 병렬 처리 지원
- Claude Code: 지시문/프롬프트를 받아 코드 기반 워크플로우를 제어하거나, 리포에 이슈를 생성하도록 함
- Miniax M2.7: 저비용(30센트/백만토큰), 빠른 실행의 LLM. Anthropic·GLM 5.1 등 타 LLM도 유연하게 교체 가능
- VPS 환경(4GB RAM): CPU/메모리 리소스를 모니터링하며 브라우저 자동화, 병렬 처리 제한 등 실전 인프라 튜닝 과정 시연

### GitHub Issues를 AI 개발 파이프라인의 단일 진입점으로 삼아 설계·운영함

- 모든 문제점, 신규기능 요청, 개선의견을 자연어로 GitHub Issue에 기재
- Claude Code가 이슈 분류/병합(그룹화) 및 세부 이슈 생성 지원
- 이슈별로 Archon 워크플로우가 자동 할당되어 실제 코드 변경·테스트·리뷰·머지·배포가 이루어짐
- 이 과정에서 워크플로우/룰/YAML파일 작성·수정은 주로 AI가, 파라미터(우선순위, 라벨 등)는 운영자가 조정 가능

### 이슈의 우선순위 배치, 트리아지, 효율화 전략을 데이터 중심으로 자동화함

- 새 이슈가 오면 바로 하나씩 에이전트를 띄우는 방식이 아니라, 다수 이슈를 그룹화·트리아지해서 우선순위·유사 건 중복 판단, 효율화 운영
- 사용자(운영자)도 라벨, 프라이어티, 승인 규칙을 실시간으로 수정해 AI 우선배치 로직을 동적으로 교정
- 실전에서는 낮은 사양 VPS로 인해 병렬 진행 이슈 수 제한, 토큰 소모량 실시간 체크 등 현실적 리소스 관리도 같이 고안

### 풀 워크플로우: 코드 작성, 브라우저 자동화 테스트, 자동 머지·배포까지 자동화됨

- 코드 변경점이 PR로 생성되면, 별도의 워크플로우가 브라우저 셀레니움 기반 자동 클릭/테스트로 검증
- Pull Request 자동 리뷰(테스트 실패, 보안 체크 등), 머지 및 배포까지 사람이 개입하지 않음
- 주요 예시로 채팅앱 UI/UX 개선(입력창 센터링, 대화명 오토네이밍, 오토스크롤 등)을 이슈 –> AI 수정 –> PR –> 테스트/머지까지 자동 반복

### 지속적인 회귀 테스트와 자동 이슈 생성 루프 등 자기개선 피드백 시스템이 탑재됨

- 주 1회 크론잡이 메인 브랜치 전체에 대해 회귀(browser automation)테스트를 수행하고, 실패시 AI가 자동 이슈 생성
- 이를 통해 버그, UI/UX 누수 등을 단일 루프에서 자동 탐지→수정→배포하는 ‘자가 학습’ 구조 실험
- 이상징후, 잘못된 머지, 반복 실피 등에 대해 운영자가 YAML 워크플로우 수정 및 룰 업데이트로 최적화

### 다양한 LLM 모델과 API 요금제 비교, 동적 모델 변경 등 비용효율성과 유연성을 확보함

- Miniax M2.7(30센트/백만토큰), GLM 5.1(더 비싸고 3~4배 높음), Quen 3.6, Kimi 등 성능/비용/속도 실사례 비교
- 개인 구독제/직접 API 결제 여부, 대규모 병렬처리시 스케일 여부 등 실험
- 모델 교체는 오직 환경변수(API키, base URL, 모델명)만 수정해 무코드로 즉시 전환 지원
- 라우팅(GPU별 다중 모델 혼합), API 명시적 선택 및 모델간 워크플로우 즉시 변경 시연

### 이슈 재현 자동화(Archon 재현 워크플로우)로 AI가 버그 여부를 스스로 판별함

- 사용자는 이슈 번호만 입력하면, Archon 워크플로우가 문제 구간(웹UI, CLI, API 등)을 자동분류
- 자동으로 백엔드/프론트 구동, 테스트 인스턴스 실행, 브라우저 자동화/CLI 등의 방법으로 재현 시도
- 성공/실패/부분적재현 시에는 해당 이슈에 자동으로 코멘트
- 프롬프트 튜닝/수정, 프로세스 단절/세션 유지 등 개선요구 예시도 수회 반복

### 실전 prompt engineering 및 워크플로우 개선·오류 수정과정이 상세히 시연됨

- 이슈마다 워크플로우 내 prompts(자연어 지시문) 세부 리뷰 및 직접 수작업 개선
- Bash 노드 등 결정론적 단계와 AI 노드(프롬프트 기반) 단계의 적절한 병행 및 구조적 분리 예시 제공
- 한 workflows에서 오류 발생시 원인분석 및 prompt 실질 개선 루틴 반복(삭제, 기준 강화, 논리 추가 등)

### 오픈소스 harness(매개체) 활용에 있어 Archon과 타 툴(OhMyCodeex, Open Harness 등)의 차별성을 설명함

- BMAD, OhMyCodeex, Open Harness 등은 ‘의견이 강한(Opinionated) 단일 접근’인데, Archon은 자신만의 하니스 및 전체 SW개발주기(SDLC) 커스텀화를 지원
- 명확한 YAML 구조, 확장성(엔터프라이즈 적용) 및 병렬 처리, 커스텀 워크플로우 편집, 사내 도입 시 기존 프로세스와의 통합이 용이함
- 실제 기업 대상 AI 코드 자동화 교육 사례, 대형 조직의 도입 난점, BMAD 불만족 예시 등도 포함

### 다크 팩토리와 Archon의 생산성 향상 및 한계, 미래 전망이 솔직히 공유됨

- 코드 품질 극대화보다는 한계 측정, AI 코딩의 미래/발전 가능성 실험에 더 중점
- 개발자/솔로프리뉴어, 팀 모두 생산성 향상 가능—이슈 등록~개선/배포가 단일 파이프라인으로 압축
- 워크플로우가 미완성/불안정(예: 브라우저 자동화 실패, 프론트/백엔드 기동 실패시 우회 없이 강제 실패, 수동 보완 필요)
- AI 코딩이 인간 개발자를 전적으로 대체하는데는 아직 현실적으로 상당한 시간(5년 이상)이 걸릴 것이란 현실적 관점 제시

### Dynamis AI 커뮤니티 및 강의, 라이브 Q&A, 워크샵 참여 안내 및 시청자 소통으로 마무리함

- 운영자는 실제로 Dynamis라는 AI 커뮤니티를 운영하며 주 1회 워크샵, 심화강의, Q&A 세션 등 교류 확대
- Dynamis 내에서 다양한 실전 케이스, 베스트 프랙티스, 전략 토론 등 커뮤니티 주도 발전 지향
- Archon 실습, 두뇌 개발(Second Brain), 최신 AI 코드 자동화 기법 등 추가 콘텐츠 예고
- 라이브 스트림 진행 체계(무편집, 즉석 소통, 실시간 문제해결, 후속 라이브 예고)로 참여 독려하며 마무리
