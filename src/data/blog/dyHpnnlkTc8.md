## *VS 코드에서 에이전트로 개발 자동화하기* 핵심 요약

- 본 강연은 Visual Studio Code(이하 VS 코드)에서 다양한 AI 에이전트(GitHub Copilot 및 Claude 등)를 활용해 개발 생산성을 높이는 실제 방법을 다룸
- 에이전트의 종류(로컬, 백그라운드, 클라우드)와 각각의 특성, VS 코드에서의 통합 활용 방식이 구체적인 예시와 함께 소개됨
- 로컬 에이전트는 개발자가 직접 코드나 테스트 등 세부 작업에 관여할 때, 백그라운드 에이전트는 UI 작업 등 반자동화가 필요한 작업에, 클라우드 에이전트는 문서 작성 등 완전 자동화가 필요한 작업에 적합함
- 강연자는 하나의 코드베이스에서 동시에 세 가지 에이전트를 병행해 서로 다른 문제(프론트엔드 생성, 테스트 작성, 문서화)를 동시 해결하는 시연을 진행함
- GitHub Copilot과 각종 에이전트는 VS 코드의 단일 진입점(모달)에서 손쉽게 세팅·확장·커스터마이즈 가능함
- 클라우드 에이전트는 GitHub Actions 기반 격리 환경에서 안전하게 동작하며, MCP 서버와 Playwright MCP 서버를 활용해 테스트 및 CI/CD 워크플로우 자동화 지원
- 각 에이전트는 지능형 프롬프트, 커스텀 스킬, 에이전트 별 역할/제약, 후크 등의 세세한 사용자 정의 및 자동화가 가능함
- Github 및 오픈소스 프로젝트(awesomecopilot 등)를 통해 다양한 커뮤니티 리소스 활용 및 확장 방법도 제시함
- MCP(모노 컨텍스트 프로토콜)를 도입해 Azure, GCP, AWS 등 외부 리소스와 통합 가능성을 시사
- VS 코드는 다양한 에이전트와 워크플로우를 통합해 높은 개발 효율성을 제공하며, 에이전트 활용 ROI 극대화를 위한 최신 실전 사례를 공유함

## 세부 요약 - 주제별 정리

### 개발자들은 '에이전트'에 과도한 기대를 하고 있지만 실제 현장은 복합적 과제를 동시 처리해야 함을 강조함

- 많은 개발자들이 AI 에이전트를 ‘원샷 프롬프트’로 모든 문제를 해결할 수 있을 것이라고 착각하나, 현실은 다양한 개발 단계에서 각기 다른 에이전트 활용이 필요함
- 기업은 에이전트 도입 시 투자 대비 효과(ROI), 생산성 향상, 토큰 소비 그리고 실제 효용성에 대해 신중히 접근해야 함
- 토큰 비용을 아끼기 위한 다양한 실험적 접근(예: 해적 영어 등 토큰 수를 줄이는 프롬프트 활용)이 커뮤니티에서 활발히 시도됨

### 현 시점에서 사용 가능한 에이전트는 로컬, 백그라운드, 클라우드로 구분되며 각각의 활용 목적이 다름

- 로컬 에이전트: VS 코드 내에서 직접 실행, Claude 등 다양한 LLM 및 자체 호스팅 모델과 연결 가능 / 개발자가 직접 컨텍스트를 유지하며 개발
- 백그라운드 에이전트: GitHub Copilot CLI 및 VS 코드에서 지원 / Git worktree 등 격리된 하위 디렉터리에서 운영되어 주요 브랜치와 분리, 자동화된 작업과 부분적 인간 관여에 적합
- 클라우드 에이전트: GitHub Actions 등 외부 클라우드 리소스를 사용 / 완전 자동화된 반복 작업(문서화 등)에 최적 / 기업 내 확장성 및 규모화 제공

### 로컬, 백그라운드, 클라우드 에이전트 각각의 실전 활용 시나리오를 구체적으로 제시함

- 로컬 에이전트: 테스트 코드 작성 등 세밀하고 직접 다뤄야 하는 작업에 활용, 빠른 피드백 루프 및 인간 개입 보장
- 백그라운드 에이전트: 프론트엔드(UI) 개발 등 개발자가 일정 수준만 관여하고 나머지는 AI에 위임해야 하는 작업에 사용됨
- 클라우드 에이전트: 레포지토리의 오픈소스 친화성 강화, readme 및 기여 가이드 자동 생성 등 반복적이고 상세 관여가 필요없는 작업에 이상적임

### 실제 데모를 통해 세 가지 종류의 에이전트가 한 코드베이스에서 동시에 병행 동작하는 과정을 상세히 보여줌

- 단순한 Python CRUD 애플리케이션 예시 선택
- 백그라운드(CLI) 에이전트로 UI 추가 작업을 자동화시킴(AutoPilot 기능 활용, 이때 자동 툴 호출 질문을 생략)
- 클라우드 에이전트로 오픈소스 친화화(README/CONTRIBUTING 파일 자동 생성 등) 요청
- 로컬 에이전트로 커스텀 테스트 에이전트를 선택해 테스트 코드 추가 및 오류 처리 개선을 직접 검수·수정
- 각 에이전트의 작업 결과와 상호 동작 상태를 실시간으로 VS 코드 내에서 병렬 관리

### 각 에이전트 작업 결과를 VS 코드 풀 리퀘스트/디렉터리 등에서 직관적으로 확인하며 개발자 관점에서 관리가 용이함

- 에이전트가 생성한 코드·테스트·문서를 PR(풀 리퀘스트) 형태로 확인 및 병합 가능
- Git worktree를 활용해 별도의 디렉터리에서 새 UI 등 변경사항을 안전하게 체크 가능
- 직접 로컬 실행 및 시연(포트 충돌 등 작은 트러블슈팅 포함) 과정을 소개함으로써 실무적인 리얼리티 강화

### 클라우드 에이전트 실행 구조와 보안상 장점, 추가 워크플로 자동화 사례를 구체적으로 설명함

- 클라우드 에이전트는 격리된 GitHub Actions 환경에서 동작해 기업 보안 요구 사항 충족
- MCP 서버 및 Playwright MCP 서버를 통해 자동화 테스트 실행(스크린샷 포함), 동적 워크플로우, 네트워크 화이트리스트 등 안전장치 적용
- 메인 브랜치 직접 접근 불가 등 보안성 중시 설계로, 기업에 적합한 안전한 에이전트 환경 구현

### 에이전트별로 다양한 커스텀 기능(스킬, 프롬프트, 후크 등)과 VS 코드 내 유저화 기능을 손쉽게 관리할 수 있음

- VS 코드 내 모달 UI에서 모든 에이전트·스킬·명령어(프롬프트), 후크 등을 한 화면에서 세밀하게 설정·편집 가능
- 커스텀 에이전트(예: 테스트 자동 생성), 내장 에이전트(Ask, Explore, Plan 등), 내장 스킬, 프롬프트 파일 등 연동
- Claude 등 써드파티 에이전트 및 각종 플러그인(Claude 플러그인, 후크 등)도 동일하게 확장·연동 가능함

### 강연자는 'Awesome Copilot' 등 오픈소스/커뮤니티 리소스를 통한 추가 확장성과 타사 AI 도구 연동법도 제시함

- awesomecopilot(aka.ms/awesomecopilot) 프로젝트에서 다양한 커스텀 스킬, 프롬프트, 플러그인 등 예제/자료 제공
- VS 코드 및 GitHub Copilot 외에도 여타 AI 도구(Claude 등)와도 설정 공유·재사용이 자유로움
- 실무 커뮤니티에서 마이크로소프트/GitHub 범위를 넘어 다양한 LLM 및 워크플로우에 적용되고 있음

### MCP(모노 컨텍스트 프로토콜)를 활용해 외부 클라우드 리소스 통합 및 채팅 커스터마이즈 확장성을 강조함

- MCP(Mono Context Protocol): LLM 및 에이전트와 Azure, GCP, AWS 등 클라우드 리소스를 보안연동 및 확장 가능
- 인증 기반 접근 및 Playwright, 문서 생성 등 일부 오픈 MCP 서버 존재(즉시 활용 가능)
- MCP로 사용자 워크플로우에 맞춘 맞춤형 에이전트/자동화 라우팅 실현

### VS 코드는 에이전트 중심 개발 자동화의 단일 진입점을 제공하며, 플러그인/확장/CLI 연동 모두 하나로 통합 가능함

- VS 코드 한 곳에서 모든 에이전트, 챗봇 커스터마이즈, 다양한 플러그인, MCP 및 Copilot CLI 세션을 순차적이면서도 동시적으로 활용 가능
- 워크플로우 단순화와 생산성 실질적 증가, 지속적으로 확대되는 에이전트/AI 도구 생태계 소개
- 강연은 이후에도 참가자 피드백 및 워크플로우 공유를 독려하며 마무리됨
