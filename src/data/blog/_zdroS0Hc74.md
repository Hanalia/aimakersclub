## *AIE 유럽 2일차: Google Deepmind, Anthropic, Cursor, Factory, Linear, Hugging Face, Cerebras 등 AI 엔지니어 컨퍼런스* 핵심 요약

- 영상은 2024년 런던에서 열린 AI Engineer Europe(유럽 AI 엔지니어 컨퍼런스) 2일차의 모든 키노트, 주요 토크, 데모, 대화, 폐막 순서까지 자세하게 기록함.
- Google Deepmind, Anthropic, Cursor, Factory, Linear, Hugging Face, Cerebras, Arena, Microsoft, Bit.ly, Lagora 등 글로벌 AI 및 생산성 툴 주요 인사들이 대거 토크에 참여.
- Google은 최신 오픈 소스 모델인 Gemma 4의 특징·성능·라이선스·커뮤니티 반응·활용 사례를 상세히 소개하며 기업·개발자 직접 배포와 파인튜닝을 강조.
- Anthropic의 MCP(모노컨텍스트 프로토콜)는 1억 1천만 월간 다운로드 이상을 기록, 에이전트와 SaaS 간 연결을 혁신하며 2026년 ‘완전한 연결성의 해'를 예고.
- 에이전트 오케스트레이션, 멀티 에이전트 자동화, 코드 자동화, 대화형 협업, 사람과 에이전트의 작업 분담 흐름에서 필요한 심층적인 실무 전략이 공유됨.
- 대표적인 오픈 소스 코딩 에이전트 사례(Pi, OpenClaw)는 고도의 확장성, 사용자 주도형 커스터마이징, 슬로우 개발 문화와 코드베이스 품질 유지의 노하우를 집중 설명.
- 복수 발표에서 빠른 모델(1,200 token/s 출력 등) 등장의 위험과 ‘느린 개발자가 반드시 필요함’을 강조하며 에이전트의 생산력 확대와 인간 판단·윤리·품질 관리의 균형이 강조됨.
- Arena, Hugging Face, Cerebras 등은 각자 모델 벤치마크, 코딩 에이전트, GPU 커널 자동화, 오토리서치, 스킬 관리 등 최전선 배포와 실사용 환경의 실제 데이터를 다룸.
- Linear 등 웰메이드 SaaS 개발팀은 ‘기능 남발의 함정’, 품질 관리 문화(버그 제로, Quality Wednesday), 고객 경험 내재화, 채팅 인터페이스를 넘는 AI U/I 혁신을 중점 토론.
- 컨퍼런스 운영팀(세계 최대 AI 행사 주최진)이 실제 행사 업무를 'AI + 에이전트'로 어떻게 완전히 자동화·혁신했는지, 실무 생산성·의사결정 패턴 변화를 사례로 제시.
- 영상의 전반부와 후반부에는 ‘Heat’로 반복되는 시적 서사와, AI 시대의 인간적 성찰(페이스, 판단 중시, 코드와 인간의 경계, 집단지성, 새로운 작업 윤리 등)이 강조되어, 기술과 철학이 융합된 최신 AI 현장의 분위기를 생생하게 포착.

## 세부 요약 - 주제별 정리

### Google Gemma 4는 초소형부터 32B까지 다양하며 모바일·디바이스 내 실행 및 라이선스 혁신을 달성함

- Google DeepMind 소속 Omar Santoro는 최신 오픈 모델 Gemma 4의 상세 스펙, 차별점, 신규 아키텍처(E2B, per layer embeddings), 운영 사례를 발표.
- Gemma 4는 2B~32B 파라미터 크기로 제공되며, 초소형(2B)은 안드로이드, iOS, 라즈베리파이, Nintendo Switch 등에서 로컬, 오프라인 실행 가능.
- 개발자 친화 크기와 Mistral, Llama 등 오픈 모델 벤치마크에서 경쟁력 확보(10M 다운로드/주, Gemma 4 파생 모델 1,000종, 패밀리 전체 5억 다운로드).
- E2B/E4B per layer embedding 구조 적용으로, GPU 외 CPU·디스크로도 임베딩 활용이 가능. 모바일·온디바이스 추론에 최적화.
- 멀티모달 입력(텍스트, 이미지, 오디오, 음성 인식 및 번역)에 탁월, 140개+ 언어 지원, Gemini 기반 멀티링구얼 토크나이저 사용.
- 라이선스를 기존 제한적 OSS → 완전한 Apache 2.0으로 변경, 상업적/기업적 활용 자유화.
- 대형 모델(31B)도 소비자 GPU에서 실행 가능하고, 커뮤니티 중심 파인튜닝·양자화·멀티 플랫폼 배포 협업(MLX, Olama, Huggingface 등).
- 공식 파생 모델(Medma: 의료, Shield Gemma: 콘텐츠 필터링) 및 싱가폴·인도 정부 등 다양한 멀티링구얼·주권 AI 훈련사례 소개.
- “내가 가진 데이터, 내 언어로, 내 기기에 맞춰 커스텀 모델을 구축하는” 진정한 ‘퍼스널 AI’ 시대 도래를 시사.

### MCP는 1억이 넘는 월간 다운로드를 기록하며 에이전트-SaaS 연결성을 산업 표준으로 끌어올림

- Anthropic의 David Sora Par, MCP(MonoContext Protocol)의 발전 현황, 생태계, 향후 과제 발표.
- MCP는 애플리케이션, 툴, UI, 인증, 실험적 기능 연동 등 클라우드·로컬·에이전트 워크플로우를 단일 프로토콜로 연결.
- 2023~2025년 18개월 동안 1억1천만 월 다운로드 달성(React의 2배 속도), OpenAI, Google, Langchain, Linear, Notion, Slack 등 주요 산업체 채택.
- 2026년부터 “실제 지식노동/엔터프라이즈용 일반 에이전트”의 시대가 본격화될 것임을 전망.
- 스킬, CLI, MCP가 에이전트 기능 연결의 3대 축, 실무에 맞는 커넥티비티 스택 조립법(진보적 디스커버리, 프로그램 기반 툴콜) 제안.
- 토큰 절약·지연 최소화·성능 최적화·권한 제어를 위해 Progressive Discovery 패턴, REPL 툴 지정, 구조화 출력(JSON, 타입 명시) 강조.
- REST API를 단순 래핑하는 실수를 경계하며, 에이전트 행위에 맞는 프로토콜 설계 및 TypeScript/Python 2.0 SDK, Stateless Transport 등 향후 계획 공개.
- 기업용 Cross-App Access(SSO), Server Discovery, MCP Apps, Skill-Over-MCP 등 실사용 강화.
- “2026년은 연결성의 해, 최고의 에이전트는 모든 연결방법(MCP, CLI, Skill...)을 혼합 사용한다”는 전망.

### 멀티 에이전트 시스템은 게이밍에서 영감을 받아 가시성, 분업, 협업, 자동화 오케스트레이션으로 작업 효율을 혁신함

- Agent Craft 창시자(이도 살라몬)는 멀티에이전트 오케스트레이터를 게이밍(특히 RTS 게임) 방식에서 착안해 설계.
- 한 번에 수십~수백 명 수준의 에이전트 관리가 ‘병목’임을 식별, 파일 시스템 맵핑, 에이전트 분포 시각화(히트맵), 변경내역 실시간 추적, 충돌 방지로 가시성 강화.
- 메뉴-패널은 미션별 상태, 파일별 변동 이력, 에이전트별 작업 라인이지, PR 묶음 단위 리뷰, 스크린샷·비디오 기반 빠른 검수 가능.
- ‘캠페인’ 기능으로 내가 직접 관리하지 않고도 에이전트 팀이 중간계획, 작업수행, 검수까지 자체 분업.
- 인간-에이전트, 인간-인간, 에이전트-에이전트 등 다양한 협업 모드(워크스페이스, 채팅, 소프트/하드 콜라보레이션) 지원.
- ‘플래닝에서 리뷰로’ 인간의 부담 이동/PR 대량 리뷰 지원 등 최적화.
- “전통적 엔지니어링 기술+게임에서 익힌 관리법 응용이 AI-에이전트 시대의 필수 실무 스킬”임을 강조.

### 오픈 소스 코딩 에이전트 생태계(PI, OpenClaw)는 확장성·관찰성·보안·자율성·자기 진화성을 극대화함

- Pi 창시자 Mario Zechn은 코딩 에이전트 세대교체기를 ‘세 막극’(코딩 생산성, OSS 침식, 과속 경계) 관점으로 진단.
- 클라우드코드 등 상용 harness의 문제점(관찰 불가, 툴정의 임의조정, hooks 미흡, 모델 고정, 확장 불가)을 지적.
- Pi는 미니멀 코드, 핫리로드 확장, TypeScript 기반 직접 확장 지원, 오너블 도구/체계(예: ai, agent core, bespoke framework, coding agent) 구현.
- 사용자는 시스템프롬프트, skill(markdown), 공식 문서 조합만으로 자신만의 Pi 확장/자체 조정 가능.
- “에이전트 자기 수정 가능성” - 개발자 혹은 에이전트가 스스로 harness, 기능, 보안 모델(예: 요롱게 YOLO 모드)등을 증설.
- Pi, OpenClaw 등 포크 생태계, MPM/Github 통한 저변 확장/핫리로드 배포 가능.
- OSS 이슈 트래커 공격에 자동화(휴먼식 PR만 통과)로 대응, 사용자-에이전트의 코드 검토 책임성 문제 제기.
- “빠르고 손쉬운 자동화 생산 속에서도, 중요한 코드는 반드시 직접 읽고 리뷰하라”는 원칙 강조.

### 프릭션(마찰)은 개발자 고유의 판단력이며, 빠른 모델·대량생산 환경에서 '느린 사고'와 아키텍처 설계가 핵심이 됨

- Flask 창시자/Arendil 설립자 Armen Ronacher와 크리스티나(Arendil 개발자)는 에이전트 코드 기반 환경에서 ‘속도만능주의의 함정’과 체험 사례를 공유.
- AI 도구가 압도적으로 생산성을 올리자 코드 리뷰, 설계, 시스템 이해 등에서 철저한 마찰 지점 확보(프릭션)에 방점을 찍음.
- “생산 압박→대형 PR·대량 메이지→리뷰 포기→품질 저하”의 악순환 실례 반복 제시.
- 라이브러리/모듈러 코드의 구성에선 에이전트 성능이 극대화, UI·제품처럼 많은 조건이 엮인 영역에서는 에이전트의 전체 구조 인식력 한계 드러남.
- “AI/에이전트가 설득력 있는 모듈을 잘 만들지만, 복합 상태에서는 데먼트(이상 행위), 로컬 최적화 및 불확실성, 설계 의도 은닉 등 문제 발생.”
- 타입 명확화, 유일 함수명, 한 곳에서만 SQL 실행, Lint 규칙, erasable syntax, Mechanical(기계적) 체크법 개발 및 개발자 리마인더(파이 확장)로 인간의 고민 영역 선별.
- ‘마찰’을 긍정적으로 수용: 인간의 경험, 고유 판단, 큰 그림/변화탐지, 설계 이해를 유도하는 장치임.
- "빠르고 효율적인 AI 흐름 속에서, 진정 중요한 결정에는 반드시 마찰(=판단 개입)을 남겨라" 강조.

### 코드베이스의 에이전트 자동화는 사용성·확장성·검증성에 따라 ‘슬로우+파워 유저’ 지향으로 재설계되고 있음

- Cursor 개발팀 발표자는 1. 오래된 코드 15,000줄을 200줄의 skill(실질적으로는 마크다운 프롬프트)로 대체한 사례 공개
- Git worktree 기반 병렬 에이전트(worktree/Best event), 여러 모델로 동시 실험, 작업 분리, PR 비교, 실제적 UI/UX 개선(중간적용, 다중 코드베이스, 심층 디프 등).
- 이 과정에서 프롬프트 기반형은 “모델 이탈, 자료실행 제한” 등 허점(속도체감, 자동화한계, 회귀에 취약) 및 고급 사용성/발견성 저하의 문제가 존재.
- Evals(브레인트러스트 활용) 통한 자동 프롬프트 최적화/품질 검증, RL파이프라인에 신규 task 반영.
- 차후 Git 이외의 다른 워크플로우 패러다임 지원, 에이전트 기반의 Agentic 인터페이스 강화 시사.
- “에이전트-기반 코딩 생산성 혁신은, 힘 있는 파워유저/정보 시스템 효율에 따름.”

### 빠른 코딩 모델의 시대에는 '느린 개발자'가 필수이며, 실시간 협업·선택적 검증·맥락 관리가 개발 핵심이 됨

- Cerebras, OpenAI 등 신형 모델(Codex Spark)은 1,200 token/s 이상의 초고속 코드생성(기존 모델 60 token/s) 달성.
- Sarah Chang(Cerebras)은 “속도만큼 사고/검증속도가 따라가지 못하면 오류/슬롭 대량 생산된다”고 경고.
- 벤치마크: 하드웨어(메모리 벽), 구조(Layer, KV cache), 디코드-프리필 분리, Mixture of Expert, 캐시 최적화 등 인프라 혁신 소개.
- 새로운 워크플로우 플레이북 제안: 계획/실행 이원화(플래닝은 느린 거대모델, 실행은 빠른 모델), 세션 경험 재구성(리얼타임 협업), 프리커밋-자동 검증-자동 리팩토링 내재화.
- 검증의 ‘비용이 제로’임을 강조하며, 대량 버전 실험/체리픽 등 ‘취향 유도형 검색’ 중요.
- 컨텍스트 윈도우 관리(외부 메모리, Plan.md, Progress.md, Verify.md 등), Role별 분리, 세션별 지속성 강조.
- “빠른 모델, 느린 개발자, AI-휴먼 완전 협업 패러다임” 실무 전략 제안.

### 복잡한 AI 시스템 진단은 'AI를 활용한 내부 분석 자동화'로 전환됨

- Incident.io의 Lawrence는 SRE용 내부 프롬프트/루틴/에이전트/코드 평가(Eval) 파일 체계를 운영하면서, “AI가 AI를 진단”하는 파이프라인 확립.
- 실제 인시던트 보고서 생성→수백~수천 개의 텔레메트리, 로그, 트레이스, 시스템 프롬프트, 추적 등 다단계 프로세스를 파일 시스템 형태로 저장·분석.
- Eval CLI(검색, 수정, 추가, 통합), Agents의 Red-Green 사이클, 반복적 프롬프트 개선-테스트 단계, 자동 Flow→버그 위치 추적 및 힐링 루프 구축.
- 대규모 백테스트(수천 케이스), 유사 원인 클러스터링, 코드베이스-로그-프롬프트 연동 등 인간이 접근 불가능하던 영역까지 AI 분석 자동화.
- 모든 단계를 “파일 기반 파이프라인”+AI 도구+자동화된 런북으로 재구성, 시간-비용 절감 및 신뢰도 극대화.

### 멀티데이·멀티미션 에이전트 팀은 목적과 검증을 명확히 구조화하며 시스템적 복원력으로 확장 가능함

- Factory·Goose 프로젝트 주도 Luke(팩토리)는 미션 기반 멀티에이전트 자동화 구조를 제안.
- ‘Delegration, Creator-Verifier, Direct communication, Negotiation, Broadcast’ 5가지 멀티에이전트 프레임워크를 분해·조합.
- ‘미션’이란 기획-분해-실행-검증을 엄격히 구조화한 3롤(Orchestrator-Worker-Validator) 시스템, 16일 이상(실무 한달분) 연속 자동운영.
- 검증 단계가 자동화의 핵심: 계획상의 Validation contract, 시행 전 Assert, 코드리뷰용 Code Review 에이전트, User Testing Validator(실제 브라우저, 클릭 등 사용성 평가까지).
- 각 단계간 핸드오프(누가, 무엇을, 언제, 어디까지), 마일스톤-체크포인트 내에서 “자동 복원력(자기 치유적)” 설계.
- 동시처리는 사전에 제한적으로 적용(로컬/코드 리뷰 한정), 추가 에러율 하락에 더 효과적.
- 모델 종류와 조합(플래닝-거대모델, 실행-속도모델, 검증-신뢰성모델) 동적으로 배치, 프롬프트/스킬 기반 아키텍처로 향후 진화 가능.
- 벌써 엔터프라이즈 실사용: 신제품 프로토타이핑, 내부툴 신속 개발, 대규모 리팩터링, 코드베이스 현대화 등 업무 혁신 사례.

### AI 시스템 엔지니어링은 코드 생성에서 시스템 커널·파인튜닝·오토리서치 등 '실리콘-근접형 자동화'로 진화함

- HuggingFace의 Ben은 “코딩 에이전트를 GPU 커널 작성, LLM 파인튜닝, 오토리서치 등 최첨단 프로그래밍 영역에 실제 투입”하는 3단계 데모 시현.
- 커널 자동화: 커널스(Kernels) 허브에 toml파일과 스킬/벤치마크 예시, 업스킬(Eval 및 타 모델 비교)를 통한 최적화·상용 배포.
- 제로샷 태스크: HF CLI, Unsloth, 자체 RL파이프 등 활용 파인튜닝 “프롬프트 하나로” LLM 재학습(데이터세트→체크포인트→허브→즉각 배포).
- 오토리서치(멀티에이전트 오토랩): HF Paper 기반 논문 스카우트, Planner-Worker-Reviewer-Reporter 에이전트로 Hypothesis 큐 생성->트레이닝->대시보드 추적 등 자동화 R&D 실현.
- 완전 오픈소스+Parquet 기반 데이터레이크(Drachio)와 자동화 시각화(Gantt 차트, 자유 테이블)로 반복적 개선 가능.
- “Agent-friendly API, Skill 관리, 데이터 완전 개방을 통한 엔지니어링 자동화가 실현 중.”

### 분산/원격 코딩 에이전트 세션 관리 및 '포모(FOMO)' 극복 솔루션이 실무협업의 필수가 됨

- Bit.ly의 Michael Richmond는 ‘포모(Fear Of Missing (Agent) Time, FORMAT)’를 공식화.
- 여러 터미널/IDE/코딩 에이전트 세션을 동시에 운영할 때, 세션잃음/응답 대기/상태파악 못함이 혁신의 방해 요인임을 실사례로 입증함.
- ‘Command and Control’ 앱(iOS/Android/Web)의 대시보드, 실시간 세션 알림, 원격 입력 및 세션 관리, 협업 히스토리, 단일 뷰 제공 기능 소개.
- 복수의 에이전트 환경(클라우드, 로컬, 다양한 플랫폼)에서 세션 중복 파악과, 작업수행 확인, 통합 알림, "모든 에이전트 단일판넬" 작동 시연.
- 아키텍처: 각 플랫폼마다 데몬, 중앙 컨트롤 플레인, 유저 인터페이스 연결 → 분산 환경에서도 실시간 Noti+상호작용 보장.
- “코딩 에이전트 혁명은, 언제 어디서나 연결된 세션 관리/통합 UI 도구가 없으면 업무 전환 비용이 폭증함”을 강조.

### VS Code·Copilot Agent 등 메이저 환경의 에이전트 활용은 실제 생산력·확장성·보안성·인간태스크 분리를 지원함

- GitHub Copilot, VSCode 클라우드팀 라이언은 에이전트 사용 전략, 예시·코드커스텀·테스트/프론트/도큐 자동화/백그라운드 활용법 등 공유.
- Local/Background/Cloud Agent로 태스크 분리: 로컬은 테스트/핸즈온, 백그라운드는 반복·프론트엔드, 클라우드는 도큐/허드성 업무.
- Git worktree, MCP 서버, Secure Sandbox, 네트워크 제한, 메인 브랜치 접근 차단 등 실무 환경에 맞는 확장/보안/권한 관리.
- VSCode 전용 콘트롤패널(에이전트 선택, Skill/Prompt 관리, 서드파티 연계, 커스텀 에이전트 생성, 프롬프트/훅/MCP 등 커스터마이징).
- “실제 생산성은 에이전트 메이저 툴 내 완전 통합·서드파티+사내툴 결합, Chat+Skill+MCP+직관적 채널에서 파생된다” 강조.

### Linear 사례에서 'AI/에이전트 남발은 평균 품질 추락을 부르고', 체계적 품질문화와 고객 경험 내재화가 필수임을 시사

- Linear CTO(토마스 아트맨)는 AI 활용 시대 '하이퍼쉽핑' “무조건 모든 요청 반영, 기능 남발”의 함정 경계.
- “결국 승자는 품질/심미성/디자인 테이스트, ‘설계-엔지니어링-고객 피드백-UX 최적화’ 사이클에 집중하는 팀”임을 강조.
- 기존 고성장 SaaS 사례(우버)와 비교, 품질측정 어려움, 수치(매출, DAU, 트립)만 본 뒤 장기적 품질추락→점진적 고객 이탈 위험 제시.
- Linear는 'Quality Wednesday'(매주 팀원 전체가 작은 품질 문제 스스로 찾아내고 개선 발표)과 ‘Zero Bug Policy’(버그 발견시 즉시 작업 우선 순위, 7일 내 해결, 필요시 무시도 가능)로 특유의 품질문화를 유지.
- 에이전트는 버그 자동 수정에 활용, 실제로 수작업이 아닌 에이전트가 전체 버그 10%+ 자동처리 중.
- 채팅 인터페이스로는 한계, “에이전트와 인간이 함께 ‘공유 아티팩트'에서 고도의 협업을 해야 한다”고 지속적으로 강조.
- 채용·온보딩에서부터 직접 실제 피드백에 노출되고, 코드베이스 내 디자인 의도를 내재화하는 세밀한 문화(고객과 슬랙, PM 회의 등) 실현.

### '모델은 아직도 멀었다': 최첨단 모델도 넌센스·불확실성·미창의 문제에서 약점을 보임

- Arena Peter Gstif는 실시간 벤치마크·Arena 투표 데이터를 바탕으로 “최신 GPT, Claude조차 넌센스(헛된) 질문, 고차원 작업, 게임 설계, 창의 등에서 실패율 9~15% 유지” 데이터 공개.
- 논리력·팩트체크에서는 꾸준한 개선 두드러지나, 창의적 문제/라이프스타일·금융·법률·게임 등 세부 분야에서는 거의 성능 개선이 없음.
- reasoning level 높일수록 오히려 오답률 증가, 모델 크기=품질 등식은 성립 안 함, 단일 프롬프트/단일 벤치마크가 실환경과 괴리.
- 결론: “라인은 올라가지만, 실제 Human-level Expert 작업은 훨씬 복잡․창발적이며, 자동화·AGI 환상에 취해선 안 된다.”.

### AI 도구 + 인간 조직은 철저한 업무 프로세스 자동화와 팀 생산성 혁신을 동시에 달성함

- AI Engineer 컨퍼런스 운영(9명 소수팀, 연매출 900만 달러 규모)이 ‘Figma->Devon 코드->자동화->디자인/사이트/명단/이메일/일정관리/로스터 관리' 등 전체 행사 엔지니어링을 AI에이전트로 처리.
- 디자이너가 피그마 변경→에이전트가 웹까지 구현, 직원 외부 의뢰 없이 잦은 빠른 반복, 창의적 실험/폴리시, Easter Egg 추가 등 신규 업무 경험.
- 스프레드시트-ETL-외부 API 연동-노션화 등 업무처리형 Agent 활용을 모든 비즈니스 업무로 확장(금융, 외주관리, 데이터 관리 등도 포함).
- 직원 피드백 루프 단축, 생산성·자율성 증대, 조직 사회적·기계적 생산성 혁신 사례로 제시.
- 차세대 ‘Agent Experience’(에이전트가 사용자가 되는 경험)이 진정한 디지털 트랜스포메이션의 핵심임을 강조.
