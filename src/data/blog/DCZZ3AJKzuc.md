## *채팅 에이전트에 음성을 추가하는 방법 — Luke Harries, ElevenLabs* 핵심 요약

- 2025년은 채팅 에이전트(Chat Agents)의 해로 평가받으며, SaaS 기업들도 모두 AI 중심으로 전환하거나 자사 앱에 채팅 에이전트를 도입하기 시작함
- 챗 인터페이스가 정부기관(gov.uk) 등 다양한 분야와 서비스의 기본 UI로 빠르게 확산됨
- 텍스트 채팅만으로는 미래지향적이지 않으며, 음성 인터페이스가 더 빠르고 인터랙티브하며 접근성도 높음(예: 키보드 사용이 어려운 사람, 난독증 등)
- 음성을 채팅 에이전트에 추가하면 Zoom 회의 참여, 콜센터 고객 응대 등 ‘옴니채널’ 형태의 새로운 상호작용 가능
- ElevenLabs는 처음에는 세계 최고의 텍스트 음성 변환(TTS) 모델 개발을 목표로 했으나, 기업들과의 협력 과정에서 에이전트 전체 솔루션 구축으로 확장함
- 주요 고객 사례로 Revolut의 고객 지원 시스템이 언급되고, 음성 에이전트의 설계 기본 구조(음성 엔진+에이전트 오케스트레이션) 설명
- 다수 기업들이 이미 자체 에이전트를 갖췄기 때문에, 기존 인프라를 전면 교체하지 않고 음성만 손쉽게 추가할 수 있는 새로운 음성 엔진(Voice Engine) 제품을 곧 출시할 것임을 예고
- Voice Engine은 Scribe(STT), V3(TTS) 등 ElevenLabs의 최신 음성 모델을 결합하고, 이모션/문맥 파악, 수천 개 음성과 언어 지원, 개발자 친화적 SDK 및 UI 컴포넌트 제공
- 개발자는 몇 줄의 코드(SDK 래퍼)만으로 기존 챗봇을 음성 서비스로 변환 가능하며, 웹·전화 솔루션 등 다양한 채널에도 즉시 적용 가능
- 데모 시연 및 코드 예시를 통해 실제 적용 방법과 단순화된 통합 방식을 직접 보여줌

## 세부 요약 - 주제별 정리

### 채팅 에이전트가 SaaS 시장의 기본 UI로 자리 잡은 현 상황 설명

- 2025년에는 챗봇을 도입하지 않은 SaaS 기업은 도태되고, AI 중심의 챗봇 UI가 필수로 자리 잡았다는 평가가 등장
- Linear, Posthog, SEO 등 다양한 SaaS 앱이 홈 스크린을 챗 인터페이스로 변경하는 사례가 트위터에서 바이럴됨
- 이러한 변화 흐름에 대해 “이제 챗 인터페이스가 기본 대화 방식이 됐다”는 입장을 밝힘
- 챗 에이전트는 명령형(Declarative) 대화방식과 빠른 시작이 장점임
- 영국 정부(GOV.UK) 또한 공식 홈페이지에서 챗봇 UI 채택 사례를 들어 대세화 추세를 강조

### 음성은 텍스트 채팅보다 더 자연스럽고 미래지향적 상호작용 수단임을 주장

- 음성은 채팅보다 훨씬 빠르고, 실시간성과 상호작용성이 뛰어난 커뮤니케이션 매체로 소개
- 키보드 활용이 어려운 사람이나 난독증 등 다양한 사용자에게 더 높은 접근성을 제공
- 음성 지원을 통해 다양한 ‘옴니채널(Omni-channel)’ 상호작용, 예를 들어 Zoom 회의 중 실시간 통계 교정이나, 고객지원용 전화 응대가 가능해짐
- 즉, 단순 채팅 에이전트에서 벗어나, 모든 채널/상황에 적용 가능한 음성 에이전트로의 업그레이드 필요성을 강조

### ElevenLabs가 초기에는 TTS 전문이었으나, 이제 에이전트 전체 솔루션을 제공하게 된 배경

- ElevenLabs는 처음에는 최고의 텍스트-음성 변환(TTS) 모델을 개발하는 데 집중함
- 다양한 기업과 협업하면서 음성 인식(STT), 턴-테이킹, LLM 연동 등 통합된 ‘에이전트 오케스트레이션’ 요구가 지속적으로 제기됨
- 대표적인 B2B 사례로 Revolut의 고객지원 자동화 프로젝트가 언급됨
- 대부분의 실전 배치 프로젝트에서, 음성 엔진(음성인식/STT, TTS, 턴테이킹)과 챗GPT 등 LLM, 도구호출(tool calling), RAG, 외부 솔루션 연동 등 복합 시스템이 기본 구조임
- 업계 내 많은 개발자·기업들이 유사한 시스템/구조를 스스로 구축하거나, 솔루션 홍보자료(슬라이드)로 활용하는 경우 다수

### 이미 챗봇 에이전트를 구축한 기업들이 음성 서비스 도입시 겪는 현실적 문제 설명

- 여러 기업들은 자체 챗봇(에이전트) 인프라, 평가(evaluations), 음성 인식, 트랜스크립션 등에 많은 시간과 비용을 투자함
- 기존 시스템을 ‘전면 대체’하기보다는 손쉽게 음성(voice)만 추가할 수 있는 방법 필요
- 기존 챗에이전트 아키텍처를 완전히 재구성하지 않고, 음성 기능만 간편하게 붙일 수 있는 솔루션의 역할이 중요해짐

### 새로운 제품 Voice Engine으로 기존 챗봇에 음성을 손쉽게 통합함을 강조

- ElevenLabs가 곧 출시할 ‘Voice Engine’은 음성 엔진 파트를 독립적인 1급(First-class) 컴포넌트로 묶어 제공
- 개발자는 기존 챗봇 에이전트를 변경하지 않고, Voice Engine 래퍼만 덧붙이면 됨
- 서버 SDK의 구조: 클라이언트 인스턴스 생성 → Voice Engine 생성 → 챗에이전트에 세션 단위로 래퍼(attach) 연동
- 세션 시작마다 기존 챗에이전트와의 통신 프록시 역할을 자동으로 수행

### Scribe, V3 등 첨단 음성 모델과 풍부한 기능을 결합한 점을 구체적으로 설명

- STT(음성-텍스트 변환)에는 Scribe — 가장 높은 정확도 제공
- TTS(텍스트-음성 변환)에는 ElevenLabs V3 — 고품질, 감정·문맥 인식형, 수천 개 음성 및 언어 선택 지원
- 고도화된 턴테이킹(turn-taking) 지원: 대화 중 멈춤(pause) 감지, 문맥 및 감정 인식(semantic & emotion context-aware)
- 기능적 다양성(다언어, 수천개 음성, 다양한 상호작용 상황 등)
- 개발자 사용성을 중시하여, 텍스트 대응 경험을 그대로 음성지원으로 확장하는 것에 초점

### SDK와 UI 컴포넌트로 개발자 경험을 극한까지 단순화함

- 서버 SDK, 클라이언트 SDK 모두 제공
- 클라이언트 SDK 활용시 웹사이트에 단 3줄 코드로 음성 에이전트 위젯 즉시 탑재
- SDK를 통해 전화(telephony), CCaaS(콜센터형 음성서비스) 시스템까지 개발자 친화적으로 지원
- Shaan CN, Vercel 스타일의 UI 컴포넌트를 함께 제공해, 빠르고 직관적인 인터페이스 구현이 가능함

### 실제 데모와 코드 공개로 적용 용이성 입증

- 기존 챗 지원 에이전트를 ‘Hello, how are you?’ 대화 테스트와 함께 실제로 음성으로 동작하는 데모 시연
- 로컬로 실행되는 통합 코드의 일부를 설명하고, Voice Engine의 wrapper 함수 구조, 세션 기반 프록시 연동 코드를 코드 스니펫으로 보여줌
- 새로운 제품 출시 시 ‘베스트 프랙티스’를 내장한 스킬(skill) 및 원프롬프트(one-prompt) 코드 변환 지원 예정
- 개발자는 프롬프트만 입력하면 자동으로 기존 코드베이스 분석 및 래핑 로직 생성, 배포 가이드 제공

### 챗봇에서 음성 에이전트로 가는 ‘추상 레이어 업그레이드’의 가치 제안

- 단순히 ‘음성변환(STT, TTS) API’로 접근하지 않고, 더 상위 레벨의 추상화(voice engine bundle)로 전환해야 한다는 패러다임 제시
- 개발자는 개별 음성 API 호출 대신, 한 번의 래퍼 및 프록시 적용으로 챗봇 전체에 음성을 확장하는 구조
- “차세대 챗에이전트는 ‘단순 채팅’에 머무르지 않고, 반드시 ‘음성’을 추가하게 될 것”이라는 전망

### 사후 Q&A에서 도구호출(tool calling)·통합 관련 상세 답변

- 기존 챗봇(에이전트)의 대부분은 이미 백엔드에서 ‘도구호출(tool calling)’ 기능 내장
- Voice Engine은 기본적으로 프론트(UI)에서 챗 에이전트 전체를 래핑하므로, 별도의 도구호출 처리 필요 없음
- ElevenLabs는 클라이언트 측/서버 측 도구 연동 개념 모두 도입하고 있으며, 차후 래핑된 에이전트로 일부 tool call 프록시 연동 방식도 추가 예정
- 챗에이전트가 이미 완비된 기업이라면 음성 엔진 래퍼만 추가하면 손쉽게 음성 기능 포함 가능

### 11Labs의 협력 의사와 향후 시장 전망 언급

- “채팅 에이전트들은 음성을 추가하지 않으면 도태될 것”이라는 단언적 전망으로 마무리
- 얼리어답터·디자인 파트너 참여 의사 밝히며, 다양한 개발자와 협업 희망
- 이후 간단한 질의응답, Q&A, 박수로 발표 종료
