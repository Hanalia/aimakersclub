**영상 링크:** [The Kiro AI Coding Hackathon has Officially Started! Build ANYTHING, Win Big Prizes.](https://www.youtube.com/watch?v=O8KI2paEBbM)  
**채널명:** Cole Medin

## *키로 AI 코딩 해커톤이 공식적으로 시작되었습니다! 무엇이든 만들고, 푸짐한 상금의 주인공이 되세요.* 핵심 요약

- Dynamus와 Kiro AI가 공동 주최하는 공식 코딩 해커톤이 1월 5일부터 시작되어 1월 23일까지 참가 접수를 받음
- 총 상금 규모는 **17,000달러**로, 누구나 참여할 수 있으며, 참가자는 키로 AI(큐로)를 사용해 자유롭게 프로젝트를 제작해야 함
- 참가 방법은 Dynamus 커뮤니티에 가입하고, 해커톤 섹션에서 안내에 따라 제출하는 방식으로 진행됨
- Kiro 계정 생성 시 **500개의 무료 크레딧**이 즉시 지급되며, 이메일을 통해 추가 크레딧 코드도 추후 제공됨
- 프로젝트 제출 시 소스 코드, 문서화, README, 2~5분 분량의 데모 영상 등이 반드시 포함되어야 함
- 심사 기준은 **애플리케이션의 완성도, Kiro 사용 수준, 문서화 품질, 독창성, 데모 영상의 전달력** 등 5가지로 구성됨
- 팀 참가가 아닌 **개인 단위**로만 응모 가능하며, 1인 1작품만 최종 제출 가능
- GitHub 템플릿 리포지토리와 퀵스타트 가이드(설치, 문서화, 개발 흐름 등)가 지원 도구로 제공됨
- 큐로 CLI 또는 IDE 중 한 가지를 선택하여 프로젝트를 제작하면 됨(CLI가 권장됨)
- 최종 수상자는 2월 7일 Cole Medin 유튜브 채널 라이브 스트림에서 발표 예정

## 세부 요약 - 주제별 정리

### 해커톤 개요와 일정이 명확하게 안내되며 모든 개발자에게 참여 기회가 제공됨
- 키로 AI와 Dynamus가 합작하여 대규모 해커톤을 개최, 누구나 참가 가능함을 강조
- 이벤트 개시는 2024년 1월 5일, 접수 마감은 1월 23일(태평양 표준시 기준 자정)
- 결과 심사는 접수 마감 후 수행되고, **2024년 2월 7일** Cole Medin 유튜브 라이브에서 수상자 발표
- 참가 등록은 Dynamus 커뮤니티 '해커톤 참가하기' 버튼 클릭으로 간편하게 가능
- 이미 Dynamus 회원이라면 해당 섹션에 자동으로 접근, 비회원은 커뮤니티 가입 후 참여
- 해커톤 관련 공지사항은 커뮤니티 채널 및 이메일을 통해 모두 전달
- 참가자 가이드 및 링크, 제출 절차 등은 커뮤니티 고정 공지글에 정리되어 있어 접근 용이

### 큐로 계정 생성 시 무료 크레딧과 추가 지원이 제공되어 누구나 손쉽게 시작할 수 있음
- 참가자는 큐로 계정 생성만으로 **초기 500 크레딧** 자동 지급(30일 내 사용 가능)
- 기존 큐로 이용자도크레딧이 이미 적용되어 있음
- 추가로 주최 측이 일주일간 순차적으로 이메일을 통해 특별 크레딧 코드 제공 예정
- 해커톤 참가에 필요한 비용이 따로 없어 장벽이 매우 낮음
- 지급된 크레딧만으로도 큐로 기능 대부분을 충분히 활용할 수 있다고 안내

### 프로젝트 주제와 형식은 'Kiro AI 사용'만 제외하면 완전히 자유로움
- 유일한 필수 요건: **큐로 AI(키로 또는 Curo) 활용**
- 그 외 하드웨어, 소프트웨어, 웹/모바일 앱, AI 에이전트, CLI 도구, API, 자동화툴 등 장르 제한 없음
- 실질적으로 "상상할 수 있는 모든 종류의 애플리케이션" 제작 가능함을 강조
- 큐로 프로젝트 개발 방법은 CLI(명령줄)와 IDE(통합개발환경) 모두 허용, CLI를 권장
- 공식 문서와 퀵스타트 가이드, 도구별 매뉴얼 등 지원 문서가 풍부하게 제공
- 어떤 도구나 스타일을 써도 심사와는 무관, 큐로의 창의적 활용 자체가 중점 요소임

### 프로젝트 제출 시 요구되는 파일과 문서화 기준이 구체적으로 안내됨
- 필수 제출자료: **소스코드, 프로젝트 전체 문서화, 실습/설치 안내(README), 2~5분 분량의 시연 영상**
- 제안 영상 방식: 비공개 유튜브, Loom 등 자유 포맷 출처 가능
- 소스코드는 집 파일(zip) 또는 공개 깃허브(GitHub) 저장소로 제출
- README 파일로 운영 및 설치, 동작 확인 방법을 명확히 설명해야 함

### 큐로만의 개발 일지와 글로벌 규칙 파일 작성을 통해 AI 협업 과정을 강조함
- 일반 코드 외에, 큐로 특유의 '스티어링 문서(글로벌 룰)' 제출 필요
- 프롬프트/명령(워크플로우)은 Markdown 형식 작성 및 제출
- 개발 일지(Devlog)에는 주요 결정, 개발 단계상의 문제, 해결 방식, 시간 흐름 등을 기록해야 함
- 이런 문서화를 통해 AI 코딩 과정 자체를 투명하게 공유할 것 요구

### 프로젝트 심사는 다섯 가지 명확한 기준을 토대로 이루어짐
- 심사 항목: **애플리케이션 완성도, 큐로 도구 활용 수준, 문서화 품질, 솔루션 독창성/혁신성, 데모 영상 전달력**
- 큐로 사용 방식(IDE/CLI 등)은 자유이며, 각 방식의 기능 활용도가 심사에 반영
- '문서화' 및 프롬프트/스티어링 문서도 심사에 적극 반영되며, 데브로그 등 창의적 접근도 가점 요인
- 데모 영상 요청의 주요 목적은 참가자의 열정, 프로젝트 전달력 및 사용자 경험 확인

### 참가 방식은 반드시 개인 단위로 제한되며, 1인 1작품 원칙이 명확히 적용됨
- 팀 단위 참가 불가, 오직 개인 계정만 인정
- 여러 프로젝트 진행은 가능하나, 취합 후 최종적으로 1개만 제출 가능
- 제출 방법은 깃허브 링크 또는 zip 파일 업로드형 중 자유 선택
- 참가자는 지원 랜딩 페이지를 통해, 제출 전 프로젝트를 충분히 다듬을 수 있음

### 참가 성공을 돕기 위한 큐로 퀵스타트 템플릿과 개발 워크플로우가 제공됨
- 공식 깃허브 템플릿 저장소를 현지 복제(clone)하여 프로젝트 베이스로 활용 가능
- 템플릿 내에는 큐로 툴 설치, 계정 설정, 프로젝트 초기화 등 단계별 안내가 포함됨
- '퀵스타트 명령'을 활용하면 참가자가 쉽게 스티어링 파일을 작성 및 세팅할 수 있음
- 여러 워크플로우 예시 명령이 탑재되어 있어 필요에 따라 자유롭게 변형, 확장 가능
- 템플릿의 설계 이유: 높은 진입장벽 해소 및 기초 작업 자동화

### 큐로 CLI 기준의 상세 설치 가이드와 기능 차별점 안내가 포함됨
- 가이드에는 운영체제별(윈도우, 맥OS, 리눅스) 설치법이 모두 나열됨
- CLI 중심으로 작성되었으나, IDE로도 유사하게 적용 가능
- 큐로 내 고급 기능(탄젠트 모드, 세션 간 지식 관리, Sub-agent, MCP 서버 등) 특징도 요약 설명됨
- 기존 AI 코딩 도우미들과 큐로의 차이, 특장점을 집중적으로 소개

### 전 과정은 영상 및 문서 가이드와 링크로 연결되어 있으며, 문의 채널도 별도로 마련됨
- 전체 퀵스타트 가이드 영상(상세 버전)과 랜딩 페이지가 별도 링크로 제공됨
- 참가 중 궁금한 사항은 커뮤니티 또는 Cole Medin 채널을 통해 직접 질의 가능
- 공식 안내 내 모든 지원 도구는 설명 내 또는 랜딩 페이지 내에 일괄 링크 제공됨

### 해커톤을 통한 도전, 성장, 수상 기회라는 분위기로 영상이 마무리됨
- 영상 호스트는 참가자 성장을 돕고 싶다는 취지와 함께 열정적 분위기를 유도
- 마지막까지 해커톤의 주요 규정, 도구, 지원 방안 등을 반복 강조
- 모든 안내 내용을 바탕으로 참가자들에게 성실한 준비와 창의적 도전을 권유함
