## *20일 컴퓨트 vs 7시간: SOTA(최첨단) 모델의 재정의* 핵심 요약

- 영상은 "최첨단(State-of-the-Art, SOTA) AI 모델"의 의미와 평가방식에 대한 오해 및 한계를 다룸
- 일반적 SOTA 판단법으로는 ‘퍼블릭 리더보드(top1 모델)’ 확인과 ‘내부 벤치마크’(수동 평가 또는 자동화 평가)가 있음
- 퍼블릭 리더보드는 각 리더보드마다 모델 순위가 상이하며, 구체적 사용 사례별로 편차가 매우 큼
- 동일 모델도 리더보드마다 순위·점수(예:Elo 점수)가 달라 신뢰가 어려움; 일부 모델은 한 리더보드에는 등장, 다른 곳엔 없음
- 다수 리더보드에서 가장 높은 순위 모델이 반드시 모든 유즈케이스에서 우수한 것도 아님
- 리더보드 순위는 적은 샘플(수천장) 기준이고, 실제 배포 상황에서는 수백만 트래픽으로 통계적 유의성이 부족함
- 내부 벤치마크 역시 소수 샘플에 대한 개인적 취향(수동 확인)과, 한정 메트릭에 의존할 경우 편향 발생 가능성 높음
- 메트릭 예시(CLIP score)는 데이터셋·지표마다 모델 순위가 미세하게 변동, 실제 목표과제와 맞는 메트릭 선정이 핵심임
- SOTA 선정 시 ‘품질’뿐 아니라 연산 시간·비용·에너지와 같은 ‘효율성’ 지표도 반드시 고려해야 함
- 품질이 약간 상승해도 효율성이 20배 이상 차이날 수 있으며, 파레토 프론트(Pareto front)를 활용하면 적합 모델 다수 도출 가능
- 모델 최적화 및 경량화(예: 양자화, 프루닝, 캐싱 등)로 품질·효율성 균형 추구하는 새로운 접근을 강조함

## 세부 요약 - 주제별 정리

### 퍼블릭 리더보드만 활용할 경우 순위와 실제 활용이 크게 괴리됨

- 많은 사람들이 SOTA 모델을 찾으려 할 때 첫 번째 방법으로 퍼블릭 리더보드(예: Design Arena, LM Arena, Artificial Analysis)를 참고함
- 리더보드에 따라 상위 1위 모델, 하위 모델의 순위가 크게 차이남 (예: Human 모델은 Artificial Analysis에서는 10위, Arena에서는 5위)
- 동일 모델이 한 리더보드에는 있으나 다른 곳엔 없어 정보수집에 혼선을 야기
- Elo 점수(모델간 상대적 강점 평가) 역시 리더보드마다 분포·스케일이 달라 모델간 비교 신뢰도 약화
- 여러 리더보드를 동시에 살펴보고, 순위가 비슷하게 나타나는 경우 대체로 여러 모델 성능이 비슷함을 의미할 수 있음
- 실무에서는 하나의 리더보드 결과만 신뢰할 경우 최적 모델을 놓칠 수 있음

### 유즈케이스 별 적합성 평가가 리더보드 전체 점수보다 중요함

- 대부분의 리더보드는 다양한 작업(예: 객체 제거, 배경 변경, 텍스트 편집 등)에 대해 평균점수를 제공
- 실제 서비스를 위한 모델 선정에서는 ‘특정 작업’(예: 객체 제거 특화)에 최적화된 리더보드 결과를 참고해야 함
- 예시: 객체 제거 항목에서 특정 모델이 1위, 텍스트 편집에선 또 다른 모델이 1위로, ChatGPT 이미지 등 어떤 모델도 모든 작업에서 일관되게 1위를 하지 않음
- 각 모델은 주로 어떤 데이터로 훈련되었는지에 따라 특화된 작업의 성능 차이가 나타남

### 리더보드 평가 샘플 수 및 통계적 의미에 한계가 존재함

- 대표 리더보드(예: Artificial Analysis)는 대부분 몇 천장의 이미지만으로 평가됨
- 실제 모델 배포 시에는 일일 수백만~수천만 사용량이 가능해, 리더보드의 샘플수가 실사용 환경을 크게 반영하지 못함
- ChatGPT 이미지 예시: 26,000개 이미지 평가를 위해 20일 연산 시간이 필요, 실제 운영 환경과 괴리 존재
- 리더보드 전적(예: 승률) 상위 모델도 100% 승률은 불가, 대부분 주요 모델이 대결의 40%는 패배함
- 인하우스 모델 트래픽 혹은 실제 API 로그 기반 내부 평가가 오히려 참고로 더 신뢰도 높음

### 내부 벤치마크 평가 또한 개인편향 및 샘플 편향의 위험을 안고 있음

- 이미지/비디오 생성 연구에서는 직접 몇 개의 프롬프트와 모델을 비교(수동 주관 평가)하는 게 흔함
- 실시간 현장투표 예시: 세 이미지에 대한 선호도 설문에서 참여자 간 선호도 편차 존재, 동일 인물도 다른 프롬프트에서 선택이 달라짐
- 표본 자체의 임의성, 참여자의 주관적 취향이 평가 결과를 뒤흔들 수 있음
- 따라서 단일 평가자가 소수 샘플만 보는 수동평가 결과를 그대로 신뢰할 수 없으며, 다양한 사람/충분한 샘플로 인적 평가 확장 필요

### 자동화 메트릭 중심 평가도 일관된 기준·메트릭 선정이 필수적임

- 자동화 평가에서 널리 쓰이는 메트릭(CLIP score) 역시, 데이터셋마다 혹은 메트릭 스케일 특성상 모델간 차이가 미미하게 나타나 실제 우열판단을 어렵게 만듦
- 세 가지 메트릭(예: CLIP score A, B, C)로 똑같은 8개 모델 평가 시, 순위 및 점수 차이가 미세해 해석이 미묘해짐
- 과제(유즈케이스)가 명확할 경우(예: 텍스트 렌더링) 그에 최적화된 메트릭 사용하는 것이 모델간 차별력 및 평가 신뢰도 모두 높아짐
- 적합한 평가 메트릭을 선정하지 않고 단순 자동벤치마크만 의존할 시 올바른 SOTA 판단이 불가

### 품질과 효율성(연산시간/비용/에너지)의 균형이 SOTA 선정의 핵심임

- 대규모 SOTA 모델(예: ChatGPT 이미지)은 리더보드 기준 26,000회 평가에 20일의 연산, 약 5,000달러, 556kWh(마라톤 400번 분량 에너지) 소요
- 대안 경량화 모델(예: Pruna의 Timed Generation Model)은 동일 26,000회 평가에 7시간, 265달러, 4회 마라톤 수준 에너지로 처리 가능
- 품질향상폭이 적어도, 연산 효율성 차이는 20배까지 발생 → SOTA 판단에 무조건적 품질 추구는 경제/실행 상의 문제가 됨
- 파레토 프론트(Pareto front) 시각화로 품질과 효율성(속도, 비용, 에너지 등) 관계 분석 필요

### 파레토 프론트를 활용하면 다양한 SOTA 모델군이 도출됨

- x축에 효율성(예: 레이턴시, 단일 이미지 생성 비용), y축에 품질 점수(Elo score)를 놓고 파레토 시각화
- SOTA는 한 모델이 아니라 각 효율성 구간마다 ‘다중 최상위 모델’이 존재함을 확인 가능
- 효율성(속도/비용)별, 유즈케이스별로 SOTA 모델군을 구분하면, 품질은 유사하나 효율성에서 수십 배 차이날 수 있음
- 예: 텍스트 렌더링 특화 파레토 프론트 분석 시, FLUX-2 및 FLUX2 Flex 모델이 해당 과제의 최적 해법으로 부상

### 벤치마킹 자체는 무의미하지 않으나 평가 방법론 혁신이 필수임

- “벤치마킹은 죽지 않았다(Benchmarking isn’t dead)”라는 메시지: 올바른 평가와 다각적 검증이 되면 유의미함
- 다양한 샘플, 실제 유즈케이스 조건, 효율성 평가 등 규칙 준수시 최적 ‘퍼포먼스 모델’ 다수를 선별 가능

### Pruna에서 효율적(빠르고 저비용) AI 모델을 제공하는 전략을 실천함

- Pruna(연사의 소속업체)는 다양한 퍼포먼스 모델(이미지/비디오 등)을 Endpoint형 서비스로 신속하게 제공
- 자체 모델: 이미지/비디오 생성 1~5초 내 실현, 최적 효율성 달성
- 오픈소스 패키지 제공: 사용자용 모델 압축/경량화 알고리즘, 효율성 연구 논문, 비용 최적화 사례를 커뮤니티에 공개
- 다양한 고객 니즈에 최적화된 모델 맞춤형 제공

### 모델 효율화 기법(양자화, 프루닝, 캐싱 등)을 적극 도입해야 함

- 오픈 Q&A에서 효율화 기법 답변: 모듈별 양자화, 불필요 컴포넌트 프루닝, 생성/디노이저 단계 캐싱 및 디스틸레이션 적용
- 예: 20~50 스텝 소요되는 이미징 모델을 캐싱 도입으로 4~20스텝으로 연산 최적화 가능
- 오픈소스 및 사내 전용 알고리즘 활용, 모델형 서비스의 효율성 대폭 향상

### SOTA 모델 선정은 다중 기준 평가와 철저한 목적 적합성 검증이 필요함

- “여러 샘플, 실제 유저 환경, 다중 벤치마크, 효율성” 네 가지 원칙을 강조
- 품질·효율성 균형 및 과제별 파레토 상 최적 모델 선택이 오늘날 SOTA 선정의 최신 방법론임
