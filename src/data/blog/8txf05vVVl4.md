## *코드 모드: 코드를 말하게 하라* 핵심 요약

- 본 강연은 Sunil Pai(Cloudflare, PartyKit 창립자)가 소개하는 "코드 모드"라는 새로운 AI-시스템 상호작용 패러다임에 대한 탐색이다.
- 기존의 툴 호출 방식(JSON 왕복) 대신 LLM에게 직접 실행 가능한 코드를(주로 JavaScript) 생성·실행하게 해 시스템 연동 효율을 극대화하는 접근법을 제안한다.
- 이 방식은 API 엔드포인트가 매우 많은 대규모 시스템(예: Cloudflare API 2,600개 엔드포인트)을 다룰 때 토큰 소비량을 99% 이상 감소시키고, 상호작용 과정을 즉각적·간결하게 만든다(1.2M → 1,000토큰).
- 코드 방식 도입 시 반복, 상태 유지, 순차·병렬처리 등 엔지니어 특유의 고급 프로그래밍 패턴을 LLM이 네이티브하게 활용할 수 있다.
- 실제 데모에서는 단일 코드 실행으로 여러 API 작업을 병렬·순차적으로 처리할 수 있음을 시연한다(예: DDoS 공격 중 모든 악성 IP 차단).
- 소프트웨어 상호작용의 전문성과 대중성 간 경계가 LLM과 코드모드 덕분에 크게 약화됨을 강조한다.
- 코드 실행의 안전성·감사가능성을 위해 '샌드박스(harness)' 및 능력 기반 보안(capability-based security) 아키텍처의 필요성을 설명한다.
- 코드를 통한 동적 워크플로우, 장기 실행, 사용자별 맞춤 UI 생성 등 새로운 소프트웨어 구현 지평을 제시한다.
- 개발자 경험(DevX)이 더 이상 인간만의 몫이 아니며, 앞으로는 코드를 생성하고 시스템과 상호작용하는 AI 에이전트들의 경험도 함께 설계해야 함을 강조한다.

## 세부 요약 - 주제별 정리

### LLM 활용 방식에서 툴 호출 한계가 명확히 드러남

- AI 애플리케이션을 설계할 때 흔히 "툴 콜링(툴 호출)"—즉, JSON을 통한 모델-시스템 간 왕복—방식이 선택된다.
- 이 방식은 소수 툴·짧은 실행 환경에서는 문제없으나, 수십·수백 개 외부 서비스와 연동하고 도구 수가 늘어나면 파편화·비효율로 인한 한계가 발생한다.
- Google 서비스, Jira, 위키 등 많은 툴이 컨텍스트를 점유하게 되면, 토큰 소모가 치솟고, 응답 속도는 느려지며, 조합 자체가 깨질 위험이 매우 커진다.
- 결국, 모델-시스템 간 반복/왕복 호출로 과도한 오버헤드가 발생한다.

### 코드 생성을 통한 "코드 모드" 접근이 급격한 효율 향상을 이끌다

- Cloudflare 내부에서는 JSON 툴 콜링 대신, LLM이 즉시 실행 가능한 JavaScript 코드를 "문자열"로 생성·실행하도록 환경을 바꿈.
- 이럴 경우, API 타입 시스템과 타입 체크·문법 검사 등 코드의 고유한 이점을 활용할 수 있다.
- LLM 자체가 수많은 코드 데이터를 이미 학습하여, 매우 다양한 코드 생성이 가능하다.
- 단발 호출로 복잡한 작업도 한 번에 실행할 수 있어, "실행-결과"의 왕복 수를 획기적으로 줄일 수 있다.

### 대규모 API 연동의 토큰 비용을 1/100로 줄이는 실전 사례 소개

- Cloudflare의 전체 API 엔드포인트는 약 2,600개에 달한다.
- 만약 각 엔드포인트마다 도구를 만들어 노출한다면, 첫 호출 시 무려 1.2백만~1.5백만 토큰이 필요하여 비현실적이다.
- 팀원 Matt Carey가 고안한 방식은 단 2개의 엔드포인트(검색, 실행)에만 "코드 문자열" 입력을 허용(입력은 전체 Open API 스펙), 필요한 기능만 추출하고 코드로 실행하는 구조다.
- 이로써 토큰 소모량을 1,000토큰 수준(99.9% 감소)으로 대폭 절감함을 실제로 증명했다.

### "코드 모드" 실시간 데모를 통해 복잡한 API 호출이 한 번에 처리됨을 보여줌

- 실제 강연 데모에서는 "나의 워커들을 모두 나열해줘"와 같은 자연어 요청에 대해, AI가 즉시 JavaScript 코드를 생성해 단발 실행한다.
- 기존 방식이었다면 여러 번의 API 호출·왕복이 필요했으나, 이 방식은 한 번의 코드 실행으로 모든 작업을 완료한다.
- 데모 중에는 실제로 자바스크립트 에러와 페이지네이션 처리 등의 구체적 실행 흐름을 보여주며, 코드는 API 표면에 바로 접속하여 실질적 업무(예: DDoS 방어 자동화)를 신속히 처리한다.

### LLM과 코드 모드가 프로그래머·비프로그래머 구분의 벽을 허무는 계기가 됨

- 전통적으로 프로그래머는 문제를 받으면 IDE에서 코드·스크립트를 작성, 자동화한다(예: 사진 200장 자동 분류·이름 변경).
- 비프로그래머는 GUI 앱(월 $7)이나 타인의 도움에 의존해야 했다.
- LLM과 코드 모드 보급으로, 누구나 "날짜와 위치로 파일명 바꿔줘"와 같은 문장으로 즉석 코드 생성 및 실행이 가능해졌으며, 프로그래밍 전문성과 일상적 요구의 경계가 크게 약화된다.
- 단, 실행은 기본적으로 안전하게(SAFELY) 이뤄져야 한다는 점을 강조한다.

### 코드 기반 상호작용 패러다임이 소프트웨어 구조에 혁신을 일으킴을 실험적으로 보이다

- 예시: Cloudflare Workers의 창시자인 Kenton이 자신만의 코드 환경에서 캔버스 UI를 만들고, LLM에게 "틱택토 게임을 플레이 해달라"고 요청.
- LLM이 기존 코드 없이 캔버스 상의 드로잉(선, X/O 등) 상태만 분석해 바로 게임 룰을 유추하고 대결에 나섰음(결국 Kenton 승리, Opus 모델이 일부러(?) 져줌).
- 이처럼 LLM이 코드를 "생성"하는 단계를 넘어, 시스템 상태머신을 "직접 조작"하는 새로운 상호작용 방식을 시연.
- 이는 "프로그램 생성"이 아닌 "상태머신 거주(live state interaction)" 현상이라고 명명.

### 코드 실행의 안전성과 감사성을 위한 ‘샌드박스’(Harness)·능력 기반 보안 구조가 필수적임

- 최근 수개월간 많은 업체들이 “코드 에이전트는 범용 컴퓨팅 머신”이라는 판단 아래, 별도 샌드박스(하네스) 실행 환경 구축.
- Cloudflare에서는 V8 Isolate 기반 동적 워커(dynamic worker)로, 처음엔 무능(무권한) 상태에서 서서히 필요한 API 등만 선별적으로 노출.
- 외부 네트워크 연결(feth 등)은 기본적으로 봉쇄하고, 필요시 엄격히 관리하도록 권장.
- 각 코드 실행은 언제, 누가, 어떤 거래/변경을 했는지(예: $230만 원 거래 등) 완벽히 추적·관찰 가능해야 하며, 신속한 실행·감사 가능성이 중시됨.

### 샌드박스 종류는 다양할 수 있으나, 실행 안전성과 기능 노출의 주도성 확보가 본질임

- V8 Isolate, WebAssembly, Custom JS 인터프리터 등 다양한 엔진이 가능한데, 핵심은 "임의 코드 실행+필요 능력만 on-demand 노출+최소 권한" 을 보장하는 구조.
- 이런 환경에서는 단일 코드 실행뿐 아니라, 장기 워크플로우(수 일/수 년 실행), 사용자별 상태 유지, 완전 맞춤형 UI 자동 생성도 가능해진다.

### AI 코드 에이전트의 활동 범위가 맞춤 UI·이커머스 등으로 크게 확대됨

- 이커머스 등에서는 "이번 주문의 지연 이유 알려줘", "거래 반품 후 유사상품 추천" 등 요청에 맞게 LLM이 맞춤형 프로그램/워크플로우를 상황별 즉석 생성–실행 가능.
- 기존에는 UI가 불특정 다수에게 맞춰져 무난히 통일된 UX만 제공했다면, 코드 모드 도입으로 각 사용자에 대한 울트라 맞춤화(dynamic UI, 기능 노출 등)이 실현 가능해짐.
- 시스템 설계자는 AI 에이전트가 주고받을 프로그램 상호작용의 "개발자 경험"도 중시할 필요.

### 미래 소프트웨어에서 사용자는 인간만이 아니라 "코드 생성형 AI 에이전트"도 대상임

- 앞으로 대규모 서비스의 실질적 상호작용 주체는 코드를 생성·실행하는 소형 자동화 에이전트(로봇)가 될 가능성이 크다.
- 따라서 문서/가이드/탐색 UX도 사람이 아닌, 코드 에이전트(타입, 에러 처리, 상태 등)가 원활히 활용할 수 있도록 설계해야 함을 강조(마크다운 문서, 검색성, 에이전트 전용 에러 메시징 등).
- 언어(Python, JS, WASM, Lisp 등)은 중요하지 않고, 이벤트·샌드박싱·능력 기반 보안·신속 가동성을 강화하는 것이 본질이라 설명.

### "코드를 말하게 하라"는 패러다임의 전환이 개발자와 소프트웨어 미래 양쪽에 큰 변화를 예고함

- 지금까지는 프로그래머만이 무한한 자유와 시스템 접근 권한을 누린 반면, 타인은 버튼과 폼에 눌려왔다.
- 이제, 코드 모드와 LLM의 확산으로 이 경계선이 사라지며 미래 소프트웨어는 "코드 생성·실행·상호작용" 중심으로 재편될 것으로 보임.
- "코드가 직접 말한다"는 선언으로, 더 많은 실험과 논의가 필요함을 청중에게 제안하며 강연을 마무리한다.
