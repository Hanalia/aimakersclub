## *앱의 종말 — Kitze, Sizzy.co* 핵심 요약

- 발표자는 어린 시절부터 생산성 도구와 할일관리 앱에 집착해왔으며, 완벽한 솔루션을 찾지 못해 직접 여러 앱을 개발
- 앱의 진화 과정(텍스트 기반, 자동화, 할일 우선순위화, 습관/일정 통합 등)을 경험하며 ‘완전한 라이프 OS’를 꿈꿔옴
- 기성 할일 앱과 다양한 SaaS의 한계(플랫폼·기능 단절, 유저 인터페이스 불편, 지속적 사용 어려움)를 지적
- 오픈AI/클라우드 기반 AI 및 플러그인 등장 시 “이제 앱은 끝났다”는 파괴적 변화 감지, 하지만 실제론 아직 한계 많음
- 클라우드 에이전트(예: OpenClaw) 및 자가호스팅 에이전트 실험에 전념, 데이터 소유와 개별화된 생산성 환경 강조
- 에이전트 기반 개인비서·조직 시스템을 구축했으나, 실사용에서는 UI/기억/자동화 등 여러 문제점 노출
- 생산성 도구/에이전트의 미래는 “모두가 자신의 업무 대부분을 AI에 위임”하는 방향이라 전망
- 기존 앱 중심 워크플로는 사라지고, 대부분의 사용자는 앱 없이, AI-OS에서 자연어로 각종 작업을 처리하게 될 것
- 애플 등 주류 OS 제공사가 로컬 기반 AI·툴 연동으로 시장에서 강력한 표준을 만들 가능성 시사
- 실험적 도구 개발, 다양한 커뮤니티/협업(Discord, Tinkerer Club 등), 개인화/특수화된 멀티에이전트 활용 등이 언급됨

## 세부 요약 - 주제별 정리

### 생산성 앱과 할일 관리에 평생 집착한 경험에서 시작함

- 발표자는 만 10세 때부터 할일 관리에 관심을 가짐(어린 시절 메모장에 ‘스트링 주스 먹기’ 등 항목 체크)
- 할일 관리 앱(To-do app)에 만족하지 못해, 15년 전부터 text 파일 기반 직접 시스템 개발
- Android Tasker 등 자동화 도구로 ‘와이파이 접속 시’ 등 다양한 상황 기반 리마인더 설정 시도
- Google Assistant와 IFTTT 연동·자동 기록 등도 이용하여 생각 정리와 처리 자동화 실험
- 본질적으로 원하는 것은 ‘할일 앱’이 아니라 자기 삶 전체를 관리·보조하는 ‘Life OS’임을 인식

### 10년 넘는 기간 동안 다양한 자체 생산성 툴을 개발하는 과정

- 2017년 ‘toodo’: 태그별(예: health, crisis 등) 우선순위 자동 합산으로 주요 할일 상위노출 기능 추가
- ‘better’ 개발 및 리브랜딩: 할일·습관·캘린더 일정 등 통합, 미니 OS 지향
- 하지만 주의력 산만(ADHD)과 흥미 전환으로 다양한 앱 개발 후에도 완결적 앱 출시에 어려움 경험
- 앱마다 웹/Android/iOS, 구독제/프리미엄 등 다양한 한계 겪으면서, “하나의 앱에 모든 기능 융합” 시도
- 2022년 ‘Benji’(자신의 반려견 이름): 궁극의 생산성 앱 목표, but 마케팅 미진·기능 추가 욕심으로 여전히 출시 전 상태

### 사유 데이터와 개인화된 OS 구축에 대한 갈증 (Benji OS 및 Benji Phone 상상)

- 분산된 서비스(다양한 웹/앱/OS)에 실망, “하나로 통합된 Life OS”을 갈망
- 사용자 데이터 입력의 마찰(폼(Forms) 작성 등)이 생산성 앱 지속 사용의 가장 큰 걸림돌임을 여러 차례 언급
- 실제로 Benji 등 생산성 시스템에 몰입하는 기간(월 단위)과 완전히 무시하는 기간이 반복
- 장기 목표로 “Benji Phone”, “Benji OS” 등 직접 하드웨어 및 소프트웨어까지 확장된 관리 환경을 구상

### ChatGPT, 플러그인, 클라우드 에이전트 열풍이 ‘앱 패러다임’의 종말을 예고했으나 실제론 한계 부각

- ChatGPT 및 플러그인 등장 시 ‘이제 SaaS 모두 사라질 것’이라며 시장 변화 촉각
- 2023년 이전까지는 JSON 출력조차 강제 프롬프트·파싱으로 힘들게 구현(‘please don’t write any markdown!’ 등 우스개)
- Benji에 실시간 음성입력→API 호출 기능 추가, 캘린더/할일 자동 갱신, 트위터 등에서 화제 모음
- 하지만 수많은 멀티-기능(본인은 60개) 보유에도 단일 간단한 기능(사진 분석 통한 식사칼로리 기록 등)만 잘 파는 스타트업이 수백만 달러 매출 달성 사례와 비교, 개발 과잉의 한계 실감

### OpenClaw, 클라우드봇 등 신개념 멀티에이전트 시스템 실험과 자가 데이터 호스팅 흐름

- Claude Code, tool calls, 개인 스킬을 추가하며 ‘본인만의 어시스턴트’ 구축 시도
- 하지만 AI툴·MCP 등 서비스마다 환경·UI 제약, 매우 높은 진입장벽 실감
- 클라우드봇(WhatsApp/Telegram 연동) 등장에 열광, “이게 진짜 Cloud Setup의 진화”라 평가
- OpenClaw 커뮤니티(‘lobster suit’ 등 밈이 된 100명 미만의 초기 멤버), Discord 활용, 본인도 로고/공식 밈 제작에 참여
- Google Drive/iCloud 등 클라우드 데이터를 전부 NAS·로컬로 이전(에이전트가 접근 가능하도록)

### 실제 에이전트 활용 환경의 혼란, 불안정, 그리고 생산성 앱/에이전트의 구조적 한계

- OpenClaw 등 에이전트 활용 후 “완전히 통합된 삶의 OS”를 상상하지만 실상은 사용성, 자동화, 기억 등에서 미흡
- 다양한 앱·플랫폼(5개의 Discord, 다수 채널과 스레드, 포럼 등)로 오히려 시스템이 복잡·혼란해짐
- 커뮤니티도 처음엔 활발했지만 점차 참가자가 줄며 활동성 감소, “OpenClaw Anonymous”라 자조
- 가장 큰 이유는 신뢰성 부족(크론 작업, 멀티에이전트 연동, 에이전트 간 소통 미흡, 맥락 잊어버리기 등)
- 디스코드/텔레그램 등 채팅 중심 플랫폼의 UX 한계 지적: 삶의 OS에 최적화된 UI가 아님

### 에이전트의 ‘대화 느낌’, 피상적 UX의 한계 및 개인화의 욕구 고조

- OpenClaw 등은 채팅 인터페이스 제한적, 반복적으로 ‘지시 작업 수행 유무’만 확인하는 대화 흐름 발생
- “마치 오트밀 상자(box of oats)와 이야기하는 느낌”이라 풍자, 실제 집행력 및 업무 자동화 미흡
- 대화형 비서의 통찰력·능동적 피드백 부족(작업 완료 여부 알림, 맥락 기억 등)

### 개인화된 자체 도구(Wolffer) 개발 시도의 이유와 장점/한계 정리

- 결국 직접 실험적 AI에이전트 UI·플랫폼(Wolffer) 개발에 착수(일반 공개 의도 없음, 순수 실험)
- 단점:
    - 외부 채널연동 불가, 매우 한정적(플러그인 없음, 확장성 부족, 메모리 시스템 미비)
    - ADHD식 개발로 완결성 부족, OpenAI 등 자금/브랜딩 전무
- 장점:
    - 대화·작업의 예측가능성, 멀티에이전트 오케스트레이션에 최적화된 UI 구현
    - ‘메모리’를 믿지 않고, 트리형 주제(프로젝트/업무/고객지원 등)별 상위 맥락을 프로토콜화해 직접 전달
    - 워크스페이스 전환·툴콜 표시/제어, 버튼 기반 작업, 크론의 예측가능성, 관리 UI 등 본인에게 최적
    - 지식베이스 문서, 패스워드, 스킬 등도 채팅에 인라인으로 자유롭게 언급·결합 가능

### 에이전트·AI 생산성 환경의 미래는 ‘앱이 사라지고, AI가 인간을 프롬프트’하는 방향으로 전환될 것

- 오늘날 ‘사용자가 AI에 명령’을 내리지만, 미래엔 “AI가 99%의 작업 위임·처리, 인간에게만 의사결정 질의”로 전환될 것
- 앱 UI는 실시간 필요에 따라 생성, 사용자는 자연스러운 결제/입력/업무처리에만 집중
- “할일을 하려면 앱이 필요하다”는 인식 대신, OS 자체에서 즉시 각종 업무·욕구 해결 예상
- 소프트웨어 자체는 영상/음악/특수목적으로 일부 전문가만 이용, 대다수는 AI-OS에서 대화·응답만 충분

### 대중화의 열쇠는 ‘로컬 에이전트’와 주류 OS(예: 애플)의 툴 통합 기능

- 애플 등 기존 OS제공자는 ‘로컬 모델+툴 연동’ 발전을 통해 신형 시장 표준 지배 가능성
- 최근 Google Pixel 등도 앱 백그라운드 실행, 자동화 명령, 주문 등 이미 일부 실현
- 대중은 클라우드·자가호스팅 대신, 로컬 데이터·모델 기반 에이전트(OS 내장형) 선호 확산될 전망

### 결론: 에이전트 세상은 아직 미완성이나, 장차 대다수 사용자가 ‘앱 없는 시대’로 진입할 것임

- 현재는 개인화 에이전트·AI 도구 마다 한계와 학습곡선이 명확, ‘실생활 조직화’에 미흡
- 하지만 혁신은 지속, 점차 “인간의 자연스러운 업무·생활 조직이 직접적 AI 기반 시스템으로 이전”
- 실험적 커뮤니티 및 개발자들의 노력, 자체 도구 확장 등은 미래 변화의 전조임을 시사하며 마무리
