## *Any-to-Any: 네이티브 멀티모달 에이전트 구축하기* 핵심 요약

- 본 영상은 Google DeepMind의 Patrick Löber가 Gemini API를 활용해 'Any-to-Any' 멀티모달 네이티브 에이전트 구축 과정을 시연하며, 핵심 기능과 실제 사용 예제를 소개함
- Gemini는 텍스트, 코드, 이미지, 오디오, 비디오, URL, Google 검색 등 다양한 입력을 네이티브로 처리 가능하며, 텍스트뿐만 아니라 이미지, 음성, 비디오, 함수 호출, 코드도 생성 가능
- 하지만 모든 기능이 단일 멀티모달 모델로 구현된 것은 아니며, 실제로는 Gemini 3(텍스트 출력), Nano Banana(이미지/음성 생성) 등 각각 특화된 모델을 조합하여 사용
- Gemma 모델은 텍스트, 이미지, 비디오(소형 모델은 오디오 포함) 입력을 지원하여 로컬 환경에서도 멀티모달 에이전트 구축이 가능
- 실습 예제로 '노트북 LM'과 유사한 에이전트(여러 자료 요약, 팟캐스트, 인포그래픽 생성 등)를 구성, 사용자는 PDF/이미지/동영상/음성 메모 등 다양한 자료를 입력하여 종합적 요약 결과를 얻음
- Gemini API는 언어별 SDK 제공, 무료 API 키를 통해 PDF, 동영상, MP3 등 다양한 파일을 업로드하고, 간단한 코드로 다양한 멀티모달 처리가 가능
- 오디오(1분=1,920토큰, 최대 100만 토큰=9시간), 비디오(약 1시간) 등 긴 입력도 처리 가능하며, 타임스탬프 설정, URL 입력, 컨텍스트 캐싱 등 다양한 편의 기능 제공
- 함수 호출(Function/Tool Calling) 기능으로, 필요에 따라 이미지 생성, 음성 생성 등 특화 모델 호출 및 복잡한 에이전트 루프 구축이 가능
- Nano Banana 등 네이티브 생성 모델은 Gemini의 세계 지식 활용 및 멀티모달 연계 처리가 가능해 지도 교정, 방정식 자동화, 다양한 언어/억양의 TTS 등 고급 활용이 가능
- 실시간 오디오 인-아웃이 가능한 Live API(Gemini 3.1 Flash Live) 소개로 자연스러운 인터랙션 시연, 멀티모달 임베딩 등 최신 연구 결과 및 향후 활용 방향 제시

## 세부 요약 - 주제별 정리

### Gemini API는 텍스트 이외의 다양한 입력과 출력을 네이티브로 지원함을 강조함

- Gemini API는 단순한 텍스트 인식 모델이 아니며, 여러 입력 모달리티를 자연스럽게 다룸
- 입력 지원 모달리티: 텍스트, 코드, 이미지, 오디오, 비디오, URL, Google 검색 등
- 출력 지원 모달리티: 텍스트, 이미지 생성, 음성 생성, 비디오 생성, 함수 호출, 코드 생성 등
- 예시로 텍스트 분석 뿐만 아니라 직접 이미지나 음성 결과물을 생성할 수 있음을 시연
- 다양한 사용 사례(Use Cases)로 확장 가능하다고 언급

### 실제로는 각 기능별로 특화된 모델이 존재하며 Gemini의 비전과 현재 위치를 설명함

- 모든 기능이 단일 멀티모달 모델로 구현된 것은 아님을 명확히 함
- 현재는 Gemini 3가 메인 모델로 텍스트 중심 출력, 이미지·음성·비디오 등은 별도 특화 모델 활용
- 대표적 특화 생성 모델: Nano Banana(이미지·음성 생성), 등
- Gemini와 Gemma 모델군, Gemma는 텍스트/이미지/비디오(및 소형 모델에서는 오디오) 입력 지원으로 로컬 구현 가능성 설명

### Gemini 기반 멀티모달 이해 및 출력 구조(에이전틱 루프)를 구체적으로 설명함

- 멀티모달 이해 단계(Phase 1)와 멀티모달 생성 및 반복적 에이전트 루프(Phase 2)로 구조화
- 에이전트가 직접 “어떤 결과물을 생성할지” 결정(Workflow 하드코딩이 아니라 Agent Reasoning 방식)
- Gemini가 추론/사고 담당 – 필요에 따라 특화 생성 모델 호출(Function/Tool Call)
- 루프 내에서 추가 Asset 필요 여부를 스스로 판단하여 반복, 최종적으로 텍스트·음성·인포그래픽 등 통합 결과물 도출
- “노트북 LM”과 유사한 구조: 유저가 여러 형식의 자료 입력 → 멀티모달 통합 요약 및 각종 부가 자료 생성

### 다양한 입력(텍스트, PDF, 오디오, 비디오 등)과 API/SDK 활용을 예시 코드와 함께 상세히 안내함

- Google AI SDK 설치, 무료 API 키 획득 과정 간략 설명
- 다양한 언어(SDK), 파일 업로드 및 인라인 데이터 입력 가능
- 예시: PDF, 동영상, MP3 업로드 및 한 번에 통합 분석 가능
- inline data(소형 파일), 파일 API(대형 파일), 직접 URL(Youtube 등 포함) 지원
- Gemini API의 "API Skill" 활용 – 복잡한 코드 암기 없이 간편하게 에이전트 연결 가능

### 오디오 및 비디오 입력 처리 한계와 활용법, 토큰 정보 등을 구체적으로 안내함

- 오디오 1분 = 1,920 토큰, Gemini 토큰 한도 1,000,000 → 9시간 분량 입력 가능
- 비디오 입력은 약 1시간(토큰 기준), 설정 바꿔 더 긴 입력 가능
- 타임스탬프 지정 등 다양한 부분적 입력 분석 가능(예: 5분~15분 범위만 분석)
- 컨텍스트 캐싱 기능 → 대용량 파일 반복 질의시 90% 비용 절감
- 긴 파일 효율적 처리 및 비용 효율성 강조

### 멀티모달 이해(요약, 전사 등) 활용 팁 및 시연 코드 예시를 제공함

- 오디오 파일 전사(transcribe) 기능: Flash, Flashlight 모델에서 프롬프트에 “transcript” 지정 시 효과적
- 파일마다 내용 설명 정보를 함께 입력하여 요약/분석 명확성 증대 가능
- Gemini를 통해 여러 모달리티 자료를 복합적으로 요약·요약본 생성 가능

### 멀티모달 생성 단계에서는 에이전트-함수호출을 통한 이미지, 음성 생성 프로세스를 자세히 다룸

- Gemini 3.5 flash image preview(Nano Banana 2) 활용 이미지 생성 실습
- 인포그래픽, 슬라이드 형태 시각자료 자동 생성 – “create infographic” 프롬프트 활용
- 텍스트-투-스피치(TTS): Gemini 2.5 기반 음성합성 모델, 2화자 오디오 등 다양한 형식 지원
- 실습 예시: 학습 주제를 2분 만에 음성 오디오로 요약해주는 팟캐스트 형식 시연

### 함수(도구) 호출 로직 및 프롬프트 설계 예시를 구체적으로 안내함

- 함수 선언(이름, 설명, 파라미터 등) 방식 명시
- 이미지 생성, 음성 생성 등 각 함수에 대해 기능별 구분 필요
- 프롬프트 예시: 에이전트 역할, 생성할 자료 유형 안내 → 어떤 컨셉이 시각자료가 필요한지, 어떤 부분이 음성 요약이 필요한지 직접 결정 후, 해당 함수 호출

### 네이티브 생성 모델의 실제 활용 예시와 차별점을 상세히 보여줌

- Nano Banana(이미지 생성): 지도에 화살표를 표시하고, 프롬프트에 따라 ‘골든게이트 브릿지’ 이미지 자동 생성
- 교육용 활용 케이스: 수학 숙제 직접 교정, 수식 포함 이미지 자동 생성, 이미지 내 코드 자동생성 등 실제 사례 소개
- TTS(음성 합성) 모델: 다국어, 다양한 억양·발음(예: 영국, 바이에른 억양) 지원 및 시연
- 복수 언어, 억양 연출력도 데모를 통해 입증

### 실시간 오디오 인-아웃 지원 Live API 및 멀티모달 임베딩 등 최신 기능을 간략히 소개함

- Live API(Gemini 3.1 Flash Live): 오디오 입력-오디오 출력이 미리 파이프라인화된 모델 구조(즉, 인풋 오디오→아웃풋 오디오)
- 자연스럽고 대화형 음성 인터페이스, 단일 모델 구조(복수 모델 연결 단계 필요 없음)
- 웹(https://ai.studio/live)에서 직접 체험 가능, 실시간 예시 데모(착용자 외모, 옷차림 인지 등)
- 멀티모달 임베딩 소개 – 모든 모달리티를 통합된 벡터 공간으로 매핑(예: 멀티모달 검색 등 신기능)
- 로컬 환경(Gemma 4)에서도 멀티모달 이해/생성 가능성 언급
- 영상 마무리: 기존 패턴은 타 분야에도 즉시 전이·확장 가능, 개발자로서 직접 구현해볼 것을 독려
