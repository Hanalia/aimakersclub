## **Omnigent: 모든 코딩 에이전트를 위한 새로운 메타 하니스** 핵심 요약

- **Omni Agent**는 최근 주말에 오픈소스로 공개된 강력하고 무료 메타 하니스(meta-harness) 툴로, 여러 AI코딩 에이전트의 장점을 결합하여 복잡한 코딩 워크플로우를 손쉽게 실행할 수 있게 지원함
- 메타 하니스란 코드 구현(예: Claude Code)과 코드 리뷰(예: Codex) 등 다양한 AI 코딩 에이전트를 동시에 관리·오케스트레이션 할 수 있는 상위 계층의 시스템
- 최신 추세는 단일 LLM(대형 언어 모델)이나 단일 하니스에 의존하지 않고, 다양한 AI 코딩 보조 도구의 강점을 조합하는 형태임
- Omni Agent를 사용하면 GitHub 레포지토리 링크만 있으면 단일 커맨드로 몇 분 안에 설치 가능하며, 로그인 인증도 별도로 재설정할 필요 없음
- DataBricks의 CTO가 주도하여 개발하였고, 사내 엔지니어링에 광범위하게 활용 중임을 언급
- Polly, Debbie와 같은 예제 오케스트레이터 에이전트가 미리 포함되어 있고, 사용자가 쉽게 커스텀 에이전트 및 오케스트레이터도 구축할 수 있음
- 다양한 접근 방식을 지원: 네이티브 앱, REST API, 터미널 CLI, 웹 UI 등
- 워크트리, 병렬 개발, 다양한 모델/서비스와의 연동, 세션 공유, 정책・가드레일, 샌드박스 기능 등 실전 개발에 필요한 다양한 기능 내장
- 코드 리뷰는 구현 에이전트와 분리된 별도의 세션에서 진행하도록 쉽게 설계되어 LLM 편향 최소화 및 품질 향상 가능
- Omnigent를 통해 여러 기기(폰+데스크톱)에서 동시에 동일 세션 협업까지 가능

## 세부 요약 - 주제별 정리

### 메타 하니스는 AI 코딩의 신뢰성과 확장성을 획기적으로 높임

- AI 코딩 분야에서 올해 가장 중요한 교훈 중 하나는 LLM(모델) 만큼이나 하니스(작업 자동화와 워크플로우 운영 도구)가 중요함을 깨달은 것임
- 최고의 LLM(예: Fable 5)이 벤(ban) 당하면서, 항상 최고 모델에만 의존할 수 없는 현실이 부각됨
- 하니스를 개선해 시스템 전체 신뢰성과 확장성을 높여야 한다는 필요성이 높아짐
- 하니스는 시스템 프롬프트, 도구 선택, 스킬, 워크플로우, 룰 등 단일 코딩 에이전트를 더욱 강력하게 만드는 요소의 집합체임
- 메타 하니스는 이보다 한 단계 위: 여러 AI 코딩 도우미가 하나의 큰 작업을 효과적으로 분배·연결하게 해줌
- Cole Medin은 자신의 프로젝트(Archon)도 Omni Agent의 메타 하니스 개념이 반영될 것이라 밝힘

### Omni Agent는 모든 AI 코딩 에이전트의 오케스트레이션을 쉽게 구현함

- 설치 방법이 대단히 간단: GitHub 레포지토리 링크를 받아 커맨드 한 줄이면 완료됨
- 별도의 인증 과정을 거치지 않아도, 이미 CLI에 등록된 인증 정보를 자동 활용
- Claude, Codex, Pi 등 주요 AI 코딩 모델들과 바로 연동 가능
- 여러 기기에서 동일 환경으로 작업 가능, 세션 공유 기능이 내장되어 있음
- 오픈소스로 공개되어 누구나 자유롭게 사용 및 커스터마이즈 가능

### DataBricks CTO 주도, 실제 사내 현업 환경에서도 성공적으로 활용 중임

- Omni Agent는 대형 빅데이터 기업 DataBricks에서 자체적으로 내부 실무에서 활발히 활용하는 검증된 도구임
- 회사 CTO가 주도적으로 프로젝트를 이끌었으며, 개발 규모 및 신뢰성에서 차별화
- 내부에서의 “dogfooding” 사례가 소개되며, 매우 긍정적인 평가를 받고 있음

### Polly 등 내장 오케스트레이터 에이전트로 복합적인 워크플로우가 가능함

- Omni Agent에는 예제 오케스트레이터인 Polly(폴리), Debbie(데비)가 내장되어 있음
- Polly는 다양한 코드 구현(Claude)과 코드 리뷰용(Codex) 에이전트 사이에서 워크플로우를 분배함
- 복수의 harness(하니스)를 넘나들면서 로컬 개발까지 지원 (예: Pi+Olama)
- 작업 디렉토리, 워크트리 설정 등 병렬 개발에 필수적인 기능 제공

### 워크플로우 예시: 코드 구현은 Claude, 코드는 Codex로 리뷰하는 자동화 흐름

- 사용자는 GitHub 이슈나 자유 형식 텍스트로 요청 작성 가능
- Polly가 “구현은 Claude, 검토는 Codex”에게 자동 할당
- 각 에이전트는 독립된 세션에서 작업하고, 프로세스가 샌드박스에서 분리 실행 가능
- 인증 정보를 재설정하지 않고, 기존 CLI 인증을 그대로 활용해 10분 내로 워크플로우 실행 가능

### 분리된 세션을 통한 코드 구현-리뷰는 LLM 편향을 줄이고 품질을 높임

- 구현과 리뷰를 동일한 에이전트가 처리 시, LLM의 편향(바이어스)이 누적될 수 있음
- 코드 리뷰는 따로 분리된 에이전트(예: Codex) 세션에서 처리하는 것이 권장됨
- 실제 워크플로우 예제로 이러한 구성이 Omni Agent에서 정말 간단하게 구현됨을 시연

### 높은 수준의 커스터마이징과 정책・가드레일 기능 통합

- 각 오케스트레이터(Polly 등)는 Config, Skills, Agents 세 가지 요소로 구성
- 에이전트/오케스트레이터 설정은 시스템 프롬프트, 실행자(executor), 샌드박싱, 가드레일(guardrails), 도구 접근권한, 스킬(workflow) 등으로 세분화
- 예시: Claude에게 “강제 push(–force)” 명령 실행 시 사용자 승인(human-in-the-loop)이 필요하도록 커스텀 가드레일 설정 가능
- 가드레일 정책은 파이썬 코드로 간단히 정의, 에이전트/오케스트레이터 설정 옆에 바로 위치시킴

### AI 코딩 보조 에이전트 자체도 Omni Agent에서 자동 구축할 수 있음

- Polly, Debbie 같은 예제를 AI 코드 에이전트에게 분석–참조하도록 요청하면, 커스텀 에이전트 자동 생성 가능
- 사용 예시: Claude 기반 시스템 프롬프트, 가드레일 등 간단한 템플릿 기반으로 에이전트 신속 생성
- 커스터마이즈된 정책/가드레일도 에이전트 별로 유연하게 반영 가능

### Debbie 오케스트레이터는 에이전트 간 토론/합의형 워크플로우도 지원

- Debbie 오케스트레이터를 이용해 Claude, GPT 등 다른 AI 코딩 에이전트가 각각 다른 입장에서 질문에 대해 토론–피드백–합의 결론 도출 가능
- 토큰 사용량, reasonaing 과정 등 각 에이전트의 추론 과정을 별도로 모니터링 가능
- Debbie가 두 라운드에 걸친 토론 후 결론을 종합해 사용자에게 전달

### Omni Agent는 여러 기기·환경에서의 동시 협업도 지원함

- 동일 Wi-Fi 네트워크에서 PC와 모바일에서 같은 세션과 대화 참여 가능
- 전화기로 메시지 전송→바로 데스크톱 화면에서 실시간 반영
- 서버에 호스팅하면 인터넷을 통한 원격 협업 및 글로벌 협업 환경도 구현 가능
- README에 멀티기기 협업 관련 별도 설정 안내가 제공되고 설치·설정이 극히 간편함

### Omni Agent는 다양한 응용과 커스터마이즈가 가능해 메타 하니스의 미래상을 제시함

- 단일 LLM/모델/프로바이더에 의존하지 않고, 여러 에이전트의 장점 최대한 활용–조합
- 코드 구현, 리뷰, 정책, 토론 등 복잡한 워크플로우조차 간편하게 통합
- harness engineering(하니스 공학)과 AI 코딩 워크플로우 혁신의 새로운 패러다임임을 적극적으로 제안함
