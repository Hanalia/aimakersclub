## *왜 TTS(텍스트-음성 변환) 모델이 최근 LLM과 비슷해졌는가 — Samuel Humeau, Mistral* 핵심 요약

- **미스트랄(Mistral)의 AI 연구원 Samuel Humeau가 공개한 자사 최초의 오픈소스 텍스트-음성 변환(TTS) 모델의 특징과 기술적 구조를 상세하게 설명**
- **텍스트-음성 변환(TTS)의 주요 활용처는 에이전트를 통한 대화형 인터페이스로, 실시간 응답과 짧은 레이턴시(지연 시간)가 핵심 요건임**
- **오디오 스트리밍 구조를 구현하여, 전체 음성 데이터 생성이 완료되기 전부터 첫 오디오 패킷을 즉시 재생함으로써 레이턴시를 최소화**
- **TTS 모델은 소수의 음성 샘플(수 초 분량)만으로 실시간으로 음성을 클론(복제)할 수 있으며, 언어마다 고유의 억양과 발음을 정확히 재현**
- **프레임 단위(80밀리초)로 오디오를 잘라 토큰화하고, 약 37개의 토큰이 한 프레임을 구성하도록 데이터 압축 및 처리**
- **주요 구조는 오토리그래시브(autoregressive) 디코더 백본과 토큰 기반 시퀀스 모델링이 결합된 형태이며, 대형 LLM(약 40억 파라미터 수준) 설계와 유사**
- **모델의 훈련은 오디오 재구성과 정보 압축(재구성 손실 및 적대적 손실 포함)을 통해 필수적인 특징만을 추출하도록 설계**
- **오픈소스 배포의 경우, 음성 클로닝용 인코더 부분은 제공하지 않으며, 미스트랄에서 인증된 오픈 음성만 사용 가능(보안상의 이유)**
- **실시간 텍스트 입력 스트림과 오디오 생성 동시 진행에 대해 다양한 아키텍처 패턴(오디오/텍스트 인터리브, 듀얼 스트림)이 실험되고 있음**
- **결론적으로 TTS 모델들은 점점 LLM의 시퀀스 토큰 생성 방식을 채택하며, 인터페이스 측면에서 강력한 실시간 에이전트 구현이 가능해지고 있음**

## 세부 요약 - 주제별 정리

### 미스트랄의 TTS 오픈소스 공개 및 회사 소개

- 미스트랄(Mistral)은 최첨단(프론티어) AI 연구소로, 기업 맞춤형 AI 도입을 지원하는 B2B 사업도 겸하고 있음
- 최근 최초의 오픈소스 텍스트-음성 변환(TTS) 모델을 공개
- TTS 모델은 음성 생성 품질이 매우 뛰어나고 기업 및 개발자를 위한 활용도를 강조
- 미스트랄의 TTS 오픈소스 모델은 누구나 사용할 수 있으며, 사내 기술 보고서도 참고 가능

### TTS의 핵심 사용 사례는 실시간 에이전트 인터페이스임

- 초기 TTS 활용 사례는 “블로그 읽기”, “기사 듣기” 등 오프라인 사용 위주였음
- 최근 주력 사용처는 챗봇, 대화 에이전트에 편입되어 음성 입출력을 구현하는 것임
- STT(음성→텍스트), TTS(텍스트→음성)가 결합됨으로써 대화형 에이전트가 구현됨
- 사용자의 체감 레이턴시 감소가 대단히 중요하며, 빠른 실시간 처리와 즉각적 응답이 요구됨

### 오디오 스트리밍 구조 구현으로 레이턴시를 최소화함

- 시스템은 LLM이 첫 토큰을 생성하는 즉시, 첫 오디오 패킷을 생성·재생하는 스트리밍 구조를 지향
- 전체 음성이 다 생성되기 전에도 사용자는 오디오 응답을 들을 수 있어, 사용자 경험이 대폭 개선
- 실시간 응답의 예시로 “Paul(가상 에이전트 목소리)”과 즉시 대화하는 데모 시연 포함
- 실제 LLM 응답이 매우 짧더라도, 오디오 스트리밍 덕분에 레이턴시가 체감상 거의 없음

### 현대 TTS 모델은 소수 음성 샘플로 고품질 음성 클로닝이 가능함

- 몇 초짜리 음성 샘플만으로 화자의 억양, 발음, 말투 등을 정밀하게 복제
- 언어를 바꿔도 화자 고유의 억양이 TTS 음성에 그대로 재현됨(예시: 프랑스 성우의 영어 클로닝)
- 발표자는 자신의 음성도 실시간으로 복제하여 “스스로와 대화” 시연을 보여줌
- 기업의 브랜드 관점에서 “음성 아이덴티티(Voice Identity)” 개념이 새로운 마케팅 자산으로 부상

### 오디오 데이터는 프레임 단위로 자르고, 압축 및 토큰화 작업이 필수임

- 오디오는 초당 수천 번 샘플링되어 데이터량이 엄청남(예: 200kbps 수준의 MP3)
- TTS 시스템은 오디오를 약 80ms 프레임으로 자르고, 프레임당 여러 토큰(자사 모델의 경우 37개)으로 변환함
- 이 작업으로 초당 약 500개의 오디오 토큰으로 압축되며, 정보 손실 최소화 및 처리 효율성을 확보
- 오디오에서 부차적인 정보를 덜어내고 핵심 음성 정보를 추출하도록 모델 훈련 설계

### 텍스트-음성 변환을 위한 토큰 시퀀스 모델링 구조 설명

- LLM(대형 언어모델)과 유사하게, 오토리그래시브(autoregressive) 방식의 디코더 백본 이용
- 모델은 오디오 샘플 개별 생성이 아니라, 프레임(패치) 단위로 토큰 시퀀스를 생성
- 시퀀셜 토큰 열을 만든 뒤, 이를 기반으로 음성 합성 오디오를 복원(디코드)
- 메인 트랜스포머는 약 40억 파라미터(4B) 규모로 대형 LLM에 준하는 구조

### 오디오 코덱 훈련 방법과 정보 압축 설계를 소개함

- 대규모 오디오 재구성 작업을 통해, 토큰 단위로 정보가 압축되도록 코덱 학습
- 재구성 손실(loss), 적대적 손실(adversarial loss) 등 다양한 손실 함수로 최적화
- 일부 토큰에는 텍스트 정보가 반드시 포함되도록 가이드, 텍스트 복원이 가능하도록 처리
- 미스트랄 TTS는 프레임별 37개 토큰을 diffusion model 기반으로 한 번에 생성(기술 차별점 강조)

### 텍스트-음성 변환에서 텍스트 컨디셔닝의 구현 방식

- 실제 '텍스트 기반 음성' 생성을 위해, 텍스트 컨텍스트(예: 화자 음성 샘플, 변환할 텍스트)를 함께 입력
- 대부분의 시스템은 전체 컨텍스트를 초기 입력으로 주고, 오디오 완성 후 재생
- 미스트랄 모델은 이 방식을 채택하며, 필요시 텍스트 입력 후 17ms 만에 첫 오디오 출력을 지원(네트워크/단일 GPU 가정)

### 실시간 텍스트 입력 스트림 처리 방식에 여러 실험적 패턴이 존재함

- 실시간 텍스트 스트림을 입력받아 오디오를 연속 생성할 때 두 가지 주요 패턴이 있음
    - 텍스트와 오디오를 한 스트림에 인터리브(interleave) 방식으로 혼합 처리
    - 오디오/텍스트 각각 독립적인 스트림으로 처리 후 추론(inference) 시 결합(blend) 처리
- 미스트랄도 차기 연구 주제로 이 구조들을 실험할 예정, 현시점에서 “최고의 아키텍처”는 미정
- 기대효과는 LLM이 텍스트를 생성하는 즉시, 실시간으로 오디오 재생이 시작되는 더 빠른 레이턴시

### 미스트랄 오픈소스 TTS의 오픈 범위와 제한 사항

- 텍스트-음성 모델 자체와 학습된 주요 음성 샘플(오픈 보이스)은 공개
- 단, 「음성 클로닝용 인코더」 부분은 오픈하지 않아, 사용자가 임의 음성을 직접 복제하진 못함
- 공식 제공하는 음성(보이스)로만 사용 가능하며, 추후 오픈 보이스를 더 늘릴 계획
- 보안 및 책임성 이유로 범용 음성 클로닝은 제한하는 방침

### TTS는 LLM 시퀀스 모델링과 점점 더 유사해지며, 새로운 실시간 인터페이스 가능성을 연다

- 텍스트→음성 시스템은 LLM의 토큰 생성 및 시퀀스 예측 원리를 적극적으로 차용
- 음성 에이전트, 챗봇 등 새로운 인터페이스 구성에서 TTS와 LLM 결합이 핵심 인프라로 부상
- 중앙 LLM이 텍스트를 실시간 생성·스트리밍하고, TTS가 이를 즉시 음성화하여 저지연 대화형 시스템 구축이 가능해짐
- 앞으로도 TTS 모델의 아키텍처는 실시간성과 품질 모두를 추구하는 방향으로 빠르게 진화할 예정
