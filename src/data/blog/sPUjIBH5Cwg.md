## *tldraw 캔버스에서 에이전트의 활용 — Steve Ruiz, tldraw* 핵심 요약

- 영상 제목: **tldraw 캔버스에서 에이전트의 활용 — Steve Ruiz, tldraw**
- 발표자는 tldraw 창업자 Steve Ruiz로, tldraw는 온라인 화이트보드, 스타트업, 그리고 다른 제품 제작에 활용 가능한 SDK임을 소개함
- tldraw의 캔버스는 오픈소스 React 컴포넌트로 구성되어 커스터마이즈와 AI 연동에 최적화되어 있음
- AI 활용 초기 사례로 2023년 공개된 ‘Make Real’ 프로젝트를 언급하며, 비개발자도 직접 그린 도면을 프로토타입으로 전환할 수 있었던 혁신적 경험을 설명함
- 텍스트 구조화 방식의 AI 작업(예: 캣 드로잉, 다이어그램 자동 완성)을 통해 이미지 기반 AI보다 안정적이고 복잡한 지시 수행이 가능함을 시연
- ‘에이전트 루프(Agentic Loop)’를 도입해 AI가 반복적으로 작업 산출물 리뷰 및 개선, 완료 판단 등 ‘코딩 에이전트’ 방식의 협업을 실현함
- 형상화된 캐릭터(페어리)를 도입해 캔버스 내에서 AI가 작업 상태·위치 등 시각적으로 표시하고, 다중 에이전트 협업 및 리더-팔로워 구조까지 구현함
- 데스크톱 앱과 Electron 환경 내에서 AI가 보안 위험을 감수하고 파일/코드를 직접 조작하는 ‘로컬 우선’ 접근법 실험 사례도 공유함
- 클라우드와 웹에서 제한적인 부분을 로컬 환경에선 자유롭게 활용하며, 사용자가 위험 부담을 인지하고 도구를 다루는 방식의 패러다임 변화를 제안함
- 발표자는 fairies.tldraw.com에서 데모 체험을 권장하며, 앞으로 에이전트와 캔버스의 결합 실험을 계속해서 확장할 계획임을 알림

## 세부 요약 - 주제별 정리

### tldraw는 무료 온라인 화이트보드이자 SDK로 다양한 제품에 적용되고 있음

- tldraw는 무료로 사용할 수 있는 온라인 화이트보드 서비스임
- 스타트업으로 런던에 기반을 두고 있음
- tldraw는 SDK(소프트웨어 개발 키트)로 제공되어, 다른 제품 및 서비스에 손쉽게 통합 가능
- 예시: Replet의 에이전트 캔버스, Luba AI의 캔버스 등 유명 서비스에 tldraw 캔버스가 적용됨
- Angular 기반 앱에서도 어노테이션 모드 등에 tldraw 캔버스가 활용됨
- SDK로서 “멋진 것”을 만들 수 있음을 증명하는 것이 팀의 역할 중 하나임
- 최근 실험의 초점은 AI 및 ‘해킹 가능한 캔버스 런타임’에 있음

### Make Real 프로젝트는 비개발자를 위한 AI 동작 전환의 혁신을 보여줌

- 2023년 공개된 ‘Make Real’ 앱은 비개발자가 그림을 그려 AI를 통해 동작하는 프로토타입(HTML 등)으로 전환할 수 있도록 설계됨
- 당시에는 '바이브 코딩'이라는 용어도 없었고, 유사 서비스가 거의 전무했음
- 이 트위터 기반 데모는 사용자들이 코딩 없이 기술적 산출물을 만들 수 있도록 해 반향을 일으킴
- 사용자는 색상 변경 지시 등 추가 어노테이션을 하고, 이 프롬프트가 통째로 AI에게 전달됨
- 이후 Stitch 등 소수의 앱에서만 유사한 시도가 이루어짐
- API 키 노출 등의 위험과 더불어, 실시간 데모 시 네트워크 상태에 따라 결과 품질이 좌우됨

### 텍스트 기반 AI 구조화는 이미지 모델 대비 더 복잡하고 안정적인 작업을 가능케 함

- 사용자가 “고양이 그려줘”, “다이어그램 완성해줘”, “슬라이드 마무리해줘”, “그래프 완성해줘” 등 자연어로 지시하면, AI가 캔버스 내 구조화된 도형·도표·객체로 바로 구현
- 이미지 생성 AI(예: Diffusion 등)와 달리, 텍스트→객체(HTML/SVG 등) 변환 방식이므로 수정과 상호작용이 쉽고 결과가 명확함
- 복수의 프롬프트(예: “고양이가 촛불을 끄는 모습”, “쥐도 그려줘” 등)를 동시에 실행해 의미적 맥락까지 반영
- 모델은 그림의 세부 요소(예: 촛불, 고양이 등) 의미를 학습하지 않았으나 정확하게 반영함
- 구조화 데이터(AI 모델에게)에는 이미지보다 더 적은 학습 데이터가 존재하고, 데이터 간 충돌 현상도 쉽지 않음
- 예: 웹상의 y축 증감 방향과 카르테시안 그래프의 y축 증감 방향이 다름—텍스트 기반 방식은 이런 충돌을 쉽게 조정 가능

### 에이전트 루프(Agentic Loop)는 코딩 에이전트와 비슷하게 반복 작업 및 판단을 구현함

- “버터플라이 생애주기 다이어그램 그려줘”와 같이 복잡한 요청을 입력하면, AI가 끊임없이 산출물(다이어그램)을 생성하고 리뷰하며 완성도를 높임
- 반복적으로 실행→피드백→수정하는 루프를 통해 코딩 에이전트(LLM 에이전트)의 일반적인 작동 방식(출력 리뷰, 거절, 설명 노트 등)을 반영
- 이 방식은 실제 tldraw의 다양한 에듀테크나 디자인 앱(예: love art, magic path 등)에 도입되어, 튜터 역할, 캐릭터 시트 자동완성 등 교육적 협업까지 확장됨
- D&D(던전앤드래곤) 캐릭터 시트 완성, 과제 작성 등의 교육 현장에서 효과적임이 시연됨

### 에이전트 캐릭터(페어리)는 캔버스 내 실시간 시각적 협업과 상태표시를 가능하게 함

- AI 에이전트의 ‘바깥으로의 실체화’ 실험으로 ‘페어리’(요정) 캐릭터를 도입
- 각 페어리는 캔버스 내 물리적으로 존재하며 이동, 작업, 다른 에이전트와 상호작용 등이 시각적으로 표시됨
- 각각의 페어리는 마치 각기 다른 터미널 창 혹은 콤포저의 별도 인스턴스처럼 독립적으로 작업 가능
- 예: 한 페어리에 ‘고양이 그려줘’, 다른 페어리에 ‘고양이에 모자 씌워줘’, ‘고양이 목도 그려줘’ 등 역할 분담 가능
- 각 페어리는 작업 과정, 상태, 위치, 작업 구간을 실시간 감지·표시
- 에이전트들끼리 작업물 및 상태를 인식하고, 타인의 작업을 이어받아 협력도 가능함

### 다중 에이전트 협업과 리더-팔로워 구조 도입은 작업 분배·상태관리의 새로운 방법을 시연함

- 예: 여러 명의 페어리를 선택해 “더 많은 동물 그려줘” 요청 시, 한 명이 리더로 뽑히고 나머지에게 ‘할 일 목록’을 배분함
- 리더는 전체 작업 환경(캔버스)을 탐색·스카우트하며, 팔로워들에게 분업화된 태스크를 할당하는 구조임
- 이 과정에서 리더는 직접 작업을 하지 않아도 되고, 작업 배분-상태 확인-완성 여부 판단 등 관리 역할에 집중
- 이러한 협업 모델은 2023년 10~12월 tldraw가 동시대의 ‘에이전트 오케스트레이션’ 논의를 반영하며 실험함
- 복수 에이전트 병렬 작업 시 작업물 중복·충돌을 방지하는 ‘공유 상태’, 리더-팔로워 모델, 블라인드 작업 제한 등도 함께 연구됨

### fairies.tldraw.com 사이트는 다중 에이전트 기반 실제 데모 체험의 장을 제공함

- ‘Make Real’이 텍스트→AI 전환을 쉽게 했듯 ‘Fairies’는 다중 에이전트 병렬 작업을 직관적으로 이해하는 데 탁월함
- 실제 ebook 앱 스펙 등 복잡한 요구사항을 입력해 다수의 페어리가 캔버스 내에서 협업하며 화면을 완성하는 모습 시연
- 페어리들이 직접 분업과 협업을 하며 사용자에게 시각적 피드백을 주는 점이 강점
- fairies.tldraw.com에서 누구나 실시간으로 체험 가능하며, 새로운 발견과 재미를 제공함

### 데스크톱 앱과 Electron 환경에서 AI가 ‘로컬 우선’으로 파일 및 코드를 적극 조작하도록 실험함

- 런타임 API, 브라우저 환경 등 보안·접근권한 제약 때문에, 실제 코딩 및 파일 조작이 필요할 땐 Electron 기반 데스크톱 앱(로컬환경)에서 실험 수행
- Electron으로 tldraw 앱을 감싸고, 로컬 포트 오픈→HTTP 서버 오픈→포스트된 내용(JavaScript) 바로 실행 체계 구축
- 오프라인 파일 기반 데스크톱 환경이라, 보안 위험 부담을 사용자가 직접 떠안는 전제로 ‘모든 권한’ 부여 가능
- 예시: 명령형으로 UI 위젯 생성, 인터랙티브 도구로 기능 확장(hover/click 등 tldraw 미지원 요소도), 심지어 Spotify 앱 번들 코드 변조 등 API 외부 코드 직접 조작까지 가능

### AI가 코드 단위의 직접 제어를 하게 되면서 예상외의 창조성과 위험성도 함께 드러남

- AI는 사용자 지시(예: “인터랙티브하게 만들어줘”, “이 부분 UI 바꿔줘”)에 따라 tldraw 에디터 API 코드까지 자유롭게 작성 및 실행
- 로컬 파일, HTML, 스크립트 인젝션 등 다양한 방식으로 결과물을 만드는 등 결과가 갑작스럽고 예측 불가(긍정+부정)인 상황도 빈번
- 직접 앱 구조, 코드, 상호작용 방식을 바꾸며, 심지어 팀원(맥스)의 요청으로 Spotify 앱 내 불필요한 영역 제거 작업도 AI가 코드 단위로 처리함
- 브라우저 환경에서의 보안 상 제약률이 높은 반면, 로컬 환경에서는 사용자가 위험을 감수하면 거의 모든 행동이 가능하다는 점을 강조

### ‘로컬 우선’ 패러다임 확장은 사용자 책임 하에 ‘강력한 도구’를 제공하는 길을 여는 중임

- 기존에는 실험적, 이상적으로만 여겨졌던 ‘로컬 파일 기반 앱의 에이전트 완전 접근’이 현실적 필수요건이 되고 있음
- 클라우드 및 웹의 안전장치를 넘어, 가능성을 최대화하기 위해선 사용자가 직접 위험을 인지하고 수용하는 방향이 대세가 될 수 있음을 시사
- OpenClaw 같은 프로젝트도 유사하게 ‘예리하지만 쓸만한 도구’를 제공하는 컨셉 공유
- 즉, 모든 것을 할 수 있는 도구를 내어주고, 책임(및 자유)는 사용자에게 전가하는 방식임

### tldraw와 Steve Ruiz는 에이전트+캔버스 결합 실험을 계속 확장할 계획임을 밝힘

- tldraw와 페어리 프로젝트 모두 X(트위터) @tldraw 계정에서 정보 공유
- 발표자 Steve Ruiz 역시 개인 계정(@Steve)으로 관련 내용을 계속 공유 중
- fairies.tldraw.com에서 직접 데모 체험 가능하며, 아직 실험 중이지만 지속적인 업데이트를 약속함
- 페어리는 IRC 등 부가 기능도 제공하며 커뮤니티와의 실험 확장도 가능함
