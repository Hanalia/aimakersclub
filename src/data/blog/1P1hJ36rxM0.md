**채널명:** AI Engineer

## *소프트웨어 엔지니어링은 코드 작성이 아니다* 핵심 요약

- 이번 강연은 Google DeepMind의 연구 부사장 베누아 실링스(Benoit Schillings)가 소프트웨어 엔지니어링의 본질과 AI가 코드 작성 영역에서 가져올 패러다임 전환에 대해 설명함
- 실링스는 Google X(와이모, 글래스 등)에서의 경험과 DeepMind 팀의 역할(향후 1개월~1년 내 Gemini 기술 발전 주도)에 대해 언급하며, ML 기반 코드 작성 연구의 시발점이 된 ‘Pitchfork’ 프로젝트(2018년 시작)를 소개함
- 45년간 개발자로서 겪은 언어 변화(어셈블리, C++, GC 언어, Python, V 코딩 등)를 바탕으로, 작성 코딩의 한계에서 설계 및 아키텍처 중심으로 소프트웨어 엔지니어링의 무게중심이 이동하는 과정을 설명
- 인간의 인지 한계(문맥 유지 7~9토큰)가 소프트웨어를 모듈화하고 장기적으로 관리 가능하게 만드는 설계 방식에 큰 영향을 미쳤으나, AI의 ‘거의 무한대’ 문맥 처리로 이 구조가 변화하고 있음을 강조
- AI 모델(Gemini 등)은 이미 ‘구문적 코드 작성’(함수 등)에서는 인간을 능가하는 수준에 도달했으며, 앞으로는 수백만 라인의 대규모 코드베이스 관리와 아키텍처 설계가 핵심 도전과제로 전환된다고 진단
- 코드는 방대한 훈련 데이터(공개 저장소 등), 검증 가능성(컴파일·테스트 가능성) 등에서 특별한 도메인이었으나, 현재는 ML로 생성된 코드가 80% 이상을 차지하여 인간 중심 데이터의 한계에 도달함
- 알파제로처럼 ‘Selfplay’(자가 생성 및 평가) 기법이 코드 작성에도 적용되어, 모델이 스스로 수많은 도전과 평가를 통해 코딩 능력을 계속 향상시킬 수 있음을 시사
- 작성 자체의 비용이 거의 ‘제로’가 됨에 따라 코드 폭증과 그에 따른 설계/신뢰성/보안 이슈, 평가 체계 개편, 새로운 언어 설계 필요성 등 근본적인 변화 이슈를 제시
- 코드는 실험 영역의 확장(특히 화학·생물학 등)과 예상불가한 혁신 가능성을 내포하고 있으며, 앞으로 AI를 활용한 코드 실험이 과학적 발견과 문제 해결의 큰 역할을 할 것임을 강조

## 세부 요약 - 주제별 정리

### 베누아 실링스의 경력과 Google DeepMind 연구팀의 목표가 혁신적 AI 도입에 초점을 맞춤

- 베누아 실링스는 Google X(예: 와이모, 글래스)와 Google DeepMind에서 다양한 경험을 쌓음
- DeepMind 팀 목표: "1개월~1년 내 Gemini를 혁신적으로 발전시킬 기술 개발"
- 단기(1개월)와 중기(1년) 타깃을 아우르는 개발의 중요성을 강조하며, 1년 이상 예측은 불가능하다고 언급
- 팀 내 주요 연구 분야: 코드, 추론 능력 발전, 신경망 토폴로지, 강화학습 과학 등
- 연구철학: ‘필요한 기술에 선제적으로 도전’하는 자세를 갖춤

### 2018년 Pitchfork 프로젝트는 ML 기반 코드 작성 가속화의 선구적 시도였음

- Pitchfork: 2018년 Google X에서 시작한 ML 기반 코드 작성 효율화 프로젝트
- 초창기 내부 저항: “왜 코드 작성에 ML이 필요한가?” 등 무관심과 회의론 존재
- 프로젝트 목표: 일상적이지만 불필요하게 느린 코드 리뷰·수정 주기를 ML로 단축
- ‘영어로 코딩한다’는 개념(바이브 코딩 등)에 대해 처음엔 회의적이었으나, 현재는 실제로 실현되고 있음을 인정함
- 실링스 자신도 언어별 변화를 경험(어셈블리 → C++ → Python 등)
- 새 기술 도입 초기 불신과 적응 과정에 대한 개인적 통찰 제공

### 코드 작성 도구ㆍ언어의 진화는 인간 인지의 한계와 컴퓨팅 환경 변화에 따라 발전해 왔음

- 초창기(약 1980년대): 기계 성능 제한(어셈블리 코딩)의 시대, 성능 극대화가 핵심
- 컴퓨팅 비용 하락 이후: 최적화보다 모듈성·관리성이 더 중요해짐(라이브러리, 함수, 재사용 등)
- 인간의 단기 문맥 유지 한계: 7~9 토큰 이내로, 대규모 복잡성 관리 시 ‘문맥 잃음’ 문제 발생
- 소프트웨어 엔지니어링 프로세스: 인간 인지 한계를 전제로 한 구조적 설계에 집중
- AI/ML의 ‘무한 문맥’ 능력이 전통적 소프트웨어 프로세스 구조를 변화시키고 있음

### AI는 구문적 코드 생성 영역에서 이미 인간을 넘어섰으며, 문제 해결의 중심은 설계와 아키텍처로 이동함

- 함수 생성 등 ‘슈퍼휴먼’ 수준의 코드 작성 능력은 현 시점에서 이미 확보됨
- 인간이 더 능숙한 영역: 시스템을 다양한 맥락에서 파악하고(‘유도적 사고’), 패턴 발견 및 결정(벤다이어그램 식 연관성 탐지 등)
- 진정한 소프트웨어 엔지니어링은 거대 코드베이스의 관리와 변경, 초거대 아키텍처 설계(예: 구글 내 수천만 라인, Jeff Dean의 아키텍트 역할 등)로 핵심이 옮겨감
- ‘코드 작성’이 아닌 ‘문제 정의, 요구 명세, 구조 설계, 장기적 안정성 확보’가 인간 역할로 남게 됨

### 코드 도메인은 방대한 데이터와 검증 가능성 덕분에 AI 학습에 이상적이었으나, 인간 코드의 시대는 끝나고 있음

- 코드 도메인은 공개 저장소(GitHub 등) 기반 방대한 데이터 확보 용이
- 코드의 명확한 정답 존재: 컴파일, 테스트, 유닛 테스트 등으로 정량적 검증 가능
- 현재 GitHub 신규 추가 코드 80% 이상이 기계 생성(ML), 인간 데이터 ‘고갈’ 상태
- 더 이상 인간 코드 열람/분석이 AI 발전에 결정적인 데이터가 아님

### Selfplay(자가 플레이) 학습 방식이 코드 생성 AI의 차세대 성능 도약을 이끎

- 알파제로: 인간 지식 없이 스스로 경기를 반복하며 초월적 바둑·체스 실력 획득
- Frontier 모델 역시 스스로 코딩 과제를 만들어 풀고, 아키텍처·정답도 자체 평가 가능
- Selfplay의 핵심은 무한 도전 과제 설계, 결과 평가, 성능 향상 주기 무한 반복
- 인간 엔지니어와 비교: 개발자도 자기계발을 위해 스스로 문제 설계·풀이·평가 반복함
- Selfplay는 계산 자원과 시간만 허락된다면, 더 높은 수준의 ‘슈퍼휴먼 코드’로 진화 가능

### 코드 작성의 비용이 극적으로 하락하여, 앞으로 설계, 적합성 평가, 신뢰성 유지가 핵심 과제로 부각됨

- 과거: 코드 작성이 ‘비용의 본질’, 조직문화·프로세스·비즈니스 구조 모두 여기에 최적화
- 현재: 코드 작성 ‘비용=0’에 수렴, 코드 양 급증(폭발적 증가 예고)
- 설계·적합성(Design/Adequacy), 신뢰성 관리가 미래 소프트웨어 엔지니어링의 핵심
- 코드 리뷰 등은 단기적으로 유지되겠으나, 조만간 Gemini 등 모델이 코드 생성·검증도 담당
- 과거 컴파일러의 어셈블리 코드 체크 사라진 사례와 유사

### 보안 취약점 은닉, 코드 안전성, 평가 지표 다변화 등 새로운 관리체계가 필수적으로 요구됨

- 모델 기반 취약점 탐지(E.g. Mythos): 엄청난 취약점을 자동 감지, 패치 또한 무한 반복
- 코드 안전성을 ‘사후 탐지’가 아닌 ‘작성 단계에서부터 정답을 만드는 모델’로 유도해야 함(높은 난이도)
- 모델이 문제 도메인이나 규칙 지식을 서로 전이(Inductive Architecture)하거나, 정확한 문제 분해·계획 능력을 습득해야 함
- 평가(benchmark)도 단순 실행 결과 검증(예: 3bench)에서 개방형·고난도 문제(예: 압축률 극대화 알고리즘 등)로 다변화 필요
- 새로운 평가 방식은 ‘전혀 새로운 알고리즘 설계’ 등 창의적 해결 능력까지 검증해야 함

### 인간과 AI의 사고방식 차이, 다중모달 코드 설계, 시각적 코드 구조 파악 등이 미래 발전의 새로운 축이 됨

- 코드 작성은 단순 토큰 체인이 아니라 시각적, 공간적, 동적 구조의 이해가 필요
- Google Gemini 모델은 설계단계부터 텍스트뿐만 아니라 멀티모달리티(공간·시각·동적 표현 포함) 지원
- 미래의 AI 개발 프레임워크는 공간적/시각적 추론을 활용한 문제 해결까지 도달해야 함

### AI에 최적화된 새로운 프로그래밍 언어 설계가 필요하며, 언어의 사용성보다 신뢰성과 안전성이 중심이 될 것임

- 기존 언어(Python 등)는 인간이 사용하기 위한 목적에 맞춤(가독성 등), 안전성은 희생됨
- 코드 작성 고통이 사라진 지금, 오히려 ‘작성 난이도가 높더라도 안전성이 극대화된 언어’(예: 강타입, Lean류 증명 기반 등)를 도입할 시점
- 미래 언어는 사람이 읽을 필요조차 없으며, 정형 논리적 제약조건 우선 설계가 가능함

### 코드 실험의 파급력은 화학, 생물학, 자연과학 등 물질계 영역의 혁신을 폭발적으로 진전시킬 것임

- 코드는 ‘만능 실험 언어’로, 실험 비용이 본질적으로 0에 수렴하며 과학적 탐색 속도 혁신
- 화학: 20개 이상 원자가 결합된 분자의 거동조차 인간은 파악 힘듦, AI 코딩 실험은 10,000원자 규모 구조 분석 및 신물질 설계 가능성
- 생물학: 자연은 ‘탁월한 엔지니어, 최악의 문서화담당’, AI는 복잡한 생체·분자 관계 규명에 유리
- 기존 인간의 ‘편향된 시야’ 한계를 뛰어넘는 혁신적 아이디어와 발견의 가능성(‘보이지 않는 금맥’)
- ML은 인간의 심리적·진화적 한계에 가려 왔던 문제도 다른 각도에서 획기적으로 접근 가능

### 결론: 코드와 AI의 만남은 소프트웨어 엔지니어링, 과학, 문제해결 방식 전체를 근본적으로 바꾸고 있음

- 위 모든 변화가 기존의 소프트웨어 엔지니어링 프레임을 근본적으로 뒤흔들고 있음
- AI 코드 실험 및 활용의 확대는 새로운 발견, 혁신, 사고방식 전환의 시대로 가속함
- 앞으로 인간과 머신의 역할, 프로세스, 언어, 과학적 탐구가 모두 동반 진화할 것임
