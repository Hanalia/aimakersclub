## *모델을 키우지 말고, 행동하게 만드세요 — Kobie Crawford, Snorkel* 핵심 요약

- 영상 제목은 "*모델을 키우지 말고, 행동하게 만드세요 — Kobie Crawford, Snorkel*"로, Snorkel과 UC버클리 Agentyca(RLLM) 연구팀의 공동 연구 결과를 소개함.
- 대형 AI 모델(235억 파라미터)을 단순히 크기만 키우는 것이 아니라, 전문가 데이터와 RL(강화학습)을 활용해 소형(4억 파라미터) 모델의 ‘행동’을 개선하는 것이 현명하다는 메시지가 핵심.
- 금융 분석 툴 사용 과제에서 RL을 통해 4억 파라미터 모델이 235억 파라미터 모델을 능가하는 성능을 달성함을 입증.
- 대형 모델은 추론(Reasoning) 능력은 뛰어나지만 툴 사용에 있어 주어진 환경을 이해하고 적절히 사용하는 ‘행동’ 능력이 부족함을 확인.
- 실험 환경(FinQA)은 Snorkel과 UC버클리가 자체 구축한 벤치마크로, 툴 사용 쿼리 및 복합적 데이터베이스 질의까지 포함함.
- 데이터셋 수집 과정에서 금융 분야 전문가들을 직접 섭외해 높은 데이터 품질을 확보, 이후 과제 적합성 검증을 통해 정확도와 신뢰성을 중시함.
- RL 학습 루프는 약 21시간, 비용은 500달러 미만으로 실현되어, 합리적인 비용·시간 투자로 대규모 성능 향상을 이끌어 냄.
- 모델이 단순히 출력 정답을 외우는 것이 아니라, 문제를 풀기 위해 필요한 툴(SQ L쿼리 등)을 먼저 탐색/사용하고 오류 발생 시 스스로 수정하는 ‘행동 습득’이 성능 개선의 핵심 요인임을 밝혔다.
- 단일 테이블 문제만으로 훈련할 경우, 복합(다중) 테이블 문제까지 성능이 동반 상승하는 현상을 발견, 툴 적응 ‘행동’ 자체가 일반화에 중요함을 시사.
- Snorkel은 성능 평가를 단순 정답/오답으로만 보지 않고 Rubric(세분화 평가표)을 도입해, RL 및 데이터 개선 방향을 더 구체적으로 진단/설계함.

## 세부 요약 - 주제별 정리

### 발표자는 데이터 품질과 전문가 참여의 중요성을 강조하며 Snorkel의 역할을 소개함

- Kobie Crawford는 Snorkel의 개발자 에반젤리스트로, 데이터 중심 AI 연구와 솔루션을 강조함.
- Snorkel은 창립 이래 철저히 데이터 품질에 집중, 데이터 생성·검증 과정에 분야별 전문가를 적극 참여시킴.
- 전문가는 박사급 연구자/현장 실무자 등으로, 실제 모델 학습에 쓰일 데이터의 신뢰성과 정확도를 보장.
- Snorkel이 대기업/연구실들에 제공하는 데이터는 ‘전문가 루프’(expert-in-the-loop) 구조를 반드시 거침.
- 영상 초반에는 컨퍼런스 참가자 반응 확인 및 Snorkel의 컨퍼런스 지원 동기를 밝힘.

### AI 모델의 크기만 키우는 현재 관행이 생산 환경(엔터프라이즈) 과제에 항상 효과적이지 않음을 지적함

- 개인용 AI 비서의 발전에도 불구하고, 실제 엔터프라이즈 현장에선 신뢰성·보안·비용 등이 더욱 중시됨.
- 실전 도입을 위해선 서버 직접 운영(On-prem), 제3자 의존도 감소 및 데이터 통제 등이 필요함.
- 한계에 부딪히면 모델 크기만 키우는 구조(“더 큰 모델이 더 똑똑하고 성능이 좋을 것”이라는 생각)가 만연함.
- 하지만 대형 모델은 연산비용과 추론 시간, 보안성에서 불리할 수 있으며, 항상 성능을 담보하지 않음.

### 연구 목표는 ‘강화학습+전문 데이터셋’ 기반으로 소형 모델이 대형 모델보다 실질 업무를 더 잘 수행하게 만드는 것이었음

- 목표: 4억 파라미터 모델을 235억 파라미터 모델보다 금융 툴 사용 과제에서 더 잘 동작하게 훈련.
- UC버클리(RLLM) Agentyca Lab과 협업해 실험 설계 및 검증을 공동 진행.
- 모델 크기가 아니라 문제에 꼭 맞는 데이터와 바람직한 행동을 학습시키는 방식에 주목.

### 대형 모델은 추론능력은 뛰어나지만 환경 내 ‘툴 사용 행동’ 학습에 명백한 한계를 드러냄

- 실제 실험에서 235억 파라미터 모델은 “YouTube 광고 매출의 연도별 성장률은?” 질문에 잘못된 DB 테이블을 질의.
- 환경 내 존재하지 않는 테이블명을 아무 근거 없이 사용해 쿼리하다가 미답을 받자, 끝내 ‘환각적 답변(hallucination)’ 생성.
- 이는 환경 인지나 툴 적응력보다 단순 추론(Reasoning)에 치우친 큰 모델의 맹점을 보여줌.
- 발표자는 이를 ‘Terence Tao 효과’에 비유: 천재 수학자(초고난이도 수학도 풀 능력)가 실전 문제(기본 데이터 분석)에 오히려 비효율적일 수 있듯, 모든 문제에 최고급 지성을 동원할 필요가 없다는 것.

### Snorkel의 데이터 수집 및 전문가 검증 방식이 RL 성능 개선의 핵심이었음

- 금융 데이터, SQL 등 도메인지식이 필요한 작업에 해당 분야 전문가(PhD, 실무진)를 데이터 수집·가공·과제 검증 과정에 투입.
- 과제 생성 후 전체 유효성·적합성 검토, 실제로 쿼리 가능하고 정답 검증이 명확한 질문/정답 세트만 선별.
- 데이터 품질 강화는 RL 및 모델 일반화력 향상의 핵심 기반임을 강조.
- Snorkel 고유의 데이터 루프와 품질 기준이 후속 연구 및 벤치마크에도 큰 역할.

### RL(강화학습) 학습 프로세스는 환경 셋업 및 비용·시간 면에서 접근성을 입증함

- 4억 파라미터 모델로 RL(GRPO 방식) 수행, 학습 시간 약 21시간, 비용 500달러 이하.
- RL 실험 환경(FinQA)은 Snorkel과 UC버클리가 자체 구축·공개한 오픈소스 벤치마크 환경임.
- 모든 도구/테이블/DB 쿼리 환경이 포함되어 완전히 폐쇄된 셋업으로, 외부 인프라 의존도 없음.
- PrimeIntellect 인프라, OpenEnv(GitHub), Hugging Face Spaces 등에서 사용 가능하도록 함.

### RL 학습 결과, 행동 개선이 대형 모델 대비 월등한 성능 향상을 이끌어냄

- RL과 고품질 데이터셋으로 훈련한 4억 파라미터 모델은 235억 파라미터 모델 대비 패스앳원(pass@1) 점수가 두 배 가까이 상승.
- 복합 쿼리(다중 테이블 질의)도 단일 테이블 위주 훈련만으로도 14%에서 27%로 성능이 거의 두 배 상승.
- RL 학습 후 모델은 ‘get table names’ → ‘get table info’ → SQL 쿼리 → 에러시 자체 수정→ 정답 도출 등, 일련의 올바른 툴 사용 행동을 습득.
- 실전 과제에서 중요한 것은 정답 자체가 아니라, 환경에 맞게 올바른 도구와 전략을 선택/응용하는 능력임을 입증.

### 단일 테이블 위주 훈련이 다중 테이블 고난도 문제에서도 성능향상을 이끌어, 행동 자체의 일반화 효능이 강조됨

- 학습 데이터셋: 총 290개(기본 FinQA), 79개(고급 Reasoning 버전·다중 테이블).
- 단일 테이블 질문만 주입해 훈련해도, 다중 테이블 질의 난도에서도 동등하게 성능이 크게 오름.
- 이는 툴사용 ‘행동’ 패턴 학습이 보다 다양한 문제로의 일반화에 결정적임을 시사.
- Curriculum learning 등 점진적 학습법보다 단순 단일-테이블 중심 학습이 실효적 결과를 냄.

### Snorkel 연구팀은 평가 지표로 단순 정답율이 아닌, 세분화된 Rubric(루브릭) 기반 평가 방법론을 활용함

- RL에서 단순 예/아니오 보상 신호 외에도, 각 문제 단계를 나눠 행동 분석이 가능한 평가 기준을 제작.
- Rubric 기반 피드백을 통해 실제로 어떤 행동이 실패 원인인지 분석, 추가 데이터셋 선정 시에도 지침으로 삼음.
- 기존 RL은 단일 가치 보상만 사용하나, Rubric은 세밀한 행동원인 진단·피드백 및 데이터 기획에 필수적.
- 이 방식이 앞으로의 행동 기반 LLM 발전 및 RL 연구에도 중요한 모범으로 활용될 것임을 시사.

### RL 및 고품질 행동 데이터 적용만으로 소형 모델이 대형 모델 성능을 능가할 수 있음을 실증했고, 자세한 내용은 논문/블로그를 통해 공개됨

- Snorkel, UC버클리 팀의 추가 정보와 블로그∙기술 문서가 링크로 제공됨.
- 전문 데이터, 환경, 평가 방법까지 일련 프로세스가 외부에도 공개되어 추가 연구 및 실무 현장 적용이 용이함.
- 발표 마지막에는 질문 시간을 별도로 할애하지 못했으나, 관심자 대상 후속 질의 대응을 약속함.
