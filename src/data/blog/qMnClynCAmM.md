## *AI 코딩의 다음 진화는 하니스(Harnesses) - 직접 만드는 방법* 핵심 요약

- 이 영상은 오픈소스 AI 코딩 하니스 빌더 "Archon"의 대규모 업데이트와 그 활용법을 상세히 소개함
- 하니스(harness)는 여러 코딩 에이전트 세션들을 조율하는 상위 레이어로, AI 코딩을 결정적이고 반복 가능하게 만들어줌
- Archon을 사용하면 개발자의 소프트웨어 개발 프로세스를 워크플로우로 손쉽게 코드화하고, 다양한 코드베이스에 적용 가능
- 기존 프롬프트 엔지니어링(LLM에게 한 번에 최고의 결과 얻기)과 컨텍스트 엔지니어링(단일 에이전트에 최적의 맥락 제공)을 넘어 하니스 엔지니어링(다수 에이전트 세션을 연결)으로 진화 설명
- 하니스 적용 시, PR 수용률이 기존 단일 LLM(6.7%) 대비 70%까지 증가한 사례 및 Stripe Minion, Anthropic 등 실제 기업/커뮤니티 사례 언급
- Archon은 수많은 기본 워크플로우(이슈 수정, PR 생성, 리뷰, PRD 작성 등)를 제공하며, 사용자가 직접 새로운 커스텀 워크플로우를 YAML로 손쉽게 만들 수 있음
- CLI, 웹 UI, Claude, GitHub, Telegram, Slack 등 다양한 인터페이스를 지원하여 통합적 멀티 플랫폼 활용이 가능함
- 전체 설치 및 사용할 프로젝트 준비, 에이전트 연결, 기본/커스텀 워크플로우 실행 등 구체 설치 및 실습 과정을 안내
- 워크플로우 병렬 실행, 각 단계별 모델 지정(예: Haiku, Sonnet 등) 등으로 효율성·유연성을 확보함
- 마지막으로 완전히 커스텀한 워크플로우 생성 방법과 Archon의 확장 계획, 향후 라이브스트림 소식을 알림

## 세부 요약 - 주제별 정리

### Archon은 AI 코딩 하니스 빌더로서 기존보다 한층 강력한 자동화와 반복 가능성을 제공함

- Archon은 기존 AI 커맨드 센터의 대대적 오버홀 버전으로, AI 코딩 하니스를 누구나 쉽게 만드는 오픈소스 툴로 진화함
- 하니스는 여러 코딩 에이전트 세션을 조율하여 작업을 체계적으로 구성할 수 있는 상위 개념의 레이어임
- Archon을 이용하면 기존의 코딩 에이전트 활용방식에 자동화, 판단 가능성, 반복성이 더해진 워크플로우로 진보할 수 있음
- 워크플로우에는 프롬프트, 결정적 명령, 컨텍스트 생성, 검증, 휴먼 피드백, PR 생성 등 다양한 노드(조각 작업)를 포함
- 기본 제공 워크플로우 외에도, 사용 목적에 따라 커스텀 워크플로우를 만들어 자신만의 개발 사이클을 완벽히 반영 가능
- Archon은 백엔드 복잡성을 자동으로 처리해주며, 사용자는 워크플로우 설계 및 실행에 집중할 수 있음

### 프롬프트와 컨텍스트 엔지니어링을 넘어 하니스 엔지니어링이 AI 생산성의 핵심으로 부상함

- 2022~2024년까지 프롬프트 엔지니어링은 한 번의 프롬프트로 LLM의 최적 성능을 끌어내는 데 집중함
- 이후 컨텍스트 엔지니어링으로 발전, 단일 에이전트에 적절한 맥락을 주입해 대작업을 처리
- 최근에는 하니스 엔지니어링으로, 다수의 에이전트 세션을 논리적으로 연결하여 보다 복잡한 대형 작업을 처리하는 단계로 진화
- 기존 Ralph loop, Anthropic 오픈 하니스 등과 달리 Archon은 완전 커스터마이징이 가능해, 각자의 소프트웨어 개발 주기를 자유롭게 반영할 수 있음
- 하니스는 도구 체이닝, 프롬프트 결합, 작업 단계 관리 등 LLM을 최대치로 능력 확장시켜줌

### 대형 LLM도 하니스 적용만으로 PR 수용률이 6%→70%까지 향상됨을 사례와 수치로 증명함

- 단일 LLM 기반 코드 PR의 수용률은 단 6.7%에 불과(연구 결과)
- 맞춤형 하니스(예시: Ralph loop 등) 적용 시 PR 수용률이 70%에 육박하는 사례 보고
- Stripe는 자체 AI 코딩 하니스를 활용해 매주 1,300건 이상의 AI 전용 PR을 생성·머지하며, 해당 워크플로우에 컨텍스트 큐레이션, 다단계 검증, 휴먼 검수 절차를 도입
- Anthropic의 코딩 에이전트 코드베이스의 40%가 하니스 관련 코드임이 유출 코드에서 드러남(하니스가 얼마나 중요한지 증명)
- 결과적으로 하니스는 모델 자체의 성능보다도 결과 품질에 결정적으로 기여함

### Archon의 워크플로우는 프롬프트, 결정적 명령, 휴먼 피드백 등 노드 단위로 설계 가능하며 반복적 신뢰성을 보장함

- YAML 기반 워크플로우 문법으로, 한 번 정의한 흐름을 여러 프로젝트에 재사용 가능
- 노드 단위로 명령(프롬프트) 또는 결정적 액션(테스트, 컨텍스트 생성 등) 지정 가능
- 각 노드와 세션별로 새로운 코드 세션 여부·모델종류·문맥 주입 빈도 모두 세밀하게 제어 가능
- 계층적 루프나 휴먼 승인 게이트 삽입 등으로 워크플로우를 현실 업무 흐름에 근접하게 커스터마이즈 가능
- 병렬 실행·에러 리트라이·동시 검증 등도 지원하여, 실제 개발 현장의 요구에 부응
- 각 워크플로우는 CLI, Claude, 웹UI, GitHub 등 다양한 방식으로 호출·감시·조작 가능

### Archon의 설치 및 프로젝트 연결 과정은 단 5분 만에 완료할 수 있음

- 저장소 클론 → Claude 또는 코딩 에이전트에서 'Archon 설치' 요청 → 준비된 스킬로 단계별 안내 진행
- 기본적으로 Bun, 대상 프로젝트(예: YouTube Chat 프로젝트) 지정 후 여러 플랫폼(예: CLI, GitHub, Telegram, Slack 등) 중 연동 플랫폼 선택
- 각 플랫폼(예: GitHub, Slack)마다 API 키 발급 및 입력, 사용 AI 코딩 에이전트(현재 Claude 위주, Codex·Pi Agent 등 순차 지원 예정) 선택 등 설정 진행
- 인증정보 등록 및 Archon 스킬을 대상 레포에 복사, 필요 시 문서 디렉토리 등록 등 진행
- 모든 셋업 과정이 완료되면, 명령어 한 번으로 워크플로우 목록 표시 및 테스트 실행됨
- 새로운 프로젝트를 추가하고 싶으면, Archon CLI 1회 실행 또는 웹UI 통해 손쉽게 추가 가능

### Archon 워크플로우는 기본 제공 외에도 직접 커스텀 생성이 매우 쉬움

- 모든 워크플로우는 .archon 폴더 내 YAML 파일 형태로 보관되어 바로 수정 및 확장 가능
- 워크플로우 내에서 단계별로 모델(예: Haiku, Sonnet 등) 지정 지원, 일부 단순 분류 노드는 경량 모델 할당 가능(코스트/속도 최적화)
- 각 노드는 단순 프롬프트, Bash 명령, 또는 특정 커맨드(예: 웹리서치 등)를 실행하도록 정의
- 예: 'Archon fix Github issue' 워크플로우의 경우, 단계별로 이슈번호 추출→분류→리서치→구현→검증→PR 생성이 순차적으로 실행됨
- Human-in-the-loop 방식 워크플로우, 아이디어→PR 생성, PR 리뷰, 이슈 생성 등 다수 기본 워크플로우 제공
- 커스텀 워크플로우 빌더(워크플로우 생성용 워크플로우) 활용 시, 필요한 절차를 대화식으로 설계·적용 가능

### Archon은 병렬 워크플로우 실행과 웹·CLI 실시간 모니터링 등 뛰어난 확장성과 시각화를 지원함

- 한 번에 여러 이슈 번호 목록을 입력하면, 각 이슈별로 워크플로우가 백그라운드에서 병렬 처리됨(예: 이슈 5,7,8,9,10,11번 동시 실행)
- CLI 명령·웹 UI 모두에서 실행 중 워크플로우의 상세 로그, 각 노드별 진행상황, 실행 내역을 실시간으로 시각화·조회할 수 있음
- Claude Code 등 에이전트에서도 워크플로우 상태 주기적 체크, 에러 감지 및 재시작(자동/수동) 등 유지보수 가능
- 워크플로우의 고도화 및 디버깅(로그 기록) 기능이 내장되어, 반복 실행·상태 모니터링이 매우 용이

### 워크플로우 빌더와 오픈소스 도구 접목으로 다양한 전략(예: Beads 전략 등)도 바로 구현 가능함

- 영상에서는 Beads(지속적 구조화 메모리 제공 오픈소스 도구) 전략을 반영한 Archon 워크플로우 생성 예시 제시
- 워크플로우 빌더 워크플로우 활용: 대화로 원하는 기능 설명 시, 자동으로 적합한 워크플로우 YAML 파일 생성
- 예시) 특정 리포 분석, 아이디어 분해, 개별 작업 반복 구현, 최종 검증 등 Beads 프로세스 전체를 Archon 워크플로우로 자동화
- 생성된 워크플로우를 원하는 프로젝트에 즉시 적용 후 실행 가능
- 향후 n8n 스타일의 워크플로우 시각화 빌더 등 추가 UI 확장 예정

### Archon은 다양한 개발 환경(프로젝트, 인터페이스, 에이전트)과의 통합 유연성을 갖추고 있음

- GitHub, Slack, Telegram, CLI, 웹UI 등 다양한 플랫폼에서 프론트엔드/백엔드 실행 및 상태 확인 가능
- 새로운 프로젝트는 웹 UI에서 "프로젝트 추가"로 간편 연결, 각 프로젝트별 워크플로우 목록 관리 및 실행 가능
- 각 워크플로우는 상황/목적에 따라 CLI, 에이전트 대화, 웹 UI 버튼 등 다채로운 방식으로 실행
- 다양한 에이전트(Coding Agent, Claude Code, 향후 Codex, Pi Agent SDK 등)와도 연동 지원

### 향후 Archon의 기능 확장과 커뮤니티 참여(라이브스트림 등) 계획이 제시됨

- Archon은 베타 단계로, 향후 신규 기능 및 버그픽스, UI/UX 개선, 더 많은 기본 워크플로우 제공 예정
- 사용자 참여를 독려(예: 라이브스트림 / 질의응답), Archon 워크플로우 디자인 및 병렬 실행, 고급 활용법 등 추가 공유 예정
- GitHub 스타, YouTube 구독 등 커뮤니티 구축에 대한 안내와 차후 콘텐츠 예고로 마무리
