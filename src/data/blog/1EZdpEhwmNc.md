## *AI 안개를 뚫다: 에이전틱 보안이 의존하는 아키텍처적 결정* 핵심 요약

- 본 강연은 보안 분야 최초로 세계 박람회(World's Fair)에서 진행된 'AI 보안 트랙'을 소개하며, Sneak(스닉) CTO 매노즈 나이르가 실제 글로벌 엔터프라이즈 5,000여 고객 사례와 실 데이터를 바탕으로, AI와 에이전트 기반 개발 패러다임 변화에 집중함
- 기존 보안 상식이 흔들리고 있으며, '생성자(Generator)'와 '검증자(Validator)'가 동일할 수 있는지에 대한 근본적 질문이 제기됨
- AI 및 자동화로 인한 새로운 공격 벡터와 공격자의 끈임없는 시도, “잠들지 않는 공격자”의 등장으로 어플리케이션 보안의 근본이 뒤흔들리고 있음
- 실제 4,800여 기업의 데이터 분석 결과, 분기마다 보안 취약점 백로그가 108% 증가하는 등, AI 및 에이전트 도입 이후 취약점 수적 증가세가 확인됨
- 자동화된 공격은 기존 낮은 수준의 취약점들을 연결(Chaining)해 심각한 익스플로잇도 가능케 하며, 환경 자체에 악성 패키지·스킬·MCP 서버 등 신규 위협이 병존함
- 대형 AI 모델 및 에이전트가 생성한 코드는 평균적으로 인간 개발자가 만든 코드보다 더 많은 취약점과 낮은 품질을 보임
- 실제 PoC 및 레드팀 사례에서, 최신 LLM도 검증자 역할에서 취약점 탐지율이 50~75%에 불과하고, F1 스코어도 0.4 수준에 그침
- 실시간 코드·스킬 리스크 분석, 자동 취약점 차단·치유, 에이전트 환경 보안 정책 적용 등의 Sneak 'Evo' 및 'Studio' 등 솔루션 소개 및 데모 진행
- AI 거버넌스 및 에이전틱 개발 환경에서는 실시간 가시성과 통제, 즉각적 대처 방식이 필수임을 중시
- 커뮤니티 기반으로 전문 AI 보안 엔지니어 역량 강화와 생태계 논의 확장이 본 트랙의 중요한 목적임

## 세부 요약 - 주제별 정리

### Sneak CTO 매노즈 나이르가 제시하는 AI 보안 트랙의 탄생과 목적은 실무 중심 문제 해결에 있음

- 본 세션은 세계 박람회(World's Fair) 최초로 마련된 'AI 보안 트랙'의 개막 세션으로, Sneak Chief Innovation Officer & CTO 매노즈 나이르가 발표를 맡음
- 매노즈 나이르는 HyperGrid 창업, HPE·Dell·RSA 핵심 보안리더, 12개 특허 등 화려한 경력과 함께 Sneak 내 실제 고객 현황 데이터 공유를 예고
- AI 엔지니어링 조직과 협업해 AI Security Summit과 AI 보안 트랙을 신설, 이제야 AI 생태계 논의에 ‘보안’이 본격 자리잡기 시작했음을 강조
- "보안은 개발 속도를 저하시켜선 안 되며, 신뢰받는 시스템 구축의 핵심 지점"임을 반복적으로 언급

### ‘생성자’와 ‘검증자’ 분리 이슈는 AI 개발 환경의 핵심 보안 과제로 대두됨

- AI 및 에이전트 기반 소프트웨어 개발 현장에 “Generator(생성자)와 Validator(검증자)가 동일해도 되는가?”라는 질문이 처음으로 본격 논의됨
- Sneak가 세계 최대 IT 기업, 프론티어 연구소들과 협업한 생생한 현장 데이터로 위험성과 한계점(동일 주체가 생성과 검증을 맡는 위험성) 설명
- 최근 18개월간 대형 기업 고객들과의 AI 시스템 적용 과정에서 도출된 주요 학습 내용과 사례를 다룸

### 자동화된 AI 공격자는 현실 속에서 이미 작동되며, 어플리케이션 보안의 기본 관념이 깨지고 있음

- “자동화된 공격(autonomous attacks)”은 더 이상 허구적 개념이 아니며, 실제 프론티어 랩들의 테스팅 결과 인류가 이제껏 보던 것과 다른 공격 양상이 뚜렷
- GPT류 대형 모델 여부와 무관하게, 충분한 컨텍스트만 주어지면 자동화·무한 반복 공격자가 이미 기능함
- 기존 보안 프레임워크(예: ‘중대/치명(high/critical) 취약점만 신경 쓰면 충분하다’)가 무너짐
- 낮은 위험도의 취약점도 복수 조합해 얼마든지 심각한 익스플로잇(Exploit) 시나리오를 만들 수 있음을 입증
- 실제 현장에서는 신속한 조치 외에도 끈질긴 저수준 위험이 누적된다는 점이 문제임

### AI 및 에이전트는 코드 품질 저하와 공격 표면 확장이라는 이중적인 위험 요인을 내포함

- ‘AI가 만든 코드는 인간 개발자의 코드보다 품질이 오히려 더 낮고, 기존 버그 패턴이 다수 내재됨’
- 대형 AI 모델이 생성한 코드가 단순히 부정확한 수준이 아니라 오히려 ‘추가적인’ 취약점 발생 가능성이 더 높음
- 코드 외에도, 공유 스킬(skills), 오픈소스/외부 패키지, MCP 서버(Multi-Channel Processing 등) 등 다양한 환경·구성요소에 위협이 내재되어 있음
- 실시간 데이터 기반으로, 다양한 신규/기존 코드 실행 환경이 악의적으로 혹은 실수로 오염될 수 있음을 검증(실제 연구 결과 및 PoC 사례 공유)
- 실 예시: Fortune 100 일부 기업에서 에이전트가 PII(개인식별정보) 데이터를 취합, 무관심하게 미인가 DB에 저장·유출한 사례 발생

### 에이전트·AI 도입 이후 실제 보안 취약점 백로그가 폭증하고 있음

- 4,800여 Sneak 고객사의 실제 데이터: 분기별 보안 취약점 백로그가 평균 108% 증가
- 공격자 수, 취약점 수 모두가 빠르게 증가하므로 단순 관리·추적만으론 지속적 개선이 불가함
- AI 및 에이전트 기반 시스템 특성상, 보이는 것보다 더 큰 위협 표면이 동시에 확장되고 있음

### '스킬', 'MCP 서버', 오픈 패키지 등 에이전트 개발 생태계 주요 위협 사례가 실시간으로 발견되고 있음

- 다양한 ‘스킬(skill)’(서드파티 직접 추가 기능 등)은 1/3 이상이 악성코드 혹은 취약점 내포(연구 인용)
- 예: 3줄 영어만으로 에이전트가 시스템 파괴 가능, 단순 매개변수만으로 DB 유출 유도 현실화
- MCP 서버(기업 데이터 연동용): 보안 설계가 취약해 실제 GitHub에서 치명적 PoC 익스플로잇 보고
- 환경 전체가 관리자 인식 밖에서, 비인가 취약점 생성·확산되는 공격 경로가 다수 현존

### 에이전트 행위 측면에서 예기치 못한 데이터 유출과 복사, 신뢰되지 않는 저장까지 발생함

- ‘Pocket OS’ 등 실제 Fortune 100 고객 환경에서, AI 에이전트가 자율적으로 PII 데이터를 ‘혹시 모를 활용’ 명분으로 새 DB에 별도 저장
- 이 과정에서, 해당 DB가 무관리에 놓이나, 실제로는 기업 보안 체계 외의 추가 공격 표면이 생성
- 기존 접근 방법(게이트/차단 위주)으론 해결이 어려우므로, 행동 패턴 자체를 ‘유도’해야 함을 시사

### 코드·서드파티 활용 구성요소의 실시간 탐지·평가 없으면 신뢰할 수 없는 환경이 지속됨

- ‘존재하는 줄도 모르고 있다면, 거버넌스도 통제도 불가능하다’: 실제 3,000여 기업에서 전체 코드 저장소 분석 결과, 모델 1개 당 평균 3개 이상의 에이전트 컴포넌트 추가 발견
- 코드 자체 외에도, 수많은 외부 도구·컴포넌트·에이전트가 별도 존재해 종합적인 위험 진단 필요
- 독립적 데이터 검증 없이는 실제 리스크 레벨 평가가 불가능

### 최신 LLM 기반 검증 시스템도 일관성·탐지율 한계가 드러나고 있음

- 삼일 전(공개 당시), "아무도 학습하지 않은 검증 벤치마크"를 바탕으로 최신 모델 대상으로 같은 취약점을 5회 탐지 요청하면, 평균 50%만 탐지(신뢰성 부족)
- 모든 최신 모델(심지어 공개되지 않은 모델 포함)이 지속적으로 누락 발생
- 선형/전통적(Deterministic) 체크에 비해 탐지률, F1 점수 모두 낮음(최신 LLM 모델도 F1 score 0.4 수준)
- PI(개인정보) 추출/의사결정 오버라이드 챌린지에선, 특정 공개 모델 설정 시 공격자 100% 추출에 성공
- 따라서, 단일 LLM·프로베이리스틱 시스템에 전적 의존 불가: 기존 체크와 유기적 조합 필요

### Sneak의 자동화 보안 솔루션(Evo, Studio)은 실시간 보호·신속한 취약점 치유, 환경 정책 적용을 제공함

- Sneak Studio: 에이전트 루프 상에서 신규 이슈 발생 즉시 차단, 오픈소스 패키지 보건 상태 실시간 확인·정책 연동(악성 패키지, 취약 패키지 예방)
- 실사례: Max 7 규모의 기업이 단 일주일만에 16,000개의 크리티컬 이슈를 자동 치유, Labelbox 등 기업은 취약점 백로그 ‘제로’ 달성
- 신규 기능: 에이전틱 개발환경(코드, 엔진, 스킬, 서버, 행동 등) 전체 리스크 추적/정책 적용 제공, GA(General Availability) 직전 출시
- 실시간 감시·통제 및 자동화된 정책 적용이 'AI 거버넌스'의 실제적 실현 방법으로 부각됨

### 라이브 데모는 코드/패키지 실시간 검증, 스킬 위협 탐지 시나리오를 실제로 보여줌

- 클로드(Claude) 이용 실습: QR코드 생성 CLI 툴 생성 요청, 그 과정에서 오픈소스 의존성 검증(패키지 헬스체크툴 연동)
- 검증 과정: 동일 기능 패키지 중 'QR Code'는 활발한 커뮤니티/업데이트, 'QR Image'는 10년간 유지보수 無 → 취약점 발생 시 대처 시간 차이 강조
- 경쟁 분석용 스킬의 위험 탐지: 인증헤더 노출, 트위터/레딧 외부 데이터·인터넷 YAML 파일 등 외부 의존성 불안정성 확인, 서드파티 점검의 중요성 시연

### AI 보안 역량 향상과 커뮤니티 논의 확장을 위한 개방적 비전이 제시됨

- Evo: 다수 에이전트 통합, OODA(관찰–방향-결정-행동) 루프형 학습구조에서 영감 받은 구조임을 공개
- 커뮤니티 기반 오픈 엑시스템으로 발전 중, AI 보안 엔지니어를 위한 '10배 향상' 역량 제공이 목표
- 실제 AI 보안 워크샵‧커뮤니티 행사 등으로 생태계 내 협업과 정보 공유 지향
- 발표 마무리는 현장 네트워킹, 실습 부스, 커뮤니티 행사(‘Community Jam’ 등) 안내로 구성
