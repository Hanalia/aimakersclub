## *프로덕션 환경에서 에이전트 구축과 확장: OpenGov OG Assist 사례* 핵심 요약

- **OpenGov는 정부 기관용 ERP 소프트웨어(예산, 구매, 자산관리, 인허가)를 제공하는 기업으로, 모든 제품에 AI 기반 어시스턴트(OG Assist)를 도입함**
- **OG Assist는 제품 내 챗 인터페이스 버튼으로, 도메인별 맞춤형 질의응답 및 다양한 에이전트 툴(예: 요금 코드 조회, 화면 정보 하이라이트 등)을 실행**
- **AI 에이전트 개발은 ‘Effect’(오픈소스 타입스크립트 라이브러리)로 전환하며 코드 구조화, 트레이싱, 에러 핸들링, 로깅 등 품질과 확장성을 확보**
- **초기에는 LangGraph 기반 루프를 사용했으나, 자체 Effect 기반 에이전트 루프로 완전 이관하여 더 높은 제어권과 커스터마이징 달성**
- **에이전트 간 상호작용 프로토콜(A2A, agent-to-agent)를 적극 도입하여, 엄격한 스펙 하에 제품 내/외부 시스템 간 일관성 확보**
- **사용자 피드백(좋아요/싫어요) 및 자동화된 eval(Test), CI와 연동된 실제 응답 평가 시스템을 구축해 정확도와 활용성을 지속적으로 고도화**
- **안전성과 신뢰성 강화를 위해 툴 콜 승인 시 인간 개입(Human-in-the-loop)과 샌드박싱(코드, 파일 실행 격리 환경)으로 프로덕션 보호**
- **‘롤링 요약(Rolling Summarization)’ 등으로 장기 대화 문맥 처리를 효율화, 대화 흐름 유지와 기억 능력 확보**
- **런타임 동적 UI(form) 생성 등으로 맞춤형, 개인화된 사용자 경험 제공**
- **Effect 기반의 자동 트레이싱과 관측성 강화로, 프로덕션 규모에서 에이전트 디버깅·운영의 속도/안정성 담보**
- **‘툴’과 ‘스킬’ 중심 설계로 기능 확장과 팀 내/외 개발 생산성을 동시에 혁신**
- **내부적으로도 Claude, Cursor, Claude Agents 등 최신 AI 도구를 활용, 자체 개발자 업무 흐름 가속화**

## 세부 요약 - 주제별 정리

### OpenGov는 정부 솔루션의 혁신과 OG Assist로 제품 사용자 경험을 크게 전환함

- OpenGov는 예산, 구매, 자산, 인허가 등 정부기관 업무용 ERP SW를 개발하는 회사임
- OG Assist는 전 제품군에 공통으로 붙는 챗 인터페이스 버튼으로, 즉시 맥락에 맞는 에이전트 경험 제공
- 예시: 유틸리티 청구 관련 화면에서 요금코드(rate codes) 정보 질의, 화면 내 정보 하이라이트, 클릭 유도 등 가능
- 모든 제품팀이 각자 도메인에 맞는 툴, 스킬 정의·개발 → OG Assist에 일원화

### AI 에이전트 팀 신설과 OG Assist의 개발 여정은 실질적 통합과 혁신의 연속이었음

- AI의 급부상 시점에서 핵심 개발자들이 AI 에이전트 전담 팀으로 합류해 OG Assist 본격 구축
- OG Assist를 백엔드 뿐 아니라 프론트엔드에도 통합, 화면 정보 인지 및 조작 등 광범위한 기능 추가
- 사용자는 무엇이 화면에 있는지 묻거나, 실행 가능한 다음 액션 등 현실적인 업무 문맥 질의에 대해 직접적인 피드백을 받음

### Effect 라이브러리 채택이 코드 품질 및 조직 생산성 혁신에 근본적인 역할을 했음

- Effect는 타입스크립트 오픈소스 라이브러리(https://effect.website/), OG Assist 전체 에이전트 루프의 기반
- Zod와 유사한 스키마, 고급 에러 핸들링, 로깅, 트레이싱, 구조적 동시성 등 프로덕션 필수 기능 내장
- 신규 서비스 및 기능 개발 속도, 아키텍처 일관성, 유지보수성 모두 크게 향상
- 실제 코드 예시에서는 effect/ai 패키지 사용: 의존성 주입 방식으로 LLM 핫스왑 용이, 전체 루프에 트레이싱·로깅 자동화

### 자체 Effect 기반 에이전트 루프 구축이 복잡한 도메인 맞춤화와 제어권을 가능하게 함

- 초기에는 LangGraph 기반 루프를 썼으나, 확장성 한계로 자체 Effect 기반 루프로 이관
- 전체 에이전트 루프를 내부적으로 완전 제어 → 복잡한 엔터프라이즈 도메인별 기능/요건 반영 용이
- dependency injection 등 아키텍처 패턴 도입으로 추후 언어 모델·컴포넌트 교체/확장 빠름
- 고도화된 트레이싱, structured concurrency, 에러 로깅, 코드 재사용성 확보

### A2A(agent-to-agent) 프로토콜 도입으로 에이전트 간 통신 엄격성과 확장성을 동시에 담보함

- Google이 만든 agent-to-agent(A2A) 프로토콜 채택, 프론트/백엔드 간 인터페이스·컨트랙트로 정형화
- agent card(에이전트명, 설명 등) 구조 등 엄격한 스펙하에 제품별/파트별 일관성 유지
- 프로토콜 기반으로 확장(메타데이터, A2UI 등) 가능, 새로운 툴·스킬 확장 시 빠른 정렬
- 스펙 기반으로 팀 간 의사소통, 개발 정합성 확보에 큰 성과

### 피드백 수집과 자동화 평가 시스템이 제품 품질 개선에 핵심적 역할을 함

- 제품 내에서 thumbs up/down 방식의 실시간 사용자 피드백을 수집, 반영하여 응답 품질 개선
- 이메일/콜 등 직접 피드백 외에도, CI 파이프라인 상에서 Eval(자동화 평가)로 실제 답변을 검증
- 툴/스킬 호출 여부 등 구체적 테스트를 반복적으로 자동화, 응답 일관성과 정확도 지속 개선
- 수동+자동 피드백 루프로 매우 짧은 반복/개선 사이클 유지

### 인간 승인(Human-in-the-loop) 체계로 변형성 있는 툴 호출에 대한 안전성과 신뢰성을 강화함

- 에이전트의 외부 툴콜, 특히 변경(mutation) 요청 시, UI 상으로 인간 승인(accept/reject) 프로세스 강제
- 행동 신뢰성과 책임성, 사용자와 운영자 모두를 위한 안전장치 마련
- 복잡·민감한 오퍼레이션에서 항상 인간이 ‘최종 결정권’을 행사

### 에이전트 샌드박싱 도입으로 코드 실행·파일 생성 안전성을 보장함

- 에이전트가 코드 실행, 파일 생성 등 작업이 필요할 때, 독립 샌드박스 환경에서 수행
- 작업 완료 시 샌드박스 자동 폐기(tear down) → 제품 환경 보호
- 예시: AI Engineer Conference 2026용 PDF 파일 생성 요청 시, 격리 공간에서 출력 및 다운로드 제공

### 롤링 요약 등 장기 대화 문맥 처리 전략이 대규모 대화 체계의 실제 문제를 해결함

- LLM의 토큰 한계/문맥 과잉 문제 해결을 위해 ‘n회 대화 후 요약’ 등 중간 요약(rolling summarization) 방식 채택
- 최근 대화 n-5~n-10개만 실 대화에 노출, 나머지는 요약본 단위로 리콜(recall) 지원
- 예전 대화 일부만 필요한 경우 ‘메모리’ 컴포넌트가 간략 핵심 요약으로 보조 → 맥락 유실 최소화 및 현실적 문맥 추적

### 런타임 동적 UI 생성 기능으로 사용자 맞춤 인터페이스를 실현함

- 에이전트가 form 등 프리미티브 UI를 런타임에 수집, 개인화된 선택지/예시/응답 방식 제공
- 예) "긴 에세이 예시를 보여줘" 요청 시, 실시간으로 form 생성 및 예시 주제 리스트 선택 지원
- 유연하고 몰입도 높은 사용자 경험 실현

### 자동 트레이싱 및 관측성 시스템이 대규모 에이전트 운영/배포의 핵심 기반이 됨

- Effect 라이브러리의 tracing 기능을 활용, 모든 함수 호출/스팬 등 트레이스 정보 자동 태깅 및 관리
- API, 핸들러, 서비스 간 호출 흐름/프로파일링, 병목 지점, 장애 진단 용이
- 대규모 에이전틱 시스템에서 타팀 API/서비스 연동 상황 실시간 파악 및 디버깅 지원

### ‘툴’과 ‘스킬’ 중심 설계 방식이 비즈니스 도메인 및 개발팀 생산성을 비약적으로 높임

- 예: dad joke 툴 → 툴킷(여러 툴 집합) 생성 → LLM에 등록, 프롬프트로 기능 유연 확장
- Effect의 툴·스킬 시스템이 근간, 각 업무화면·도메인별 맞춤형 기능 신속 제공 가능
- 외부 패키지(effect/ai) 기반 개발 권장 → 개방적 확장성 보장

### 내부 개발에서도 최신 AI 도구(Claude, Cursor, Claude Agents 등)로 업무 속도를 비약적으로 혁신함

- OG Assist·툴·스킬은 고객제품에 도입할 뿐만 아니라, 내부 개발워크플로우에도 적극 활용
- 첨단 AI 에이전트툴(Claude, Cursor 등)로 코드 읽기, 작성, 리뷰, 배포(Ship) 등 개발 전주기 가속
- 조직 전체의 개발 생산성 및 지식 확산 동시 달성
