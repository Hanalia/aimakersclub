## *MCP의 미래 — David Soria Parra, Anthropic* 핵심 요약

- 영상은 MCP(Multi Capability Protocol) 기반 에이전트의 미래 발전 방향과 현재 MCP 생태계 성과, 그리고 향후 도전 과제를 구체적으로 다룸
- **MCP 애플리케이션**은 자체 UI를 제공하며, 클라우드/로컬/다양한 환경에 구동 가능한 서버-기반 새로운 형태의 에이전트임을 강조
- 18개월 만에 MCP 생태계는 **월 1억1천만 다운로드**를 기록, 이는 React가 해당 수치에 도달하는 데 걸린 기간의 절반임
- **주요 MCPS 활용 분야**로 VS Code, WhatsApp, Blender, Linear, Slack, Notion 등 다양한 SaaS 연동 및 실무 자동화가 언급됨
- 현재 MCP 서버의 다수는 외부에 공개되지 않은 기업 내 시스템 연동용으로 사용되고 있음
- 2024~2025년에는 ‘코딩 에이전트’가 주류였으나, 앞으로는 **일반 지식노동(금융, 마케팅 등)**을 지원하는 “연결성 중심”의 에이전트가 확산될 전망
- **연결성을 위한 3대 주요 선택지**: Skills(도메인 지식), MCP, CLI 등 장점별로 병행·조합 필요
- “프로그레시브 툴 디스커버리” 및 “프로그래매틱 툴 콜링” 방법론 등, MCP 기반 에이전트 최적화를 위한 구체적인 개발 패턴 제시
- REST API를 MCP로 단순 치환하는 접근의 한계 지적, 진정한 “에이전트 지향 설계” 필요성 강조
- MCP 프로토콜 및 툴킷의 향후 발전 방향(비상태 전송 프로토콜, 에이전트-에이전트 통신, 크로스앱 인증 등)과 커뮤니티 생태계 협업 방침이 공유됨

## 세부 요약 - 주제별 정리

### MCP 애플리케이션은 자체 UI 및 툴 연동성을 갖춘 차세대 에이전트 인터페이스로 소개됨

- MCP 애플리케이션은 기존의 플러그인, SDK, 클라이언트에서 즉시 렌더링되는 인터페이스와 다르게, “MCP 서버”에서 서비스됨
- 이 MCP 서버는 클라우드, VS Code, Cursor 등 다양한 환경에 쉽게 배치 가능, **“just work”**의 사용성을 추구함
- MCP는 앱뿐 아니라 추가적인 “도구”도 함께 제공하여, 사용자가 직접 또는 모델이 툴을 활용해 상호작용 가능함
- 클라이언트와 서버가 의미적으로(semantics) 상호 이해하며, UI 렌더링 및 복잡한 상호작용이 가능하도록 “프로토콜” 기반 설계를 도입
- 이러한 형태의 에이전트는 기존 에이전트/자동화 툴과 차별화된 “양방향 확장성” 제공

### 18개월 만에 MCP 생태계는 급격히 성장하여 오픈소스 업계 유의미한 이정표를 세움

- 18개월 전 MCP는 단지 작은 “사양 문서(spec doc)”와 일부 SDK(Corely Claude에 의해 개발)만 존재
- 이후 **110만**이 아닌 “월 1억1천만(Monthly 110 million)” 다운로드를 돌파
- React가 같은 다운로드 규모에 도달하는 데 두 배 이상 소요된 것과 비교
- 이 수치는 OpenAI, Google ADK, LangChain 등 다양한 프레임워크와 수천 개의 도구, 애플리케이션이 MCP를 “공통 표준”으로 채택했음을 의미
- 외부에 잘 드러나지 않는 기업 내부, 시스템 연동용 MCP 서버가 대다수인 현실도 언급됨

### 머지않아 “코딩 에이전트”에서 “전방위 업무 지원형 일반 에이전트”로 진화할 전망임

- 2024년까지는 데모 및 기술 시연 중심, 2025년엔 “코딩 에이전트”(로컬·컴파일러 중심·개발자 친화)가 주된 흐름
- 2026년에는 금융분석가, 마케팅 등 전문지식 직군까지 지원하는 **연결성 중심 제너럴 에이전트** 수요가 폭증할 것으로 전망
- 개발자 환경과 달리, 전방위 실무자들은 **다양한 SaaS(5개 이상), 공유 드라이브 등** 여러 시스템과의 ‘연결성’이 필수
- 연결을 위한 “단일 해법”은 존재하지 않으며, 적합한 도구 조합이 필요함을 역설

### 실질적인 “연결성” 확보를 위해서는 Skills, MCP, CLI 등 3가지 방법론을 병행 활용해야 함

- Skills: 단순 도메인 지식 파일화, 여러 플랫폼에서 재사용 가능
- CLI: 로컬 에이전트·코딩 환경에서 강력하며, 환경 탐지(auto-discovery) 및 명령 조합(bash, git 등)에 효과적
- MCP: 리치 세만틱, UI 제공, 장기 작업, 플랫폼 독립성, 승인·거버넌스 등 **엔터프라이즈 기능** 지원
- 각 방식은 용도/환경에 따라 보완적으로 활용할 것을 제안

### MCP 기반 에이전트 개발에서는 “프로그레시브 툴 디스커버리”와 “프로그래매틱 툴 콜링” 기법이 핵심임

- “프로그레시브 디스커버리(점진적 탐색)”는 모든 도구를 컨텍스트 윈도우에 일괄 로딩하여 비대화되는 문제(‘컨텍스트 블로트’)를 해결하는 패턴
- 툴 검색/필요시 로딩 방식(tool search & on-demand loading)을 통해 컨텍스트 자원 감소
- 예시: Anthropic Cloud Code의 도입 전/후 도구 사용량 비교에서 큰 절감 효과 시연
- “프로그래매틱 툴 콜링”(Programmatic Tool Calling, 일명 코드 모드)은, 모델이 도구 호출-결과 처리-다음 호출 등을 일일이 추론(inference)·조정하는 대신, 코드(script)를 직접 작성·실행하여 퍼포먼스와 조합성을 대폭 향상시킴
- 예시로는 모델이 Bash 명령문 또는 MCP 상 API 호출용 코드를 생성해서 복수의 작업을 한 번에 처리하는 시나리오를 제시
- MCP의 “구조화된 출력(structured output)” 기능은 타입 정보를 기반으로, 더 효과적인 조립/필터링 지원
- 구조화 출력이 없는 경우엔 기대 반환 타입을 명시하거나, 저가 모델로 캐스팅 요청하여 수동적으로 조합도 가능

### 단순 REST-to-MCP 변환 방식은 심각한 한계가 있으며, 진정한 “에이전트 지향 API 설계”가 필요함

- 기존 REST API를 1:1로 MCP 서버로 래핑하는 관습이 비효율적이고, “cringe”한 결과만 초래함을 명확히 지적
- 실제 사용 시나리오/사용자 경험(UX) 중심의 API 설계가 선행되어야 하고, 에이전트 관점에서 편리한 실행 환경 제공이 중요함
- 서버 측에서도 단순 툴만 제공하는 것보다 “실행 환경(execution environment)”과 프로그램 조립 지원이 훨씬 강력함
- MCP의 다양한 시멘틱(Tasks, Elicitation 등)과 실험적 기능(MCP Applications, skills over MCP 등) 활용 장려

### MCP 프로토콜은 비상태 전송 등 주요 확장과 업데이트를 앞두고 있음

- 현행 streamable HTTP 방식 한계(대규모 환경에서 확장성 부족)를 보완하기 위해 Google 주도로 “Stateless Transport Protocol”을 6월 내에 도입 예정
- 이 개선안은 MCP 서버를 일반 REST 서버처럼 쉽게 배포(Cloud Run, Kubernetes 등)할 수 있게 함
- 에이전트-에이전트 비동기 통신 지원(Asynchronous task primitive)도 실험적 버전에서 점진적 확장 추진

### 멀티언어 SDK, 서버 스킬 확장, 타사 인증 등 “생태계 통합성” 향상 작업도 진행 중임

- TypeScript 및 Python SDK 2버전(다양한 피드백과 경험에 기반한 리라이팅) 순차 출시 계획
- FastMCP(커뮤니티 주도 더 성능 좋은 MCP SDK)와 기존 Python SDK 비교, 자체 개선의 필요성 인정
- 기업 대상 “크로스앱 액세스(Cross-App Access)”: Google, Okta 등의 인증 정보를 한 번만 연결해, 여러 MCP 서버 연동 시 재인증 필요 없게 함(6월 적용 목표)
- 서버 자동 탐색(Server Discovery): 웹사이트의 well-known URL을 활용, 브라우저·에이전트·크롤러가 MCP 서버 검색·연동 가능케 함
- Extension(확장) 메커니즘: GUI осн 앱(web), CLI 등 클라이언트 특성에 따라 지원 기능 다르게 오픈
- “skills over MCP” 기능: 서버 운영자가 특정 도메인 지식을 Skills로 패키징하여 지속적으로 업데이트/배포 가능케 함(플러그인/레지스트리 방식 대체)

### 오픈 커뮤니티·재단 주도로 협업하며, MCP는 향후 “최고의 연결성 에이전트” 실현의 중심 역할을 담당할 것임

- MCP 재단 설립, 오픈소스 기반 커뮤니티 운영(Discord, 이슈트래커 등)
- 사용자/개발자 피드백을 적극 수렴하여 프로토콜, 도구, 환경 모두 지속 개선 추진
- 2026년엔 연결성(connectivity)이 에이전트 발전의 핵심 키워드로 자리매김하며, 각종 방법론·도구(컴퓨터 사용, CLI, MCP, Skills 등) 결합 운용이 새로운 표준이 될 것으로 전망

### 실제 MCP 애플리케이션 데모를 통해 실전 활용 사례를 시각적으로 시연함

- 클라우드 코드 기준, MCP Application이 어떻게 복잡한 인터페이스·실행 로직을 단일 서버/모델/앱으로 통합해 구현하는지 실시간 데모 시각 제공
- 모델이 동적으로 그래프 렌더링, 대화형 UI 등 실제적인 업무 자동화·지식작업 보조에 활용 가능함을 보여줌
